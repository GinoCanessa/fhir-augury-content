## Resolution Description

> Agreed, we will remove the list of FHIR tools as suggested

## Feature Proposal

Remove the "list of FHIR tools" bullet point and all of its sub-items from the "What You Will Need" section of the CDS Service Specification page. The list contains outdated and broken links (including a note "<b>The HAPI links DO NOT work</b>") and is no longer useful to implementers.

After removal, the "What You Will Need" section will retain its three substantive bullet points covering (1) the CDS rule execution engine web service, (2) FHIR data processing capability, and (3) EHR FHIR server access.

## Detailed Implementation Plan

### Step 1: Remove the FHIR Tools List

**File:** `input/pagecontent/cds-service-spec.md`

Locate the "What You Will Need" section (heading: `#### What You Will Need`). Remove the **entire fourth bullet point** and all of its nested sub-items. The content to remove is:

```markdown
* Here is a list of FHIR tools that might be of interest to implementers:

   * [Public](http://wiki.hl7.org/index.php?title=Publicly_Available_FHIR_Servers_for_testing)

   * Local (preferred)

           <b> The HAPI links DO NOT work </b>
       * [HAPI RESTful Server](http://hapifhir.io/doc_rest_server.html)

           * [HAPI JPA Server](http://hapifhir.io/doc_jpa.html)

           * [HAPI JAX-RS Server](http://hapifhir.io/doc_rest_server_jaxrs.html)

       * [.NET Server](https://github.com/ewoutkramer/fhir-net-api)
```

**Note:** This bullet point appears to occur in **two places** in the file (the section content is duplicated in the rendered output). Verify and remove both occurrences if present.

### Step 2: Verify Remaining Section Structure

After removal, the "What You Will Need" section should contain only these three bullet points:

1. A web service that processes CDS requests using a CDS rule execution engine (SHOULD process CDS Hooks requests and return CDS Hooks Card responses)
2. A web service that processes CDS requests SHOULD be able to process FHIR data including clinical and terminology resources
3. The EHR system MAY provide access to relevant patient data via a FHIR server

The next heading after these bullet points should be `#### CDS Function`.
