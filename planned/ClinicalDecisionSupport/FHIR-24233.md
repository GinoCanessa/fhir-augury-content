## Problem Statement

The ballot commenter questions why the CPG IG introduces an external WHO classification system for health worker personas instead of using existing FHIR constructs (Patient, PractitionerRole, and the practice-codes value set). The concern is that the CPG IG's "persona" concept appears to be an orthogonal approach that does not leverage the standard FHIR resource model for representing actors in care scenarios.

## Resolution Description

> Will work with HTA to ensure we are referencing the external classification correctly.
>
> In addition, we will specify personas using the existing FHIR representation for those as currently specified by Patient, Practitioner, PractitionerRole, and RelatedPerson, and make clear that the personas code system is an example.

**Implementation is required** — resolution is "Persuasive with Modification."

## Feature Proposal

The resolution requires two categories of changes:

1. **Clarify persona representation**: Explicitly state that CPG personas are represented using standard FHIR resources (Patient, Practitioner, PractitionerRole, RelatedPerson) — not as an alternative to them. The persona concept is a use-case modeling abstraction that maps directly to these resources.

2. **Mark persona code system as example**: Make clear that the CPGCommonPersonaCS code system is provided as an _example_ starting point, not a required classification. The ISCO-based codes are illustrative and content implementers may use other coding systems.

3. **ISCO URL coordination**: Ensure the ISCO code system URL is correctly referenced following HTA coordination with the ILO Department of Statistics.

## Detailed Implementation Plan

### Task 1: Update Personas Section in methodology.md

**File:** `input/pagecontent/methodology.md`
**Location:** Lines ~157–165 (Personas subsection under "Select" step)

**Current text (lines 157–165):**
```markdown
#### Personas

Personas are narrative descriptions of the actors and roles, or the _who_, referred to by the guideline. For example, nurse midwife, community healthworker, and pregnant mother are all examples of personas.

To provide a potential starting point for formally identifying personas, this implementation guide provides a [Common Personas](CodeSystem-cpg-common-persona-cs.html) CodeSystem based on the WHO recommendation for [Classifying health workers](https://www.who.int/hrh/statistics/Health_workers_classification.pdf). This recommendation uses codes from the [International Standard Classification for Occupations](http://www.ilo.org/public/english/bureau/stat/isco/index.htm) but defines several additional categories of workers. In addition, the codes in that recommendation are focused on health workers, so codes for patient and care partner personas need to be considered as well.

Where a code from the ISCO exists, it is used. Where a WHO recommended health worker category is used, a code is constructed beginning with a `W`. Where a code is introduced by this implementation guide, it is constructed beginning with a `C`.

Content conforming to this implementation guide MAY identify personas using the [Common Personas](CodeSystem-cpg-common-persona-cs.html) code system.
```

**Proposed changes:**

1. Add a paragraph after the first paragraph explicitly connecting personas to FHIR resources:

   > In FHIR, personas are represented using the existing resource types as appropriate: [Patient](StructureDefinition-cpg-patient.html) for the subject of care, [Practitioner](StructureDefinition-cpg-practitioner.html) and [PractitionerRole](StructureDefinition-cpg-practitionerrole.html) for healthcare workers and their roles, and [RelatedPerson](StructureDefinition-cpg-relatedperson.html) for care partners and other non-professional participants. The persona concept is a use-case modeling abstraction that maps to these standard FHIR resources; it does not replace them.

2. Modify the code system introduction paragraph to frame it as an example:

   > To provide an _example_ starting point for formally identifying personas, this implementation guide provides a [Common Personas](CodeSystem-cpg-common-persona-cs.html) CodeSystem...

3. Update the final conformance sentence to reinforce the example nature:

   > Content conforming to this implementation guide MAY identify personas using the [Common Personas](CodeSystem-cpg-common-persona-cs.html) code system or any other appropriate coding system for the roles being described.

## Risks and Considerations

1. **Partial implementation already exists**: The CPG IG already has profiles for all four resources and uses `preferred` binding strength. The main gap is narrative clarity, not structural conformance. Changes are primarily editorial.

2. **ISCO URL is an external dependency**: The endorsed ISCO URL coordination has been pending since 2020. The remaining changes should proceed independently; the URL can be updated when available.

3. **Code system naming**: Renaming from "Common Personas" to "Example Personas" in the value set may affect downstream implementers who reference the value set by name or description. The formal ID (`cpg-common-persona-vs`) would remain unchanged.

4. **No cross-references found**: No Zulip threads, GitHub issues, or PRs were found linked to this ticket, suggesting it has not been partially addressed through other channels.
