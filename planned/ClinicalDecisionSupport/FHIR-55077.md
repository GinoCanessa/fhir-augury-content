## Description

The RiskAssessment resource references several value sets that are potential candidates for migration to HL7 Terminology (THO). While value set migration is not required, if the value set definition is likely to change in the coming years, or the value sets might be useful in CDA, V2, or other specs, migration should be considered.

**Affected artifacts:**

| Artifact Type | Current ID | Bound Element |
|---------------|-----------|---------------|
| CodeSystem | `risk-assessment-method` | (backing CodeSystem for ValueSet) |
| ValueSet | `risk-assessment-method` | `RiskAssessment.method`, `RiskAssessment.code` |
| CodeSystem | `risk-assessment-outcome` | (backing CodeSystem for ValueSet) |
| ValueSet | `risk-assessment-outcome` | `RiskAssessment.prediction.outcome` |

## Approved Resolution

> Agreed, mark the ValueSets and CodeSystems involved experimental, and add Example to the name, title, id, and url for each.

This means:

1. **Mark experimental** — both CodeSystems and ValueSets already have `<experimental value="true"/>`, so no change needed here.
2. **Add "Example" to id, url, name, and title** — rename each artifact to clearly indicate these are example/illustrative code systems and value sets, not normative terminologies.
3. **Update all internal references** — the StructureDefinition bindings and CodeSystem→ValueSet cross-references must point to the new URLs.
4. **Rename source files** — file names should match the new artifact ids.

---

## Feature Proposal

Rename the four RiskAssessment terminology artifacts (2 CodeSystems + 2 ValueSets) by appending "Example" to their identifiers, and update all internal cross-references within the FHIR Core repository. This makes it clear these are illustrative examples rather than normative terminologies, supporting potential future migration of real terminologies to THO.

### Breaking Change Assessment

- **Binding strength is `example`** for all three binding sites — this means consumers are not required to use these value sets, minimizing breaking impact.
- The CodeSystems and ValueSets already have `<experimental value="true"/>`, so implementations should already treat these as non-stable.
- URL changes will break any implementations that hard-code the old URLs, but since these are example-strength bindings, this is acceptable for an R6 ballot change.
