### Description

Both `cqf-calculatedValue` (`http://hl7.org/fhir/StructureDefinition/cqf-calculatedValue`) and `cqf-expression` (`http://hl7.org/fhir/StructureDefinition/cqf-expression`) allow the use of an `Expression` datatype (typically CQL or FHIRPath) to dynamically calculate a value for an element. Although `cqf-calculatedValue` may have been intended for narrower scenarios (e.g., calculation of fixed constraints or limits), this distinction was **never formally documented** and any contextual guidance has since been removed. As a result, `cqf-expression` has emerged as the more appropriate general-purpose extension and is the one used in the SDC IG.

### Resolution Description

> Agreed, deprecate calculatedValue as suggested:
>
> DEPRECATED: This extension has been deprecated in favor of the more general cqf-expression extension.

---

## Feature Proposal

**Deprecate the `cqf-calculatedValue` extension** by marking it as retired/deprecated in the FHIR Extensions Pack (`HL7/fhir-extensions`), following the established deprecation pattern used throughout the repository. Update associated extensions that reference `cqf-calculatedValue` to reflect the deprecation. The replacement is the existing `cqf-expression` extension, which requires no changes itself.

## Established Deprecation Pattern

The `HL7/fhir-extensions` repo has a well-established deprecation pattern, observed in extensions such as `cqf-relativeDateTime`, `cqf-systemUserType`, `cqf-targetInvariant`, `questionnaire-constraint`, and others. The pattern involves the following changes to the StructureDefinition XML:

1. **`structuredefinition-standards-status` extension**: Change `<valueCode>` from `trial-use` to `deprecated`, and add a nested `structuredefinition-standards-status-reason` extension with a `<valueMarkdown>` explaining the deprecation.
2. **`<status>` element**: Change from `active` to `retired`.
3. **`<description>` element**: Append `DEPRECATED: <reason>` text.
4. **`<short>` in differential**: Prepend `DEPRECATED` to the short description.
5. **`<definition>` in differential**: Append `DEPRECATED: <reason>` text.

### Change 1: Deprecate `cqf-calculatedValue` StructureDefinition

**File:** `input/definitions/Element/StructureDefinition-cqf-calculatedValue.xml`

Apply all five deprecation pattern changes:

#### 1a. Change `structuredefinition-standards-status` (lines 11–13)

**Before:**
```xml
<extension url="http://hl7.org/fhir/StructureDefinition/structuredefinition-standards-status">
  <valueCode value="trial-use"/>
</extension>
```

**After:**
```xml
<extension url="http://hl7.org/fhir/StructureDefinition/structuredefinition-standards-status">
  <valueCode value="deprecated">
    <extension url="http://hl7.org/fhir/StructureDefinition/structuredefinition-standards-status-reason">
      <valueMarkdown value="This extension has been deprecated in favor of the more general cqf-expression extension."/>
    </extension>
  </valueCode>
</extension>
```

#### 1b. Change `<status>` (line 22)

**Before:**
```xml
<status value="active"/>
```

**After:**

### Change 2: Update `cqf-initialValue` Cross-Reference

**File:** `input/definitions/Element/StructureDefinition-cqf-initialValue.xml`

The `<purpose>` element (line 33) contrasts `cqf-initialValue` with `cqf-calculatedValue`. This reference should be updated to note that `cqf-calculatedValue` is now deprecated and `cqf-expression` is the recommended alternative.

**Before:**
```xml
<purpose value="This extension supports providing an expression that determines a calculated value for an element, but one that is only appropriate as the initial value (e.g. the calculated value that should be used when a form is first displayed to capture the information in the resource, as opposed to the cqf-calculatedValue extension, which provides the calculated value for an element whenever any change is made to the values of the resource)."/>
```

**After:**
```xml
<purpose value="This extension supports providing an expression that determines a calculated value for an element, but one that is only appropriate as the initial value (e.g. the calculated value that should be used when a form is first displayed to capture the information in the resource, as opposed to the cqf-expression extension, which provides the calculated value for an element whenever any change is made to the values of the resource)."/>
```

### Change 3: Update `cqf-alternativeExpression` Cross-Reference

**File:** `input/definitions/datatypes/StructureDefinition-cqf-alternativeExpression.xml`

The `<comment>` element (line 58) lists `cqf-calculatedValue` as one of the expression extensions that `cqf-alternativeExpression` can be used with. Update to note the deprecation.

**Before:**
```xml
<comment value="This extension can be used on any Expression, including other Expression extensions such as cqf-expression, cqf-initialValue, and cqf-calculatedValue."/>
```

**After:**
```xml
<comment value="This extension can be used on any Expression, including other Expression extensions such as cqf-expression and cqf-initialValue."/>
```

> **Rationale:** Since `cqf-calculatedValue` is deprecated, it should be removed from the list of recommended usage contexts to avoid guiding implementers toward the deprecated extension.
