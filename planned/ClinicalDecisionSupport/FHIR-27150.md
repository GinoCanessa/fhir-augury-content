## Summary

This ticket requests the definition of two code systems and two corresponding extensions to characterize knowledge artifacts by their **capability** and **representation level**, replacing the original "deliveryAspect" concept. The resolution renames "Distributable" to "Computable" and adds a second axis for knowledge representation.

## Resolution Description

> Add code system for **knowledgeCapability**: Shareable, Computable, Publishable, Executable
>
> Add code system for **knowledgeRepresentationLevel**: Narrative, Semi-Structured, Structured, Executable
>
> Add extension for knowledgeCapability and knowledgeRepresentationLevel

### FHIR Core Integration

The `Shareable` resource type in `HL7/fhir` defines both elements as first-class properties:

- **File:** `HL7/fhir` → `source/shareable/shareable-spreadsheet.xml`
- `Shareable.knowledgeCapability` — 0..*, CodeableConcept, bound to `KnowledgeCapability`
- `Shareable.knowledgeRepresentationLevel` — 0..*, CodeableConcept, bound to `KnowledgeRepresentationLevel`

> **Note:** A comment in the spreadsheet observes: *"Discussion in SDC indicates that the knowledgeCapability modeling here would be better handled with profiles"* — this is addressed by later tickets (e.g., FHIR-42973, FHIR-41367) and does not affect the scope of FHIR-27150.
