## 1  Summary

The reporter (Rob McClure, a terminology expert) identified that the PDDI IG defines many explicitly enumerated drug value sets that are brittle and difficult to maintain. He recommended using SNOMED CT drug classes via NLM RxClass/RxMix APIs and the `ValueSet RulesText` extension to derive RxNorm expansions dynamically, rather than hard-coding individual drug codes.

After discussion between the CDS work group and the reporter on 2025-03-05, the resolution was **Persuasive with Modification**. The accepted modification focuses on three concrete actions (see §2).

---

## 2  Resolution Description (Verbatim)

> 1. Add more language about where to find and how to ensure content such as these value sets is appropriate for use (and maintained).
> 2. Add references to:
>    - FHIR Terminology Module — <https://hl7.org/fhir/terminology-module.html>
>    - FHIR Terminology IG Guidance — <https://build.fhir.org/ig/FHIR/ig-guidance/terminology.html>
> 3. Remove value set definitions from the IG, replacing them with references to the value sets in VSAC. Publisher will likely complain about not being able to resolve these references, but that issue is resolvable by a separate process that adds VSAC value sets referenced in HL7 FHIR publications to the VSAC package.

## 4  Feature Proposal

### 4.1  Goal

Transition the PDDI IG away from locally-maintained, explicitly-enumerated drug value sets to externally-maintained VSAC-hosted value sets, and add educational/guidance content pointing implementers to authoritative terminology resources.

### 4.2  Approach

1. **Enhance the terminology page** with guidance on locating, evaluating, and maintaining drug value sets—including references to the FHIR Terminology Module and IG Guidance.
2. **Remove all drug ValueSet XML definitions** from the IG source.
3. **Replace with VSAC canonical references** so that CQL and other artifacts point to VSAC-hosted value sets instead of IG-local definitions.
4. **Update the IG definition** (`ig.xml`) to remove the locally-registered ValueSet resources.
5. **Update CQL libraries** to reference the new VSAC canonical URLs.
6. **Suppress expected IG publisher warnings** about unresolvable VSAC references (via `ignoreWarnings.txt` or sushi configuration).

## 5  Impact Analysis

### 5.1  Files to Modify

#### 5.1.1  Terminology Page — `input/pagecontent/terminology.md`

**Current state:** Contains a brief disclaimer (added in response to ballot comment) about the informative nature of value sets and the Value Set RulesText extension, followed by the Composite/Element/Element-exclude framework description.

**Required changes:**
- Add a new section at the top (or expand the existing introductory paragraph) with guidance on:
  - Where to find canonical drug value sets (VSAC, NLM RxClass)
  - How to evaluate whether a value set is appropriate for a given PDDI use case
  - How to ensure ongoing maintenance of value set content
  - The distinction between pharmacodynamic DDIs (where drug-class–based value sets are appropriate) and pharmacokinetic DDIs (where they may not be)
- Add explicit references and hyperlinks to:
  - [FHIR Terminology Module](https://hl7.org/fhir/terminology-module.html)
  - [FHIR Terminology IG Guidance](https://build.fhir.org/ig/FHIR/ig-guidance/terminology.html)
- Update the "ValueSet" and "Code Systems" listing sections to note that value sets are now hosted in VSAC

#### 5.1.2  Value Set XML Files — `input/vocabulary/valueset-*.xml` (88 files)

**Current state:** 88 ValueSet XML files containing explicitly enumerated drug codes (RxNorm concepts). These range from ~1 KB (single-drug stubs) to ~177 KB (`valueset-hydrocortisone.xml`).

**Composite ValueSets (6 drug-class composites + 1 code system + 1 SNOMED-based):**

## 6  Detailed Implementation Plan

### Phase 1: Identify VSAC Value Set Mappings

**Objective:** Map each current IG-local drug value set to a corresponding VSAC value set OID.

**Tasks:**
1. Inventory all drug value sets currently in `input/vocabulary/` (see §5.1.2 for complete list)
2. For each composite value set (AAS, NSAIDS, SCS, PPIS, LOOPDIURETIC, CYCS), search VSAC for an existing equivalent or plan to create one
3. For individual drug value sets referenced only via composites, determine if VSAC versions exist or if the composite alone suffices
4. For the SNOMED-based `Hx-UGIB-snomed` value set, search VSAC for an equivalent condition value set
5. Document the mapping: `{IG ValueSet name} → {VSAC OID} / {VSAC canonical URL}`
6. For lab LOINC value sets (digoxin-LOINC, potassium-LOINC, magnesium-LOINC, calcium-LOINC, renal-LOINC), determine if VSAC versions exist or if these should remain IG-local

**Dependencies:** VSAC access, potentially coordination with NLM to create/publish needed value sets.

### Phase 2: Update Terminology Page

**Objective:** Add guidance and references per resolution items 1 and 2.

**Tasks:**
1. Expand the introductory paragraph in `input/pagecontent/terminology.md` to include:
   - Guidance on identifying and selecting appropriate drug value sets for PDDI CDS
   - Information about VSAC as the authoritative source for drug value sets used in HL7 IGs
   - Explanation that value sets in this IG are now maintained externally in VSAC
   - Discussion of drug-class-based vs. individually-enumerated value sets and when each approach is appropriate
   - Reference to the Value Set RulesText extension and NLM RxClass as advanced approaches
2. Add a new subsection with links to:
   - [FHIR Terminology Module](https://hl7.org/fhir/terminology-module.html)
   - [FHIR Terminology IG Guidance](https://build.fhir.org/ig/FHIR/ig-guidance/terminology.html)
3. Update the value set listing section to note VSAC as the hosting location
4. Add a mapping table showing the VSAC canonical URLs for each value set used by the IG

### Phase 3: Remove Local Value Set Definitions

**Objective:** Remove the 80+ drug ValueSet XML files per resolution item 3.

**Tasks:**
1. Delete all drug-related ValueSet XML files from `input/vocabulary/`:
   - All composite value sets: `valueset-AAS.xml`, `valueset-CYCS.xml`, `valueset-LOOPDIURETIC.xml`, `valueset-NSAIDS.xml`, `valueset-PPIS.xml`, `valueset-SCS.xml`
   - All element value sets (individual drugs): `valueset-aspirin.xml`, `valueset-warfarin.xml`, `valueset-ibuprofen.xml`, etc.
   - All element-exclude value sets: `valueset-betamethasone-exclude.xml`, etc.
   - The SNOMED condition value set: `valueset-Hx-UGIB-snomed.xml`
2. Retain `valueset-PDDICDS-indicator.xml` (CDS indicator code system, not a drug value set)
3. Assess and decide on LOINC lab value sets — retain if no VSAC equivalent exists
4. Remove corresponding `<resource>` entries from `input/ig.xml` for all deleted value sets
5. If the IG also uses `input/resources/ig-new.xml` or `input/resources/ig-new.json` for resource registration, update those as well

### Phase 4: Update CQL and Library Artifacts

**Objective:** Point CQL valueset declarations at VSAC canonical URLs.

**Tasks:**
1. In `input/images/artifacts/warfarin-nsaids/warfarin-nsaids-cds-logic.cql`:
   - Replace all `http://hl7.org/fhir/ig/PDDI-CDS/ValueSet/...` URLs with the corresponding VSAC canonical URLs from Phase 1
2. In `input/images/artifacts/digoxin-cyclosporine/digoxin-cyclosporine-cds-logic.cql`:
   - Same URL replacement
3. Re-encode updated CQL as base64 and update the `content[].data` in:
   - `input/images/artifacts/warfarin-nsaids/warfarin-nsaids-cds-library.json`
   - `input/images/artifacts/warfarin-nsaids/warfarin-nsaids-cds-library-sign.json`
   - `input/images/artifacts/digoxin-cyclosporine/digoxin-cyclosporine-cds-library.json`
   - `input/images/artifacts/digoxin-cyclosporine/digoxin-cyclosporine-cds-library-sign.json`
4. Also update the `pddi-cds-common-library.json` if it contains value set references

### Phase 5: Handle IG Publisher Warnings

**Objective:** Ensure the IG builds cleanly despite unresolvable VSAC references.

**Tasks:**

3. Test the IG build with `_genonce.bat` to verify:
   - Build completes successfully
   - Only expected warnings about VSAC resolution appear
   - No broken page links or missing resource references

### Phase 6: Review Examples and Documentation

**Objective:** Ensure consistency across all IG content.

**Tasks:**
1. Review example resources in `input/examples/` for any references to deleted value set files
2. Review `input/pagecontent/knowledge-artifacts.md` — update CQL examples that show value set declarations to use VSAC URLs
3. Review `input/pagecontent/documentation.md` for any inline references to value set filenames or IG-local URLs

## 8  Risks & Considerations

1. **VSAC Value Set Availability:** Not all IG-local value sets may have existing VSAC equivalents. New VSAC value sets may need to be created and published, which requires coordination with NLM.

2. **IG Publisher Resolution:** The IG publisher cannot natively resolve VSAC value set references. The resolution acknowledges this and notes it is "resolvable by a separate process that adds VSAC value sets referenced in HL7 FHIR publications to the VSAC package." Until that process is in place, build warnings must be suppressed.

3. **CQL Execution Impact:** CQL engines that execute these libraries at runtime will need access to a terminology server that can resolve VSAC value sets. This is a deployment consideration for implementers.

4. **Pharmacokinetic DDIs:** As noted in the ballot comment discussion, drug-class–based value sets are not always appropriate for pharmacokinetic drug interactions (e.g., digoxin-cyclosporine). The terminology page guidance should acknowledge this nuance.

5. **Breaking Change for Existing Implementers:** Implementations that reference the current IG-local ValueSet URLs will need to update their CQL and configuration to use the new VSAC URLs. The terminology page should include migration guidance.

6. **Lab LOINC Value Sets:** These may be candidates for retention as IG-local definitions if they are small, stable, and not yet available in VSAC.
