## Problem Description

The reporter noted that the PDDI CDS IG lists two conformance resource profiles on a "Profiles" page, but they are only available as downloadable XML files (placed in the `input/images/requests/` folder). They are not rendered as proper FHIR IG profile pages using the standard IG Publisher representation, nor are they listed in the artifact index. The profiles are also not tracked as Jira-related artifacts.

## Resolution Description

> "Agreed, these profile definitions are not part of the actual specification. We will remove the Profiles page (this IG doesn't actually define any profiles) and remove the structureDefinition files from the 'request' folder so they do not appear in the published output."

**Disposition**: Persuasive — **implementation is required.**

## Feature Proposal

Remove the Profiles page and its associated StructureDefinition XML files from the PDDI CDS IG. The IG does not actually define any profiles; the StructureDefinition files were incorrectly included as static downloadable XML content rather than proper conformance resources. They should be deleted entirely, and all references to the Profiles page should be removed from the IG definition.

## Impact Analysis

### Scope

This is a **low-risk removal** of non-functional content. The IG currently does not define any valid profiles — the StructureDefinition files are static XML placed in the images/requests directory (not processed by the IG Publisher as conformance resources). Removing them will not affect any IG functionality, break any resource references, or alter the normative content.

### Files Affected

| # | File Path | Action | Reason |
|---|-----------|--------|--------|
| 1 | `input/pagecontent/profiles.md` | **DELETE** | Profiles page content; links to non-existent profile renderings |
| 2 | `input/images/requests/structuredefinition-PDDICDS-valueset.xml` | **DELETE** | StructureDefinition XML file for "PDDI ValueSet" profile (not a real IG artifact) |
| 3 | `input/images/requests/structuredefinition-PDDICDS-requestgroup-action-indicator.xml` | **DELETE** | StructureDefinition XML file for "PDDI RequestGroup" profile (not a real IG artifact) |
| 4 | `input/images/requests/structuredefinition-PDDICDS-detectedissue-potentiation.xml` | **DELETE** | StructureDefinition XML file for "PDDI DetectedIssue Potentiation" (also in requests folder, should be removed) |
| 5 | `input/ig.xml` | **EDIT** | Remove the `<page>` element for `profiles.html` from the page hierarchy |

### Files NOT Affected

- `input/resources/codesystem-PDDICDS-indicator.xml` — Real conformance resource, stays
- `input/vocabulary/` — ValueSet definitions, unrelated
- `input/examples/` — Example resources, unrelated
- Other pagecontent files (`index.md`, `cds-service-spec.md`, etc.) — No references to the profiles page found in these files
- `FHIR-pddi.xml` (specification registry) — The two StructureDefinition artifacts are already marked `deprecated="true"` in this file; no change needed

## Detailed Implementation Plan

### Step 1: Delete the Profiles page content

**File**: `input/pagecontent/profiles.md`

**Action**: Delete the entire file.

**Current content**:
```markdown

The following profiles set the minimum expectations:

* [PDDI ValueSet](requests/structuredefinition-PDDICDS-valueset.xml)

* [PDDI RequestGroup](requests/structuredefinition-PDDICDS-requestgroup-action-indicator.xml)
```

This page contains only links to downloadable XML files and no rendered profile content.

### Step 2: Delete StructureDefinition XML files from the requests folder

**Files to delete** (all in `input/images/requests/`):

1. `structuredefinition-PDDICDS-valueset.xml` (7,317 bytes)
2. `structuredefinition-PDDICDS-requestgroup-action-indicator.xml` (6,047 bytes)
3. `structuredefinition-PDDICDS-detectedissue-potentiation.xml` (7,722 bytes)

> **Note**: The third file (`detectedissue-potentiation`) is not referenced from `profiles.md` but is also a StructureDefinition in the same `requests/` folder and should be removed per the resolution's instruction to "remove the structureDefinition files from the 'request' folder."

The other JSON files in `input/images/requests/` (CDS Hook request/response examples like `digoxin-cyclosporine-order-select-coordination.json`, `warfarin-nsaids-order-sign-coordination.json`, etc.) are **not** StructureDefinition files and should be **retained**. These are CDS service request examples used in the IG documentation.

### Step 3: Remove the Profiles page from the IG definition

**File**: `input/ig.xml`

**Action**: Remove the following `<page>` element from the `<page>` hierarchy (currently between the `downloads.html` and `testdata.html` pages):

```xml
<!-- REMOVE THIS BLOCK -->
<page>
    <nameUrl value="profiles.html"/>
    <title value="Profiles defined as part of the PDDI CDS Implementation Guide"/>
    <generation value="markdown"/>
</page>
```

The surrounding context in `input/ig.xml`:
```xml
            <page>
                <nameUrl value="downloads.html"/>
                <title value="PDDI CDS – Downloads"/>
                <generation value="markdown"/>
            </page>
            <!-- DELETE THE profiles.html page block here -->
            <page>
                <nameUrl value="testdata.html"/>
                <title value="Test data defined as part of the PDDI CDS Implementation Guide"/>
                <generation value="markdown"/>
            </page>
```

### Step 4: Verify no other references

Verify that no other page content files reference the profiles page or the deleted StructureDefinition files. Based on analysis:

- `input/pagecontent/index.md` — Should be checked for any navigation links to the profiles page
- `input/pagecontent/downloads.md` — Should be checked for any download links to the StructureDefinition files
- `input/includes/` — Should be checked for any menu/navigation includes referencing profiles
