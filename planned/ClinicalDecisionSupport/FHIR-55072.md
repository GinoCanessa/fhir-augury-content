## Description

The RequestOrchestration resource references three CodeSystem/ValueSet pairs that are potential candidates for future migration to the HL7 Terminology (THO / UTG). The resolution directs that rather than migrating now, each ValueSet and CodeSystem should be:

1. Marked as **experimental** (`experimental = true`)
2. Renamed to include **"Example"** in the `id`, `url`, `name`, and `title`

This signals that these terminologies are illustrative, not normative, and may be replaced or migrated in the future.

## Resolution Details

> Agreed, mark the ValueSets and CodeSystems involved experimental, and add Example to the name, title, id, and url for each.

Affected artifacts:
- [valueset-action-code](https://build.fhir.org/valueset-action-code.html)
- [valueset-action-reason-code](https://build.fhir.org/valueset-action-reason-code.html)
- [valueset-action-participant-function](https://build.fhir.org/valueset-action-participant-function.html)

## Feature Proposal

Rename and mark as experimental the three CodeSystem/ValueSet pairs used by `RequestOrchestration` (and shared with `PlanDefinition` and `ActivityDefinition`). Update all element bindings in consuming StructureDefinitions to reference the new URLs. Rename the source files to match the new ids.

## Naming Convention Reference

The "Example" naming pattern follows established precedent in the FHIR spec. Existing examples include:

| File | id | name | title |
|---|---|---|---|
| `valueset-object-role-example.xml` | `object-role-example` | `AuditEventEntityRoleExample` | `Example Audit Event Entity Role` |
| `codesystem-audit-entity-detail-type-example.xml` | `audit-entity-detail-type-example` | `AuditEntityDetailTypeExample` | `Example Codes for AuditEvent.entity.detail.type` |

Pattern: `id` gets `-example` suffix, `name` gets `Example` suffix, `title` gets `Example` prefix.
