## Feature Proposal

### Objective

Replace all occurrences of the word **"minimum"** (when used to describe the PDDI information model) with **"minimal"** across the PDDI CDS Implementation Guide. The concept should be consistently referred to as the **"minimal information model"** — not the "minimum information model" — throughout all IG pages.

### Background

The PDDI CDS IG builds on a W3C Community Group project that defines a set of core information items for representing clinically actionable PDDI knowledge. The IG body text already *mostly* uses the correct term "minimal information model," but several references and link texts still use "Minimum" — specifically in the titles referencing the W3C Community Group Note titled "A Minimum Representation of Potential Drug-Drug Interaction Knowledge and Evidence." Since this ticket's resolution is **Persuasive**, the IG text (including descriptive link text) should be updated to use "Minimal" consistently, even though the underlying W3C document's actual title uses "Minimum."

### Scope

- **In scope:** Changing IG-authored descriptive text that uses "Minimum" when referring to the PDDI information model concept.
- **Out of scope:** The `profiles.md` use of "minimum expectations" (standard FHIR IG phrasing, not related to the information model). The filename `MIM.md` is acceptable as-is since the heading inside already reads "Minimal Information Model."
