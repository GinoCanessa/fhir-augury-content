## Disposition

**Persuasive** — requires implementation. The work group agreed with the submitter's request and specified a concrete editorial change.

---

## Feature Proposal

### What

Add a paragraph to the PDDI-CDS Implementation Guide that references the [EBM-on-FHIR IG](http://hl7.org/fhir/uv/ebm) (`hl7.fhir.uv.ebm`), explaining that the Evidence-Based Medicine on FHIR resources (in particular the `Evidence`, `EvidenceVariable`, and `Citation` resources) may be useful for providing structured, computable references to evidence that underlies PDDI recommendations.

### Why

The PDDI-CDS IG's minimal information model already includes an "Evidence for a Suspected Drug-Drug Interaction" element in its knowledge artifacts (visible in both the Warfarin + NSAIDs and Digoxin + Cyclosporine use cases). However, evidence citations are currently expressed only as free-text narrative within the MIM tables and CDS Hooks Card content. The EBM-on-FHIR IG defines FHIR profiles and value sets specifically designed to represent scientific evidence in a structured, machine-readable way. Adding a reference creates a path for implementers to adopt computable evidence representation alongside narrative text.

### How (High Level)

Add a new paragraph to the **Related Projects** section of the PDDI-CDS IG's main page (`index.md`), referencing the EBM-on-FHIR IG and explaining its potential applicability to structured evidence in PDDI recommendations.

## Detailed Implementation Plan

### Step 1: Add EBM-on-FHIR Reference Paragraph to `index.md`

**File:** `input/pagecontent/index.md`

**Location:** The "Related Projects" section (currently under the `#### Related Projects` heading, approximately after the CDS Connect paragraph and before the `### Scope` heading).

**Current content at insertion point:**

```markdown
#### Related Projects

The [Use Case](./use-cases.html) section of this implementation guide describes a representative
clinical PDDI example that is used for illustration throughout this implementation guide. ...

The example CDS services shown in this implementation guide were written without the use of a
specific authoring tool. The [Clinical Decision Support Authoring Tool](...) is a component of
the CDS Connect project funded by ... However, the authoring tool was not used to assist with
PDDI CDS that uses CDS Hooks as specified in this implementation guide.

### Scope
```

**Action:** Insert a new paragraph **after** the existing CDS Connect paragraph and **before** `### Scope`. The paragraph should:

1. Introduce the EBM-on-FHIR IG by name and link
2. Briefly describe what the IG provides (Evidence, EvidenceVariable, Citation resources)
3. Explain how it could be used in the PDDI context (structured references to evidence supporting PDDI recommendations)

**Proposed text:**

```markdown
The [Evidence Based Medicine on FHIR (EBMonFHIR) Implementation Guide](http://hl7.org/fhir/uv/ebm)
defines FHIR profiles and value sets for representing scientific knowledge, including the Evidence,
EvidenceVariable, and Citation resources. This IG may be useful to implementers seeking to provide
structured, computable references to the evidence that supports PDDI recommendations. For example,
the "Evidence for a Suspected Drug-Drug Interaction" elements described in this implementation
guide's minimal information model are currently expressed as narrative text. The EBMonFHIR IG
provides a standardized approach for representing such evidence in a machine-readable form that
could complement the narrative evidence descriptions used in PDDI CDS artifacts.
```

### Verification

1. **Build the IG** locally using `_genonce.bat` to confirm the new paragraph renders correctly on the index page.
2. **Review** that the hyperlink to the EBM-on-FHIR IG resolves correctly.
3. **Confirm** no unintended changes to other sections of the page.
