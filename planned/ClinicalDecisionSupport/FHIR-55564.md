## Problem Statement

Several elements across CDS-owned resources remain marked as `deprecated` (via `structuredefinition-standards-status` extension) in the R6 normative ballot. Since R6 is a new major version intended for normativity, retaining deprecated elements forces implementers to model and consider elements explicitly flagged as "should not be used." Elements that were deprecated in R5 meet the criteria for withdrawal in R6 per [FHIR versioning policy](http://hl7.org/fhir/versions.html#deprecation).

The ticket identifies these CDS-scoped deprecated elements:

| Element | Deprecation Status | Type |
|---------|--------------------|------|
| `ActivityDefinition.topic` | Deprecated since R5 | `0..* CodeableConcept` |
| `EventDefinition.topic` | Deprecated since R5 | `0..* CodeableConcept` |
| `Library.topic` | Deprecated since R5 | `0..* CodeableConcept` |
| `PlanDefinition.topic` | Deprecated since R5 | `0..* CodeableConcept` |
| `Measure.topic` | Deprecated since R5 | `0..* CodeableConcept` |
| `Measure.guidance` | Deprecated (in favor of `Measure.usage`) | `0..1 markdown` |
| `Measure.subject[x]` | Deprecated (in favor of `Measure.group.subject[x]`) | `0..1 CodeableConcept\|Reference(Group)` |

## Approved Resolution

From the ticket resolution:

> Agreed, remove the topic element from ActivityDefinition, EventDefinition, PlanDefinition, and Library since they were deprecated in R5.
>
> Ensure that the R4 conversions to R5 and R6 handle representing topic using a useContext with a code of topic.
>
> Note that due to an issue with structure definition rendering in the ballot4 content, these elements did not display as "deprecated", but they were marked as deprecated in the StructureDefinitions.

**Scope note:** The resolution explicitly names `ActivityDefinition.topic`, `EventDefinition.topic`, `PlanDefinition.topic`, and `Library.topic` for removal. The status of `Measure.topic`, `Measure.guidance`, and `Measure.subject[x]` should be confirmed with the CDS work group — they are listed in the ticket body but not named in the resolution text. The analysis below covers all seven elements for completeness.

## Feature Proposal

### What Changes

Remove the deprecated `topic` element from `ActivityDefinition`, `EventDefinition`, `PlanDefinition`, and `Library` StructureDefinitions. Optionally also remove `Measure.topic`, `Measure.guidance`, and `Measure.subject[x]` (pending CDS WG confirmation). Update all examples to convert `topic` data to equivalent `useContext` entries. Update or remove associated search parameters. Update version conversion maps so that R4→R5 and R4→R6 transformations correctly represent `topic` as `useContext` with a code of `topic`.

### Why

R6 is intended as a normative release. Carrying forward deprecated elements contradicts the deprecation lifecycle policy and adds unnecessary burden on implementers who must support elements flagged as "do not use."

### Migration Path

The `topic` element's data maps directly to `useContext` entries using:
- `useContext.code` = `http://terminology.hl7.org/CodeSystem/usage-context-type#topic`
- `useContext.value[x]` = the `CodeableConcept` formerly in `topic`

For `Measure.guidance`, content moves to `Measure.usage`. For `Measure.subject[x]`, the per-group `Measure.group.subject[x]` is the replacement.

### 8. Version Conversion Maps

The resolution explicitly requires:

> Ensure that the R4 conversions to R5 and R6 handle representing topic using a useContext with a code of topic.

Version conversion maps are maintained in the **FHIR publisher tooling** (Java-based, `org.hl7.fhir.core` repository) rather than in the `HL7/fhir` specification source. The conversion logic will need to be updated to:

1. **R4 → R5/R6:** Map `Resource.topic[]` to `Resource.useContext[]` entries where:
   - `useContext.code.system` = `http://terminology.hl7.org/CodeSystem/usage-context-type`
   - `useContext.code.code` = `topic`
   - `useContext.valueCodeableConcept` = the original `CodeableConcept` from `topic`

2. **R5/R6 → R4:** Map `Resource.useContext[]` entries with code `topic` back to `Resource.topic[]`.

This affects conversions for: `ActivityDefinition`, `EventDefinition`, `Library`, `PlanDefinition` (and `Measure` if its `topic` is also removed).
