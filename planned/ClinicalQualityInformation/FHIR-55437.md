## Resolution Summary

The resolution is **Persuasive** with the description: *"Remove Measure.guidance as proposed."*

The ticket is a child of [FHIR-53722](https://jira.hl7.org/browse/FHIR-53722), which set policy for handling deprecated elements in R6. `Measure.guidance` is a `0..1 markdown` element that has been flagged as deprecated (`structuredefinition-standards-status = deprecated`, with `comment="NOTE: This element is deprecated in favor of the usage element"`) and qualifies for outright removal under the parent policy. The implementation must delete the element from the Measure StructureDefinition, refresh the resource diagram, and record the change in the resource introduction. Surrounding artifacts (examples that still carry a `<guidance>` value) need to be reconciled because the element will no longer exist after this change.

## Feature Proposal

### Problem Statement

`Measure.guidance` (`0..1 markdown`, summary, deprecated) duplicates information that authors are now expected to convey via `Measure.usage`. It has been carrying a `deprecated` standards-status flag and a comment instructing implementers to use `Measure.usage` instead. Leaving the element in R6 perpetuates two ways to express the same information and contradicts the deprecation already announced in earlier releases. The parent ticket FHIR-53722 established that elements meeting the deprecation criteria should be removed in R6; this child ticket applies that policy to `Measure.guidance`.

### Proposed Change

Delete `Measure.guidance` from the Measure resource and update the artifacts that reference it:

- **Remove the `<element id="Measure.guidance">` block** from `source/measure/structuredefinition-Measure.xml` (the `<differential>` element definition that includes `path="Measure.guidance"`, `min=0`, `max=1`, `type code="markdown"`, `isSummary=true`, the `structuredefinition-standards-status = deprecated` extension, and the deprecation comment "NOTE: This element is deprecated in favor of the usage element").
- **Refresh `source/measure/measure.svg`** so the class diagram no longer renders a `guidance : markdown [0..1]` row. Remove the entire `<text … class="diagram-class-detail">…guidance : markdown [0..1]</text>` element rather than only its inner content (the partially-stripped row currently leaves an orphan `<text … class="diagram-class-detail"> [0..1]</text>` at `y="574.0"` in the diagram).
- **Record the change** in `source/measure/measure-introduction.xml` under the R6 change list section. The existing partial commit added an entry: `FHIR-55437: Remove deprecated Measure.guidance` — keep this and ensure it sits alongside the FHIR-55436 ("Remove deprecated Measure.topic") entry rather than in a separate `<ul>` (currently `measure-introduction.xml` opens a new `<ul>` for FHIR-55437 instead of appending to the same list as FHIR-55436).
- **Reconcile examples** that still serialise a `<guidance value="…">` field on a Measure instance. After removal the element will no longer exist, so any `<guidance>` child of a `<Measure>` resource will fail validation:
  - `source/measure/measure-cms146-example.xml` (line 100, `<guidance value="This is an episode of care measure …"/>`)
  - `source/measure/measure-exclusive-breastfeeding.xml` (line 428, `<guidance value="A discharge to a designated cancer center …"/>`)
  - `source/measure/measure-predecessor-example.xml` (line 427, `<guidance value="A discharge to a designated cancer center …"/>`)

  For each, either delete the `<guidance>` element or move its content into `Measure.usage` (markdown, also `0..1`). Moving the prose into `usage` is consistent with the deprecation comment that pointed at `usage` as the replacement.

- **No change** to `Measure.usage` (line 587, `alias="guidance"`, `comment="This metadata element was typically referred to as Guidance in HQMF"`) is required by the resolution; the alias and HQMF comment legitimately reflect history and do not need to be touched.

The unrelated `Measure.group.improvementNotationGuidance` element (line 1358 of `structuredefinition-Measure.xml`) is **not** in scope.

### Design Rationale

The parent ticket FHIR-53722 documents the deprecation-handling policy used here:

- If an element has carried the `deprecated` flag since R4, **remove it**.
- If the element only has a comment indicating deprecation, but additionally has a trial-use flag, lived on a trial-use resource in R4, or was added in R5+, **remove it**.
- Otherwise, add a formal `deprecated` flag.

`Measure.guidance` already has the formal `structuredefinition-standards-status = deprecated` extension on its definition and a comment naming `Measure.usage` as the replacement. It therefore satisfies the first criterion and qualifies for removal rather than re-flagging. Choosing to delete (rather than further deprecate) keeps R6 free of two parallel ways to express usage guidance, matches the sibling change for `Measure.topic` (FHIR-55436), and removes a long-standing source of confusion for measure authors.

### Breaking Changes

This is a **breaking change** to the Measure resource:

- Removal of an existing element. Any R5 instance that populated `Measure.guidance` will not validate against R6, and any client/server code that reads or writes `Measure.guidance` will need to migrate to `Measure.usage`.
- The deletion is the explicit intent of the resolution and is consistent with the parent policy in FHIR-53722, but it must be called out in the R6 change notes for Measure.

There are no terminology binding, type, or cardinality changes to other elements; `Measure.usage` is unchanged.
