## Resolution Summary

The FamilyMemberHistory example in QI Core (titled "Father Died of Heart Attack") contains two inconsistencies:

1. **Outcome uses an unrelated SNOMED code:** The `condition.outcome` element currently uses SNOMED CT code `520004` ("Congenital bent nose"), which has no relation to a fatal myocardial infarction scenario. It must be changed to SNOMED CT code `419099009` ("Dead (finding)").

2. **Note is clinically irrelevant:** The `condition.note` currently reads *"Was fishing at the time. At least he went doing something he loved."* This must be replaced with *"Father suffered a myocardial infarction and subsequently died from complications."*

These changes must be applied across all representations: XML source, JSON test files, and generated narrative content (HTML `<div>` blocks).

## Feature Proposal

### Problem Statement

The QICoreFamilyMemberHistory example instance (id: `example`) demonstrates a scenario where a father died of a heart attack. However, the `condition.outcome` element illogically references "Congenital bent nose" (SNOMED `520004`), a condition entirely unrelated to cardiac death. The `condition.outcome` element is bound to the **ConditionOutcomeCodes** value set (example binding), which includes codes like "Dead (finding)" (`419099009`), "Patient condition finding" (`370996005`), and disability codes. Additionally, the note provides anecdotal rather than clinical content.

These inconsistencies undermine the example's educational value and clinical credibility.

### Proposed Change

In the QI Core FamilyMemberHistory example (id: `example`), make two data corrections:

1. **Replace `condition.outcome`:**
   - **From:** `system: http://snomed.info/sct`, `code: 520004`, `display: Congenital bent nose`
   - **To:** `system: http://snomed.info/sct`, `code: 419099009`, `display: Dead (finding)`

2. **Replace `condition.note.text`:**
   - **From:** `"Was fishing at the time. At least he went doing something he loved."`
   - **To:** `"Father suffered a myocardial infarction and subsequently died from complications."`

3. **Regenerate narrative `<div>` content** in all JSON representations to reflect the updated outcome and note values.

### Design Rationale

- SNOMED CT `419099009` ("Dead (finding)") is a member of the **ConditionOutcomeCodes** value set (`http://hl7.org/fhir/ValueSet/condition-outcome`) and directly aligns with the scenario — the father died from the condition.
- The example already has `contributedToDeath` semantics (via the severity extension coded as "Fatal" with SNOMED `399166001`), so using "Dead (finding)" as outcome is fully consistent.
- The revised note text provides clinically meaningful context that matches the coded data (myocardial infarction → death from complications).
