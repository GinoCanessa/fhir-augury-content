## Feature Proposal

### Problem Statement

The Quality Reporting narrative page (`clinicalreasoning-quality-reporting.html`) under FHIR Clinical Reasoning contained the phrase "**To do** this using criteria-based stratifiers …" in the discussion that follows the criteria-based-stratification CQL example (around the `criteria-based-stratifiers` anchor). Lloyd McKenzie's bulk pass on incomplete-content indicators (Comment 45) flagged this as a "To do" marker that implies missing/incomplete content. Even though the words were used in their natural-English sense ("In order to do …"), the appearance of the literal token "To do" in a specification page is exactly what the comment-45 cleanup is meant to scrub. The work group agreed to reword rather than rely on readers to disambiguate.

This is a pure documentation / narrative change. There is no resource model, terminology, or schema change.

### Proposed Change

In `source/clinicalreasoning-quality-reporting.html`, in the paragraph that immediately follows the "Stratifier Male" / "Stratifier Female" CQL example (the paragraph beginning the discussion of why combining age-range + gender via criteria-based stratifiers does not scale), replace the leading clause as follows:

- **Before** (the original wording the ticket calls out):

  > To do this using criteria-based stratifiers would require specifying a stratifier expression for each possible combination of age range group and gender. With even larger stratifications, this is untenable.

- **After** (per the resolution):

  > Doing this using criteria-based stratifiers is not recommended because it would require specifying a stratifier expression for each possible combination of age range group and gender. With even larger stratifications, this is untenable.

The paragraph remains a single `<p>...</p>` block, immediately after the `</code></pre>` of the `define "Stratifier Male"` / `define "Stratifier Female"` example, and immediately before the next `<pre><code>` block introducing the value-based "Age Range Stratifier" example. No other prose, no anchors, no section headings, and no examples on the page change.

The current clone at HEAD already shows the post-change wording at line 437:

```html
<p>Doing this using criteria-based stratifiers is not recommended because it would require specifying a stratifier expression for each possible combination of age range group and gender. With even larger stratifications, this is untenable. Value-based stratifiers simplify this, so you could say:</p>
```

(Note that the trailing fragment "Value-based stratifiers simplify this, so you could say:" was already part of the same paragraph and is preserved in the applied commit.)

### Design Rationale

- The resolution explicitly acknowledges the change is editorial ("Even though it was not necessary, we agree to change the language"). The minimal-touch rewrite avoids any normative or semantic shift in the Quality Reporting guidance.
- Replacing only the lead-in clause keeps the rest of the sentence (and the surrounding example flow: criteria-based example → critique → value-based example) intact, so the page's pedagogical structure is unchanged.
- The new wording strengthens the message: it now makes explicit that criteria-based stratifiers across multiple dimensions are *not recommended*, which is consistent with the value-based-stratifier example that immediately follows. This aligns with the Clinical Reasoning module's general guidance to prefer value-based stratifiers for multi-dimensional stratification.
- Alternatives considered (implicit from the resolution): leaving the text as-is and instead adding an implementation note (the other option offered by the FHIR-55079-style "incomplete content" cleanup pattern). The work group chose the reword path because it removes the trigger token entirely and produces clearer guidance in the same edit.
