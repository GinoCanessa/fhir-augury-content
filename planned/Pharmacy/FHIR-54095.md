## Resolution Summary

Resolution: **Persuasive**.

The ticket is a Technical Correction that reports a typographical error in the
MedicationRequest specification prose. The phrase `"Device Request"` (two
words) must be corrected to `"DeviceRequest"` (one word, the actual FHIR
resource name). The sentence in question also appears to be the one the
reporter quoted verbatim:

> "This resource would not be used when ordering a device(s) that may have a
> medication coating e.g. heparin coated stints, or similar types of devices.
> These types of devices would be ordered using the **Device Request** or the
> SupplyRequest resources."

The approved fix is a pure prose/spelling correction — replace
`Device Request` with `DeviceRequest` in that sentence. No structural, schema,
terminology or behavioural changes are required.

## Feature Proposal

### Problem Statement

The MedicationRequest Scope and Usage introductory text refers the reader to
two sibling resources (DeviceRequest and SupplyRequest) for the case of
ordering medication-coated devices (e.g., heparin-coated stents). The
reference to DeviceRequest is written as two words — `Device Request` —
which:

- Does not match the FHIR resource name (`DeviceRequest`), and
- Is inconsistent with how the same resource is rendered later in the same
  file (`medicationrequest-introduction.xml` line 24 correctly writes
  `<a href="devicerequest.html">DeviceRequest</a>`).

This makes the incorrect reference ambiguous to readers and breaks the
convention used elsewhere on the same page.

### Proposed Change

Edit the single occurrence in
`source/medicationrequest/medicationrequest-introduction.xml` at line 8:

Current text:

```html
<p>This resource would not be used when ordering a device(s) that may have a
medication coating e.g. heparin coated stints, or similar types of devices.
These types of devices would be ordered using the Device Request or the
SupplyRequest resources.</p>
```

Replace `Device Request` with `DeviceRequest`. To keep the fix consistent with
the Boundaries and Relationships paragraph on line 24 of the same file, also
turn the corrected token into a hyperlink to `devicerequest.html` (the same
link target already used on line 24). Proposed replacement:

```html
<p>This resource would not be used when ordering a device(s) that may have a
medication coating e.g. heparin coated stints, or similar types of devices.
These types of devices would be ordered using the
<a href="devicerequest.html">DeviceRequest</a> or the
<a href="https://build.fhir.org/ig/HL7/oo-incubator/StructureDefinition-SupplyRequest.html">SupplyRequest</a>
resources.</p>
```

If the applying committer prefers the minimal change (text-only, no added
hyperlinks), the minimum-viable fix is simply
`Device Request` → `DeviceRequest` on line 8. Either satisfies the
resolution.

### Design Rationale

- The resolution is marked *Persuasive* for a one-word spelling
  correction; there is no scope to alter cardinality, bindings, or resource
  relationships.
- Adding the hyperlink is a low-cost, high-value alignment with the sibling
  paragraph on line 24 of the same file which already links to
  `devicerequest.html` and the oo-incubator SupplyRequest build page. It is
  offered as a suggested enhancement but is not required by the resolution.
- No alternative was considered (spelling a FHIR resource name in two words
  is simply wrong).
