## Resolution Summary

Resolution: **Persuasive**.

Per the resolution description, a new MedicationDispense example will be added that demonstrates how a partial fill is represented. The example must include at least the following elements:

- `status`
- `type` (the dispense-type code that flags this as a partial fill)
- `fillNumber`
- `authorizingPrescription` (referencing the prescription at <https://build.fhir.org/medicationrequest0301.html> — i.e., `MedicationRequest/medrx0301`)
- `medication`
- `daysSupply`

The editor may add other elements they consider pertinent (e.g., `subject`, `performer`, `quantity`, `whenHandedOver`, `dosageInstruction`).

## Feature Proposal

### Problem Statement

MedicationDispense supports the concept of partial fills via the `type` element (which is bound — example strength — to `http://terminology.hl7.org/ValueSet/v3-ActPharmacySupplyType`, whose `Partial Fill` code is `PF`) and via the `fillNumber` element (a `positiveInt` whose definition explicitly states that partial fills increment fillNumber by 1, not by a fractional value). However, the existing example set in `source/medicationdispense/` (35 instances, `medicationdispenseexample8.xml` and `medicationdispense0301.xml` … `medicationdispense0335.xml`) does **not** contain an example that combines these elements to demonstrate the partial-fill pattern. Implementers reading the specification therefore have no canonical reference for how a partial fill should look on the wire. The Pharmacy WG agreed to close this gap by adding a worked example.

### Proposed Change

Add a new MedicationDispense example instance (proposed id `meddisp0336`, file `medicationdispense0336.xml`) that represents a **partial fill** dispense for the prescription already used as the canonical IV-antibiotic prescription example (`MedicationRequest/medrx0301`, "Vancomycin Hydrochloride", per `medicationrequest0301.xml`). The instance MUST contain at least the elements called out in the resolution.

Concrete element values for the new instance (a strawman that satisfies the resolution; the editor may refine wording, codes, dates, and quantities):

- `id` = `meddisp0336`
- `status` = `completed`
- `medication.concept` (or `medication.reference` to a contained `Medication`) — Vancomycin Hydrochloride (e.g., NDC `0409-6531-02`, matching the `med0301` contained Medication used in `medicationdispense0301.xml`)
- `subject` → `Patient/pat1` ("Donald Duck") — consistent with the existing dispense examples for this prescription
- `performer.actor` → `Practitioner/f006`
- `authorizingPrescription` → `MedicationRequest/medrx0301`
- `type` — CodeableConcept with `coding`:
  - `system` = `http://terminology.hl7.org/CodeSystem/v3-ActCode`
  - `code` = `PF`
  - `display` = `Partial Fill`
  - (This code is in the example-bound `v3-ActPharmacySupplyType` value set, matching the precedent set by `medicationdispense0301.xml`, which uses the same CodeSystem with `code=EM` "Emergency Supply".)
- `quantity` — e.g., `4 Vial` (a partial dispense against a `12 Vial` total ordered)
- `daysSupply` — e.g., `1 d` (using UCUM `http://unitsofmeasure.org`, code `d`), reflecting that only a subset of the prescribed days has been supplied
- `fillNumber` = `1` (first partial fill of the prescription; the comment in MedicationDispense.fillNumber notes that subsequent partial fills would be `2`, `3`, …, regardless of fractional quantity)
- `whenHandedOver` — a realistic ISO datetime
- Optional but recommended: a brief `dosageInstruction` mirroring the existing dispense pattern, and a `note` explaining the partial-fill scenario in human-readable text

In addition to the new example file, two registrations must be updated so the publisher picks the example up:

1. Add a new `<entry>` to `source/medicationdispense/list-MedicationDispense-examples.xml` (the manifest of MedicationDispense examples) with build extensions for `description` and `title` and an `<item>` `<reference value="MedicationDispense/meddisp0336"/>`. Title pattern: `medicationdispense0336`. Description: `"Dispense for Vancomycin (NDC code) - completed - Partial Fill (type=PF) with fillNumber=1, against MedicationRequest/medrx0301"`.
2. Add a new `<li>` to `source/medicationdispense/medicationdispense-examples-header.xml` so the example is linked from the rendered "Examples" tab. The closest matching bucket is the **INJECTABLE DOSAGE FORMS / Medication Resource for a manufactured product** group (which already contains `medicationdispense0301.html`).

No StructureDefinition, ValueSet, CodeSystem, search-parameter, or narrative-page changes are required — `type`, `fillNumber`, `daysSupply`, and `authorizingPrescription` already exist on `MedicationDispense` in the current build.

### Design Rationale

- **Reuse of `medrx0301`** keeps the example self-contained against existing fixtures. The resolution explicitly named the URL `https://build.fhir.org/medicationrequest0301.html` (which renders `MedicationRequest/medrx0301`), and `medicationdispense0301.xml` is already a dispense against that same request — so the new partial-fill instance can be read side-by-side with `medicationdispense0301` to highlight precisely the partial-fill delta.
- **`type` code `PF` from v3-ActCode** is the canonical "Partial Fill" entry in the example-bound value set (`v3-ActPharmacySupplyType`), matching how `medicationdispense0301.xml` already uses `code=EM` from the same CodeSystem. This keeps the new example consistent with existing examples and with the binding pattern documented in `structuredefinition-MedicationDispense.xml` (lines 497–528).
- **`fillNumber=1`** demonstrates the rule from the element's own definition: partial fills are counted as whole numbers, not fractional quantities (`structuredefinition-MedicationDispense.xml` line 574). Setting it to `1` (rather than e.g. `0.4`) shows implementers the correct encoding.
- **A separate quantity (`4 Vial`) vs. the original order (`12 Vial`)** illustrates the partial-fill semantics without requiring any prose commentary, because the reader can compare the new example's `quantity` to either the prescription's `dispenseRequest.quantity` or to `medicationdispense0301.xml`'s `12 Vial` quantity.
- The resolution explicitly grants the editor latitude ("and other things as deemed pertinent by the editor of the changes"), so the additional elements above (`whenHandedOver`, `note`, `dosageInstruction`) are recommended rather than mandated.

## Implementation Plan

### Prerequisites

None. All referenced fixtures (`Patient/pat1`, `Practitioner/f006`, `MedicationRequest/medrx0301`) and the `PF` terminology code already exist in the clone at HEAD `db79dcd`.

### Step-by-Step Tasks

#### HL7/fhir

1. **Create the new MedicationDispense partial-fill example.**
   - File: `source/medicationdispense/medicationdispense0336.xml`
   - Action: Create a new XML instance following the same structure as `source/medicationdispense/medicationdispense0301.xml` (root element `<MedicationDispense xmlns="http://hl7.org/fhir">` with `xsi:schemaLocation="http://hl7.org/fhir ../../schema/medicationdispense.xsd"`).
   - Details: Populate at minimum:
     - `<id value="meddisp0336"/>`
     - `<status value="completed"/>`
     - `<medication>` — either a `<reference>` to a `<contained><Medication>` block (mirroring `medicationdispense0301.xml`'s `med0301` Vancomycin Hydrochloride) **or** a `<concept>` with NDC `0409-6531-02` / RxNorm equivalent. Reusing the contained `Medication` pattern from `0301` keeps consistency.
     - `<subject><reference value="Patient/pat1"/><display value="Donald Duck"/></subject>`
     - `<performer><actor><reference value="Practitioner/f006"/></actor></performer>`
     - `<authorizingPrescription><reference value="MedicationRequest/medrx0301"/></authorizingPrescription>`
     - `<type><coding><system value="http://terminology.hl7.org/CodeSystem/v3-ActCode"/><code value="PF"/><display value="Partial Fill"/></coding></type>`
     - `<quantity><value value="4"/><unit value="Vial"/><system value="http://snomed.info/sct"/><code value="733026001"/></quantity>` (or another partial value smaller than the `12 Vial` used in `medicationdispense0301.xml`).
     - `<daysSupply><value value="1"/><unit value="Day"/><system value="http://unitsofmeasure.org"/><code value="d"/></daysSupply>`
     - `<fillNumber value="1"/>` (positiveInt; whole number per `MedicationDispense.fillNumber` definition).
     - `<whenPrepared value="2015-01-15T10:20:00Z"/>` and/or `<whenHandedOver>` consistent with the existing 0301 timeline.
     - Optional: a short `<note><text value="Partial fill of authorizingPrescription medrx0301; remaining quantity to be dispensed in subsequent fill (fillNumber=2)."/></note>` to make the scenario explicit for human readers, plus a copy-paste of the `<dosageInstruction>` block from `medicationdispense0301.xml` for completeness.
   - Verify the file parses against `schema/medicationdispense.xsd` (publisher will do this; see Validation Checklist below).

2. **Register the new example in the manifest list.**
   - File: `source/medicationdispense/list-MedicationDispense-examples.xml`
   - Action: Append a new `<entry>` block (after the final `meddisp0335` entry) of the form:
     ```xml
     <entry>
       <extension url="http://hl7.org/fhir/build/StructureDefinition/description">
         <valueString value="Dispense for Vancomycin (NDC code) - completed - Partial Fill (type=PF) with fillNumber=1, authorized by MedicationRequest/medrx0301"/>
       </extension>
       <extension url="http://hl7.org/fhir/build/StructureDefinition/title">
         <valueString value="medicationdispense0336"/>
       </extension>
       <item>
         <reference value="MedicationDispense/meddisp0336"/>
         <display value="Partial Fill"/>
       </item>
     </entry>
     ```
   - Details: Pattern matches every existing entry in the file (build extensions for `description` + `title`, an `<item>` with `reference` + `display`).

3. **Link the new example from the rendered "Examples" tab.**
   - File: `source/medicationdispense/medicationdispense-examples-header.xml`
   - Action: Append a new `<li>` to the `<ul>` immediately under the "INJECTABLE DOSAGE FORMS / The following examples of MedicationDispense resources for Injectable Dosage Forms that reference a Medication Resource for a manufactured product" paragraph (the `<ul>` that already begins with the `medicationdispense0301.html` link):
     ```xml
     <li><a href="medicationdispense0336.html">Dispense for Vancomycin (NDC code) - completed - Partial Fill (type=PF) with fillNumber=1, authorized by MedicationRequest/medrx0301</a></li>
     ```
   - Details: Keep the wording aligned with the manifest description from Task 2 so the rendered link text matches the example's stated purpose.

### Open Questions

- The resolution does not specify which prescription/medication to dispense; this plan reuses the IV Vancomycin context from `medrx0301` because the resolution explicitly links to `https://build.fhir.org/medicationrequest0301.html`. If the editor (Frank McKinney) prefers a more typical retail-pharmacy partial-fill scenario (e.g., a controlled-substance Schedule II oral medication), the same task structure applies — only the inner element values change.
- The resolution lists `medication` and `daysSupply` but does not state whether to use `medication.reference` (to a contained `Medication`) or `medication.concept` (inline CodeableConcept). The plan recommends the contained-`Medication` pattern from `medicationdispense0301.xml` for symmetry, but `medication.concept` (per `medicationdispense0335.xml`) is equally conformant; the editor may pick either.
- The resolution does not call for a partner `MedicationDispense` example representing the **subsequent** fill (e.g., `fillNumber=2`). If reviewers want the partial-fill semantics fully demonstrated as a sequence, a follow-up ticket could add `meddisp0337` showing the second portion of the same prescription. This is out of scope here.
