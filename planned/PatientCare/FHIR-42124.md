## Feature Proposal

### Problem Statement

In Use Case 3 ("Data Transfer") of the IBCM IG, the FHIR resource instance graph is difficult to correlate with the narrative text. Readers cannot easily trace which text-described clinical events (e.g., "Third visit to Obstetrician at Teaching Hospital") correspond to which nodes/edges in the graph diagram. The original single large graph (`usecase3.png`, 156 KB) was especially problematic.

### Current State (Partial Progress)

The IG has already been partially updated since the ticket was filed. The original monolithic `usecase3.png` graph has been supplemented with three smaller, visit-specific diagrams:

- `input/images/usecase3-firstMOHvisit.png` — First MOH visit
- `input/images/usecase3-followupMOHvisit.png` — Follow-up MOH visit
- `input/images/usecase3-thirdKandyvisit.png` — Third visit to Teaching Hospital Kandy

Each sub-diagram is accompanied by a resource mapping table linking concepts to FHIR example resources. This addresses the **"break this diagram into smaller diagrams"** part of the reporter's suggestion.

### Remaining Work

What has **not yet been done** per the resolution:

1. **Numbering/labeling in diagrams**: Add identifiers (numbers or letters) to graph nodes so readers can cross-reference between the narrative text and the graph elements (e.g., "Encounter MOH Visit **(1)**", "Patient Ms. MS **(2)**").
2. **Additional bridging text**: Add explanatory narrative text that explicitly connects the textual description of each clinical event to the corresponding FHIR resource instances shown in the graph.
3. **Old monolithic graph cleanup**: Evaluate whether `usecase3.png` (the original large graph) should be removed or retained with numbering, since the broken-up diagrams now serve the same purpose.

### Design Rationale

- Numbered labels in diagrams are a standard technique in technical documentation to create cross-references between prose and figures.
- The ClinFHIR-generated graphs may not support embedded labels natively, so alternative approaches include: (a) post-processing the PNG images to overlay numbers, (b) recreating the diagrams using PlantUML/Mermaid with labels, or (c) adding a numbered legend alongside each graph.
- Adding bridging text in the narrative (e.g., "The Patient resource for Ms. MS **(1)** is linked to the Encounter resource **(2)** representing the first MOH visit...") creates a direct textual-to-visual mapping.

## Implementation Plan

### Prerequisites

1. Clone the `HL7/fetal_records` repository (not currently cached)
2. Coordinate with assignee **Lilian Minne** (LilianMinne on GitHub) and the Patient Care work group
3. Determine whether ClinFHIR tooling can generate labeled graphs, or whether diagrams must be manually annotated or recreated

### Step-by-Step Tasks

#### Task 1: Assess Diagram Labeling Strategy

Determine the best approach for adding numbered labels to the existing graph images:

- **Option A**: Re-export from ClinFHIR with labels (if the tool supports it)
- **Option B**: Manually annotate the existing PNG files using an image editor to overlay numbered labels
- **Option C**: Recreate the diagrams using PlantUML or Mermaid with embedded labels in the IG build pipeline

**Recommendation**: Option B (manual annotation) is the most pragmatic and lowest-risk approach, given the resolution's acknowledgment that ClinFHIR tooling may not fully support this.

#### Task 2: Add Numbered Labels to Use Case 3 Sub-Diagrams

For each of the three sub-diagrams, add numbered labels to every resource node:

#### Task 3: Update Narrative Text in `use-case3-data-transfer.md`

Add bridging text before each graph that explicitly maps numbered items to the narrative. For example, before the first MOH visit graph, add:

```markdown
The graph below shows the FHIR resources for the first MOH visit. 
Ms. MS is represented as a Patient resource **(1)**, with her pregnancy 
recorded as a Condition **(2)**. The first visit is modeled as an 
Encounter **(3)** at the local MOH office **(4)**. The baseline 
examinations are captured as Observations: pregnancy test **(5)**, 
body weight of 47kg **(6)**, body height of 147cm **(7)**, hemoglobin 
of 12g/dl **(8)**, and urine for sugar **(9)**.
```

Similar bridging text should be added for each sub-diagram.

#### Task 4: Update or Remove Monolithic `usecase3.png`

Evaluate whether the original full graph (`usecase3.png`) is still referenced in the IG pages. If it is no longer referenced (the current `use-case3-data-transfer.md` does not reference it), consider removing it to avoid confusion. If retained, apply the same numbering scheme.

#### Task 5: Cross-reference with Related Applied Tickets

Ensure the changes are consistent with the work done for:
- FHIR-47865 (figure labels and captions — may already add `Figure N:` labels)
- FHIR-47998 (descriptive modeling text — may overlap with bridging text)

Coordinate to avoid duplicated or conflicting changes.
