## Resolution Summary

**Resolution:** Persuasive.

The ticket reports that in R6 §10.4.9.5 *DICOM Tag Mapping*, the row for
`ImagingStudy.series.started` uses a literal `+` between the two DICOM tag
references where, for consistency with neighbouring rows, the connector should
be the word `and`.

The applied change (already prepared on PR
[HL7/fhir#4043](https://github.com/HL7/fhir/pull/4043), commit
`79ad0b019`) replaces the DICOM mapping string for
`ImagingStudy.series.started` from:

> `Series Date (0008,0021) + and Series Time (0008,0031)`

to:

> `Series Date (0008,0021) and Series Time (0008,0031)`

This is a pure editorial / consistency fix in the DICOM mapping table; no
schema, cardinality, binding, or invariant changes.

## Feature Proposal

### Problem Statement

The DICOM Tag Mapping table rendered for the ImagingStudy resource (R6
spec §10.4.9.5) is generated from the `<mapping identity="dicom">/<map>`
strings inside `structuredefinition-ImagingStudy.xml`. The
`ImagingStudy.series.started` element currently maps to two DICOM tags
combined with a stray `+` sign (`(0008,0021) + and (0008,0031)`).
Surrounding rows (e.g., `series.performer`, `series.specimen`,
`series.modality`, `series.instance.title`) consistently use the natural
language word `and` to join multi-tag mappings. The lone `+` in the
`series.started` row is inconsistent with the rest of the table and reads
as an editorial defect.

### Proposed Change

Edit a single XML attribute value inside the differential of the
ImagingStudy StructureDefinition.

- **File:** `source/imagingstudy/structuredefinition-ImagingStudy.xml`
- **Element:** `ImagingStudy.series.started` (DICOM mapping inside the
  element with `id="ImagingStudy.series.started"`).
- **Change:** In the `<mapping>` whose `<identity value="dicom"/>`,
  replace the `<map>` value:
  - From: `Series Date (0008,0021) + and Series Time (0008,0031)`
  - To:   `Series Date (0008,0021) and Series Time (0008,0031)`

No other elements, attributes, or files are affected by this ticket
(FHIR-54786). The companion technical-correction
[FHIR-54785](https://jira.hl7.org/browse/FHIR-54785) was rolled into the
same commit and addresses the analogous `+` issues on the `ist-3`
constraint human text and on `series.specimen`'s DICOM mapping; those
changes are out of scope for this plan but should be expected to land
together.

### Design Rationale

- Consistency with the rest of the DICOM Tag Mapping table is the
  explicit reason given in the ticket.
- The DICOM mapping prose has no machine semantics — the change is a
  purely cosmetic / readability fix that keeps the published mapping
  table uniform.
- No alternative was discussed in Jira; the resolution is a
  straightforward "Persuasive" textual correction.
