## Resolution Summary

**Resolution:** Persuasive — "Remove sentence as described."

The first sentence of the ImagingSelection "Scope and Usage" section —
*"This resource is an event resource from a FHIR workflow perspective -
see Workflow."* — is redundant with the already-generated
"References to this Resource" section and should be deleted. The
committer comment on the ticket points at
[HL7/fhir PR #4043](https://github.com/HL7/fhir/pull/4043) (commit
`c429239`) as the implementing change.

## Feature Proposal

### Problem Statement

The ImagingSelection resource introduction page opens its "Scope and
Usage" section with a boilerplate sentence stating the resource is an
*event resource from a FHIR workflow perspective*, linking to
`workflow.html#event` and `workflow.html`. The reporter observes that
this information is already surfaced by the auto-generated "References
to this Resource" / workflow classification block on the resource
summary page, so repeating it in the human-authored scope prose adds no
value and clutters the intro. The WG agreed (Persuasive) that the
sentence should be removed.

### Proposed Change

In `HL7/fhir`, edit the ImagingSelection introduction page to delete
the redundant sentence from the Scope and Usage section. Concretely:

- **File:** `source/imagingselection/imagingselection-introduction.xml`
- **Section:** Inside the `<div>` that begins with
  `<a name="scope"></a> <h2>Scope and Usage</h2>`.
- **Remove:**

  ```xml
  <p>This resource is an <a href="workflow.html#event">
    <em>event resource</em>
  </a> from a FHIR workflow perspective - see <a href="workflow.html">Workflow</a>.</p>
  ```

- Leave the rest of the section (the "A selection of data from an
  ImagingStudy" paragraph and lists that follow) unchanged.
- No other edits — no StructureDefinition change, no search parameters,
  no terminology, no examples — the workflow classification itself
  stays intact; only the duplicate prose is deleted.

### Design Rationale

- The classification of ImagingSelection as an "event" resource in the
  FHIR workflow pattern is authoritative via the workflow page and the
  auto-generated resource-relationships section; hand-written prose
  does not need to restate it.
- Removing the sentence is consistent with the goal of keeping Scope
  and Usage narrative focused on domain content, not cross-cutting
  workflow taxonomy.
- The change is pure prose deletion in an authored HTML fragment, so
  no alternative design (e.g., rewording, moving to Notes) is needed —
  the WG resolution explicitly says "Remove sentence as described."
- Other event resources (`AdverseEvent`, `MedicationAdministration`,
  `MedicationDispense`, `MedicationStatement`) still contain the same
  boilerplate sentence. Those are **out of scope** for this ticket but
  are flagged under Open Questions in case the WG wants a follow-up.
