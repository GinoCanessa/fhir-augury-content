## Resolution Summary

Resolution is **Persuasive**. The approved change is to add an identical note paragraph at the end of both the *2D Image Selection* (section 10.5.5.3.3) and *3D Image Selection* (section 10.5.5.3.4) subsections of the ImagingSelection notes page. The exact wording from the resolution is:

> Note that the `imageRegion2D` and `imageRegion3D` elements are mutually exclusive; an imaging selection cannot include both types of image region.

A subsequent comment from `jwhitby` (2026-04-01) links the implementation commit [`c429239e`](https://github.com/HL7/fhir/commit/c429239eac28753c38f5211945b94996e2c86b05) in `HL7/fhir`, which lands this change in `source/imagingselection/imagingselection-notes.xml` along with several other unrelated Imaging Integration tickets bundled in the same commit. The plan below covers only the FHIR-54830 portion of that commit.

## Feature Proposal

### Problem Statement

The `ImagingSelection` resource defines a constraint that requires `imageRegion2D` and `imageRegion3D` to be mutually exclusive on a single imaging selection (an instance can carry one or the other, never both). The narrative documentation of these two element groups in the ImagingSelection notes page documents each one in isolation but never tells the reader that they cannot co-occur. The reporter (Ana Kostadinovska) flagged this as an incomplete description that risks misleading implementers who read the prose without consulting the formal invariants on the StructureDefinition. The fix is purely editorial — the constraint already exists; only the narrative needs to surface it.

### Proposed Change

Edit `source/imagingselection/imagingselection-notes.xml` and add an identical `<p>` element at the end of each of the two subsections that describe `imageRegion2D` / `imageRegion3D` selections.

Insert (after the existing `See <a href="imagingselection-example-2d-image-region-selection.html">2D Image Region Selection example</a>.` paragraph in the *2D Image Selection* `<h4>` block):

```xml
<p>
Note that the <code>imageRegion2D</code> and <code>imageRegion3D</code> elements are mutually exclusive; an imaging selection cannot include both types of image region.
</p>
```

Insert the same `<p>` element after the existing `See <a href="imagingselection-example-3d-image-region-selection.html">3D Image Region Selection example</a>.` paragraph in the *3D Image Selection* `<h4>` block.

No element-level changes (cardinality, type, binding, invariants) are required — the constraint enforcing mutual exclusivity already exists on the `ImagingSelection` StructureDefinition. This is a documentation-only change.

### Design Rationale

- The mutual-exclusivity rule already exists as a formal constraint on `ImagingSelection`. Restating it in narrative is consistent with how other FHIR resource notes pages surface formal invariants in human-readable form (e.g., the existing bullet lists that describe what each region type "requires" in the same `imagingselection-notes.xml` file).
- Placing the note at the *end* of each subsection — rather than once at the top of the combined "Image Region Selection" area — keeps the warning visible to any reader who navigates directly to either anchor (`#10.5.5.3.3` or `#10.5.5.3.4`) without scrolling up.
- Duplicating the identical sentence in both subsections (rather than putting it once with a cross-reference) is the simplest reader-facing fix; it matches the resolution wording verbatim and the discussion in the ticket showed no preference for a more elaborate alternative.
