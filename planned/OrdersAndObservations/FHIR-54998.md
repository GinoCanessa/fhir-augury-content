## Resolution Summary

The committee voted **Persuasive** with the following motion (verbatim):
"Motion to remove the profile from FHIR core, after the 52021 is completed."

In other words, the `MedicalProductOfHumanOrigin` (MPHO) profile — a profile on
`BiologicallyDerivedProduct` that was supposed to have been moved to the
OO-Incubator IG per the note to balloters — must be removed from the FHIR Core
build. The removal is **gated** on the prior completion of FHIR-52021 ("MPHO
profile slicing on category"), because that ticket touches the same profile and
its disposition needs to land first (or be confirmed as moot once the profile
leaves Core).

Per the imported ticket comment (JIRAUSER19407, 2026-01-29) the dependency is
explicit: *"See FHIR-52021"*.

## Feature Proposal

### Problem Statement

`StructureDefinition/medicalproductofhumanorigin` and its supporting authoring
files still ship in the FHIR Core ballot build (`source/biologicallyderivedproduct/`)
even though the WG previously communicated to balloters that the profile would
be relocated to the **OO-Incubator IG** (`HL7/oo-incubator`). Leaving the
profile in Core:

- Misleads balloters who were told it would no longer be in Core.
- Maintains a duplicate definition once the profile is published in OO-Incubator.
- Keeps a `draft`/FMM 1 profile inside an otherwise normative resource folder,
  which is inconsistent with the WG's intent to incubate it separately.

### Proposed Change

Remove the MPHO profile and all of its supporting authoring artefacts from the
FHIR Core repo (`HL7/fhir`, `source/biologicallyderivedproduct/`). Concretely:

1. **Delete the StructureDefinition itself.**
   - `source/biologicallyderivedproduct/structuredefinition-profile-medicalproductofhumanorigin.xml`
     (the constraint StructureDefinition with
     `id = medicalproductofhumanorigin`, canonical
     `http://hl7.org/fhir/StructureDefinition/medicalproductofhumanorigin`, name
     `MedicalProductOfHumanOrigin`).

2. **Delete the profile's narrative authoring fragments.**
   - `source/biologicallyderivedproduct/profile-mphobiologicallyderivedproduct-introduction.xml`
   - `source/biologicallyderivedproduct/biologicallyderivedproduct-mphobiologicallyderivedproduct-profile-notes.xml`
     (Both are empty `<div/>` placeholders today, but they are still wired into
     the profile's authoring set and should be removed alongside the SD.)

3. **Delete the profile-only example.**
   - `source/biologicallyderivedproduct/biologicallyderivedproduct-example-red-cell-antigen.xml`
     This example claims `meta.profile =
     http://hl7.org/fhir/StructureDefinition/medicalproductofhumanorigin` and
     uses MPHO-specific coding (`code = mpho`, ISBT 128 MPHOUniqueIdentifier
     system). Without the profile in Core, the validation target is gone and
     the example becomes orphaned; move it to OO-Incubator (out of scope here)
     or delete it from Core.

4. **Strip the MPHO entries from the BDP "core profiles" IG.**
   - `source/biologicallyderivedproduct/implementationguide-BiologicallyDerivedProduct-core.xml`
     currently lists the MPHO profile as its **only** resource:
     ```xml
     <definition>
       <resource>
         <reference>
           <reference value="StructureDefinition/medicalproductofhumanorigin"/>
         </reference>
       </resource>
     </definition>
     ```
     Remove that `<resource>` entry. If no other content remains for the
     BiologicallyDerivedProduct-core IG, also delete the file (and any pages
     that reference `BiologicallyDerivedProduct-core`).

5. **Audit and strip references in companion examples.**
   - `source/biologicallyderivedproduct/biologicallyderivedproduct-example-apheresis-platelet.xml`
     contains an MPHO reference (matched via the file scan). Remove or rework
     the reference so the example no longer depends on the MPHO profile/canonical.

6. **Update the BDP release notes.**
   - `source/biologicallyderivedproduct/BiologicallyDerivedProduct-release-notes.xml`
     – add an `<li>` entry referencing this ticket, e.g.:
     ```xml
     <li>The <code>MedicalProductOfHumanOrigin</code> profile has been removed
     from FHIR Core and migrated to the OO-Incubator IG
     [%jira-task FHIR-54998%]</li>
     ```

7. **Update `source/history.html`.**
   - The `history.html` page currently announces the profile's *addition*:
     > "...the profile [Medical Product of Human Origin](medicalproductofhumanorigin.html) profile has been added"
     Either delete that bullet (if the prior addition is being undone wholesale)
     or replace it with a removal entry pointing at FHIR-54998. The current
     bullet leaves a dangling link to `medicalproductofhumanorigin.html`, which
     will no longer be generated once the SD is gone.

The R6-Release-note label confirms a release-note entry is expected.

### Design Rationale

- The committee's motion is unambiguous: the profile is moving out of Core into
  the OO-Incubator IG. Implementation simply executes a decision already taken.
- Keeping the profile in Core after balloters were told it had moved would be
  confusing and would split the source of truth for MPHO. Removing it now
  honours the published ballot expectation.
- Sequencing: per the resolution, FHIR-52021 ("MPHO profile slicing on
  category") must be completed first. Either (a) FHIR-52021's outcome is
  applied to the profile in Core before this ticket deletes it (so OO-Incubator
  can pick up the corrected version), or (b) FHIR-52021 is closed as moot
  because the profile is leaving Core. Confirm with the WG which path is
  intended; either way, this ticket does not start until FHIR-52021's status is
  resolved.
- Migration into `HL7/oo-incubator` is owned by that repo and is **not** part
  of this Core ticket. This plan only covers the Core-side removal and the
  housekeeping it triggers (release notes, history bullet, IG file, examples).

### Breaking Changes

Yes — this **is** a breaking change in the spec sense, even if it executes a
previously-announced intent:

- **Removed StructureDefinition.** Implementers who built against
  `http://hl7.org/fhir/StructureDefinition/medicalproductofhumanorigin` will
  find the canonical no longer resolves under FHIR Core. Their dependency must
  shift to the OO-Incubator IG once it republishes.
- **Removed example.** The `biologicallyderivedproduct-example-red-cell-antigen`
  example URL will disappear from the Core spec.
- **Removed page.** `medicalproductofhumanorigin.html` will no longer be
  generated; any external bookmarks become 404s.

The R6-Release-note label is the right channel to flag this. The
`BiologicallyDerivedProduct-release-notes.xml` entry must call it out
explicitly so balloters and implementers are alerted.

## Implementation Plan

### Prerequisites

1. **FHIR-52021 must be resolved/applied first.** This is the explicit
   sequencing condition in the resolution motion. Do not start this ticket
   until FHIR-52021's outcome is committed (either as an applied change to the
   MPHO profile or as a confirmed moot-because-removed disposition).

   (`biologicallyderivedproduct-example-red-cell-antigen.xml`) and the IG file

   the WG that the OO-Incubator IG (`HL7/oo-incubator`) has — or will have —
   the corresponding artefacts so nothing is lost in transit. Migration to
   OO-Incubator is itself out of scope for this ticket.
3. **Repo-wide reference scan.** Before applying, run a
   case-insensitive search across `HL7/fhir` for
   `medicalproductofhumanorigin`, `MedicalProductOfHumanOrigin`,
   `mphobiologically`, and the profile canonical
   `StructureDefinition/medicalproductofhumanorigin` to catch any reference
   not already enumerated (search params files, list files, narrative pages,
   `fhir.ini`). The current scan found references only in the files listed
   below plus `source/history.html`.
