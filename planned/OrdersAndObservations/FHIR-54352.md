## Resolution Summary

The resolution is **Persuasive** with the resolution description "Will fix as
requested." The ticket reports a single spelling error in a diagram on the
Diagnostics module page: the word `(Aenarobic Culture)` must be corrected to
`(Anaerobic Culture)`. No semantic, structural, or conformance change is
involved — purely a cosmetic correction in the diagram artwork.

## Feature Proposal

### Problem Statement

The Diagnostics module page (`source/diagnostics-module.html`) embeds a set of
hand-authored diagrams illustrating how `DiagnosticReport` and `Observation`
relate for complex microbiology reports (Culture + Susceptibility). Those
diagrams were added by FHIR-48299 ("Add diagram(s) showing the structure for
complex reports …"). One of the diagrams contains the misspelling
`Aenarobic Culture` where it should read `Anaerobic Culture`. The surrounding
HTML prose on the same page already uses the correct spelling
("anaerobic culture" appears at lines 151, 163, and 172 of
`source/diagnostics-module.html`), so the diagram is the sole inconsistency.
The published PNG renders the misspelled label and is therefore visible to
every reader of the Diagnostics module.

### Proposed Change

Correct the spelling inside the source artwork and regenerate the published
PNG, then commit both:

1. In the PowerPoint source for the affected diagram under
   `images/source/parent-child-structure-*.source.pptx` — most likely
   `parent-child-structure-2.source.pptx` (the diagram described as showing
   "the specific resources and attributes … using a sample anaerobic culture
   as the basis for these relationships" at `diagnostics-module.html` lines
   149–157), with `parent-child-structure-1.source.pptx` and
   `parent-child-structure-3.source.pptx` as fallbacks if the typo is in one
   of the other two — find the text label `Aenarobic Culture` and replace it
   with `Anaerobic Culture`. Preserve font, size, color, position, and any
   surrounding parentheses (the ticket text shows the label is parenthesised:
   `(Aenarobic Culture)` → `(Anaerobic Culture)`).
2. Re-export the slide as PNG with the same dimensions and filename
   (`images/parent-child-structure-1.png`,
   `images/parent-child-structure-2.png`, or
   `images/parent-child-structure-3.png` to match the source that was
   edited) so the embed in `source/diagnostics-module.html` continues to
   resolve. Width is currently 1000px in the HTML embeds.
3. No changes are required to `source/diagnostics-module.html` itself — the
   prose is already correct and the `<img>` `src` / `alt` attributes do not
   reference the misspelling.

There are no element, cardinality, type, binding, terminology, search-
parameter, invariant, or example changes. This is a documentation/artwork-
only correction.

### Design Rationale

- The Diagnostics module diagrams in `images/` are generated from
  PowerPoint sources kept under `images/source/` (file naming convention
  `<diagram-name>.source.pptx`). The clone shows three matched pairs:
  `parent-child-structure-{1,2,3}.png` and
  `parent-child-structure-{1,2,3}.source.pptx`. The PPTX is the editable
  master; the PNG is the deliverable consumed by the bespoke HL7 publisher
  via the `<img>` tag in `source/diagnostics-module.html`.
- Editing the PPTX (rather than the PNG) preserves the established
  authoring workflow and keeps future edits cheap. Editing only the PNG
  (e.g., with an image editor) would diverge the source-of-truth from the
  rendered output and was rejected for that reason.
- Replacing the diagram with a generated representation (e.g., PlantUML)
  is out of scope: this ticket is a `Technical Correction` and the
  resolution is "Will fix as requested", which targets the misspelling
  only.

## Implementation Plan

### Prerequisites

- A working installation of Microsoft PowerPoint (or LibreOffice Impress
  with PPTX fidelity) capable of opening
  `images/source/parent-child-structure-*.source.pptx` and re-exporting
  the modified slide as PNG at the existing dimensions. No code or build
  prerequisite is required.

### Step-by-Step Tasks

#### HL7/fhir

1. **Locate the misspelled label**
   - File: `images/source/parent-child-structure-2.source.pptx` (then
     fall back to `parent-child-structure-1.source.pptx` and
     `parent-child-structure-3.source.pptx` if not found).
   - Action: Open each PPTX in PowerPoint, run a Find for `Aenarobic`
     (case-insensitive). Identify the slide and text shape containing
     `(Aenarobic Culture)`.
   - Details: The misspelling will appear inside parentheses per the
     ticket description. Note which file/slide contains the typo so
     subsequent steps target the correct artifact. If `Aenarobic`
     appears in more than one PPTX of the set, all occurrences must be
     fixed.

2. **Correct the spelling**
   - File: `images/source/parent-child-structure-<N>.source.pptx`
     (whichever was identified in Task 1).
   - Action: Replace `Aenarobic Culture` with `Anaerobic Culture` in
     each text shape where it occurs.
   - Details: Preserve font, font size, color, bold/italic, alignment,
     parenthesisation, and shape geometry. Do not relayout the slide.
     Save as `.pptx` (do not change format).

3. **Re-export the diagram PNG**
   - File: `images/parent-child-structure-<N>.png` (matching the PPTX
     edited in Task 2).
   - Action: From the corrected PPTX, export the affected slide as PNG
     and overwrite the existing file. Preserve filename, dimensions
     (the HTML embeds use `width="1000"`), and visual styling (padding,
     transparency where applicable).
   - Details: `source/diagnostics-module.html` line 155 styles
     `parent-child-structure-2.png` with `padding: 20px` in the
     `<img>` tag, so the PNG itself does not need internal padding. The
     other two are embedded plain. After regeneration, verify the file
     opens and renders the corrected label.

4. **Verify no other occurrences exist**
   - File: repo-wide.
   - Action: Run a case-insensitive search (`git grep -i Aenarobic`)
     across the clone to confirm `Aenarobic` is not present anywhere
     else (the current clone HEAD `db79dcd` shows zero matches in
     tracked text files; this rules out hidden duplicates in HTML or
     SVG and leaves only the binary PPTX/PNG pair as the source of
     the typo).
   - Details: This is also a guard against the typo being repeated in a
     PUML/SVG that might be added in the future on top of this fix.

5. **Commit and review**
   - File: commit message + PR description.
   - Action: Commit the modified `.pptx` and the regenerated `.png`
     together with a clear message such as
     `FHIR-54352: fix typo "Aenarobic" -> "Anaerobic" in microbiology diagram`.
     In the PR description, attach a before/after crop of the diagram
     label so reviewers can confirm the fix without opening PowerPoint
     (the binary PPTX/PNG diff is otherwise opaque on GitHub).
   - Details: Do not include unrelated changes (e.g., re-exporting the
     other two PNGs unless they were also edited). Touching only the
     edited pair keeps the diff minimal and reviewable.
