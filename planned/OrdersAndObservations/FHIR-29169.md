## Resolution Description

## Original Request

The reporter suggests adding a description/expansion of the acronym "EHR" (Electronic Health Record) at its first occurrence in the IG. Although EHR is a well-known acronym, best practice is to spell it out on first use.

## Cross-References

No cross-references found for this ticket.

---

## Feature Proposal

Add "Electronic Health Record (EHR)" as a new entry in the IG's glossary page. The resolution explicitly chose to address the concern via the glossary rather than expanding the acronym inline at first occurrence, so the implementation is a single glossary addition.

### Current State of `glossary.md`

The glossary currently contains 7 entries:

```markdown
# Glossary
This Implementation Guide defines and uses a number of terms and abbreviation as follows:
1. Durable Medical Equipment (DME)
2. Home Health Services (HH)
3. Medications associated with DME items (e.g. nebulizers and infusion pumps)
4. Prior Authorization (PA)
5. Appropriate Use (AU)
6. Medical Review (MR)
7. HealthCare Procedure Code System (HCPCS)
8. Clinical Procedure Terminology (CPT)
```

## Detailed Implementation Plan

### Step 1: Add EHR to the Glossary

**File:** `input/pagecontent/glossary.md`

**Action:** Append a new numbered entry for "Electronic Health Record (EHR)" to the glossary list.

**Proposed change:**

```markdown
# Glossary
This Implementation Guide defines and uses a number of terms and abbreviation as follows:
1. Durable Medical Equipment (DME)
2. Home Health Services (HH)
3. Prior Authorization (PA)
4. Appropriate Use (AU)
5. Medical Review (MR)
6. HealthCare Procedure Code System (HCPCS)
7. Clinical Procedure Terminology (CPT)
8. Electronic Health Record (EHR)
```

### Step 2 (Optional Enhancement): Expand Acronym at First Use

Although the resolution says "Will add to the glossary", for additional clarity consider also expanding "EHR" at its first occurrence in the IG narrative pages:

- **`input/pagecontent/restful_fhir_exchanges.md`** — Change "query the EHR" to "query the Electronic Health Record (EHR)" in the first assumptions list.
- **`input/pagecontent/fhir_messaging_exchanges.md`** — Change "information in the EHR" to "information in the Electronic Health Record (EHR)" in assumption 1.

This step is optional since the resolution specifically chose the glossary approach, but inline expansion on first use is a documentation best practice.
