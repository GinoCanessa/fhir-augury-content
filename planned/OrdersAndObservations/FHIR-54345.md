## Resolution Summary

The committee voted **Persuasive** on the motion: *"Motion to fix the broken
link to point to the incubator resources."* The Diagnostics Module page links
to `MolecularDefinition` and `GenomicStudy` using URLs under
`https://build.fhir.org/ig/HL7/cg-incubator/StructureDefinition-*.html` that
return HTTP 404. The fix is to repoint those links to the actual published
location of the CG Incubator IG artifacts.

The single open comment from `marti_velezis` (2026-02-03) flags an
implementation dependency: *"This needs to be coordinated with CG when they
have their Incubator IG set up."* The `Pending-SME-Input` label corroborates
that the final destination URL must be supplied by Clinical Genomics before
the change can land.

## Feature Proposal

### Problem Statement

The Diagnostics Module page (`source/diagnostics-module.html` in `HL7/fhir`)
contains an in-page table that lists the FHIR Resources involved in
diagnostics. Two of those entries — `MolecularDefinition` (line 54) and
`GenomicStudy` (line 55) — link out to
`https://build.fhir.org/ig/HL7/cg-incubator/StructureDefinition-MolecularDefinition.html`
and
`https://build.fhir.org/ig/HL7/cg-incubator/StructureDefinition-GenomicStudy.html`
respectively. Both URLs currently 404, so a reader who navigates the
Diagnostics Module looking for these incubator resources hits a dead end.

The same broken URL pattern is used in three additional places in the
repository (see Affected Files below), which means the underlying issue is
specification-wide rather than localized to one bullet list.

### Proposed Change

Update the four occurrences of the broken `cg-incubator` URLs in
`HL7/fhir/source/` to point at the live publication location of the CG
Incubator IG StructureDefinitions, once Clinical Genomics confirms it.
Concretely:

1. In `source/diagnostics-module.html`:
   - **Line 54** — replace
     `https://build.fhir.org/ig/HL7/cg-incubator/StructureDefinition-MolecularDefinition.html`
     in the `<li><a href="…">MolecularDefinition</a></li>` entry of the
     Diagnostics resources table with the working CG Incubator IG URL.
   - **Line 55** — replace the equivalent
     `…/StructureDefinition-GenomicStudy.html` URL in the
     `<li><a href="…">GenomicStudy</a></li>` entry.
   - **Line 108** — replace the same
     `…/StructureDefinition-MolecularDefinition.html` URL inside the
     `[%impl-note%] MolecularSequence resource has been replaced by …
     [%end-note%]` block.
2. In `source/genomics.html`:
   - **Line 111** — replace
     `…/StructureDefinition-GenomicStudy.html` in the GenomicStudy
     paragraph.
   - **Line 118** — replace
     `…/StructureDefinition-MolecularDefinition.html` in the
     MolecularDefinition paragraph.
3. In `source/ballot-intro.html`:
   - **Line 81** — replace
     `…/StructureDefinition-GenomicStudy.html`.
   - **Line 88** — replace
     `…/StructureDefinition-MolecularDefinition.html`.

The visible link text (`MolecularDefinition`, `GenomicStudy`,
`Molecular Definition`) is correct and should be preserved; only the `href`
target changes. The exact replacement URL must be supplied by Clinical
Genomics (see Open Questions / Prerequisites). Likely candidates are the
permanent CG Incubator IG canonical pages (e.g.
`https://hl7.org/fhir/uv/genomics-reporting/…`,
`https://build.fhir.org/ig/HL7/genomics-reporting/…`, or a new
`hl7.org/fhir/uv/cg-incubator/…` once published) but these MUST be
verified before the URLs are written into the spec.

### Design Rationale

The resolution does not call for any structural or semantic change to the
Diagnostics Module — it preserves the resources table layout, the link
text, and the surrounding prose. The minimal change that satisfies the
motion is a URL substitution. Repointing all four occurrences (rather
than just the two on the Diagnostics Module page that were quoted in the
ticket) keeps the spec internally consistent: the same incubator
resource is referenced from `genomics.html` and `ballot-intro.html` with
the same broken URL pattern, and silently leaving those broken would
re-open the issue under a different ticket.

The `Pending-SME-Input` label and the comment from `marti_velezis`
acknowledge that the editor cannot pick the replacement URL unilaterally
— the destination is owned by the Clinical Genomics work group's
Incubator IG. This is therefore a coordination-bound documentation fix,
not a design decision.
