## Resolution Summary

The resolution is **Persuasive** with resolution description "make the change
as indicated". The ticket asks for the literal removal of one forward-looking
sentence from the Security page summary list:

> "FHIR MAY also define a set of resources to administer access control
> management, but does not define any at present"

No other prose change, no new normative content, and no replacement wording is
called for. Because the Security page is going Normative, the sentence — which
hints at unplanned future resources — is to be deleted outright.

## Feature Proposal

### Problem Statement

The FHIR Security page (`security.html`) contains a top-level summary bullet
under the "Authorization/Access Control" item that reads:

> Authorization/Access Control - FHIR defines a Security Label infrastructure
> to support access control management. **FHIR MAY also define a set of
> resources to administer access control management, but does not define any
> at present**

The bolded clause is a forward-looking statement implying the spec may, in a
future release, introduce an entire new family of access-control resources.
There is no concrete plan to do so. With the Security page now progressing to
Normative, leaving an unplanned, speculative "MAY" statement in normative
prose is inappropriate: it creates an open-ended commitment that is neither
specified, balloted, nor scheduled.

### Proposed Change

Delete the trailing sentence from the Authorization/Access Control bullet in
the security summary list so that the bullet ends after "…to support access
control management."

- **File:** `source/security.html`
- **Location:** the `<li>` for `Authorization/Access Control` in the summary
  `<ol>` (currently line 38 in the clone at `db79dcd`).
- **Before:**

  ```html
  <li><a href="#binding">Authorization/Access Control</a> - FHIR defines a Security Label infrastructure to support access control management. FHIR MAY also define a set of resources to administer access control management, but does not define any at present</li>
  ```

- **After:**

  ```html
  <li><a href="#binding">Authorization/Access Control</a> - FHIR defines a Security Label infrastructure to support access control management.</li>
  ```

No other text on the page needs to change. The downstream
`<a name="binding"></a>` / `<h2>Authorization/Access Control</h2>` section
(line 236+) does not contain the offending sentence and is unaffected.

### Design Rationale

- The resolution wording ("make the change as indicated") and the ticket body
  both specify exactly which sentence to remove. No alternative wording or
  replacement is proposed by the submitter or the work group.
- The sentence is purely promissory ("MAY also define") and not a normative
  conformance statement. Deleting it cannot break any existing implementation.
- Keeping the bullet's first sentence preserves the cross-link to the
  `#binding` (Authorization/Access Control) section and the description of
  the existing Security Label infrastructure, so navigability and accuracy of
  the page are maintained.
- An alternative — replacing the sentence with a hedged "future work" note —
  was implicitly rejected by the resolution: the submitter argues that
  speculative language is inappropriate for Normative content, and the work
  group accepted this as Persuasive.
