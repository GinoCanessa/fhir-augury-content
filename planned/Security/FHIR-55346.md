## Problem Statement

`Provenance.entity.role` is a mandatory element (`min=1`) with a **required** binding to the `ProvenanceEntityRole` ValueSet. The current code system contains five codes (`revision`, `quotation`, `source`, `instantiates`, `removal`) but provides no escape valve concept such as "unknown" or "other". This makes the element impossible to populate when none of the existing codes apply — for example, when mapping from a legacy system with incomplete data, or when down-converting from a future FHIR version that introduces new concepts.

## Resolution Description

> Add "other" to the codeSystem.
>
> Definition: "The authoring/source system knows that none of the codes apply. An extension may elaboration on the way the data was used."

## Feature Proposal

Add an `other` concept to the `ProvenanceEntityRole` code system so that implementers have an escape valve when none of the existing role codes apply. Because the ValueSet includes the entire code system, the new concept will automatically be available through the required binding. The `short` value on the element definition must also be updated to reflect the new code.

### Backwards Compatibility

- **Non-breaking** for consumers: existing code values remain unchanged; a new optional code is added.
- **Non-breaking** for producers: the binding remains `required` with the same ValueSet; producers *may* use the new `other` code but are not required to.
- Systems that validate against the current code list will need to update their validation to accept the new `other` code.

## Implementation Plan

### Change 1: Add `other` concept to the CodeSystem

**File:** `source/provenance/codesystem-provenance-entity-role.xml`
**Repo:** HL7/fhir

Add a new `<concept>` element after the existing `removal` concept (before the closing `</CodeSystem>` tag):

```xml
  <concept>
    <code value="other"/>
    <display value="Other"/>
    <definition value="The authoring/source system knows that none of the codes apply. An extension may elaborate on the way the data was used."/>
  </concept>
```

> **Note:** The resolution text says "elaboration" but the grammatically correct form is "elaborate". This should be confirmed with the assignee during implementation.

**Current state (lines 71–76):**
```xml
  <concept>
    <code value="removal"/>
    <display value="Removal"/>
    <definition value="An entity that is removed from accessibility, usually through the DELETE operator."/>
  </concept>
</CodeSystem>
```

**After change:**
```xml
  <concept>
    <code value="removal"/>
    <display value="Removal"/>
    <definition value="An entity that is removed from accessibility, usually through the DELETE operator."/>
  </concept>
  <concept>
    <code value="other"/>
    <display value="Other"/>
    <definition value="The authoring/source system knows that none of the codes apply. An extension may elaborate on the way the data was used."/>
  </concept>
</CodeSystem>
```

### Change 2: Update StructureDefinition `short` and committee-notes

**File:** `source/provenance/structuredefinition-Provenance.xml`
**Repo:** HL7/fhir

#### 2a. Update `short` value (line 676)

**From:**
```xml
      <short value="revision | quotation | source | instantiates | removal"/>
```

**To:**
```xml
      <short value="revision | quotation | source | instantiates | removal | other"/>
```

#### 2b. Update committee-notes extension (line 673)

The existing committee-notes text states "No uses cases have been identified for extensibility for this role." This is now outdated since the ballot identified use cases (legacy system mapping, version down-conversion). Update:

**From:**
```xml
      <extension url="http://hl7.org/fhir/build/StructureDefinition/committee-notes">
        <valueString value="No uses cases have been identified for extensibility for this role."/>
      </extension>
```

**To:**
```xml
      <extension url="http://hl7.org/fhir/build/StructureDefinition/committee-notes">
        <valueString value="An 'other' code was added per FHIR-55346 for use cases such as legacy system mapping and version down-conversion."/>
      </extension>
```

### Change 3: ValueSet — No Change Required

**File:** `source/provenance/valueset-provenance-entity-role.xml`
**Repo:** HL7/fhir

The ValueSet includes the entire code system via:
```xml
  <compose>
    <include>
      <system value="http://hl7.org/fhir/provenance-entity-role"/>
    </include>
  </compose>
```

The new `other` code will be automatically included. No modification needed.
