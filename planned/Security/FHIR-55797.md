## Resolution Summary

Resolution: **Persuasive** — "add it".

The `Provenance.activity` binding (example-strength, ValueSet
`http://hl7.org/fhir/ValueSet/provenance-activity-type`) does not currently
enumerate a code for the "destroy / delete record" activity, even though the
Provenance narrative ("Provenance of Removal and Replace" in
`provenance-notes.xml`) explicitly describes recording the deletion of a
resource and the page links the `provenance-example-delete.xml` example. The
fix is to:

1. Add the `destroy` code (from
   `http://terminology.hl7.org/CodeSystem/iso-21089-lifecycle`) to the
   `provenance-activity-type` ValueSet, so the "destroy" lifecycle activity
   is one of the example codes surfaced for `Provenance.activity`.
2. Update the Provenance narrative section that talks about removal so it
   explicitly names `destroy` (from iso-21089-lifecycle) as the recommended
   vocabulary for the `Provenance.activity` element when recording the
   destruction/deletion of a resource.

## Feature Proposal

### Problem Statement

`Provenance.activity` is bound (strength `example`) to the ValueSet
`http://hl7.org/fhir/ValueSet/provenance-activity-type`. That ValueSet
currently composes a small enumerated list of codes from
`http://terminology.hl7.org/CodeSystem/iso-21089-lifecycle`
(`amend`, `originate`, `merge`, `deidentify`, `receive`, `transform`,
`verify`) plus narrative pointers to several other code systems (v3-ActCode,
v3-DataOperation, v3-DocumentCompletion, FHIR RESTful interactions, W3C
PROV). It does **not** enumerate `destroy`, even though:

- The Provenance page itself (section "Provenance of Removal and Replace"
  in `source/provenance/provenance-notes.xml`, anchor `removal`) is built
  around recording the removal/deletion of a resource and links the
  `provenance-example-delete.html` example.
- The `provenance-example-delete.xml` example uses a `DELETE` code from
  `v3-DataOperation`, but the narrative does not give implementers a
  consistent code from the lifecycle code system that the rest of the
  ValueSet draws from.
- ISO 21089 lifecycle includes a dedicated `destroy` code that is the
  natural match for this activity, and the work group already prefers that
  code system for record-lifecycle activities (it is the only enumerated
  include in the ValueSet today).

The result is a discoverability / consistency gap: implementers reading
the Provenance "removal" narrative do not see a recommended
`Provenance.activity` code in the bound ValueSet that matches the activity
the section is teaching.

### Proposed Change

Two coordinated changes inside `HL7/fhir`:

**A. ValueSet `provenance-activity-type` — add the `destroy` lifecycle code.**

In `source/terminologies/valueset-provenance-activity-type.xml`, inside the
existing `<compose><include system="http://terminology.hl7.org/CodeSystem/iso-21089-lifecycle">`
block, add a new `<concept>` entry:

```xml
<concept>
  <code value="destroy"/>
</concept>
```

The code SHOULD be appended to the existing seven concepts in this include
block (no other includes change). Bump the ValueSet `<version>` per the
HL7/fhir publisher conventions (typically the current cycle's version
roll), and refresh `<date>` to the day of the edit.

The binding strength on `Provenance.activity` remains `example` (no
StructureDefinition change required); this is purely a ValueSet content
addition.

**B. Narrative — name `destroy` in the Provenance "removal" section.**

In `source/provenance/provenance-notes.xml`, update the
"Provenance of Removal and Replace" section (anchor `removal`, currently at
lines ~74–87) so the prose explicitly mentions that the recommended
`Provenance.activity` code for recording the destruction/deletion of a
resource is `destroy` from
`http://terminology.hl7.org/CodeSystem/iso-21089-lifecycle`. Concretely,
extend the first paragraph (currently ending at line 81) with a sentence
along the lines of:

> The recommended `Provenance.activity` value when recording the removal
> of a resource is `destroy` (from
> [ISO 21089 Lifecycle](http://terminology.hl7.org/CodeSystem/iso-21089-lifecycle)),
> which is one of the codes enumerated in the bound
> [ProvenanceActivityType](valueset-provenance-activity-type.html)
> ValueSet.

Keep the existing link to `provenance-example-delete.html`. Do **not**
silently change the `provenance-example-delete.xml` example's use of
`v3-DataOperation#DELETE` — the resolution does not call for changing the
example, only for adding `destroy` to the ValueSet and naming it in the
narrative. (See Open Questions for whether to also realign the example.)

### Design Rationale

- **Consistency with the existing include set.** The ValueSet's only
  enumerated include today is `iso-21089-lifecycle`; `destroy` is the
  iso-21089-lifecycle code that maps cleanly to the activity the narrative
  already describes. Adding it to the same include keeps the enumerated
  list internally consistent and avoids introducing a new code system into
  the enumerated portion.
- **Example-strength binding preserved.** Because the binding is
  `example`, this is an additive, non-breaking change — implementers
  already free to use any code can now find a recommended one inside the
  ValueSet itself.
- **Narrative ↔ ValueSet alignment.** The ticket explicitly asks that "the
  section speaks to the vocabulary `destroy` for activity"; updating the
  `removal` section ensures readers of the Provenance page see the same
  vocabulary they will find in the bound ValueSet.
- **Alternative considered:** adding a `v3-DataOperation#DELETE` enumerated
  include to mirror `provenance-example-delete.xml`. Rejected because (a)
  the resolution explicitly says "add `destroy`", (b) the existing enumerated
  include is iso-21089-lifecycle, and (c) DELETE is already discoverable
  via the description's pointer to v3-DataOperation.

### Step-by-Step Tasks

#### HL7/fhir

1. **Add `destroy` to the `provenance-activity-type` ValueSet.**
   - File: `source/terminologies/valueset-provenance-activity-type.xml`
   - Action: Inside the existing
     `<compose><include system="http://terminology.hl7.org/CodeSystem/iso-21089-lifecycle">`

     `deidentify`, `receive`, `transform`, `verify`), append a new

   - Details: Bump `<version>` from `2.0.0` to the current publication
     cycle's value (per the publisher's conventions — match what other
     Security-WG ValueSets in the same build are using), and update
     `<date>` to the edit date. Do not change the binding strength or any
     other include. Do not touch the `<description>` (it already lists
     `iso-21089-lifecycle` as an underlying code system).

2. **Update the Provenance "removal" narrative to name `destroy`.**
   - File: `source/provenance/provenance-notes.xml`
   - Action: In the section anchored by `<a name="removal"> </a>` and
     headed `<h3>Provenance of Removal and Replace</h3>` (currently
     ~lines 74–87), extend the first paragraph (which today ends at
     "...Provenance.target contains the non-version reference.") with a
     sentence naming `destroy` from
     `http://terminology.hl7.org/CodeSystem/iso-21089-lifecycle` as the
     recommended `Provenance.activity` value when recording deletion, and
     link to `valueset-provenance-activity-type.html`.
   - Details: Suggested wording (adjust to house style):

     ```html
     <p>
     The recommended <code>Provenance.activity</code> value when recording
     the removal of a resource is <code>destroy</code> (from
     <a href="http://terminology.hl7.org/CodeSystem/iso-21089-lifecycle">ISO 21089 Lifecycle</a>),
     which is one of the codes enumerated in the
     <a href="valueset-provenance-activity-type.html">ProvenanceActivityType</a>
     ValueSet bound to <code>Provenance.activity</code>.
     </p>
     ```

     Keep the existing anchor and the existing `provenance-example-delete.html`
     link intact.

3. **(Verification only — no edit.)** Re-read
   `source/provenance/structuredefinition-Provenance.xml` element
   `Provenance.activity` (lines 310–326) to confirm the binding still
