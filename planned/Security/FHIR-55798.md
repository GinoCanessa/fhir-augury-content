## Resolution Summary

**Resolution:** Persuasive. Resolution description (verbatim): *"Note that the
description element does not render on the human readable page."*

The ticket reports that the `description` element on the `ProvenanceActivityType`
ValueSet (`http://hl7.org/fhir/ValueSet/provenance-activity-type`, file
`source/terminologies/valueset-provenance-activity-type.xml`) is "not appropriate":
it consists of an unexplained header line ("code systems are:") followed by a
bulleted list of links to several CodeSystems (v3-DocumentCompletion,
v3-DataOperation, v3-ActCode, ISO 21089 Lifecycle, FHIR RESTful interactions,
W3C PROV Activity Types) — most of which are *not* even composed into the
ValueSet (only `iso-21089-lifecycle` is `<include>`d). The resolver agreed this
is "Persuasive" and noted the qualifier that this prose does not render on the
human-readable page (so the user-visible impact is minimal, but the source of
truth still needs to be corrected).

The required change is therefore to:

1. Replace the current `<description>` value with a short prose explanation of
   what `ProvenanceActivityType` is for (the representative activity-type codes
   bound at example strength to `Provenance.activity`).
2. Remove the bulleted list of links to "unrelated" CodeSystems from the
   description. (The actual composition is already declared in `<compose>` and
   does not need to be duplicated as a link list in the description.)

No `<compose>` content, identifiers, status, or binding changes are required.

## Feature Proposal

### Problem Statement

The `description` element on `ValueSet/provenance-activity-type` reads:

> code systems are:
> - [v3-DocumentCompletion](http://terminology.hl7.org/CodeSystem/v3-DocumentCompletion)
> - [v3-DataOperation](http://terminology.hl7.org/CodeSystem/v3-DataOperation)
> - [v3-ActCode](http://terminology.hl7.org/CodeSystem/v3-ActCode)
> - [ISO Lifecycle](http://terminology.hl7.org/CodeSystem/iso-21089-lifecycle)
> - [FHIR RESTful interactions](codesystem-restful-interaction.html)
> - [W3C PROV Activity Types](http://hl7.org/fhir/w3c-provenance-activity-type)

This is problematic for two reasons:

- It does not actually describe what the ValueSet is or how it should be used.
  A `ValueSet.description` is meant to convey purpose, intent, and scope to
  readers and tooling — not to enumerate links.
- It lists CodeSystems that are not part of the ValueSet's `<compose>` block.
  The current `<compose>` only `<include>`s the `iso-21089-lifecycle` system
  with a curated set of seven concepts (`amend`, `originate`, `merge`,
  `deidentify`, `receive`, `transform`, `verify`). Listing v3-DocumentCompletion,
  v3-DataOperation, v3-ActCode, FHIR RESTful interactions, and W3C PROV
  Activity Types as if they were members is misleading — those are "unrelated
  things" with respect to the ValueSet's actual content.

The Security WG accepted the ticket as Persuasive. The committer noted the
prose does not currently render on the human-readable page, so this fix is a
data-quality correction to the ValueSet resource itself rather than a
visible-rendering fix.

### Proposed Change

Edit the single file `source/terminologies/valueset-provenance-activity-type.xml`
and replace the value of the `<description>` element with a short, focused
explanation of the ValueSet. A suggested replacement (Security WG to confirm
exact wording):

> A representative set of codes that may be used to describe the kind of
> activity that produced or modified a resource version, for use as the
> example-strength binding on `Provenance.activity`. The codes included here
> are drawn from the ISO 21089 Lifecycle code system; implementers may extend
> or replace the binding with another activity-type code system appropriate to
> their environment.

Concretely, in
`source/terminologies/valueset-provenance-activity-type.xml`:

- **Line 44** — replace the existing
  `<description value="code systems are:&#xA;- [v3-DocumentCompletion]…- [W3C PROV Activity Types](http://hl7.org/fhir/w3c-provenance-activity-type)"/>`
  with a single-line `<description value="…"/>` containing the new prose
  (encoded as needed; no embedded link list).

No other element of the file changes:

- `id` / `url` / `identifier` / `version` (`2.0.0`) / `name`
  (`ProvenanceActivityType`) / `title` / `status` (`active`) /
  `experimental` / `date` / `publisher` / `contact` — unchanged.
- `<extension>` block (`structuredefinition-wg = sec`,
  `structuredefinition-standards-status = informative`,
  `structuredefinition-fmm = 6`, `valueset-extensible = true`) — unchanged.
- `<compose>` (the `iso-21089-lifecycle` include with the seven concepts) —
  unchanged.

The binding on `Provenance.activity` in
`source/provenance/structuredefinition-Provenance.xml` (line 325) continues to
reference this ValueSet at `example` strength; no edit is required there.

### Design Rationale

- The ticket asks to (a) "explain the valueSet" and (b) "remove links to
  unrelated things." Both are addressed by rewriting the description to
  describe purpose/intent and dropping the inline bullet list.
- Keeping the `<compose>` content unchanged honours the resolver's intent —
  the ticket scope is narrative cleanup, not a re-curation of which codes the
  ValueSet contains. Changing the composition would be a separate, breaking
  change and was not requested.
- The phrase "representative set of codes …" mirrors the existing `<title>`
  ("ValueSet of representative activity type codes.") and the `example`
  binding strength on `Provenance.activity`, which together signal that this
  is an illustrative — not normative — list.
- The resolver's note that "the description element does not render on the
  human readable page" explains why no separate intro page or HTML edit is
  required: there is no narrative file under `source/terminologies/` for this
  ValueSet to keep in sync.
