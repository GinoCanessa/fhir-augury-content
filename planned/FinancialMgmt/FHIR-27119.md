## Context Gathered Before Halt (for reference)

### Resolution Summary

Resolution: **Persuasive with Modification**.

The committee determined that the requested addition of a `replacedBy` code is
not appropriate because the concept is the same as the existing `prior` code.
Instead, the resolution is to **amend the definition of the existing `prior`
code** in the `ex-relatedclaimrelationship` CodeSystem to read:

> "A prior Claim instance for the same suite of services. This is typically
> used when an amended Claim (prior authorization, predetermination or claim)
> is submitted."

No new codes are to be added; the value set membership remains `prior` and
`associated`.

### Original Request

The reporter (Saul Kravitz, 2020-05-12) noted that the current
claim-relationship value set is an example with two values (`Prior`,
`Associated`) and asked that `ReplacedBy` be added to better reflect industry
practice.
