## Resolution Summary

Resolution: **Persuasive**.

Per the ticket text: *"Remove the self pay codesystem from the valueset and change
the .name and .title to remove the reference to self pay codes."*

Concretely, this means in the `Coverage Type` ValueSet
(`http://hl7.org/fhir/ValueSet/coverage-type`):

1. Drop the `<include><system value="http://terminology.hl7.org/CodeSystem/coverage-selfpay"/></include>`
   composition entry.
2. Rename the ValueSet metadata so it no longer advertises self-pay codes:
   - `ValueSet.name`:  `CoverageTypeAndSelfPayCodes` → `CoverageTypeCodes`.
   - `ValueSet.title`: `Coverage Type and Self-Pay Codes` → `Coverage Type Codes`.

Self-pay status on a Coverage instance is now expressed via the dedicated
`Coverage.kind` element (with code `self-pay`) plus `Coverage.paymentBy`, so the
self-pay codes no longer need to overload the insurance-type binding.

## Feature Proposal

### Problem Statement

The `Coverage Type` ValueSet bound to `Coverage.type` currently mixes two distinct
concepts:

- The intended insurance-type axis (v3 `ActCoverageTypeCode`).
- The `coverage-selfpay` CodeSystem from
  `http://terminology.hl7.org/CodeSystem/coverage-selfpay`.

In R5, `Coverage` introduced an explicit `Coverage.kind` element (
`insurance | self-pay | other`, bound to `coverage-kind`) and an associated
`Coverage.paymentBy` backbone, including the invariant `cov-self-pay-1`
("When patient is self-pay, paymentBy must exist") seen in
`source/coverage/structuredefinition-Coverage.xml`. Self-pay is therefore
already first-class on the resource. Leaving `coverage-selfpay` codes in the
insurance-type ValueSet is now redundant, encourages double-encoding, and is
misleadingly reflected in the ValueSet's `name` and `title`
(`CoverageTypeAndSelfPayCodes` / "Coverage Type and Self-Pay Codes").

### Proposed Change

In `source/coverage/valueset-coverage-type.xml`:

- Update metadata:
  - `ValueSet.name`  = `CoverageTypeCodes`
  - `ValueSet.title` = `Coverage Type Codes`
  - `ValueSet.description` may stay as "This value set includes Coverage Type
    codes." (already self-pay-free).
- Update `compose`:
  - Remove the entire `<include><system value="http://terminology.hl7.org/CodeSystem/coverage-selfpay"/></include>` block.
  - Retain `<include><valueSet value="http://terminology.hl7.org/ValueSet/v3-ActCoverageTypeCode"/></include>` as the sole composition.

The standalone `source/coverage/valueset-coverage-selfpay.xml`
(`CoverageSelfPayCodes`) is **not** in scope of this ticket — the resolution only
addresses the combined `coverage-type` ValueSet. That ValueSet should remain as
the dedicated wrapper for the `coverage-selfpay` CodeSystem unless a follow-up
ticket retires it.

`Coverage.type`'s binding (`http://hl7.org/fhir/ValueSet/coverage-type`,
`extensible`) is **not** changed by URL — only the contents narrow. Existing
instances that currently use `coverage-type` codes drawn from
`v3-ActCoverageTypeCode` are unaffected.

Example (post-change `valueset-coverage-type.xml` `compose`):

```xml
<compose>
  <include>
    <valueSet value="http://terminology.hl7.org/ValueSet/v3-ActCoverageTypeCode"/>
  </include>
</compose>
```

### Design Rationale

- Avoids a single coded element conflating two orthogonal facts (kind of
  payment vs. insurance product type) now that R5 has dedicated structure
  (`Coverage.kind`, `Coverage.paymentBy`, invariant `cov-self-pay-1`) for
  self-pay scenarios.
- Keeps the ValueSet URL stable (`http://hl7.org/fhir/ValueSet/coverage-type`)
  so the binding on `Coverage.type` does not need to be re-pointed and
  downstream IGs don't need URL fix-ups.
- The renaming brings `name` and `title` back in line with what the ValueSet
  actually contains after the include is removed; mismatched metadata would
  otherwise confuse implementers and tooling.
- The standalone `valueset-coverage-selfpay.xml` is left untouched, because the
  resolution explicitly scopes the change to "the valueset" containing both —
  i.e., the combined one — and downstream consumers may still need a value set
  for self-pay codes alone.

## Impact Analysis

### Affected Repositories

#### HL7/fhir (FhirCore)

- **Role:** Owns the authored ValueSet bound to `Coverage.type`.
- **Affected Files:**
  - `source/coverage/valueset-coverage-type.xml` — drop the `coverage-selfpay`
    `include`; rename `name` to `CoverageTypeCodes` and `title` to
    "Coverage Type Codes".
- **Change Scope:** Minor.

### Breaking Changes

Removing the `coverage-selfpay` codes from the ValueSet bound to `Coverage.type`
*narrows* the value set. The binding strength on `Coverage.type` is `extensible`
(it permits codes outside the value set when no suitable code exists), which
softens the impact, but:

- Existing R4/R4B/early-R5-draft instances that populated `Coverage.type` with a
  code from `http://terminology.hl7.org/CodeSystem/coverage-selfpay` (e.g., `pay`)
  will no longer produce a value-set membership match. Validators will report
  the code as outside the value set; under `extensible` they should still
  validate, but conformance reports change.
- Implementers should migrate self-pay representation to `Coverage.kind` =
  `self-pay` (with `Coverage.paymentBy` populated, per invariant
  `cov-self-pay-1`).

This is a content/binding-content change rather than a structural break, but it
should be called out in R5 / R6 ballot/change-tracking notes for FM.

### Related Specifications

- `Coverage.type` (binding to `http://hl7.org/fhir/ValueSet/coverage-type`) in
  `source/coverage/structuredefinition-Coverage.xml`.
- `Coverage.kind` (binding to `coverage-kind`,
  `source/coverage/codesystem-coverage-kind.xml` /
  `valueset-coverage-kind.xml`) — the now-canonical place to express self-pay.

## Implementation Plan

### Prerequisites

None. The change is self-contained in a single ValueSet file in `HL7/fhir`. The
referenced CodeSystem (`coverage-selfpay`) and the alternative include
(`v3-ActCoverageTypeCode`) already exist; `Coverage.kind` /
`Coverage.paymentBy` are already in place to absorb the self-pay use case.

### Step-by-Step Tasks

#### HL7/fhir

1. **Rename the ValueSet metadata to drop the self-pay reference**
   - File: `source/coverage/valueset-coverage-type.xml`
   - Action:
     - Change `<name value="CoverageTypeAndSelfPayCodes"/>` to
       `<name value="CoverageTypeCodes"/>`.
     - Change `<title value="Coverage Type and Self-Pay Codes"/>` to
       `<title value="Coverage Type Codes"/>`.
   - Details: Leave the `id`, `url`, identifiers, version, status,
     experimental flag, publisher, contact, description, and copyright as-is.
     Description already reads "This value set includes Coverage Type codes."
     and does not require an edit.

2. **Remove the self-pay CodeSystem include from the ValueSet composition**
   - File: `source/coverage/valueset-coverage-type.xml`
   - Action: Delete the include block:
     ```xml
     <include>
       <system value="http://terminology.hl7.org/CodeSystem/coverage-selfpay"/>
     </include>
     ```
     Retain the remaining include
     `<include><valueSet value="http://terminology.hl7.org/ValueSet/v3-ActCoverageTypeCode"/></include>`
     unchanged.
   - Details: The resulting `<compose>` should contain a single `<include>`
     pointing at `http://terminology.hl7.org/ValueSet/v3-ActCoverageTypeCode`.

3. **Sanity-check Coverage.type binding consumers (no edit expected)**
   - File: `source/coverage/structuredefinition-Coverage.xml`
   - Action: Confirm the `Coverage.type` element binding still references
     `http://hl7.org/fhir/ValueSet/coverage-type` (currently does, around
     line 277). No structural edit required; the URL is unchanged.
   - Details: Self-pay-related text on `Coverage.kind` (`short`
     "insurance | self-pay | other"), `Coverage.paymentBy`, and invariant
     `cov-self-pay-1` (lines ~194, 200–203, 225, 230, 510) are about
     `Coverage.kind`/`paymentBy`, **not** `Coverage.type`, and must remain
     untouched.

4. **Review the self-pay Coverage example for consistency (optional but
   recommended)**
   - File: `source/coverage/coverage-example-selfpay.xml`
   - Action: If the example currently encodes self-pay using a
     `Coverage.type.coding` with a code from
     `http://terminology.hl7.org/CodeSystem/coverage-selfpay`, update it to
     express self-pay through `Coverage.kind = self-pay` (with
     `Coverage.paymentBy` populated as needed by `cov-self-pay-1`) instead.
   - Details: Out of strict scope for the resolution text but in keeping with
     its intent; only edit if the example would otherwise become misleading.
