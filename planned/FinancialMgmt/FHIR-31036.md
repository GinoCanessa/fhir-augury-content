## Feature Proposal

### Problem Statement

Payer adjudication systems need a v2-0203 (Identifier Type) concept to
flag identifiers that are payer-assigned **claim** identifiers, distinct
from generic claim numbers (`FILL`/`PLAC`) or other existing identifier
types. The original ticket also asked for two additional payer-oriented
codes (Unique Member ID, Patient Account Number) and a cosmetic rename
of the code system, but FM judged only the claim identifier to be
sufficiently justified after review (see `pknapp` 2025-01-30 comment
asking for a use case, and the resulting modified resolution).

### Proposed Change

Add exactly one new concept to CodeSystem
`http://terminology.hl7.org/CodeSystem/v2-0203`:

- **code:** `UCID`
- **display:** `Unique Claim Identifier`
- **definition:** `A unique identifier assigned by a payer for a claim
  received from a provider or subscriber.`
- **status property:** `active`
- **placement:** alphabetical position between `UDI` and `UMB` (i.e., the
  position where the cached UTG clone already shows it at concept id
  `12093`).

XML form (matches the existing concept blocks in `cs-v2-0203.xml`):

```xml
<concept id="12093">
  <code value="UCID"/>
  <display value="Unique Claim Identifier"/>
  <definition value="A unique identifier assigned by a payer for a claim received from a provider or subscriber."/>
  <property>
    <code value="status"/>
    <valueCode value="active"/>
  </property>
</concept>
```

No other v2-0203 concept should be added or modified under this ticket.
In particular, do **not** add `UM`, `PAT`, or any "uc" lowercase variant,
and do **not** retitle/rename the CodeSystem.

The associated ValueSet `http://terminology.hl7.org/ValueSet/v2-0203`
includes the entire CodeSystem and therefore picks up `UCID`
automatically; no edits to `vs-v2-0203.xml` are required.

### Design Rationale

- FM's modified resolution is conservative: the only use case backed by
  a sponsoring proposal at review time was the claim identifier, so only
  that code was approved.
- `UCID` (4-letter mnemonic) was chosen over the lowercase `uc` proposed
  here because UP-699 (the implementing terminology ticket) follows the
  established v2-0203 convention of upper-case mnemonics (e.g., `UDI`,
  `UMB`, `UPIN`, `USID`).
- Definition wording aligns with the originally requested semantics
  ("payer-assigned identifier for a claim received from a provider or
  subscriber") with minor copy-edit for THO publication.
- Renaming v2-0203 → "Identifier Type" is **not** done as part of FHIR-31036
  because (a) the modified resolution scopes the approval to the new code
  only and (b) the CodeSystem `.name`/`.title` already convey
  "IdentifierType" in the current cached source of truth.
