## Resolution Summary

**Resolution type:** Persuasive.

Resolution description: *"Change 'the FHIR' to 'the FHIR specification'."*

The ticket asks that the sentence in `services.html#considerations` reading
"Rollback is not expressly specified as part of the FHIR." be corrected so it no
longer uses "the FHIR" as if FHIR were a noun preceded by a definite article. The
applied resolution (slightly different from the original proposal of "change 'the
FHIR' to just 'FHIR'") is to replace "the FHIR" with "the FHIR specification",
yielding: "Rollback is not expressly specified as part of the FHIR specification."

## Feature Proposal

### Problem Statement

On the FHIR Core "RESTful API Considerations" page (`services.html`, section
*Transactional Integrity*), the current prose says:

> Rollback is not expressly specified as part of the FHIR.

"the FHIR" is ungrammatical — FHIR is a proper noun / specification name, not a
noun that takes the definite article. The same paragraph already uses the
correct forms elsewhere ("the FHIR specification", "the FHIR server", etc.), so
this sentence is inconsistent with the rest of the page and with FHIR editorial
usage generally.

### Proposed Change

Edit `source/services.html` in `HL7/fhir` to replace the phrase "part of the
FHIR" with "part of the FHIR specification" in the Transactional Integrity
paragraph.

Exact before/after of the affected sentence:

- **Before:** `Rollback is not expressly specified as part of the FHIR.`
- **After:**  `Rollback is not expressly specified as part of the FHIR specification.`

No other text on the page changes. This is a pure narrative/prose correction
with no schema, StructureDefinition, terminology, or example impact.

### Design Rationale

The resolution text (Persuasive) explicitly states "Change 'the FHIR' to 'the
FHIR specification'", so the wording is prescribed by the work group's vote
rather than left to the implementer. This form is preferred over the reporter's
original suggestion ("just 'FHIR'") because:

- It keeps the sentence grammatical — "specified as part of FHIR" reads
  acceptably, but "specified as part of the FHIR specification" matches the
  register used in the surrounding paragraph and in analogous passages on the
  same page ("The FHIR specification does not inherently provide for
  concurrency management ..." appears two sentences later).
- It is explicit about what is and isn't in scope: rollback is not specified by
  the FHIR specification itself, but an implementation may still layer it on.
