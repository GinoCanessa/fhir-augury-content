## Resolution Summary

The reporter identified a typo on the "Resource Definitions" page of the FHIR specification. Under the **"Key to Type Icons"** heading, the fifth bullet item describing the complex-type icon contains the misspelling **"uppwer case letter"** instead of **"upper case letter"**. The work group resolved this as **Persuasive** — the change is accepted as submitted and requires implementation.

---

## Feature Proposal

Correct the single-character typo in `source/formats.html` by changing `uppwer` to `upper` on the line describing the complex datatype icon.

**Before:**
> All complex type names defined in this specification start with an uppwer case letter

**After:**
> All complex type names defined in this specification start with an upper case letter

## Implementation Plan

<li><img title="Datatype" style="background-color: white;" alt="." class="hierarchy" src="icon_datatype.gif"/>: An element of a datatype which describes an element that has other elements.
