## Resolution Summary

The Version Management Policy page (`source/versions.html`) shows an example of an
R4 `Immunization` instance that carries the R5
`http://hl7.org/fhir/5.0/StructureDefinition/extension-Immunization.administeredProduct`
extension. The trailing prose said:

> *Note that this is an R5 extension, since valueAttachment was introduced in R5.*

This wording is wrong — the example does not use `valueAttachment` at all; it
uses the `administeredProduct` extension (which itself is what was introduced
in R5). The approved resolution (Persuasive) is purely a prose correction:

> Will change *"…since valueAttachment was introduced in R5"* to *"…since
> administeredProduct was introduced in R5"*.

No model, schema, profile, terminology, or example-instance changes are
required.

## Feature Proposal

### Problem Statement

In the *Version Management Policy* page, the worked example that demonstrates
how to use an R5 extension on an R4 resource is followed by an explanatory
sentence that misnames the extension. The sentence references `valueAttachment`
— a `value[x]` choice that was introduced in R5 — but the example above it
does not use `valueAttachment`. The example uses the
`extension-Immunization.administeredProduct` complex extension, whose internal
`reference` sub-extension is populated with `valueReference`, not
`valueAttachment`. Readers who are trying to learn the cross-version-extension
pattern from this example are pointed at the wrong vocabulary, which both
breaks the example's pedagogical value and risks propagating the
misunderstanding into downstream guidance.

### Proposed Change

Edit `source/versions.html`. In the paragraph immediately following the worked
`Immunization` example (the one whose `extension.url` is
`http://hl7.org/fhir/5.0/StructureDefinition/extension-Immunization.administeredProduct`),
replace the single word `valueAttachment` with `administeredProduct`, so the
sentence reads:

```html
<p>
Note that this is an R5 extension, since administeredProduct was introduced in R5.
</p>
```

No other text on the page changes. No anchors, links, examples, or surrounding
markup are touched.

### Design Rationale

- The resolution explicitly dictates the exact replacement string, so the
  change is mechanical.
- `administeredProduct` is the actual name of the R5 extension shown in the
  example and is what the prose is meant to motivate; using it makes the
  sentence describe what the reader just saw.
- `valueAttachment` is unrelated to this example and would only be relevant in
  an example demonstrating a `value[x]` slot newly typed `Attachment` in R5.
  Mentioning it here is simply a copy-edit error.
- A pure HTML prose edit on `source/versions.html` is the correct mechanism in
  this repo; the Version Management Policy page is authored directly in HTML
  and is not generated by the publisher.
