## Resolution Summary

**Resolution:** Persuasive.

The resolution (recorded by Ron Shapiro in the FHIR‑I discussion and copied into the ticket by jmandel on 2023‑09‑14) is that the curating work group declared on each of the following ten specification pages must be changed from *Modeling and Methodology (MnM)* — which is no longer an active HL7 Work Group — to *FHIR Infrastructure (FHIR‑I)*, the work group that inherited MnM's FHIR responsibilities:

- `datatypes.html`
- `datatypes-definitions.html`
- `datatypes-examples.html`
- `datatypes-mappings.html`
- `datatypes-profiles.html`
- `metadatatypes.html`
- `metadatatypes-definitions.html`
- `metadatatypes-examples.html`
- `metadatatypes-mappings.html`
- `metadatatypes-profiles.html`

Concretely, in the bespoke HL7 publisher templating used by `HL7/fhir`, the `[%wg mnm%]` / `[%wgt mnm%]` tokens in the `"Responsible Owner"` table row on each page must be replaced with `[%wg fhir%]` / `[%wgt fhir%]` (the token key for FHIR Infrastructure).

## Feature Proposal

### Problem Statement

As of this ticket (filed 2023‑07‑27) the HL7 Modeling and Methodology (MnM) Work Group was no longer an active HL7 Work Group — its FHIR‑related responsibilities had transitioned to FHIR Infrastructure (FHIR‑I). However, the curating work group displayed on the FHIR Core specification's Data Types and MetaDataTypes pages still referenced MnM. Readers of the specification would therefore be directed to a defunct work group when looking for the owner of these pages, and any process that routes tickets / change requests by the page's declared owner would likewise be misrouted.

### Proposed Change

On each of the ten pages listed under **Resolution Summary**, update the "Responsible Owner" header row so that the curating work group link points at FHIR Infrastructure instead of MnM.

For `source/datatypes.html` (example; all ten pages follow the same pattern):

```diff
- <table class="colsn"><tr><td id="wg"><a _target="blank" href="[%wg mnm%]">[%wgt mnm%]</a> Work Group</td> …
+ <table class="colsn"><tr><td id="wg"><a _target="blank" href="[%wg fhir%]">[%wgt fhir%]</a> Work Group</td> …
```

The replacement is mechanical: swap `mnm` → `fhir` inside both `[%wg …%]` and `[%wgt …%]` tokens on the header row of each page. No other content on these pages changes; maturity level, normative / informative status, and all body content remain as-is.

Additionally, the curating-work-group comment lines at the top of the `[types]` and `[infrastructure]` sections of `source/fhir.ini` (which govern membership of the same data types and metadata types lists) should be updated from `; MnM must approve changes to this list` to `; FHIR-I must approve changes to this list` so the publisher metadata stays consistent with the page headers.

### Design Rationale

- **Single source of truth for work-group attribution.** The publisher token `[%wg fhir%]` / `[%wgt fhir%]` is the canonical pattern used throughout `HL7/fhir` for FHIR‑I–owned pages. Reusing it keeps the Data Types and MetaDataTypes pages consistent with every other FHIR‑I page and lets the publisher render the correct URL/name automatically if the FHIR‑I entry ever moves.
- **No structural change.** The ticket is a technical correction (metadata only). Editing the existing `<table class="colsn"> … <td id="wg"> …` row preserves the page's normative flags, IDs and layout.
- **Consistency with work-group transition.** After MnM was retired, all MnM‑attributed FHIR pages were expected to be re-attributed to FHIR‑I. This ticket covers the Data Types / MetaDataTypes subset of that cleanup; ticket FHIR‑45488 later extended the same rewrite to additional related files (`fhir.ini` comment lines, `codesystem-resource-status.xml`, `timing-extensions-spreadsheet.xml`, `iso-21090-spreadsheet.xml`) so the whole cluster is consistent — see **Related Tickets**.
