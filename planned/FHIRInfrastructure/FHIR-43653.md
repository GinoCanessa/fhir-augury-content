## Feature Proposal

### Problem Statement

In the GraphDefinition "notes" page, under the section "Summarize a set of
profiles on resources", the text reads:

> Here's [an example](http://fhir.hl7.org.au/fhir/rcpa/pprofiles.html) of the
> kind of summary this represents. (Todo: make this an actual graph
> definition, and clone into the main spec)

Two problems:

1. The external URL `http://fhir.hl7.org.au/fhir/rcpa/pprofiles.html` does
   not resolve (broken link to the RCPA AU pathology profiles page).
2. The parenthetical "Todo" has been carried in published normative
   specification text since at least the R4 era and is editorial debt that
   should not appear in a published page.

The reporter (Bas van den Heuvel, FHIR Infrastructure) asks editors to
resolve the TODO and have the link refer to a real example that the spec
itself can guarantee.

Note on repo scope: GraphDefinition has been **removed from HL7/fhir core**
in the R6 normative work (commit `270e0ceb7b`, "normative changes for R6")
and migrated to the **HL7/api-incubator-ig** repository. The notes file
carrying the offending paragraph now lives there at
`input/pagecontent/StructureDefinition-GraphDefinition-notes.xml`. The
ticket was opened against "FHIR Core (FHIR)" before that migration; the
actual edit must be made in the incubator IG.

### Proposed Change

In `HL7/api-incubator-ig`, edit
`input/pagecontent/StructureDefinition-GraphDefinition-notes.xml`,
"Summarize a set of profiles on resources" subsection (lines ~31–35), to:

- Remove the broken external hyperlink to
  `http://fhir.hl7.org.au/fhir/rcpa/pprofiles.html`.
- Remove the parenthetical "(Todo: make this an actual graph definition,
  and clone into the main spec)".
- Replace the sentence with a forward reference to the in-spec
  `GraphDefinition/example` ("Document Generation Template"), which is the
  canonical worked example shipped alongside the resource.

Concrete proposed wording (HTML, suitable for the `.xml` pagecontent
file — the publisher renders it through the IG template):

```xml
<p>
For a worked example of the kind of summary this represents, see the
<a href="GraphDefinition-example.html">Document Generation Template</a>
(<code>GraphDefinition/example</code>) shipped with this specification,
which uses GraphDefinition to describe the set of resources to include
when generating a document with the <code>$document</code> operation.
</p>
```

If editors prefer to keep a pointer to a real-world clinical example
(the original RCPA intent), the parenthetical can additionally cite the
AU pathology IG by canonical URL rather than by deep link to a moved page,
but the broken HTTP link must be removed and the TODO must be deleted in
either case.

### Design Rationale

- **Match what the ticket asks for.** The reporter explicitly asks to
  "resolve the todo and have the link refer to it" — i.e., point at a real,
  in-spec GraphDefinition. The repository already contains exactly such an
  instance (`graphdefinition-example.xml`), so no new artifact is required.
- **Stable in-spec link.** `GraphDefinition-example.html` is generated by
  the IG publisher from `input/resources/graphdefinition/graphdefinition-example.xml`,
  so the link remains valid as long as the example does, with no external
  dependency that can rot.
- **Editorial hygiene.** Published normative pages should not carry
  authoring TODOs. The same notes file contains a second residual TODO at
  line ~165 ("Todo: how would this be validated? - where is the graph
  referred to?") and two TODOs in inline XML comments at lines ~202 and
  ~232. These are out of scope for FHIR-43653 (the ticket targets one
  specific link), but they are flagged in **Open Questions** for the
  workgroup to decide whether to roll into this change.
- **Alternative considered:** finding the new home of the RCPA AU pathology
  pprofiles page and re-pointing the external link there. Rejected because
  (a) it leaves the spec dependent on a third-party page that can move
  again, and (b) it does not satisfy the "have the link refer to [an
  actual graph definition]" intent.
