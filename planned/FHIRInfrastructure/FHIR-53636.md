## 1 Summary

The description text for `Element.id` currently says the value "does not contain spaces." The intent was always "no whitespace" (including tabs, carriage returns, non-breaking spaces, etc.). Because the regex pattern for the `id` type (`[A-Za-z0-9\-\.]{1,64}`) already implicitly prohibits all whitespace, and `Element.id` is typed as a `string` (not `id`), the prose description is the only place that formally communicates the constraint to implementers. This ticket corrects the wording from "spaces" to "whitespace" to make the prohibition unambiguous.

## 2 Feature Proposal

Change every occurrence of the phrase **"does not contain spaces"** in the Element.id definition text to **"does not contain whitespace"** across the FHIR Core specification source.

The `id` type regex `[A-Za-z0-9\-\.]{1,64}` (in `source/datatypes/primitives.xml` line 309) already enforces the no-whitespace constraint at the formal level; this ticket corrects the human-readable description to match that intent.

## 5 Detailed Implementation Plan

### 5.1 Primary change — Element.id definition text

**File:** `source/datatypes/element.xml`
**Line:** 629

**Current text:**
```
Unique id for the element within a resource (for internal references). This may be any string value that does not contain spaces
```

**Proposed text:**
```
Unique id for the element within a resource (for internal references). This may be any string value that does not contain whitespace
```

**Rationale:** This is the canonical definition of `Element.id` in the spreadsheet that drives the spec build. All downstream renderings (StructureDefinition JSON/XML, HTML pages, tooltips) are generated from this text.

### 5.2 Downstream / generated content — resource-notes.xml tooltip text

**File:** `source/resource/resource-notes.xml`
**Lines:** 409, 520

These lines contain pre-rendered HTML table snippets with tooltip text referencing Element.id:

```html
title="Unique id for the element within a resource (for internal references). This may be any string value that does not contain spaces."
```

**Action:** Change `does not contain spaces` → `does not contain whitespace` in both occurrences.

> **Note:** If these HTML tables are auto-generated by the spec build toolchain from the Element definition in `element.xml`, then only §5.1 is needed and these will regenerate automatically. If they are manually maintained, both occurrences must be updated by hand.

### 5.3 Files confirmed as NOT requiring changes

| File | Reason |
|------|--------|
| `source/datatypes/primitives.xml` | The `id` type regex `[A-Za-z0-9\-\.]{1,64}` (line 309) already prohibits all whitespace — no change needed |
| `source/subscriptiontopic/subscriptiontopic-notes.xml` | Contains "no spaces or special characters" in unrelated context (search parameter naming conventions) — not about Element.id |
| `tools/history/release*/*.xml` | Historical snapshots of published releases — must not be modified |
| `source/datatypes.html` | Does not contain the "does not contain spaces" text |
