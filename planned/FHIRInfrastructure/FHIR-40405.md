### Resolution (verbatim)

> We will propose a new Identifier.type code called "USER" in the existing v2 code system and, once accepted, will add it to the FHIR Identifier.type value set.
>
> Display "User Name"
>
> Definition "An identifier for an individual (or occasionally another system) that has access to a system. It is used in authenticating the individual and/or logging their actions. AKA User Id."
>
> We will **not** define a profile on Identifier in the core spec as we don't have any other profiles on identifier, nor do we have a place where we would point to 'user' in the core spec.
>
> (We're open to feedback from v2 if they feel CAAI should be used for this purpose instead.)
