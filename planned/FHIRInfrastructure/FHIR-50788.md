## Resolution Summary

**Resolution: Persuasive (pre-applied).** The reporter's fix has been approved by
Lloyd McKenzie and already merged into the main FHIR branch via
[HL7/fhir#3466](https://github.com/HL7/fhir/pull/3466) (commit
`662f186eb3`). The resolution is to change the UCUM unit code used by the
Questionnaire example `questionnaire-profile-example-ussg-fht` from the
non-existent `[lb_i]` to the valid UCUM code `[lb_av]` (pounds
avoirdupois), so the FHIR publisher no longer crashes during a full build
when validating this example against UCUM.

## Feature Proposal

### Problem Statement

During a full FHIR build, the publisher validates example instances
against the UCUM code system. The example
`questionnaire-profile-example-ussg-fht.xml` declared a weight unit with
UCUM code `[lb_i]`, which is not a valid UCUM code. The publisher could
not resolve the code in the UCUM system and aborted the build with a
validation error, meaning the entire core specification failed to build
because of a single bad example. The reporter verified that substituting
the valid UCUM code `[lb_av]` (pounds avoirdupois) allows the build to
complete successfully.

### Proposed Change

Replace the invalid UCUM unit code `[lb_i]` with `[lb_av]` in the
Questionnaire profile example and any companion example/value-set content
that mirrors it. Specifically:

- **File:** `source/questionnaire/questionnaire-profile-example-ussg-fht.xml`
  - In the inline `Weight Units` contained `ValueSet` (near line 56 of the
    current HEAD — the `<compose>/<include>` block with
    `<system value="http://unitsofmeasure.org"/>`), change the weight
    concept's `<code value="[lb_i]"/>` to `<code value="[lb_av]"/>`.
    Leave the companion `km` (kilograms) concept untouched. No display,
    definition, or cardinality changes — a pure one-character-class fix
    to a coded value.
- **File:** `source/questionnaire/$populate-response.txt`
  - Line 129 currently carries `<code value="[lb_av]"/>`, matching the
    corrected example. Verify it is still consistent (no change needed
    at HEAD; noted here because the file mirrors the same UCUM concept).

No StructureDefinition, search parameter, binding, or documentation
changes are required. This is a data-only correction inside an example
instance.

### Design Rationale

UCUM `[lb_av]` is the canonical code for pounds (avoirdupois) and is
already accepted by the FHIR publisher's UCUM validator; `[lb_i]` is not
a recognised UCUM code. Choosing `[lb_av]` is consistent with the
companion kilogram concept (`km`) used in the same `Weight Units`
ValueSet and with how UCUM-validated weight examples are expressed
elsewhere in core. The fix is the minimum viable change required to
restore a green build and has already been pre-applied on that basis
(Pre-applied comment by Khalid Shahin, approved by Lloyd McKenzie).
