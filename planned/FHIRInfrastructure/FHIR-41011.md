## Resolution Summary

**Resolution:** Persuasive.

The `<comment>` text on the `Resource.language` element in the base `Resource`
StructureDefinition contains a grammatical error — the word "it" appears twice
in the phrase "it should it also be specified". The ticket requests removing
the second, duplicate occurrence of "it" so the sentence reads "…If a
language is specified, **it should also be specified** on the div element in
the html…".

This is a pure prose / technical-correction change to a single element
comment; no cardinality, type, binding, invariant, or search-parameter changes
are implied.

## Feature Proposal

### Problem Statement

The `comment` attribute on the `Resource.language` element definition in
`source/resource/structuredefinition-Resource.xml` (line 114) reads:

> "…If a language is specified, it should it also be specified on the div
> element in the html (see rules in HTML5 for information about the
> relationship between xml:lang and the html lang attribute)."

The clause "it should it also be specified" contains a duplicated "it" and is
ungrammatical. Because this text is surfaced as the standard comment on
`Resource.language` — which every resource inherits — the error appears
throughout the published specification wherever the `Resource.language`
element comment is rendered (resource pages, differential/snapshot tables,
`profiles-resources.xml` publisher output, etc.).

### Proposed Change

Edit the `Resource.language` element's `<comment value="…"/>` in
`source/resource/structuredefinition-Resource.xml` (inside the `<differential>`
block, line 114) to remove the duplicate "it".

- **Before:** `…If a language is specified, it should it also be specified on the div element in the html…`
- **After:**  `…If a language is specified, it should also be specified on the div element in the html…`

No other part of the comment changes. No other element, binding, cardinality,
or invariant is affected.

### Design Rationale

- The resolution type is **Persuasive** and the ticket is classified as a
  **Technical Correction** with priority **Lowest** — i.e., an unambiguous
  editorial / grammar fix with no design debate.
- Per the HL7/fhir briefing, only `<differential>` content in
  StructureDefinitions is hand-edited; the corresponding `<snapshot>`
  content is regenerated by the publisher. The broken sentence lives in
  the `<differential>` of the base `Resource` StructureDefinition, so a
  single-line edit there propagates correctly to all published artifacts
  on the next publisher run.
- No alternative was considered because the ticket description is
  explicit: "Recommend updating to remove the second instance of the
  word 'it'."
