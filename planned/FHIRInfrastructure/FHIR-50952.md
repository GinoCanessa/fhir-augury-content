### Description

The definition of the FMM extension (`structuredefinition-fmm`) says it is only allowed on `DomainResource`, but it should be allowed in many other places (e.g., `ImplementationGuide.definition.page`), so its context should be changed to `Element`.

### Resolution Description (Persuasive with Modification)

The resolution has four distinct parts:

1. **Extension change (HL7/fhir-extensions):** Add a link to the [FHIR Maturity Model Confluence Page](https://confluence.hl7.org/spaces/FHIR/pages/35718679/FHIR+Maturity+Model) on the `structuredefinition-fmm` extension definition. The context must also be broadened from `DomainResource`/`Resource` to `Element`.

2. **Confluence documentation (external — FMG action):** Ask FMG to update the FHIR Maturity Model Confluence Page with:
   - A note about use of FMM level on Elements with a brief description
   - A note that element FMM levels *cannot* exceed the FMM of the resource they are defined in
   - An example of why an element FMM level is appropriate (e.g., FMM 0 label on a Trial-Use element in a more mature resource) plus description of intent
   - A note that use of FMM level in other contexts (e.g., on a `code` in a `CodeSystem`) is currently undefined

3. **FHIR spec versions page (HL7/fhir):** Per FHIR-40463, the text on the FHIR Versions page regarding maturity will be updated from the Confluence page content. When doing the next update, the "frozen snapshot" paragraph should be moved to the **top** of the maturity section (above the actual FMM level definitions) so readers are directed to the current Confluence content first. If an appropriate style exists (e.g., `reader-note`), the paragraph should be styled as such.

### Originating Zulip Discussion

In [IG creation > How to give IG pages an FMM?](https://chat.fhir.org/#narrow/stream/IG%20creation/topic/How%20to%20give%20IG%20pages%20an%20FMM%3F), Isaac Vetter reported an IG Publisher error when applying the `structuredefinition-fmm` extension to `ImplementationGuide.definition.page` elements. The IG Publisher rejected it because the extension only allowed `DomainResource` and `Resource` as contexts. Grahame Grieve confirmed the extension definition was wrong and filed this ticket.

---

## Feature Proposal

Broaden the allowed context of the `structuredefinition-fmm` extension from `DomainResource`/`Resource` to `Element`, update its description to link to the authoritative FHIR Maturity Model Confluence page, and reposition informational text on the FHIR spec versions page so that readers encounter the "living document" notice before the (potentially stale) inline FMM level definitions.

### Risk Assessment

- **Low risk.** The context broadening is additive — all existing usages on `DomainResource` and `Resource` are still valid because `Element` is their ancestor type. No existing IGs or implementations will break.
- The versions page change is purely editorial text reordering.
- The companion extension `structuredefinition-fmm-support` (currently context `DomainResource` only) is **not** explicitly mentioned in the resolution, but may warrant a parallel change in a future ticket.

### Backward Compatibility

- **Fully backward compatible.** Widening context from specific types to their parent type (`Element`) means all existing uses remain valid. New uses on non-resource elements become valid.

## Implementation Plan

### Task 1: Broaden FMM Extension Context (HL7/fhir-extensions)

**File:** `input/definitions/Element/StructureDefinition-structuredefinition-fmm.xml`

#### 1a. Replace context entries with a single `Element` context

**Current (lines 41–48):**
```xml
  <context>
    <type value="element"/>
    <expression value="DomainResource"/>
  </context>
  <context>
    <type value="element"/>
    <expression value="Resource"/>
  </context>
```

**Proposed:**
```xml
  <context>
    <type value="element"/>
    <expression value="Element"/>
  </context>
```

**Rationale:** This is the core change requested by the ticket. Replacing the two specific context entries (`DomainResource`, `Resource`) with a single `Element` context allows the extension to be used on any element in any resource — including `ImplementationGuide.definition.page`, data type elements, backbone elements, etc.

#### 1b. Update the extension description to include Confluence link

**Current (line 32):**
```xml
  <description value="The FMM level assigned to the artifact."/>
```

**Proposed:**
```xml
  <description value="The FMM level assigned to the artifact. See the [FHIR Maturity Model](https://confluence.hl7.org/spaces/FHIR/pages/35718679/FHIR+Maturity+Model) for further details."/>
```

**Rationale:** The resolution explicitly says to add a link to the FHIR Maturity Model Confluence Page on the extension.

#### 1c. Update the comment on the Extension element

**Current (line 57):**
```xml
      <comment value="Though this is defined for resources, it can be used for any artifact."/>
```

**Proposed:**
```xml
      <comment value="This extension can be used on any element. When used on individual elements within a resource, the FMM level cannot exceed that of the containing resource. See the [FHIR Maturity Model](https://confluence.hl7.org/spaces/FHIR/pages/35718679/FHIR+Maturity+Model) for details on element-level maturity."/>
```

**Rationale:** The old comment referenced the previous (resources-only) constraint. The new comment reflects the broadened context and the resolution's guidance that element FMM levels cannot exceed their containing resource's FMM.

### Task 2: Reposition "Frozen Snapshot" Paragraph (HL7/fhir)

**File:** `source/versions.html`

#### 2a. Move the "frozen snapshot" paragraph to the top of the maturity section

The paragraph currently at lines 159–161 (after the FMM levels table and "tested across scope" list) should be moved to immediately after the introductory paragraph (after line 135), before the FMM level definitions begin.

**Current structure (simplified):**
```
<h3>The FHIR Maturity Model</h3>
<p>The content of this release has been subject to significant review...</p>
<p>All artifacts in this specification are assigned a "Maturity Level"...</p>
<table>...FMM 0 through FMM 6...</table>
<p>Tested across scope means:...</p>
<p>The Maturity level is strongly related to stability...</p>
<p>The table above represents a frozen snapshot...</p>    ← MOVE THIS UP
<p>The maturity model is significantly influenced...</p>
```

**Proposed structure:**
```
<h3>The FHIR Maturity Model</h3>
<p>The content of this release has been subject to significant review...</p>
<p class="reader-note">The content below represents a frozen snapshot of the maturity levels maintained by the FHIR Management Group on the <a href="...">FHIR Confluence page</a>. Further information and discussion about these levels can be found there.</p>    ← MOVED HERE
<p>All artifacts in this specification are assigned a "Maturity Level"...</p>
<table>...FMM 0 through FMM 6...</table>
<p>Tested across scope means:...</p>
<p>The Maturity level is strongly related to stability...</p>
<p>The maturity model is significantly influenced...</p>
```

**Specific edits:**

1. **Remove** lines 159–161 (the paragraph containing "The table above represents a frozen snapshot...").
2. **Insert** a new paragraph after line 135 (after the intro paragraph ending "Guidance from early implementation will help address these areas.") with the relocated text. Adjust wording from "The table above" to "The content below" since it now precedes the table. Apply `reader-note` class if available in the FHIR build tooling, otherwise use a `<blockquote>` or `<div class="note">` pattern consistent with the rest of the spec.

**Rationale:** The resolution explicitly says this paragraph "should be moved to the top of that section (above the actual contents) so that readers can be directed to the current content first" and styled with an appropriate style (e.g., `reader-note`).

---

### Task 3: Confluence Page Updates (External — FMG Action Item)

> **Note:** This task is NOT a code change. It requires FMG to update the [FHIR Maturity Model Confluence Page](https://confluence.hl7.org/spaces/FHIR/pages/35718679/FHIR+Maturity+Model).

The following content should be added to the Confluence page:

1. A note about use of FMM level on Elements with a brief description of what this means
2. A rule that element FMM levels *cannot* exceed the FMM of the resource they are defined in
3. An example: e.g., an FMM 0 label on a Trial-Use element within a more mature resource, plus a description of intent (to show something is *less mature* than its containing definition)
4. A note that use of FMM level in other contexts (e.g., on a `code` in a `CodeSystem`) is currently undefined

**Action:** Open a request to FMG or file a follow-up ticket to track this Confluence documentation update.
