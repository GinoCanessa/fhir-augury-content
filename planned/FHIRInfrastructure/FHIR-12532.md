## Resolution Summary

The resolution is **Persuasive** (auto-approved), meaning all issues raised in the ticket are accepted as valid and should be corrected. The ticket identifies multiple typos and broken/incorrect links on the DAF Research Query Responder CapabilityStatement narrative page (`CapabilityStatement-daf-research-queryresponder.html`), including non-functional link anchors, missing whitespace, and search parameter type links pointing to incorrect targets.

## Feature Proposal

### Problem Statement

The DAF Research Query Responder CapabilityStatement page (rendered from `resources/capabilitystatement-daf-research-queryresponder.xml`) contains several documentation errors in its narrative HTML (`<text>` element):

1. **Non-functional "SHALL" links in RESTful Operations Summary table:** The "SHALL" text in the Search column for DAF-OperationDefinition and DAF-QueryResults rows is marked up as a link but does not have a valid `href` attribute — it renders as styled text with no destination.
2. **Invalid anchor links for DAF-Task and DAF-Provenance SHALLs:** The "SHALL" links in the DAF-Task and DAF-Provenance rows point to `#DAFTaskSearch` and `#DAFProvenanceSearch` respectively. While these IDs now exist in the search parameters table, the original ballot version had them pointing to nonexistent anchors.
3. **Missing space between "FHIR" and "specification":** In the Data Mart Search Parameters section, the sentence reads "...within the FHIRspecification" — missing a space.
4. **Incorrect "FHIR" link target:** The "FHIR" hyperlink in that sentence should link to `search.html#2.44.1.2` (the search contexts section) rather than `search.html#Introduction`.
5. **Incorrect search parameter type links:** In the search parameters table, the "reference" type links for DAF-Task (`part-of`, `owner`, `requester`) and DAF-Provenance (`target`) incorrectly pointed to `search.html#token` instead of `search.html#reference`.
6. **Incorrect "date" type link:** The "date" type link for DAF-Provenance search parameters (`start`, `end`) incorrectly pointed to `search.html#token` instead of `search.html#date`.
7. **Incorrect "Modifiers" link:** The "Modifiers" link was case-sensitive and pointed to the wrong anchor.
8. **Incorrect "Composite Search Parameters" link:** The "Composite Search Parameters" link pointed to the wrong anchor.

### Proposed Change

Fix all typos and broken/incorrect links in the narrative HTML of the CapabilityStatement resource at `resources/capabilitystatement-daf-research-queryresponder.xml`. The specific changes are:

1. **DAF-OperationDefinition and DAF-QueryResults Search column SHALLs:** Either remove the link styling (if no search parameters section exists for these resources) or link to a valid anchor. Since there are no search parameters defined for OperationDefinition or QueryResults, the "SHALL" text should be plain text (`<p>` tag), consistent with how the other columns (Read, Version Read, etc.) are marked up.
2. **FHIR/specification space:** Ensure proper whitespace between the closing `</a>` tag for "FHIR" and the word "specification".
3. **FHIR link target:** Change the `href` from `search.html#introduction` to `search.html#2.44.1.2` (or the current equivalent anchor for search contexts).
4. **Modifiers link:** Ensure it links to `search.html#modifiers` (lowercase).
5. **Composite Search Parameters link:** Ensure it links to `search.html#composite`.
6. **Reference type links:** Ensure all "reference" search parameter type links point to `search.html#reference`.
7. **Date type links:** Ensure all "date" search parameter type links point to `search.html#date`.

### Design Rationale

These are straightforward documentation corrections (typos and broken links). The resolution is "Persuasive" meaning the issues are accepted as stated. The fixes improve the usability and accuracy of the CapabilityStatement narrative, allowing implementers to navigate correctly to the referenced FHIR specification sections. No behavioral or structural changes to the IG are required.
