## Resolution Summary

Resolution: **Persuasive**. The ticket reports a typo in the "Simple form"
paragraph of the FHIR Mapping Language page
(`https://hl7.org/fhir/mapping-language.html#simple`). The current sentence
reads:

> "If no dependent rules are specified, and if **the** is only one source and
> target, and they both specify a variable, the rule can be written in an
> abbreviated form:"

The word **"the"** must be corrected to **"there"** so the sentence reads:

> "If no dependent rules are specified, and if **there** is only one source
> and target, and they both specify a variable, the rule can be written in
> an abbreviated form:"

No other content changes are requested. There are no attached applied-vote
comments on the Jira ticket, and no cross-references (GitHub PRs, other
tickets, or Zulip threads) were found for this ticket.

## Feature Proposal

### Problem Statement

The narrative text introducing the abbreviated ("simple") form of a mapping
rule on the FHIR Mapping Language specification page contains a grammatical
typo: the subject "there" was written as "the", producing an ungrammatical
sentence. This is purely an editorial defect and does not affect the
normative definition of the simple form syntax — but it is confusing to
readers first encountering the FML abbreviated rule form.

### Proposed Change

Edit the FHIR Mapping Language HTML page to change a single word in the
paragraph immediately preceding the simple-form example:

- **File:** `source/mapping-language.html` (HL7/fhir)
- **Location:** line 862 (paragraph starting "If no dependent rules are
  specified…", anchored from `#simple`).
- **Before:**
  ```html
  <p>
  If no dependent rules are specified, and if the
  is only one source and target, and
  they both specify a variable, the rule can be written in an abbreviated form:
  </p>
  ```
- **After:**
  ```html
  <p>
  If no dependent rules are specified, and if there
  is only one source and target, and
  they both specify a variable, the rule can be written in an abbreviated form:
  </p>
  ```

No other edits are required. The surrounding examples, grammar table,
anchors, and section ordering are left untouched.

### Design Rationale

This is a one-word editorial correction to the reporter's literal request,
which was accepted as "Persuasive" by FHIR Infrastructure. No design trade-off
is involved:

- It restores the intended English grammar of the sentence.
- It does not alter the abbreviated-rule syntax, grammar, examples, or any
  normative FML behaviour.
- It preserves the `#simple` anchor and section structure used by inbound
  cross-references from elsewhere in the spec.

No alternative approaches were considered because the resolution specifies
the exact textual change.
