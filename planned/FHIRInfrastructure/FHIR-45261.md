## Resolution Summary

Resolution: **Persuasive**. The ticket reports a typo on the FHIR R5
`operations.html#executing` specification page. The text reads
"…containing **ne or more** output parameters", and should read
"…containing **one or more** output parameters". The fix is a single
character correction in the narrative prose.

## Feature Proposal

### Problem Statement

The R5 specification page describing how to execute an operation contains
a typo in the paragraph that explains the response shape. The published
sentence currently says:

> Upon completion, the operation returns an HTTP status code indicating
> what the outcome of performing the operation, and optionally another
> Parameters resource, containing **ne or more** output parameters.

The "ne or more" is a clear misspelling of "one or more". It does not
change normative behavior, but it is visible on the published spec page
and harms readability.

### Proposed Change

In the FHIR Core authoring source `source/operations.html`, update the
"Executing an Operation" paragraph so that the word `ne` becomes `one`.
Concretely (current clone HEAD `db79dcd`):

- File: `source/operations.html`
- Line 87: change

  ```html
  performing the operation, and optionally another <code>Parameters</code> resource, containing
  ne or more output parameters. If there is a single output parameter named 'return' with
  ```

  to

  ```html
  performing the operation, and optionally another <code>Parameters</code> resource, containing
  one or more output parameters. If there is a single output parameter named 'return' with
  ```

No other elements, structures, examples, terminology bindings, or search
parameters are affected.

### Design Rationale

This is a Technical Correction with resolution `Persuasive` — the change
is editorial and the fix follows the reporter's wording verbatim. There
is no design decision to make; "one or more" is the only sensible reading
in context (the same paragraph immediately discusses the case of "a
single output parameter" and "zero output parameters"). No alternative
phrasings need to be considered.
