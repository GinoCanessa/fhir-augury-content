## Resolution Snapshot (for context only — not a substitute for the plan)

The resolution table on the Jira ticket (vote outcome: **Persuasive**)
proposes the following changes to CodeSystem `v2-0203` and to the
ValueSet `identifier-type` that binds it on `Identifier.type`:

| Code | Display | Definition (resolved) |
|------|---------|------------------------|
| `UDI` | Universal Device Identifier | An identifier assigned to a device using the Unique Device Identification framework as defined by IMDRF (<http://imdrf.org>). **This value is deprecated and Distribution Identifier should be used instead.** |
| `DISTID` *(new)* | Distribution Identifier | An identifier appearing on the packaging of a marketed product used to track it through its lifecycle (e.g., transport, inventory, use, and disposal). |
| `DIRECT-MARKING` *(new)* | Identifier directly marked on the identified item | An identifier physically marked on the item that it identifies. |
| `SINGLE-ITEM` *(new)* | Identifier for any single item of a homogeneous collection | An identifier pre-defined by the assigner used for identifying an individual item in a homogeneous collection. An example would be a box of syringes where a SINGLE-ITEM identifier would refer to any arbitrary single syringe in the box. Another example of use of this value is the "Unit of use DI" as defined in the regulatory medical device databases (e.g. GUDID for US and EUDAMED for EU). |

The vote also blocks on the linked UTG proposal **UP-232**, which is the
mechanism by which the new codes will actually be added to the v2-0203
CodeSystem in `HL7/UTG`. Any plan must distinguish UTG-side
CodeSystem/ValueSet edits from any FHIR-core mirror/binding updates,
and the UTG briefing is required to name those paths accurately.
