## Feature Proposal

### Goal

Ensure that the SDC Questionnaire profile's CapabilityStatement and/or search conformance expectations include `name` and `publisher` as supported search parameters, enabling implementers to search for SDC Questionnaires by their computer-friendly name and by publisher identity.

### Background

The base FHIR Questionnaire resource already defines both `name` and `publisher` as standard search parameters:

- **`name`** — type `string`, expression `Questionnaire.name` ("Computationally friendly name of the questionnaire")
- **`publisher`** — type `string`, expression `Questionnaire.publisher` ("Name of the publisher of the questionnaire")

These exist in the base spec at `source/questionnaire/bundle-Questionnaire-search-params.xml` (lines 327–366) in **HL7/fhir**. The ticket's request is specifically about making these search parameters required or expected within the SDC Questionnaire conformance artifacts (CapabilityStatement / search profile), not about creating new SearchParameter definitions.

## Impact Analysis

### Affected Repositories

> **REPO_NOT_CACHED: HL7/sdc**
> **REPO_NOT_CACHED: HL7/patient-reported-outcomes**

The primary implementation belongs in the **SDC IG** (`HL7/sdc`, `https://github.com/HL7/sdc`) and potentially the **PRO IG** (`HL7/patient-reported-outcomes`, `https://github.com/HL7/patient-reported-outcomes`). Neither repository is available in the local cache.

### HL7/fhir (base spec) — No changes needed

The base Questionnaire resource already defines both `name` and `publisher` search parameters in:

- **File**: `source/questionnaire/bundle-Questionnaire-search-params.xml`
  - `Questionnaire-name` (lines 327–345): type `string`, expression `Questionnaire.name`
  - `Questionnaire-publisher` (lines 348–366): type `string`, expression `Questionnaire.publisher`

No new search parameter definitions are required in the base spec.

### HL7/sdc (SDC IG) — Changes required ⚠️

The SDC IG is the primary target. Based on the ticket and related FHIR-49026 (which was subsequently Applied), the following types of changes are expected:

1. **CapabilityStatement**: The SDC IG's CapabilityStatement resource(s) for Questionnaire need to list `name` and `publisher` as supported search parameters with appropriate expectation levels (e.g., `SHOULD` or `SHALL`).
   - Likely file pattern: `input/resources/CapabilityStatement-*.json` or `input/resources/CapabilityStatement-*.xml`
   - The `rest.resource` entry for `Questionnaire` should include `searchParam` entries referencing:
     - `http://hl7.org/fhir/SearchParameter/Questionnaire-name`
     - `http://hl7.org/fhir/SearchParameter/Questionnaire-publisher`

2. **Search Questionnaire Profile**: The SDC "Search Questionnaire" profile (`sdc-questionnaire-adapt-srch` or similar) may need updates to its conformance documentation to note these search parameters.

3. **Documentation/Narrative**: Any IG pages discussing Questionnaire search capabilities should list `name` and `publisher` among the expected search parameters.

### HL7/patient-reported-outcomes (PRO IG) — Potentially affected

The ticket was filed against the PRO specification. If the PRO IG defines its own CapabilityStatement or conformance expectations for Questionnaire, those would also need `name` and `publisher` search parameter entries. However, the PRO IG appears to be a lightweight overlay (version 0.1.0, dated 2018Sep) and may not have extensive conformance artifacts.

## Implementation Plan

### Prerequisite

Clone and inspect the `HL7/sdc` repository to identify the exact files requiring changes. The following plan is based on standard SDC IG structure patterns and the related ticket FHIR-49026.

### Step 1: Update SDC CapabilityStatement(s)

**Target**: SDC IG CapabilityStatement resource(s) in `HL7/sdc`

Locate the CapabilityStatement resource(s) that define Questionnaire interactions (likely under `input/resources/`). Add `name` and `publisher` to the `searchParam` list for the Questionnaire resource entry:

```json
{
  "name": "name",
  "definition": "http://hl7.org/fhir/SearchParameter/Questionnaire-name",
  "type": "string",
  "documentation": "Computationally friendly name of the questionnaire"
},
{
  "name": "publisher",
  "definition": "http://hl7.org/fhir/SearchParameter/Questionnaire-publisher",
  "type": "string",
  "documentation": "Name of the publisher of the questionnaire"
}
```

The expectation level should be determined by the work group. Given the related FHIR-22292 context (which lists `name` and `publisher` alongside other critical search parameters), `SHOULD` is a reasonable default.

### Step 2: Update SDC Search Documentation

**Target**: IG narrative pages discussing search parameters in `HL7/sdc`

Update any pages or sections documenting Questionnaire search capabilities to include `name` and `publisher` in the list of supported search parameters. This may include:
- The SDC capability/conformance page
- The search-related profile documentation

### Step 3: Verify PRO IG impact

**Target**: `HL7/patient-reported-outcomes`

Check if the PRO IG has its own CapabilityStatement or conformance artifacts for Questionnaire. If so, add the same `name` and `publisher` search parameter expectations. Given the PRO IG's age (0.1.0 from 2018), this may be deferred if the IG is no longer actively maintained.

### Step 4: Coordinate with FHIR-49026

The related ticket FHIR-49026 ("SearchParameter expectations need work") has already been Applied with a Persuasive with Modification resolution. That ticket addresses broader SDC search parameter expectation issues, including listing standard search parameters in the CapabilityStatement and reviewing expectation levels. Implementation of FHIR-22089 should be coordinated with or verified against the changes made for FHIR-49026 to avoid duplication.
