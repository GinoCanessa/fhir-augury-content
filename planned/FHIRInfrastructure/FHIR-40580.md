## Problem Statement

In section 12.4.5 "Pattern Content" of the FHIR Core specification, the `deliverTo` element's type includes a reference to `HealthCareService` (uppercase "C" in "Care"). The correct resource name is `HealthcareService` (lowercase "c"), as defined in the [HealthcareService resource definition](http://build.fhir.org/healthcareservice.html). This typo also appeared in binding descriptions within the HealthcareService StructureDefinition and associated ValueSet.

## Resolution Description

The resolution was **Persuasive**: this is clearly a misspelling of the intended resource name. A PR was submitted and merged to correct all three occurrences.

## Feature Proposal

Correct all instances of the misspelling `HealthCareService` → `HealthcareService` in FHIR Core source files where the resource name is used in reference types and binding descriptions.
