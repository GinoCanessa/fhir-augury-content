## Resolution Summary

The reporter identified that the Request pattern's `deliverTo` element listed
`HealthCareService` (capital "C") among its allowed reference target types,
while the actual resource is named `HealthcareService` (lower-case "c"). The
work group accepted the change as **Persuasive**, with the resolution
description: *"This is clearly a misspelling of the intended resource name.
PR https://github.com/HL7/fhir/pull/2811 fixes it."*

Concretely, every occurrence of the literal string `HealthCareService` in the
core spec must be corrected to `HealthcareService`. PR #2811 (already merged
into HL7/fhir master) is the authoritative implementation of this fix and
modifies three files (see Implementation Plan below).

## Feature Proposal

### Problem Statement

`HealthcareService` is the canonical resource name throughout FHIR Core (the
file `source/healthcareservice/structuredefinition-HealthcareService.xml`
defines `<id value="HealthcareService"/>`, `<name value="HealthcareService"/>`,
`<type value="HealthcareService"/>`, etc.). However, in section 12.4.5
("Pattern Content") of the Request pattern the type list of the `deliverTo`
element used the misspelt form `HealthCareService`. Two prose `description`
strings on the HealthcareService resource itself also used the same incorrect
casing. Because FHIR resource names are case-sensitive, the misspelt form is
not a valid reference target and confuses readers and tooling that scrape
type lists from the published spec.

### Proposed Change

Replace the string `HealthCareService` with `HealthcareService` in every
location where it currently appears in the core spec. Three files are
affected (verified against the cached clone @ `db79dcd`, where the fix is
already present — re-creating the change conceptually):

1. **Request pattern spreadsheet** — `source/request/request-spreadsheet.xml`,
   the row defining `deliverTo`. Change the cell:
   ```
   Reference(Patient|RelatedPerson|Practitioner|PractitionerRole|
             Organization|Location|HealthCareService|CareTeam|Device|Endpoint)
   ```
   to:
   ```
   Reference(Patient|RelatedPerson|Practitioner|PractitionerRole|
             Organization|Location|HealthcareService|CareTeam|Device|Endpoint)
   ```

2. **HealthcareService StructureDefinition** —
   `source/healthcareservice/structuredefinition-HealthcareService.xml`,
   the `referralMethod` binding description (currently around line 573).
   Change:
   `"The methods of referral can be used when referring to a specific HealthCareService resource."`
   to:
   `"The methods of referral can be used when referring to a specific HealthcareService resource."`

3. **service-referral-method ValueSet** —
   `source/healthcareservice/valueset-service-referral-method.xml`,
   the top-level `<description>` element (line 46). Make the same casing
   correction as above.

No element names, paths, cardinalities, types, bindings, or invariants are
changed. This is a pure casing/typo fix to a reference type list and two
descriptive strings.

### Design Rationale

The `HealthcareService` resource has been published with that exact casing
since DSTU2; the `HealthCareService` form is not a valid FHIR resource name
and never resolves to anything. There is no ambiguity in the resolution and
no design alternative — the only correct action is to align the typo with
the canonical name. The work group reviewed and approved PR #2811 as
implementing the fix, marking the disposition Persuasive.

## Implementation Plan

### Prerequisites

None. This is a self-contained typo fix with no dependencies on other

### Step-by-Step Tasks

#### HL7/fhir

1. **Fix `deliverTo` reference target list in the Request pattern spreadsheet**
   - File: `source/request/request-spreadsheet.xml`
   - Action: In the `<Data ss:Type="String">` cell that lists the
     `deliverTo` element's target types, replace `HealthCareService`
     with `HealthcareService`. PR #2811's diff showed this as the cell
     containing
     `Reference(Patient|RelatedPerson|Practitioner|PractitionerRole|Organization|Location|HealthCareService|CareTeam|Device|Endpoint)`.
   - Details: Edit only the textual content of the `<Data>` element.
     Preserve all surrounding SpreadsheetML markup (`<Cell>`,
     `<NamedCell>`, `ss:StyleID`, etc.) byte-for-byte. Do not introduce
     a trailing newline if the original file lacks one (PR #2811's diff
     shows the file had no trailing newline before the fix).

2. **Fix `referralMethod` binding description in HealthcareService StructureDefinition**
   - File: `source/healthcareservice/structuredefinition-HealthcareService.xml`
   - Action: In the `<element id="HealthcareService.referralMethod">`
     block's `<binding>`, change
     `<description value="The methods of referral can be used when referring to a specific HealthCareService resource."/>`
     to
     `<description value="The methods of referral can be used when referring to a specific HealthcareService resource."/>`
   - Details: This change is inside `<differential>` (the
     hand-authored portion); do not touch `<snapshot>`, which is
     regenerated by the publisher. Keep `<strength value="example"/>`
     and the bound `<valueSet>` URL unchanged.

3. **Fix top-level description on the service-referral-method ValueSet**
   - File: `source/healthcareservice/valueset-service-referral-method.xml`
   - Action: Replace the top-level
     `<description value="The methods of referral can be used when referring to a specific HealthCareService resource."/>`
     with
     `<description value="The methods of referral can be used when referring to a specific HealthcareService resource."/>`
   - Details: One-line text change. Do not modify `<contact>`,
     `<immutable>`, `<compose>`, or any other element. Preserve file's
     existing trailing-newline convention.

4. **Repo-wide sanity sweep for stragglers**
   - File: _N/A — verification step_
   - Action: Run `git grep -n HealthCareService` (or equivalent
     ripgrep/PowerShell `Select-String`) under `source/` to confirm no
     other file still contains the misspelt form. If any matches surface
     outside the three files above, evaluate each case on the same
     principle (any reference to the FHIR resource type must use
     `HealthcareService`) and apply the same correction.
   - Details: PR #2811 only touched the three files listed; the cached
     clone @ `db79dcd` confirms zero remaining occurrences of
     `HealthCareService`. New stragglers could only appear if other
     authoring changes since PR #2811 reintroduced the typo, which is
