## Feature Proposal

### Problem Statement

The FHIR Core *Profiling FHIR* page (`source/profiling.html`) contains a
typographical error in the *"Profiling descriptive elements"* section.
The narrative reads:

> These elements can be refined to reflect constraints applied
> **elsehere** for the element and/or to reflect the context of use of
> the element within the profile.

`elsehere` is not an English word; the intended word is `elsewhere`.
The error is purely cosmetic — it has no semantic effect on the
specification — but it appears on a normative-looking prose page of the
core spec, so it warrants correction to preserve readability and the
spec's professional polish. There is no indication in the ticket
discussion that anything beyond the spelling fix was requested.

### Proposed Change

Single-character spelling correction in one HTML file in `HL7/fhir`:

- **File:** `source/profiling.html`
- **Section:** *"Profiling descriptive elements"* (anchor
  `<a name="descriptive"></a>`, `<h3>Profiling descriptive elements</h3>`)
- **Before (one occurrence):**
  > `…constraints applied elsehere for the element and/or…`
- **After:**
  > `…constraints applied elsewhere for the element and/or…`

Diff (matches PR #3457 exactly):

```diff
@@
-Some properties of an element are purely descriptive and aren't used as part of the validation process.  These elements can be refined to reflect constraints applied elsehere for the element and/or to reflect the context of use of the element within the profile.  Changes need to be made cautiously however.  The meaning and guidance provided in the base resource or profile can't be invalidated, only constrained or contextualized.
+Some properties of an element are purely descriptive and aren't used as part of the validation process.  These elements can be refined to reflect constraints applied elsewhere for the element and/or to reflect the context of use of the element within the profile.  Changes need to be made cautiously however.  The meaning and guidance provided in the base resource or profile can't be invalidated, only constrained or contextualized.
```

No other files, no schema changes, no terminology changes, no new
elements, no behavioural changes.

### Design Rationale

Pure typo correction. There is no design decision to make: `elsewhere`
is unambiguously the intended word in context (the sentence is talking
about constraints that apply somewhere other than within the profile
itself). The change preserves all surrounding sentence structure,
punctuation, anchors, and surrounding paragraphs. The single in-clone
occurrence of the misspelling lets us correct it without any wider
sweep.
