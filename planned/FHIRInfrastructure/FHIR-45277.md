## Resolution Summary

**Resolution:** Persuasive.

The Operations specification page (`operations.html`) contains a typo in the
sentence describing the response of an operation invocation: the word "ne"
appears where "one" is intended. The approved correction is to change the
phrase so it reads "containing **one** or more output parameters" (currently
"containing **ne** or more output parameters"). This is a pure
documentation/typo fix — no normative behavior changes.

## Feature Proposal

### Problem Statement

On the FHIR Operations specification page
([build.fhir.org/operations.html](https://build.fhir.org/operations.html)),
the paragraph describing the response of an operation invocation reads:

> Upon completion, the operation returns an HTTP status code indicating what
> the outcome of performing the operation, and optionally another `Parameters`
> resource, containing **ne** or more output parameters.

The bolded "ne" is a typo — the intended word is "one". The error is
preserved in the authored source `source/operations.html` (line 87 of the
current `HL7/fhir` master clone), so it surfaces in every published build of
the FHIR Core specification. The reporter (Rik Smithies) flagged it as a
Technical Correction and the FHIR Infrastructure work group resolved it
Persuasive.

### Proposed Change

Edit the single occurrence in `source/operations.html` at line 87:

- **Before:** `ne or more output parameters.`
- **After:** `one or more output parameters.`

Full sentence after the change:

> Upon completion, the operation returns an HTTP status code indicating what
> the outcome of performing the operation, and optionally another
> `<code>Parameters</code>` resource, containing **one** or more output
> parameters.

No other text on the page needs to change. No other files in the repo
contain the same typo in this context (the surrounding sentence is unique to
`source/operations.html`).

### Design Rationale

Trivial typographical correction. The intended word is unambiguous from
context ("ne" is not a word; the surrounding clause "containing _ or more
output parameters" requires a positive integer quantifier and "one" is the
natural completion that mirrors the immediately following sentence which
talks about a "single output parameter"). No alternative wording was
discussed in the ticket — the resolution simply accepts the reporter's
correction.
