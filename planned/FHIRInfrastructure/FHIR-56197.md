## Halted: Required Repo Briefing Missing

The resolution for FHIR-56197 is **Persuasive with Modification** (rename the
proposed extension name from `allowsDescendants` → `allowDescendants`). The
implementation requires changes in **two** in-scope repositories:

1. **HL7/fhir** (FhirCore) — additional documentation on
   `StructureDefinition.context` / `StructureDefinition.context.type`
   describing how the `expression` is evaluated for each `type` value
   (`fhirpath`, `element`, `extension`), and clarifying that for `element`
   and `extension` the new definition only targets the root element.
2. **HL7/fhir-extensions** (FhirExtensionsPack) — define a **new core
   extension** on `StructureDefinition.context.type`:
   - Name: `allowDescendants` (boolean, 0..1)
   - Whether the context expression includes all descendants of the
     nominated expression (only valid when `type` = `element` or
     `extension`).
