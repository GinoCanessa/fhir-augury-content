## Resolution Summary

Resolution is **Persuasive**. The Questionnaire resource notes contain an external hyperlink to the "Australian New South Wales blue book" PDF
(`http://www.health.nsw.gov.au/Kids/Publications/my-personal-health-record.pdf`) that no longer resolves. The reporter proposes replacing the
broken link with the current NSW Health landing page for the Child Personal Health Record (Blue Book):
`https://www.health.nsw.gov.au/kidsfamilies/MCFhealth/Pages/child-blue-book.aspx`. No other text or example data needs to change.

## Feature Proposal

### Problem Statement

The Questionnaire resource notes page (rendered in the spec from `source/questionnaire/questionnaire-notes.xml`) cites the NSW "blue book"
as the real-world inspiration for the bundled `questionnaire-example-bluebook` example. The cited URL has rotted: the legacy
`/Kids/Publications/my-personal-health-record.pdf` path under `health.nsw.gov.au` is no longer served, leaving readers of the FHIR
specification with a broken external citation. The link is in the prose only — neither the example resource itself nor any conformance
artifact is affected.

### Proposed Change

Replace the single anchor `href` in `source/questionnaire/questionnaire-notes.xml`:

- **From:** `http://www.health.nsw.gov.au/Kids/Publications/my-personal-health-record.pdf`
- **To:** `https://www.health.nsw.gov.au/kidsfamilies/MCFhealth/Pages/child-blue-book.aspx`

The visible link text ("Australian New South Wales blue book") and the surrounding prose remain unchanged. The change is one line:

```xml
<!-- before -->
the <a href="http://www.health.nsw.gov.au/Kids/Publications/my-personal-health-record.pdf">Australian New South Wales blue book</a>

<!-- after -->
the <a href="https://www.health.nsw.gov.au/kidsfamilies/MCFhealth/Pages/child-blue-book.aspx">Australian New South Wales blue book</a>
```

The new URL points at the current NSW Health landing page for the Personal Health Record (Blue Book), which is the modern equivalent of the
publication previously hosted at the broken PDF path. It is also served over HTTPS, which the legacy URL was not.

### Design Rationale

- The reporter explicitly proposed the replacement URL in the ticket description; the resolution is "Persuasive" with no counter-proposal in
  the comments, so the change is a direct application of the suggested fix.
- Updating the citation rather than removing it preserves the pedagogical value of the existing `questionnaire-example-bluebook` example
  (the reader can still see the source publication that motivated the example).
- The replacement is a stable NSW Health landing page rather than a versioned PDF, reducing the chance of repeat link rot.
- Switching from `http://` to `https://` matches current FHIR practice for external citations and avoids mixed-content warnings in the
  rendered spec.
