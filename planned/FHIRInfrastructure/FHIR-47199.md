## 1. Summary

The reporter (IgnacioJauregui) noted that the deprecated artifacts paragraph on the FHIR Versions page still reflects R5-era wording — implying the list only covers items deprecated *in the current version*. The resolution (Persuasive with Modification) directs that the heading be reworded to clarify the list shows **all** items currently flagged as deprecated, regardless of when the deprecation originally occurred.

### Resolution Text (verbatim)

> Will change *"The following artifacts are deprecated in this version of FHIR"*
>
> to
>
> *"The following artifacts are marked as deprecated in this version of FHIR (though the deprecation may have occurred in a previous release)"*
>
> The intention of this element isn't to identify what's newly deprecated, but rather show everything that is flagged as deprecated.

## 2. Feature Proposal

Update the deprecated-artifacts heading on the Versions page (`source/versions.html`) to use the resolution's revised wording. This is a documentation/prose-only change — no schema, conformance, or tooling impact.

## 6. Additional Observations

### Deprecated items currently in the spec but NOT in the list

The deprecated-items `<ul>` on the Versions page currently contains only **one entry**:

- `OperationOutcome.issue.location`

However, many other artifacts carry `standards-status="deprecated"` in the R6 source:

| Category | Deprecated Artifact | Source File |
|----------|-------------------|-------------|
| Element | `Bundle.signature` | `source/bundle/structuredefinition-Bundle.xml` |
| Element | `Observation.bodySite` | `source/observation/structuredefinition-Observation.xml` |
| Element | `Measure.subject[x]` | `source/measure/structuredefinition-Measure.xml` |
| Element | `Measure.guidance` | `source/measure/structuredefinition-Measure.xml` |
| Element | `StructureDefinition.keyword` | `source/structuredefinition/structuredefinition-StructureDefinition.xml` |
| Element | `MetadataResource.topic` (+ concrete subtypes: ConceptMap, NamingSystem, ValueSet, ActivityDefinition, CodeSystem, EventDefinition, Library, Measure, PlanDefinition `.topic`) | `source/metadataresource/structuredefinition-MetadataResource.xml` and subtype files |
| Operation | `Measure/$evaluate-measure` | `source/measure/operationdefinition-Measure-evaluate-measure.xml` |
| Operation | `Measure/$submit-data` | `source/measure/operationdefinition-Measure-submit-data.xml` |
| Search Param | `Observation:body-site` | `source/observation/bundle-Observation-search-params.xml` |
| Search Param | `StructureDefinition:keyword` | `source/structuredefinition/bundle-StructureDefinition-search-params.xml` |

The resolution for **FHIR-47199** does not explicitly require updating the list contents — it only changes the heading wording. Whether the list itself should be expanded to include all deprecated artifacts is a separate editorial question. Related tickets [FHIR-53722](https://jira.hl7.org/browse/FHIR-53722) ("Deprecated elements should be withdrawn where applicable in R6") and [FHIR-55442](https://jira.hl7.org/browse/FHIR-55442) ("Remove deprecated OperationOutcome.issue.location") address the broader deprecated-element lifecycle and may interact with the list contents.
