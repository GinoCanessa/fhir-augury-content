## Feature Proposal

### Problem Statement

The `ActorDefinition` resource introduction page (`source/actordefinition/actordefinition-introduction.xml`) contained, in its "Boundaries and Relationships" section, the sentence:

> Actors may also be referenced from `<a href="examplescenario.html">ExampleXcenario</a>`.

The visible link text `ExampleXcenario` is a misspelling — there is no FHIR resource by that name. The link target (`examplescenario.html`) is correct, so the page rendered an obviously broken word inside an otherwise valid cross-link to the `ExampleScenario` resource. This is a pure copy/typography defect introduced when ActorDefinition / Requirements were added (commit `63a82280f5`, "Add ActorDefinition and Requirements"); the typo also survived an intermediate edit in commit `4b2d24769e` (FHIR-44757) before being fixed in `c188530fa1` (FHIR-43422).

### Proposed Change

Single, surgical text edit in one file:

- **File:** `source/actordefinition/actordefinition-introduction.xml`
- **Section:** `<h2>Boundaries and Relationships</h2>`
- **Change:** within the second paragraph of that section, replace the link text `ExampleXcenario` with `ExampleScenario`. The `href` (`examplescenario.html`) and surrounding sentence are unchanged.

Before:

```xml
<p>
Actors may also be referenced from <a href="examplescenario.html">ExampleXcenario</a>.
</p>
```

After:

```xml
<p>
Actors may also be referenced from <a href="examplescenario.html">ExampleScenario</a>.
</p>
```

No other elements, anchors, or files require modification for this ticket. (The same commit that fixed this typo also fixed two unrelated grammar issues in the same file — "can plays" → "can play"; "refer the actors" → "refer to the actors"; and "asociated" → "associated". Those are tracked under FHIR-43422 and are not in scope for FHIR-43804.)

### Design Rationale

The resolution wording is unambiguous and the reporter's recommendation ("changed back into 'ExampleScenario'") is the only sensible fix: `ExampleScenario` is the canonical FHIR resource name and matches the link target `examplescenario.html`. No alternative spelling, link relocation, or sentence rewrite is justified for a typo of this kind. Per the FhirCore playbook this is a "Update narrative / prose" recipe — author the fix directly in the `source/actordefinition/*.xml` introduction file; no spreadsheet, StructureDefinition, or generated artifact is involved.
