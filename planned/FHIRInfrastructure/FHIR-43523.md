## Feature Proposal

### Problem Statement

The GraphDefinition notes page includes a sentence that links to an external RCPA pathology profiles example to illustrate "the kind of summary this represents." That link (`http://fhir.hl7.org.au/fhir/rcpa/pprofiles.html`) no longer resolves, so the illustrative anchor in "Summarize a set of profiles on resources" is dead. Readers land on a 404/redirect and lose the concrete example the narrative depends on. A parenthetical TODO in the same paragraph already acknowledges that the intent was eventually to replace this external reference with a first-party GraphDefinition example — so the broken URL is effectively a long-standing placeholder that has now rotted.

### Proposed Change

Repair the broken external reference in the GraphDefinition notes page. The affected file is:

- `input/pagecontent/StructureDefinition-GraphDefinition-notes.xml` (lines 31–35) in `HL7/api-incubator-ig`.

Current content (verbatim):

```xml
<p>
Here's <a href="http://fhir.hl7.org.au/fhir/rcpa/pprofiles.html">an example</a> of
the kind of summary this represents. (Todo: make this an actual graph definition,
and clone into the main spec)
</p>
```

Implement one of the following, in priority order:

1. **Preferred — replace with a working AU Pathology IG URL.** The RCPA profiles page was part of the HL7 AU pathology (RCPA) FHIR guide, which now lives at `https://build.fhir.org.au/ig/hl7au/au-fhir-pathology-rcpa/` (with a published build under `http://hl7.org.au/fhir/rcpa/`). Locate the current equivalent of the original "pprofiles.html" summary page inside that IG (commonly the "Reports" / "Profile Summary" page) and update the `<a href>` to that URL. Keep the anchor text ("an example") and surrounding prose unchanged.
2. **Fallback — remove the hyperlink but keep the prose.** If a stable replacement page cannot be confirmed, strip the `<a href="…">…</a>` wrapper so the paragraph reads as narrative only:

   ```xml
   <p>
   A typical example of this kind of summary is a diagnostic report profile bundle
   that aggregates individual observation profiles. (Todo: make this an actual
   graph definition, and clone into the main spec)
   </p>
   ```

   The inline TODO should be preserved verbatim so the follow-up (authoring a real GraphDefinition example) remains discoverable.
3. **Ideal follow-up (out of scope for this ticket unless explicitly requested).** Author a real GraphDefinition instance inside `HL7/api-incubator-ig` that expresses a DiagnosticReport → panel Observation → atomic Observation profile graph, publish it as an example, and link to the rendered page from this paragraph, replacing the TODO.

The ticket's `Persuasive` resolution is satisfied by option 1 or 2; option 3 closes the embedded TODO but is a separate deliverable.

### Design Rationale

- GraphDefinition and this narrative page migrated out of `HL7/fhir` into the API incubator IG (`HL7/api-incubator-ig`, `input/pagecontent/StructureDefinition-GraphDefinition-notes.xml`). The fix must land in that repo, not in `HL7/fhir`. This is consistent with the reminder that GraphDefinition has moved out of FHIR Core.
- The reporter did not ask for the TODO to be resolved, only that the link work. Replacing the dead URL with a live equivalent is the smallest change that honors the `Persuasive` resolution while preserving the paragraph's pedagogical intent.
- Removing the link entirely (option 2) is an acceptable fallback because the surrounding prose already states the concept ("summarize the overall picture and present a summary to the user"); the external page was illustrative, not normative.
- Rewriting to a live URL should prefer the canonical HL7 AU publication path (`hl7.org.au/fhir/rcpa/...` or the current CI build under `build.fhir.org.au/ig/hl7au/au-fhir-pathology-rcpa/`) rather than third-party mirrors, to reduce the chance of future rot.
