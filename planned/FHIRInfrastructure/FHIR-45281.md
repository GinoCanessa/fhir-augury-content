## Resolution Summary

Resolution: **Persuasive**. The ticket reports a grammatical typo on
`http://build.fhir.org/documents.html`: the article "a" is used before a
word beginning with a vowel sound ("XSLT"), and should be "an". The
agreed fix is to change "**a** XSLT transform" to "**an** XSLT
transform" in the sentence:

> The XML Tools reference implementation includes **a** XSLT transform
> that converts an XML document into browser-ready XHTML.

The comment from `corey_spears` (2024-05-13) confirms the affected page
is `https://build.fhir.org/documents.html`. No other content changes
were requested.

## Feature Proposal

### Problem Statement

The published FHIR documents page (`documents.html`) contains an
English grammar error: it uses the indefinite article "a" before
"XSLT", a noun phrase whose initial sound is the vowel /ɛ/ ("ex"). The
correct article is "an". This is a small documentation defect that
nonetheless surfaces in user-facing build output of the FHIR Core
specification.

### Proposed Change

Edit `source/documents.html` in `HL7/fhir`. On the line currently
reading:

```html
The <a href="downloads.html#refimpl">XML Tools reference implementation</a> includes a
XSLT transform that converts an XML document into browser-ready XHTML.
```

change the trailing word `a` (end of the line containing the anchor)
to `an`, so the rendered prose reads:

> The XML Tools reference implementation includes **an** XSLT
> transform that converts an XML document into browser-ready XHTML.

No other text, anchors, or markup changes are required. The change is
confined to a single word in a single `<p>` block on a single
specification page.

### Design Rationale

The fix matches standard English usage: the indefinite article is
chosen by the *initial sound* of the following word, not its initial
letter. "XSLT" is pronounced "ex-ess-ell-tee", which begins with a
vowel sound, so "an" is correct. The change is a pure technical
correction with no behavioural, schema, terminology, or conformance
implications.
