## Feature Proposal

### Problem Statement

Section 5.10.4.4 of the FHIR R5 narrative for **GraphDefinition**
("Document Relationship rules" / compartment-consistency codes) ends with a
stranded authoring note:

> Todo: how would this be validated? – where is the graph referred to?

This is an editorial leftover that leaks unfinished-spec language into the
published IG. The question is not actionable as written (it is an open
thought aimed at a future author) and the resolution explicitly declines to
answer it in-line; instead the committee chose to strike the line so the
rendered page flows directly from the compartment-rule table into the
"Simple Examples" subsection.

Note that since R5, **GraphDefinition has been migrated out of
`HL7/fhir`** into the Application Programming Interfaces (API) Incubator
IG (`HL7/api-incubator-ig`). The affected narrative therefore no longer
lives in `source/graphdefinition/` of HL7/fhir; it lives in
`input/pagecontent/StructureDefinition-GraphDefinition-notes.xml` of
HL7/api-incubator-ig.

### Proposed Change

Delete the three-line `<p>…</p>` block containing the `Todo` sentence from
the GraphDefinition notes page in the API Incubator IG. Specifically, in
`input/pagecontent/StructureDefinition-GraphDefinition-notes.xml` at
lines 164–166:

```xml
<p>
Todo: how would this be validated? - where is the graph referred to?
</p>
```

must be removed, leaving the immediately preceding `</table>` (the
compartment-code table at line 163) directly followed by the blank line and
the `<a name="examples"></a>` / `<h4>Simple Examples</h4>` block at lines
168–169. The trailing blank line (167) may be left in place or collapsed;
either way the final rendered page goes straight from the compartment-rule
table to the examples.

After the edit, lines 163–169 should read:

```xml
</table>

<a name="examples"></a>
<h4>Simple Examples</h4>
```

No other content on the page changes. No section anchors are removed (the
deleted `<p>` block carries no `<a name="…">` anchor), so there are no
inbound fragment links to audit.

### Design Rationale

- The ticket is pure editorial cleanup; the committee's resolution is a
  single sentence ("Will remove this line"), so the smallest possible
  diff is the right answer.
- Preserving the `<a name="examples"></a>` anchor and the `Simple
  Examples` heading keeps existing cross-links (e.g., any `#examples`
  fragments) intact.
- Writing new prose to answer the Todo is out of scope per the
  resolution. If a future WG discussion concludes that GraphDefinition
  compartment rules need a validation story, that belongs in a separate
  ticket — it should not be silently rolled into this fix.
