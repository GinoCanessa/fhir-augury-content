## Resolution Summary

**Resolution:** Persuasive (no separate resolution description was recorded; the
applied change is the one described in the ticket itself).

The QuestionnaireResponse specification page contains a contraction
(`they're`) that violates the FHIR specification's house style requiring
abbreviations and contractions to be written out in full. Replace `they're`
with `they are` on the QuestionnaireResponse page. The ticket explicitly
scopes the fix to that single token (`Specifics: "they're"`).

## Feature Proposal

### Problem Statement

The FHIR house style for normative specification prose requires contractions
to be written out — "they're" should be "they are", "don't" should be "do
not", and so on. The ticket flags that the QuestionnaireResponse page
violates this rule and identifies the specific occurrence (`they're`). The
ticket's `Description` field reads:

> The page contains one or more misspelled or abbreviated words.
> Abbreviations need to be fully spelled out. Specifics: "they're"

A repository-wide search of the QuestionnaireResponse authoring directory
(`source/questionnaireresponse/`) confirms exactly one occurrence of
`they're`, in the **Boundaries and Relationships** section of
`questionnaireresponse-introduction.xml`:

```
source/questionnaireresponse/questionnaireresponse-introduction.xml:45:
  the definition of the questionnaire form, including what questions are
  asked, how they're organized and the constraints on the allowed answers.
```

No other instances of `they're` exist anywhere under
`source/questionnaireresponse/`.

### Proposed Change

Edit `source/questionnaireresponse/questionnaireresponse-introduction.xml`,
line 45, replacing `they're` with `they are`.

**Before (line 45):**

```xml
the definition of the questionnaire form, including what questions are asked, how they're organized and the constraints on the allowed answers.
```

**After (line 45):**

```xml
the definition of the questionnaire form, including what questions are asked, how they are organized and the constraints on the allowed answers.
```

No other files require changes for this ticket. The fix is confined to a
prose narrative file consumed by the HL7 publisher; no StructureDefinition,
ValueSet, search parameter, example, or schema is affected.

### Design Rationale

- The ticket's resolution (`Persuasive`) endorses the reporter's request as
  filed. The reporter (Lloyd McKenzie) named the exact token to be changed
  (`they're`), so no design discussion is needed about scope or wording.
- Expanding `they're` → `they are` is the minimal, mechanical change that
  satisfies the FHIR style rule against contractions while preserving the
  sentence's meaning and grammar.
- The change is made directly in the authored XML narrative
  (`questionnaireresponse-introduction.xml`) because, per the FhirCore
  briefing, `source/<resource>/*-introduction.xml` is hand-authored content
  picked up by the HL7 publisher; it is not a generated artifact.
