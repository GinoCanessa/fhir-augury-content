## Feature Proposal

### Problem Statement

Since STU3, the FHIR spec carried an example referencing an external AU RCPA profile page with a parenthetical TODO to replace the externally-linked illustration with a proper, in-spec `GraphDefinition` resource and re-use it as the canonical example. The external link has since rotted (only available via web.archive.org), so the example no longer works for readers, and the placeholder TODO has been visible to ballot reviewers for several releases. FHIR-43656 (which FHIR-43524 is rolled into) commits the FHIR Infrastructure WG to finally authoring a real `GraphDefinition` example and dropping the TODO.

A complicating fact surfaced by the Repo Context (see below): between the time the ticket was filed and today, `GraphDefinition` has been removed from FHIR Core and moved to the API Incubator (`http://build.fhir.org/HL7/api-incubator/StructureDefinition-GraphDefinition`). Every in-core reference to `GraphDefinition` in `HL7/fhir` now links out to the Incubator. Any new "actual GraphDefinition" example must therefore be authored where the resource lives (the API Incubator IG) and can only be linked to — not cloned into — the core spec.

### Proposed Change

Two coordinated pieces of work:

**A. Author the `GraphDefinition` example (in the API Incubator IG).**

- Create a new `GraphDefinition` instance that illustrates "the kind of summary" the retired AU RCPA pprofiles page was meant to convey: a GraphDefinition whose root is a CDA-style summary `Composition`, with links walking into the typical sections (Problem List, Medications, Allergies, Results, etc.) and each link's `target` constraining the referenced resources to the profiles used in that summary.
- Give the instance a stable `id`, canonical `url`, `name`, `title`, `status = draft`, `experimental = true`, and a `description` that explicitly states: *"This example is provided for illustration only; it is not a definitive best practice for representing a CDA Continuity-of-Care / Patient-Summary document in FHIR."* (matching the "example only" caveat FHIR-43656 requires for the sibling fix.)
- Add XML/JSON/Turtle serializations following whatever example convention the Incubator IG already uses for its other resource examples.

**B. Replace the TODO prose in the core spec with a link to the new example.**

- Find the single remaining occurrence of the legacy "Here's an example / *(Todo: make this an actual graph definition, and clone into the main spec)*" block in `HL7/fhir` (if any still survives; initial inspection of the current clone at `db79dcd` did not find the exact string, suggesting the TODO has already been stripped during the GraphDefinition relocation — see Open Questions).
- If it still exists: replace the dead external link and the TODO with a short narrative plus a link to the new Incubator example (`http://build.fhir.org/HL7/api-incubator/GraphDefinition-<id>.html`), and add the "example only, not best practice" caveat.
- If it has already been removed as part of the GraphDefinition relocation, record that in the ticket and confirm with the WG that FHIR-43524 is a no-op on `HL7/fhir` and only FHIR-43656's other two deliverables (the "in both places" CDA-section comment and the removal of the "how would this be validated" TODO) remain.

### Design Rationale

- `GraphDefinition` was moved to the API Incubator precisely because its design is still maturing, so authoring the new example there matches the current location of the resource and avoids shipping an example for a resource that no longer lives in core.
- Keeping the example explicitly marked "for illustration only" mirrors the instruction in FHIR-43656's resolution and keeps the summary / CCD discussion out of normative-looking core content — the WG deliberately chose not to canonize a specific summary structure.
- Cross-linking from core to the Incubator (rather than duplicating the resource in both repos) is consistent with how every other `GraphDefinition` reference in the current `HL7/fhir` clone is written (see the grep results in Impact Analysis below).
