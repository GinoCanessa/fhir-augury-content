## Resolution Summary

Resolution: **Persuasive**.

The Questionnaire specification page contains a sentence that points readers to "the QuestionnaireResponse page" for a diagram showing how QuestionnaireResponse items link back to Questionnaire items, but the hyperlink target is `questionnaire.html#link` — the page the reader is already on, not the QuestionnaireResponse page. The correct target is `questionnaireresponse.html#link`. The fix is a single-attribute change: update the `href` of the existing `<a>` element so it resolves to the diagram on `questionnaireresponse.html#link`.

## Feature Proposal

### Problem Statement

In the Questionnaire resource notes, the "LinkIds" subsection tells the reader: *"The QuestionnaireResponse page includes a diagram showing how response items are linked to their corresponding Questionnaire items."* The word "diagram" is hyperlinked, but the link points to `questionnaire.html#link` (the same Questionnaire page, anchor `link`) rather than the QuestionnaireResponse page. As a result, the reader is not taken to the diagram described in the prose; they land on the wrong page (or, depending on what the `#link` fragment resolves to on `questionnaire.html`, on no useful anchor at all).

The matching anchor `<a name="link"> </a>` does exist on the QuestionnaireResponse notes page (`source/questionnaireresponse/questionnaireresponse-notes.xml` line 26), so the intended target is unambiguous.

### Proposed Change

Change the broken hyperlink in the "LinkIds" subsection of the Questionnaire notes so that it points to the QuestionnaireResponse linkage diagram.

- **File:** `source/questionnaire/questionnaire-notes.xml`
- **Location:** line 130, inside the `<h3>LinkIds</h3>` block (anchor `<a name="linkId"> </a>` on line 128).
- **Current:**
  ```xml
  The QuestionnaireResponse page includes a <a href="questionnaire.html#link">diagram</a> showing how response items are linked to their corresponding Questionnaire items.
  ```
- **Replace with:**
  ```xml
  The QuestionnaireResponse page includes a <a href="questionnaireresponse.html#link">diagram</a> showing how response items are linked to their corresponding Questionnaire items.
  ```

No other text changes; only the `href` attribute is updated.

### Design Rationale

- Matches the prose: the sentence explicitly says "The QuestionnaireResponse page includes a diagram", so the link must resolve to a fragment on `questionnaireresponse.html`.
- The target anchor already exists: `source/questionnaireresponse/questionnaireresponse-notes.xml` line 26 declares `<a name="link"> </a>`, which the bespoke HL7 publisher will render as `questionnaireresponse.html#link`. No new anchor needs to be created.
- Minimal-diff fix. Alternative approaches considered:
  - *Inline the diagram into `questionnaire-notes.xml`* — rejected; the diagram lives with the QuestionnaireResponse content and the prose deliberately cross-references it.
  - *Remove the link and just describe the diagram inline* — rejected; loses the cross-reference value and removes a useful navigation aid.
- Consistent with adjacent links in the same paragraph that already use `questionnaireresponse.html` (the `<a href="questionnaireresponse.html">QuestionnaireResponse</a>` references on the surrounding lines).
