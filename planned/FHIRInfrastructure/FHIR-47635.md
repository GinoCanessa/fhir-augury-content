## Resolution Summary

Resolution type: **Persuasive**.

The ticket points out that the FHIR "Versions" page attributes the labels
`deprecated` / `withdrawn` to the HTML 4.0 Standard, but HTML 4.0 actually
uses `deprecated` and `obsolete` (not `withdrawn`). Rather than renaming
FHIR's `withdrawn` bucket, the approved resolution reframes the HTML 4.0
citation as an inspiration rather than a direct source.

Exact wording from the resolution description (reformatted):

> Will change `(from the HTML 4.0 Standard)` to
> `(inspired by 'deprecated' and 'obsolete' from the HTML 4.0 Standard)`.

No change to the terms `deprecated` / `withdrawn` themselves, to the
semantics of the table rows, or to any downstream ballot rules is
required.

## Feature Proposal

### Problem Statement

On the FHIR Versions page (`source/versions.html`), the paragraph
introducing the "Deprecation / Withdrawal of Normative Content"
table (section anchor `#deprecation`, line 551) reads:

> Once content is normative, there is a process for removing it from the
> standard by marking it as `deprecated` or `withdrawn` (from the
> [HTML 4.0 Standard](https://www.w3.org/TR/html4/conform.html)):

This implies both labels are taken from HTML 4.0. In fact, HTML 4.0
defines **`deprecated`** and **`obsolete`** — it does not define
`withdrawn`. The attribution is therefore incorrect and misleading to
implementers and editors who follow the citation.

### Proposed Change

Make a single, local wording change in `source/versions.html` so the
citation is accurate without renaming the FHIR labels themselves.

- File: `source/versions.html`
- Section: `<h4>Deprecation / Withdrawal of Normative Content</h4>`
  (under `<a name="deprecation"></a>`).
- Line 551 currently reads:

  ```html
  <code>deprecated</code> or <code>withdrawn</code> (from the <a href="https://www.w3.org/TR/html4/conform.html">HTML 4.0 Standard</a>):
  ```

- Replace the parenthetical so the line becomes:

  ```html
  <code>deprecated</code> or <code>withdrawn</code> (inspired by 'deprecated' and 'obsolete' from the <a href="https://www.w3.org/TR/html4/conform.html">HTML 4.0 Standard</a>):
  ```

No other lines in the file change. The table, the surrounding prose, the
deprecation policy (lines 554–563), and the list of currently deprecated
artifacts (line 566+) are all unaffected.

### Design Rationale

The resolution deliberately keeps the FHIR-specific term `withdrawn`
rather than renaming it to `obsolete`, because:

- `withdrawn` is already referenced extensively elsewhere on
  `versions.html` (lines 555, 559, 562, 563) and in downstream prose. A
  rename would be a sweeping edit across the specification.
- The ticket reporter's own secondary suggestion is to keep `withdrawn`
  and just clarify the citation — which is what the resolution does.
- "Inspired by" is the minimally invasive phrasing that honours the HTML
  4.0 precedent while acknowledging that FHIR's taxonomy is its own
  (two-state `deprecated` / `withdrawn`, not HTML's `deprecated` /
  `obsolete`).

No alternative wording is recorded in the ticket comments; the
resolution was captured as a single applied-vote change.
