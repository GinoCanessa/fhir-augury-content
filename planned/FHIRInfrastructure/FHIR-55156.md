## Summary

The element `CapabilityStatement.rest.interaction.code` is a **mandatory** element (min=1) with a **required** binding to the `SystemRestfulInteraction` value set (`http://hl7.org/fhir/ValueSet/system-restful-interaction`). The value set currently contains only four codes: `transaction`, `batch`, `search-system`, and `history-system`. There is no `unknown` or `other` escape valve concept, which makes the containing `interaction` BackboneElement impossible to populate when the interaction type is not one of these four predefined codes.

## Resolution Description

> Do the same as per [FHIR-55154](https://jira.hl7.org/browse/FHIR-55154) (should be able to use the same code, just add it to this value set)

FHIR-55154 (the sibling ticket for `CapabilityStatement.rest.resource.interaction.code`) resolved:

> Will add 'other' to the code system and value set with a definition of "An interaction type that is not one of the standard interaction types defined in this version of FHIR." Unknown wouldn't make sense here.

## Feature Proposal

Add an `other` code to the `restful-interaction` CodeSystem and include it in the `SystemRestfulInteraction` value set. This provides an escape valve for `CapabilityStatement.rest.interaction.code` so that system-level interactions not yet defined in the spec can still be represented. This change mirrors what FHIR-55154 does for the `TypeRestfulInteraction` value set.

## Implementation Plan

### Step 1: Add `other` concept to CodeSystem `restful-interaction`

**File:** `source/codesystem/codesystem-restful-interaction.xml`
