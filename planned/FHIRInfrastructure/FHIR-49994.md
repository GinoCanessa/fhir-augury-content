## Resolution Summary

**Resolution: Persuasive.** The Request "Boundaries and Relationships" (BnR) narrative contains an unterminated parenthetical and a trailing `</a>/` that leaves the sentence truncated. Per the applied resolution description, the existing text

> "(or is intended to be followed by a number of [other FHIR resources](https://build.fhir.org/workflow.html#list)/"

must be rewritten as

> "(or is intended to be followed) by a number of [other FHIR resources](https://build.fhir.org/workflow.html#list)."

i.e. (a) close the parenthesis after `followed`, (b) move `by a number of` outside the parenthesis, and (c) replace the stray trailing slash with a period terminating the sentence.

## Feature Proposal

### Problem Statement

The Request resource's "Boundaries and Relationships" section is authored in `source/request/request-spreadsheet-introduction.xml`. Its second sentence reads:

> This pattern is followed by (or is intended to be followed by a number of [other FHIR resources](workflow.html#list)/

This sentence has three defects, all of which the ticket and its resolution call out:

1. The parenthesis opened after "followed by" is never closed.
2. The sentence has no terminating period.
3. A stray `/` character appears immediately after `</a>`, producing a visible artifact (`other FHIR resources/`) in the published HTML.

The resolution is categorised as a pure technical correction: there is no change to semantics or cross-references, only to the prose.

### Proposed Change

Edit the single `<p>` element that opens the "Boundaries and Relationships" section in `source/request/request-spreadsheet-introduction.xml` (line 23–26 of the clone) so that its closing sentence reads:

```html
This pattern is followed by (or is intended to be followed) by a number of <a href="workflow.html#list">other FHIR resources</a>.
```

Concretely:

- Insert a closing `)` after the word `followed` (was: `followed by a number of`; becomes: `followed) by a number of`).
- Delete the trailing `/` that currently follows `</a>`.
- Append a `.` to terminate the sentence.

No other elements, anchors, or links change. The link target `workflow.html#list` and its anchor in `source/workflow.html` are left untouched.

### Design Rationale

The resolution description dictates the exact replacement text verbatim; this plan applies that text with no editorial interpretation. Keeping the edit minimal (one sentence, one file) matches the "Update narrative / prose" recipe from the FhirCore playbook and mirrors how other `*-spreadsheet-introduction.xml` BnR blocks are authored. No alternative wording was discussed in the ticket (there are no comments).
