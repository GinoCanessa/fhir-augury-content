### Preliminary scoping notes (not a substitute for the plan)

The following is included only as context for the maintainer who
refreshes the briefings; it is **not** an implementation plan and must
not be acted on without first loading the refreshed briefings.

- **Resolution commits to** (a) more visually distinct CI-build output
  — e.g., different page background, border warnings, an "under
  construction" icon — and (b) an "existing yellow box" link-out to a
  per-specification "release guide" Confluence page, pending FMG/TSC
  discussion.
- **Resolution explicitly rejects** the reporter's two suggestions:
  the base URL (e.g., `https://www.hl7.org/fhir/us/core/`) will
  continue to resolve to the most recent official release, the CI
  build will remain discoverable by search engines, and IG
  status/maturity will not be reset on each CI build.
- **Likely touch points** (to be confirmed against briefings):
  - `HL7/fhir-ig-publisher` — banner/header rendering for CI-build
    vs. official releases; wiring for a new "release guide" link slot
    in the yellow box; any `publication-request.json` / version-status
    plumbing that decides which banner variant to emit.
  - `HL7/ig-template-hl7` — Jekyll includes / CSS that render the
    yellow box and page chrome, where the new background, border
    warnings, and "under construction" icon would live.
  - `HL7/fhir` — only if the FHIR Core publish templates carry their
    own copy of the yellow-box markup that must stay in sync.
- **Out of scope** for this ticket: SEO-level changes, base-URL
  redirects, maturity resets, regulatory policy.
- **Cross-org coordination** required by the resolution itself: FMG
  and TSC sign-off on the "release guide" Confluence concept before
  the publisher-side link can be wired up.
