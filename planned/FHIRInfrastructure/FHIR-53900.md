## Summary

The contained resources section (Chapter 2.1.3.0.14) still includes an open "Note to Implementers" asking whether the inability to search for contained resources that reference their container is a problem. This open question should be removed before the specification goes normative, as unresolved questions are inappropriate in a normative specification.

## Resolution

**Persuasive** — The committee agreed to remove the note. The resolution description states:

> Will remove the following note:
> > Note to Implementers: There is no way to search for contained resources that reference their container. Is this a problem?
> >
> > Feedback is welcome here.

## Feature Proposal

Remove the `[%stu-note dstu2%]` block from the contained resources section of the references page. This is a simple editorial deletion of a legacy feedback-solicitation note that is no longer appropriate for a normative specification.

## Impact Analysis

### Scope: Minimal

This change is purely editorial — it removes an informational note from the HTML source of the specification. It does not affect:

- Any normative content, constraints, or rules
- Resource definitions, profiles, or data types
- Search parameters or operations
- Any generated artifacts (schemas, StructureDefinitions, etc.)
- Conformance expectations

### Risk: Very Low

Deleting an informational note has zero functional impact. The note was a request for feedback, not normative text.
