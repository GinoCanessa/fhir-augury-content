## Resolution Summary

Resolution: **Persuasive**.

Per the resolution description, add a hyperlink to `https://www.rfc-editor.org/rfc/rfc2119` on the text "RFC 2119" in the sentence:

> This specification uses the conformance verbs SHALL, SHOULD, and MAY as defined in RFC 2119.

This is a pure documentation / technical-correction change. No schema, profile, or terminology artifacts are affected.

## Feature Proposal

### Problem Statement

The FHIR Core specification references RFC 2119 (the IETF document that normatively defines the conformance keywords SHALL / SHOULD / MAY) as plain text rather than a hyperlink. Readers have to search for the RFC manually. Other RFCs cited elsewhere in the spec (e.g., `source/http.html`, `source/signatures.html`, `source/subscriptions.html`, `source/rdf.html`, `source/terminology-service.html`) are already linked to `https://www.rfc-editor.org/rfc/rfcNNNN`, so the unlinked "RFC 2119" mention is both an inconvenience and a stylistic inconsistency.

### Proposed Change

Wrap the text `RFC 2119` with an anchor pointing at the canonical RFC Editor URL, using the plain-anchor pattern already used throughout `source/` for RFC citations (no `class="external-link"`, no `target="_blank"`, no `rel="nofollow noopener"` â€” those were artifacts of the Jira paste, not FHIR authoring style; see `source/http.html:275`, `source/signatures.html:215`, `source/rdf.html:205`).

Specifically, in `source/help.html` line 396 (verified in clone `db79dcd`), change:

```html
    HL7 FHIR specification uses the conformance verbs SHALL,
    SHOULD, and MAY as defined in RFC 2119. The correct verb for indicating
```

to:

```html
    HL7 FHIR specification uses the conformance verbs SHALL,
    SHOULD, and MAY as defined in <a href="https://www.rfc-editor.org/rfc/rfc2119">RFC 2119</a>. The correct verb for indicating
```

This is the sentence quoted verbatim in the resolution description.

Additionally, for consistency â€” the only other page carrying `RFC 2119` prose is `source/conformance-rules.html` line 84:

```html
<p>This specification uses the conformance verbs SHALL, SHOULD, and MAY as defined in the RFC 8174 clarification of RFC 2119. Unlike RFC 2119, however, this specification allows ...
```

Link the *first* occurrence of `RFC 2119` on this line to `https://www.rfc-editor.org/rfc/rfc2119`. The subsequent "Unlike RFC 2119" mention should be left unlinked (standard practice: link the first occurrence in a section). Optionally, also link `RFC 8174` to `https://www.rfc-editor.org/rfc/rfc8174` in the same sentence â€” this is mentioned as an Open Question below because it is outside the literal resolution text.

### Design Rationale

- **Matches the resolution verbatim.** `source/help.html:396` contains the exact sentence the resolution quotes. Linking just that one occurrence satisfies the resolution as written.
- **Consistency with existing FHIR authoring style.** The established pattern in `source/` is a bare `<a href="...">` around descriptive text â€” not Jira's `class="external-link" target="_blank" rel="nofollow noopener"` variant. Following the existing pattern keeps the diff minimal and conforms to how other RFC links are authored on adjacent pages.
- **First-occurrence linking.** Linking only the first mention per page (for `conformance-rules.html`) is standard technical-writing style and is how other repeated RFC references are handled (e.g., RFC 3987 on `source/rdf.html` is linked at each new section, not on every sentence).
- **No alternatives seriously on the table.** The ticket is a Technical Correction with a single obvious fix; no competing designs were discussed in comments (the snapshot carries no comment thread).
