# Implementation Plan: FHIR-50461

**Title:** Add data-monitoring to ResearchStudyPartyRole CodeSystem
**Status:** Resolved - change required
**Resolution:** Persuasive
**Work Group:** Biomedical Research & Regulation
**Specification:** FHIR Core (FHIR)
**Resolved:** 2025-05-20

---

## Resolution Summary

Add the `data-monitoring` code to the ResearchStudyPartyRole CodeSystem, move the ResearchStudyPartyRole CodeSystem and ValueSet to THO (HL7 Terminology), and update the binding in the ResearchStudy StructureDefinition to point to the THO ValueSet.

## Feature Proposal

### Problem Statement

The ResearchStudyPartyRole CodeSystem was missing a code to represent data monitoring entities (such as Data Monitoring Committees or Data and Safety Monitoring Boards). This gap was discovered while mapping the SPIRIT checklist for trial protocols to the ResearchStudy StructureDefinition during EBMonFHIR project meetings. Additionally, the CodeSystem and ValueSet were maintained inline in FHIR Core rather than in THO, which is the standard home for HL7 terminology resources.

### Proposed Change

Three changes are required:

1. **Add `data-monitoring` concept** to the ResearchStudyPartyRole CodeSystem with code `data-monitoring`, display `Data Monitoring`, and a definition describing an entity that monitors data validity and integrity while prioritizing participant safety.
2. **Move the CodeSystem and ValueSet to THO**, changing URLs from `http://hl7.org/fhir/research-study-party-role` / `http://hl7.org/fhir/ValueSet/research-study-party-role` to `http://terminology.hl7.org/CodeSystem/research-study-party-role` / `http://terminology.hl7.org/ValueSet/research-study-party-role`.
3. **Update the binding** in `ResearchStudy.associatedParty.role` to reference the THO ValueSet URL.

### Design Rationale

- The SPIRIT checklist (Standard Protocol Items: Recommendations for Interventional Trials) identifies data monitoring as a key role in clinical trials. Adding this code aligns FHIR's ResearchStudy resource with established clinical trial reporting standards.
- Moving terminology to THO follows HL7's standard practice of managing code systems and value sets in the Unified Terminology Governance (UTG) repository, enabling independent versioning and broader reuse.

### Breaking Changes

- **CodeSystem URL change:** The CodeSystem URL changes from `http://hl7.org/fhir/research-study-party-role` to `http://terminology.hl7.org/CodeSystem/research-study-party-role`. Systems using the old URL for code validation or code lookups will need to update. This is a **breaking change** for implementations referencing the old URL.
- **ValueSet URL change:** The ValueSet URL changes from `http://hl7.org/fhir/ValueSet/research-study-party-role` to `http://terminology.hl7.org/ValueSet/research-study-party-role`. Terminology servers and profiles binding to the old URL will need updating.
- **New code addition:** Adding `data-monitoring` is **non-breaking** — it is an additive change that expands the set of valid codes.

### Terminology Impact

- **New code:** `data-monitoring` added to CodeSystem `ResearchStudyPartyRole` with display "Data Monitoring" and definition: "An organization or individual who monitors the validity and integrity of collected data while prioritizing participant safety and study integrity. The organization may be called a Data Monitoring Committee (DMC) or Data and Safety Monitoring Board (DSMB)."

- **Binding strength:** Per FHIR-53696, the binding strength on `ResearchStudy.associatedParty.role` should change from `example` to `preferred` when the binding URL is updated.
