**Title:** Remove external SEVCO codes from research-study-classifiers CodeSystem and add an external code system.
**Status:** Resolved - change required

**Work Group:** Biomedical Research & Regulation

## Resolution Summary

The resolution is **Persuasive**: "Use https://fevir.net/sevco when it is listed with THO."

The original ticket identified that the `research-study-classifiers` CodeSystem contained 75 terms copied from the Scientific Evidence Code System (SEVCO, maintained by Scientific Knowledge Accelerator Foundation at https://fevir.net/sevco). These were included as a temporary convenience for the R5 ballot to support the `study-design` ValueSet used by both `ResearchStudy.studyDesign` and `Evidence.studyDesign`. The ticket calls for:

1. Getting an external code system identifier from HTA for SEVCO
2. Creating a full convenience copy of the SEVCO external code system in THO
3. Requesting a THO publication
4. Revising the Study Design ValueSet CLD to use the SEVCO code system
5. Removing the SEVCO terms from the Research Study Classifiers CodeSystem

Vote history: Brian Alper / Catherine Hosage-Norman: 5-0-0 (Impact: Compatible, substantive; Category: Correction).

## Feature Proposal

### Problem Statement

The `research-study-classifiers` CodeSystem (`http://hl7.org/fhir/research-study-classifiers`) was being used as a dumping ground for 75 SEVCO terms that properly belong in an external code system. This made the FHIR Core CodeSystem an improper home for externally-maintained terminology, causing maintenance and sustainability concerns. The `study-design` ValueSet currently enumerates 73 individual SEVCO codes with full designations in its compose section (~1,890 lines), rather than referencing the codes via a compact CLD using the external code system with filters or descendant-of relationships.

### Proposed Change

**Phase 1 (partially complete):** Register SEVCO as an external code system in THO (HL7 Terminology) with the canonical URI `https://fevir.net/sevco`, and remove SEVCO codes from the `research-study-classifiers` CodeSystem.

**Phase 2 (remaining work):** Create a full convenience copy CodeSystem resource for SEVCO in UTG, revise the `study-design` ValueSet CLD to use a compact intensional definition (3 root codes + descendants per companion ticket FHIR-39004), and publish via THO.

### Design Rationale

- External terminologies should be referenced, not copied into FHIR Core code systems
- The Scientific Knowledge Accelerator Foundation (SKAF) maintains SEVCO independently; coupling it to FHIR release cycles creates drift
- A proper external code system registration in THO enables long-term sustainability
- Compact CLD definitions (filter-based) are more maintainable than enumerating 73+ individual codes
- CC BY-SA 4.0 licensing of SEVCO permits convenience copying

### Breaking Changes

- **Terminology resolution change:** The `study-design` ValueSet CLD will change from an enumerated list of 73 SEVCO codes to an intensional definition (3 root codes + descendants). This is semantically equivalent if the SEVCO code system hierarchy is correct, but may produce different expansion results if the SEVCO code system has been updated with new descendant codes since the enumeration was created. This is **intentional** — the ValueSet should track SEVCO updates.
- **No element-level breaking changes:** No StructureDefinition elements are added, removed, or have their types/cardinalities changed.
- **Binding compatibility:** Both `ResearchStudy.studyDesign` (preferred) and `Evidence.studyDesign` (extensible) continue to bind to the same ValueSet URL. Systems already using SEVCO codes from `https://fevir.net/sevco` will continue to work.
- **Risk:** If the SEVCO convenience copy CodeSystem is not yet published in THO when the FHIR spec build runs, the ValueSet expansion may fail. The convenience copy must be published first.

### Terminology Impact

- **CodeSystem `http://hl7.org/fhir/research-study-classifiers`:** No change needed — already contains only 4 non-SEVCO codes (`fda-regulated-drug`, `fda-regulated-device`, `mpg-paragraph-23b`, `irb-exempt`)
- **External CodeSystem `https://fevir.net/sevco`:** Needs a convenience copy CodeSystem resource created in UTG/THO. The NamingSystem registration already exists. The CodeSystem resource should contain the full SEVCO hierarchy including all codes referenced by the `study-design` ValueSet (73+ codes currently enumerated, plus ancestor codes like SEVCO:01000, SEVCO:00998, SEVCO:00999 per FHIR-39004)
- **ValueSet `http://hl7.org/fhir/ValueSet/study-design` / `http://terminology.hl7.org/ValueSet/study-design`:** CLD must change from enumerated codes to intensional filter-based definition. Both the FHIR Core copy and the UTG/THO copy must be updated in sync.
- **Licensing:** SEVCO is CC BY-SA 4.0, copyright held by Scientific Knowledge Accelerator Foundation. Open license permits convenience copying.

## Implementation Plan

### Prerequisites

1. **SEVCO external code system registration in THO** — ✅ Already complete (NamingSystem `SEVCO.xml`, metadata record `ECS_MR-SEVCO.xml`, listing in `external_code_systems.xml`)
2. **SEVCO canonical URI assigned** — ✅ `https://fevir.net/sevco` (TSMG-endorsed)
3. **Obtain current SEVCO code system content** from https://fevir.net/resources/CodeSystem/27270 for the convenience copy
4. **Coordinate with FHIR-39004** — the ValueSet CLD change is specified in the companion ticket and should be implemented together

### Step-by-Step Tasks

#### Task 1: Create SEVCO Convenience Copy CodeSystem in UTG (HL7/UTG)

**File:** Create new `input/sourceOfTruth/external/v3/codeSystems/CodeSystem-SEVCO.xml` (or appropriate path per UTG conventions for external code systems)

1. Obtain the full SEVCO code system content from https://fevir.net/resources/CodeSystem/27270
2. Create a FHIR CodeSystem resource with:
   - `url`: `https://fevir.net/sevco`
   - `name`: `SEVCO`
   - `title`: `Scientific Evidence Code System (SEVCO)`
   - `status`: `active`
   - `content`: `complete` (convenience copy)
   - `hierarchyMeaning`: `is-a`
   - `caseSensitive`: `true`
   - `copyright`: CC BY-SA 4.0, Scientific Knowledge Accelerator Foundation
   - `publisher`: Scientific Knowledge Accelerator Foundation / Computable Publishing LLC
   - All concept codes with display, definition, designations (synonyms), and hierarchy
3. Register the CodeSystem in UTG's `input/utg.xml` if required by UTG build conventions
4. Verify the CodeSystem includes at minimum the 73 codes currently enumerated in `valueset-study-design.xml` plus the 3 root codes from FHIR-39004 (SEVCO:01000, SEVCO:00998, SEVCO:00999)

#### Task 2: Revise Study Design ValueSet CLD in FHIR Core (HL7/fhir)

**File:** `source/evidence/valueset-study-design.xml`

1. Replace the current `compose` section (which enumerates 73 individual SEVCO codes with full designations across ~1,850 lines) with a compact intensional definition:
   ```xml
   <compose>
     <include>
       <system value="https://fevir.net/sevco"/>
       <filter>
         <property value="concept"/>
         <op value="is-a"/>
         <value value="SEVCO:01000"/>
       </filter>
     </include>
     <include>
       <system value="https://fevir.net/sevco"/>
       <filter>
         <property value="concept"/>
         <op value="is-a"/>
         <value value="SEVCO:00998"/>
       </filter>
     </include>
     <include>
       <system value="https://fevir.net/sevco"/>
       <filter>
         <property value="concept"/>
         <op value="is-a"/>
         <value value="SEVCO:00999"/>
       </filter>
     </include>
   </compose>
   ```
2. Update the `immutable` element to `false` (since the expansion now depends on external content)
3. Remove the `version` pin (`<version value="2.0"/>`) from the include, or update to current SEVCO version
4. Update `meta.lastUpdated` to reflect the change date

#### Task 3: Revise Study Design ValueSet CLD in UTG (HL7/UTG)

**File:** `input/sourceOfTruth/fhir/valueSets/ValueSet-study-design.xml`

1. Apply the same CLD change as Task 2 — replace the enumerated 73 SEVCO codes with the 3 filter-based includes
2. Keep the ValueSet URL as `http://terminology.hl7.org/ValueSet/study-design`
3. Update version and metadata as appropriate

#### Task 4: Verify No Residual SEVCO Codes in Classifiers CodeSystem (HL7/fhir)

**File:** `source/researchstudy/codesystem-research-study-classifiers.xml`

1. Confirm only 4 codes remain: `fda-regulated-drug`, `fda-regulated-device`, `mpg-paragraph-23b`, `irb-exempt`
2. ✅ **Already verified** — no SEVCO codes present

#### Task 5: Request THO Publication

1. Submit a publication request to THO to include the new SEVCO convenience copy CodeSystem
2. Coordinate timing so the CodeSystem is published before the FHIR Core build depends on it for ValueSet expansion

1. **ValueSet expansion test:** Expand `study-design` ValueSet against the SEVCO convenience copy CodeSystem and verify it includes all 73 previously-enumerated codes plus any new descendants under the 3 root codes
2. **Build validation:** Run FHIR Core build to verify no broken references from `ResearchStudy.studyDesign` or `Evidence.studyDesign` bindings

### Open Questions

1. **Exact path for SEVCO convenience copy in UTG:** The UTG repository has multiple patterns for external code systems. Confirm the correct directory (e.g., `input/sourceOfTruth/external/v3/codeSystems/` vs. another path) with UTG maintainers.
2. **SEVCO version pinning:** Should the ValueSet CLD pin a specific SEVCO version, or track the latest? The current enumerated version references `version="2.0"`. Removing version pinning allows automatic tracking of new codes but may introduce unexpected expansion changes.
3. **Coordination with FHIR-39004:** The companion ticket specifies the same ValueSet CLD change. These two tickets should be implemented together to avoid duplicate or conflicting changes.
4. **SEVCO hierarchy completeness:** The 3 root codes in FHIR-39004 (SEVCO:01000, SEVCO:00998, SEVCO:00999) must be ancestors of all 73 currently-enumerated codes. Verify against the full SEVCO hierarchy at https://fevir.net/resources/CodeSystem/27270.
5. **THO publication timing:** The convenience copy CodeSystem must be published in THO before the FHIR Core spec build can successfully expand the revised ValueSet. Coordinate release timing.
