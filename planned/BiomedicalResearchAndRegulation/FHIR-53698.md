## Resolution Description

> Change binding to THO-defined ValueSet with Preferred binding strength.

## Problem Statement

`ResearchStudy.studyDesign` currently binds to the locally-defined ValueSet `http://hl7.org/fhir/ValueSet/study-design` (file `source/evidence/valueset-study-design.xml` in the FHIR core spec). This ValueSet was migrated to HL7 Terminology (THO) at `http://terminology.hl7.org/ValueSet/study-design`, but the binding references in the FHIR core StructureDefinitions were **not updated** in time for the R6 ballot.

The binding URI in both `ResearchStudy.studyDesign` and `Evidence.studyDesign` must be changed to point to the THO-hosted ValueSet, and the local copy of the ValueSet must be removed from the FHIR core repository.

Completing this change also resolves:
- [FHIR-39003](https://jira.hl7.org/browse/FHIR-39003) – Remove external SEVCO codes from research-study-classifiers CodeSystem
- [FHIR-39004](https://jira.hl7.org/browse/FHIR-39004) – Change Study Design ValueSet CLD to use SEVCO once published in THO

## Feature Proposal

Update the `studyDesign` element binding in both the `ResearchStudy` and `Evidence` StructureDefinitions from the FHIR-core-local ValueSet URI to the THO-hosted ValueSet URI, maintaining the existing binding strengths (Preferred for ResearchStudy, Extensible for Evidence). Remove the local ValueSet XML file from the FHIR core repository since its authoritative home is now THO. Update associated SVG diagrams to point to the THO-hosted ValueSet page.
