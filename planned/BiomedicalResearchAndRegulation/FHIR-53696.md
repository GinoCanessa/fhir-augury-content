## Resolution Description

> Change binding to THO-defined ValueSet with Preferred binding strength.

## Problem Statement

`ResearchStudy.associatedParty.role` currently has an **Example** binding to the FHIR-core-defined ValueSet at `http://hl7.org/fhir/ValueSet/research-study-party-role`. The CodeSystem and ValueSet have already been migrated to THO (HL7 Terminology) at `http://terminology.hl7.org/ValueSet/research-study-party-role` and `http://terminology.hl7.org/CodeSystem/research-study-party-role` (confirmed present in HL7/UTG), but the binding in the StructureDefinition was not updated before the R6 ballot.

This change also completes the resolution of [FHIR-50461](https://jira.hl7.org/browse/FHIR-50461) ("Add data-monitoring to ResearchStudyPartyRole CodeSystem"), which requested moving the CodeSystem/ValueSet to THO and adding a `data-monitoring` concept.

## Feature Proposal

Update the `ResearchStudy.associatedParty.role` element binding to:
1. Point to the THO-defined ValueSet URL (`http://terminology.hl7.org/ValueSet/research-study-party-role`)
2. Strengthen the binding from `example` to `preferred`

Additionally, update example resources to use the new THO CodeSystem URL, and retire the FHIR-core-local CodeSystem and ValueSet definitions.

### Breaking Change Assessment

**Binding Strength Change: Example → Preferred**
- **Example** bindings are non-normative; systems are free to use any codes
- **Preferred** bindings encourage use of the specified codes but still permit alternatives
- Moving from Example to Preferred is a **non-breaking, tightening change** — no existing valid data becomes invalid

**ValueSet URL Change: `http://hl7.org/fhir/ValueSet/...` → `http://terminology.hl7.org/ValueSet/...`**
- Validators referencing the old URL will need to resolve the new URL
- The concepts in the THO ValueSet are identical to the FHIR-core version (same 12 codes including `data-monitoring`)
- This is a **low-risk change** — standard THO migration pattern

**CodeSystem URL Change in Examples: `http://hl7.org/fhir/research-study-party-role` → `http://terminology.hl7.org/CodeSystem/research-study-party-role`**
- Existing data using the old CodeSystem URL will reference a deprecated system
- This is expected as part of THO migration and is **non-breaking** for R6
