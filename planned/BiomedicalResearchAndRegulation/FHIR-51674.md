## Description

The ticket reports a missing space typo in the ConditionEra profile definition. The text reads `"very firstchronologically"` but should read `"very first chronologically"`. The affected page is the rendered StructureDefinition for ConditionEra at:

https://build.fhir.org/ig/HL7/fhir-omop-ig/StructureDefinition-ConditionEra.profile.json.html

## Resolution Description

> We will fix the identified typo.

**Resolution is "Persuasive" → Implementation required.**

## Feature Proposal

Fix the missing-space typo in the `condition_era_start_date` element description within the ConditionEra FSH definition. Change `"firstchronologically"` to `"first chronologically"`.

## Detailed Implementation Plan

### Step 1: Fix the typo in ConditionEra.fsh

**Repository:** `HL7/fhir-omop-ig`
**File:** `input/fsh/ConditionEra.fsh`
**Branch:** `main` (current HEAD: `2a497142`)

**Current text** (in the `condition_era_start_date` element description string):

```
"The start date for the Condition Era constructed from the individual instances of Condition Occurrences. It is the start date of the very firstchronologically recorded instance of the condition with at least 31 days since any prior record of the same Condition."
```

**Corrected text:**

```
"The start date for the Condition Era constructed from the individual instances of Condition Occurrences. It is the start date of the very first chronologically recorded instance of the condition with at least 31 days since any prior record of the same Condition."
```

**Change:** Insert a single space between `first` and `chronologically`.

3. Confirm the generated `StructureDefinition-ConditionEra.profile.json` now contains `"first chronologically"` instead of `"firstchronologically"`.
