# Implementation Plan: FHIR-20958

**Title:** Number sections for ease of reading/commenting
**Status:** Resolved - change required
**Resolution:** Persuasive
**Work Group:** Biomedical Research & Regulation
**Specification:** US Coordinated Registry Network (CRN) (FHIR)
**Resolved:** 2019-08-28

---

## Resolution Summary

The ballot comment requests that section numbers be added throughout the entire CRN IG (overview, mappings, capability statements, etc.) for easier reading and reference. The work group found this persuasive and resolved to "correct section numbering in the IG." This requires adding numeric section prefixes to all headings across all narrative pages of the IG so that readers and commenters can reference specific sections by number.

## Feature Proposal

### Problem Statement

The WHT Coordinated Registry Network (CRN) FHIR Implementation Guide uses markdown headings (`###`, `####`) across its narrative pages without any numeric section prefixes. This makes it difficult for readers and ballot commenters to reference specific sections when providing feedback, since there is no stable numbering scheme to cite. The IG contains substantial narrative content across multiple pages (overview, capability statements, guidance, profiles, etc.), and without section numbers, cross-referencing between and within pages is cumbersome.

### Proposed Change

Add hierarchical numeric section numbers as prefixes to all heading text across every narrative markdown page in the IG. The numbering scheme should be hierarchical and consistent across the entire IG, with top-level page sections numbered sequentially and sub-sections numbered with dot-notation (e.g., "1.1", "1.1.1"). The Table of Contents page (`toc.md`) should also be updated to include the corresponding section numbers.

For example, on `crn-overview.md`:
- `### Background` → `### 2.1 Background`
- `### Purpose and Goals of the Project` → `### 2.2 Purpose and Goals of the Project`
- `### Scope of the Project` → `### 2.3 Scope of the Project`

The top-level page numbering convention across the IG would be:
1. Home (`index.md`)
2. Overview (`crn-overview.md`)
3. Mappings and Profiles (`profiles.md`)
4. Capability Statements (`capstatements.md`)
5. Terminology (`terminology.md`)
6. Implementation Guidance (`guidance.md`)
7. Downloads (`downloads.md`)
8. History (`history.md`)

### Design Rationale

Section numbering is a standard editorial practice for specifications and implementation guides. It enables precise references in ballot comments (e.g., "Section 4.2.1 should clarify..."), improves navigability, and aligns with how other FHIR IGs handle section identification. The numbering is applied to the markdown heading text itself rather than through CSS auto-numbering, ensuring the numbers appear in all rendered formats and in the auto-generated `{:toc}` tables of contents.

## Implementation Plan

### Prerequisites

- Clone or fork the [HL7/coordinated-registry-network](https://github.com/HL7/coordinated-registry-network) repository (note: this repository is not present in the local cache under `cache/github/repos/`)
- Agree on a numbering scheme with the work group (the scheme below follows the page order in `toc.md`)

### Step-by-Step Tasks

#### HL7/coordinated-registry-network

1. **Establish the top-level numbering scheme**
   - File: `source/pages/toc.md`
   - Action: Add numeric prefixes to each Table of Contents entry
   - Details: Update the `<ul>/<li>` list items to include section numbers:
     - 1 — Home
     - 2 — Overview
     - 3 — Mappings and Profiles
     - 4 — Capability Statements
     - 5 — Terminology
     - 6 — Implementation Guidance
     - 7 — Downloads
     - 8 — History

2. **Number sections in Home page**
   - File: `source/pages/index.md`
   - Action: Add numeric prefixes to `###` headings
   - Details:
     - `### Introduction` → `### 1.1 Introduction`
     - `### Guidance to the readers` → `### 1.2 Guidance to the readers`

3. **Number sections in Overview page**
   - File: `source/pages/crn-overview.md`
   - Action: Add numeric prefixes to all `###` and `####` headings
   - Details: Number all headings sequentially (e.g., `### 2.1 Background`, `### 2.2 Purpose and Goals of the Project`, etc.) and sub-sections with dot notation (e.g., `#### 2.2.1 ...`)

4. **Number sections in Mappings and Profiles page**
   - File: `source/pages/profiles.md`
   - Action: Add numeric prefixes to all `####` headings
   - Details:
     - `#### Mappings of CDEs to FHIR` → `#### 3.1 Mappings of CDEs to FHIR`
     - `#### Profiles` → `#### 3.2 Profiles`
     - `#### Extensions` → `#### 3.3 Extensions`

5. **Number sections in Capability Statements page**
   - File: `source/pages/capstatements.md`
   - Action: Add numeric prefixes to all `###` and `####` headings
   - Details: This is the largest narrative page. Number the top-level actor sections (e.g., `### 4.1 CRN Instrument and Metadata Repository Conformance Requirements`) and their sub-sections (e.g., `#### 4.1.1 Behavior and Formats`, `#### 4.1.2 Profiles and Interactions`, `#### 4.1.3 Search and Other Parameters`, `#### 4.1.4 Security Requirements`). Repeat for each actor: External CRN Data Collection System (4.2), EHR (4.3), GUDID Database (4.4), Research Registry (4.5), etc.

6. **Number sections in Terminology page**
   - File: `source/pages/terminology.md`
   - Action: Add numeric prefixes to headings
   - Details: Apply `5.x` numbering to all headings

7. **Number sections in Implementation Guidance page**
   - File: `source/pages/guidance.md`
   - Action: Add numeric prefixes to all `###` and `####` headings
   - Details: Number sequentially (e.g., `### 6.1 Introduction`, `#### 6.2 CRN Instrument and Metadata Repository Implementation`, `#### 6.3 External CRN Data Collection System Implementation`, etc.)

8. **Number sections in Downloads page**
   - File: `source/pages/downloads.md`
   - Action: Add numeric prefixes to headings
   - Details: Apply `7.x` numbering to all headings

9. **Number sections in History page**
   - File: `source/pages/history.md`
   - Action: Add numeric prefixes to headings
   - Details: Apply `8.x` numbering to all headings

### Testing Considerations

- **IG Build Verification:** Run the FHIR IG publisher (`_genonce.sh` or equivalent) against the modified repository to confirm all pages render correctly and no build errors are introduced.
- **Anchor Link Verification:** Verify that any internal cross-page anchor links (e.g., `capstatements.html#gudid-database-conformance-requirements`) still resolve correctly, since adding section numbers to heading text will change auto-generated HTML anchor IDs. Update any internal links that reference heading anchors.
- **Visual Inspection:** Review each rendered HTML page to confirm section numbers appear correctly in both headings and auto-generated `{:toc}` tables of contents.

### Open Questions

1. **Numbering scheme confirmation:** The proposed numbering scheme assigns top-level numbers based on the page order in `toc.md`. The work group should confirm whether this ordering and the depth of numbering (e.g., two vs. three levels) meets their needs.
2. **Anchor ID stability:** Adding section numbers to headings will change the auto-generated HTML anchor IDs (e.g., `#background` → `#21-background`). Any external systems or documents linking to specific section anchors will need to be updated. The work group should assess whether there are known external links that would break.
3. **Repository staleness:** The HL7/coordinated-registry-network repository has not had commits since March 2019 (pre-dating the ticket resolution in August 2019). It is unclear whether the IG is still actively maintained or if the work group intends to resume development to apply this change.
