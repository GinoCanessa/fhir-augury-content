## Resolution Summary

The ticket's resolution description is, verbatim, **"fix the typo"** — i.e. change
the misspelled string `Acetaminophin` to `Acetaminophen` in the
MedicinalProductDefinition example list. The applied vote is `Persuasive` and
the change is classified as a `Technical Correction`. While the ticket
specifically calls out the `Acetaminophin product, pack and tablet (using
contained resources)` description string, the implementing committer (Rik
Smithies) confirmed via the linked commit that a second, related typo
(`Acetaminopen`) exists in the same example file and was corrected as part of
the same fix.

A commit implementing this resolution already exists in the repo cache
(`HL7/fhir@cdb85a4d`, message: *"FHIR-53866 changed spelling of Acetaminophin
to Acetaminophen (and Acetaminopen fixed also)"*, dated 2026-04-16). This plan
therefore documents what the change must contain so it can be re-applied or
verified.

## Feature Proposal

### Problem Statement

`source/medicinalproductdefinition/list-MedicinalProductDefinition-examples.xml`
ships a curated list of MedicinalProductDefinition examples that the FHIR
publisher renders into the resource page. One entry advertises an example as
`Acetaminophin product, pack and tablet (using contained resources)` — the
drug name is misspelled (`Acetaminophin` should be `Acetaminophen`). The
linked example resource itself, the file name, and the body of the example
all already use the correct spelling (`acetaminophen` /
`medicinalproductdefinition-acetaminophen-500mg-tablets-box-of-20.xml`),
making the misspelling in the listing both visible to readers and inconsistent
with the underlying example.

A neighbouring entry in the same `<entry>` further misspells the same drug as
`Acetaminopen 500mg box of 20 tablets` in the `<display>` value of the
referenced `MedicinalProductDefinition/Acetamin-500-20-generic`. The reporter
flagged only `Acetaminophin`; the second misspelling (`Acetaminopen`) is in
the same enclosing `<entry>` and was caught while applying the fix.

### Proposed Change

Edit one file and update two adjacent string values:

- **File:** `source/medicinalproductdefinition/list-MedicinalProductDefinition-examples.xml`
- **Change 1 (line 69 in the post-fix tree, originally line 69 of the
  pre-fix file):** Inside the `<entry>` whose `title` extension is
  `medicinalproductdefinition-acetaminophen-500mg-tablets-box-of-20`, the
  `description` extension's `valueString` must read:

  ```xml
  <valueString value="Acetaminophen product, pack and tablet (using contained resources)"/>
  ```

  (Replace `Acetaminophin` → `Acetaminophen`.)

- **Change 2 (line 76 in the post-fix tree):** In the same `<entry>`, the
  `<item>` whose `<reference>` is
  `MedicinalProductDefinition/Acetamin-500-20-generic` must have its
  `<display>` value read:

  ```xml
  <display value="Acetaminophen 500mg box of 20 tablets"/>
  ```

  (Replace `Acetaminopen` → `Acetaminophen`.)

No other files require changes. A repo-wide grep across `source/` for
`Acetaminophin` and `Acetaminopen` returns zero hits in the post-fix clone
(verified at HEAD `db79dcd`), confirming both misspellings live exclusively
in this single XML file. The dictionary file `source/spelling/english.dic`
already contains the correctly spelled `acetaminophen`.

### Design Rationale

- **Pure data-string fix.** No structural or schema change, no cardinality
  shift, no terminology binding change. The publisher reads
  `list-<Resource>-examples.xml` files as the authored source of the
  per-resource example listing, so editing the `valueString`/`display`
  attributes directly is the canonical way to correct rendered text on the
  resource page.
- **Consistency with the underlying example.** The referenced example file
  (`medicinalproductdefinition-acetaminophen-500mg-tablets-box-of-20.xml`)
  and its filename already use the correct spelling, so this is the only
  change needed to keep the listing consistent with the example it points
  to.
- **Including `Acetaminopen` in the same edit.** The reporter named only
  `Acetaminophin`, but the second typo is in the very next nested element of
  the same `<entry>`. Bundling both into one change avoids leaving a known
  typo in the immediately adjacent line and matches what the assignee
  actually committed under this ticket key.

### Step-by-Step Tasks

#### HL7/fhir

1. **Correct the `description` valueString for the Acetaminophen product
   listing**
   - File: `source/medicinalproductdefinition/list-MedicinalProductDefinition-examples.xml`
   - Action: In the `<entry>` whose nested `title` extension has
     `valueString="medicinalproductdefinition-acetaminophen-500mg-tablets-box-of-20"`,
     change the preceding `description` extension's `valueString` from
     `"Acetaminophin product, pack and tablet (using contained resources)"`
     to `"Acetaminophen product, pack and tablet (using contained
     resources)"`.
   - Details: Single-character substitution (`i` → `e`). Leave the
     surrounding XML structure, attribute order, indentation, and the
     extension URL
     (`http://hl7.org/fhir/build/StructureDefinition/description`)
     unchanged.

2. **Correct the `display` value on the referenced
   `Acetamin-500-20-generic` item**
   - File: `source/medicinalproductdefinition/list-MedicinalProductDefinition-examples.xml`
   - Action: In the same `<entry>`, in the `<item>` whose
     `<reference value="MedicinalProductDefinition/Acetamin-500-20-generic"/>`
     follows, change the `<display>` from
     `"Acetaminopen 500mg box of 20 tablets"` to
     `"Acetaminophen 500mg box of 20 tablets"`.
   - Details: Insert the missing `h` (`Acetaminopen` → `Acetaminophen`).
     Do not change the `<reference value="…"/>` — only the human-readable
     `<display>` is wrong. Preserve casing of the rest of the string and
     the `mg` / `box of 20 tablets` wording.

3. **Verify no other instance of either misspelling remains**
   - File: repository-wide
   - Action: Run a case-insensitive search across the clone
     (`git grep -in "acetaminophin\|acetaminopen"`) and confirm zero
     matches.
   - Details: This guards against silent regressions if either string was
     copy-pasted into other example files or narrative HTML pages. At the
     current clone HEAD `db79dcd`, this search returns no results — that
     is the expected post-fix state.
