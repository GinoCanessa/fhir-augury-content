## Resolution Description

> Change binding to THO-defined ValueSet with Preferred binding strength.

## Problem Statement

`ResearchStudy.progressStatus.state` currently has an **Example** binding to the FHIR-Core-defined ValueSet at `http://hl7.org/fhir/ValueSet/research-study-status`. The underlying CodeSystem and ValueSet have been migrated to HL7 Terminology (THO) at `http://terminology.hl7.org/ValueSet/research-study-status`, but the binding in the StructureDefinition was not updated before the R6 ballot. The binding strength should also be increased from Example to Preferred.

## Feature Proposal

Update the `ResearchStudy.progressStatus.state` element binding in the FHIR Core StructureDefinition to:

1. **Point to the THO-defined ValueSet** (`http://terminology.hl7.org/ValueSet/research-study-status`) instead of the FHIR-Core-defined ValueSet (`http://hl7.org/fhir/ValueSet/research-study-status`).
2. **Change binding strength** from `example` to `preferred`.
3. **Update the example resource** to reference the THO CodeSystem URL.

This aligns with the broader effort to migrate terminologies to THO and strengthens interoperability by making the binding Preferred, encouraging consistent use of standardized study status codes while still allowing extensions for specialized needs.

## Impact Analysis

### Breaking Change Assessment: **Low Risk**

- **Binding strength increase (Example → Preferred):** This is a **tightening** of the binding. Systems that previously used arbitrary codes for `progressStatus.state` will now be encouraged (but not required) to use codes from the THO ValueSet. Since Preferred is not Required or Extensible, this is a soft guidance change and will not cause validation failures for existing data.
- **ValueSet URL change:** The ValueSet URL changes from `http://hl7.org/fhir/ValueSet/research-study-status` to `http://terminology.hl7.org/ValueSet/research-study-status`. The content of both ValueSets is semantically identical (same codes). Systems referencing the old URL in their validation tooling may need to update, but the actual code values remain unchanged.
- **Example resource update:** The example file references the old FHIR-Core CodeSystem URL; updating it is a non-breaking cosmetic change.

## Detailed Implementation Plan

### Repository: HL7/fhir (FHIR Core)

#### Change 1: Update binding in StructureDefinition (REQUIRED)

**File:** `source/researchstudy/structuredefinition-ResearchStudy.xml`
**Lines:** 811–818

**Current content:**
```xml
      <binding>
        <extension url="http://hl7.org/fhir/StructureDefinition/elementdefinition-bindingName">
          <valueString value="ResearchStudyStudyStatus"/>
        </extension>
        <strength value="example"/>
        <description value="defn."/>
        <valueSet value="http://hl7.org/fhir/ValueSet/research-study-status"/>
      </binding>
```

**Updated content:**
```xml
      <binding>
        <extension url="http://hl7.org/fhir/StructureDefinition/elementdefinition-bindingName">
          <valueString value="ResearchStudyStudyStatus"/>
        </extension>
        <strength value="preferred"/>
        <description value="Codes that convey the current status of the research study."/>
        <valueSet value="http://terminology.hl7.org/ValueSet/research-study-status"/>
      </binding>
```

**Changes:**
1. `<strength value="example"/>` → `<strength value="preferred"/>`
2. `<valueSet value="http://hl7.org/fhir/ValueSet/research-study-status"/>` → `<valueSet value="http://terminology.hl7.org/ValueSet/research-study-status"/>`
3. `<description value="defn."/>` → `<description value="Codes that convey the current status of the research study."/>` (use the actual ValueSet description instead of the placeholder)

#### Change 2: Update example resource CodeSystem URL (RECOMMENDED)

**File:** `source/researchstudy/researchstudy-example-ctgov-study-record.xml`
**Lines:** 564, 573

The example resource references the old FHIR-Core CodeSystem URL. Update to the THO CodeSystem URL for consistency:

**Current** (line 564):
```xml
<system value="http://hl7.org/fhir/research-study-status"/>
```

**Updated:**
```xml
<system value="http://terminology.hl7.org/CodeSystem/research-study-status"/>
```

Apply the same change at line 573 (second `progressStatus` entry).

#### Change 3: Evaluate FHIR-Core ValueSet and CodeSystem (FOLLOW-UP)

The following files contain the FHIR-Core copies of the terminology that has been migrated to THO. With the binding now pointing to THO, these files may be candidates for deprecation or removal:

- `source/researchstudy/valueset-research-study-status.xml` — URL: `http://hl7.org/fhir/ValueSet/research-study-status`
- `source/researchstudy/codesystem-research-study-status.xml` — URL: `http://hl7.org/fhir/research-study-status`

**Note:** Deprecation/removal of these files should be coordinated with the FHIR build tooling maintainers and may require a separate ticket if not already covered. The `oids.ini` file (lines 802, 1444) also references these URLs and would need corresponding updates if the resources are retired.

### Repository: HL7/UTG (THO) — No Changes Required

The THO ValueSet and CodeSystem already exist and are correctly defined:

- **ValueSet:** `input/sourceOfTruth/fhir/valueSets/ValueSet-research-study-status.xml`
  - URL: `http://terminology.hl7.org/ValueSet/research-study-status`
  - Includes CodeSystem: `http://terminology.hl7.org/CodeSystem/research-study-status`
- **CodeSystem:** `input/sourceOfTruth/fhir/codeSystems/CodeSystem-research-study-status.xml`
  - URL: `http://terminology.hl7.org/CodeSystem/research-study-status`
  - Contains 17 concepts (overall-study, active, active-but-not-recruiting, etc.)
- Both are registered in `input/sourceOfTruth/control-manifests/fhir-Rendering.xml`

No changes are required in the UTG repository.
