## Resolution Summary

**Resolution:** Persuasive with Modification.

The committee agreed to extend the definitions of the `on-study` and `off-study`
codes in the `ResearchSubjectState` code system so that they cover retrospective
study use, but the change is **not** to be applied as a FHIR-Core ValueSet edit.
Instead it is being executed through the Terminology HL7 Org (THO / UTG)
process under tracker **UP-585** ("Change the ResearchSubjectState CodeSystem
terms for ResearchSubject").

The wording agreed in comments (with the editorial "the" added by the
assignee in comment 55218) is:

- **on-study** — append: *"In a retrospective study context: the start of the
  period during which a subject's data is included, observed, or considered
  as part of the study analysis."*
- **off-study** — append: *"In a retrospective study context: the end of the
  period during which a subject's data is included, observed, or considered
  as part of the study analysis."*

Per the resolution, no change is made to the FHIR-Core ResearchSubject
artifacts themselves; the codes already bound from
`ResearchSubject.progress.state` continue to flow through unchanged from
`http://terminology.hl7.org/CodeSystem/research-subject-state`.

## Feature Proposal

### Problem Statement

The `ResearchSubjectState` code system at
`http://terminology.hl7.org/CodeSystem/research-subject-state` (defined in
`CodeSystem-research-subject-state.xml` in HL7/UTG) currently defines:

- `on-study` — *"A person that is enrolled or registered on a study."*
- `off-study` — *"A person that has ended their participation on a study
  either because their treatment/observation is complete or through not
  responding, withdrawal, non-compliance and/or adverse event."*

Both definitions are framed exclusively from a prospective enrolment/
participation viewpoint. They do not describe how the same codes should be
interpreted in a **retrospective study**, where there is no enrolment event
and "on/off study" instead delimits the window during which the subject's
existing data is in scope of the analysis. Implementers reusing these codes
in retrospective analyses therefore have no normative anchor for that meaning.

### Proposed Change

The change is purely a **definition / description amendment** in the
`ResearchSubjectState` CodeSystem maintained in HL7/UTG. No code values are
added, removed, renamed, or rebound. The fix appends the agreed retrospective
clause to the existing definitions:

- `on-study` (concept code `on-study`):
  Replace the current `<definition value="A person that is enrolled or
  registered on a study."/>` with text combining the prospective definition
  with the retrospective clause — e.g.:

  > A person that is enrolled or registered on a study. In a retrospective
  > study context: the start of the period during which a subject's data
  > is included, observed, or considered as part of the study analysis.

- `off-study` (concept code `off-study`):
  Replace the current `<definition value="A person that has ended their
  participation on a study either because their treatment/observation is
  complete or through not responding, withdrawal, non-compliance and/or
  adverse event."/>` with text combining the prospective definition with the
  retrospective clause — e.g.:

  > A person that has ended their participation on a study either because
  > their treatment/observation is complete or through not responding,
  > withdrawal, non-compliance and/or adverse event. In a retrospective
  > study context: the end of the period during which a subject's data is
  > included, observed, or considered as part of the study analysis.

Display values (`On-study`, `Off-study`) and concept codes are unchanged.

Because UP-585 is described in comments as a broader CodeSystem overhaul
(removal/renaming of codes — see comments by reuben.daniels and
bryn.rhodes), the definition tweak from FHIR-46450 must be merged **into
the UP-585 change set** rather than applied as a parallel edit, so that
the two streams do not collide on the same `<concept>` blocks. The
resolution explicitly delegates the change to UP-585.

### Design Rationale

- **THO/UTG ownership:** `research-subject-state` is a THO-published
  CodeSystem (`http://terminology.hl7.org/...`), not a FHIR-Core
  CodeSystem, so the editorial source of truth is HL7/UTG. The committee
  rejected creating a FHIR-side ValueSet override, which keeps the
  semantics in one place and avoids divergence.
- **Definition extension, not a new code:** The retrospective meaning is
  semantically the same state ("subject is in scope of the study /
  subject is no longer in scope"), so the concept identity is preserved.
  Adding a new code (e.g. `on-study-retrospective`) would force every
  retrospective-study implementer to differentiate from prospective use,
  which is exactly what the reporter wanted to avoid.
- **Editorial cleanup:** The assignee's own comment normalises the wording
  ("the start of the period…") which should be the text actually applied.
- **Major-version implications:** Comment thread on UP-585 (bryn.rhodes,
  khalid_shahin) flags that other UP-585 changes warrant a major version
  bump; the FHIR-46450 contribution alone is non-breaking, but it inherits
  whatever version increment UP-585 lands with.
