## Resolution Summary

The example FHIR `AllergyIntolerance` resource on the "Value as Concept Pattern" page uses SNOMED CT code **294930007** for "Allergy to benzylpenicillin". This code is invalid — it does not exist in either the US or International Edition of SNOMED CT. The resolution directs that the code be corrected to **294499007**, which is the actual SNOMED CT code for "Allergy to benzylpenicillin".

> *Resolution description:* "We change the SNOMED code to the proper code."

---

## Feature Proposal

Replace every occurrence of the invalid SNOMED code `294930007` with the correct code `294499007` across all files in the `HL7/fhir-omop-ig` repository. This is a straightforward find-and-replace correction affecting narrative text, a JSON example resource, SQL examples, an HTML table, and an SVG diagram.

No structural, logical, or behavioral changes are required. The clinical meaning ("Allergy to benzylpenicillin"), display text, and the OMOP decomposition pattern (observation concept 439224 → value concept 1728416) remain unchanged.

### Additional Considerations

- **`observation_source_concept_id: 4222295`**: The SQL INSERT example and the page reference this OMOP concept_id as the source concept for the (invalid) SNOMED code. Since SNOMED 294930007 is invalid, this OMOP concept_id may also be incorrect. However, the ticket resolution only asks to fix the SNOMED code itself. If SNOMED 294499007 has a different OMOP concept_id, that value should also be updated, but that is **out of scope** for this ticket as written. This should be verified during implementation.
- **Display text "Allergy to benzylpenicillin"** remains correct for SNOMED 294499007, so no display text changes are needed.
