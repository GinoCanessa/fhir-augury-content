# Implementation Plan: FHIR-20967

**Title:** Remove sentence from section about NIH CDE Repository
**Status:** Resolved - change required
**Resolution:** Persuasive
**Work Group:** Biomedical Research & Regulation
**Specification:** US Coordinated Registry Network (CRN) (FHIR)
**Resolved:** 2019-08-28

---

## Resolution Summary

The resolution (Persuasive) requires replacing the existing paragraph in the CRN overview page under the "NLM's CDE Repository" section. The current paragraph contains a misleading sentence that references a "Questionnaire Representation of CDE" at the NLM, which was intended only as a visualization of the data elements — not an actual representation of the Questionnaire Resource in the IG.

The exact approved replacement wording from the resolution is:

> The National Library of Medicine (NLM) has made the CDEs publicly available for reuse in other activities. The full set of WHT-CRN CDEs can viewed in the NIH CDE Repository.

This removes the second sentence about accessing CDEs "as Questionnaire Resource" and replaces the entire paragraph with clearer, corrected language.

## Feature Proposal

### Problem Statement

The CRN Overview page (`crn-overview.html#nlms-cde-repository`) in the US Coordinated Registry Network Implementation Guide contains a paragraph about the NIH CDE Repository that includes a misleading sentence. The current text states:

> "The Common Data Elements The data elements are published by National Library of Medicine (NLM) at CDE Repository for others to use the data elements as part of their activities. The CDEs can also be accessed as Questionnaire Resource at the NLM's Questionnaire Representation of CDE."

The second sentence ("The CDEs can also be accessed as Questionnaire Resource at the NLM's Questionnaire Representation of CDE") is problematic because the form in the NIH CDE Repository was intended to be a visualization of the data elements, not an actual representation of the FHIR Questionnaire Resource as used in the IG. This causes confusion about the relationship between the CDE Repository's form view and the FHIR Questionnaire Resource defined in the implementation guide.

### Proposed Change

Replace the entire paragraph under the "NLM's CDE Repository" section heading in `source/pages/crn-overview.md` with the following text:

**Current text (to be removed):**
```
The [Common Data Elements](https://drive.google.com/file/d/1GGRZOJcLBAW_p_czcfaSe7xAirs8KPw6/view?usp=sharing) 
The data elements are published by National Library of Medicine (NLM) at [CDE Repository](https://cde.nlm.nih.gov/cde/search) for others to use the data elements as part of their activities. The CDEs can also be accessed as Questionnaire Resource at the NLM's [Questionnaire Representation of CDE](https://cde.nlm.nih.gov/formView?tinyId=XJwYSNJ4I).
```

**Replacement text:**
```
The National Library of Medicine (NLM) has made the CDEs publicly available for reuse in other activities. The full set of WHT-CRN CDEs can viewed in the [NIH CDE Repository](https://cde.nlm.nih.gov/cde/search).
```

Key changes:
- Removes the misleading sentence about accessing CDEs "as Questionnaire Resource"
- Removes the link to the NLM form view (`https://cde.nlm.nih.gov/formView?tinyId=XJwYSNJ4I`)
- Removes the link to the Google Drive Common Data Elements template file
- Retains a link to the NIH CDE Repository search page (`https://cde.nlm.nih.gov/cde/search`)
- Simplifies language to clearly state the CDEs are publicly available

**Note:** The resolution text says "can viewed" (likely a typo for "can be viewed"). The implementer should decide whether to fix this typo or preserve the exact approved wording. Recommended: use "can be viewed" to correct the grammatical error.

### Design Rationale

The change was proposed by the ticket reporter (Robin Taylor) who noted that the NLM CDE Repository's form view was being misrepresented as a FHIR Questionnaire Resource. The CDE Repository's form view is a visualization tool for browsing data elements, not a FHIR-conformant Questionnaire resource. Keeping the misleading sentence could cause implementers to incorrectly attempt to use the CDE Repository's form view as a FHIR Questionnaire, leading to interoperability issues.

The resolution simplifies the paragraph to focus on the availability of CDEs for reuse, which is the primary purpose of the section, without making claims about FHIR resource representations.
