## Description

UMC (Uppsala Monitoring Centre) are now a generator of substance names (from their UMC SRS-system) and have requested that they be added to the `substance-name-authority` code system.

## Resolution Description

> Add code 'UT' and display 'UMC Terminologies' to the CodeSystem.

---

## Feature Proposal

Add a new concept with code **`UT`** and display **`UMC Terminologies`** to the FHIR-defined CodeSystem `substance-name-authority` (`http://hl7.org/fhir/substance-name-authority`). This recognises the Uppsala Monitoring Centre as an official substance naming authority alongside existing entries (BAN, INN, USAN, USP, etc.). A ballot note should also be added to the SubstanceDefinition resource introduction to highlight this change for balloters.

### Breaking Changes

**None.** This is a purely additive change—a new concept in an existing CodeSystem. The ValueSet binding on `SubstanceDefinition.name.official.authority` is `preferred`, so implementers are not required to use codes from this set. No existing codes are modified or removed.

### Downstream / Cross-Cutting Concerns

- The ValueSet `substance-name-authority` automatically expands to include the new code because it includes the entire CodeSystem.
- Implementers using the `substance-name-authority` ValueSet in validation will accept `UT` after updating to the new version.
- No migration or data conversion is needed.

## Implementation Plan

### Change 1: Add `UT` concept to CodeSystem

**File:** `source/substancedefinition/codesystem-substance-name-authority.xml`
**Repository:** HL7/fhir

**Action:** Append a new `<concept>` element before the closing `</CodeSystem>` tag.

**Insertion (after the IUIS concept, before `</CodeSystem>`):**

```xml
  <concept>
    <code value="UT"/>
    <display value="UMC Terminologies"/>
  </concept>
```

**Context:** The CodeSystem currently contains 15 concepts (BAN, COSING, Ph.Eur., FCC, INCI, INN, JAN, JECFA, MARTINDALE, USAN, USP, PHF, HAB, PhF, IUIS). The new concept `UT` becomes the 16th entry.

### Change 2: Add ballot note to SubstanceDefinition introduction

**File:** `source/substancedefinition/substancedefinition-introduction.xml`
**Repository:** HL7/fhir

**Action:** Insert a ballot note blockquote after the medication-definition-module link paragraph, highlighting the addition for balloters.

**Insertion (after line 5, before the main descriptive paragraph):**

```html
    <blockquote class="ballot-note" id="bn1">
    <p><b>Note to Balloters:</b> To ensure this resource is ready for Normative status, we are seeking ballot comment on the substantive content. The key changes made since the last R6 ballot are:</p>
    <ul>
        <li>Updates to the following elements:</li>
            <ul>
                <li>SubstanceDefinition.name.official.authority, value set (and code system) substance-name-authority has been extended with code: "UT", UMC Terminologies</li>
            </ul>
    </ul>
    </blockquote>
```
