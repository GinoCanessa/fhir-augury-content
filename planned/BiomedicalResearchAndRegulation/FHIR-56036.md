## Resolution Summary

**Resolution:** Persuasive.

**Resolution wording (from Jira):** *"Add code 'UT' and display 'UMC Terminologies' to the CodeSystem."*

The change extends the `http://hl7.org/fhir/substance-name-authority` CodeSystem with one new concept so that Uppsala Monitoring Centre (UMC) — which now generates substance names from its UMC SRS system — can be cited as a naming authority on `SubstanceDefinition.name.official.authority` (and any other element bound to the `substance-name-authority` ValueSet).

| Field | Value |
|-|-|
| `code` | `UT` |
| `display` | `UMC Terminologies` |
| Parent CodeSystem URL | `http://hl7.org/fhir/substance-name-authority` |
| OID | `2.16.840.1.113883.4.642.4.2095` |

The ValueSet `http://hl7.org/fhir/ValueSet/substance-name-authority` includes the entire CodeSystem (`<compose><include><system value=".../substance-name-authority"/>`), so the new code is automatically in scope without a ValueSet edit.

## Feature Proposal

### Problem Statement

`SubstanceDefinition.name.official.authority` is bound to the `substance-name-authority` ValueSet, which enumerates recognised official naming authorities (BAN, INN, JAN, USAN, USP, Ph.Eur., COSING, FCC, INCI, JECFA, MARTINDALE, PHF, HAB, PhF, IUIS). UMC (Uppsala Monitoring Centre) now publishes substance names from its SRS system and has requested to be listed as a naming authority. Without a code, an implementer using SubstanceDefinition cannot record `name.official.authority` as UMC without resorting to a non-standard / extension code.

### Proposed Change

Add a single concept to the `substance-name-authority` CodeSystem:

```xml
<concept>
  <code value="UT"/>
  <display value="UMC Terminologies"/>
</concept>
```

Append the concept after the existing `IUIS` concept inside `source/substancedefinition/codesystem-substance-name-authority.xml`, immediately before the closing `</CodeSystem>` element. No `definition` was added by precedent — the existing concepts in this CodeSystem use only `code` and `display` (e.g., `BAN`, `INN`, `IUIS`); follow that pattern.

In addition, add a balloter note to the SubstanceDefinition Scope and Usage page so reviewers of the next R6 ballot are explicitly directed at this CodeSystem extension. Insert a `<blockquote class="ballot-note" id="bn1">` immediately after the `<h2>Scope and Usage</h2>` block in `source/substancedefinition/substancedefinition-introduction.xml`, listing the change to `SubstanceDefinition.name.official.authority` (value set / code system `substance-name-authority` extended with `UT` — UMC Terminologies).

No change is needed to:
- `source/substancedefinition/valueset-substance-name-authority.xml` — composes by `<include><system .../></include>` and inherits the new code.
- `SubstanceDefinition` StructureDefinition itself — element `name.official.authority` and its binding are unchanged.

### Design Rationale

- The existing CodeSystem is `complete` and lists naming authorities by short mnemonic (`BAN`, `INN`, `JAN`, …). `UT` (for "UMC Terminologies") is consistent with that mnemonic style.
- Adding to a `complete` `active`/`normative` CodeSystem (FMM 1) is additive; existing producers and consumers continue to validate. It is the minimal change that satisfies the resolution.
- Editing the include-by-system ValueSet is unnecessary and would risk drift; the existing `<include><system>` form is the idiomatic FHIR pattern for "all codes of this system".
- The introduction-page ballot note follows the established R6 ballot-comment convention used elsewhere in `source/<resource>/<resource>-introduction.xml` (a `<blockquote class="ballot-note">` block) and points balloters at the substantive change so it is not silently shipped under "Normative" status.

## Implementation Plan

### Prerequisites

- None. The change is self-contained in `HL7/fhir`. No upstream UTG / extensions / IG ticket must clear first.

### Step-by-Step Tasks

#### HL7/fhir

1. **Add the `UT` concept to the `substance-name-authority` CodeSystem.**
   - File: `source/substancedefinition/codesystem-substance-name-authority.xml`
   - Action: Insert a new `<concept>` block immediately before the closing `</CodeSystem>` tag, after the existing `IUIS` concept.
   - Details: Use exactly the form
     ```xml
     <concept>
       <code value="UT"/>
       <display value="UMC Terminologies"/>
     </concept>
     ```
     Do not add a `<definition>` (none of the sibling concepts carry one — match the existing style).
     Do not change `<version>`, `<date>`, `<status>`, `<content>`, `<caseSensitive>`, or any extension; the publisher manages versioning at release time and the file is `content="complete"` already.

2. **Add a ballot note to the SubstanceDefinition Scope and Usage page.**
   - File: `source/substancedefinition/substancedefinition-introduction.xml`
   - Action: Immediately after the existing `<h2>Scope and Usage</h2>` paragraph (the `<p style="border:2px; …">` "module page" callout) and **before** the next `<p>SubstanceDefinition is used for rich descriptions…</p>`, insert:
     ```xml
     <blockquote class="ballot-note" id="bn1">
     <p><b>Note to Balloters:</b> To ensure this resource is ready for Normative status, we are seeking ballot comment on the substantive content. The key changes made since the last R6 ballot are:</p>
     <ul>
         <li>Updates to the following elements:</li>
             <ul>
                 <li>SubstanceDefinition.name.official.authority, value set (and code system) substance-name-authority has been extended with code: "UT", UMC Terminologies</li>
             </ul>
     </ul>
     </blockquote>
     ```
   - Details: Mirrors the ballot-note convention used elsewhere in the spec. If a `bn1` block already exists on the page when the work begins, append the bullet under its `<ul>` rather than introducing a duplicate `id="bn1"`.

3. **Verify no companion ValueSet edit is needed.**
   - File: `source/substancedefinition/valueset-substance-name-authority.xml`
   - Action: Confirm the file's `<compose>` is exactly `<include><system value="http://hl7.org/fhir/substance-name-authority"/></include>` (no enumerated `<concept>` list). If so, leave untouched.
   - Details: Documented in the briefing as an additive-include style ValueSet; the new code is picked up automatically.
