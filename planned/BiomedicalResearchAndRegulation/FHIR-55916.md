## Resolution Description

> Add a new extension in the Extensions Pack for ResearchStudy with the extension name Blinded Role (url path blinded-role) with valueCodeableConcept datatype.
>
> Create a value set for listing in THO that includes the values expected in M11 and CTIS specifications.

**Note:** Per Jira comments (2026-03-24), discussion with Bryn concluded the value set can be created in the Extensions Pack instead of THO.

## Background & Motivation

The `ResearchStudy.studyDesign` element uses CodeableConcept values for study design characteristics including blinding concepts (e.g., "blinding of study participants" from SEVCO). However, for EU CTIS study records and ICH M11 regulatory reporting, there is a separate requirement to list the **roles** that are blinded — such as "Investigator", "Care Provider", "Participant" — rather than the study design concept itself.

The existing `studyDesign` value set (from SEVCO) describes *what kind of blinding* is used as a study design process (e.g., SEVCO:01060 "blinding of study participants"), while the proposed extension captures *which roles* are blinded using NCI Thesaurus role codes. These are complementary but semantically distinct.

The BRIDG 5.1 mapping for `ResearchStudy.studyDesign` already references `InterventionalStudyProtocol.blindedRoleCode`, confirming this is a recognized concept that needs distinct representation.

## Feature Proposal

### New Extension: `researchStudy-blindedRole`

Add a simple extension on `ResearchStudy` that carries a `valueCodeableConcept` identifying a role that is blinded in the study. The extension is repeating (0..*) to allow listing multiple blinded roles.

- **Extension URL:** `http://hl7.org/fhir/StructureDefinition/researchStudy-blindedRole`
- **URL Path:** `blinded-role` (per the ticket)
- **Context:** `ResearchStudy`
- **Data Type:** `valueCodeableConcept`
- **Cardinality:** 0..*
- **Binding Strength:** extensible
- **Work Group Code:** `brr`

### New ValueSet: `research-study-blinded-role`

Create a value set containing NCI Thesaurus codes for trial blinding roles, sourced from the ICH M11 Technical Specification and EU CTIS requirements.

- **ValueSet URL:** `http://hl7.org/fhir/ValueSet/research-study-blinded-role`
- **System:** `http://ncicb.nci.nih.gov/xml/owl/EVS/Thesaurus.owl` (NCI Thesaurus)
- **7 concepts** (per final Jira comment combining M11 + CTIS + Statistician):

| NCI C-Code | Display | Definition |
|------------|---------|------------|
| C142710 | Participant | A member of the clinical study population from whom data are being collected. |
| C17445 | Care Provider | The primary person in charge of the care of a patient, usually a family member or a designated health care professional. |
| C25936 | Investigator | A person responsible for the conduct of the clinical trial at a trial site. |
| C207599 | Outcomes Assessor | The individual who evaluates the outcome(s) of interest. |
| C51877 | Statistician | A person responsible for the compilation, organization, and analysis of mathematical data. |
| C70793 | Sponsor | An individual, company, institution, or organization that takes responsibility for the initiation, management, and/or financing of a clinical study. |
| C48660 | Not Applicable | Determination of a value is not relevant in the current context. |

## Detailed Implementation Plan

### Repository: HL7/fhir-extensions

All changes are in the `input/definitions/ResearchStudy/` directory, following existing patterns from sibling extensions in the same folder.

#### File 1: `input/definitions/ResearchStudy/StructureDefinition-researchStudy-blindedRole.xml` (NEW)

Create a new StructureDefinition XML file for the extension. Use the pattern from `StructureDefinition-encounter-reasonCancelled.xml` (simple valueCodeableConcept extension) adapted to the BRR work group and ResearchStudy context.

**Key elements:**
- `id`: `researchStudy-blindedRole`
- `structuredefinition-wg` extension: `brr`
- `structuredefinition-fmm` extension: `1` (new extension, start at FMM 1)
- `structuredefinition-standards-status` extension: `trial-use`
- `url`: `http://hl7.org/fhir/StructureDefinition/researchStudy-blindedRole`
- `name`: `RSBlindedRole`
- `title`: `ResearchStudy Blinded Role`
- `publisher`: `HL7 International / Biomedical Research and Regulation`
- `contact` telecom URL: `http://www.hl7.org/Special/committees/rcrim`
- `description`: `A role that is blinded in the research study, identifying which participants or personnel are unaware of intervention assignments.`
- `context.expression`: `ResearchStudy`
- Differential:
  - `Extension` element: min=0, max=* (repeating — multiple roles can be blinded)
  - `Extension.extension`: max=0 (no sub-extensions)
  - `Extension.url`: fixedUri to the extension URL
  - `Extension.value[x]`: min=1, type=CodeableConcept, binding strength=extensible to `http://hl7.org/fhir/ValueSet/research-study-blinded-role`
- OID identifier: needs assignment (placeholder `urn:oid:2.16.840.1.113883.4.642.5.XXXX`)

**Template reference:** `input/definitions/Encounter/StructureDefinition-encounter-reasonCancelled.xml` (lines 1–76) and `input/definitions/ResearchStudy/StructureDefinition-researchStudy-studyRegistration.xml` (for BRR metadata conventions).

#### File 2: `input/definitions/ResearchStudy/ValueSet-research-study-blinded-role.xml` (NEW)

Create a new ValueSet XML file with NCI Thesaurus concepts enumerated inline.

**Key elements:**
- `id`: `research-study-blinded-role`
- `meta.profile`: `http://hl7.org/fhir/StructureDefinition/shareablevalueset`
- `structuredefinition-wg` extension: `brr`
- `structuredefinition-standards-status` extension: `trial-use`
- `structuredefinition-fmm` extension: `1`
- `url`: `http://hl7.org/fhir/ValueSet/research-study-blinded-role`
- `name`: `ResearchStudyBlindedRole`
- `title`: `Research Study Blinded Role`
- `publisher`: `HL7 International / Biomedical Research and Regulation`
- `description`: `Codes identifying roles that are blinded in a research study, based on ICH M11 and EU CTIS specifications.`
- `jurisdiction`: World (code `001`)
- `compose.include`:
  - `system`: `http://ncicb.nci.nih.gov/xml/owl/EVS/Thesaurus.owl`
  - `valueset-systemName` extension: `NCI Thesaurus`
  - 7 enumerated `concept` entries (C142710, C17445, C25936, C207599, C51877, C70793, C48660) each with `code`, `display`, and `valueset-concept-definition` extension
- OID identifier: needs assignment (placeholder `urn:oid:2.16.840.1.113883.4.642.3.XXXX`)

**Template reference:** `source/device/valueset-device-safety.xml` in HL7/fhir (lines 1–103) for the NCI Thesaurus concept enumeration pattern.

### OID Assignment

Both new resources need OID assignments:
- StructureDefinition OID: `urn:oid:2.16.840.1.113883.4.642.5.XXXX` (5-series for extensions)
- ValueSet OID: `urn:oid:2.16.840.1.113883.4.642.3.XXXX` (3-series for value sets)

These should be assigned during the build/publishing process per HL7 conventions.
