**Title:** Change mapping of Hospital Discharge
**Status:** Resolved - change required
**Resolution:** Persuasive

**Resolved:** 2019-08-28

## Resolution Summary

The resolution is **Persuasive**: the tracker agrees with the submitter's rationale. The approved resolution states:

> Correct the mapping of Hospital.Discharge. Rename the data element as Hospital Discharge Date. The mapping should be Encounter.period.end.

The current CDE-to-FHIR mapping table incorrectly maps the "Hospital Discharge" Harmonized Common Data Element to the full `Encounter.period` (a Period data type representing the interval from admission to discharge). Since "Hospital Discharge" refers specifically to the discharge date/time — not the full admission-to-discharge interval — the mapping must be corrected to point to the `.end` component of the period.

## Feature Proposal

### Problem Statement

In the CRN Implementation Guide's CDE-to-FHIR mapping table (`CDEs_FHIR.html`), the Harmonized Common Data Element "Hospital Discharge" is currently mapped to `Encounter.period`. However, `Encounter.period` is a `Period` data type representing the entire time interval from hospital admission to discharge. Mapping a single discharge date/time concept to the full period is semantically incorrect — `Encounter.period` captures both admission and discharge as a range.

The data element name "Hospital Discharge" is also ambiguous: it could refer to the event of discharge or the discharge date. The resolution clarifies that the intended CDE represents a date, so the name should be explicit.

### Proposed Change

Two modifications to the CDE-to-FHIR mapping table:

1. **Rename the CDE label** from `Hospital Discharge` → `Hospital Discharge Date`
2. **Correct the FHIR mapping** from `Encounter.period` → `Encounter.period.end`

The FHIR Resource/Profile reference (`us-core-encounter`) remains unchanged.

### Design Rationale

- `Encounter.period` is a `Period` data type with `.start` (admission dateTime) and `.end` (discharge dateTime). Mapping only the discharge date to the full period misrepresents the data element's semantics.
- `Encounter.period.end` precisely captures the hospital discharge dateTime, which is what the CDE represents.
- Renaming to "Hospital Discharge Date" removes ambiguity and aligns the CDE label with its narrower semantic scope (a dateTime, not an event or interval).
- This aligns with standard clinical informatics practice where admission and discharge are separate, distinct data elements.

- **None.** This is a documentation/mapping correction within an Implementation Guide. It does not change any StructureDefinitions, profiles, or FHIR resource definitions. It corrects the semantic mapping guidance published in the IG.
- Implementers who relied on the previous (incorrect) mapping of "Hospital Discharge" → `Encounter.period` should update their implementations to use `Encounter.period.end` instead, but this is a clarification of the intended mapping rather than a structural API change.

### Related Specifications

- **US Core Encounter Profile (`us-core-encounter`):** The CRN IG references this profile for the Hospital Discharge mapping. The profile inherits `Encounter.period` from the base Encounter resource. No changes needed to US Core.
- **FHIR Core Encounter Resource (`Encounter`):** Defines `Encounter.period` as a `Period` element representing the start and end time of the encounter. The `.end` sub-element captures when the encounter ended (i.e., discharge). No changes needed to the core resource.
