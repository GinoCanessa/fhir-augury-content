## Resolution Summary

The ballot commenter (Mitra Rocca) recommended adding BRIDG (Biomedical Research Integrated Domain Group) as one of the in-scope artifacts for the CDISC Lab FHIR IG, alongside the existing CDASH, SDTM, and CDISC LAB model standards. The resolution was **Persuasive** (voted 17-0-0), classified as non-substantive / enhancement.

The work group agreed to implement this by:
1. Adding a link to the existing BRIDG-to-CDASH/SDTM mapping.
2. Creating a new mapping to cover BRIDG-to-CDISC LAB XML.
3. Adding both to the Overview page and Downloads page.

## Feature Proposal

### Problem Statement

The CDISC Lab FHIR IG currently maps FHIR resources to three CDISC standards (LAB model, CDASH, SDTM) but does not formally include BRIDG as an in-scope data standard, despite the Overview page already containing a dedicated BRIDG section with background information and instructions for browsing the BRIDG model. The "Project Data Standards" table in `overview.md` lists only LAB, CDASH, and SDTM. This omission makes it harder for implementers to understand the relationship between FHIR laboratory data and BRIDG, which serves as BR&R's domain information model and semantic foundation.

### Proposed Change

1. **Add BRIDG to the "Project Data Standards" table** in `source/pages/overview.md` — adding a row for BRIDG with a link to the BRIDG model website and its description as the semantic domain model.
2. **Add a link to the existing BRIDG-to-CDASH/SDTM mapping** on both the Overview page and Downloads page. This mapping already exists as a BRIDG deliverable and should be referenced with a hyperlink.
3. **Create a new BRIDG-to-CDISC LAB XML mapping document** — either as an Excel spreadsheet or similar artifact that maps BRIDG model entities/attributes to the CDISC LAB XML standard elements. Add this as a downloadable artifact.
4. **Update the Downloads page** (`source/pages/downloads.md`) to include links to both BRIDG mapping artifacts under a new "BRIDG Mappings" section.
5. **Update the scope description** in the Overview's "FHIR to CDISC Laboratory Data Mapping" subsection to mention BRIDG alongside LAB, CDASH, and SDTM.

### Design Rationale

BRIDG is already referenced in the IG's Overview page as BR&R's domain information model and semantic foundation. Formally adding it to scope aligns the IG's stated standards with the broader BR&R strategy. The BRIDG model provides the conceptual framework that underpins the CDISC standards mapped in this IG, and making this relationship explicit benefits implementers who need to trace data semantics across standards. The existing BRIDG section in `overview.md` already provides navigational guidance to the BRIDG model; this change formalizes the relationship by adding BRIDG to the data standards table and providing downloadable mapping artifacts.

## Implementation Plan

### Prerequisites

1. Obtain or locate the existing BRIDG-to-CDASH/SDTM mapping document. This is an external BRIDG deliverable available via the BRIDG model website at [https://bridgmodel.nci.nih.gov/](https://bridgmodel.nci.nih.gov/) under the CDISC Views package.
2. Review the BRIDG model CDISC Lab Model v1.0.1 diagram and its associated metadata Excel report (accessible via the BRIDG model browser as described in the existing `overview.md` BRIDG section).
3. Coordinate with FHIR-24125 implementation, as both tickets modify the Overview and Downloads pages.

### Step-by-Step Tasks

#### Task 1: Update "Project Data Standards" table in Overview

**File:** `source/pages/overview.md`

Add a new row for BRIDG to the "Project Data Standards" table (currently at approximately line 75 of the file):

```markdown
| CDISC Standard  | Description of Standard |
:----------------|-------------------------------------------------------------:|
| Laboratory Data Model (LAB) | [LAB Description from CDISC](https://www.cdisc.org/standards/data-exchange/lab) |
| CDASH | [CDASH Description from CDISC](https://www.cdisc.org/standards/foundational/cdash) |
| SDTM | [SDTM Description from CDISC](https://www.cdisc.org/standards/foundational/sdtm) |
| BRIDG | [BRIDG Model](https://bridgmodel.nci.nih.gov/) |
```

#### Task 2: Update mapping description to include BRIDG

**File:** `source/pages/overview.md`

In the "FHIR to CDISC Laboratory Data Mapping" subsection (under "Purpose and Goals"), update the description paragraph to mention BRIDG. Change:

> "FHIR resources are compared (mapped) to three existing CDISC standards"

to:

> "FHIR resources are compared (mapped) to four existing CDISC standards"

And add BRIDG to the list: "the CDISC LAB model, CDASH LB Domain variables, SDTM LB variables, and the BRIDG model."

#### Task 3: Add BRIDG mapping links to the BRIDG section

**File:** `source/pages/overview.md`

In the existing "### BRIDG" section, add links to:
- The existing BRIDG-to-CDASH/SDTM mapping (external URL from BRIDG model site)
- The new BRIDG-to-CDISC LAB XML mapping (internal link to new artifact once created)

#### Task 4: Create BRIDG-to-CDISC LAB XML mapping spreadsheet

**New file:** `source/pages/BRIDG_to_CDISC_LAB_mapping.xlsx` (or similar)

Create a mapping spreadsheet that maps BRIDG model entities and attributes from the CDISC Lab Model v1.0.1 diagram to the CDISC LAB XML standard elements. The structure should follow the pattern of the existing `LAB_LB_FHIR_mapping_and_gaps_for_IG.xlsx` mapping file, with columns such as:
- BRIDG Class/Attribute
- CDISC LAB XML Element/Attribute
- Mapping Notes/Gaps
- Data Type

This requires domain expertise and access to the BRIDG model metadata Excel report.

#### Task 5: Update Downloads page

**File:** `source/pages/downloads.md`

Add a new section for BRIDG mappings after the existing use case sections:

```markdown
#### BRIDG Mappings

BRIDG-to-CDASH/SDTM Mapping: [BRIDG to CDASH/SDTM Mapping](https://bridgmodel.nci.nih.gov/) (available via BRIDG model CDISC Views)<br/>
BRIDG-to-CDISC LAB XML Mapping: [BRIDG_to_CDISC_LAB_mapping.xlsx](BRIDG_to_CDISC_LAB_mapping.xlsx)<br/>
```

#### Task 6: Optionally update Mappings and Profiles page

**File:** `source/pages/mappingsandprofiles.md`

Update the introductory paragraph to mention BRIDG as a fourth standard. Change:

> "FHIR resources are compared (mapped) to three existing CDISC standards that are used in clinical research laboratory data exchange: the CDISC LAB model, CDASH LB Domain variables, and SDTM LB variables."

to:

> "FHIR resources are compared (mapped) to four standards used in clinical research laboratory data exchange: the CDISC LAB model, CDASH LB Domain variables, SDTM LB variables, and the BRIDG model."

### Open Questions

1. **BRIDG-to-CDASH/SDTM mapping URL:** The exact URL for the existing BRIDG-to-CDASH/SDTM mapping needs to be confirmed. The BRIDG model website has undergone changes; the current link format and availability of the CDISC Views package should be verified.
2. **BRIDG model version:** The existing BRIDG section references BRIDG Model 5.3.1. Confirm whether a newer version of the BRIDG model should be referenced, or if 5.3.1 remains the target version.
3. **Mapping spreadsheet authorship:** The BRIDG-to-CDISC LAB XML mapping spreadsheet requires domain expertise to create. Identify who will author this artifact — this may require coordination with the BRIDG working group or NCI.
4. **Coordination with FHIR-24125:** FHIR-24125 also modifies the Overview and Downloads pages (moving use case content from Downloads to Overview). These changes should be sequenced or merged together to avoid conflicts.
5. **IG tooling migration:** The IG currently uses an older Jekyll-based IG Publisher toolchain (`ig.json` configuration). If the IG is being migrated to the newer `sushi`/FSH-based toolchain, the file structure and page locations may differ from what is documented here.
