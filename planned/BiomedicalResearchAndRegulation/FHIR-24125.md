## Resolution Summary

The ballot comment (submitted by Mitra Rocca during the September 2019 ballot cycle) identified two issues with the CDISC Lab IG:

1. The Overview page mentions Use Cases but does not include hyperlinks to the actual use case materials (e.g., downloadable mapping spreadsheets and code).
2. The Downloads page contains detailed use case descriptions that belong in the Overview → Use Cases section. The Downloads page should be simplified to contain only hyperlinks to downloadable files, without lengthy descriptions.

The resolution was **Persuasive** (voted 17-0-0 by Christi Denny/Myron Finseth), categorized as a non-substantive clarification. The workgroup agreed to move content from the Downloads page to the Overview page and limit the Downloads page to hyperlinks.

> **Note — Implementation Already Applied:** Commit [`eda3ecb6`](https://github.com/HL7/cdisc-lab/commit/eda3ecb6edaefadf32f45addc7a029bdc28f11d3) (2020-02-28, "Updates to overview and downloads pages") and commit [`d4eb4997`](https://github.com/HL7/cdisc-lab/commit/d4eb4997f8144e7fd1e727b27c4260c38839020c) (2020-05-08, "Updates per Sept 2019 ballot comments") appear to have already implemented this ticket. The current state of the repository reflects the requested changes. The Jira ticket was formally resolved later (2021-10-27) due to a Jira status bug requiring re-open/re-close. The plan below documents what was done and what should be verified.

## Feature Proposal

### Problem Statement

In the September 2019 ballot version of the CDISC Lab IG (`http://hl7.org/fhir/uv/cdisc-lab/2019SEP/`), the Downloads page contained detailed use case descriptions alongside download links. Meanwhile, the Overview page's Use Cases section referenced use cases but did not link to the actual materials. This created a confusing user experience where:

- Readers looking for use case context on the Overview page found no actionable links
- Readers visiting the Downloads page encountered narrative content better suited for the Overview
- The IG's information architecture did not follow standard IG conventions of separating narrative from downloads

### Proposed Change

1. **Move use case descriptions** from `source/pages/downloads.md` to the Use Cases section in `source/pages/overview.md`
2. **Add hyperlinks** in the Overview → Use Cases section pointing to the actual downloadable use case materials (mapping spreadsheets, code)
3. **Simplify the Downloads page** to contain only labeled hyperlinks to downloadable files (mapping spreadsheets, narrative document, domain model, sample data, code, IG zip), without detailed use case descriptions

### Design Rationale

- Follows standard FHIR IG information architecture: Overview pages provide narrative context; Downloads pages provide file links
- Improves discoverability by placing use case descriptions where readers expect them (Overview)
- Reduces content duplication between pages
- Non-substantive clarification — no changes to profiles, mappings, or conformance artifacts

2. Ensure Downloads page links back to Overview page for use case descriptions
3. **Current state (verified):** Bidirectional links are in place
