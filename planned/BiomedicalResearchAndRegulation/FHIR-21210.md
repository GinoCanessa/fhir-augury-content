# Implementation Plan: FHIR-21210

**Title:** Mappings of CDEs to FHIR section could be more useful
**Status:** Resolved - change required
**Resolution:** Persuasive
**Work Group:** Biomedical Research & Regulation
**Specification:** US Coordinated Registry Network (CRN) (FHIR)
**Resolved:** 2019-08-28

---

## Resolution Summary

The ballot commenter (Patricia Craig) noted that the "Mappings of CDEs to FHIR" table in the WHT CRN IG would be more useful if:

1. The data elements in column 1 ("Harmonized Common Data Element") included definitions — e.g., for "Gestational Age at Delivery," specifying whether it is reported in days, weeks, and how it is calculated.
2. Column 2 ("FHIR Data Element") contained hyperlinks to the FHIR data element definitions.

The resolution accepted as **Persuasive** directs:
> "Add a definition / description of the data element into the IG as a column. Add links to the FHIR data Elements."

## Feature Proposal

### Problem Statement

The "Mappings of CDEs to FHIR" table on the Mappings and Profiles page (`profiles.md`) of the WHT CRN IG currently has three columns:

| Column | Content | Problem |
|--------|---------|---------|
| Harmonized Common Data Element | CDE name only (e.g., "Gestational Age at Delivery") | No definition — reviewers cannot determine units, calculation method, or scope of the data element |
| FHIR Data Element | Plain-text FHIR element paths (e.g., `Observation.code, Observation.value[x]`) | No hyperlinks — reviewers must manually locate the FHIR element documentation |
| FHIR Resource/Profile/Extension | Profile/resource name (e.g., `us-core-observationresults`) | Adequate as-is |

This makes it difficult for reviewers and implementers to provide informed feedback or correctly implement the mappings without separate research.

### Proposed Change

Modify the CDE-to-FHIR mapping table in `source/pages/_includes/CDEs_FHIR.html` to:

1. **Add a new column "CDE Definition"** (column 2, shifting existing columns right) that provides a concise definition or description for each harmonized CDE. Definitions should specify units, value types, calculation methods, or any relevant context. Definitions can be sourced from the NIH CDE Repository or the WHT CRN consensus definitions.

2. **Add hyperlinks to column 2** (now column 3, "FHIR Data Element") pointing to the relevant FHIR specification element pages. For FHIR base resources, link to `http://hl7.org/fhir/R4/{resource}.html`. For US Core profiles, link to the US Core IG profile pages.

The resulting table will have **four columns**:

| Harmonized Common Data Element | CDE Definition | FHIR Data Element | FHIR Resource/Profile/Extension |
|-------------------------------|---------------|-------------------|-------------------------------|
| Gestational Age at Delivery | The gestational age of the fetus at the time of delivery, measured in completed weeks and/or days. | [Observation.code](http://hl7.org/fhir/R4/observation-definitions.html#Observation.code), [Observation.value\[x\]](http://hl7.org/fhir/R4/observation-definitions.html#Observation.value_x_) | us-core-observationresults |

### Design Rationale

- Adding definitions directly in the table keeps all mapping context co-located, reducing the need for external lookups.
- Hyperlinks to FHIR element definitions follow standard IG authoring practices and improve navigation.
- A fourth column rather than inline annotations preserves the clean tabular structure and allows definitions to be arbitrarily detailed.
- This is purely a documentation/narrative change — no StructureDefinition, profile, or conformance artifact changes are required.

## Impact Analysis

### Affected Repositories

#### HL7/coordinated-registry-network (IG)

- **Role:** The WHT CRN Implementation Guide repository containing all page content, profiles, and resources for the Coordinated Registry Network IG.
- **Affected Files:**
  - `source/pages/_includes/CDEs_FHIR.html` — **Primary file.** Contains the HTML `<table>` with the CDE-to-FHIR mapping. Must be modified to add a definition column and hyperlinks. Currently has 32 data rows across CDEs covering Patient demographics, Procedure, Device, Observation, Condition, Encounter, Location, and AdverseEvent mappings.
  - `source/pages/profiles.md` — **Secondary file.** Contains the introductory text for the mapping table ("The following is a mapping of each of the specific CDE element..."). The text describes the table as having "three columns" — this must be updated to say "four columns" and describe the new definition column.
- **Change Scope:** Moderate — The HTML table needs a new column header and 32 new `<td>` cells with CDE definitions, plus 32+ hyperlinks added to existing FHIR element cells.

## Implementation Plan

### Prerequisites

1. **Obtain CDE definitions:** Gather official definitions for all 32 harmonized CDEs listed in the mapping table. Sources include:
   - NIH CDE Repository (https://cde.nlm.nih.gov/)
   - WHT CRN project consensus documents
   - The CRN overview page (`source/pages/crn-overview.md`) which contains contextual descriptions for some CDEs
2. **Verify FHIR element URLs:** Confirm the correct FHIR R4 specification URLs for each element path referenced in the table (e.g., `http://hl7.org/fhir/R4/patient-definitions.html#Patient.birthDate`).
3. **Verify US Core profile URLs:** Confirm the correct US Core IG profile page URLs for elements that map through US Core profiles.

- Base FHIR resources: `http://hl7.org/fhir/R4/{resource-lowercase}-definitions.html#[Resource].[element]`
- US Core profiles: `http://hl7.org/fhir/us/core/STU3.1.1/StructureDefinition-us-core-{profile}.html`

```html
<!-- Before -->
<td class="maptable17">Patient.birthDate</td>

<td class="maptable17">
```

### Open Questions

1. **Source of CDE Definitions:** The resolution says to add definitions, but does not specify the authoritative source. Should definitions come from the NIH CDE Repository, the WHT CRN project consensus documents, or be written by the IG editors? The NIH CDE Repository is the most authoritative source and was already referenced elsewhere in the IG.

2. **FHIR Version for Links:** The IG's `ig.json` specifies `"version": "4.0.0"` (FHIR R4). All links should use the R4 specification URLs (`http://hl7.org/fhir/R4/`). Should links use the permanent R4 URL or the `build.fhir.org` URL?

3. **US Core Version for Links:** The IG depends on US Core but the version is listed as `"current"`. Links to US Core profiles should target a specific published version (e.g., STU3.1.1) or the current build. Clarification from the work group is recommended.

4. **Table Width:** Adding a definition column to an already 3-column table may cause layout issues on narrow viewports. Should the table use responsive CSS, or is horizontal scrolling acceptable?

5. **Repo Activity:** The `HL7/coordinated-registry-network` repository has not had commits since March 2019 (last commit: `6e16ed5` on 2019-03-26). This ticket was resolved in August 2019 but never implemented. The repo may need to be brought up to date with current IG publisher tooling before this change can be applied.
