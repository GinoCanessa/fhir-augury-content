## Resolution Summary

Resolution: **Persuasive with Modification**. The agreed actions, as recorded
in the ticket's resolution description, are:

- Talk to PC (FHIR Product Coordinators).
- **Let PC remove the profile and its extensions from Core.**
- Clinical Genomics WG will review the use cases enabled by the profile in
  conjunction with other R6 resources (e.g., `PersonalRelationship`) and
  consider re-publishing the content in R6 guidance and/or any other related
  IGs that get published for R6.

Per the most recent comment (james.jones, 2025-09-17), there is **no expected
date** for the content to be re-published in an IG. The change scope for
HL7/fhir is therefore strictly **removal**; no replacement content is
introduced into Core, and no parallel add into another HL7 repo is required by
this ticket.

## Feature Proposal

### Problem Statement

FMG asked work groups to move workgroup-owned profiles and extensions out of
the FHIR Core specification and into work-group-managed Implementation Guides.
The `FamilyMemberHistoryForGeneticsAnalysis` profile (URL
`http://hl7.org/fhir/StructureDefinition/familymemberhistory-genetic`) and
its three companion complex extensions
(`family-member-history-genetics-parent`,
`family-member-history-genetics-sibling`,
`family-member-history-genetics-observation`) currently live under
`source/familymemberhistory/` in HL7/fhir, contrary to that policy. They are
Clinical Genomics WG content and should not be in Core.

### Proposed Change

Remove the `FamilyMemberHistoryForGeneticsAnalysis` profile, its companion
extensions, the example List(s) that demonstrate them, the placeholder
"Genetic Pedigree" page, and all narrative cross-links that point at the
deleted page from HL7/fhir master.

Concretely, in HL7/fhir:

1. Delete the profile StructureDefinition file
   `source/familymemberhistory/structuredefinition-profile-familymemberhistory-genetic.xml`
   (defines `url=http://hl7.org/fhir/StructureDefinition/familymemberhistory-genetic`,
   `name=FamilyMemberHistoryForGeneticsAnalysis`, and references the three
   companion extension URLs at lines 74, 83, 92).
2. Delete the genetic-pedigree introduction page
   `source/familymemberhistory/familymemberhistory-genetic-introduction.xml`
   (the narrative shown on `familymemberhistory-genetic.html`).
3. Delete the two ImplementationGuide registration files that publish the
   profile and its companion extensions in Core:
   - `source/familymemberhistory/implementationguide-family-member-history-genetics.xml`
     (id `family-member-history-genetics`, references
     `StructureDefinition/familymemberhistory-genetic` and the three
     `family-member-history-genetics-{parent,sibling,observation}` SDs).
   - `source/familymemberhistory/implementationguide-FamilyMemberHistory-core.xml`
     (id `FamilyMemberHistory-core`, name
     `FamilyMemberHistoryHL7Extensions`, references
     `StructureDefinition/familymemberhistory-genetic`).
4. Delete the example List(s) that exercise the deleted profile/extensions:
   - `source/list/list-example-familyhistory-genetics-profile.xml`
   - `source/list/list-example-familyhistory-genetics-profile-annie.xml`
   - `source/list/list-example-double-cousin-relationship-pedigree.xml`
5. Remove the four `oids.ini` rows that reserve OIDs for the deleted
   structures (these stay in `tools/history/release{2,3,4,4b,5}/…` for
   historical record but are deleted from current `source/oids.ini`):
   - `…/family-member-history-genetics-observation = …5.1224`
   - `…/family-member-history-genetics-parent = …5.1231`
   - `…/family-member-history-genetics-sibling = …5.1223`
   - `…/familymemberhistory-genetic = …5.1229`
6. Update `source/familymemberhistory/familymemberhistory-introduction.xml`
   (lines 13–15) to remove the "refer to the
   [Genetic Pedigree](familymemberhistory-genetic.html) profile for an
   example" sentence so the resource introduction no longer links to the
   deleted page.
7. Run a repo-wide search for `familymemberhistory-genetic.html`,
   `familymemberhistory-genetic`, `FamilyMemberHistoryForGeneticsAnalysis`,
   `family-member-history-genetics-parent`,
   `family-member-history-genetics-sibling`, and
   `family-member-history-genetics-observation` and clean up any remaining
   inbound references (only matches found in the current clone outside
   `tools/history/` are the seven files enumerated above; new arrivals merged
   after this plan was written must also be swept).

No replacement content is added to HL7/fhir. CG's eventual R6 IG work is
explicitly out of scope for this ticket (per the 2025-09-17 comment, "no
expected date" for an IG home).

### Design Rationale

This implements the ticket's resolution literally ("Let PC remove Profiles
and Extensions from Core"). FMG's policy of moving WG-specific content out of
Core motivates the deletion; the WG is responsible for any future
re-publication in an IG and that is tracked separately (the ticket was
resolved without binding the removal to a parallel IG add). Removing all
companions (the example Lists, the two IG registration files, the OID
reservations, and the inbound link from the FamilyMemberHistory introduction)
together avoids dangling references, which is exactly the gotcha-class issue
the briefing flags for narrative deletions.

### Breaking Changes

This is a **breaking change for any implementer that conforms to or
references** the deleted artifacts:

- The profile URL
  `http://hl7.org/fhir/StructureDefinition/familymemberhistory-genetic` is
  withdrawn from Core.
- The complex-extension URLs
  `http://hl7.org/fhir/StructureDefinition/family-member-history-genetics-parent`,
  `…/family-member-history-genetics-sibling`, and
  `…/family-member-history-genetics-observation` are withdrawn from Core.
- Their OID reservations (in current `source/oids.ini`) are removed.
- Inbound link from the FamilyMemberHistory introduction is removed; any
  external page or IG that deep-links to `familymemberhistory-genetic.html`
  on the published spec will 404 in the next cut.

The companion FamilyMemberHistory **base resource** is not changed and
remains unaffected.

### Related Specifications

- **FamilyMemberHistory** (base resource) — its introduction page loses one

- **List** — not changed; only three of its example instances are removed.

### Step-by-Step Tasks

#### HL7/fhir

1. **Delete the genetic-pedigree profile StructureDefinition.**
   - File: `source/familymemberhistory/structuredefinition-profile-familymemberhistory-genetic.xml`
   - Action: Remove the file entirely (`git rm`).
   - Details: This is the only artifact defining
     `http://hl7.org/fhir/StructureDefinition/familymemberhistory-genetic`
     (`name=FamilyMemberHistoryForGeneticsAnalysis`). It also references
     the three companion extension URLs at lines 74, 83, 92.

2. **Delete the genetic-pedigree narrative page source.**
   - File: `source/familymemberhistory/familymemberhistory-genetic-introduction.xml`
   - Action: `git rm`.
   - Details: This is the body content the publisher emits at
     `familymemberhistory-genetic.html`. After removal, the page itself
     stops being generated.

3. **Delete the genetics-pedigree publisher IG manifest.**
   - File: `source/familymemberhistory/implementationguide-family-member-history-genetics.xml`
   - Action: `git rm`.
   - Details: This IG (id `family-member-history-genetics`, name
     `FamilyMemberHistoryForGeneticsAnalysis`) is what the bespoke publisher
     uses to register the deleted profile and the three
     `family-member-history-genetics-{parent,sibling,observation}` SDs.
     With the manifest gone, the publisher will stop trying to resolve those
     four references.

4. **Delete the secondary `FamilyMemberHistory-core` IG manifest.**
   - File: `source/familymemberhistory/implementationguide-FamilyMemberHistory-core.xml`
   - Action: `git rm`.
   - Details: This second manifest (id `FamilyMemberHistory-core`, name
     `FamilyMemberHistoryHL7Extensions`) also publishes
     `StructureDefinition/familymemberhistory-genetic`. It exists only to
     ship the genetic profile alongside the base resource; with the profile
     removed, the manifest has no remaining purpose.

5. **Edit the FamilyMemberHistory resource introduction to drop the
   genetic-pedigree link.**
   - File: `source/familymemberhistory/familymemberhistory-introduction.xml`
   - Action: Remove the sentence that reads "However, it can also be the
     basis for capturing a more rigorous history useful for genetic and
     other analysis - refer to the
     [Genetic Pedigree](familymemberhistory-genetic.html) profile for an
     example." The surrounding paragraph about elementary family history
     can stand on its own; either drop the paragraph entirely or trim it to
     end after "particular family member."
   - Details: This is the only inbound link to
     `familymemberhistory-genetic.html` inside `source/`. Failing to remove
     it leaves a dangling page reference once the introduction file in
     step 2 is gone — exactly the class of issue the briefing's gotcha
     about anchor cleanup warns about.

6. **Delete the genetics-pedigree List examples.**
   - File: `source/list/list-example-familyhistory-genetics-profile.xml`
   - Action: `git rm`.

     URL is withdrawn.

7. **Delete the second genetics-pedigree List example.**
   - File: `source/list/list-example-familyhistory-genetics-profile-annie.xml`
   - Action: `git rm`.
   - Details: Companion to step 6 (the "Annie Proband" pedigree, paired

8. **Delete the double-cousin pedigree List example.**
   - File: `source/list/list-example-double-cousin-relationship-pedigree.xml`
   - Action: `git rm`.
   - Details: Also depends on the deleted genetics-pedigree extensions; no
     longer valid against Core after this change.

9. **Remove the four genetic-pedigree OID reservations from
   `source/oids.ini`.**
   - File: `source/oids.ini`
   - Action: Delete the four lines whose left-hand sides are
