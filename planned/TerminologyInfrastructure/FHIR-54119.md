## Resolution Summary

The committee accepted the ticket as **Persuasive** with the resolution
"Will fix as requested". The submitter observed that the short
description (one-line `<short>`) for `NamingSystem.author` reads
**"Who authored the CodeSystem"**, which is leftover copy-paste from the
sibling `CodeSystem.author` element. It must be changed to
**"Who authored the NamingSystem"**.

No other elements, definitions, cardinalities, types, bindings, or
behavior are affected by this resolution.

## Feature Proposal

### Problem Statement

`NamingSystem` (Foundation.Terminology, normative since 6.0.0) defines
an `author` element of type `ContactDetail` (0..*). Its short
description in the StructureDefinition currently says
`Who authored the CodeSystem`, mis-naming the parent resource. This is
inconsistent with:

- The sibling element `NamingSystem.editor` whose `<short>` already
  reads "Who edited the NamingSystem"
  (`source/namingsystem/structuredefinition-NamingSystem.xml:597`).
- The element's own `<definition>` (line 588) which already correctly
  references `NamingSystem`: *"An individiual or organization primarily
  involved in the creation and maintenance of the NamingSystem."*
- `CodeSystem.author` (`source/codesystem/structuredefinition-CodeSystem.xml:544`),
  which is the legitimate owner of the "Who authored the CodeSystem"
  short text.

The mismatch is a documentation defect that surfaces in every rendered
view of the resource (logical view, snapshot table, JSON/XML schema
descriptions, FHIR IG renderings).

### Proposed Change

Edit a single `<short>` value in the NamingSystem StructureDefinition's
**differential**:

- **Resource:** `NamingSystem`
- **Element id / path:** `NamingSystem.author`
- **Change:** `<short value="Who authored the CodeSystem"/>` →
  `<short value="Who authored the NamingSystem"/>`

No other element fields change:
- Type stays `ContactDetail`.
- Cardinality stays `0..*`.
- Definition, requirements, comments, bindings, and constraints are
  unchanged.
- The `committee-notes` extension on the element is unchanged.

### Design Rationale

The new wording mirrors the established pattern used by all other
contributor-role elements on `NamingSystem`
(`editor` → "Who edited the NamingSystem", and the surrounding
`reviewer`/`endorser` siblings follow the same `<verb> the
NamingSystem` form). Re-using the existing pattern is the minimal,
least-surprise correction and aligns with the resolution's
"Will fix as requested" directive. No alternative was discussed in the
ticket — there is only one correct value for this element's short
description.
