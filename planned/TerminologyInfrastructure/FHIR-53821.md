## Resolution Summary

Resolution: **Persuasive** — "Will fix."

On the "Using Codes in Resources" (Terminologies) page, the second bullet of
the introductory list currently reads:

> **Value Sets** specifies a set of codes defined by code systems that can be
> used in a specific context

The subject "Value Sets" is plural, so the verb must be "specify". The
resolution requires changing "specifies" to "specify" in that bullet. No
other editorial or structural changes are in scope.

## Feature Proposal

### Problem Statement

The Terminologies overview page (`terminologies.html`, section headed "Using
Codes in Resources") introduces the three core terminology concepts —
Code Systems, Value Sets, and Bindings — as a bulleted list. The Value Sets
bullet has a subject–verb agreement error: "**Value Sets** specifies …".
This is a cosmetic grammar defect in normative-adjacent narrative and was
raised as a Technical Correction with Highest priority. It does not affect
conformance, but because the page is the primary on-ramp to FHIR
terminology for implementers, the grammar error is highly visible.

### Proposed Change

In `source/terminologies.html`, change the verb in the second list item of
the opening `<ul>` so the plural subject "Value Sets" agrees with the verb:

- Before (line 106):
  ```html
  <li><b><a href="valueset.html">Value Sets</a></b> specifies a set of codes defined by code systems that can be used in a specific context</li>
  ```
- After:
  ```html
  <li><b><a href="valueset.html">Value Sets</a></b> specify a set of codes defined by code systems that can be used in a specific context</li>
  ```

No other lines are touched. The sibling "Code Systems" and "Bindings" bullets
already use plural-agreeing verbs ("define", "apply … and specify") and do
not require changes.

### Design Rationale

This is a pure grammar fix with a single correct form; there is no design
alternative. The surrounding list already follows the pattern
`<b><a>Term</a></b> <verb-plural> …` for "Code Systems" and "Bindings", so
switching "specifies" → "specify" makes the Value Sets bullet parallel in
both grammar and style with its peers. The resolution is tagged
**Persuasive** with resolution description "Will fix", confirming no
discussion of alternatives is needed.
