## Resolution Summary

Resolution: **Persuasive** ŌĆö *"Will fix"*.

Fix a single-word typo in the sentence immediately preceding the Internal Code
Systems table on the Code Systems page. The change is purely textual:

- From: *"These code systems are tightly bound to the resource definitions, and **a** defined by and considered part of FHIR itself."*
- To:   *"These code systems are tightly bound to the resource definitions, and **are** defined by and considered part of FHIR itself."*

No semantic, structural, conformance, or terminology changes are involved.

## Feature Proposal

### Problem Statement

The narrative paragraph that introduces the Internal Code Systems table on the
FHIR Core *Code Systems* page contains an ungrammatical fragment ("ŌĆ”and a
defined by and considered part of FHIR itself."). The intended word is "are".
The malformed sentence makes the introductory text difficult to parse and
projects an unprofessional impression on a normative-adjacent page used by
implementers to understand which code systems are owned by FHIR itself.

### Proposed Change

Edit one line of authored HTML on the Code Systems page so the sentence reads
correctly:

- File: `source/terminologies-systems.html`
- Location: line 155 (within the `<p>` block beginning at line 153, immediately
  before the Internal Code Systems table).
- Change: replace the word `a` with `are` in the phrase
  `and a defined by and considered part of FHIR itself`, yielding
  `and are defined by and considered part of FHIR itself`.

No other tokens, anchors, IDs, or surrounding markup change. The full corrected
paragraph reads:

> These code systems are tightly bound to the resource definitions, and are
> defined by and considered part of FHIR itself.

### Design Rationale

This is a Technical Correction with resolution *Persuasive ŌĆö Will fix*. The
ticket itself dictates the exact before/after wording, so there is no design
choice to be made. The minimal surgical edit (one word) preserves line offsets
elsewhere in the file, avoids any risk of disturbing adjacent prose, and
matches the standing FhirCore recipe of editing `source/*.html` pages directly
through the bespoke HL7 publisher.
