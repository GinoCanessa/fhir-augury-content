## Resolution Summary

**Resolution: Persuasive with Modification.**

The resolution explicitly defers any code-base change:

> It is unlikely that there will be another R4B publication. We will make
> users aware of these OID conflicts in the Errata that is published via
> this page on Confluence:
> https://confluence.hl7.org/display/FHIR/Known+Issues+with+the+published+FHIR+Specifications

In other words, the five remaining R4B CodeSystem OID discrepancies
identified in the attached spreadsheet will **not** be patched in the
R4B specification source. Instead, they will be documented as known
issues on the Confluence "Known Issues with the published FHIR
Specifications" page. No change is required in any FHIR source repository
(`HL7/fhir`, `HL7/UTG`, etc.) under this ticket.

## Feature Proposal

### Problem Statement

A December 2022 effort by TSMG, FMG, and Grahame Grieve resolved the
bulk of CodeSystem OID discrepancies between FHIR versions, but five
CodeSystem OIDs were left differing across FHIR versions (the list lives
on the attached spreadsheet referenced from
[FHIR Core and THO OID Issue Summary and Resolution](https://confluence.hl7.org/display/TSMG/FHIR+Core+and+THO+OID+Issue+Summary+and+Resolution),
linked there as `OID Fix List R4B.csv`). The discrepancies primarily
affect R4B implementers who cross-reference the same code systems in
other FHIR versions or in THO.

### Proposed Change

Per the workgroup's resolution, the change is **documentation-only and
out-of-band** to the FHIR source repositories:

1. Add an entry (or entries, one per affected CodeSystem) to the
   Confluence page
   [Known Issues with the published FHIR Specifications](https://confluence.hl7.org/display/FHIR/Known+Issues+with+the+published+FHIR+Specifications)
   listing each of the five remaining R4B CodeSystem OID discrepancies.
2. For each entry, include:
   - The CodeSystem URL / name.
   - The OID published in R4B.
   - The OID(s) used in other FHIR versions and/or in THO.
   - A short note that the R4B value will not be corrected in the
     specification because no further R4B publication is planned, and a
     pointer to the
     [TSMG OID Issue Summary and Resolution page](https://confluence.hl7.org/display/TSMG/FHIR+Core+and+THO+OID+Issue+Summary+and+Resolution)
     for context.
3. Keep the source spreadsheet (`OID Fix List R4B.csv`) attached to the
   TSMG page so implementers have machine-readable details.

No StructureDefinition, CodeSystem, ValueSet, NamingSystem, or
publication-tooling change is required in any GitHub repository.

### Design Rationale

- R4B is in maintenance and TSMG/FMG do not plan another R4B
  publication, so editing R4B source CodeSystems would not produce a
  corrected published artifact and could reintroduce drift between THO
  and the (current) R4B source.
- Errata via the Confluence "Known Issues" page is the established
  channel for post-publication FHIR corrections that cannot be
  re-published. Implementers already consult that page.
- The `TechnicalCorrection` label confirms this is a documented
  technical correction rather than a normative redesign.

## Implementation Plan

### Prerequisites

- Editor access to the HL7 Confluence space `FHIR` (specifically the
  "Known Issues with the published FHIR Specifications" page).
- A copy of the `OID Fix List R4B.csv` spreadsheet attached to
  [FHIR Core and THO OID Issue Summary and Resolution](https://confluence.hl7.org/display/TSMG/FHIR+Core+and+THO+OID+Issue+Summary+and+Resolution)
  to source the five concrete CodeSystem rows.

### Step-by-Step Tasks

#### Confluence — `FHIR / Known Issues with the published FHIR Specifications`

1. **Add an R4B CodeSystem OID Discrepancies section.**
   - Page: https://confluence.hl7.org/display/FHIR/Known+Issues+with+the+published+FHIR+Specifications
   - Action: Add a new section (e.g., "R4B CodeSystem OID Discrepancies
     (FHIR-49641)") near the existing R4B errata entries.
   - Details: Introduce the five remaining R4B CodeSystem OID
     discrepancies and explain that they will not be corrected in
     R4B because no further R4B publication is planned.

2. **Insert a row-per-CodeSystem table.**
   - Page: same as above.
   - Action: Add a table with columns `CodeSystem URL`, `R4B OID`,
     `Conflicting OID(s)`, `Source(s) using conflicting OID`, `Notes`,
     populating one row for each of the five CodeSystems listed in
     `OID Fix List R4B.csv`.
   - Details: Pull the values directly from the attached spreadsheet so
     the Confluence page and the spreadsheet stay in sync.

3. **Cross-link the TSMG resolution page and ticket.**
   - Page: same as above.
   - Action: Add explicit links to
     [FHIR Core and THO OID Issue Summary and Resolution](https://confluence.hl7.org/display/TSMG/FHIR+Core+and+THO+OID+Issue+Summary+and+Resolution)
     and to [FHIR-49641](https://jira.hl7.org/browse/FHIR-49641) so
     implementers can trace the history.

4. **(Optional) Re-attach the spreadsheet for convenience.**
   - Page: same as above.
   - Action: Attach a copy of `OID Fix List R4B.csv` (or link the TSMG
     attachment) so the machine-readable list is one click from the
     Errata page.

#### FHIR source repositories

No tasks. Per the resolution, no edits are made in `HL7/fhir`, `HL7/UTG`,
or any other FHIR source repository.
