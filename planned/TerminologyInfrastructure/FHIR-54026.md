## Problem Statement

The `lastActiveVersion` parameter on the CRMI ValueSet/$expand operation currently overrides the code system version in the expansion when a code is no longer active. Instead, the information about the last active version should be conveyed through a **concept property** (not an extension, per FHIR-55965 resolution), leaving the expansion results otherwise unchanged. The parameter also needs to be renamed.

## Resolution Description

Per the approved resolution:

1. **Rename** the parameter from `lastActiveVersion` to `includeLastVersionActive`
2. **Change behavior**: When `includeLastVersionActive` is set and a code explicitly named in `compose.include.concept.code` is not active in the version of the code system used for the expansion:
   - Include the code in the expansion with `inactive = true`
   - Set the version to the **requested** version of the code system (not the version where it was last active)
   - Add a **`lastVersionActive` concept property** indicating the version of the code system in which the code was last active
3. The parameter can only be used with `activeOnly = false`; setting `includeLastVersionActive` when `activeOnly = true` has no effect
4. Servers **MAY** throw an exception if they do not support this capability
5. The expansion result with `activeOnly=false` should be the **same** with or without `includeLastVersionActive`, except for the addition of the `lastVersionActive` concept property
