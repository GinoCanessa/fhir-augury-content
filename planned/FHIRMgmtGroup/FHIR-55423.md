Resolution description (verbatim from the ticket):
> Agreed, clarify that the confluence page is the source truth as proposed in both the spec and the confluence page.

Concretely, two clarifying edits are required:

1. **In the FHIR Core specification** — update the FHIR Maturity Model paragraph in `source/versions.html` so that it explicitly states the HL7 Confluence "FHIR Maturity Model" page is the *source of truth* for maturity levels, not merely a place where "further information" can be found. The current text describes the in-spec table as a "frozen snapshot of the maturity levels maintained by the FHIR Management Group on the FHIR Confluence page" but never names the Confluence page as authoritative.
2. **On the Confluence page itself** (https://confluence.hl7.org/display/FHIR/FHIR+Maturity+Model) — add a prominent statement at the top of the page declaring that this page is the source of truth for FHIR resource/artifact maturity levels. There are no behavioural / conformance changes — this is a documentation clarification only.

## Feature Proposal

### Problem Statement

The FHIR Maturity Model section in the FHIR Core specification (`source/versions.html`, anchor `#maturity`, lines ~125–166 in the current `master` clone HEAD `db79dcd`) presents a table of FMM levels and then states (line 160):

> The table above represents a frozen snapshot of the maturity levels maintained by the FHIR Management Group on the FHIR Confluence page. Further information and discussion about these levels can be found there.

This wording tells the reader that the in-spec table is a *snapshot* — but it never explicitly identifies the Confluence page as the **source of truth**. As the ticket reporter (Bryn Rhodes) notes, the fact that the Confluence page is authoritative is currently only stated in scattered comments, never on the Confluence page itself nor in the spec. The result is that implementers and authors may treat the in-spec table (or other historical artefacts) as the canonical maturity assignment, when in fact the FMG-maintained Confluence page is the single source of truth between publications.

### Proposed Change

**Spec change (`HL7/fhir`, `source/versions.html`):**

Replace the existing paragraph at line ~159–161:

```html
<p>
The table above represents a frozen snapshot of the maturity levels maintained by the FHIR Management Group on the <a href="https://confluence.hl7.org/display/FHIR/FHIR+Maturity+Model">FHIR Confluence page</a>.  Further information and discussion about these levels can be found there.
</p>
```

with prose that names the Confluence page as authoritative, e.g. (illustrative — exact wording is for the FMG to confirm):

```html
<p>
The <a href="https://confluence.hl7.org/display/FHIR/FHIR+Maturity+Model">FHIR Maturity Model page on HL7 Confluence</a>, maintained by the FHIR Management Group (FMG), is the source of truth for the current FMM level of every resource and artifact in FHIR. The table above is a frozen snapshot of those maturity levels at the time this version of the specification was published; the Confluence page is updated continuously and may be ahead of the snapshot above. Further information and discussion about how levels are assigned can also be found there.
</p>
```

The change keeps the existing `https://confluence.hl7.org/display/FHIR/FHIR+Maturity+Model` link intact so no inbound references break, and preserves the "frozen snapshot" framing for the in-spec table (which remains accurate for the published artefact).

**Confluence change (out-of-repo, FMG-owned):**

At the top of https://confluence.hl7.org/display/FHIR/FHIR+Maturity+Model, add a callout/banner of the form:

> **This page is the source of truth for FHIR artifact maturity (FMM) levels.** It is maintained by the FHIR Management Group (FMG). The maturity tables published in any released version of the FHIR specification are frozen snapshots of this page taken at publication time and may be out of date.

(Wording again for FMG to finalize.)

### Design Rationale

- The resolution explicitly says the clarification belongs in **both** the spec and the Confluence page; doing only one half leaves the gap the reporter described.
- Keeping the existing "frozen snapshot" language preserves the meaning the spec already conveys about the relationship between the published table and the live Confluence page; the change only adds the missing "source of truth" attribution.
- The link target (the Confluence page URL) is unchanged, so no other spec page that references `#maturity` or the Confluence URL needs to be touched.
- No FMM levels themselves are being changed — this is intentionally a documentation-only edit, in line with the ticket's `Persuasive` resolution and the absence of any technical/coded follow-ups in the comments.
