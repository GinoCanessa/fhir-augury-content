## Resolution Summary

Resolution: **Persuasive** — "resolve as requested".

The reporter identified a typographical error in the Composition resource's
"Note for CDA aware readers" narrative section: one sentence contains the
duplicated phrase "where the where the". The committee agreed the duplication
should be removed. No semantic change to the Composition resource or to the
CDA-mapping guidance is intended; only the redundant words are deleted so the
sentence reads "...is equivalent to a CDA document, where the
`Bundle.identifier` is equivalent to `ClinicalDocument.id` ...".

## Feature Proposal

### Problem Statement

The Composition resource page in FHIR Core includes a narrative subsection
titled "Note for CDA aware readers" (rendered from
`source/composition/composition-introduction.xml`). Inside the bulleted list
contrasting Composition with CDA, the second clause of the first bullet
contains an accidental word duplication ("where the where the") immediately
before the `Bundle.identifier` reference. The duplication is a pure surface
defect — it makes the sentence ungrammatical and slightly harder to read,
though the meaning is still recoverable. There is no functional or
specification impact, but the published HTML inherits the typo verbatim and
the ticket is queued under Block Vote 1, so the fix needs to land before the
next publication of FHIR Core.

### Proposed Change

Single-token narrative edit in the Composition introduction page.

- **File:** `source/composition/composition-introduction.xml`
- **Location:** Line 135, inside the `<ul>` that begins on line 131, in the
  first `<li>` (the bullet that contrasts Composition's `identifier` with
  `ClinicalDocument.setId`).
- **Current text (line 135, verbatim):**

  ```html
   where the where the <code>Bundle.identifier</code> is equivalent to <code>ClinicalDocument.id</code> and <code>Bundle.meta.security</code> is equivalent to <code>ClinicalDocument.confidentialityCode</code>.</li>
  ```

- **Replace with:**

  ```html
   where the <code>Bundle.identifier</code> is equivalent to <code>ClinicalDocument.id</code> and <code>Bundle.meta.security</code> is equivalent to <code>ClinicalDocument.confidentialityCode</code>.</li>
  ```

That is, remove the second occurrence of `where the ` (one instance, including
the trailing space) so the sentence reads correctly. No other characters,
indentation, or surrounding markup change. No StructureDefinition,
ValueSet, CodeSystem, search parameter, example, or operation is touched.

### Design Rationale

The resolution text ("resolve as requested") and the ticket type
(`Technical Correction`) confirm this is a non-controversial editorial fix.
Removing the duplicate words is the minimal change that satisfies the
reporter's request while preserving the original sentence intent — namely
the equivalence mapping between FHIR Bundle metadata and CDA
`ClinicalDocument` header fields. No alternative wording is needed; the
sentence as written (with the duplication removed) already renders the
intended meaning consistently with the surrounding CDA-mapping bullets.
