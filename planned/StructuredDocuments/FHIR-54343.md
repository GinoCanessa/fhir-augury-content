## Feature Proposal

### Problem Statement

In FHIR 6.0.0-ballot4, the `<short>` (Short Display) of
`Composition.participant.type` enumerated example codes — including
`author`, `authenticator`, `custodian`, `legal authenticator`,
`record target`, and `subject` — that are **explicitly excluded** by the
bound ValueSet
`http://terminology.hl7.org/ValueSet/fhir-clinical-doc-participant`
(see https://terminology.hl7.org/7.0.1/ValueSet-fhir-clinical-doc-participant.html).
These types are surfaced through dedicated Composition fields
(`author`, `attester`, `custodian`, `subject`) rather than through
`participant`, so listing them in the short description was misleading
and inconsistent with the binding's own `<description>`, which calls
those types out as excluded.

The reporter (and four sibling tickets that the workgroup chose to
coordinate) asked for a Short Display that either reflects the codes
actually allowed by the ValueSet or uses neutral explanatory text.

### Proposed Change

Apply the agreed wording to `Composition.participant.type` and document
it in the resource ballot note:

1. **Composition StructureDefinition — change the `<short>` value.**
   - File: `source/composition/structuredefinition-Composition.xml`
   - Element: `Composition.participant.type` (the differential element
     with `id="Composition.participant.type"` / `path` of the same
     name).
   - Replace the current `<short value="..."/>` with:
     `<short value="Meaning and purpose of participation, in creation of the clinical document"/>`
   - Do **not** touch the `<definition>`, `<min>`, `<max>`, `<type>`, or
     `<binding>` (binding strength `preferred`, ValueSet
     `http://terminology.hl7.org/ValueSet/fhir-clinical-doc-participant`,
     binding name `FHIRClinicalDocParticipantVs`). Cardinality stays
     `1..*` and type stays `CodeableConcept`.

2. **Composition introduction — record the ballot note and release-notes link.**
   - File: `source/composition/composition-introduction.xml`
   - Inside the existing `<blockquote class="ballot-note" id="bn1">`
     "Note to Balloters" block, ensure there is a paragraph stating that
     the `Composition.participant.type` short description was changed to
     "Meaning and purpose of participation, in creation of the clinical
     document" as a correction.
   - In the `Release Notes` `<ul>` block, ensure there is a list item
     linking to `https://jira.hl7.org/browse/FHIR-54343`. If the
     coordinated siblings (FHIR-53695, FHIR-54103, FHIR-54182,
     FHIR-54986) are resolved with the same wording, they should appear
     here too.

No new elements, no extension definitions, no terminology changes, and
no new ValueSet/CodeSystem bindings are required. The ValueSet itself
is unchanged.

### Design Rationale

The Composition workgroup chose to harmonise the `Composition.participant.type`
short description across five overlapping tickets rather than pick from
the various reword proposals individually. The selected wording mirrors
the existing `<definition>` text up to the first sentence, so the short
display becomes a true subtitle of the formal definition rather than an
example enumeration. This:

- Removes the contradiction between the `<short>` (which previously
  listed codes excluded by the ValueSet) and the binding's own
  `<description>`, which explicitly excludes Author, Authenticator,
  Custodian, Legal Authenticator, Record Target and Subject.
- Avoids enumerating any specific codes in the `<short>`, which keeps
  the description stable if the ValueSet is later extended.
- Uses neutral, purpose-oriented language consistent with how other
  CodeableConcept-typed `.type` elements are described elsewhere in the
  spec.

Alternative wordings considered in the duplicate tickets (e.g. listing
only the *allowed* codes) were rejected because any in-line enumeration
would re-introduce the same drift problem the original short caused.
