## Feature Proposal

### Problem Statement

The first three codes (`dicom-wado-rs`, `dicom-qido-rs`, `dicom-stow-rs`) in the `endpoint-connection-type` CodeSystem carry definition text whose embedded DICOM Standard URLs return **HTTP 404** (the `part18/sect_6.5.html`, `part18/sect_6.7.html`, and `part18/sect_6.6.html` pages no longer exist on `dicom.nema.org`). The `dicom-wado-uri` code's URL points at the long-obsolete `dicom/2013/output/...` snapshot. The result is that any consumer following the published DICOMweb references from `Endpoint.connectionType` lands on dead pages.

A prior ticket (FHIR-50770, *"DICOM references in endpoint-connection-type are incorrect"*) flagged the same problem and was resolved Persuasive, but the required follow-up THO/UTG ticket was never created, so the broken URLs were never actually corrected at the source of truth. This ticket re-raises the issue and pairs it with a concrete UTG follow-up (UP-775).

Patient Administration owns the resource binding but Imaging Integration (WG) had to supply the authoritative replacement URLs since they own the DICOMweb content.

### Proposed Change

Update the `definition` text of four `concept` entries inside the THO `CodeSystem/endpoint-connection-type` resource to point at the correct, currently-served DICOM Standard pages. No code values change, no display values change, no concepts are added or removed, and no element cardinalities or bindings change. This is a pure text correction inside `<concept><definition value="…"/></concept>`.

Specific edits (each is a single attribute-value replacement):

1. Concept `dicom-wado-rs`
   - Old: `definition = "DICOMweb RESTful Image Retrieve - http://dicom.nema.org/medical/dicom/current/output/chtml/part18/sect_6.5.html"`
   - New: `definition = "DICOMweb RESTful Image Retrieve - https://dicom.nema.org/medical/dicom/current/output/chtml/part18/sect_10.4.html"`

2. Concept `dicom-qido-rs`
   - Old: `definition = "DICOMweb RESTful Image query - http://dicom.nema.org/medical/dicom/current/output/chtml/part18/sect_6.7.html"`
   - New: `definition = "DICOMweb RESTful Image query - https://dicom.nema.org/medical/dicom/current/output/chtml/part18/sect_10.6.html"`

3. Concept `dicom-stow-rs`
   - Old: `definition = "DICOMweb RESTful image sending and storage - http://dicom.nema.org/medical/dicom/current/output/chtml/part18/sect_6.6.html"`
   - New: `definition = "DICOMweb RESTful image sending and storage - https://dicom.nema.org/medical/dicom/current/output/chtml/part18/sect_10.5.html"`

4. Concept `dicom-wado-uri`
   - Old: `definition = "DICOMweb Image Retrieve - http://dicom.nema.org/dicom/2013/output/chtml/part18/sect_6.3.html"`
   - New: `definition = "DICOMweb Image Retrieve - https://dicom.nema.org/medical/dicom/current/output/chtml/part18/chapter_9.html"`

Also bump the CodeSystem `version` per UTG conventions (current value `2.1.1` in the cache snapshot — UP-775 owners should select the next THO publishing version; it is not a substantive content change, just a definition-text correction).

The `ValueSet/endpoint-connection-type` resource (which only `compose.include`s the CodeSystem with no enumerated concepts) does not need to change — the corrected definition text propagates automatically to expansions.

### Design Rationale

- **Fix at the source of truth (UTG/THO).** The CodeSystem URL is `http://terminology.hl7.org/CodeSystem/endpoint-connection-type`, which is maintained in `HL7/UTG` (`input/sourceOfTruth/fhir/codeSystems/`). FHIR Core only mirrors a ValueSet that re-includes this CodeSystem, so editing it inside FHIR Core would not address the underlying defect and would drift from THO.
- **Definition-text edit, not a display or code change.** Codes and displays are stable identifiers used in instances. Editing only the `definition` value is non-substantive (matches the WG vote: "Impact: Non-substantive; Category: Correction"), preserves all existing `Endpoint` instance data, and avoids any conformance break.
- **Use the URLs supplied by Imaging Integration.** Authority for the correct DICOMweb pages rests with II; the URLs above were posted by II in the Zulip thread linked from the ticket and re-quoted by `egelkraut` in the ticket comments. No alternative URL set was discussed.
- **Switch `http://` → `https://`.** The current definitions use `http://`, but `dicom.nema.org` redirects HTTP to HTTPS, and the new URLs from II are written as `https://`. Using HTTPS in the canonical definition text avoids an avoidable redirect hop and matches modern Standard practice.
- **Track the deferred FHIR-50770 work.** This change closes the loop FHIR-50770 left open by ensuring the THO ticket (UP-775) actually exists and is paired with this PA ticket.
