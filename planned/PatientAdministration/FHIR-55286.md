## Resolution Summary

Resolution: **Persuasive**. The ticket reports that the `appointment` specification page contains contracted forms that should be fully spelled out. The reporter specifically calls out `didn't` and `doesn't`. The resolution accepts the change: expand the contractions on the Appointment page to their fully spelled-out equivalents (e.g., `doesn't` → `does not`, `didn't` → `did not`). An `egelkraut` comment (2026-01-27) notes that FMG is concurrently discussing a global FHIR style policy on contractions (Zulip `179192-fmg > Do we have a policy on contractions in FHIR specs?`); resolving this Persuasive Patient Administration ticket does not require waiting on that policy — the narrow fix is simply to remove the flagged contractions on this one page. No resolution_description text was provided by the WG beyond the standard Persuasive disposition.

## Feature Proposal

### Problem Statement

HL7 editorial convention for FHIR narrative prose discourages contractions: published specification pages should spell abbreviations out. The `Appointment` resource page (built from `source/appointment/appointment-introduction.xml` in HL7/fhir) currently contains at least the contractions `doesn't`, `don't`, `won't`, and `didn't` in user-facing prose. The reporter (Lloyd McKenzie) called this out as "Comment 250" on the Appointment page; the ticket was accepted as Persuasive by Patient Administration.

### Proposed Change

Edit the Appointment-page narrative source and replace every prose contraction with its fully spelled-out form. Verified occurrences in the clone at HEAD `db79dcd`:

| File | Line | Current text | Replacement |
|------|------|--------------|-------------|
| `source/appointment/appointment-introduction.xml` | 40 | `indicated as available doesn't guarantee` | `indicated as available does not guarantee` |
| `source/appointment/appointment-introduction.xml` | 166 | `has no template information, and thus don't use the` | `has no template information, and thus does not use the` (also corrects subject/verb agreement — the subject `A series appointment` is singular) |
| `source/appointment/appointment-introduction.xml` | 171 | `likely won't use the <code>originatingAppointment</code>` | `likely will not use the <code>originatingAppointment</code>` |
| `source/appointment/appointment-introduction.xml` | 175 | `just as it would be normally if they didn't have a sequence` | `just as it would be normally if they did not have a sequence` |

Explicitly out of scope:

- `source/appointment/appointment-example-request.xml` line 84 contains `doesn't` inside an XML comment (`<!-- This resource auto accepts, so doesn't need negotiation -->`). This is authoring metadata embedded in an example instance, not prose rendered to the Appointment page; leave it unchanged unless the WG explicitly asks for example-comment normalization.
- `source/appointment/appointment-request-mapping-exceptions.xml` line 34 contains `'authorization'` / `'reason'` (possessive-style single-quoting of terms, not contractions) — not in scope.
- Other appointment-area files (`appointment-notes.xml`, mapping exceptions, value sets, code systems) were scanned and contain no prose contractions that need expansion.

### Design Rationale

- The fix is purely editorial — same narrative content, fewer contractions. It touches only `<p>` / `<li>` prose inside an authored HTML fragment that the bespoke HL7 publisher copies into `appointment.html`.
- "Doesn't" → "does not" and "didn't" → "did not" are the exact cases the reporter flagged. Sweeping up `don't` and `won't` on the same page in the same commit keeps the page internally consistent and avoids a trailing ticket for the remaining contractions on the same page.
- The FMG Zulip discussion referenced by `egelkraut` may later produce a FHIR-wide contraction policy. That policy will dictate whether comments in example files (like `appointment-example-request.xml#84`) also have to be expanded. Waiting on it is not required to execute this Persuasive resolution on the Appointment page; scope the change narrowly to the cases Patient Administration accepted.

### Step-by-Step Tasks

#### HL7/fhir

1. **Expand `doesn't` on the availability-check paragraph.**
   - File: `source/appointment/appointment-introduction.xml`
   - Action: On line 40, replace `Just because a time is indicated as available doesn't guarantee that an appointment can be made.` with `Just because a time is indicated as available does not guarantee that an appointment can be made.`.
   - Details: Located inside the `Checking Availability on the Schedule (optional)` `<li>` of the Making-an-Appointment ordered list. Preserve surrounding whitespace and sibling markup.

2. **Expand `don't` and fix subject/verb agreement in the Series intro.**
   - File: `source/appointment/appointment-introduction.xml`
   - Action: On line 166, replace `A series appointment has no template information, and thus don't use the` with `A series appointment has no template information, and thus does not use the`.
   - Details: The subject `A series appointment` is singular, so the grammatically correct expansion is `does not`, not `do not`. This sits inside the `<h4>Series (simple chain)</h4>` section.

3. **Expand `won't` in the originatingAppointment sentence.**
   - File: `source/appointment/appointment-introduction.xml`
   - Action: On line 171, replace `This style of recurring appointment likely won't use the <code>originatingAppointment</code> property either.` with `This style of recurring appointment likely will not use the <code>originatingAppointment</code> property either.`.
   - Details: Same paragraph as task 2; keep the `<code>originatingAppointment</code>` inline element exactly as-is.

4. **Expand `didn't` in the per-series acceptance paragraph.**
   - File: `source/appointment/appointment-introduction.xml`
   - Action: On line 175, replace `just as it would be normally if they didn't have a sequence.` with `just as it would be normally if they did not have a sequence.`.
   - Details: This is the `<p>` immediately following the Series intro paragraph.

5. **Verify no collateral occurrences remain on the Appointment page.**
   - File: `source/appointment/` (all authored XML/HTML fragments included into `appointment.html`).
   - Action: Run a contractions sweep over `source/appointment/appointment-introduction.xml`, `appointment-notes.xml`, `appointment-fivews-mapping-exceptions.xml`, and `appointment-request-mapping-exceptions.xml` (e.g., regex `[A-Za-z]n['’]t|[A-Za-z]['’](ve|re|ll|s|m|d)\b`), confirm the only remaining hits are in XML comments of `appointment-example-request.xml` (out of scope) and the possessive-style single-quoted terms in `appointment-request-mapping-exceptions.xml` line 34 (not contractions).
   - Details: This matches the ticket's stated scope ("page: appointment") and avoids pulling unrelated files into the fix.
