## Halt: Required repo briefing missing

This ticket asks for changes to the codes inside the `service-type` value set
(remove `BIST`, `ATSS`, `PADP`, `GPAH Other`; rename `HACC` to "Home and
Community Care"). The FHIR Core ValueSet at
`source/healthcareservice/valueset-service-type.xml` (in `HL7/fhir`,
clone HEAD `db79dcd`) only contains a single `compose.include` of
`http://terminology.hl7.org/CodeSystem/service-type` and **no inline codes**.
The codes themselves are owned by **HL7/UTG** (terminology), so any
implementation plan must touch that repository as well as (potentially) the
FHIR Core ValueSet metadata.

A current per-repo briefing is required for every in-scope repository before
this skill can produce a grounded plan. The status is:

- **`HL7/fhir`** — owns `source/healthcareservice/valueset-service-type.xml`,
