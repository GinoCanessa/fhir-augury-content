## Resolution Summary

**Resolution:** Persuasive.

The resolution description states:

> The error occurs twice in https://build.fhir.org/organizationaffiliation.html#jv — will be fixed accordingly.

This is a documentation-only technical correction: the misspelling "seperate"

## Feature Proposal

### Problem Statement

The `organizationaffiliation` resource page contains a spelling error flagged
by the automated spelling/abbreviation QA check (ticket source: "Comment 268"
imported by Lloyd McKenzie). The word **"seperate"** appears twice in the
`Joint Ventures` section of the page and must be replaced with the correct
spelling **"separate"**. This is a pure prose / specification-page fix — there
are no semantic, structural, or conformance implications.

### Proposed Change

Within the FhirCore authoring source for the `OrganizationAffiliation`
resource page (`source/organizationaffiliation/organizationaffiliation-notes.xml`,
which is the `<notes>` fragment included into `organizationaffiliation.html`
by the HL7 publisher), replace both occurrences of `seperate` with `separate`
inside the `Joint Ventures` section (anchor `<a name="jv"></a>` / `<h2>Joint
Ventures</h2>`):

Current text (lines 26–27 of `organizationaffiliation-notes.xml`):

```html
<p>
With a joint venture between multiple organizations, a seperate Organization is created which represents the Venture.
Each Organization in the venture is created through a seperate OrganizationAffiliation resource with
the telecom details provided in each being the contact point to call for enquiries relating to their affiliation in the venture.
</p>
```

Corrected text:

```html
<p>
With a joint venture between multiple organizations, a separate Organization is created which represents the Venture.
Each Organization in the venture is created through a separate OrganizationAffiliation resource with
the telecom details provided in each being the contact point to call for enquiries relating to their affiliation in the venture.
</p>
```

No other changes — no structural edits, no re-flow, no anchor changes, no
element definitions altered.

### Design Rationale

The ticket is classified `Technical Correction` with resolution `Persuasive`
and a resolution description that explicitly limits the fix to the two
misspelled occurrences on `organizationaffiliation.html#jv`. Per the
FhirCore repo-analysis briefing, HL7 publisher pages under `source/<resource>/`
are authored directly as HTML/XML fragments, so the correction is applied at
the source fragment file, not on the generated `organizationaffiliation.html`
output. No alternatives were considered: a verbatim spelling correction is
the only action the resolution asks for.
