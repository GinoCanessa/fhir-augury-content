## Resolution Summary

Resolution type: **Persuasive**. The approved change is:

> Will update the spec to include the correct link below on Appointment.class:
> `http://terminology.hl7.org/ValueSet/encounter-class`

In concrete terms: fix the `Appointment.class` binding so that its `valueSet`
canonical points at the valid THO ValueSet `http://terminology.hl7.org/ValueSet/encounter-class`
(lowercase, hyphenated) instead of the broken `http://terminology.hl7.org/ValueSet/EncounterClass`
(CamelCase) that currently 404s.

## Feature Proposal

### Problem Statement

`Appointment.class` has a preferred binding whose `valueSet` canonical URL is
`http://terminology.hl7.org/ValueSet/EncounterClass`. That URL returns "Not
Found" on terminology.hl7.org, so the binding link rendered on the FHIR
specification page for Appointment is broken. The intent is to bind to the
same value set already used by `Encounter.class`, whose canonical is
`http://terminology.hl7.org/ValueSet/encounter-class` (confirmed in
`source/encounter/structuredefinition-Encounter.xml` line 253). The two
siblings — Appointment and Encounter — were meant to share a binding but
Appointment's StructureDefinition carries a typo-level CamelCase variant
that does not resolve in THO.

### Proposed Change

Single-file technical correction to the Appointment StructureDefinition:

- **Repo:** `HL7/fhir`
- **File:** `source/appointment/structuredefinition-Appointment.xml`
- **Element:** `Appointment.class` (element id `Appointment.class`, line 252)
- **Change:** in the `<binding>` block at line 262–269, update the
  `<valueSet>` on line 268 from

  ```xml
  <valueSet value="http://terminology.hl7.org/ValueSet/EncounterClass"/>
  ```

  to

  ```xml
  <valueSet value="http://terminology.hl7.org/ValueSet/encounter-class"/>
  ```

- **No other binding fields change:** `strength` stays `preferred`,
  `description` stays `"Classification of the encounter."`, and the
  `elementdefinition-bindingName` extension `valueString` stays
  `"EncounterClass"` (the binding *name* is a display label and already
  matches the sibling `Encounter.class` binding on line 249 of
  `source/encounter/structuredefinition-Encounter.xml`).

Resulting binding block (after edit):

```xml
<binding>
  <extension url="http://hl7.org/fhir/StructureDefinition/elementdefinition-bindingName">
    <valueString value="EncounterClass"/>
  </extension>
  <strength value="preferred"/>
  <description value="Classification of the encounter."/>
  <valueSet value="http://terminology.hl7.org/ValueSet/encounter-class"/>
</binding>
```

### Design Rationale

- **Consistency with the sibling binding.** `Encounter.class` already binds to
  `http://terminology.hl7.org/ValueSet/encounter-class`. Aligning
  `Appointment.class` with that exact canonical restores the intended
  cross-resource consistency between Encounter and Appointment (an
  appointment is, semantically, a prospective encounter of a given class).
- **The value set lives in THO, not core.** The canonical starts with
  `http://terminology.hl7.org/`, which is owned by the HL7/UTG repository.
  The value set itself is correct and published; only the reference in
  Appointment is wrong. There is therefore nothing to change in UTG.
- **Lowercase-hyphenated is the THO convention.** THO value-set IDs are
  lowercase-hyphenated (e.g., `encounter-class`), while the
  `bindingName` extension is free-form display text and conventionally
  CamelCase (`EncounterClass`). The ticket author's broken link conflated
  the two. The fix only touches the canonical URL; the bindingName stays.
- **Alternative considered and rejected: introduce a separate, Appointment-
  specific value set.** The resolution explicitly directs reuse of the THO
  `encounter-class` value set — same semantics, same codes — so a new
  value set is not warranted and would fragment terminology.

## Impact Analysis

### Affected Repositories

#### HL7/fhir (FhirCore)

- **Role:** Authors the core Appointment StructureDefinition whose
  `Appointment.class` binding currently references a non-resolving THO
  canonical URL. Fixing the canonical here is the entire change.
- **Affected Files:**
  - `source/appointment/structuredefinition-Appointment.xml` — update the
    `<valueSet value="…"/>` on line 268 inside the `Appointment.class`
    `<binding>` element from `…/ValueSet/EncounterClass` to
    `…/ValueSet/encounter-class`.
- **Change Scope:** Minor (one-line value change in a differential).

### Breaking Changes

No breaking changes. The binding strength (`preferred`), the element
cardinality (`0..*`), the type (`CodeableConcept`), the bindingName
extension (`EncounterClass`), and the semantic value set (THO
`encounter-class`) are all unchanged from what was intended. Consumers
that were relying on the *working* link — there were none, because the
link was broken — are unaffected. Consumers that had locally mapped the
broken URL to the correct value set will see the authored URL now match
reality.

### Related Specifications

- `Encounter.class` in `source/encounter/structuredefinition-Encounter.xml`
  already uses `http://terminology.hl7.org/ValueSet/encounter-class` with
  bindingName `EncounterClass` (verified lines 249 and 253). After this
  fix, Appointment and Encounter agree on both the bindingName and the
  canonical.
- The value set itself,
  `http://terminology.hl7.org/ValueSet/encounter-class`, is owned by
  `HL7/UTG`. No change is required there.

### Related Tickets

- No related Jira tickets surfaced via `cross-referenced FHIR-54232`
  beyond the artifact back-link to `Appointment.class` itself. A keyword
  review (`appointment.class`, `encounterclass`, `binding`) surfaced no
  competing or dependent tickets in the ingested dataset.

### Terminology Impact

No terminology impact. The target value set
`http://terminology.hl7.org/ValueSet/encounter-class` already exists and
is already used correctly by `Encounter.class`. No changes to HL7/UTG
are required; this fix is purely a reference correction in `HL7/fhir`.
