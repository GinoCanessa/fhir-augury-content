## Summary

The ticket asks the FHIR Core editors to make a set of editorial and
status changes to the **Known Identifier Systems** page, plus a small
follow-on update on the **Terminology Module** page so its outbound
link matches the renamed page. Specifically, the page should be
renamed to *Identifier Systems*, its Standards Status should be
promoted to **Normative**, two introductory sentences should be
removed, and the second paragraph should be reworded to (a) explain
that the historical list of registered URIs has been migrated to
HL7 Terminology (`http://terminology.hl7.org`), which is now the
actively-curated registry of identifier systems, and (b) clarify that
all identifier-system URIs defined on `terminology.hl7.org` prior to
the release of FHIR R6 are expected to remain usable in systems
claiming conformance to FHIR R6. The closing pointer to the code
systems list is also reworded for clarity. The Terminology Module
page must then have its "Known Identifier Systems" link text updated
to "Identifier Systems".

The only cross-reference surfaced by Augury is to the Confluence
page **"FHIR R6 Terminology Content Review"** (page id `256510055`,
in space `VOC`), which the comment from `jsnell` identifies as the
TI co-chair review record that authorized the change. No related
Jira tickets, Zulip threads, or GitHub PRs/issues were returned by
the cross-reference index.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Apply the six edits to the *Known Identifier Systems* page exactly
as written in the ticket:

1. Rename the page from "Known Identifier Systems" to **Identifier
   Systems** (page title and any in-page heading; preserve the
   existing URL/anchor — `identifier-registry.html` — to avoid
   breaking inbound links, and add a small note or `<a id="known-identifier-systems"/>`
   shim so old anchor references keep resolving).
2. Promote the **Standards Status** of the page to **Normative**.
3. Delete the second and third sentences of the first paragraph.
4. Insert the new first sentence in the second paragraph that
   points readers at HL7 Terminology
   (`http://terminology.hl7.org`) as the actively-curated
   identifier-system registry.
5. Reword the closing sentence of the second paragraph to read
   exactly: *"For example, all system URIs defined on
   terminology.hl7.org prior to the release of FHIR R6 are
   expected to be used in systems claiming conformance to FHIR
   R6."*
6. Reword the final sentence to *"See the list of code systems
   that can be used in the system element of the Coding
   datatype"*, retaining the existing link target (the Code
   Systems tab).

Then, on the *Terminology Module* page, replace the link text
"Known Identifier Systems" with "Identifier Systems" (link target
unchanged).

#### Justification

The change is purely editorial plus a status promotion and is
unambiguously specified by the reporter. The TI co-chair review
captured at Confluence page `256510055` ("FHIR R6 Terminology
Content Review") explicitly endorsed it, which satisfies the
balloting threshold for a Normative promotion of an introductory
page whose substantive registry has already moved to THO. Pointing
implementers at `terminology.hl7.org` as the canonical, actively
curated list of identifier systems also matches the long-standing
direction of the Vocabulary WG (registry content lives in THO,
narrative explanation lives in the FHIR spec). Preserving the
existing URL avoids breaking the many in- and out-of-spec
references to `identifier-registry.html`.

---

### Disposition B: Alternative Approach

#### Proposal

Accept the substance of the change but tighten the rename and
promotion mechanics:

- **Page title:** rename to "Identifier Systems" as requested,
  but do **not** change the source filename or anchor; instead
  add an explicit redirect/alias entry so both
  `identifier-registry.html` and a new
  `identifier-systems.html` resolve, and add a short editorial
  note ("formerly *Known Identifier Systems*") in the page
  intro so search engines and bookmarks aren't silently
  broken.
- **Normative promotion:** scope the Standards Status change to
  the *narrative* on the page only, and call out in the page
  preamble that the **registry of identifier systems is
  maintained at terminology.hl7.org and is not itself part of
  the Normative content**. This avoids accidentally normatising
  THO-managed content by reference.
- **Wording:** adopt the new second-paragraph sentence about THO
  but link the phrase "HL7 Terminology" to a stable THO landing
  page rather than the bare `http://terminology.hl7.org/` URL,
  so the link survives any future THO URL rearrangements.
- **Terminology Module link:** make the same text update as in
  Disposition A.

#### Justification

The substantive intent of the ticket is correct, but two
mechanical risks are worth handling explicitly: (1) renaming
without a redirect would silently break a heavily-linked page,
and (2) a flat "page is Normative" status, when the page now
mostly serves as a pointer to externally-curated THO content,
risks creating ambiguity about what is actually under the
Normative ballot. Tightening both before merge keeps the
implementer-facing benefit of the rename and promotion while
avoiding scope creep into THO governance.

---

### Disposition C: Decline

#### Proposal

Decline the rename and the Normative promotion. Keep the page
titled *Known Identifier Systems* and at its current Standards
Status, but adopt only the *non-controversial* wording changes
(items 3, 4, 5 and 6) so the page accurately reflects that the
URI list now lives at THO. Also leave the Terminology Module
link text unchanged.

#### Justification

A rename plus a Normative promotion is more than a typo fix and
is being requested without a Zulip discussion thread or any
linked implementer feedback in the cross-reference index. If the
page's substantive content is now a pointer to an
externally-curated THO registry, declining the Normative
promotion avoids binding the FHIR Core ballot to content that
the FHIR editors do not directly control. The ticket would
remain partially addressed by the wording fixes, and the
larger change could be reintroduced once the TI review record
is itself adopted as the rationale alongside a public discussion.

---

### Recommendation

**Recommended disposition:** A

This is a TI co-chair–endorsed editorial cleanup whose intent
is unambiguous: the page is no longer a registry, it is a
pointer to the THO registry, and the wording should say so.
Disposition A delivers exactly what the reporter and the TI
review captured at Confluence `256510055` asked for, including
the Standards Status promotion that signals the page itself is
now stable. The mechanical concerns raised in Disposition B
(URL preservation, scoping the Normative status to the
narrative, link stability) are real and worth honouring at
implementation time, but they are best handled as routine
publication-tooling decisions during the edit rather than as a
distinct workgroup disposition. Disposition C under-delivers:
declining the rename and the status change leaves the
implementer-facing description of where the identifier-system
registry actually lives in a half-finished state, and the TI
review on Confluence already provides the workgroup endorsement
that would otherwise justify holding the change.
