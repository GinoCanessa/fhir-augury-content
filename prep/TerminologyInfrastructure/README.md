These tickets ask the Terminology Infrastructure WG to clean up how
parameters of the terminology operations are specified — the location
semantics of `$validate-code` output `issues`, the wording of input
parameter descriptions across `$validate-code` / `$expand` / `$translate`
/ `$lookup` / `$subsumes`, and the data type used for parameters that
identify versionable terminology resources (`uri` vs. `canonical`).

#### Topic: Duplicate terminology resources vs. HL7 Terminology (THO)

Both tickets report that ValueSets and CodeSystems shipped in the FHIR
Extensions Pack are also defined in HL7 Terminology (THO) under
different canonical URLs, and ask Terminology Infrastructure (with
FMG) to choose a single canonical home per artifact, deprecate the
duplicate, and prevent recurrence. The work has since been folded into
the cross-specification consolidation ticket
[FHIR-55033](./FHIR-55033.md).

##### Linked Ticket Group: FHIR-47270

FHIR-47270 originally raised the duplication problem, and FHIR-47498
catalogued the 14 CodeSystem / 14 ValueSet duplicates in an attached
spreadsheet two days later; the two tickets directly link each other
and must be tracked as one workstream — both have been superseded by
FHIR-55033 (in the FHIR Core specification bucket above).
