### Change Request

#### Topic: Duplicate terminology resources between Extensions Pack and THO

Both tickets report the same underlying problem: a substantial set
of `CodeSystem` and `ValueSet` resources are defined in both the
FHIR Extensions Pack and HL7 Terminology (THO) under different
canonical URLs, and they need to be reconciled (single master copy,
deprecation of duplicates). They cite each other and roll up into
the FHIR-Core consolidation ticket [FHIR-55033](./FHIR-55033.md).

##### Linked Ticket Group: FHIR-47270

These two tickets are explicitly linked in Jira and describe the
same duplicate-terminology-resource problem from complementary
angles — FHIR-47270 motivates the issue and proposes a policy
direction; FHIR-47498 supplies the enumerated list of 14 + 14
duplicate CodeSystems and ValueSets. They must be dispositioned
together so the policy and the concrete resource list stay
consistent.
