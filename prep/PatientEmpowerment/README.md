### Change Request

#### Topic: Scope, purpose, and definition of the PHR IG

A cluster of tickets asking the Patient Empowerment WG to sharpen
exactly what the PHR FHIR IG is — what "PHR" means in scope, whether
the IG defines a data model for a PHR system or just a FHIR exchange
profile, and what conformance "MUST use FHIR" claims actually require
of implementers. Tackling these together yields a single, consistent
purpose statement that the rest of the IG can lean on.

##### Linked Ticket Group: FHIR-49070

`FHIR-49071` is explicitly linked to `FHIR-49070` and both ask the
same root question — whether the IG's "MUST use FHIR" / "data is
stored in FHIR" statements describe a conformance requirement, an
exchange-format requirement, or an implementation assumption. They
should be resolved together so the answer is consistent across the
Conformance and Data Model sections.
