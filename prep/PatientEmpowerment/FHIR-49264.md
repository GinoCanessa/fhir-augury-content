## Summary

The reporter asks why the Personal Health Record (PHR) FHIR Implementation
Guide — which is a universal-realm (UV) specification — references US Core
in Table 4.1. The implicit concern is that an international IG should not
appear to depend on, or privilege, a single national realm's profiles. No
cross-referenced Jira tickets were returned by the corpus, so the
question stands on its own and is not part of a broader linked
discussion. A single comment from `awatson1978` clarifies that the US
Core references in Table 4.1 are examples only — neither prescriptive
nor required — and notes that the editors are actively soliciting more
international examples (e.g., Netherlands, Japan) for future ballots.

## Proposed Dispositions

> Note: because no PHR-IG repo briefing is available in the local
> `cache/github/repos/` tree, the dispositions below describe the
> editorial change in spec terms (Table 4.1 of the PHR IG) rather than
> naming a specific authoring file path. If/when the PHR IG is added
> via `repo-analysis`, the implementing PR target paths should be
> filled in before action.

### Disposition A: Accept As Requested

#### Proposal

Treat the appearance of US Core in Table 4.1 as a defect for a UV IG.
Edit Table 4.1 of the Personal Health Record (FHIR) IG to remove the
US Core profile references, replacing each row that currently cites a
US Core profile with either (a) the corresponding base FHIR R4/R5
resource (e.g., `Patient`, `AllergyIntolerance`, `Condition`,
`Observation`), or (b) a UV-realm equivalent if one exists in the
international ecosystem (e.g., the IPS profiles in
`hl7.fhir.uv.ips`). Add a short note above Table 4.1 stating that
realm-specific profiles (US Core, UK Core, Nictiz, etc.) are out of
scope for the table and that implementers in a given realm may
substitute their realm's profile.

#### Justification

Universal-realm IGs are expected to be realm-neutral in their normative
or quasi-normative tabular content. Even when a national profile is
listed only as an "example", its presence in a structured table is
easily read by implementers as guidance to use that profile, and it
implicitly privileges one jurisdiction. Removing the US Core entries
restores realm neutrality and aligns with the editor's own statement
that the references are "merely examples" — if they are merely
examples, they do not belong in a normative-looking table in a UV IG.
This is also the disposition most directly responsive to what the
reporter asked.

---

### Disposition B: Alternative Approach

#### Proposal

Keep Table 4.1 structurally as-is but (1) clearly relabel the column
that currently shows US Core profiles as "Example realm-specific
profile (illustrative only)", (2) add a header note stating that the
PHR IG is realm-neutral and that the entries in this column are
non-normative illustrations of how a national realm might bind a row
to a national profile, and (3) commit to populating additional rows /
a parallel column with examples from other realms (Netherlands /
Nictiz, Japan / JP Core, IPS, etc.) as they become available, as
already signalled in the comment from `awatson1978`. Optionally move
the column out of Table 4.1 into an informative appendix
("Realm-specific profile examples") so the main table contains only
base-resource references.

#### Justification

This preserves the editorial intent — concrete, implementer-friendly
examples drawn from a real, mature national IG — while explicitly
defusing the realm-bias concern. US Core is, in practice, the most
mature published realm profile set, so removing it entirely (per
Disposition A) deletes useful reference material before its
international counterparts are ready. Relabelling and adding a header
caveat is a low-cost change that is responsive to the reporter and
consistent with the comment author's stated intent of expanding to
more realms over future ballots. Moving the column to an informative
appendix further separates "normative table content" from
"illustrative examples", which is a pattern several other UV IGs
follow.

---

### Disposition C: Decline

#### Proposal

Close as Not Persuasive / Resolved – No Change. The text in the
existing comment thread already makes clear that the US Core entries
in Table 4.1 are illustrative examples, not prescriptive content, and
that more realms will be added as material becomes available. No
specification change is required.

#### Justification

The reporter's concern is satisfied by clarification rather than by a
spec change: US Core examples are not normative, do not constrain
non-US implementers, and do not change the IG's UV scoping. The
existing comment from the editor on the ticket is sufficient to put
that on the record. Declining avoids churn in Table 4.1 immediately
before more international examples land and avoids deleting concrete,
implementer-useful references that today have no published
equivalent. A reply on the ticket repeating the clarification is the
proportionate response.

---

### Recommendation

**Recommended disposition:** B

The reporter's underlying concern — that a UV IG appears to depend on
a single realm's profile set — is real and not adequately answered by
"these are just examples" if the examples sit unlabelled in a
structured table that looks normative. Disposition C (decline) leaves
that perception in place and is likely to attract repeat tickets from
other international reviewers. Disposition A (delete) is responsive
but throws away the only mature, concrete examples available today,
ahead of the international replacements landing. Disposition B
relabels and isolates the realm-specific column so the main table is
realm-neutral, keeps the useful US Core illustrations until peers
from other realms catch up, and operationalises the editor's already-
stated commitment to expand to Netherlands, Japan, and others. It is
the minimum editorial change that fully addresses the reporter's
question without losing implementer value.
