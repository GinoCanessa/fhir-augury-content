## Summary

The reporter (Aaron Nusstein) asks the Clinical Quality Information work
group to revise the *short descriptions* on the sliced
`Measure.group.population` elements in the **CQFM Ratio Measure** profile
(US FHIR Guidance – Quality Reporting, §17.54.1.1). The current short
descriptions are characterized as "not clear or descriptive of what [each]
slice means, despite being somewhat intuitive." The reporter proposes
reworking the short descriptions to be more helpful to the reader. The
single cross-reference picked up by FHIR Augury is the FHIR element
`Measure.group.population` itself, confirming that the scope is limited
to the slice metadata (short / definition text) on that element within
the CQFM Ratio Measure profile — no behavior, cardinality, or binding
changes are requested.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Rewrite the `short` text (and, where it adds clarity, the
`definition` text) on each sliced
`Measure.group.population` element in the CQFM Ratio Measure profile
(`ratio-measure-cqfm`) so that each slice's purpose is self-evident
from its short description alone. Concretely, replace the current
terse labels with descriptions that name (a) the role of the
population in a *ratio* measure and (b) what it counts. Suggested
replacements (final wording subject to CQI editorial review):

- `initial-population` → "Patients/events eligible to be evaluated by
  this ratio measure (the denominator's source set)."
- `denominator` → "Subset of the initial population that forms the
  denominator of the ratio."
- `denominator-exclusion` → "Cases removed from the denominator (and
  therefore from the ratio) because they do not meet measurement
  criteria."
- `numerator` → "Subset of the denominator (or, for some ratio
  measures, of the initial population) that meets the measure's
  numerator criteria — i.e., the events being counted in the ratio."
- `numerator-exclusion` → "Cases removed from the numerator that
  would otherwise have qualified."
- `measure-population` (if present on the ratio profile) → "The
  population over which the ratio is computed when an explicit
  measure-population slice is supplied."

The change is editorial: no cardinalities, types, bindings, invariants,
or slice discriminators are altered.

#### Justification

The request is narrowly scoped to documentation quality on a single
profile, and it targets the user-facing `short` text that appears in
the rendered IG and in tooling. Improving slice short text is fully
consistent with the IG's stated audience (measure authors and
implementers) and with FHIR's general guidance that
`ElementDefinition.short` should make the element's purpose
immediately recognizable. Because nothing about the *conformance*
surface changes, the change is non-breaking for existing CQFM Ratio
Measure instances and tooling. The reporter explicitly framed this as
an editorial improvement; there is no community signal (Zulip, linked
tickets) suggesting controversy.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than (or in addition to) rewriting the per-slice `short`
descriptions in-place, expand the **narrative** for §17.54.1.1 *CQFM
Ratio Measure* in the IG to include a small table that maps each
population slice (`initial-population`, `denominator`,
`denominator-exclusion`, `numerator`, `numerator-exclusion`,
`measure-population`) to a one-sentence plain-language description and
to its role in the ratio formula. Keep the existing slice `short`
values but tighten them only where they are actively misleading. The
narrative table becomes the canonical "what each slice means"
reference; the StructureDefinition `short` values stay terse and
machine-friendly.

#### Justification

CQFM is a profile on the base `Measure` resource, and the slice
discriminator codes (`initial-population`, `denominator`, …) come from
`http://terminology.hl7.org/CodeSystem/measure-population`. Rewording
the `short` text in only the *Ratio* profile risks divergence with the
parallel CQFM Proportion / Continuous Variable / Cohort profiles,
where the same population codes are used with subtly different
semantics. A narrative table localized to §17.54.1.1 explains the
*ratio-specific* meaning without forcing every other CQFM measure
profile to track wording changes, and avoids creating per-profile
divergence in element `short` text that downstream tooling might
display side-by-side.

---

### Disposition C: Decline

#### Proposal

Decline the change as out-of-scope editorial polish that is already
addressed by the existing IG narrative. Close as **Persuasive with
Modification – No Change** if the reporter accepts the existing
documentation, otherwise as **Not Persuasive** with a pointer to the
profile narrative and to the underlying FHIR `Measure` documentation
that defines each population code.

#### Justification

The reporter themselves notes the meanings are "somewhat intuitive,"
and the population slice codes are defined normatively in the
`measure-population` CodeSystem and in the base `Measure` resource
documentation. CQI's editorial budget is finite and the work group has
historically prioritized conformance-affecting tickets over short-text
rewording, particularly when no implementer has reported a
misinterpretation incident. Declining keeps the profile stable and
avoids a wave of look-alike rewording requests against the parallel
CQFM measure-type profiles.

---

### Recommendation

**Recommended disposition:** A

This is a low-risk, editorial change against a single profile in a
single IG. The reporter has identified a specific, real readability
weakness; the fix costs only editor time and rendered IG regeneration;
and the change cannot break conformance because it touches only
`ElementDefinition.short` (and optionally `definition`) text. The work
group should accept the change, agree on final wording in the meeting
(the strings proposed in Disposition A are a starting point, not a
final draft), and apply the edits to the `ratio-measure-cqfm`
StructureDefinition in the `CQF-Measures` IG. The narrative-table
idea from Disposition B is worth folding in as a *bonus*: tightening
the slice `short` text *and* adding a population-role table in
§17.54.1.1 gives readers both an in-tooling hint and a single
reference table, with no additional conformance risk.
