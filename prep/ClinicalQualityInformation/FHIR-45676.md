## Summary

The reporter is reading section **1.3.2 "Quality Measurement Standards
Landscape"** of the *HL7 FHIR® Implementation Guide: Quality Measures,
Edition 1 — US Realm* and notes that the IG's link to the **Clinical
Quality Framework** points at an HL7 Confluence page. They are asking
whether general readers / implementers of the IG will actually have
access to that Confluence page when they follow the link, i.e. whether
the link is publicly resolvable or sits behind an HL7 / Confluence
login. This is a documentation-quality / reader-accessibility question
rather than a request to change normative IG content. No
cross-referenced Jira tickets, Zulip threads, or GitHub items were
surfaced for this ticket, so the request stands on its own.

## Proposed Dispositions

> Note on grounding: because no GitHub cross-references were surfaced
> for this ticket, no repo briefing was loaded. The dispositions below
> describe what *would* change in the published Quality Measures IG
> narrative (the page that hosts §1.3.2). The exact authoring path
> should be confirmed by the CQI work group / IG editors before the
> change is applied; the briefing for the Quality Measures IG repo was
> not consulted because the ticket itself surfaced no repo
> cross-reference.

### Disposition A: Accept As Requested

#### Proposal

Treat this as a documentation defect against the published Quality
Measures IG (Edition 1, US Realm). In the IG's §1.3.2 "Quality
Measurement Standards Landscape" narrative source:

1. Identify the anchor text "Clinical Quality Framework" and the
   Confluence URL it currently points to.
2. Replace the Confluence link with the **public** Clinical Quality
   Framework landing page on HL7.org (the CQI Work Group / CQF project
   page) so it resolves without an HL7 Confluence login.
3. If a public landing page does not exist, file a parallel request
   with the CQI work group to publish a short public CQF overview page
   on HL7.org and point §1.3.2 at that page once available; in the
   interim, annotate the link in the IG with text such as "(HL7
   Confluence — login may be required)" so readers are not surprised.
4. Republish the IG narrative (no FHIR conformance artifacts change).

#### Justification

The reporter's concern is concrete and easy to verify: HL7 Confluence
spaces are frequently visible only to logged-in HL7 members or to
specific work-group members, while the Quality Measures IG is intended
to be readable by any implementer. A broken or gated link in §1.3.2
degrades the IG's usefulness as an on-ramp for new implementers and
contradicts the general FHIR principle that published normative
narrative should be self-sufficient and independently navigable.
Repointing to a public page (or, failing that, labeling the link as
gated) is the minimal, lowest-risk fix and does not affect any
StructureDefinition, ValueSet, or measure logic.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than treating §1.3.2 as a single-link fix, take the opportunity
to harden *all* outbound links in the Quality Measures IG against the
same class of problem:

1. In §1.3.2, replace the Confluence link with a brief in-IG paragraph
   summarizing what the Clinical Quality Framework is and citing the
   primary public references (HL7 CQI work group page, CQL spec,
   QI-Core / Quality Measure IG family) instead of relying on a single
   external Confluence URL.
2. Add a short editorial note (or a CI link-checker job in the IG's
   build) that flags any `confluence.hl7.org` URL in the IG's
   narrative for editor review before publication, so future editions
   do not re-introduce gated links.
3. Apply the same treatment to any other Confluence URLs found in the
   IG narrative.

#### Justification

The literal request (Disposition A) only fixes one link. The
underlying risk — HL7 Confluence pages that *look* like public
references but require login, or that get reorganized and 404 — applies
to any Confluence URL the IG cites. Embedding a short summary in the
IG itself, plus a build-time guard, makes the IG narrative more
durable and more useful offline. The trade-off is more editor effort
for this and future editions, and a small narrative-text expansion in
§1.3.2; against that, it removes a recurring class of reader
complaints.

---

### Disposition C: Decline

#### Proposal

Close as **Not Persuasive / Working as Intended**: leave §1.3.2 as
written, on the rationale that the linked Confluence page is the HL7
work-product home for the Clinical Quality Framework and that any HL7
member or CQI participant — the IG's primary audience for the
"Standards Landscape" pointer — can authenticate and read it. Optionally
add a one-line note in the IG that an HL7 / Confluence login may be
required.

#### Justification

The link is informational background in a "landscape" section, not a
normative reference an implementer must follow to conform to the IG.
Many HL7 IGs link to Confluence work-group pages by long-standing
practice. If the link target itself is correct and intentional,
"declining" the change keeps the IG aligned with how the rest of HL7
treats CQF references and avoids editorial churn for what is, at the
end of the day, a single hyperlink.

---

### Recommendation

**Recommended disposition:** A

The ticket is a low-risk documentation question, not a spec-design
debate, and the reporter's concern is legitimate — an IG's
"Standards Landscape" section is exactly the kind of place where new
implementers go to orient themselves, and a gated link there is a
poor first impression. Disposition A fixes the actual problem with
minimal effort and no impact on any conformance artifact. Disposition
B is a reasonable longer-term hardening but is more work than this
ticket, on its own, justifies; if the work group wants that broader
cleanup, it should be tracked as a separate IG-quality task and
referenced from this ticket. Disposition C is hard to defend: even if
some readers can authenticate to Confluence, a published IG should not
assume that, and labeling the link as gated (which Disposition C
allows as an option) is functionally a partial Disposition A anyway.
