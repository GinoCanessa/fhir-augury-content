## Summary

The reporter, Aaron Nusstein, asks the Clinical Quality Information work
group to revise the **short descriptions of the slices on
`Measure.group.population`** in the **CQFM Continuous Variable Measure**
profile (cqfmeasures IG, section 17.48.1.1). The current short
descriptions are characterised as "not clear or descriptive of what the
slice means" — the meanings can be intuited, but a reader of the
rendered StructureDefinition does not get an immediate, plain-language
explanation of each population role (initial-population,
measure-population, measure-population-exclusion, measure-observation).
The change requested is purely an editorial / definitional improvement to
the slice's `short` (and possibly `definition`) text; no constraint,
cardinality, binding, or slicing-discriminator change is asked for.

The only cross-reference Augury surfaces is the FHIR element mention
itself (`Measure.group.population`); there are no related Jira tickets,
Zulip discussions, or GitHub items linked to the ticket key.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

In the **CQFM Continuous Variable Measure** profile
(`StructureDefinition-measure-cv`, section 17.48.1.1 of the cqfmeasures
IG), rewrite the `short` text on each slice of
`Measure.group.population` so that it names the population role and
states what it represents in a continuous-variable measure, rather than
echoing the slice name. Suggested replacements (workgroup to refine
wording):

- `initial-population` slice — short: *"Initial Population — patients/events eligible to be considered by the measure"*.
- `measure-population` slice — short: *"Measure Population — the subset of the Initial Population for whom the measure-observation is computed"*.
- `measure-population-exclusion` slice — short: *"Measure Population Exclusion — members of the Measure Population removed before observation"*.
- `measure-observation` slice — short: *"Measure Observation — the value computed for each member of the Measure Population (the continuous variable)"*.

Where the corresponding `definition` text is similarly terse, extend it
in the same pass with one additional sentence pointing at the QI-Core
expression-style guidance for that population role. No
slicing-discriminator, cardinality, binding, must-support, or invariant
change is made; this is an editorial revision of the SD's slice
metadata only.

#### Justification

The request is narrow, low-risk, and aligns with how other CQFM measure
profiles (Proportion, Ratio, Cohort) describe their population slices.
Improving slice `short` text increases readability of the rendered
StructureDefinition and the IG's "Differential" / "Snapshot" tables,
which is exactly where implementers first encounter the
continuous-variable population shape. There is no normative impact:
`short` and `definition` are descriptive metadata, so existing
instances, validators, and conformance behaviour are unaffected.

---

### Disposition B: Alternative Approach

#### Proposal

Address the same underlying clarity problem at the **profile-narrative
level** rather than (or in addition to) editing slice `short` text:

- Add or expand a "Population criteria for Continuous Variable measures"
  subsection in the profile's intro / notes page (the `intro`
  pre-fragment that renders above the StructureDefinition tables in
  section 17.48.1.1), explaining each of the four population roles in
  plain language, with one worked example.
- Touch up the slice `short` only where it is actively misleading; rely
  on the new narrative for the explanatory content.
- Optionally, add a single shared `cqfm-population-roles` glossary
  entry in the IG's general "Measure Conformance" page and link the
  profile narrative to it, so the same wording can be reused by the
  Proportion / Ratio / Cohort measure profiles.

#### Justification

Slice `short` text is constrained by what renders well in narrow IG
tables (often truncated to a few words). Pushing the explanatory
content into the profile narrative gives room for a definition that
actually teaches the reader, while keeping the slice rows tidy. It also
avoids one-off wordings on every measure-type profile diverging over
time; centralising the explanation in the IG narrative is more
maintainable than refining `short` text on four slices in isolation.
Trade-off vs. Disposition A: a reader who only opens the rendered SD
table still sees the original short text, so for the strictest read of
the request this option is incomplete unless paired with at least a
light edit of the slice `short`s.

---

### Disposition C: Decline

#### Proposal

Decline as **Not Persuasive — Editorial Only / Insufficient Specifics**.
The current slice short descriptions correspond to the canonical FHIR
Measure population codes (`initial-population`, `measure-population`,
`measure-population-exclusion`, `measure-observation`) whose meanings
are normatively defined in the base FHIR `Measure` resource and the
`measure-population` code system. The reporter acknowledges the
meanings are "somewhat intuitive". Without proposed replacement text or
a concrete example of a misread, the workgroup has no actionable
change.

#### Justification

`short` text on a profile slice is intentionally minimal and is meant
to be read alongside the binding to `measure-population` and the IG
narrative, both of which already define each role. Re-wording four
strings without a concrete proposal risks churn in the published IG
without measurable benefit. If the reporter (or another implementer)
returns with proposed wording or a citation showing the current text
caused a real implementation error, the request can be reopened with a
bounded scope.

---

### Recommendation

**Recommended disposition:** A (with a light dose of B)

Disposition A is recommended because the request is small, the cost of
the change is low (text-only edits to a single profile in the
cqfmeasures IG authoring source for `StructureDefinition-measure-cv`),
and the benefit — clearer first-read comprehension of the
Continuous-Variable population shape — accrues to every implementer
who lands on section 17.48.1.1. Decline (C) is hard to justify for a
purely editorial fix that the reporter has already framed as a
readability problem. The workgroup should consider folding in the
narrative/glossary improvement from Disposition B at the same time, so
the slice `short`s and the surrounding profile narrative stop at a
consistent place; that combination is the highest-value, lowest-risk
outcome for the next CQFM publication.
