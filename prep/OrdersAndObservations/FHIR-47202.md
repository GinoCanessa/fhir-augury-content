## Summary

Giorgio Cangioli proposes a purely editorial revision to a single sentence in
the "Design choices" paragraph of the Laboratory Report (FHIR) IG. The current
sentence is awkward and contains a duplicated clause ("proposes a scalable
approach This is done by supporting a scalable approach…"). The reporter
supplies a cleaner replacement that preserves the original meaning: the IG
presents a scalable approach that lets different jurisdictions choose the
solution that best fits their requirements while still ensuring laboratory
report data can be retrieved via `DiagnosticReport` searches. No new
normative content, no resource or element changes, and no cross-referenced
Jira tickets, Zulip discussions, or GitHub items are associated with the
ticket.

## Details

**Description:**

> Suggest to change the sentence in the Design choices paragraph
>
> "This guide proposes a scalable approach This is done by supporting a
> scalable approach allowing the different jurisdictions to select the
> solution that better fits with their requirements; while assuring the
> capability for everyone to retrieve laboratory report data by searching per
> DiagnosticReport."
>
> into
>
> "This guide presents a scalable approach that enables different
> jurisdictions to select the solution that best fits their requirements,
> while ensuring the ability for all users to retrieve laboratory report data
> through DiagnosticReport searches."

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Replace the current sentence in the "Design choices" paragraph of the
Laboratory Report (FHIR) IG narrative with the reporter's exact rewording:

> "This guide presents a scalable approach that enables different
> jurisdictions to select the solution that best fits their requirements,
> while ensuring the ability for all users to retrieve laboratory report
> data through DiagnosticReport searches."

This is a pure narrative edit (no StructureDefinition, ValueSet, search
parameter, or example changes). The edit lands on whichever markdown/XML
narrative source backs the IG's Design choices page in the Laboratory
Report (FHIR) IG repository.

#### Justification

The current sentence is grammatically broken — it duplicates the clause
"proposes a scalable approach / supporting a scalable approach" — and is
clearly a copy-paste artefact rather than intentional content. The proposed
replacement preserves the original meaning verbatim (scalability,
jurisdictional choice, DiagnosticReport-based retrievability) while reading
cleanly. The ticket is `block-vote-4` labelled, which is consistent with
trivial editorial fixes batched for an en-bloc disposition. There is no
downstream impact on conformance, validation, or implementations.

---

### Disposition B: Alternative Approach

#### Proposal

Accept the editorial intent but tighten the wording further while also
explicitly naming the two retrieval guarantees the paragraph is trying to
convey. Suggested revision:

> "This guide adopts a scalable approach: jurisdictions can select the
> solution that best fits their requirements, while consumers retain the
> ability to retrieve laboratory report data through `DiagnosticReport`
> searches."

In addition, code-format `DiagnosticReport` so it is visually consistent
with other resource references in the IG narrative.

#### Justification

The reporter's phrasing is a strict improvement over the broken original,
but it still buries the two distinct guarantees (jurisdictional flexibility
and consistent `DiagnosticReport`-based retrieval) in a single subordinate
clause. Splitting them with a colon makes the design intent more legible
to first-time readers, and code-formatting the resource name aligns with
typical FHIR IG narrative conventions. The trade-off is that this departs
from a verbatim accept and may slightly slow the en-bloc vote because the
text now differs from what the reporter literally requested.

---

### Disposition C: Decline

#### Proposal

Decline the change on the grounds that the existing sentence, while
awkward, is comprehensible and that editorial-only changes can be deferred
to a later, batched copy-edit pass over the IG.

#### Justification

This is the weakest of the three options. The only defensible rationale
for declining would be process-based (avoid touching narrative under a
freeze) rather than substantive. The current sentence contains a clear
duplication ("proposes a scalable approach This is done by supporting a
scalable approach"), which a reader will reasonably perceive as a defect
rather than a stylistic choice, so leaving it in place harms the IG's
perceived quality with no offsetting benefit.

---

### Recommendation

**Recommended disposition:** A

The request is a verbatim editorial fix to a sentence that is objectively
broken (duplicated clause). The reporter has supplied a clean replacement
that preserves the original meaning, the ticket is flagged
`block-vote-4` — explicitly intended for en-bloc approval of low-risk
edits — and there is no impact on resources, profiles, terminology,
search parameters, or examples. Disposition B is a reasonable polish but
introduces wording that the reporter did not request and would need its
own review; Disposition C has no substantive grounding. Accepting as
requested is the fastest path to removing a visible defect from the
Laboratory Report (FHIR) IG narrative.
