## Summary

The ticket reports that the navigation menus in the Clinical Order Workflow
(COW) implementation guide are inconsistent in their capitalization style:
some menus use title case for every word, while others only capitalize the
first word (sentence case). The reporter, Brian Swinkels, supplies three
screenshots from the published guide and from a working branch to illustrate
the inconsistency, and states a preference for title case ("capitalizing the
menus looks better"). The request is for the COW IG to adopt a single
consistent capitalization convention across all menus.

No related Jira tickets, Zulip discussions, or GitHub items were surfaced
through cross-referencing or text search, so the request stands on its own
without prior community context.

## Proposed Dispositions

The dispositions below are limited to editorial / IG-build-time changes
because the ticket does not touch any FHIR resource, profile, terminology
artifact, or normative content. No repo briefings were available to ground
specific file paths, so the proposals describe the change at the IG level
and call out where the COW IG project would need to be located before
implementation.

### Disposition A: Accept As Requested

#### Proposal

Adopt **title case** as the single capitalization convention for all top-level
and submenu entries in the Clinical Order Workflow (COW) implementation
guide, matching the reporter's branch screenshot. Concretely:

- Audit the COW IG's navigation source — typically the `menu.xml` (or
  equivalent `_menu.md` / `sushi-config.yaml` `menu:` block) in the COW IG
  repository — and rewrite every menu label so that all major words are
  capitalized (e.g., `Order Workflow Overview`, `Profiles And Extensions`,
  `Examples`, `Downloads`).
- Apply title case to the existing entries that currently use sentence case
  (e.g., `Profiles and extensions` → `Profiles and Extensions`).
- Add an editorial note in the IG's contribution guidance / `README.md`
  stating that menu labels MUST use title case so future contributors do
  not reintroduce the inconsistency.
- Re-publish the IG so the change takes effect on the next build.

#### Justification

This directly addresses what the ticket asks for and matches the reporter's
explicit preference. Title case is the dominant convention used elsewhere in
HL7 IGs (e.g., the published US Core, IPS, and base FHIR specification
navigation use title case for menu items), so adopting it brings COW into
line with the rest of the FHIR ecosystem. Because the change is purely
editorial and confined to navigation labels, there is no impact on
conformance, profiles, search parameters, or implementer behavior.

---

### Disposition B: Alternative Approach

#### Proposal

Adopt **sentence case** instead, and document the choice in the IG. The
substantive action is the same as Disposition A — pick one style and apply
it across every menu — but the chosen style is the modern web/UX convention
of capitalizing only the first word and proper nouns (e.g.,
`Order workflow overview`, `Profiles and extensions`,
`HL7 Order Workflow`). The COW IG would then update its menu source and
contribution guidance to call out sentence case as the rule.

#### Justification

Sentence case is the convention used by most contemporary documentation and
UI style guides (Google, Microsoft, GitHub Primer, Atlassian) on the grounds
that it is easier to read in long labels, more accessible for screen readers,
and friendlier to non-English locales where title case has no analog. If the
workgroup is willing to depart from the rest of the FHIR ecosystem on
editorial grounds, sentence case is the more defensible long-term style.
The trade-off versus Disposition A is consistency-with-FHIR (favors A)
versus consistency-with-modern-web (favors B); both are equally cheap to
implement.

---

### Disposition C: Decline

#### Proposal

Mark the ticket as **Not Persuasive — Editorial / Out of Scope** with a
note that the FHIR Jira tracker is intended for content and conformance
issues, and that menu capitalization should be handled in the COW IG's
own repository as a routine editorial PR rather than through a tracked
change request.

#### Justification

The change has zero impact on the spec content (no resource, profile,
extension, value set, search parameter, or operation is affected). It is
also a pure styling preference — neither title case nor sentence case is
"wrong" — and resolving it through the workgroup ballot/tracker process
risks setting a precedent that every label-styling preference must be
adjudicated by O&O. A direct PR against the COW IG repository, reviewed by
the IG editors, is the lighter-weight and more conventional path for
purely cosmetic IG changes.

---

### Recommendation

**Recommended disposition:** A

Disposition A is the cleanest outcome for the workgroup. The fix is
trivially small and entirely editorial, the reporter has already done the
work to demonstrate the preferred style on a branch, and title case keeps
COW visually consistent with the rest of the FHIR ecosystem (base spec,
US Core, IPS, etc.) — which is the larger consistency concern and
out-weighs the modern-web argument for sentence case in B. Disposition C
is technically defensible but feels heavy-handed for a request that costs
roughly one PR to satisfy and that genuinely improves the reader's
experience of the published guide. Accept as requested, apply title case
across the COW IG menu, and add a one-line editorial rule to prevent
regression.
