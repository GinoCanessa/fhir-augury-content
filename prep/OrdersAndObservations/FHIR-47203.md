## Summary

The reporter (Giorgio Cangioli) asks the Orders & Observations work group
to verify whether the "Navigating the profiles" section of the
international **Laboratory Report (FHIR)** implementation guide — which
was originally taken from the EU Laboratory Report guide — is still
useful, and, if it is retained, to update the diagrams it contains so
they reflect the current state of the IG. The ticket has already been
triaged with a resolution of **Persuasive with Modification**, the
agreed modification being to **remove the "Navigating the profiles"
section and the diagrams included in it** rather than refresh them.
There are no cross-referenced Jira tickets, Zulip threads, or GitHub
items associated with FHIR-47203 in the augury cache, so the request
stands on its own as an editorial / IG-narrative cleanup.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Retain the "Navigating the profiles" section in the Laboratory Report
(FHIR) IG and refresh its content so it accurately reflects the
current set of profiles. Concretely:

- Re-walk the IG's profile list (DiagnosticReport-lab profile,
  Observation-lab profile, supporting Specimen / Patient / Practitioner
  / Organization profiles, etc.) and rewrite the narrative so each
  profile mentioned in the section actually exists in the IG with the
  name used.
- Re-render the "navigation" diagram(s) (UML / class-style images
  inherited from the EU Laboratory Report guide) so the boxes,
  cardinalities, and reference arrows match the current Laboratory
  Report IG profile graph rather than the EU guide's.
- Replace any EU-specific terminology, examples, or links that survived
  the import with international-IG equivalents.

#### Justification

A "navigating the profiles" / profile-map section is a common and
useful onboarding aid for IG readers, especially in an IG with as many
inter-related profiles as a Laboratory Report. Keeping the section but
correcting it preserves that onboarding value and respects the
reporter's underlying concern (the page exists, it was inherited from
the EU guide, and it is currently not maintained in lock-step with the
international IG). This is consistent with how other HL7 IGs (e.g., US
Core, IPS) present a profile overview page.

---

### Disposition B: Alternative Approach

#### Proposal

Replace the inherited "Navigating the profiles" section with a
lightweight, auto-generated profile overview rather than a hand-curated
narrative + hand-drawn diagrams. Specifically:

- Remove the prose section and EU-derived diagrams.
- Add a short "Profiles in this IG" page that lists each profile with
  its purpose in one or two sentences, links to the StructureDefinition
  page, and shows a small reference graph generated from the IG
  publisher's own dependency output (or a Mermaid diagram checked in
  alongside the page) so it cannot drift from the underlying
  StructureDefinitions.
- Cross-link this page from the IG home and from the
  DiagnosticReport-lab and Observation-lab profile pages.

#### Justification

The reporter's concern is essentially a maintenance one: the diagrams
were imported from another guide and have drifted. Replacing the
hand-maintained artifacts with generated / mechanically-checkable
artifacts addresses the root cause (drift) rather than the symptom
(out-of-date diagrams) and reduces ongoing maintenance burden on the
work group. It still gives readers a navigation aid, which is the value
the original section was intended to provide.

---

### Disposition C: Decline (as recorded — Persuasive with Modification: remove)

#### Proposal

Adopt the resolution already recorded on the ticket: **remove the
"Navigating the profiles" section and the included diagrams entirely**
from the Laboratory Report (FHIR) IG, rather than refreshing or
replacing them. Concrete actions:

- Delete the "Navigating the profiles" page (or section, depending on
  how it is structured in the IG sources) from the IG's narrative
  pages.
- Delete the inherited diagram image assets that were used only by that
  section.
- Remove any TOC / menu / cross-link entries that point at the removed
  section so the published IG has no broken links.

#### Justification

This is the disposition that the ticket has actually been resolved
with ("Persuasive with Modification" + resolution text "Remove the
Navigating the profiles section and included diagrams"). The
justification recorded by the work group is implicitly that the section
is not pulling its weight: it was inherited from the EU guide, has
drifted, and the cost of keeping it accurate (and its diagrams in
sync with the profile set) outweighs its onboarding value, especially
given that each profile already has its own page in the IG. Removing
it eliminates a known source of incorrect documentation rather than
committing the work group to ongoing diagram maintenance.

---

### Recommendation

**Recommended disposition:** C

The ticket has already been triaged and resolved as "Persuasive with
Modification" with the explicit resolution **"Remove the Navigating the
profiles section and included diagrams"**, and it is included in
**block-vote-4**. The recommendation here aligns with that recorded
disposition: removing the section is the lowest-risk, lowest-cost
outcome, it directly addresses the reporter's concern that the
inherited content is no longer reliable, and it does not commit the
work group to maintaining hand-drawn UML-style diagrams against an
evolving profile graph. Disposition A (refresh in place) and
Disposition B (replace with a generated overview) are both defensible
if the work group later decides that a profile-navigation aid is worth
re-introducing, but neither should override the already-agreed
removal in this ticket; either could be filed as a follow-up change
request once the section is gone.
