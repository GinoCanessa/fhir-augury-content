## Summary

Gay Dolin requests that the Laboratory Report (FHIR) IG add a "Plain
Language Summary about HL7 and this Guide" section to its Home Page, modeled
on the example in the US Multiple Chronic Conditions (MCC) IG
(https://hl7.org/fhir/us/mcc/#plain-language-summary-about-hl7-and-this-guide).
The intent is to mirror an emerging convention in peer-reviewed publishing
where plain-language summaries (PLS) make technical artifacts accessible to
non-professional audiences such as patients and the general public, citing
BioMed Central, Science Editor, and PLOS as precedent. The ticket has
already been resolved Persuasive with two modifications: (1) coordinate
with template developers so the "About HL7" paragraphs become part of the
standard IG publication chrome (header or footer), and (2) always include
an "About the Guide" block — using the visible header "About the Guide"
rather than the phrase "Plain Language Summary" — at the start of the Home
Page. No cross-referenced Jira tickets, Zulip threads, or GitHub items
were surfaced for this ticket, so the context is limited to the ticket
itself and the prior committee comment recommending broader review.

## Details

**Description:**

Please add a "Plain Language Summary about HL7 and this Guide" to the home
page.

See https://hl7.org/fhir/us/mcc/#plain-language-summary-about-hl7-and-this-guide

HL7 Would like to see IGs voluntarily add a "Plain Language Summary about
HL7 and this Guide" to the home page.

Many peer-reviewed journals now require plain language summaries (PLS) to
make technical artifacts accessible to non-professional audiences, such as
patients and the general public. PLS simplify complex artifacts, improving
understanding and engagement. Some journals mandate them, while others
provide optional guidelines to authors, with publishers like SAGE and
Taylor & Francis offering templates. This trend is growing in response to
demands for transparency and public involvement, especially in fields like
healthcare (BioMed Central:
https://researchinvolvement.biomedcentral.com/articles/10.1186/s40900-022-00358-6;
Science Editor:
https://www.csescienceeditor.org/article/give-it-to-me-straight-plain-language-summaries-and-their-role-in-scholarly-journals/;
PLOS:
https://journals.plos.org/plosone/article?id=10.1371/journal.pone.0268789).

**Resolution:** Persuasive

Motion to find persuasive with the following modifications:

- Review with the template developers to have the About HL7 paragraphs
  part of the standard IG publications process in either the footer or
  header.
- Always include the About the Guide as visible text with the header
  "About the Guide" instead of "Plain Language" etc., at the start of the
  Home Page.

**Comments:**

- **hbuitendijk (2024-10-18):** This should be reviewed with both OO Main
  and the template developers (FMG? FHIR-I? Other?).

## Proposed Dispositions

Note: because there are no GitHub cross-references, the dispositions name
the conventional authoring location for the Laboratory Report IG home page
(typically the IG's `input/pagecontent/index.md` or equivalent) without
asserting a specific repo path. The committee should confirm the actual
repository when assigning the change.

### Disposition A: Accept As Requested

#### Proposal

Add a new section titled "Plain Language Summary about HL7 and this Guide"
near the top of the Laboratory Report IG home page (`index.md`), copying
the structure used by the US MCC IG: a short paragraph describing HL7 as
the standards organization, a short paragraph describing what this IG is
and who it is for in lay terms, and a pointer to where readers can learn
more. Keep the section as IG-local narrative content authored by Orders &
Observations; do not gate the change on template-developer coordination.

#### Justification

This is the literal request from the reporter and matches an existing,
implementable example (US MCC). It can be delivered entirely within the
Laboratory Report IG's authoring content with no dependency on the IG
Publisher templates or on FMG/FHIR-I, so it is the lowest-risk path to
delivering the patient-and-public-facing narrative the ticket calls for.
It also aligns with the broader publishing trend the reporter cites
(BioMed Central, PLOS, Science Editor) of providing plain-language
summaries alongside technical artifacts.

---

### Disposition B: Alternative Approach

#### Proposal

Adopt the resolution already recorded on the ticket: (1) work with the IG
template developers (FMG / FHIR-I / IG Publisher template owners) to make
an "About HL7" block part of the standard IG publication chrome (header or
footer) so every IG inherits it automatically, and (2) in the Laboratory
Report IG itself, add a visible block at the top of the Home Page titled
"About the Guide" (not "Plain Language Summary") containing a short,
non-technical description of the IG's purpose, audience, and scope.
Coordinate the template change with FMG/FHIR-I per hbuitendijk's comment
before merging the IG-local change, but do not block the IG-local "About
the Guide" block on the template work landing.

#### Justification

This mirrors the disposition already captured on the ticket and addresses
two concerns that pure acceptance does not. First, putting the "About
HL7" boilerplate into the publication template avoids every IG
maintaining its own copy and drifting over time — a known maintenance
hazard for boilerplate. Second, using the header "About the Guide"
instead of "Plain Language Summary" is more honest about the content
(IGs are still technical documents), reduces the risk of readers
expecting a fully lay-language rewrite of the spec, and keeps the
section title consistent with how most readers already think about an
IG's introductory matter. It also explicitly threads the FMG / FHIR-I
review hbuitendijk asked for.

---

### Disposition C: Decline

#### Proposal

Decline the request as out of scope for the Laboratory Report IG on the
grounds that HL7-wide boilerplate ("About HL7") and cross-IG home-page
conventions belong to the IG Publisher template / FMG governance, not to
an individual O&O publication. Recommend the reporter reroute the request
to FMG / FHIR-I to be handled once for all IGs.

#### Justification

A defensible "decline" rests on scope: the substance of the requested
text is generic HL7 organizational boilerplate and a generic IG-purpose
statement, neither of which is Laboratory-Report-specific. Solving it
inside one IG creates N copies of the same content across the
ecosystem, which is exactly the maintenance problem the IG Publisher
templates were designed to avoid. However, this disposition leaves the
patient-and-public-facing accessibility need unmet in the near term and
does not honour the existing Persuasive resolution, so it is the
weakest of the three.

---

### Recommendation

**Recommended disposition:** B

Disposition B is recommended because it is the option already recorded
as the ticket's resolution and because it addresses both halves of the
underlying need without forcing a single-IG-only fix. The IG-local
"About the Guide" block can ship now under O&O's control, giving readers
of the Laboratory Report IG immediate access to a plain-language framing
of the guide (satisfying the reporter's intent), while the parallel
template-developer conversation that hbuitendijk asked for keeps the
broader "About HL7" boilerplate from being duplicated across every IG.
Disposition A would deliver the visible content but bake in a
maintenance liability and ignore the committee comment; Disposition C
would honour the architectural concern but leave the user-facing need
unaddressed and contradict the existing Persuasive vote. B threads
both.
