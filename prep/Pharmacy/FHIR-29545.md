## Summary

The ticket asks the PDMP / US Medication IG authoring team to add
**US Core PractitionerRole Profile** to the list of allowed reference
targets on an `actor` element that today permits only US Core
Practitioner, US Core Patient, and RelatedPerson. The reporter argues
that US Core Practitioner alone carries no information about the
practitioner's organization, so PractitionerRole is the preferred
profile when the consuming system needs the practitioner's affiliation
and role context (which is a common need in PDMP query/response
scenarios where the requester's role and organization matter for
audit, attribution, and authorization).

The change is purely additive: the proposed wording widens the
`Reference(...)` choice to
`Reference(US Core PractitionerRole Profile | US Core Practitioner
Profile | US Core Patient Profile | RelatedPerson)` and notes that
PractitionerRole is preferred over plain Practitioner.

Context from related Pharmacy WG tickets (no formal cross-references
exist, but keyword similarity surfaces near-duplicates):

- [FHIR-29546](https://jira.hl7.org/browse/FHIR-29546) is a sibling
  ticket from the same reporter against the same IG that asks for the
  identical change on a different element (an element whose existing
  reference target is `US Core Practitioner Profile | US Core
  Organization Profile`). FHIR-29545 and FHIR-29546 should almost
  certainly be dispositioned together.
- [FHIR-26524](https://jira.hl7.org/browse/FHIR-26524) is a closely
  related precedent in another Pharmacy IG (CARIN RTPBC) where the
  workgroup accepted ("Persuasive with Modification") the argument
  that custom Practitioner profiles add little over US Core and that
  US Core Practitioner / PractitionerRole should be referenced
  directly.

The ticket carries the `PDMP-old-issues` label. Per the November-2023
comments from `smrobertson`, the ticket was previously closed
"Considered for Future Use" because the PDMP IG had been withdrawn,
then reopened when the IG was picked back up under PSS-2225. It is
currently in `Triaged` status awaiting a Pharmacy WG vote.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Apply the wording change exactly as the reporter proposed: widen the
`actor` element's allowed reference targets from
`Reference(US Core Practitioner Profile | US Core Patient Profile |
RelatedPerson)` to
`Reference(US Core PractitionerRole Profile | US Core Practitioner
Profile | US Core Patient Profile | RelatedPerson)` on the affected
PDMP / US Medication IG profile, and add narrative to the profile (and
to the IG's general "Actors" / "Provenance" / equivalent guidance
section) stating that **US Core PractitionerRole is preferred over US
Core Practitioner** when both the practitioner identity *and* their
organization/role are required, which is the typical case for PDMP
data exchange. Disposition the sibling ticket
[FHIR-29546](https://jira.hl7.org/browse/FHIR-29546) the same way for
consistency across the IG.

#### Justification

This is a purely additive change to a `Reference(...)` choice, which
is backward-compatible: any existing instance that points at US Core
Practitioner, US Core Patient, or RelatedPerson remains valid. The
request is well-aligned with US Core's own modeling, which exposes
practitioner-organization affiliation through PractitionerRole
(`PractitionerRole.organization`, `PractitionerRole.code`,
`PractitionerRole.specialty`) — information the bare US Core
Practitioner profile does not carry. The Pharmacy WG has previously
accepted the same line of reasoning on
[FHIR-26524](https://jira.hl7.org/browse/FHIR-26524) ("Persuasive with
Modification") in the CARIN RTPBC IG, so accepting here is consistent
WG policy. PDMP query responses are explicitly expected to include
`Organization` resources alongside `Practitioner` (see the API-example
narrative discussed in
[FHIR-29526](https://jira.hl7.org/browse/FHIR-29526)), so allowing
PractitionerRole as the actor reference closes the modeling gap
between "who acted" and "for which organization."

---

### Disposition B: Alternative Approach

#### Proposal

Accept the underlying need but resolve it slightly differently:

1. Add **US Core PractitionerRole Profile** to the `Reference(...)`
   choice as proposed (additive).
2. **Reorder** the choice so PractitionerRole is listed first, making
   the "preferred" status visible in the Logical View / StructureDefinition
   itself rather than relying solely on narrative.
3. Add a `MustSupport` clarification (or `obligation`, if the IG is on
   a build that supports them) on the `actor` element stating that
   conformant servers SHALL support PractitionerRole as the target
   when the actor is a clinician; clients MAY also receive
   Practitioner-only references for legacy systems.
4. Apply the same shape to [FHIR-29546](https://jira.hl7.org/browse/FHIR-29546).

Optionally, add an `invariant` (warning, not error) reminding authors
that when the actor is a healthcare provider acting on behalf of an
organization, PractitionerRole is the expected target.

#### Justification

Disposition A's narrative-only "preferred" guidance is easy to miss
during implementation; encoding the preference in element ordering
and `MustSupport`/obligations gives validators and IG-publication
tooling something concrete to surface. This still preserves backward
compatibility (existing references remain valid) but raises the
probability that new implementations will actually use PractitionerRole.
The cost is modest — a handful of extra IG authoring lines — and
mirrors how US Core itself signals preference among reference choices.
The trade-off vs. A is added authoring complexity now in exchange for
clearer downstream behavior.

---

### Disposition C: Decline

#### Proposal

Decline the change. Rationale: the existing `Reference(...)` choice
on `actor` already permits US Core Practitioner, and implementations
that need organizational context can `_include` or chain to the
practitioner's `PractitionerRole` resources separately. Mark the
ticket "Not Persuasive" and close. Apply the same disposition to
[FHIR-29546](https://jira.hl7.org/browse/FHIR-29546).

#### Justification

A decline would be defensible only on the narrow ground that the
PDMP / US Medication IG is currently low-activity (the ticket sat
under "Considered for Future Use" while the IG was withdrawn, per
smrobertson's 2023-11-28 comment) and that no implementer has been
blocked to date. The existing model is *technically* sufficient
because PractitionerRole information can be retrieved separately via
search/include. However, this is a weak rationale: the request is
additive, low-cost, and explicitly aligned with US Core's intended
use of PractitionerRole, so the bar for declining is high.

---

### Recommendation

**Recommended disposition:** A (with a friendly nod to B if the WG
wants the extra rigor).

This is a textbook additive, backward-compatible widening of a
`Reference(...)` choice. The reporter's reasoning — that US Core
Practitioner alone cannot carry the practitioner-organization linkage
that PDMP exchanges actually need — is correct, and PDMP response
bundles already include `Organization` resources alongside
`Practitioner` (see [FHIR-29526](https://jira.hl7.org/browse/FHIR-29526)),
so PractitionerRole is the natural binding between the two. The
Pharmacy WG has already accepted the same argument on
[FHIR-26524](https://jira.hl7.org/browse/FHIR-26524) in a sibling IG,
which makes accepting here a matter of consistency. The change should
be dispositioned together with [FHIR-29546](https://jira.hl7.org/browse/FHIR-29546)
so the IG's actor-modeling stays uniform across profiles. Disposition
B is a reasonable upgrade path if the WG wants the "preferred" status
to be machine-readable rather than narrative-only; declining
(Disposition C) is not recommended given the change's low cost and
the existing WG precedent.
