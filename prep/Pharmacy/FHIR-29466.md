## Summary

The reporter contends that the NCPDP SCRIPT 2017071 mapping referenced
in the US Prescription Drug Monitoring Program (PDMP) FHIR
Implementation Guide — currently labelled "NCPDP Data Elements for
Request Prescription Date Range" — points at the wrong NCPDP SCRIPT
elements. According to the ticket, the correct source elements are
`<RequestedDates>/<StartDate>` and `<RequestedDates>/<EndDate>`, and
the IG text / mapping table should be updated to reflect that.

The ticket originated against the PDMP FHIR IG, was originally
dispositioned in January 2023 as **Considered for Future Use** (vote
4-0-0, John Hatem / Tim McNeil) on the basis that the IG had been
withdrawn, and was then **reopened on 2023-11-28** by smrobertson under
the renewed [PSS-2225 PDMP on FHIR](https://confluence.hl7.org/display/PHAR/US+Prescription+Drug+Monitoring+Program+%28PDMP%29+FHIR+Implementation+Guide)
project. It is now tracked under the broader "US Medication FHIR
Implementation Guide (FHIR)" specification banner. No cross-referenced
Jira tickets, Zulip threads, or GitHub items were found in Augury for
this key, so the only recorded context is the ticket body and its three
comments.

## Proposed Dispositions

The dispositions below reflect that this ticket targets a withdrawn-
then-revived IG (PDMP) and is essentially a mapping-correction request.
No GitHub repo briefings are available because Augury surfaced no
GitHub cross-references; any concrete file paths would have to be
identified at implementation time by the workgroup against whichever
repo currently hosts the revived IG under PSS-2225.

### Disposition A: Accept As Requested

#### Proposal

Persuasive — Update the NCPDP SCRIPT 2017071 mapping in the
PDMP / US Medication FHIR IG so that the source-side identifier for the
"Request Prescription Date Range" data elements is corrected to
`<RequestedDates>/<StartDate>` and `<RequestedDates>/<EndDate>` (rather
than the current generic phrase "NCPDP Data Elements for Request
Prescription Date Range"). Concretely:

- In the IG mapping table / narrative that describes how the
  prescription-history date-range request maps to FHIR (typically a
  `Period` carried on the request resource — e.g. a
  `MedicationRequest` / Task / parameter `period` with `start` and
  `end`), replace the existing source label with the two NCPDP element
  paths:
  - `RequestedDates/StartDate` → `Period.start`
  - `RequestedDates/EndDate` → `Period.end`
- Add a citation of the NCPDP SCRIPT 2017071 segment that defines
  `RequestedDates` so reviewers can verify the mapping.
- Apply the change wherever the same "Request Prescription Date Range"
  wording appears (mapping table, any prose referencing it, and any
  StructureDefinition `mapping` element that targets the NCPDP SCRIPT
  identity).

#### Justification

The reporter is citing a specific, verifiable NCPDP SCRIPT 2017071
construct (`RequestedDates/StartDate|EndDate`), and the existing IG
text is a generic phrase rather than an actual element path — i.e. the
mapping is currently ambiguous on its face. Mapping tables in FHIR IGs
are explicitly intended to name the source-standard element so an
implementer can round-trip; a one-time textual correction satisfies that
goal at minimal cost and aligns the IG with how every other
NCPDP-mapped element in the PDMP/US Medication IG is described. Because
the IG has now been re-chartered under PSS-2225, deferring the fix
again would simply re-create the same defect in the next ballot.

---

### Disposition B: Alternative Approach

#### Proposal

Persuasive with Modification — Accept the substance of the request
(point the mapping at `RequestedDates/StartDate` and
`RequestedDates/EndDate`) but, rather than only correcting the prose
label, restructure the affected mapping entry into the standard FHIR
`StructureDefinition.mapping` form keyed to a declared NCPDP SCRIPT
identity (e.g. `identity: "ncpdp-script-2017071"` with a `uri` and
`name`). Each FHIR target element (`Period.start`, `Period.end`) would
then carry a `mapping` entry whose `map` is the literal NCPDP path
(`RequestedDates/StartDate`, `RequestedDates/EndDate`). At the same
time, sweep the IG for other NCPDP SCRIPT mappings written in the
older free-text style and convert them to the same structured form.

#### Justification

The reporter's defect is real, but fixing only this one phrase leaves
the IG with mixed mapping styles (free text vs. structured `mapping`
elements) and no machine-discoverable link from FHIR elements to NCPDP
SCRIPT 2017071. A structured mapping makes the correction durable,
gives implementers and tooling (e.g. FHIR Mapping Language, IG
publisher mapping pages) a single canonical source-of-truth, and
reduces the chance of the same class of typo recurring elsewhere in the
IG. The trade-off vs. Disposition A is a larger change set and a
broader review surface, but the work is mechanical and a natural fit
for the IG's revival under PSS-2225.

---

### Disposition C: Decline

#### Proposal

Not Persuasive — Close the ticket without change on the basis that the
underlying PDMP FHIR IG was withdrawn, the current "US Medication FHIR
Implementation Guide" content does not contain a normative NCPDP SCRIPT
2017071 "Request Prescription Date Range" mapping in the same form
described in the ticket, and the requested correction would be to text
that no longer ships. Re-file as a fresh ticket against the active IG
once PSS-2225 produces a draft that actually contains the affected
mapping table.

#### Justification

This is defensible only if the workgroup confirms that the specific
mapping prose called out in the ticket is no longer present in the
revived IG. The original 2023-01-18 disposition (*Considered for Future
Use*, 4-0-0) used essentially this rationale — the IG had been
withdrawn — and was reverted in November 2023 precisely because the
project was being restarted. Declining therefore depends on a
content-level check that the offending wording did not survive the
revival; if it did survive, declining would knowingly preserve a
documented error and a `PDMP-old-issues`-labelled defect into the next
ballot.

---

### Recommendation

**Recommended disposition:** A

The ticket describes a narrowly-scoped, well-evidenced mapping
correction against an IG whose project (PSS-2225) is active again. The
reporter has named the exact NCPDP SCRIPT 2017071 path
(`RequestedDates/StartDate` and `RequestedDates/EndDate`), and the
existing wording ("NCPDP Data Elements for Request Prescription Date
Range") is an obvious placeholder rather than a real element
identifier — there is no plausible argument for keeping it. Disposition
B's structured-mapping refactor is attractive but expands a small
defect fix into a cross-cutting cleanup that is better tracked as its
own work item against the revived IG; bundling it here risks blocking
the obvious fix on a wider editorial debate. Disposition C is hard to
justify now that the project is no longer withdrawn and the ticket is
explicitly tagged `PDMP-old-issues` for triage as part of the revival.
Accept as requested, and — if the workgroup agrees the structured-
mapping cleanup has value — open a separate ticket to capture
Disposition B.
