## Summary

The ticket requests that all instances of the word "abuse" in the US
Prescription Drug Monitoring Program (PDMP) FHIR Implementation Guide —
tracked in Jira under the specification "US Medication FHIR Implementation
Guide (FHIR)" — be replaced with the word "misuse". The rationale given by
the reporter (Molly Reese) is that "abuse" is a stigmatizing term, and
"misuse" is the preferred, less-stigmatizing alternative used in current
public-health and behavioral-health discourse around opioids and controlled
substances.

The ticket was originally imported from a spreadsheet of PDMP IG issues in
October 2020. It was resolved as "Considered for Future Use" on 2023-01-18
(John Hatem / Tim McNeil, 4-0-0) because the PDMP IG had been withdrawn at
that time. It was subsequently reopened on 2023-11-28 by smrobertson in
connection with PSS-2225 ("US Prescription Drug Monitoring Program (PDMP)
FHIR Implementation Guide"), which signals that the project sponsor and/or
workgroup is preparing to take the IG back up. The ticket therefore needs a
fresh disposition now that the IG is active again.

No Jira, Zulip, or GitHub cross-references were returned by FhirAugury for
this ticket, so the summary above is based on the ticket body and comments
alone.

## Proposed Dispositions

Because FhirAugury returned no GitHub cross-references for this ticket, no
repository briefing drives the dispositions below. The specification in
scope is the US Medication FHIR Implementation Guide / PDMP IG, and any
accepted change would be made in the authoring source of that IG once the
workgroup has confirmed which repository currently hosts it (the prior
PDMP IG publication was withdrawn, and per PSS-2225 the project is being
resumed).

### Disposition A: Accept As Requested

#### Proposal

Perform a case-insensitive find-and-replace of the word "abuse" with
"misuse" across all authored narrative in the US Medication / PDMP FHIR
IG, specifically:

- IG narrative / page content (intro pages, usage guidance, security and
  privacy discussion, examples commentary).
- `display` strings on local codes and local value-set expansions authored
  by this IG where the display text uses "abuse" in a person-referential
  sense (e.g., "drug abuse", "substance abuse", "abuser").
- Example resource narrative, example `text.div` HTML, and any example
  `Condition.code.text` / `Observation.code.text` strings that use
  "abuse" editorially.
- IG-authored glossary, FAQ, and change-log entries.

Do **not** alter:

- External code systems' official display names (e.g., SNOMED CT, ICD-10,
  RxNorm, NDC, LOINC) — those displays are owned by the code system and
  cannot be unilaterally rewritten by this IG.
- Literal codes or `code` values anywhere.
- Quoted regulatory or statutory language (e.g., DEA, state PDMP law)
  that uses the word "abuse" as a term of art, where changing the quote
  would misrepresent the source.

Add a short note in the IG's terminology/guidance section stating that
the IG uses "misuse" in its own narrative as non-stigmatizing language,
while preserving the original wording of external code system displays
and cited statutory text.

#### Justification

"Misuse" is the term now favored by SAMHSA, CDC, and mainstream
behavioral-health style guides over "abuse" when describing patterns of
non-medical use of controlled substances; major US public-health bodies
have moved deliberately away from "abuse" because of its stigmatizing
effect on patients. Adopting person-first, non-stigmatizing language in
HL7 IG narrative is consistent with FHIR's broader editorial direction
and with current practice in other Pharmacy / behavioral-health IGs. The
carve-outs above respect the integrity of externally governed
terminologies and of cited law.

---

### Disposition B: Alternative Approach

#### Proposal

Replace the wording contextually rather than mechanically. Specifically:

1. In IG-authored narrative, replace "abuse" with the most clinically
   accurate alternative in each context — typically "misuse" for
   non-medical / non-prescribed use, but "use disorder" (e.g., "opioid
   use disorder") where the original text was actually describing the
   DSM-5 diagnostic concept, and "diversion" where the original text was
   describing illicit transfer of prescribed drugs to a third party.
2. For local value-set displays or local code displays that include
   "abuse", update the displays to the contextually appropriate
   replacement and add `designation` entries (or a deprecation note) so
   implementers can still map legacy "abuse"-based displays.
3. Leave external code system displays unchanged, same as Disposition A.
4. Add a single IG-level "Language and Terminology" page documenting the
   editorial policy and listing the specific substitutions made, so
   reviewers and implementers can audit the change.

#### Justification

A blanket "abuse → misuse" substitution is linguistically lossy: in
current clinical usage, "misuse", "use disorder", and "diversion" are
distinct concepts, and conflating them can actually degrade the
precision of the IG's guidance (e.g., PDMP content legitimately needs
to talk about diversion as a separate concern from misuse). A
context-aware pass yields better IG quality at modest additional
editorial cost, and the accompanying terminology page gives implementers
a discoverable record of what changed — which is important because the
PDMP IG is being resumed after being withdrawn (per PSS-2225) and
implementers will be cross-referencing against the prior published
version.

---

### Disposition C: Decline

#### Proposal

Decline the request as stated and close the ticket as "Not Persuasive"
(or "Considered for Future Use" if the workgroup wants to re-raise it
once the resumed PDMP IG has stable content to edit). Rationale: a
bulk find-and-replace of "abuse" with "misuse" in an IG whose publishing
status is in flux risks churning text that may itself be rewritten as
part of the PDMP IG's resumption under PSS-2225, and the two words are
not semantic equivalents in all clinical contexts (see Disposition B's
proposal for counter-examples involving "use disorder" and
"diversion").

#### Justification

The earlier disposition of this exact ticket (2023-01-18, 4-0-0)
deferred the change specifically because the IG had been withdrawn; the
reopening comment (2023-11-28) notes only that the IG is being taken up
again, not that the editorial stance changed. Declining now, and asking
the PDMP-IG authors to apply person-first language as part of their
editorial pass on the resumed IG, avoids imposing a mechanical edit on
text that is likely to be rewritten anyway and avoids the semantic
hazards of a blind substitution. It also keeps the edit under the eye
of the authors who best understand which instances of "abuse" should
become "misuse", which should become "use disorder", and which are
quoted external material that must not change.

---

### Recommendation

**Recommended disposition:** B

Disposition B best serves the underlying intent of the ticket — moving
the IG to non-stigmatizing language — while also protecting the clinical
precision of the PDMP content, which explicitly deals with related-but-
distinct concepts (misuse, use disorder, diversion) that a raw
`abuse → misuse` substitution would flatten. It is also the disposition
most respectful of the ticket's history: the request was reopened
specifically because the PDMP IG is being resumed under PSS-2225, so
there is a live editorial pass in which a context-aware substitution
can be folded in at low marginal cost. Disposition A is acceptable as
a fallback if the workgroup prefers to minimize editorial judgment
calls per-instance, and Disposition C should only be adopted if the
resumed PDMP IG work is not yet ready to absorb any narrative edits.
