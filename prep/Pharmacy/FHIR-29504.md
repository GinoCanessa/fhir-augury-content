## Summary

FHIR-29504 raises a regulatory/scope objection against the (then-current)
US Medication FHIR Implementation Guide. The reporter (Molly Reese)
points out that **NCPDP SCRIPT is the mandated standard for outpatient
e-prescribing in the United States**, and flags the IG's "Use Case 3 —
Create new outpatient Prescription" as the section in conflict with that
mandate. The implication is that the IG should not define a FHIR-native
mechanism for creating outpatient prescriptions that competes with, or
substitutes for, NCPDP SCRIPT.

The ticket has had a complicated lifecycle. It was originally imported
from a spreadsheet (comment #54) on 2020-10-26. On 2023-01-18 it was
voted **Considered for Future Use** (John Hatem / Tim McNeil, 4-0-0)
because the IG had been withdrawn. On 2023-11-28 that resolution was
reverted by smrobertson and the ticket was **reopened** under the
PSS-2225 PDMP on FHIR project. On 2023-11-29 it was resolved as **Not
Persuasive** with the rationale: *"creating a prescription is not in
the revised IG's scope."* The current state is `Triaged`, `Resolution =
Not Persuasive`, awaiting workgroup confirmation/closeout.

There are **no cross-referenced Jira tickets, Zulip threads, or GitHub
items** indexed against this ticket in FHIR Augury, so the review must
be made on the ticket text alone plus the Pharmacy WG's standing
position on NCPDP SCRIPT vs. FHIR for outpatient e-prescribing.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Affirm the reporter's point in the IG narrative: explicitly remove (or
re-scope) **Use Case 3 — Create new outpatient Prescription** from the
US Medication FHIR Implementation Guide and add a normative statement
in the IG's *Scope*, *Use Cases*, and (where applicable) the
`MedicationRequest` profile pages stating that **outpatient
e-prescribing in the US is conducted via NCPDP SCRIPT and is out of
scope for this IG**. Provide a non-normative pointer to NCPDP SCRIPT
and to any HL7/NCPDP harmonization material so implementers know where
to go. Reflect this in the IG's `MedicationRequest` profile by either
removing the profile (if it existed solely to support Use Case 3) or
narrowing its `intent` / `status` constraints to communication-of-an-
existing-prescription scenarios (e.g., dispense, administration,
medication history, PDMP query/report) rather than order origination.

#### Justification

This matches the federal regulatory reality: CMS / DEA EPCS and Part D
rules effectively require NCPDP SCRIPT for outpatient e-prescribing,
and a parallel FHIR-native "create prescription" workflow inside a US
Medication IG would create an unsanctioned alternative path that no
implementer could lawfully use as a substitute. It also aligns with
the most recent comment-of-record from the workgroup
(*"creating a prescription is not in the revised IG's scope"*), turning
the resolution rationale into explicit, durable IG text rather than a
ticket-level statement.

---

### Disposition B: Alternative Approach

#### Proposal

Keep `MedicationRequest` in the IG but **re-frame Use Case 3** away
from "Create new outpatient Prescription" toward
**"Represent / exchange an existing outpatient prescription in FHIR for
downstream uses"** — e.g., medication reconciliation, PDMP submission
and query, prior authorization, patient access (e.g., US Core /
International Patient Access alignment), and clinical decision support.
Add an explicit IG-level constraint (and narrative) stating that
`MedicationRequest` instances within the scope of this IG are
**representations of prescriptions originated via NCPDP SCRIPT** (or
inpatient orders), not the wire-format used to transmit a new
prescription to a pharmacy. Cross-link to NCPDP SCRIPT in the
narrative and, where useful, document a mapping table between
`MedicationRequest` elements and the equivalent NCPDP SCRIPT fields so
systems can round-trip the data without competing with SCRIPT as the
transmission standard.

#### Justification

Disposition A solves the regulatory problem but throws away
legitimately useful FHIR representation work. There are real downstream
US use cases — medication reconciliation, PDMP, prior auth, patient
access — where a FHIR `MedicationRequest` representing an outpatient
prescription is exactly what implementers need, and where NCPDP SCRIPT
is not the right tool. Reframing Use Case 3 preserves that value while
unambiguously deferring the *origination* path to NCPDP SCRIPT, which
addresses the reporter's underlying concern without removing
representation capability.

---

### Disposition C: Decline

#### Proposal

Confirm the existing **Not Persuasive** resolution. Close the ticket
without further IG changes on the grounds that the revised US
Medication FHIR IG no longer contains a "create new outpatient
prescription" use case (per the 2023-11-29 resolution comment), so the
specific defect the ticket reports — IG text proposing a FHIR-native
e-prescribing path that conflicts with the NCPDP SCRIPT mandate — is
no longer present in the spec. Note the disposition in the workgroup
minutes and in any successor PDMP IG so the concern is on the record
when prescribing scope is revisited.

#### Justification

The ticket is essentially asking the workgroup not to do something the
workgroup has already decided not to do. The current IG scope per the
resolution comment excludes prescription creation, so there is no
extant text to amend, and adopting Dispositions A or B would re-open
scope discussion on a topic that has effectively been settled. The
`PDMP-old-issues` label and the import-from-spreadsheet provenance
also indicate this is a legacy item carried forward from a withdrawn
IG; closing it as Not Persuasive keeps the backlog clean.

---

### Recommendation

**Recommended disposition:** B

The reporter is raising a genuine US regulatory issue that will recur
every time this IG (or its successor) re-enters balloting — simply
declining (Disposition C) leaves the IG vulnerable to the same comment
on every cycle and depends on readers inferring scope from a Jira
resolution comment rather than from the IG itself. Disposition A fully
removes the conflict but discards real, non-prescribing US use cases
where a FHIR `MedicationRequest` is the right artifact (PDMP, med
reconciliation, prior auth, patient access). Disposition B captures the
benefit of A — explicit, durable IG text that defers prescription
*origination* to NCPDP SCRIPT — while preserving the IG's value for the
legitimate downstream use cases that are very much in the Pharmacy WG's
remit and that align with adjacent HL7 work (US Core, Da Vinci PAS,
PDMP). Pharmacy WG should adopt B, restate the scope explicitly in the
IG narrative, and reference NCPDP SCRIPT as the authoritative
e-prescribing transmission standard.
