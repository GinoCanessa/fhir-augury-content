## Summary

FHIR-29494 is an editorial Change Request against the US Medication FHIR
Implementation Guide (PDMP / "US Meds" IG, owned by the Pharmacy work
group). The reporter (Josh Mandel) observed that the narrative pages of
the IG mix two different fictional patients in their inline examples —
one example uses "Sherlock Holmes" and another uses "Peter Jacobs" — and
asks that, within a single page, examples be unified around one
patient/person to improve readability.

The ticket sits in a small cluster of feedback Josh Mandel filed against
the same IG draft, all touching the example/search-parameter narrative:

- [FHIR-29487](https://jira.hl7.org/browse/FHIR-29487) — formatting of the
  Sherlock-Holmes search-parameter example (line breaks, indentation,
  stray backticks).
- [FHIR-29493](https://jira.hl7.org/browse/FHIR-29493) — the chained
  `subject:Patient.name.given=sherlock` style itself is criticized as
  fragile (e.g., it would still match a person who was once named
  "Sherlock Smith" and later renamed to "John Holmes").
- [FHIR-29454](https://jira.hl7.org/browse/FHIR-29454) (Daniel Zhang) —
  the surrounding wording confuses dispenses with prescriptions.

These were not formally cross-referenced in Jira but are surfaced by
keyword similarity and they share both the originating page and the
example. They give context that Disposition design should consider: the
example block at issue is already targeted by multiple change requests,
so unifying the example name is best done as part of a single
example/narrative pass rather than a one-line edit.

The ticket's history is also relevant. The PDMP IG was withdrawn while
Pharmacy worked through the original triage:

- 2023-01-18 — original disposition "Considered for Future Use" (John
  Hatem / Tim McNeil, 4-0-0): "This IG has been withdrawn so these items
  will be deferred until the IG is taken up again by the project sponsor
  and/or the workgroup."
- 2023-11-28 — `smrobertson` reverted that resolution and reopened the
  ticket in connection with PSS-2225 (PDMP on FHIR), bringing it back
  into scope.

So the request is alive again pending the IG's resumption, and the
workgroup is being asked to choose what to do with these old editorial
issues now that the project is back on the table.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Accept as a small editorial change against the US Medication FHIR IG
narrative. Pick **one** fictional patient and use that single name
consistently across every example on the affected page (the
PDMP search-parameter / dispense example page that today uses both
"Sherlock Holmes" and "Peter Jacobs"). Concretely:

- Choose "Sherlock Holmes" with birthdate 1954-01-06 as the canonical
  patient (it is already used in the chained-search example and in the
  surrounding narrative referenced by FHIR-29487, FHIR-29493 and
  FHIR-29454).
- Replace every occurrence of "Peter Jacobs" (and any other ad-hoc
  patient name) on that page with the canonical "Sherlock Holmes"
  values.
- If the page renders any example `Patient`/`MedicationDispense`
  instances, regenerate or hand-edit them to share the same `Patient`
  reference / `subject` so that the search example, the dispense
  example, and any narrative all line up on the same person.

Track the change as part of the broader narrative cleanup that
FHIR-29487, FHIR-29493 and FHIR-29454 will already be touching.

#### Justification

The request is purely editorial, low-risk, and consistent with FHIR IG
authoring conventions: most HL7 IGs use a single recurring example
identity within a given narrative page so that cross-references
("the same patient as above…") read naturally. Three other open
tickets (FHIR-29487, FHIR-29493, FHIR-29454) already require revisiting
the same example block, so unifying the patient name comes essentially
for free in that pass and avoids reviewers having to read the same page
twice.

---

### Disposition B: Alternative Approach — Roll into a Single "PDMP Example
Cleanup" Disposition

#### Proposal

Rather than disposition this ticket in isolation, mark FHIR-29494 as a
duplicate-of / will-be-addressed-by a single coordinated editorial pass
on the PDMP search/dispense example block, and resolve FHIR-29487,
FHIR-29493, FHIR-29454 and FHIR-29494 together. Concretely the
coordinated pass would:

1. Choose one canonical example patient (per Disposition A).
2. Reformat the chained-search example with line breaks / indentation
   and remove the stray backticks (FHIR-29487).
3. Replace the chained `subject:Patient.name.*` query with a two-step
   pattern (first locate the `Patient`, then query
   `MedicationDispense?subject=Patient/<id>&…`), per FHIR-29493.
4. Correct the "with a prescription from … to …" wording to "for
   medications dispensed from … to …" per FHIR-29454.

FHIR-29494 itself is then closed as "Persuasive with Mod — addressed by
the coordinated example rewrite."

#### Justification

The four tickets all target literally the same paragraph; dispositioning
them independently risks contradictory edits (e.g., Disposition A
unifies names around "Sherlock Holmes", but FHIR-29493's preferred
fix arguably makes the name irrelevant by switching to a `Patient/<id>`
reference). Doing one pass keeps the IG self-consistent, gives
reviewers a single diff to vote on, and avoids re-balloting the same
page multiple times. This also matches the spirit of the 2023-01-18
"Considered for Future Use" disposition — these items were always meant
to be picked up together when the IG resumed.

---

### Disposition C: Decline / Defer until IG Resumption

#### Proposal

Close FHIR-29494 as **Considered for Future Use** (or **Not Persuasive
— deferred**) on the basis that the US Medication FHIR Implementation
Guide is currently withdrawn. Capture in the resolution note that the
example-unification edit will be picked up by the next PDMP authoring
cycle once PSS-2225 resumes, but do not commit Pharmacy bandwidth to
the editorial change while there is no active publication target.

#### Justification

This is essentially a re-affirmation of the original 2023-01-18
disposition, made before `smrobertson` reverted it: "This IG has been
withdrawn so these items will be deferred until the IG is taken up
again by the project sponsor and/or the workgroup." If PSS-2225 has
not yet produced an active IG repo / publication branch (the
FhirAugury GitHub source returned no PDMP/US-Meds repository, and
there is no PR or issue tracking the change), then any edit made now
has nowhere to land and risks bit-rotting before the next
publication. Declining/deferring is defensible until there is a live
authoring root to commit against.

---

### Recommendation

**Recommended disposition:** B (coordinated editorial pass).

The ticket itself is unambiguous and trivially actionable, but it is
one of at least four open tickets (FHIR-29454, FHIR-29487, FHIR-29493,
FHIR-29494) targeting the same example block on the same page. Treating
them as a single disposition (B) avoids the realistic failure mode
where the workgroup unifies the example on "Sherlock Holmes" today and
then, six months later, decides via FHIR-29493 to drop the chained
name-search example entirely — wasting the first edit and forcing
another ballot pass. Disposition B preserves the spirit of A (the
example *will* be unified) while giving reviewers a single, coherent
narrative diff to vote on. Disposition C is defensible only if Pharmacy
has firm evidence that PSS-2225 will not produce an active IG branch in
the near term; given that `smrobertson` explicitly reopened the ticket
in 2023 to bring it back into scope, the workgroup's revealed
preference is to keep these items live, which argues against deferring
again.
