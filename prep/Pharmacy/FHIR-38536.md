## Summary

This ticket — one of a numbered series imported into the Pharmacy WG's
US Specialty Rx (FHIR) IG ("…4." in the title and "Comment 4 - imported
by: Ron G. Parker" in the body) — asks that the canonical URL for a
Code System defined by the IG be anchored under `terminology.hl7.org`
(THO) rather than under `hl7.org/fhir`, in line with the TSMG Code
System Identifier Task Force best practice and the R5 terminology
guidance. It additionally asks that a corresponding UTG ticket be filed
to register the Code System in THO. The cross-reference inventory
returned no related Jira tickets, no Zulip threads, and no GitHub items;
the only outbound link is to the TSMG Confluence slide deck cited as
the rationale (`confluence.hl7.org/...attachments/81034628/...pptx`).

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Re-anchor the canonical URL of the Code System(s) defined by the
US Specialty Rx (FHIR) IG from `http://hl7.org/fhir/...` (or the
current IG-anchored URL) to a `http://terminology.hl7.org/CodeSystem/...`
identifier, and file a companion UTG ticket to register the Code System
under THO so that the canonical resolves and the system is centrally
managed. Update all internal references in the IG (CodeSystem
resource, dependent ValueSets, narrative, and any `system=` literals
in examples or profiles) so they point to the new THO-anchored URL,
and add a deprecation note documenting the prior URL as superseded.

#### Justification

This is the explicit TSMG / Code System Identifier Task Force best
practice cited by the reporter and aligned with the R5 terminologies
guidance (`terminologies.html#system`). It places HL7-published
terminology under the registry that exists to manage it (THO/UTG), and
keeps the Pharmacy IG consistent with the broader pattern other WGs
have adopted for IG-defined Code Systems. It also addresses the
re-open rationale captured by lynn_laakso: the original "Considered
for Future Use" disposition was deferred pending policy finalization,
which has since matured — applying the policy now is the natural
follow-through.

---

### Disposition B: Alternative Approach

#### Proposal

Treat the request as a re-anchoring obligation but defer the URL
change to the next major (breaking-change) revision of the IG, while
in the interim (a) filing the UTG registration ticket immediately so
the THO entry exists, and (b) publishing the Code System under both
URLs — the current IG-anchored URL retained for backwards
compatibility and a new THO-anchored URL added as the preferred
canonical with `CodeSystem.identifier` carrying the prior URL. Document
the migration in the IG's "Changes" page and signal the existing URL
as deprecated.

#### Justification

The Code System is already in active use by implementers of US
Specialty Rx STU2 (the re-open comment is explicitly post-STU2
publication). A bare URL swap is technically a breaking change for
any client that resolves the canonical or matches `Coding.system`
literally, and the Pharmacy WG has historically been sensitive to
NCPDP / specialty-pharmacy implementer churn. This approach satisfies
the TSMG policy intent, gets the THO registration done immediately,
and gives implementers a defined deprecation window — at the cost of
some short-term duplication.

---

### Disposition C: Decline

#### Proposal

Decline the change on the basis that the original "Considered for
Future Use" disposition (Frank McKinney / John Hatem, 6-0-0,
2023-01-30) remains correct: the request is contingent on a finalized
policy for anchoring IG-defined terminology, and absent a clear,
applied policy the WG should not unilaterally re-anchor a URL that
implementers already depend on. Close as "Persuasive with Mod" or
"Not Persuasive" pending TSMG-issued policy guidance and a
coordinated batch migration of the affected Code Systems.

#### Justification

The risk of a piecemeal re-anchor — one CS at a time, ticket by
ticket — is that the IG ends up with mixed canonicals and implementer
confusion, especially given this is one of an explicit numbered series
("…4."). If the policy framework the original vote was waiting on is
not in fact finalized, declining and waiting is defensible and
preserves implementer stability. This disposition is weakest if TSMG
policy *has* matured in the interim — see the recommendation below.

---

### Recommendation

**Recommended disposition:** B

The TSMG best practice the reporter cites is sound and the Pharmacy
WG should comply with it; Disposition C is hard to defend now that
lynn_laakso has explicitly re-opened the ticket "post STU2
publication" and reverted the deferral. However, a straight Accept
(Disposition A) silently breaks any STU2 implementer that resolves the
canonical or matches on `Coding.system` — exactly the implementer
population the IG is meant to serve. Disposition B threads the needle:
it files the UTG registration immediately so the THO record exists,
re-anchors the canonical to THO as TSMG asks, but keeps the prior URL
addressable through a defined deprecation window and surfaces it via
`CodeSystem.identifier`. The WG should treat this ticket as the
template for the rest of the imported series (the "1.", "2.", "3." …
siblings) so all of the IG's Code Systems migrate together rather than
drifting one URL at a time.
