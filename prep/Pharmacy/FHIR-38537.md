## Summary

The Terminology Steering Management Group (TSMG), via reporter Joan Harper,
is asking the Pharmacy work group to move the canonical URL of a Code
System defined in the **US Specialty Rx (FHIR)** Implementation Guide off
of the `hl7.org/fhir` namespace and anchor it instead under
`terminology.hl7.org` (THO). The ticket also asks that a corresponding UTG
ticket be opened to register the Code System in THO. The request cites
TSMG's January 2022 Code System Identifier Task Force guidance and the R5
terminologies guidance (`http://hl7.org/fhir/5.0.0-snapshot1/terminologies.html#system`)
as the best-practice references. The ticket was originally dispositioned
in early 2023 as "Considered for Future Use" pending finalization of the
IG-defined terminology anchoring policy, and was then re-opened by Lynn
Laakso post-STU2 publication so the work group can revisit it now that
TSMG's guidance has matured. No specific Code System (canonical URL,
file, or `id`) is named in the ticket itself; the work group will need to
identify which Code System(s) in the US Specialty Rx IG this applies to.

## Proposed Dispositions

Note: cross-reference data surfaced no GitHub repository for the US
Specialty Rx (FHIR) IG, so no `repo-analysis` briefing is available to
ground concrete file paths. The dispositions below name the *kind* of
artifact a change would touch (the IG's `input/resources/CodeSystem-*`
authoring files and the corresponding UTG entry); the work group should
confirm the exact repo and file paths during disposition.

### Disposition A: Accept As Requested

#### Proposal

For every `CodeSystem` defined inside the US Specialty Rx (FHIR) IG:

1. Open a UTG (HL7/UTG) ticket to register the Code System in THO under
   a `http://terminology.hl7.org/CodeSystem/<id>` canonical URL,
   following the standard TSMG / Code System Identifier Task Force
   pattern.
2. Once the THO canonical is provisioned, update the IG's CodeSystem
   resource(s):
   - Change `CodeSystem.url` from
     `http://hl7.org/fhir/us/specialty-rx/CodeSystem/<id>` to
     `http://terminology.hl7.org/CodeSystem/<id>`.
   - Update every `ValueSet` that includes the system (`compose.include.system`,
     `expansion.contains.system`) to use the new canonical.
   - Update any `Coding.system` / fixed values in profiles, examples,
     and SearchParameters that reference the old URL.
   - Add an `OtherIdentifier` / aliasing entry where supported so that
     existing implementers using the old URL get a clear pointer.
3. Document the rename in the IG's change log / migration notes for the
   next ballot cycle so implementers can update bindings.
4. Coordinate publication so the UTG entry goes live before, or in
   lockstep with, the next US Specialty Rx publication that uses the
   new canonical.

#### Justification

This is the literal request and is fully aligned with TSMG's January
2022 Code System Identifier Task Force guidance and the R5
terminologies best practice (`terminologies.html#system`). Anchoring
externally-meaningful Code Systems in THO is the standard HL7 pattern:
it lets TSMG manage canonical-URL governance, prevents collisions
across IGs, and decouples the terminology from the IG's publication
cadence. The 2023-05-04 re-open note explicitly says the original
deferral was contingent on the IG-defined-terminology policy being
finalized; the policy is now well-established, so the original
rationale for deferring no longer applies.

---

### Disposition B: Alternative Approach

#### Proposal

Apply the THO migration only to Code Systems in the IG that are
*reusable outside US Specialty Rx* (i.e., concept content that any
implementer or other IG could plausibly bind to). For Code Systems that
are intrinsically tied to the workflow of US Specialty Rx — for
example, transient status enumerations whose meaning is only
well-defined inside this IG's transactions — keep the existing
`hl7.org/fhir/us/specialty-rx/CodeSystem/<id>` canonical and document
in the IG that the system is IG-scoped by design.

In practice this means the work group would, for each CodeSystem in the
IG: (a) classify it as "reusable" or "IG-scoped"; (b) raise UTG tickets
and migrate canonicals only for the reusable bucket per Disposition A;
(c) leave IG-scoped Code Systems where they are, with a note in the
IG's terminology page explaining the split.

#### Justification

TSMG guidance is strongest for terminologies that are reused across
IGs or that need cross-realm governance. Some IG-internal
enumerations don't meaningfully exist outside the IG's transactions,
and migrating them to THO adds governance overhead (UTG tickets,
release-coordination across HL7/UTG and the IG, dual-canonical churn
for implementers) for negligible reuse benefit. This option still
delivers TSMG's intent for the cases that matter while keeping the IG's
own internal vocabulary lightweight. The cost is a per-CodeSystem
classification call, and the work group must defend the
"IG-scoped" label in TSMG review.

---

### Disposition C: Decline

#### Proposal

Mark the ticket "Persuasive with Mod" → declined as written, on the
basis that the request is non-specific (it does not name which Code
System(s) in the US Specialty Rx (FHIR) IG should move to THO) and
that the migration would constitute a breaking canonical-URL change to
a published STU artifact. Recommend instead that the request be re-filed
as discrete tickets, one per Code System, scheduled against a
major-version (non-patch) ballot of US Specialty Rx, after the work
group has consulted with TSMG on the desired THO id(s).

#### Justification

The ticket as written has no actionable target — it asserts a policy
without naming the artifact(s) it applies to. Acting on it requires the
work group to do the discovery TSMG should arguably be doing in
collaboration with Pharmacy. Additionally, changing a published
CodeSystem canonical URL is a breaking change for any implementer that
has bound to it, and warrants its own ballot cycle and per-system
discussion rather than a blanket migration driven by a single,
non-specific ticket.

---

### Recommendation

**Recommended disposition:** A

The TSMG / Code System Identifier Task Force guidance has been the
established HL7 pattern for several years, and the re-open comment
explicitly ties the original deferral to a policy that is now in
place. Disposition C's "the ticket is too vague" objection is real but
fixable inside Disposition A — the work group can do the
classification of which Code Systems exist in the IG as part of
acceptance and raise UTG tickets per system. Disposition B is
defensible but introduces a "reusable vs IG-scoped" judgment call that
is itself contentious in TSMG review and tends to drift; in practice
most published HL7 IGs have moved their CodeSystems wholesale to THO,
and US Specialty Rx is a small enough IG that bifurcating the
treatment is more overhead than it saves. Accepting the ticket and
sequencing the canonical change against the next non-patch ballot,
with coordinated UTG registration, is the cleanest path and is what
TSMG is explicitly asking for. The Justification under Disposition A
should be amended to address the breaking-change risk by committing to
a deprecation note and migration guidance in the IG's change log.
