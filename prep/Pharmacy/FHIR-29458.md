## Summary

The reporter flags two editorial defects in the prose of the US Prescription
Drug Monitoring Program (PDMP) FHIR Implementation Guide (the ticket is
filed against the broader "US Medication FHIR IG" container, but the
content is PDMP-specific). First, the trade name **SCRIPT** — NCPDP's
prescription transaction standard — is rendered in mixed case ("script")
in at least one passage and should be uppercase. Second, the surrounding
sentence claims that NCPDP SCRIPT 10.6 is the latest version in implementer
use, but **NCPDP SCRIPT 2017071** is also widely deployed and should be
called out alongside 10.6. The reporter supplies an explicit before/after
text replacement for the affected sentence.

The ticket has a complex disposition history: it was originally resolved
**Considered for Future Use** in January 2023 because the PDMP IG had been
withdrawn. It was **reopened on 2023-11-28** under PSS-2225 ("PDMP on
FHIR"), which restarted the project. The current resolution recorded on
the ticket is **Not Persuasive with Modification**, with a resolution
narrative that goes beyond pure capitalization: it removes the specific
MedicationRequest → SCRIPT 10.6 NewRx mapping (on the basis that
MedicationRequest is only marginally used by PDMP — prescription number,
original fills authorized, etc.) and updates the related tables to
**SCRIPT 2017071 RxHistoryRequest / RxHistoryResponse**. The ticket is
currently in **Triaged** status and needs a workgroup vote to confirm or
revise that resolution.

No cross-referenced Jira tickets, Zulip threads, or GitHub items were
found in FHIR Augury for this ticket.

## Proposed Dispositions

The ticket itself is editorial in scope (capitalization + version
disclosure), but the recorded resolution narrative additionally proposes
removing a `MedicationRequest → SCRIPT 10.6 NewRx` mapping. The
dispositions below treat those as separable so the workgroup can choose
which to adopt. No GitHub repo was surfaced by Augury for this ticket;
when a disposition implies a spec edit, it would land in the PDMP IG
authoring source maintained by the Pharmacy WG (the IG is not currently
mapped in the FHIR Augury repo briefings, so the workgroup will need to
direct the editor to the correct authoring root in the PDMP IG repo
before the change is applied).

### Disposition A: Accept As Requested

#### Proposal

Apply the reporter's literal text fix to the PDMP IG narrative page
containing the "Most of the existing EHR implementations use NCPDP
script 10.6 or ASAP web services…" sentence:

- Capitalize **SCRIPT** consistently throughout the IG narrative
  (search/replace `NCPDP script` → `NCPDP SCRIPT`, preserving links and
  intentional lower-case occurrences in code identifiers, if any).
- Replace the affected sentence with: *"Most of the existing EHR
  implementations use NCPDP SCRIPT 2017071, NCPDP SCRIPT 10.6, or ASAP
  web services…"*.

Do **not** modify any conformance artifacts, mappings, or tables — keep
this purely editorial.

#### Justification

This matches what the reporter actually asked for and is unambiguously
correct: "SCRIPT" is a registered NCPDP product name and is rendered
upper-case in NCPDP's own publications, and SCRIPT 2017071 is in
production use alongside 10.6 in PDMP-adjacent EHR deployments. As a
narrative-only change it is non-breaking, requires no migration, and can
be merged without re-balloting any structural content. It also closes
the original 2020 reporter request without expanding scope into mapping
changes that the reporter did not raise.

---

### Disposition B: Alternative Approach (Accept the broader on-ticket resolution)

#### Proposal

Adopt the resolution already drafted on the ticket as the actual change
set:

1. Apply the editorial fix from Disposition A (capitalize SCRIPT, add
   "NCPDP SCRIPT 2017071" to the version list).
2. **Remove** the specific narrative/mapping note that ties
   `MedicationRequest` to **SCRIPT 10.6 NewRx**, on the grounds that
   `MedicationRequest` is only marginally used by PDMP (prescription
   number, original fills authorized, etc.) and the explicit NewRx
   mapping overstates that linkage.
3. **Update** the related tables in the IG to reference **SCRIPT
   2017071 RxHistoryRequest** and **RxHistoryResponse** in place of (or
   alongside) the older SCRIPT 10.6 references, since RxHistory* is the
   transaction family actually exercised by PDMP query/response flows.

Resolve as **Not Persuasive with Modification**, exactly matching the
text already on the ticket.

#### Justification

This honours the workgroup's prior on-ticket analysis (post-PSS-2225
restart) rather than asking it to redo the work. The two extra changes
are tightly coupled to the same NCPDP SCRIPT version concern the
reporter raised, and the RxHistoryRequest / RxHistoryResponse tables are
the actually-used transactions for PDMP, so leaving the 10.6 NewRx
mapping in place would perpetuate a misleading framing about
`MedicationRequest`'s role in PDMP. The trade-off vs Disposition A is a
larger diff and a slightly broader review surface (mapping table edits
in addition to prose), but it leaves the IG materially more accurate
rather than just cosmetically corrected.

---

### Disposition C: Decline

#### Proposal

Resolve as **Not Persuasive** — leave the IG narrative as-is, on the
basis that (a) the casing of "SCRIPT" is editorial polish on a withdrawn
/ historically-deferred IG, and (b) calling out SCRIPT 2017071 in the
narrative is redundant with whatever conformance content the reopened
PDMP project will produce under PSS-2225.

#### Justification

Defensible only if the workgroup decides that any further edits to the
existing PDMP IG text are out of scope until PSS-2225 produces a fresh
ballot draft, in which case the new draft would describe the
NCPDP SCRIPT versions correctly from scratch and this old ticket would
be superseded. The cost is leaving an objectively wrong sentence in the
published narrative until that draft lands, which is hard to justify
given how cheap Disposition A is.

---

### Recommendation

**Recommended disposition:** B

The on-ticket resolution text was authored by the workgroup itself when
PSS-2225 reopened the IG, and it captures a substantive accuracy
improvement (RxHistoryRequest / RxHistoryResponse are the transactions
PDMP actually uses) that goes meaningfully beyond Disposition A's
cosmetic fix while staying within the same NCPDP SCRIPT version theme
the reporter raised. Disposition A is a strict subset of B and would
leave the misleading `MedicationRequest → SCRIPT 10.6 NewRx` reference
in place; Disposition C is hard to defend now that the IG is no longer
withdrawn. Recommend the workgroup formally vote **Not Persuasive with
Modification** with the resolution text already drafted on the ticket,
and direct the editor to apply the prose and table changes together in
a single PR against the PDMP IG authoring repo.
