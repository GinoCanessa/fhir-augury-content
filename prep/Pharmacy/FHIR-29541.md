## Summary

The ticket asks the Pharmacy work group to add (or correct) explanatory
guidance in the US Medication FHIR Implementation Guide describing the
proper way to record substance abuse and the use of substances such as
tobacco and alcohol. The reporter proposes specific guidance language:
`MedicationStatement` is appropriate when the substance is being tracked
for clinical decision support purposes (e.g., interaction checking, the
active medication list); `Observation` is the better choice when the
intent is to populate Social History or to capture additional contextual
information such as quantity per day, intent to quit, or related
negative health effects. The ticket was originally deferred when the IG
was withdrawn, then re-opened in late 2023 in the context of the
revived PDMP (Prescription Drug Monitoring Program) on FHIR project
(PSS-2225). No related FHIR Jira tickets, GitHub items, or Zulip
discussions were surfaced through cross-references or search.

## Details

**Description:**

> A MedicationStatement may be used to record substance abuse or the
> use of other agents such as tobacco or alcohol. This would typically
> be done if these substances are intended to be included in clinical
> decision support checking (for example, interaction checking) and as
> part of an active medication list. If the intent is to populate
> social history and/or to include additional information (for example,
> desire to quit, amount per day, negative health effects), then it is
> better to record as an Observation that could then be used to
> populate Social History.

## Proposed Dispositions

> Note: Because no GitHub cross-references exist, there is no
> repo-analysis briefing to anchor a concrete file path. The US

> the work group will need to map the proposal to the current source
> location of that IG when implementing whichever disposition is

### Disposition A: Accept As Requested

#### Proposal

Add a guidance section to the US Medication FHIR IG (likely on the
`MedicationStatement` profile/usage page, with a parallel cross-link
from a Social History / Observation usage page) that adopts the
reporter's wording almost verbatim:

- Use `MedicationStatement` to record substance abuse or use of
  substances such as tobacco or alcohol **when** the data is intended
  for clinical decision support (for example, drug-substance
  interaction checking) and/or for inclusion in the patient's active
  medication list.
- Use `Observation` (typed/coded for substance use, suitable for the
  Social History category) **when** the intent is to populate social
  history or to record additional context such as quantity per day,
  desire to quit, duration of use, or negative health effects.

The change is narrative-only: a new "Recording substance use" subsection
in the IG, plus a short cross-reference note on each affected resource
page. No StructureDefinition, value set, or conformance changes are
required.

#### Justification

This is consistent with the FHIR base spec's positioning of
`MedicationStatement` (a record of medication being taken or
administered) and with the long-standing convention that lifestyle and
social-context data live in `Observation` under the
`social-history` category. The guidance closes a real interpretation
gap that PDMP implementers run into (a tobacco/alcohol entry can land in
either resource depending on intent, and downstream consumers need a
predictable rule). It also matches the original ticket import note
(comment #96 from the PDMP review spreadsheet), so adopting the
reporter's framing keeps continuity with the PSS-2225 reboot referenced
by the 2023 re-open comment.

---

### Disposition B: Alternative Approach

#### Proposal

Adopt the same intent as Disposition A, but instead of two parallel
narrative blocks, define a single canonical decision rule and surface it
in three coordinated places:

1. A short "Choosing between MedicationStatement and Observation for
   substance use" decision table in the IG's main narrative (intent →
   recommended resource → required category/code), explicitly listing
   tobacco, alcohol, and "other substances of abuse" rows.
2. A pointer/`usage` note on the IG's `MedicationStatement` profile
   page that links to the decision table.
3. A pointer/`usage` note on (or a new) Social History `Observation`
   profile in the IG that links to the same table.

Optionally, define (or reference, if one already exists in the base
spec or US Core) a recommended value set for "substance use" codes so
implementers don't each invent their own code list.

#### Justification

A single decision table is harder to drift out of sync than two
free-text guidance paragraphs and is much easier for implementers to
quote in their own implementation guides and tooling. Anchoring the
guidance in one place also lets the work group revisit it once
(rather than in N profile pages) if PDMP requirements evolve. The
optional value-set step is what turns "guidance" into something that
can actually be conformance-tested, which is more valuable in a PDMP
context where data exchange between dispensers, prescribers, and PMP
systems is the whole point.

---

### Disposition C: Decline

#### Proposal

Decline the request as **already addressed by existing FHIR base-spec
guidance**: the `MedicationStatement` resource boundary text and the
`Observation` Social History category usage notes already make this
distinction at the base level. Close the ticket as
"Not Persuasive — guidance already exists in the base specification"
and leave the US Medication IG narrative unchanged. If clarifying text
is desired specifically for PDMP, redirect the change to the PDMP IG
when it reaches ballot, rather than carrying it in the US Medication
IG.

#### Justification

The base-spec boundary text for `MedicationStatement` already covers
the substance-use case at a high level, and Social History as an
`Observation` category is a well-established US Core/FHIR pattern.
Adding the same guidance into the US Medication IG risks divergence
from the base spec if either side is later edited, and it stretches
the scope of the US Medication IG into social-history territory.
The 2023-01-18 disposition was already "Considered for Future Use"
with a clear deferral rationale (the IG had been withdrawn); the only
new fact since then is that PSS-2225 has revived the PDMP project, so
the natural home for the clarification is the PDMP IG itself.

---

### Recommendation

**Recommended disposition:** B

The substance is the same as Disposition A — the reporter's framing is
correct and useful — but expressing it as a single decision table with
linked references from the affected profile pages is materially more
durable. It avoids the "two paragraphs that drift" failure mode that
narrative-only guidance is prone to, and it gives PDMP implementers
something they can directly cite. Disposition C is defensible on
process grounds (the base spec does say something) but in practice
implementers reading the US Medication IG do not reliably consult the
base `MedicationStatement` boundary text, and the re-open in the
context of PSS-2225 indicates the work group already considers this an
IG-level concern. Recommend B, with the optional value-set work
treated as a follow-up if scope allows.
