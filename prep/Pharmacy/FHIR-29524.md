## Summary

FHIR-29524 ("Pharmacies also access PDMPs - again") is a Pharmacy work
group change request filed by Scott M. Robertson against the US
Prescription Drug Monitoring Program (PDMP) FHIR Implementation Guide.
The reporter notes that, both in the descriptive text and in Figure 2,
pharmacies should be recognised as PDMP requesters (not only as data
sources or recipients), mirroring a similar comment the reporter made
against Figure 1. The ticket has had a long lifecycle: it was first
dispositioned in January 2023 as "Considered for Future Use" (the
original PDMP IG had been withdrawn), then reopened on 2023-11-28 in
connection with PSS-2225 ("PDMP on FHIR") so the request would carry
forward into the new IG. The ticket currently carries
**Resolution = Persuasive** with the resolution note "agree that
pharmacies are both request and receive PDMP records. This is clarified
in the new IG", indicating the workgroup intends the new PDMP IG to
explicitly model pharmacies as both PDMP requesters and recipients in
the actor diagrams and supporting narrative.

No cross-referenced Jira tickets, Zulip threads, Confluence pages, or
GitHub items were returned by the cross-reference index for this key,
so the request is summarised purely from the ticket itself.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Update the US PDMP FHIR IG narrative and Figure 2 (and, by extension,
Figure 1 per the reporter's earlier comment) so the **Pharmacy actor**
is explicitly shown as a **PDMP requester** in addition to its existing
role(s) as data submitter / recipient. Concretely, the IG editors
should:

1. In Figure 2 of the PDMP IG, add an arrow / labelled flow from the
   Pharmacy actor into the PDMP system representing "query / request
   PDMP record(s)", mirroring the existing requester flow used for
   prescriber actors.
2. Update the supporting narrative (the paragraph(s) introducing
   Figure 2, and the "Actors" section) to state that pharmacies both
   request PDMP records (e.g., at point of dispense) and receive /
   contribute PDMP data, citing the same use cases as for prescribers.
3. Repeat the equivalent change against Figure 1 to keep the two
   figures consistent with the cross-referenced sibling ticket.

#### Justification

This is exactly what the reporter requested and is consistent with how
state PDMPs operate in practice: pharmacists routinely query PDMPs at
dispense as part of controlled-substance review, in addition to
submitting dispense data. The current resolution note ("agree that
pharmacies are both request and receive PDMP records. This is clarified
in the new IG") already commits the workgroup to this position, so
Accept aligns the persisted disposition (`Persuasive`) with the
narrative change in the next publication of the PDMP IG.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than treating the pharmacy-as-requester change as a one-off
narrative tweak, model the PDMP **actor catalogue** as a small
structured table (Actor → Roles: { Submitter, Requester, Recipient })
that drives both Figure 1 and Figure 2 in the PDMP IG. Add `Pharmacy`
to that table with `{ Submitter, Requester, Recipient }` roles and
regenerate the figures (or re-author them) from that source of truth.
Cross-link the actor catalogue from the workflow / use-case sections so
future role additions (e.g., care managers, behavioural-health
clinicians) only require a single edit.

#### Justification

The fact that this issue was filed as "again" — and that the reporter
had to file the same comment against Figure 1 separately — is evidence
that the figures and narrative are drifting because actor roles are
duplicated in multiple places. A small, explicit actor/role table makes
the IG self-consistent by construction, makes ballot review easier
(reviewers can compare one table instead of two figures plus prose),
and reduces the risk that a future ticket re-opens the same issue for
yet another actor. The downside is more editorial effort up front, and
it slightly expands the scope beyond what FHIR-29524 literally asks
for.

---

### Disposition C: Decline

#### Proposal

Decline the change on the grounds that the original PDMP IG was
withdrawn (per the 2023-01-18 disposition note) and that any
clarification of pharmacy roles should live entirely with the new IG
under PSS-2225, with no carry-over of the legacy ticket — i.e., close
this ticket as superseded / not-applicable and let the new IG project
gather fresh requirements.

#### Justification

Defensible only on procedural grounds: the artefact this ticket was
filed against no longer exists in its original form, and the new IG
has its own scope statement and ballot pipeline. However, this
rationale is undermined by the reporter's 2023-11-28 reopen, the
existing `Persuasive` resolution, and the explicit resolution note
that the new IG **will** clarify pharmacies as both requesters and
recipients — so declining now would discard accepted workgroup intent.

---

### Recommendation

**Recommended disposition:** A

The ticket already carries `Resolution = Persuasive` with a resolution
note that explicitly commits the workgroup to clarifying pharmacies as
both PDMP requesters and recipients in the new IG. Disposition A simply
operationalises that commitment against Figure 2 (and Figure 1, per
the sibling comment) and the surrounding narrative — a small,
well-scoped editorial change with no model-level impact on
MedicationDispense or related resources. Disposition B is attractive
as a longer-term hygiene improvement and the workgroup may want to
keep it on the backlog as a follow-up, but it should not block
closing FHIR-29524. Disposition C is not defensible given the
reporter's reopen and the existing accepted resolution.
