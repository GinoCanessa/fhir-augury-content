## Summary

The reporter (John Moehrke) suggests that the Scalable Consent Management
(SCM) FHIR IG would benefit from a cross-reference to **IHE PCF Appendix P**,
which presents an explainer of the FHIR Consent model that overlaps with the
conceptual material already in the SCM IG. The intent is purely informational:
add a pointer so SCM readers can also benefit from the IHE PCF
treatment. The ticket has already been triaged with a **Persuasive**
resolution and a stated plan to add the reference in the IG's conceptual
background section. No cross-referenced Jira tickets were surfaced.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Add an informative cross-reference to **IHE PCF Appendix P**
(`https://profiles.ihe.net/ITI/PCF/ch-P.html`, anchor
`#p4-explainer-of-fhir-consent-model`) in the conceptual background
section of the Scalable Consent Management IG. The reference would
appear as a short paragraph noting that Appendix P provides a
complementary, plain-language explainer of the FHIR Consent model
(including the R4/R5 root-provision-vs-exception semantics and DS4P
policy linkage), and pointing readers there for additional background.
Place the reference next to the existing conceptual narrative; mark it
clearly as informative (not normative) so SCM does not inherit any
normative dependency on PCF.

#### Justification

This matches the ticket's literal request and the resolution already
recorded on the Jira issue ("Persuasive"). The reporter — John Moehrke,
co-author of both the SCM IG's underlying Consent material and IHE PCF
— has, on three separate Zulip threads, treated Appendix P as the
authoritative explainer of the FHIR Consent model, including the exact
topics SCM covers (policy linkage, DS4P interaction, provision
semantics). Adding an informative pointer is low-cost, non-normative,
and helps SCM readers without duplicating PCF material into the IG.
There is no observed dissent on Zulip.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than a single inline cross-reference, add a small
**"Related external material"** subsection in the SCM IG's conceptual
background that lists complementary references in one place: IHE PCF
Appendix P (Consent model explainer), the HL7 Data Segmentation for
Privacy (DS4P) IG, and the HL7 Healthcare Classification System (HCS)
material referenced from the FHIR core security-labels page. Each entry
gets a one-line description of what the reader will find there.
Continue to mark the section as informative.

#### Justification

The Zulip evidence shows the same author repeatedly bundling Appendix P
with DS4P and HCS when explaining `Consent` and security labels. A
small grouped subsection pre-empts the "what about DS4P / HCS?"
follow-up tickets and gives implementers a single discoverable index of
external explainers. The cost is essentially the same as Disposition A
but the IG gains a more durable, expandable pattern. The trade-off is
slightly more editorial work and a need for the WG to agree on which
external references belong in the list (scope creep risk).

---

### Disposition C: Decline

#### Proposal

Do not add the reference. Note in the ticket that the SCM IG already
covers the conceptual material itself, and that IHE PCF Appendix P is
discoverable via the existing FHIR ↔ IHE liaison documentation, so a
specific in-IG pointer is unnecessary.

#### Justification

This would normally be defensible only when an IG wants to avoid
external dependencies (even informative ones) or where the external
document is unstable. Neither condition obviously applies here:
Appendix P is a published IHE document with a stable URL, and the
proposed reference is informative. Decline is therefore a weak option;
its only real merit is keeping the IG's external-reference surface
minimal.

---

### Recommendation

**Recommended disposition:** A

The reporter has stated the request narrowly and the WG has already
recorded a "Persuasive" resolution agreeing to add the reference; the
recorded resolution text matches Disposition A almost verbatim.
Disposition A is the lowest-friction action that closes the ticket as
the WG has already agreed. Disposition B is attractive as a forward
direction and the WG may want to take it as a follow-up (a separate
ticket to introduce a "Related external material" subsection, into
which Appendix P is the first entry), but it should not block this
ticket. Disposition C is not supported by the Zulip evidence or the
existing resolution and would amount to reversing the WG's prior
position without new information.
