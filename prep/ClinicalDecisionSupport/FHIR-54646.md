## Summary

This is an editorial CDS Hooks ballot comment against the *Feedback* section
of the CDS Hooks specification. The reporter (Celine Lefebvre) finds the
existing sentence — *"Note that while one or more suggestions can be
accepted, an entire card is either ignored or overridden."* — confusing,
because read literally it implies a whole card cannot be accepted. The
reporter proposes rewording to: *"Note that while one or more suggestions
(or an entire card) can be accepted, ignoring or overriding is only
possible for an entire card, not with individual suggestions."*

The work group's proposed resolution (recorded on the ticket and pre-applied
in [HL7/cds-hooks#79](https://github.com/HL7/cds-hooks/pull/79)) agrees that
the original sentence is unclear, but rejects the reporter's exact wording on
the grounds that "accept an entire card" conflicts with the feedback schema,
where `Feedback.outcome = accepted` requires `acceptedSuggestions` (i.e.,
acceptance is intrinsically suggestion-scoped). The proposed reword is:
*"Note that individual suggestions within a card can be accepted, while
ignoring or overriding applies to the entire card."* The current resolution
label is **Not Persuasive with Modification** (the resolver notes it could
arguably be Persuasive with Modification). No related Jira tickets were
surfaced by cross-referencing.

> with the `acceptedSuggestions` array in the feedback schema.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Replace the sentence in `HL7/cds-hooks` `docs/specification/current.md`
(Feedback section intro, currently around line 782 of the equivalent file
in `cds-hooks/docs`) with the reporter's exact wording:

> *"Note that while one or more suggestions (or an entire card) can be
> accepted, ignoring or overriding is only possible for an entire card,
> not with individual suggestions."*

No other text changes; no schema, table, or example changes.

#### Justification

This minimally honours the reporter's request and unblocks the ballot
comment as literally written. The reporter's reading of the original
sentence as confusing is well-founded — the original is structurally
contrastive ("while ... an entire card is ...") but the contrast targets
two different verbs (accept / ignore-or-override), which obscures the
fact that *acceptance is suggestion-scoped*. The reporter's wording is
explicit on both halves of the contrast.

### Disposition B: Alternative Approach (Work Group's Pre-Applied Reword)

#### Proposal

Replace the same sentence with the wording already pre-applied in
[HL7/cds-hooks#79](https://github.com/HL7/cds-hooks/pull/79):

> *"Note that individual suggestions within a card can be accepted, while
> ignoring or overriding applies to the entire card."*

This is a single-sentence swap in `docs/specification/current.md`
(Feedback intro paragraph). Optionally, add one clarifying half-sentence
pointing forward to the `acceptedSuggestions` field in the *Feedback
attributes* table for readers who want the schema-level grounding (e.g.,
"…— see `acceptedSuggestions` below.").

#### Justification

This preserves the editorial intent of the reporter's comment (clarifying
the contrast between *accept* and *ignore/override*) while keeping the
prose aligned with the existing feedback schema. The reporter's literal
wording introduces "accept an entire card", which is not a state the
schema can represent: `Feedback.outcome = accepted` *requires*
`acceptedSuggestions` (current.md L799), so a card has no
"card-as-a-whole accepted" outcome — only "all of its suggestions were
accepted", which is itself only meaningful suggestion-by-suggestion.
Adopting the reporter's wording risks introducing a downstream
inconsistency between the prose and the conformance schema. The WG's
reword keeps acceptance suggestion-scoped (matching the schema) and
moves the card-scoped verbs (ignore/override) into the second clause,
mirroring the *Card ignored* and *Overridden guidance* sub-sections that
follow.

### Disposition C: Decline

#### Proposal

Mark the comment **Not Persuasive** and leave the existing sentence
unchanged:

> *"Note that while one or more suggestions can be accepted, an entire
> card is either ignored or overridden."*

#### Justification

Defensible on the grounds that the existing sentence, while terse, is
technically correct and consistent with the feedback schema. However,
this disposition disregards the reporter's substantive point that the
sentence is confusing on first read, and it ignores the fact that the WG
has already invested in a pre-applied PR with a better wording. Decline
is the weakest option here.

### Recommendation

**Recommended disposition: B (Not Persuasive with Modification — apply
the WG's pre-applied reword via PR #79).**

This is the disposition the resolver has already drafted, the one already
implemented in [HL7/cds-hooks#79](https://github.com/HL7/cds-hooks/pull/79),
and the one the reporter implicitly accepted by withdrawing the in-person
request on 2026-04-15. Substantively, B is the right answer: it concedes
the reporter's editorial point (the original sentence is unclear) without
introducing the schema-incompatible phrase "accept an entire card" that
disposition A would bake into the spec prose. The resolver's own note
that this could "arguably be Persuasive with Modification" is reasonable
— if the WG wants to recognise that the reporter's underlying ask was
honoured (just not with their literal wording), upgrading the resolution
label to *Persuasive with Modification* on Disposition B is a clean call
to make on the WG call. Either label, the recommended action is the same:
land PR #79 and close the ticket.
