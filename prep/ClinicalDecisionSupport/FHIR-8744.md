## Summary

The reporter requests an editorial change to the Clinical Quality Information
Framework (CQIF) IG: the table of contents should appear on the main (landing)
page of the specification rather than being published as a separate, dedicated
page. The ticket is small in scope and purely a publication / navigation
concern — no resource, profile, or constraint changes are implied.

No cross-referenced Jira tickets, Zulip conversations, or GitHub items were
returned for this ticket, so the request stands on its own. The CQIF
specification is now flagged as **deprecated** in Jira metadata, which is
material to the disposition discussion below.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Update the CQIF IG publication so that the table of contents is rendered on
the main (index/landing) page rather than being published as a standalone
"Table of Contents" page. Remove the separate ToC page from the IG (or
redirect it to the landing page) and embed the ToC content inline at (or
near) the top of the landing page.

#### Justification

The change matches the literal request and is consistent with the typical
HL7 IG layout, where the landing page surfaces a navigable outline of the
specification. Embedding the ToC reduces an unnecessary click for first-time
readers and aligns with how most other CDS / FHIR IGs are organized.

---

### Disposition B: Alternative Approach

#### Proposal

Keep the dedicated table-of-contents page (so deep links to it remain
valid), but add a compact, auto-generated ToC block to the main page —
either a top-level outline of the IG's major sections or a sidebar /
collapsible "On this page" widget — and prominently link to the full ToC
page from the landing page. Optionally, redirect the standalone ToC page to
the landing-page anchor once the inline ToC is in place.

#### Justification

This satisfies the underlying user need (immediate navigation from the
landing page) without breaking any existing external links to the
standalone ToC page. It is a lower-risk publication change and is closer to
how the FHIR core specification handles ToC + landing-page navigation.

---

### Disposition C: Decline

#### Proposal

Decline as **Persuasive with Mod / Not Persuasive — out of scope**: the
CQIF specification is deprecated (per Jira metadata: "Clinical Quality
Information Framework (CQIF) [deprecated]"), so editorial / navigation
improvements to its publication are no longer warranted. Close the ticket
with a note pointing readers at the successor CDS-related artifacts (e.g.,
Clinical Reasoning, CQF measures/PlanDefinition work).

#### Justification

Investing publishing-tooling effort in a deprecated IG diverts work group
attention from supported specifications and risks confusing implementers
about CQIF's status. Editorial-only changes do not improve interoperability
and do not justify republishing a deprecated guide. Declining is defensible
because the underlying specification itself is no longer maintained.

---

### Recommendation

**Recommended disposition:** C

The CQIF IG is marked deprecated in Jira, and the request is a purely
editorial / publication-layout change with no normative impact. Spending
work-group cycles on republishing a deprecated guide — even for a
well-intentioned navigation improvement — sets a poor precedent and may
confuse implementers about CQIF's status. CDS WG should close this ticket
with a brief note acknowledging the suggestion's merit and pointing to the
active successor work (Clinical Reasoning / CQF). If the WG nonetheless
wants to honor the request because the CQIF artifact is still served,
Disposition B is the lowest-risk way to do so without breaking existing
links.
