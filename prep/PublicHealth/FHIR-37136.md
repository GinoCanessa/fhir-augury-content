## Summary

The ticket asks the Maternal and Infant Health Research (MIHR) IG team to
move the canonical URL of the `ProceduresDuringPregnancy` ValueSet (and,
by implication, any other ValueSets in the IG that are not bound as
`required` to a `code` data type) out of the `hl7.org/fhir/us/mihr`
namespace and re-anchor it under `terminology.hl7.org` (THO). A
companion UTG ticket would be required to register the value set in
THO. This reflects HL7 TSMG guidance that only required-bound code
value sets stay in the `hl7.org/fhir` namespace; everything else lives
in THO. The ticket was originally dispositioned "Considered for Future
Use" on 2022-06-02 (Ruby Nash / Sarah Gaunt, 21-0-2) pending IG
maturity and finalization of the draft THO process, then re-opened by
Lynn Laakso on 2023-03-29 to be picked up post-STU publication. No
cross-referenced Jira tickets, Zulip threads, or GitHub items were
returned by FhirAugury for this ticket key.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Re-anchor the canonical URL of
`http://www.hl7.org/fhir/us/mihr/ValueSet/ProceduresDuringPregnancy` to
`http://terminology.hl7.org/ValueSet/ProceduresDuringPregnancy` (or an
equivalent THO-namespaced URL), update every binding in the MIHR IG
that points at the old URL to point at the new one, and file a
companion UTG ticket to register the value set in THO. Apply the same
treatment to any other ValueSets in the MIHR IG whose bindings are
*not* `required` to a `code` data type — TSMG guidance is that only
those required-code-bound value sets remain under `hl7.org/fhir`. Add
a release note flagging the URL change for downstream implementers
since canonical URLs are normative identifiers.

#### Justification

This is exactly the policy TSMG and the Vocabulary WG have asked all
IGs to apply: implementation-guide-specific value sets that wrap
external code systems (SNOMED CT, LOINC, ICD, etc.) belong in THO so
that they can be reused across IGs and versioned independently of any
single IG. Keeping them under `hl7.org/fhir/us/mihr` couples the
terminology lifecycle to the IG release cycle and forecloses reuse by
other Public Health IGs that might also need a "procedures during
pregnancy" set. The original 2022 deferral was explicitly conditioned
on "the maturity of the IG and when the draft THO process has been
finalized" — both conditions are now substantially met, which is why
Lynn Laakso re-opened the ticket post-STU.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than relocating each value set one-by-one as individual change
requests, treat FHIR-37136 as the trigger for a single IG-wide
terminology audit: enumerate every ValueSet defined in the MIHR IG,
classify each as (a) required-binding to `code` → keep at
`hl7.org/fhir/us/mihr`, (b) anything else → migrate to THO, and bundle
the migrations into one coordinated MIHR + UTG submission. Track the
audit and the per-VS UTG tickets as sub-tasks of FHIR-37136 (or of a
new umbrella ticket) so the URL changes ship together rather than
trickling out across multiple ballot cycles.

#### Justification

Doing this once, holistically, is cheaper for implementers than
shipping a series of canonical-URL changes spread across point
releases — each canonical URL change is a breaking change for anyone
who pinned the old URL, and bundling them limits that pain to one
upgrade. It also avoids the inconsistency of an IG that has
"half-migrated" terminology, which is confusing for reviewers and
profile authors. The downside vs. Disposition A is that it is a
larger initial unit of work and depends on UTG bandwidth to register
a batch of value sets at once.

---

### Disposition C: Decline

#### Proposal

Close FHIR-37136 as "Not Persuasive — Considered for Future Use is
still the correct disposition." Leave the ValueSet canonical URLs at
`hl7.org/fhir/us/mihr/ValueSet/...` for the current STU, and revisit
during the next normative cycle when the IG and THO processes are
both stable enough that the migration can be performed once and not
revisited.

#### Justification

The original 2022 vote (21-0-2) explicitly chose this path on the
grounds that the IG was immature and the THO process was still in
draft. Re-opening post-STU does not by itself prove that either
condition has changed enough to justify the breaking canonical-URL
change for current implementers. Canonical URLs are normative
identifiers; changing them is a non-trivial cost to existing
consumers, and a "do it later, do it once" stance may still be the
lowest-total-cost option.

---

### Recommendation

**Recommended disposition:** B

The underlying policy in Disposition A is correct and aligned with
TSMG / Vocabulary guidance, but executing it on a single value set in
isolation (`ProceduresDuringPregnancy`) leaves the rest of the MIHR
IG inconsistent and sets up a series of future canonical-URL changes
— each one a breaking change for downstream implementers. Disposition
B preserves the same end-state as A while collapsing the implementer
pain into one coordinated migration and one UTG submission batch,
which is the pattern other Public Health IGs have followed when
acting on the same TSMG guidance. Disposition C is defensible only if
the workgroup judges that THO process readiness has *not* materially
improved since 2022; given that Lynn Laakso re-opened the ticket
explicitly "post STU publication," the more honest reading is that
the deferral conditions are now satisfied and the work should be
scoped and scheduled rather than deferred again.
