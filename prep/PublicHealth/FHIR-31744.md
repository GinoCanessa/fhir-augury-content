## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Replace the placeholder phrase **"Need more here"** in the opening paragraph
of the SANER **Testing and Conformance** page with substantive introductory
prose, and clean up the surrounding sentence so it parses correctly. Concretely,
rewrite the first paragraph along the lines of:

> "This chapter enumerates the FHIR conformance resources used for validating
> implementations of this guide, defines a test plan for evaluating each
> actor's behaviour, and provides reference test data that implementers can
> use to exercise their systems."

Then add a short scoping sentence (or bulleted overview) that names the
sub-sections of the page (e.g., conformance resources, actor test plan, test
data sets) so the introduction matches the chapter's actual structure. No
profiles, value sets, capability statements, or examples are touched — this
is a narrative-only edit to the SANER IG's Testing and Conformance page in
the SANER source repository.

#### Justification

The reporter's request is a pure documentation defect: an author note
("Need more here") was shipped in published narrative text. Fixing it carries
zero conformance impact, no risk to existing implementations, and is the
minimum action required to make the page read as intended. The ticket has
already been resolved as **Persuasive** and the reporter has confirmed in a
follow-up comment ("Was done, additional text was provided") that this fix
was applied, so Disposition A simply ratifies what the work group has
already done.

---

### Disposition B: Alternative Approach

#### Proposal

Treat this as the trigger for a small editorial sweep across the SANER IG
rather than a one-line fix. In addition to rewriting the Testing and
Conformance opening paragraph as in Disposition A, run a grep across the
SANER narrative pages for other author-only markers ("Need more here", "TBD",
"TODO", "FIXME", "xxx", lone "..." sentences, etc.) and either fill them in
or remove them. Capture the resulting list of fixes under this ticket (or a
follow-on tracker) so reviewers can confirm the IG no longer ships with
visible editorial scaffolding.

#### Justification

A page that opens with the words "Need more here" inside published prose is
almost always a symptom of a broader editorial gap — the same pattern often
appears elsewhere when a draft IG is moved to ballot or publication. Doing a
single targeted sweep is cheap, requires no normative changes, and prevents
the work group from being asked the same question multiple times for
different pages. The trade-off versus Disposition A is a slightly larger
authoring effort, but still entirely narrative and non-breaking.

---

### Disposition C: Decline

#### Proposal

Decline the request as **out of scope / no longer actionable**: the ticket
describes a stale snapshot of the SANER IG, the placeholder text has already
been replaced (per the reporter's own 2024-01-11 comment "Was done,
additional text was provided"), and there is no remaining normative or
narrative defect to act on. Close the ticket with no further changes.

#### Justification

Declining is defensible only on the narrow grounds that the underlying defect
no longer exists in the current SANER source. There is no design principle,
backward-compatibility concern, or scope argument against making the fix —
the fix is trivial and uncontroversial. The only reason to choose this option
is administrative: avoid re-doing work that has already been merged. If
verification shows the placeholder is still present in the current SANER
publication, this disposition collapses back into Disposition A.

---

### Recommendation

**Recommended disposition:** A

Disposition A matches both the literal request and the path the work group
has already taken (resolution = Persuasive, reporter-confirmed "Was done").
The change is narrative-only, scoped to a single SANER IG page, and carries
no impact on profiles, terminology, capability statements, or implementer
behaviour, so there is no reason to escalate it into a broader editorial
sweep (Disposition B) unless the work group independently wants to do so —
that can be tracked as its own follow-up. Disposition C should only be
selected if a quick check of the current SANER source confirms the
placeholder is already gone everywhere, in which case the ticket can be
closed administratively; otherwise apply the fix described in Disposition A
and close.
