## Summary

The reporter, on behalf of NIOSH-NPPTL, asks the SANER (Situation Awareness for
Novel Epidemic Response) IG authors to add an explicit, end-to-end use case
that demonstrates how SANER can be applied to **collect, aggregate, and report
hospital Personal Protective Equipment (PPE) inventory and supply data** —
complementing the existing patient-focused situational-awareness use cases.
Specifically, the comment proposes new narrative subsections in §2.3 that
cover (a) automated collection of PPE "eaches" by model number, manufacturer,
and NIOSH approval number across multiple hospital systems into a "trust data
platform"; (b) the corresponding process flow that normalizes and standardizes
the data and surfaces capacity, replenishment, and warning levels; (c)
querying that platform for PPE supply and shortages by hospital, region, or
state through dashboards and `Measure`/`MeasureReport` queries (per existing
§2.2.5.5); and (d) automating measure computation on that neutral platform so
hospitals are not burdened with manual entry. The reporter argues a PPE data
standard would make this computation interoperable across stockpile and
public-health actors at federal, state, and local levels.

There are no cross-referenced Jira tickets, Zulip threads, or GitHub items in
the FHIR Augury index for this ticket, so context is drawn solely from the
ticket itself and its comment history.

## Details

**Description:**

It is important to provide a Use Case for Collecting Hospital PPE Measures
including the steps for collecting PPE data, measuring reports, and automating
the data received. NIOSH-NPPTL has an example of a use case that can be
provided as an example of a supply use case to complement the existing patient
use cases.

- **2.3.1 Collecting Hospital PPE Data Measures** — A use case for the
  collection of PPE from several hospital systems in an automated system that
  communicates supply back to hospitals and helps federal, state, and local
  stockpiles anticipate shortages and demand. A central, automated system
  collects PPE data elements such as N95 respirators, surgical masks, and face
  shields by examining the "eaches" received by model number, manufacturer,
  and NIOSH approval number, and reports them to a trust data platform for
  hospital systems. Supported by PPE current inventory reporting.
- **2.3.1.1 Collecting Hospital PPE and Measures Process Flow** — PPE report
  data can come from multiple applications or tools across hospitals, but is
  reported to one trust data platform for normalization and standardization.
  Provides PPE Capacity including total number of PPE available, PPE being
  replenished, and warning levels. NIOSH-NPPTL can provide a detailed process
  flow as outlined in 2.3.1.1-1.
- **2.3.2 Accessing PPE Measure Reports** — Any federal, state, or local
  hospital or stockpile group should be able to query a reporting system to
  report on PPE supply and shortages for a particular hospital system, region,
  or state. Supported by measures and queries described in §2.2.5.5.
  Dashboards updated regularly using validated algorithms for current and
  replenished PPE supplies.
- **2.3.4 Automating Measure Computation** — A neutral trust data platform,
  through an automated procedure, can compute and report PPE measure data,
  reducing burden on hospitals and manual data entry. A PPE data standard
  (per earlier ballot comments) would make computation easier to automate and
  to communicate supply and demand.

## Proposed Dispositions

> Note: All dispositions are scoped to the **Situation Awareness for Novel
> Epidemic Response (FHIR)** IG (the SANER IG, owned by Public Health). No
> repository was surfaced for this ticket in the FHIR Augury cross-reference
> index, so file-path specifics below name SANER's documented section
> structure (the §2.3.x narrative pages) rather than concrete authoring file
> paths. If accepted, the implementer should consult the SANER IG repo's
> authoring root for the matching `pages/usecases/*.md` (or equivalent)
> sources before editing.

### Disposition A: Accept As Requested

#### Proposal

Add four new narrative subsections to the SANER IG's §2.3 use-case area:

1. **§2.3.1 Collecting Hospital PPE Data Measures** — narrative use case for
   automated, multi-hospital PPE inventory collection feeding a central
   platform, keyed off PPE "eaches" identified by model number, manufacturer,
   and NIOSH approval number. Reference the existing
   `Measure` / `MeasureReport`-based situational-awareness pattern already in
   the IG.
2. **§2.3.1.1 Collecting Hospital PPE and Measures Process Flow** — a
   sequence/flow diagram (sourced from NIOSH-NPPTL per the reporter) showing
   ingest → normalization → aggregation → capacity / replenishment / warning
   levels. Add a new SANER `Measure` definition (or a profile on `Measure`)
   for "PPE Capacity" with stratifiers for PPE category (N95, surgical mask,
   face shield, etc.), supply state (available, in-transit, replenishing),
   and warning level.
3. **§2.3.2 Accessing PPE Measure Reports** — narrative + example
   `MeasureReport` query patterns (extending the §2.2.5.5 query examples) for
   federal / state / local consumers retrieving PPE supply and shortage views
   by hospital, region, or state.
4. **§2.3.4 Automating Measure Computation** — narrative for the neutral
   trust-platform pattern, with guidance on attribution, latency, and
   provenance for automated measure computation, and a forward-pointer to a
   future PPE data standard.

Operationally: add the narrative pages, define one new "PPE Capacity"
`Measure` resource in the IG, and add at least one example
`MeasureReport` instance illustrating the stratifiers above.

#### Justification

The change extends SANER's existing patient-focused situational-awareness
pattern to supply-side data using exactly the same FHIR mechanism
(`Measure` / `MeasureReport`) the IG already uses, so the addition is
consistent with the IG's design and does not introduce a new modeling
paradigm. NIOSH-NPPTL has offered concrete real-world content for the process
flow, which addresses the most common reason ballot comments of this kind get
declined ("no actionable use case"). The 2021-04-22 vote (22-0-0) to consider
this for future STU work shows prior workgroup appetite, and re-opening it
post-STU1 was an explicit signal from the chair that it should be revisited
in a later STU.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than adding the PPE supply use case to the SANER IG itself, **publish
the PPE collection / reporting use case as a separate, supply-focused
companion IG** (or a profiling layer atop SANER) that:

- Reuses SANER's `Measure` / `MeasureReport` infrastructure verbatim.
- Defines PPE-specific `Measure` resources, value sets (PPE category, NIOSH
  approval status), and example `MeasureReport`s.
- Is owned jointly by Public Health and the appropriate supply-chain
  stakeholders (e.g., the Helios FHIR Accelerator, NIOSH-NPPTL, ASPR).
- Cross-links from SANER §2.3 with a one-paragraph pointer ("for hospital PPE
  supply use cases, see X").

The SANER IG itself would gain only a short "Related Work" pointer, not the
full §2.3.1 / §2.3.1.1 / §2.3.2 / §2.3.4 narrative.

#### Justification

The resolution comment ("Not with scope of this or Helios work") indicates the
SANER editors view supply-chain PPE reporting as adjacent to, but not part of,
the patient-acuity / hospital-status situational-awareness scope SANER was
balloted under. A companion IG keeps SANER focused on its existing scope while
giving the PPE use case a proper authoring home with the right SMEs (NIOSH,
ASPR, supply-chain WG). It also avoids forcing SANER implementers — most of
whom are reporting bed counts, ventilators, and case loads — to maintain or
ignore PPE measures they will never produce. The trade-off vs. Disposition A
is ecosystem fragmentation (two IGs to discover) and a longer time to publish
versus narrative-only acceptance, but it lines up better with HL7's general
preference for narrowly-scoped IGs.

---

### Disposition C: Decline

#### Proposal

Resolve as **Not Persuasive** (consistent with the existing 2024-01-11
disposition): hospital PPE supply-chain reporting is out of scope for the
SANER IG, which is chartered for situational awareness of patient and
facility status during novel epidemic response, not for inventory / supply
chain measures. Reporter is encouraged to bring the use case to the Helios
FHIR Accelerator, the Supply Chain workgroup, or a dedicated PPE-supply IG
project under Public Health.

#### Justification

This is the disposition the workgroup actually adopted on 2024-01-11
("Not with scope of this or Helios work"). The defensible rationale is that
SANER's `Measure` / `MeasureReport` machinery, while technically reusable for
PPE counts, was modeled and balloted against patient/bed/clinical-status
situational-awareness scenarios; adding a full PPE supply use case would (a)
expand SANER's scope materially beyond what was balloted, (b) require SME
input (NIOSH approval number, model/manufacturer identification, "eaches"
semantics) that the Public Health WG does not own, and (c) duplicate
emerging supply-chain work happening elsewhere. Declining keeps SANER tight
and pushes the work to a forum better suited to it.

---

### Recommendation

**Recommended disposition:** **C (Decline / confirm Not Persuasive), with an
explicit referral.**

The ticket has been re-opened, considered, voted on (22-0-0 for "future STU
work" in 2021), then ultimately resolved Not Persuasive in January 2024 with
the rationale that PPE supply reporting is out of SANER's and Helios' scope.
That history shows the workgroup has had multiple opportunities to absorb
this content and has consistently concluded it does not belong in SANER.
Re-litigating the same scope question without new SME engagement from
NIOSH-NPPTL or a sponsoring accelerator is unlikely to change the outcome.
However, the underlying use case is real and well-articulated, so the
disposition should be paired with a concrete pointer (Disposition B's spirit):
recommend the reporter open a project request with the Public Health WG and
Helios for a **PPE Supply Measures companion IG** that reuses SANER's
`Measure` / `MeasureReport` pattern, rather than letting the use case
disappear with the decline. If — and only if — a co-sponsoring group commits
to authoring the new content, the workgroup should reconsider Disposition A
in a future STU cycle.
