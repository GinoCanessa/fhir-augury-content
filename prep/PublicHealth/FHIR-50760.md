## Summary

Rick Geimer asks the Public Health work group to consider revising the
use cases in the **US Situational Awareness Framework for Reporting
(US SAFR)** IG so they are more generic and broaden the audience beyond
the IG's current framing. He proposes three replacement use cases —
(1) required reporting to public health authorities at the local,
state, or federal level; (2) a local emergency where surge capacity is
exceeded and hospitals need to share data with the state; and
(3) a state-declared disaster where the state needs to receive data
from local jurisdictions and forward it to the federal government —
and explicitly calls out that **STLTs (state, tribal, local, and
territorial entities) and other Public Health Authorities (PHAs)
should be represented** in the use cases.

The ticket has had a contested history. It was previously resolved as
**Considered for Future Use** by a 17-0-3 vote on 2026-03-12 with the
rationale that the IG's current use cases align with the example
measures already shipped, and that more generic use cases / example
measures would be considered in a future cycle. That resolution was
later **reverted and the ticket re-opened post-STU1 publication** by
Lynn Laakso. In ticket comments, the reporter himself signalled
flexibility ("ok keeping the use cases as is for now") but committed to
review with NHSN, and another commenter explicitly asked the team to
**work with NHSN** to produce better use-case descriptions. There are
no cross-referenced Jira tickets, Zulip threads, or GitHub items in
the data sources used for this prep.

## Details

**Description:**

Consider replacing the existing use cases with ones like this:

1. required reporting to public health authorities (local, state or federal).
2. Local emergency where surge is exceeded and hospitals need to share data with the state.
3. State declared disaster and the state needs to get data from the locals and share data with the federal government.

STLTs and other PHAs should be included in the use cases.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Update the **US SAFR (FHIR) IG narrative** — specifically the *Use
Cases* section of the IG home / introduction page — to replace the
current use cases with three generalized scenarios drawn directly from
the request:

1. **Required reporting to public health authorities** (local, state,
   or federal jurisdictions).
2. **Local emergency / surge** scenarios in which a hospital or
   facility's surge capacity is exceeded and aggregate situational
   data must flow to the state PHA.
3. **State-declared disaster** scenarios in which the state PHA
   collects data from local jurisdictions and forwards aggregate /
   situational data to federal partners.

In addition, explicitly enumerate **STLTs (state, tribal, local, and
territorial entities) and other Public Health Authorities (PHAs)** as
in-scope actors in the IG's *Actors* / *Stakeholders* discussion, and
audit example MeasureReport / Measure resources and any narrative that
implies a single (e.g., NHSN-style) reporting flow so the examples
remain consistent with the broader use cases. Coordinate the wording
with NHSN as requested in the comments to ensure the new narrative
does not invalidate the existing reporting pathway.

#### Justification

The request is editorial / scope-of-narrative rather than a normative
conformance change: it widens the IG's framing without altering
profiles, value sets, or operations. Generalizing the use cases
matches HL7's preference for IGs to describe *patterns of exchange*
rather than a single program implementation, and it directly responds
to the second commenter's request to "work with NHSN to come up with
better use case descriptions." Because Lynn Laakso explicitly
re-opened the ticket post-STU1, the work group has signalled that the
prior "Considered for Future Use" outcome is no longer adequate.

---

### Disposition B: Alternative Approach

#### Proposal

Keep the existing concrete use cases (which align with the shipped
example measures, per the original 17-0-3 vote rationale) but **add a
new "Generalized Use Cases" subsection** to the IG narrative that
frames the existing examples as instances of the broader patterns the
reporter describes:

- Mandated public-health reporting (local / state / federal).
- Surge-driven local emergency reporting (hospital → state).
- Disaster-driven jurisdictional escalation (local → state → federal).

Explicitly call out STLTs and other PHAs as actors in this new
subsection, and add a short "Applicability beyond NHSN" note clarifying
that the same IG mechanics apply to other PHA reporting programs.
Defer any actual replacement of the example measures to a future
release, consistent with the original work-group sentiment.

#### Justification

This preserves the alignment between the narrative and the example
artifacts that the work group voted to keep (17-0-3), avoids
invalidating implementer guidance that already references the existing
use cases, and still resolves the substantive concern — that the IG
reads as too narrow and omits STLTs / non-NHSN PHAs. It also matches
the reporter's own softened position ("ok keeping the use cases as is
for now") while giving the second commenter the broader framing they
asked for. Editorial layering is cheaper than re-baselining example
measures and carries no normative risk.

---

### Disposition C: Decline

#### Proposal

Resolve as **Persuasive with Mod / Considered for Future Use** (or
re-apply the prior "Considered for Future Use" outcome) on the
grounds that the existing use cases are intentionally aligned with
the example MeasureReport content shipped in STU1, that the reporter
has indicated they are "ok keeping the use cases as is for now," and
that any expansion should be performed as part of a coordinated NHSN
review in a future ballot cycle rather than as a post-publication
narrative rewrite.

#### Justification

The original disposition (17-0-3) was strongly in favor of keeping the
use cases as shipped because they align with the example measures.
The reporter has since stated he is comfortable leaving the use cases
unchanged pending an NHSN review, and there is no Zulip / GitHub /
linked-ticket evidence in the data sources of additional community
pressure for an immediate change. Re-doing the use cases without
parallel updates to the example measures would risk a narrative /
example mismatch that confuses implementers more than the current
narrowness does.

---

### Recommendation

**Recommended disposition:** B

Disposition B threads the needle between the original 17-0-3
preference to keep the IG aligned with its shipped example measures
and the explicit re-open by Lynn Laakso post-STU1, which signals that
"Considered for Future Use" is no longer acceptable. Adding a
generalized use-case subsection — with STLTs and other PHAs called out
by name and an "applicability beyond NHSN" note — directly answers
both the reporter's framing concern and the second commenter's request
to broaden the description, while avoiding the risk of de-syncing
narrative from artifacts that Disposition A carries. Because the
reporter himself has softened to "ok keeping the use cases as is for
now," a purely editorial broadening (B) is more proportionate than a
full replacement (A), and is materially more responsive than another
deferral (C). The work group should coordinate the new narrative with
NHSN before publication, as both commenters requested.
