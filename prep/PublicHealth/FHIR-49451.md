## Summary

The reporter requests that a "Plain Language Summary about this Guide" section
be added to the home page of the Medicolegal Death Investigation (MDI) FHIR
Implementation Guide. The intent is to make the IG more accessible to
non-technical audiences (patients, the general public, policy stakeholders) by
providing a short, jargon-free description of what the guide is and what it
does, modeled on the precedent set by the MCC IG
(`hl7.org/fhir/us/mcc/#plain-language-summary-about-hl7-and-this-guide`) and
on the broader practice of Plain Language Summaries (PLS) in peer-reviewed
publishing. The CDC team has already drafted candidate copy for the MDI IG
(captured in the most recent comment), so the request is effectively to
incorporate that text — minus the "About HL7" portion, per FHIR leadership
guidance — into the MDI IG home page. The ticket cross-references the HL7
Confluence page "Non-technical Summary for FHIR Based publications"
(Confluence page `281280513`), which describes the HL7 process for adding a
PLS to an already-published IG; the ticket has been re-opened for STU2 to
follow that process.

## Details

**Description:**

HL7 encourages Implementation Guides (IGs) to voluntarily include a "Plain
Language Summary about this Guide" on their home page. This addition can
enhance accessibility and understanding for non-technical audiences, including
patients and the general public.

For an example, see the "Plain Language Summary about HL7 and this Guide"
section on the MCC IG homepage:
https://hl7.org/fhir/us/mcc/#plain-language-summary-about-hl7-and-this-guide.
Note that FHIR leaders recommend excluding the "About HL7" portion for new
summaries.

To assist with this effort, IG authors can reference existing summaries
available in this GitHub repository (editable for use in an IG):
https://github.com/jmandel/chat.fhir.org-channels/tree/ae9bd11d491e3a62a0f2cd40a2ebff53dcf7afd8/summaries.

This recommendation aligns with a growing trend among peer-reviewed journals
to include Plain Language Summaries (PLS). These summaries simplify complex
concepts, improving engagement, transparency, and understanding for diverse
audiences. In fields like healthcare, PLS have become essential for fostering
public involvement. Some journals, including those published by SAGE, Taylor &
Francis, and PLOS, provide templates or require PLS as part of their
submission process (BioMed Central, Science Editor, PLOS).

By adopting this approach, HL7 can promote inclusivity, better serve its
stakeholders, and align with best practices in other domains.

## Proposed Dispositions

The dispositions below are scoped to the MDI IG. Because the FHIR Augury
GitHub catalog does not include the MDI IG repository, no `repo-analysis`
briefing is available; the dispositions therefore reference the IG by name
rather than naming a specific authoring root or file path. Implementers will
need to apply the change in the standard MDI IG narrative location
(typically `input/pagecontent/index.md` or the equivalent home-page source).

### Disposition A: Accept As Requested

#### Proposal

Add a new top-level section titled "Plain Language Summary about this Guide"
to the MDI IG home page, immediately after the introductory paragraph(s) and
before the technical scope/structure sections. Use the CDC-supplied wording
captured in the 2025-06-10 comment verbatim (or with light copy editing for
publication style). Per the guidance in the ticket description, **omit** the
"About HL7" portion that appears in the MCC IG precedent. Because the MDI IG
has already published STU2, route the change through the HL7 process for
adding a PLS to an already-published IG documented on Confluence page
[281280513](https://confluence.hl7.org/spaces/FHIR/pages/281280513/Non-technical+Summary+for+FHIR+Based+publications)
(errata / non-substantive narrative update), rather than waiting for the next
ballot cycle.

#### Justification

The request is narrative-only, additive, and carries no impact on resources,
profiles, value sets, or conformance. It directly aligns with HL7's stated
encouragement for IGs to include a PLS, follows an already-published HL7
precedent (MCC IG), and uses copy that the publishing CDC team has already
vetted. Treating it as an errata-style narrative update is exactly the path
the Confluence page describes for post-publication PLS additions, which is
why the work group reverted the earlier "Considered for Future Use"
resolution and reopened the ticket post-STU2. The 24-0-0 vote on the prior
disposition signals that the work group is comfortable acting on this; the
remaining question is purely *when* and *how*, not *whether*.

---

### Disposition B: Alternative Approach

#### Proposal

Accept the addition of a Plain Language Summary, but defer publication to the
next MDI IG ballot/release rather than processing it as a post-publication
narrative update. In the interim, capture the CDC-drafted text in the IG's
project tracker / build branch so it is ready to merge with the next STU
update, and add a short pointer on the current home page (e.g., a one-line
"A plain-language overview of this guide is in development") if any
home-page change is desired now. Optionally, also link from the MDI IG home
page to the VRDR IG's PLS (if/when added) to reinforce the relationship the
CDC-supplied summary already calls out.

#### Justification

This honors the intent of the request without invoking the
already-published-IG amendment process for narrative-only content, which some
work groups treat as reserved for higher-priority post-publication fixes.
Bundling the change with the next ballot keeps the published artifact stable
between releases and gives the work group a chance to harmonize the MDI PLS
with whatever PLS pattern the broader Public Health work group adopts across
its IGs (MDI, VRDR, and related). Trade-off: it delays the accessibility
benefit by one publication cycle and is harder to defend given that the work
group has already voted to follow the post-publication PLS process.

---

### Disposition C: Decline

#### Proposal

Decline the request to add a Plain Language Summary to the MDI IG home page,
on the basis that PLS inclusion is described by HL7 as voluntary and the MDI
IG already covers its scope and audience in the existing introduction. Close
the ticket as "Not Persuasive" or "Considered — No Action Required."

#### Justification

The HL7 guidance is encouragement, not a requirement, so non-adoption is
defensible on policy grounds. A reviewer could argue that the MDI IG's
existing "Background" / "Scope" narrative already serves the same function
for the IG's primary audience (medical examiners, coroners, vital records
offices, IT implementers), and that adding a separate PLS risks
duplication, drift between the two summaries over time, and additional
maintenance burden for future STU updates. This rationale is, however, weak
in light of the work group's recent 24-0-0 vote in favor of pursuing the PLS
process and the CDC team's willingness to supply the copy.

---

### Recommendation

**Recommended disposition:** A

The work group has already voted 24-0-0 in support of pursuing the
post-publication PLS process, the CDC team has provided ready-to-use text,
and HL7 explicitly documents an errata-style path for adding a PLS to an
already-published IG (Confluence page 281280513). The change is narrative
only, has no normative impact, and directly serves a "Highest" priority
ticket flagged with the `Narrative` label. Disposition B's deferral buys
nothing the work group hasn't already decided against, and Disposition C
contradicts the existing vote. The remaining work is execution: drop the
CDC-supplied paragraph (minus the "About HL7" preamble) into the MDI IG home
page narrative source and run it through the documented post-publication PLS
process.
