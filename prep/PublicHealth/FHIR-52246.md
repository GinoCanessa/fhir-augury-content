## Proposed Dispositions

> Note: Because there are no GitHub cross-references and no

### Disposition A: Accept As Requested

#### Proposal

Update **section 2.1** of the US Public Health Profiles Library (USPHPL)
IG narrative to explicitly add **FDA public health use cases** to the
list of priority use cases the IG aligns with, alongside the existing
USCDI+ Public Health alignment statement. Concretely:

- Extend the alignment paragraph in 2.1 to call out FDA public health
  data exchange (e.g., post-market safety surveillance, real-world
  data/evidence flows, regulatory submission of public-health-relevant
  observations) as in-scope use cases the profiles library supports.
- Add (or expand) a use-case subsection enumerating the FDA use cases
  intended to be covered, with a short description of each and a
  reference to the relevant FDA initiative or program.
- If specific profiles in the library directly serve those FDA use
  cases, link them from the new text; otherwise mark the use cases as
  "supported via the general profile set" pending future targeted
  profiling work.

#### Justification

The reporter is the FDA representative on the project and has flagged a
gap between the IG's stated alignment (USCDI+ Public Health only) and
the broader public health constituency the IG nominally serves. Adding
FDA use cases to section 2.1 is a **narrative-only change** with no
profile, value-set, or conformance impact and is therefore low risk
from an interoperability standpoint. It also corrects an inclusivity
gap before STU2 has long-term traction and aligns the IG's scope
statement with the SME label `2025_USPHPL_SME` already on the ticket.
The earlier "Considered for Future Use" disposition was reverted
explicitly to revisit the request post-STU2, which signals the WG is
open to re-incorporating the FDA use cases.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than expanding section 2.1 to enumerate FDA use cases as a peer
of USCDI+, **re-frame section 2.1** to describe the IG as a general
**US public health profile foundation** and move concrete use-case
alignment (USCDI+, FDA, CDC programs, state/local PH, etc.) into a
**dedicated "Use Cases" subsection** (e.g., 2.1.x or a new chapter).
That subsection would:

- List each supported public health use case (USCDI+ Public Health,
  FDA public health use cases, and any others the WG accepts) as a
  named entry with scope, sponsor, and current alignment status
  (aligned / partially aligned / future).
- Be designed to grow ballot-cycle by ballot-cycle without re-touching
  the IG's core scope statement.

The substantive add for this ticket is the FDA entry, but the structural
change makes future analogous tickets cheap to absorb.

#### Justification

The underlying need is recognition of FDA public health use cases, but
the broader pattern in this WG (e.g., the keyword-adjacent FHIR-21682
"public health surveillance as a use case" request, FHIR-30734 MedMorph
environmental-scan work, FHIR-35035 / FHIR-54223 USCDI references)
suggests the IG repeatedly attracts "please also list use case X"
tickets. A dedicated, structured Use Cases subsection makes the IG's
scope statement extensible and avoids the section-2.1 paragraph
becoming an ever-growing list. The marginal authoring cost over
Disposition A is small (one extra subsection), and the change is still
narrative-only.

---

### Disposition C: Decline

#### Proposal

Decline the request to add FDA public health use cases to section 2.1
on the grounds that the **USPHPL IG's scope is intentionally anchored
to USCDI+ Public Health** for STU2, and broader FDA-specific use cases
should be addressed in a separate FDA-sponsored IG (or a dedicated
profile) rather than by widening this IG's scope statement. Resolve as
**Not Persuasive** (or revert to **Considered for Future Use**) with a
note pointing the reporter at the appropriate downstream venue.

#### Justification

The original 2025-12-04 vote (29-0-6 in favour of "Considered for
Future Use") indicates the WG already weighed inclusion and
deliberately deferred. Until concrete FDA use cases with named profile
needs are submitted, expanding section 2.1 risks **scope creep** —
implementers would read the IG as claiming alignment with FDA workflows
the profiles do not actually cover, which is worse than silence.
Declining (or re-deferring) preserves the option to add the use cases
later when they can be backed by profiling work, without making a
narrative promise the IG cannot yet keep.

---

### Recommendation

**Recommended disposition:** B

The ticket is fundamentally a scope-statement complaint, and the WG has
already shown (via the reverted 2025-12-04 resolution and Lynn's
2026-02-17 re-open note) that it wants to revisit FDA inclusion
post-STU2 rather than leave it deferred. Disposition A satisfies the
reporter directly but bakes a flat "USCDI+ and FDA" pairing into a
paragraph that will keep accruing additions with each ballot cycle.
Disposition C is defensible only if the WG genuinely intends the IG to
remain USCDI+-only for the foreseeable future, which the re-open
contradicts. Disposition B delivers the substantive change the reporter
wants (explicit FDA use-case acknowledgement) **and** restructures
section 2.1 so future use-case additions are a one-line entry rather
than another scope-statement edit — which directly serves a workgroup
that clearly attracts "please also list use case X" requests.
Implementer impact is essentially nil (narrative only), and the
restructure is cheap to execute before STU3 work begins in earnest.
