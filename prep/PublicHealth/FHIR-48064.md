## Summary

Keith Boone reports that the **Unit Inclusion Criteria** section of the
NHSN Digital Quality Measure (dQM) Reporting IG does not enumerate
**burn beds** as an eligible bed/unit category. The ticket asks that
burn beds be added to the inclusion criteria so that facilities with
burn units can be represented consistently in NHSN dQM reporting. No
cross-referenced Jira tickets, Zulip threads, or GitHub items were
returned by FhirAugury, so the request stands on its own. A prior
disposition of *Considered for Future Use* (2024-11-07, Geimer/Gaunt
38-0-0) was reverted on 2025-07-09 and the ticket was reopened
post-STU1 publication.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Add **burn beds / burn unit** as an entry in the *Unit Inclusion
Criteria* of the NHSN dQM Reporting IG. Concretely, extend the value
set (or code list) that enumerates eligible inpatient unit types used
by the Bed Capacity profile(s) to include a burn-unit concept
(reusing, where possible, an existing code such as NHSN's own
location/unit taxonomy for burn critical care / burn ward, or SNOMED
CT concepts for burn unit / burn care location). Update the narrative
page that describes the inclusion criteria, any example
`Location`/bed-capacity instances, and conformance tests to exercise
the new code.

#### Justification

The request is a targeted, additive change: burn beds are a
well-recognized inpatient resource category that facilities already
report to NHSN outside of FHIR, and omitting them leaves a gap in
situational-awareness reporting for hospitals with burn units. Adding
a code to an open (extensible) value set is backward-compatible and
consistent with how the IG already differentiates ICU, med/surg, and
other specialty beds. Accepting as requested also honors the workgroup
commitment in the reverted disposition to "review the criteria with
the NHSN Bed Capacity team and add burn beds if appropriate."

---

### Disposition B: Alternative Approach

#### Proposal

**Defer the substantive change to the forthcoming US Realm
Situational Awareness IG** and, in the NHSN dQM Reporting IG, make the
Unit Inclusion Criteria value set explicitly extensible with a
documented extension point (e.g., bind as `extensible` rather than
`required` on the unit-type element, and add a non-normative note
pointing implementers to the Situational Awareness IG for the
authoritative bed-type taxonomy). No new burn-bed code is added in the
dQM IG itself; instead the IG is restructured so it does not need to
re-open each time a new bed category is identified.

#### Justification

The July 2025 comment signals that "all the bed capacity related
content will likely be moving to its own IG (a US Realm Situational
Awareness IG)," and the authors of that IG will own the canonical
bed-type list. Adding burn beds in the dQM IG now risks a short-lived
fix that will be superseded — and potentially diverge — once the
Situational Awareness IG lands. Making the binding extensible is a
smaller, forward-compatible change that unblocks burn-unit facilities
(via local extensions) without pre-empting the Situational Awareness
IG's modeling choices, and avoids a second round of breaking updates
when content migrates.

---

### Disposition C: Decline

#### Proposal

Mark the ticket *Not Persuasive / Out of Scope for this IG* on the
grounds that bed-capacity unit taxonomy is being moved to a separate
US Realm Situational Awareness IG, and the NHSN dQM Reporting IG will
simply consume whatever unit list that IG publishes. Recommend the
reporter re-file the change request against the Situational Awareness
IG once its project is launched.

#### Justification

Per the July 2025 comment, the workgroup has already signaled that
bed-capacity content is leaving this IG. Adding content that is known
to be migrating creates maintenance churn and sets up conflicting
sources of truth (dQM IG vs. Situational Awareness IG) for the same
unit taxonomy. A clean decline-with-redirect keeps the dQM IG focused
on measure reporting and routes the burn-bed question to the IG that
will actually own it.

---

### Recommendation

**Recommended disposition:** A

Burn beds are a concrete, well-scoped omission that multiple
implementers (burn centers, regional trauma systems) will hit
immediately, and the fix — adding one concept to an enumerated unit
list — is low-risk, additive, and backward-compatible. Deferring to a
not-yet-chartered Situational Awareness IG (Disposition B or C) leaves
real reporters without a way to represent burn capacity through at
least the next publication cycle, and the July 2025 reopen explicitly
reverses the earlier "future use" deferral. If the bed-capacity
content does later migrate, the added concept moves with it at
effectively zero cost; the incremental work of adding burn beds now is
far smaller than the operational cost of leaving them out. The
workgroup should still coordinate the chosen code (NHSN taxonomy vs.
SNOMED CT) with the NHSN Bed Capacity team before balloting, as the
reverted resolution anticipated.
