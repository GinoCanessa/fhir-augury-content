### Disposition A: Accept As Requested

#### Proposal

Remove the MedMorph reference from the US CCRR IG's index/home page.
Concretely:

1. In the US CCRR IG source repo, locate the index page authoring file —
   typically `input/pagecontent/index.md` (or `index.xml` / `input/pages/index.md`
   depending on the IG template in use).
2. Delete the prose paragraph(s) and any cross-reference link, image, or
   diagram caption that names **MedMorph** (or its full form, *Making EHR
   Data More available for Research and Public Health*).
3. If MedMorph is also linked from the IG's `sushi-config.yaml` `menu`, the
   IG `dependsOn`/`dependencies` list, or a dedicated "background" page that
   the index links to, evaluate whether those references are *also* now
   deprecated; remove them in the same PR if they are, otherwise leave them
   and note them in the commit message.
4. Re-build the IG locally (`./_genonce.sh` or `_updatePublisher` +
   `_genonce` per the IG-publisher template) to confirm there are no broken
   anchors or dangling links left over from the removed prose.
5. Open a PR against the US CCRR repo's main branch, reference FHIR-54581 in
   the PR title and body, and request review by the Public Health work group
   maintainers (per the IG's `CODEOWNERS`).

#### Justification

This is exactly what the ticket asks for and exactly what a "Technical
Correction" disposition is designed to handle: outdated narrative on a
publicly visible IG page is removed so readers do not infer a relationship
that is no longer current. The change is editorial/non-normative, touches no
conformance artifacts, has no cardinality or terminology impact, and cannot
break downstream implementers — the only risk is leaving stale anchors, which
the local build step in the proposal catches. The reporter is the appropriate
authority on the US CCRR IG's narrative content, and the comment trail shows
the description has been deliberately preserved through an accidental edit,
which signals the reporter is confident in the request as worded.

---

### Disposition B: Alternative Approach

#### Proposal

Instead of deleting the MedMorph reference outright, **re-frame** it as
historical context. On the index page, replace the current MedMorph paragraph
with a short note that explicitly marks the relationship as deprecated and
links the reader to the current Public Health reporting approach for cancer
registries — e.g.:

> *Earlier drafts of this IG referenced the MedMorph framework as a
> transport/orchestration option. That alignment is no longer current; see
> [Public Health reporting]({link}) for the recommended approach.*

If the workgroup confirms MedMorph genuinely has no remaining relationship to
US CCRR, fall back to Disposition A. If there is *any* residual alignment
(e.g., a knowledge artifact or a reporting workflow that US CCRR still
inherits from MedMorph), keep the reference but mark it `deprecated` with a
`<div class="deprecated">…</div>` block (or the equivalent IG-template
callout) and a target removal version.

#### Justification

Silent deletion of a public reference can confuse readers who have already
cited or bookmarked the IG's relationship to MedMorph (implementers,
jurisdictional public-health programmes, and tooling vendors who track
"reporting frameworks"). A deprecation callout preserves discoverability of
the change, gives a single release cycle of warning, and is consistent with
how other HL7 IGs (e.g., the case-reporting and genomics-reporting families)
handle removed alignments. The trade-off versus Disposition A is one extra
release of stale-but-flagged narrative; the gain is a clearer audit trail
for downstream readers.

---

### Disposition C: Decline

#### Proposal

Decline the request as submitted on the grounds that it does not provide
enough specificity to act on safely: the ticket says "MedMorph reference"
(singular) on "the index page", but does not identify which paragraph,
which link, or which version of the IG is in scope, and does not state
*why* the reference is now considered deprecated (programmatic alignment
ended? MedMorph IG itself superseded? scope of US CCRR narrowed?).
Request that the reporter re-file with (a) the exact text/link to be
removed, (b) the rationale for treating the reference as deprecated, and
(c) confirmation that no MedMorph-derived artifact (profile, value set,
example) remains in the IG that would be orphaned by the removal.

#### Justification

This is defensible **only** as a request-for-clarification, not as a
substantive "won't fix". The ticket is a one-sentence narrative cleanup
against the reporter's own IG, so the clarification cost is low and the
correction is presumed correct. Declining outright would be hard to justify
to the workgroup, which is why this disposition is offered for completeness
rather than as a serious alternative.

---

### Recommendation

**Recommended disposition:** **A — Accept As Requested.**

This is a textbook editorial Technical Correction: a single deprecated
reference on a single narrative page, raised by the IG's own reporter, with
no conformance impact, no cross-referenced discussion suggesting controversy,
and no linked tickets or PRs implying a broader programme of work. The
correct response is to remove the reference, rebuild the IG, and open a PR.
Disposition B's deprecation-callout pattern is genuinely nicer for readers,
but it is over-engineered for a single sentence on an index page and would
add a release cycle of delay for negligible benefit. Disposition C is only
appropriate if, on attempting the removal, the editor discovers that
"MedMorph reference" is ambiguous (e.g., several paragraphs reference it) —
in which case the request-for-clarification should be filed as a comment on
this ticket rather than as a decline. The Public Health work group should
adopt **A** and assign an editor against the US CCRR repo to land the change
in the next IG build.
