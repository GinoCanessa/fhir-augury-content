## Summary

The reporter notes that the SANER (Situation Awareness for Novel Epidemic
Response) implementation guide's "Alternate Criteria" extension is published
without a worked example, making it harder for implementers to know how to
populate the extension correctly when defining or computing measures. The
ticket is a documentation/example request rather than a normative change to
the extension's structure or semantics. No related Jira tickets, Zulip
threads, or GitHub PRs/issues were surfaced via cross-reference or search,
so the request stands on its own.

The ticket has already been triaged by the Public Health work group and
resolved as **Not Persuasive** on 2024-04-11. Keith Boone initially indicated
(2024-02-29) that an example would be added, and David Pyke subsequently
closed the loop (2024-04-11) with "Exists already," with the formal
resolution text "Example already exists." This review packet therefore
exists primarily to confirm the existing disposition and document the
rationale for the workgroup record.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Add a concrete, narrative example to the SANER IG page that documents the
Alternate Criteria extension. The example would:

- Show a `Measure` (or `MeasureReport`) instance in which a criterion
  uses the Alternate Criteria extension to declare an equivalent
  criterion expressed in a different language/format (e.g., a CQL
  expression alongside an alternate FHIRPath or text-based statement).
- Render the example both as XML and JSON in the IG, following the
  pattern used elsewhere in SANER for extension examples.
- Cross-link the example from the extension's StructureDefinition page
  so it is discoverable from the extension definition itself.

#### Justification

The request is purely additive documentation — adding an example does
not change any normative artifact and is consistent with the IG's
existing practice of shipping at least one worked example per
non-trivial extension. Implementer-facing examples directly reduce
mis-use of the extension and lower the barrier to conformance.

---

### Disposition B: Alternative Approach

#### Proposal

Confirm that the example **already exists** (as both reviewers concluded
in 2024) and instead invest the editorial effort in *discoverability*:

- Add an explicit "Examples" callout/link to the Alternate Criteria
  extension's StructureDefinition page pointing at the existing example
  instance(s).
- Add a short usage paragraph next to the example that walks the reader
  through which slice/value populates which element, so the existing
  example is self-explanatory rather than requiring the reader to
  reverse-engineer it.

#### Justification

The closing comments indicate the underlying example is already in the
IG; the original gap was therefore one of *findability*, not of missing
content. Improving cross-linking and adding an explanatory caption
addresses the reporter's actual pain point with less editorial churn
than authoring a new example, and avoids introducing a second example
that could drift from the canonical one.

---

### Disposition C: Decline

#### Proposal

Close the ticket as **Not Persuasive** on the basis that the example
the reporter asked for already exists in the published SANER IG and the
reporter himself has confirmed this on the ticket. No further IG change
is required.

#### Justification

This is exactly what the workgroup did on 2024-04-11. With both the
original reporter ("Exists already") and the work-group commenter
("We will add an example for alternate criteria") in agreement that the
content is present, there is no outstanding gap to address. Spending
further cycles on the ticket would not improve the specification.

---

### Recommendation

**Recommended disposition:** C

The ticket is already resolved as Not Persuasive, and the reporter
themselves verified the example exists. Re-opening to author additional
content (Disposition A) would duplicate existing material; Disposition
B's discoverability improvements are reasonable but should be raised as
a fresh editorial ticket rather than a re-litigation of this one. The
cleanest path is to confirm the existing Not Persuasive disposition
and, if the workgroup feels the discoverability concern still has
merit, spin out a new ticket targeted specifically at cross-linking the
existing example from the Alternate Criteria StructureDefinition page.
