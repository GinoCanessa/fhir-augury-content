## Summary

Craig Newman (Public Health WG) is asking the eCR (Electronic Case Reporting,
FHIR) authors to revisit how discrete microbiology results — specifically
organism-detected and antimicrobial susceptibility observations — are
modelled in the IG so that the model aligns with the guidance the FHIR R5
base spec gives in the Diagnostics Module (§10.0.3.1.1, "discrete micro
results"). The ticket notes that R4 was silent on this, so the eCR IG
defined its own approach, but R5's recommended pattern differs from what is
currently in eCR. The reporter explicitly asks the authors to coordinate
with the Orders & Observations (OO) WG about the latest R6 thinking on
complex lab results before changing the model, so that any rework is
forward-compatible with R5/R6 rather than diverging again.

The request sits in the middle of an active modelling debate that spans
several related tickets. [FHIR-39168](https://jira.hl7.org/browse/FHIR-39168)
(also filed by Craig Newman, Persuasive with Modification) asked OO to bring
the OO-authored "discrete microbiology / reflex / parent-child" guidance
into R5; [FHIR-40540](https://jira.hl7.org/browse/FHIR-40540) tracks the fact
that the agreed FHIR-39168 changes had not yet been applied and need to land
during R5 QA. In parallel, [FHIR-44631](https://jira.hl7.org/browse/FHIR-44631)
("Recommend component for AMR values", Resolved – Not Persuasive) argued
that AMR drug + S/I/R values should be **components** of the organism
Observation rather than separate member Observations — the opposite of the
direction this ticket is leaning toward. Within eCR itself,
[FHIR-54439](https://jira.hl7.org/browse/FHIR-54439) ("Susceptibility
Modelling", same ballot, Sarah Gaunt) and
[FHIR-54420](https://jira.hl7.org/browse/FHIR-54420) ("Value set in eICR
Organism Susceptibility Observation.code is too broad", Ulrike Merrick)
are open and touch the same profile(s); FHIR-54439 in particular suggests
removing the Susceptibility Observation and folding it into components on
the eICR Organism Detected Observation, which is one of the patterns the
R5 guidance describes. These together imply the disposition for FHIR-54348
should not be made in isolation from FHIR-54439 / FHIR-54420 / FHIR-40540
and the OO base-spec work.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Re-model the eICR micro / susceptibility profiles to mirror R5
Diagnostics Module §10.0.3.1.1. Concretely, in the eCR (FHIR) IG:

- Restructure **eICR Organism Detected Observation** to act as the parent
  of a discrete-micro group, using `Observation.hasMember` to point to
  child Observations (consistent with the OO parent/child guidance
  produced under FHIR-39168 / FHIR-40540, applied in R5 §10.1.5.3
  Observation Grouping).
- Keep **eICR Organism Susceptibility Observation** as a member-level
  child Observation referenced from the parent via `hasMember`, not as a
  set of `component`s on the organism (preserving OO's decision in
  FHIR-44631 to retain the member-Observation pattern).
- Adjust the IG narrative and example bundles to follow §10.0.3.1.1's
  worked example, and add a "compatibility with R5 / R6" note in the
  profile pages so implementers can see the alignment.
- Open a coordination request with OO confirming the §10.0.3.1.1 model
  is stable for R6 before publishing, and co-disposition with
  FHIR-54439 and FHIR-54420.

#### Justification

This is exactly what the reporter asked for and has the strongest
forward-compatibility story: it tracks the base spec, follows the OO
decisions captured in FHIR-39168 / FHIR-40540 / FHIR-44631, and avoids
introducing a third eCR-specific micro pattern alongside the two examples
already noted in FHIR-32860. Aligning now (during ballot reconciliation)
costs less than aligning after publication, when implementers will have
encoded the current model. The reporter explicitly flagged that R4 had no
guidance, so the current eCR model is not "wrong" historically — but
since R5 now supplies guidance, eCR has a clear target to converge on.

---

### Disposition B: Alternative Approach

#### Proposal

Do not re-model the profiles in this ballot cycle. Instead:

- Add a normative-flavoured **"Forward compatibility with FHIR R5/R6"**
  narrative section to the eCR (FHIR) IG that points to R5 Diagnostics
  Module §10.0.3.1.1 and explains where the current eCR micro /
  susceptibility model differs and where it agrees.
- Where the divergence is mechanical (element naming, slice ordering,
  example structure), apply minimum-impact tweaks now so the wire-format
  examples line up with §10.0.3.1.1 without changing element cardinality
  or the existence of the Susceptibility Observation profile.
- File a follow-up ticket (or convert FHIR-54348 to "Considered for
  Future Use") to track full structural realignment in the next eCR
  release, conditional on FHIR-40540 actually landing in the R5 base
  spec and on OO confirming the R6 direction.

#### Justification

FHIR-40540 explicitly says the FHIR-39168 narrative "has not yet been
applied" to the base spec, and FHIR-46932 documents that the broader
panel-of-Observations question lacks international agreement. Re-modelling
eCR against a target that is itself in flight risks doing the work twice
and creating yet another variant. A documentation-plus-minor-tweaks
disposition discharges the spirit of the request, gives implementers an
explicit migration story, and avoids breaking existing eCR profile
consumers in mid-cycle. It also gives eCR room to align with whichever
direction OO ultimately confirms for R6 (member Observations vs.
components — see FHIR-44631) without committing prematurely.

---

### Disposition C: Decline

#### Proposal

Resolve as **Not Persuasive** with the rationale that the eCR (FHIR) IG
targets FHIR R4 / R4 US Core, where no equivalent base-spec guidance
existed at the time the IG's micro / susceptibility profiles were
designed; that R5's §10.0.3.1.1 model is informational for an R4-based
IG; and that base-spec realignment is properly handled in a future
R5/R6-targeted version of the eCR IG rather than in the current ballot.
Cross-reference FHIR-39168, FHIR-40540, FHIR-44631, FHIR-46932,
FHIR-54439, and FHIR-54420 in the resolution comment so the issue is
preserved as background for the next major eCR revision.

#### Justification

Defensible on three grounds. First, eCR's current FHIR version is R4 and
the cited guidance is R5 / forward-looking. Second, FHIR-40540 shows the
target guidance is not yet stably applied even in the base spec, and
FHIR-46932 shows that the wider community has not yet reached agreement
on the panel/grouping pattern — declining now avoids burning ballot
cycles on a moving target. Third, FHIR-44631 (Not Persuasive) shows that
OO has already rejected one popular interpretation of "align with the
base spec" for AMR values, so any "alignment" done now risks needing to
be undone. The trade-off is explicit: implementers continue to encode
against the current eCR model for this cycle.

---

### Recommendation

**Recommended disposition:** B

The reporter's own wording ("should that model be applied here? Please
coordinate with OO WG regarding the latest R6 thoughts") frames this as
a question to be settled with OO, not a finished proposal. Disposition A
is the right *long-term* destination, but acting on it in this ballot
risks chasing FHIR-40540 (which says the base-spec guidance is not yet
applied) and pre-empting the parallel eCR tickets FHIR-54439 (proposes
collapsing the Susceptibility Observation into components — the very
direction OO declined in FHIR-44631) and FHIR-54420 (binding scope).
Disposition C under-delivers on a request the reporter has framed as
forward-looking and important. Disposition B threads the needle: it
acknowledges the divergence in narrative, makes any cheap mechanical
alignments now, sets up an explicit follow-up tied to FHIR-40540 and the
OO R6 decision, and keeps FHIR-54348, FHIR-54439, and FHIR-54420 in a
single coordinated work package so the eCR Susceptibility / Organism
profiles are restructured exactly once. The Public Health WG should
agree co-disposition with OO before closing.
