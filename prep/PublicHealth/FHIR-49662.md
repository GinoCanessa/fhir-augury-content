## Summary

The reporter has identified broken outbound links in the US Vital Records
Death Reporting (VRDR) FHIR Implementation Guide. Specifically, Section 4.2
links to the "NCHS Vital Records FHIR Messaging Implementation Guide" at
`https://build.fhir.org/ig/nightingaleproject/vital_records_fhir_messaging_ig/branches/main/index.html`,
which currently fails to resolve. A follow-up comment from the reporter
clarifies that Section 5.2 contains additional links to the same (or a
related) messaging IG that does not appear to exist either. The request is
to either update the links to a valid location or remove them. No related
Jira tickets, Zulip discussions, or GitHub items were surfaced by the
cross-reference index for this ticket.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Update both Section 4.2 and Section 5.2 of the VRDR IG narrative to point at
a working location for the NCHS Vital Records FHIR Messaging IG. Concretely:

- Confirm the current canonical home of the messaging IG with the
  nightingaleproject maintainers (it likely moved off
  `build.fhir.org/ig/nightingaleproject/vital_records_fhir_messaging_ig/branches/main/`
  to either a published HL7 ci-build path under
  `build.fhir.org/ig/HL7/vrdr-messaging/` or a versioned `hl7.org/fhir/us/...`
  URL).
- Replace the existing `<a href="…/branches/main/index.html">` references in
  the Section 4.2 and Section 5.2 narrative pages with the confirmed URL.
- If a stable published URL exists, prefer it over a current-build URL to
  avoid recurrence of this exact problem.

#### Justification

The reporter's request is purely editorial and bounded: two broken outbound
links in narrative sections of a published IG. Fixing the links preserves
the author's clear original intent — to point readers from the VRDR content
spec to the companion messaging-envelope IG — and is consistent with HL7's
general policy that published IGs should not contain known dead links.
Switching from a `branches/main` URL to a stable build or published URL
also reduces ongoing maintenance burden.

---

### Disposition B: Alternative Approach

#### Proposal

Remove the broken hyperlinks but retain the prose context. Replace each
`<a href="…">NCHS Vital Records FHIR Messaging Implementation Guide</a>`
occurrence with the bare title text (and, if appropriate, a brief sentence
such as "(publication pending — link will be added once the messaging IG is
available)"). Add a tracking note in the IG's editorial backlog so a future
publication cycle can re-introduce the link once the messaging IG has a
stable home.

#### Justification

If the companion messaging IG genuinely does not yet have a hosted location
(as the reporter's follow-up comment suggests for Section 5.2), then
pointing readers at any URL — current-build or otherwise — is misleading.
Removing the link while retaining the contextual prose preserves the
informational value of the section without falsely advertising a resource
that doesn't exist. This avoids a second broken-link ticket if the
nightingaleproject build URL changes again before publication.

---

### Disposition C: Decline

#### Proposal

Decline as "Not Persuasive — out of scope for the current VRDR ballot
cycle." Acknowledge the broken link but defer the fix to the next normal
publication of the VRDR IG, on the grounds that the change is editorial
and does not affect normative content, conformance, or implementer
behavior.

#### Justification

The change has no normative impact — it is a narrative hyperlink fix.
Workgroups sometimes batch purely editorial corrections rather than
processing them as individual ballot dispositions, particularly when the
fix depends on an external project (the messaging IG) reaching a stable
publication state. Declining here is only defensible if a parallel
editorial mechanism exists to actually correct the link before the next
publication; otherwise it leaves a known defect in the spec.

---

### Recommendation

**Recommended disposition:** A (with a fallback to B if no stable URL exists)

This is a low-risk, high-value editorial fix in narrative sections of a
published VRDR IG, and the reporter has been specific about both occurrences
(Sections 4.2 and 5.2). The work group should accept the change and update
both links to whatever the nightingaleproject / NCHS team currently
considers the canonical location of the Vital Records FHIR Messaging IG —
ideally a published or HL7-hosted ci-build URL rather than a transient
`branches/main` path. If, on investigation, no such location exists yet,
fall back to Disposition B and remove the hyperlinks while keeping the
surrounding prose, so the IG does not advertise a resource that cannot be
reached. Disposition C should only be used if there is a confirmed plan
to fix the links via an editorial pass before the next publication, since
leaving known broken links in a published IG is a poor reader experience.
