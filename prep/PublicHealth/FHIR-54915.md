## Summary

The reporter is asking the Public Health work group to correct the
descriptive text shown in the "yellow ribbon" (the standard IG
publish-box banner) of the **US Central Cancer Registry Reporting
Content IG**. Two issues are flagged:

1. The current banner text reads "Making EHR Data MOre available for
   Research and Public Health (MedMorph) Central Cancer Registry
   Reporting Content IG". The capitalization "MOre" appears to be a
   typo and should be "More".
2. The parenthetical "(MedMorph)" reference should be removed because,
   per the reporter, STU 1→2 changes already removed references to the
   MedMorph Reference Architecture IG, so the banner is now
   inconsistent with the rest of the publication.

The requested final text is:

> This page is part of the Making EHR Data More available for Research
> and Public Health Central Cancer Registry Reporting Content IG

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Update the IG's banner/title text exactly as the reporter requests:

- Change "Making EHR Data **MOre**" → "Making EHR Data **More**"
  (fix the stray uppercase `O`).
- Remove the parenthetical "**(MedMorph)**" so the banner reads:
  "This page is part of the Making EHR Data More available for
  Research and Public Health Central Cancer Registry Reporting Content
  IG".

Concretely, this is an edit to the IG's `ImplementationGuide.title`
and/or `ImplementationGuide.description` (plus any
publisher/ig.ini/sushi-config display-name strings that feed the
yellow-ribbon banner) in the **US Central Cancer Registry Reporting
Content IG** source repo. No StructureDefinitions, profiles,
extensions, value sets, or examples are touched.

#### Justification

This is a Technical Correction with no normative impact: it fixes a
typographical capitalization error and removes a stale reference to
"MedMorph" that is inconsistent with the STU 1→2 decision to drop
references to the MedMorph Reference Architecture IG. Banner text is
descriptive metadata only; correcting it improves reader comprehension
and aligns the IG's self-description with the rest of the
publication. There is no community disagreement on record (no Zulip
threads, no linked tickets) and the reporter's proposed wording is
internally consistent with the IG's current scope.

---

### Disposition B: Alternative Approach

#### Proposal

Accept the typo fix ("MOre" → "More") immediately, but **retain a
softened MedMorph reference** rather than dropping the parenthetical
entirely. Replace the current "(MedMorph)" with a sentence elsewhere
on the home page (e.g., in the IG's introduction / scope section)
along the lines of:

> "This IG was originally developed under the MedMorph initiative; as
> of STU 2, it no longer normatively depends on the MedMorph Reference
> Architecture IG."

The yellow-ribbon banner itself would then read exactly as the
reporter requests, but historical lineage is preserved in prose
elsewhere so implementers searching for "MedMorph" still land on the
right IG.

#### Justification

The reporter is correct that the banner shouldn't claim a normative
relationship to MedMorph that no longer exists. However, a number of
implementers, public-health agencies, and prior publications refer to
this work as "MedMorph CCRR", and silently dropping the term from all
front-matter risks discoverability regressions. Keeping the typo fix
in the banner while preserving a one-line lineage note in the IG
introduction satisfies both correctness and continuity, at minimal
editorial cost. Trade-off vs. Disposition A: slightly more work (one
extra paragraph to author and review) but lower risk that downstream
readers lose the historical MedMorph thread.

---

### Disposition C: Decline

#### Proposal

Decline the request, on the grounds that the IG's title /
yellow-ribbon banner is generated from `ImplementationGuide.title` and
related metadata that has already been balloted/published, and that
re-opening the banner text for a cosmetic change is out of scope for
this STU cycle. Defer the change to the next substantive publication.

#### Justification

This is the weakest of the three options and is included for
completeness. The case for declining is essentially "process /
timing": the IG has already been published with the current banner,
and a post-publication cosmetic change requires a republish. That
said, the request is specifically labeled as a **Technical
Correction**, the work group's lightweight track for exactly this kind
of typo / metadata fix, so the procedural objection is weak. There is
no normative downside to making the change.

---

### Recommendation

**Recommended disposition:** A

This is a clear, low-risk Technical Correction with two
straightforward sub-changes: a capitalization fix ("MOre" → "More")
and removal of a parenthetical that is factually inconsistent with the
IG's current STU 2 scope (per the reporter, references to the
MedMorph Reference Architecture IG were already removed in the STU
1→2 transition). There is no Zulip discussion, no linked ticket, and
no cross-referenced GitHub work that argues against the change, and
the reporter's proposed wording is internally consistent and ready to
apply. Disposition B's lineage-preserving variant is reasonable and
the work group may prefer it on discoverability grounds, but it adds
editorial work without changing the banner outcome. Disposition C is
not recommended — declining a self-described typo fix on a
Technical Correction ticket would be hard to justify to the reporter
or to downstream implementers.
