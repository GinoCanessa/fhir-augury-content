## Summary

The reporter flags a minor grammatical issue in narrative text within the
**EU Hospital Discharge Report (FHIR)** Implementation Guide, specifically
in the section that introduces the Medication profile. The current sentence
reads: *"All data elements in Medication profile are optional"*. The
reporter notes this is grammatically awkward and recommends inserting the
definite article **"the"** so it reads: *"All data elements in **the**
Medication profile are optional"*. This is a pure editorial / technical
correction with no normative impact on the profile, conformance rules, or
any artifact content. No related Jira tickets, Zulip discussions, or
GitHub items were surfaced by the cross-reference index.

## Details

**Description:**

> All data elements in Medication profile are optional

This sentence is grammatically awkward. You could go multiple ways.
Suggested:

> All data elements in **the** Medication profile are optional

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Apply the requested edit verbatim. In the EU Hospital Discharge Report
(FHIR) IG narrative that introduces the Medication profile, change the
sentence:

> All data elements in Medication profile are optional

to:

> All data elements in the Medication profile are optional

The change is restricted to inserting the definite article **"the"**
between *"in"* and *"Medication"*. No StructureDefinition, ElementDefinition,
cardinality, binding, invariant, or example needs to change. The fix
should be applied wherever this exact sentence appears in the IG's
authoring source for the Medication profile (typically the profile's
intro/description markdown or the equivalent narrative block in the
profile authoring file).

#### Justification

This is the lowest-risk, highest-fidelity outcome: it directly addresses
the reporter's grammatical concern without altering meaning or
conformance. The corrected sentence is unambiguous, reads naturally in
English, and matches typical IG prose style (e.g., "the Patient
profile", "the Encounter profile"). As a Technical Correction it is
appropriate to take exactly as filed.

---

### Disposition B: Alternative Approach

#### Proposal

Reword the sentence more substantively to be both grammatical *and* more
informative for implementers. For example:

> All data elements in this profile are optional unless otherwise
> constrained below.

Or, if the intent is purely about minimum cardinality on the Medication
resource as profiled here:

> All elements in this profile have a minimum cardinality of 0; see the
> differential below for any must-support flags or fixed values.

Apply the chosen rewording in the same narrative location identified in
Disposition A.

#### Justification

The original sentence, while now grammatical with "the" inserted, can
still mislead readers: "all elements are optional" is technically true
of cardinality but obscures must-support obligations, slicing, and any
fixed values that may exist on the profile. A short reword removes the
ambiguity for implementers at essentially the same editorial cost.
Trade-off: it goes beyond what the reporter literally asked for and
should be confirmed with the IG editor to make sure no additional
nuance (e.g., must-support semantics in the EU HDR) is misrepresented.

---

### Disposition C: Decline

#### Proposal

Decline the change as **Not Persuasive – Editorial / No Impact**, on
the grounds that the existing sentence, while slightly informal, is
unambiguous and does not affect conformance, validation, or
implementation. Leave the IG narrative as-is.

#### Justification

Pure prose-polish changes consume editorial cycles and re-publication
overhead with no benefit to implementers or conformance. If the
workgroup is operating under tight editorial bandwidth and is batching
typo fixes, this one could reasonably wait or be dropped. However, this
justification is weak in isolation because the cost of the fix is
trivially small and the reporter is correct on the merits.

---

### Recommendation

**Recommended disposition:** A

This is a textbook Technical Correction: the reporter has identified a
real grammatical defect and proposed the minimal, exact fix. Applying
the change verbatim (insert "the") incurs essentially zero risk, no
normative impact, and no implementer burden, while honoring the
reporter's contribution. Disposition B is defensible if the IG editors
already want to revisit the sentence for clarity about must-support vs.
cardinality, but it expands scope beyond what was filed and should not
be bundled in without editor sign-off. Disposition C is not justified
given how cheap the fix is. Accept as requested.
