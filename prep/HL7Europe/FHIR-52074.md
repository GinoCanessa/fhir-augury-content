## Summary

FHIR-52074 is a Technical Correction filed by Giorgio Cangioli against the
**EU Hospital Discharge Report (FHIR)** specification. It asks the HL7 Europe
work group to fix the IP / copyright statement that currently appears on the
HL7 EU terminologies, aligning it with the disposition already accepted on
the related ticket [FHIR-50968](https://jira.hl7.org/browse/FHIR-50968)
("fix copyrigth wording") against the EU Base and Core (R4) specification.

[FHIR-50968](https://jira.hl7.org/browse/FHIR-50968) was resolved as
*Persuasive* / *Applied* with the agreed wording:

> "Copyright HL7 Europe. Licensed under creative commons public domain
> (CC0 1.0)."

That replaces the previous nonsensical formulation
("all rights reserved Creative Commons License"). FHIR-52074 simply asks
that the same fix be propagated to the IP statement attached to the HL7
EU terminologies used by the Hospital Discharge Report IG (and, by
implication, anywhere else the same incorrect wording appears in HL7
Europe terminology artifacts).

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Apply the FHIR-50968 wording verbatim to the IP / copyright statement of
the HL7 EU terminologies referenced by the EU Hospital Discharge Report
IG. Concretely:

- In every HL7 Europe `CodeSystem` and `ValueSet` resource used by the
  Hospital Discharge Report IG whose `copyright` element currently reads
  some variant of *"all rights reserved Creative Commons License"*,
  replace the value with:

  > "Copyright HL7 Europe. Licensed under creative commons public
  > domain (CC0 1.0)."

- Update any IG-level pages (e.g., the IG `ImplementationGuide`
  resource's `copyright`, the rendered "Copyright" / "License" page,
  and any terminology landing pages) that repeat the old phrasing to
  the same wording.
- Bump the IG / terminology package patch version to reflect the
  technical correction and regenerate the published artifacts.

#### Justification

This is the minimum-surprise outcome and is exactly what the reporter
asked for. FHIR-50968 has already been dispositioned *Persuasive* and
*Applied* against EU Base and Core with the same fix; using the
identical wording here keeps the IP statement consistent across the
HL7 Europe FHIR family and avoids implementers seeing two different
"correct" license strings. Because the change is purely textual in
metadata (`copyright` elements and rendered narrative), it carries no
conformance impact on instances or operations.

---

### Disposition B: Alternative Approach

#### Proposal

Treat the IP statement as a single source-of-truth string maintained
centrally by HL7 Europe, rather than fixing it in-place per artifact:

- Define the canonical CC0 wording once (e.g., in the EU Base and
  Core copyright page or a shared HL7 Europe template) and reference
  it from the Hospital Discharge Report IG and from every EU
  terminology resource via include / template substitution rather than
  hard-coded `copyright` text.
- For the immediate ticket, still apply the FHIR-50968 wording to the
  EU terminologies, but additionally add a publishing-process note
  (and, if practical, a tooling check) that flags any new HL7 Europe
  terminology artifact whose `copyright` deviates from the canonical
  string.

#### Justification

FHIR-52074 is a follow-on to FHIR-50968 precisely because the same
incorrect wording leaked into more than one place. A purely textual
fix here will work for today, but the same drift will recur the next
time someone copies an old template. Centralising the IP statement
(or at least adding a lint) costs more up-front but stops the work
group from having to re-litigate the wording every time a new HL7 EU
IG or terminology bundle is published.

---

### Disposition C: Decline

#### Proposal

Reject FHIR-52074 as redundant on the grounds that FHIR-50968 already
documents the agreed wording, and direct the HL7 Europe terminology
maintainers to apply that disposition to their own publications without
opening a separate Jira ticket per IG.

#### Justification

The substantive decision (what the copyright sentence should say) has
already been made in FHIR-50968. If the work group's view is that
propagating an already-agreed editorial fix across sibling artifacts
is publishing hygiene rather than a Jira-tracked change, declining
FHIR-52074 keeps the change-request backlog focused on substantive
technical decisions. The risk is that without a tracked ticket the
fix may not actually land in the next Hospital Discharge Report
release.

---

### Recommendation

**Recommended disposition:** A

The reporter's ask is narrow, unambiguous, and a copy of an already-
*Persuasive* / *Applied* disposition on a sibling specification. The
correct license string for HL7 Europe content has been agreed in
FHIR-50968; the only remaining work is to make sure the same string
appears on the HL7 EU terminologies referenced by the Hospital
Discharge Report IG. Disposition A delivers that with no conformance
risk and keeps the IP statement consistent across the HL7 Europe FHIR
family. Disposition B is attractive as a follow-up — the work group
should consider opening a separate publishing-tooling ticket to make
the canonical CC0 string the single source of truth — but it should
not block this technical correction. Disposition C understates the
value of an explicit, tracked fix in the IG's change log and risks the
correction being missed at publish time.
