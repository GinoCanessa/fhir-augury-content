## Summary

The reporter (Bas van den Heuvel) asks the EU Imaging Study Report (R5) IG to
adopt a single, consistent approach to populating `Composition.type` and
`Composition.category`, and to align that approach with the other HL7 Europe
documents that participate in the EHDS space (notably EU Base/Core, EU
Laboratory, and other EHDS document IGs). The trigger is feedback from
Germany indicating that the imaging IG's current treatment diverges from how
sibling EHDS IGs handle these two header elements.

Although this ticket carries no cross-references in Augury, the
related-by-keyword neighbours make the substantive direction clear. The most
relevant prior decision is [FHIR-51171](https://jira.hl7.org/browse/FHIR-51171)
(*Composition: remove fixed code for type*, Applied / Persuasive with
Modification on the same imaging IG), where the working-group discussion
explicitly concluded that "to align with other EHDS IGs, `Composition.type`
should match `DiagnosticReport.code`, and `Composition.category` should match
`DiagnosticReport.category`" — `type` describes the document itself, `category`
classifies it. The companion EU Base ticket
[FHIR-55545](https://jira.hl7.org/browse/FHIR-55545) (*Composition: Location
of `.category:ehds-category`*, Applied / Persuasive with Modification) reached
the same conclusion from the EU Base/Core side: the `ehds-category` slice and
its FHIRDocumentTypeCodes-style coding belong on `Composition.type`, not on
`Composition.category`, because `category` is already used in EU Laboratory
for a more granular notion (specialty). [FHIR-51297](https://jira.hl7.org/browse/FHIR-51297)
(*Align on the use of general document fields*, Deferred) frames the broader
cross-IG alignment goal that this ticket is one slice of, and
[FHIR-51275](https://jira.hl7.org/browse/FHIR-51275) (*HL7 II: DiagnosticReport
vs Composition*, Applied) provides the design backdrop in which DiagnosticReport
is treated as the principal access mechanism while Composition carries
documentation metadata. FHIR-51298 is, in effect, the imaging IG's
follow-through ticket to apply that already-agreed alignment consistently and
confirm it against the German-side review.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

In the EU Imaging Study Report (R5) IG (`Composition` profile for the imaging
report), apply the alignment already adopted in FHIR-51171 / FHIR-55545
verbatim:

- Bind `Composition.type` to the same value set / preferred binding used by
  the sibling EHDS IGs for "what document is this" — i.e., the
  FHIRDocumentTypeCodes-style binding used by EU Base/Core's
  `ehds-category`-equivalent slice on `Composition.type`, narrowed (where
  needed) to the imaging-report-level LOINC code(s). Remove any remaining
  hard-coded local code for `Composition.type` (the reason FHIR-51171
  originally fired).
- Mirror `Composition.type` to `DiagnosticReport.code` for the same instance.
- Bind `Composition.category` to the same value set used for service/specialty
  classification across EHDS IGs (the DiagnosticReport.category pattern,
  matching how EU Laboratory uses `Composition.category` for laboratory
  specialty), and mirror it to `DiagnosticReport.category`.
- Add a short rationale paragraph in the imaging IG narrative explaining the
  `type` (= what the document is) vs `category` (= how it's classified) split
  and citing the EU Base/Core alignment.
- Cross-link this ticket's resolution to FHIR-51171 and FHIR-55545 in the IG
  changelog so the alignment story is discoverable.

#### Justification

This is precisely the conclusion the same workgroup already reached on the
two precedent tickets (FHIR-51171 on this IG, FHIR-55545 on EU Base), and
both were resolved Persuasive with Modification and have been Applied. The
"specific approach" the reporter refers to is therefore already on the
record — FHIR-51298 just asks the imaging IG to land it consistently. The
type/category split also matches IHE IDR's framing (FHIR-51275) and the
generalisation goal in FHIR-51297. Accepting as requested closes the loop on
the German review feedback without reopening a settled question.

---

### Disposition B: Alternative Approach

#### Proposal

Resolve FHIR-51298 by **publishing a single cross-IG alignment table** in EU
Base and Core (R5) and having the imaging IG (and any other EHDS document IG)
**reference** that table rather than redefining the bindings locally. The
table would cover, per EHDS document IG: the value set / preferred binding for
`Composition.type` and `Composition.category`, the matching
`DiagnosticReport.code` / `DiagnosticReport.category` bindings, the
`DocumentReference.type` and `DocumentReference.category` projections for
IHE-MHD, and any allowed local extensions. The imaging IG profile would then
constrain to the row of the table that applies to imaging reports and add no
new local bindings.

#### Justification

FHIR-51298 is one of several alignment tickets (FHIR-51297, FHIR-55545,
FHIR-51171, FHIR-55580, FHIR-51275). Treating it as a one-off imaging-IG fix
risks the same drift recurring the next time another EHDS IG ballots. A
shared, normative alignment table elevates the decision from "imaging IG
matches EU Base today" to "all EHDS document IGs are required to project the
same Composition/DiagnosticReport/DocumentReference header into MHD the same
way", which is the actual goal Bas voiced in FHIR-51297. Trade-off: more
authoring work in EU Base/Core and a coordination dependency between
publication cycles, but it pays back across every future EHDS document IG.

---

### Disposition C: Decline

#### Proposal

Mark FHIR-51298 as **Resolved – No Change**, with a pointer to FHIR-51171
(same IG, already Applied) and FHIR-55545 (EU Base, already Applied) as the
authoritative resolutions. Optionally close as a duplicate of FHIR-51171.

#### Justification

The ticket's wording ("we choose a specific approach… align this approach
with the other documents") describes an outcome that has, in substance,
already been delivered: FHIR-51171 removed the hard-coded `Composition.type`
in this IG and adopted the type/category split that matches EU Base, and
FHIR-55545 moved the `ehds-category` slice to `Composition.type` on the EU
Base side. If the German reviewers' concrete change requests are already
captured in those two tickets, opening a third change against the imaging IG
risks re-litigating a settled design and producing churn in the IG
between ballots without any new normative content.

---

### Recommendation

**Recommended disposition:** A

The workgroup has already done the design work on FHIR-51171 and FHIR-55545
and converged — across imaging, lab, and EU Base authors — on the
type-vs-category split with bindings that line up with `DiagnosticReport.code`
/ `.category`. FHIR-51298 is the German review's confirmation that this
alignment is the right one and a request to make sure it's actually applied
in the imaging IG's published artefacts. Disposition A is the cleanest
follow-through: it commits the imaging IG to the precedent without inventing
a new mechanism. Disposition B is genuinely attractive and worth raising in
the EHDS coordination call (it's the right long-term fix for the pattern
that's now repeated across at least four tickets), but it is a programme of
work, not something the imaging IG can land in this ballot cycle on its own;
it should be tracked through the deferred FHIR-51297 rather than gating
FHIR-51298. Disposition C undersells the value of an explicit "Applied"
status here — closing as duplicate would leave no auditable record that the
German feedback was specifically addressed for the imaging IG, which is
exactly what the reporter is asking for. Accept as requested, then keep the
larger alignment table conversation alive on FHIR-51297.
