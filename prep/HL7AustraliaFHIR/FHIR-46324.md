## Summary

The ticket requests that the AU Base IG provide first-class support, in the
Practitioner resource, for fellowship qualifications issued by Australian
specialist medical colleges. Specifically, the reporter (Brett Esler, mirroring
an earlier hl7au/au-fhir-base GitHub issue #463 originally raised by
@davidmckillop) asks for two things: (a) a definitive list of Australian
Specialist Medical Colleges to populate `Practitioner.qualification.issuer`,
and (b) a definitive list of fellowship qualification codes (e.g., FRACP,
FRCPA) issued by those colleges to populate `Practitioner.qualification.code`.
The authoritative source identified is the Australian Medical Council list of
specialist medical colleges. PAWG has previously agreed to extend the existing
`au-hl7v2-0360` ValueSet rather than introduce a wholly new binding, and a
draft branch (`cr-practitioner-qualification`) already publishes a
`ValueSet-au-v2-0360-extended` plus a new code system enumerating the medical
colleges. No cross-referenced Jira tickets were surfaced by the index;
the cross-reference index returns only FHIR artifact mentions
(`Practitioner.qualification` and `Practitioner.qualification.issuer`).

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Adopt the change already prototyped on the `cr-practitioner-qualification`
branch in `hl7au/au-fhir-base`:

1. Replace (or extend) the existing `ValueSet-au-hl7v2-0360` binding with
   `ValueSet-au-v2-0360-extended`, which composes the existing v2-0360 codes
   with the additional Australian fellowship codes.
2. Define a new HL7 AU `CodeSystem` enumerating the Australian Specialist
   Medical Colleges sourced from the AMC list
   (https://www.amc.org.au/accredited-organisations/assessment-and-accreditation-of-specialist-medical-programs/specialist-medical-colleges/),
   with each college's standard abbreviation as the code (e.g., `RACP`,
   `RCPA`, `RANZCO`).
3. Slice `Practitioner.qualification` in the AU Base `Practitioner` profile
   with a named slice (e.g., `australianCollegeFellowship`) that:
   - binds `qualification.code` to the extended ValueSet (preferred or
     extensible, matching the strength used elsewhere in AU Base), and
   - binds `qualification.issuer` (a `Reference(Organization)`) such that the
     referenced `Organization` is identified by a code from the new
     "Australian Specialist Medical Colleges" CodeSystem (typically via
     `Organization.identifier` with a system equal to that CodeSystem's
     canonical, since `qualification.issuer` is not directly codeable).
4. Mark FHIR-46324 Resolved → Persuasive once the branch is merged into the
   next AU Base release candidate.

#### Justification

This is exactly what PAWG has already agreed (3 Mar 2021 and 31 Mar 2021)
and what the assignee (Brett Esler) has implemented on a published draft
branch. The approach reuses existing v2-0360 semantics — preserving
implementer compatibility — while filling the gap that prompted the original
request. Treating colleges as a CodeSystem and surfacing them through
`Organization.identifier` (rather than inventing a new datatype) is
consistent with how FHIR represents identified institutions and avoids
breaking the typing of `qualification.issuer` as `Reference(Organization)`.

---

### Disposition B: Alternative Approach

#### Proposal

Keep `ValueSet-au-hl7v2-0360` unchanged and instead publish the Australian
fellowship codes and the medical-colleges enumeration as separate, externally
referenced terminology artifacts:

- A standalone `CodeSystem` and `ValueSet` for "Australian Specialist Medical
  College Fellowships" (the qualification codes), bound to
  `Practitioner.qualification.code` only inside a new dedicated
  `AUFellowshipQualification` slice on `Practitioner.qualification`.
- A standalone `CodeSystem` for "Australian Specialist Medical Colleges"
  (the issuer enumeration), referenced via `Organization.identifier` only;
  do not modify the v2-0360 binding.
- Leave the base `qualification` slice unchanged for non-fellowship
  qualifications so v2-0360 consumers see no behaviour change.

#### Justification

This avoids any compatibility risk for implementers already binding to
`au-hl7v2-0360` (the extended ValueSet introduces new codes that some
validators may flag). It also separates two semantically distinct concerns
— "what qualification" vs. "which college" — into independent artifacts,
which is easier to maintain when the AMC list changes. Trade-off: it
duplicates structure (two CodeSystems plus a slice rather than one
extended ValueSet) and diverges from the explicit PAWG decision to extend
the existing ValueSet, so it carries some governance friction.

---

### Disposition C: Decline

#### Proposal

Decline the request as currently framed. Do not introduce a normative
Australian-college CodeSystem or extend `ValueSet-au-hl7v2-0360`. Instead,
close as "Not Persuasive — already addressable" on the basis that
`Practitioner.qualification.code` is `CodeableConcept` and `qualification.issuer`
is `Reference(Organization)`, both of which can already convey fellowship and
issuing-college information today using locally-defined codes or the existing
v2-0360 binding plus free-text `Organization.name`.

#### Justification

The data model already supports the use case without any spec change; the
original need (reporting that a practitioner holds FRACP issued by RACP) is
expressible in the current AU Base Practitioner profile. Adopting a
maintained AU CodeSystem creates an ongoing curation burden (the AMC URL has
already moved once, per the 2024 comment) and the
extended ValueSet is not strictly required for interoperability.

---

### Recommendation

**Recommended disposition:** A

PAWG has already reached and recorded consensus on this exact approach
(3 Mar 2021 and 31 Mar 2021), the work has been done by the assignee, and
draft published artifacts are available on the
`cr-practitioner-qualification` branch. Disposition A turns that work into
shipped content with the lowest governance risk. Disposition B is
intellectually defensible but contradicts a recorded WG decision and adds
maintenance overhead without a clear win. Disposition C ignores a real,
repeatedly-validated implementer need (consistent representation of
Australian fellowship qualifications across systems), and the
"already-addressable with `CodeableConcept`" argument has been the de-facto
status quo since 2020 without solving the interoperability problem. The
workgroup should accept the prototyped change, confirm the slice naming and
binding strength against AU Base conventions, and merge.
