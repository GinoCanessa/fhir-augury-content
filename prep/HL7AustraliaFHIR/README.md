#### Topic: additionalBindings clarification on AU Base profiles

These tickets ask AU Base to formalise value-set guidance currently
expressed only as narrative on AU Base profiles by adopting the
`additional-binding` extension from the FHIR Tools IG. Both target
sibling AU Base profiles (Organization and PractitionerRole) and
point at the in-flight `additionalBindings` branch of
`hl7au/au-fhir-base` as the prototype.

### Change Request

#### Topic: Re-align AU eRequesting artefacts with FHIR-47141 resolution

All three tickets ask the work group to revisit AU eRequesting
artefacts in light of FHIR-47141 (resolved *Persuasive with
Modification*), which relaxed the AU eRequester Server
CapabilityStatement's `ServiceRequest` `create`/`update` interactions
from SHALL to SHOULD instead of introducing a separate Responder
actor. The set covers the ServiceRequest profile obligations, the
Server actor definition, and the Server CapabilityStatement
description so they remain internally consistent with that change.
