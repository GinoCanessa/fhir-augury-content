#### Topic: Australian organisation & payer Identifier profiles

These tickets ask AU Base to formalise identifier profiles for
Australian organisation-level identifiers — the Healthcare
Administration Entity (HAE) Organisation identifier and the
private Health Insurer identifier used in `Coverage.payor`. Both
are about defining the canonical identifier `system`/profile so
downstream IGs (AU Core, eRequesting, Patient Summary) can bind
to them consistently.

### Change Request

#### Topic: Tightening AU Core terminology bindings on observations/conditions

Both tickets ask AU Core to replace or augment a generic value-set
binding with an AU Core–specific, more clinically actionable set —
a Condition.code AU Core value set sourced from NCTS/SNOMED CT-AU
for the AU Core Condition profile, and a richer Smoking Status
binding (plus interpretation/referenceRange guidance) for the AU
Core Smoking Status Observation profile. They share the pattern of
moving from broad inherited bindings to AU-curated ones via
extensible / additional bindings.

### Change Request

#### Topic: AU eRequesting Server actor & CapabilityStatement re-review (FHIR-47141 follow-up)

Both tickets are direct follow-ups to the applied change in
FHIR-47141, which replaced the originally-proposed separate
"eRequesting Responder" actor with a relaxation of the existing
Server CapabilityStatement. They ask the work group to revisit
the **textual definition** of the AU eRequesting Server actor and
its CapabilityStatement so the prose matches the new conformance
model. They should be tackled together because the actor
definition and the CapabilityStatement narrative must agree.
