## Summary

The reporter (John Moehrke) flags an editorial inconsistency in the
Scalable Consent Management IG (build `2026Jan`). A sentence on one of
the IG's narrative pages reads "…can be found by following the links on
the **Artifacts** page", but the hyperlink actually points to the
**Artifacts Overview** page (`artifact_overview.html`). The reporter
also notes that the generated **Artifacts** page does itself include
all the artifacts, so the wording and the target page disagree, which
is confusing for readers trying to navigate the IG. No cross-referenced
Jira tickets, Zulip threads, or GitHub items were surfaced for this
ticket — it is a self-contained editorial issue.

The ticket already carries a workgroup-proposed resolution of *Not
Persuasive with Modification*: the editor (Moehrke himself) prefers to
keep the curated **Artifacts Overview** page (because it provides more
information and sorting than the generated artifacts page) and proposes
to remove the link to the generated artifacts page and ensure that **all
links go to the overview** instead.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Treat the request literally: change the link **target** so that the word
"Artifacts" in the sentence resolves to the auto-generated
`artifacts.html` page (the standard IG Publisher artifacts index)
instead of `artifact_overview.html`. Leave the link text as
"Artifacts". This is the smallest possible change consistent with the
reporter's stated complaint that "Artifacts page" wording must point to
the Artifacts page.

#### Justification

This is the most literal reading of the request and matches what an
implementer would expect when they read a sentence that says "the
Artifacts page". The generated artifacts page is a familiar, predictable
landing place across HL7 IGs, so consistency with the wider FHIR IG
ecosystem is preserved.

---

### Disposition B: Alternative Approach (Not Persuasive with Modification — already proposed by the editor)

#### Proposal

Keep the curated **Artifacts Overview** page (`artifact_overview.html`)
as the canonical entry point for browsing the IG's profiles and other
artifacts, because it provides additional grouping, sorting, and
explanatory text that the generated `artifacts.html` does not. Fix the
inconsistency by **rewording the sentence and updating any other narrative
links**, e.g.:

> The full set of profiles defined in this implementation guide can be
> found by following the links on the [Artifacts Overview](artifact_overview.html) page.

Then sweep the IG narrative for any other occurrences of "Artifacts page"
(or links to `artifacts.html` from narrative pages) and re-point them at
`artifact_overview.html` so the IG has a single, consistent navigation
story. This matches the resolution already recorded on the ticket.

#### Justification

The underlying problem is the *mismatch* between link text and link
target, not the choice of target. The Overview page is curated by the
editor and conveys more information than the generated index, so
preserving it serves readers better than reverting to the generated
page. Renaming/relinking is an editorial-only change with no normative
impact and no breaking effect on conformance. This is also the
disposition the editor has already written into the resolution field, so
it has implicit author buy-in.

---

### Disposition C: Decline

#### Proposal

Mark the ticket *Not Persuasive*: the link works, the target page
exists, and a careful reader can follow it. No spec change is required.

#### Justification

The change is purely cosmetic and the linked content is reachable
either way. Declining keeps the workgroup's editorial cycles focused on
substantive consent-management issues.

---

### Recommendation

**Recommended disposition:** B

Disposition B (the *Not Persuasive with Modification* path the editor
has already drafted into the resolution) is the right answer. The
reporter's real complaint is the inconsistency between the link text
("Artifacts") and the link target (`artifact_overview.html`); fixing
that by adjusting the wording — rather than by re-pointing links at the
auto-generated index — preserves the curated Overview page that gives
readers more useful navigation than the bare generated artifacts page.
It is editorial-only, has zero conformance impact, and is already
acceptable to the page's author. Disposition A would resolve the strict
text/target mismatch but at the cost of demoting the more useful curated
page; Disposition C ignores a real navigation papercut that the reporter
encountered while reviewing the IG.
