## Summary

This ticket (ballot comment "BC #87") against the US Breast Cancer
Radiology Reporting (FHIR) Implementation Guide reports that the IG's
narrative mentions "ANTLR4" without providing any citation or reference.
The reporter (Mitra Rocca) requests that a reference be supplied so that
readers can identify what ANTLR4 is and where to find it. The ticket has
already been triaged with a resolution of **Persuasive**: the work group
agreed and stated that, in the next publication cycle, all references to
existing tooling (including ANTLR4) would be properly cited. No Jira,
Zulip, or GitHub cross-references are recorded for this ticket in the
indexed sources.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Add an explicit reference for ANTLR4 wherever it is mentioned in the US
Breast Cancer Radiology Reporting (FHIR) IG narrative. Concretely, on
the page(s) that currently say "ANTLR4", convert the bare term into a
hyperlink to the ANTLR project (e.g.,
`https://www.antlr.org/`) and add a formal entry to the IG's
references / bibliography section that cites ANTLR4 (project name,
version where relevant, and URL). Apply the same treatment to any other
tooling names that currently appear without citation, consistent with
the resolution text.

#### Justification

This is exactly what the comment requests, and it is also what the work
group already agreed to in the resolution ("In the next cycle we will
make sure that all references to existing tooling are included"). The
change is purely editorial — no resource, profile, value set, or
constraint is touched — so it carries essentially zero implementer
impact while materially improving the IG's usability and meeting basic
HL7 publication expectations for citing third-party tooling.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than spot-fixing only the ANTLR4 mention, add a dedicated
"References" / "Tooling" appendix to the IG that enumerates every
external standard, library, and tool referenced anywhere in the guide
(ANTLR4, FHIRPath grammar, validators, etc.), and replace each in-line
bare mention with a short link into that appendix. Establish this
appendix as the canonical place for future tooling references so
subsequent ballot cycles do not repeatedly surface "a reference is
needed" comments.

#### Justification

The reporter's underlying concern — readers cannot identify what ANTLR4
is — is almost certainly not unique to ANTLR4. A single editorial fix
satisfies the letter of the comment but leaves the same gap open for
every other unreferenced tool name. A consolidated appendix is a small
amount of extra editorial work now that prevents a recurring pattern of
ballot comments and makes the IG self-contained for new readers. It is
fully compatible with Disposition A (it is a superset).

---

### Disposition C: Decline

#### Proposal

Close the ticket without further action on the grounds that it is a
trivial editorial nit on a publication that has already moved on, and
that ANTLR4 is sufficiently well-known in the FHIR tooling community
(grammar for FHIRPath, CQL, etc.) that an explicit citation is not
strictly required.

#### Justification

The ticket is a single-line editorial request with no normative impact.
If publication resources are constrained, the work group could choose
to defer indefinitely on the basis that the term is industry-standard
and easily searchable. This is the weakest position, however, because
the work group has *already* agreed in writing that the reference
should be added; declining now would contradict the recorded
"Persuasive" resolution and reopen a settled item.

---

### Recommendation

**Recommended disposition:** B

The work group has already committed (resolution: Persuasive) to add
the missing reference, so Disposition C is off the table — it would
walk back a public agreement on a request that costs nothing to honor.
Between A and B, Disposition A literally fulfills the ticket but leaves
the IG one ballot cycle away from the same comment being raised about
the next un-cited tool. Disposition B is only marginally more effort
(one appendix, plus link conversions) and converts a one-off editorial
fix into a durable convention for the IG, which is the better return on
the work group's editorial time. Recommend adopting B and treating
ANTLR4 as the first entry in the new tooling/references appendix.
