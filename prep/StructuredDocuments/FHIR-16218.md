## Summary

The reporter (Giorgio Cangioli) asks the Structured Documents work group
to revisit and improve the prose descriptions of some of the sections
defined in the International Patient Summary (IPS) FHIR Implementation
Guide, calling out **Social History** as an illustrative example. The
ticket is intentionally broad: it does not name a specific replacement
text, a constraint to add, or a structural change — it flags that
section-level descriptive guidance in the IPS is uneven and asks that
the gaps be filled in.

The cross-reference index surfaces no other Jira tickets directly
linked to FHIR-16218. However, a keyword-similar Jira ticket,
[FHIR-52097](https://jira.hl7.org/browse/FHIR-52097) ("Improve description
of sections"), filed in 2025 against the EU Hospital Discharge Report
IG, makes essentially the same request against a sibling IG (calling
out Physical Findings and Anthropometry, and noting the descriptions
should also clarify section *scope*, e.g., anthropometry vs. vital
signs). That ticket is useful comparator context: the same family of
IGs has a recurring need for clearer section-level scope/description
text. FHIR-16218 itself is not formally linked to FHIR-52097.

The ticket has a non-trivial process history. A previous resolution
("Considered for Future Use", vote Rob Hausam / Gay Dolin 16-0-3 on
2018-08-02) was reverted on 2022-11-22 by Lynn Laakso, who re-opened
the item post-STU-update publication. It currently sits in **Triaged**
state with no assignee.

## Proposed Dispositions

Because no current repo briefing has been loaded, the dispositions
below name the *target artifact in the IG* (IPS Composition section
definitions and their narrative pages) rather than specific source
file paths. Before any of these is implemented, the work group should
have `repo-analysis` run for the IPS source repo so the disposition
can be pinned to authoring roots and concrete files.

### Disposition A: Accept As Requested

#### Proposal

Accept the request and update the IPS IG to provide a clearer,
implementer-oriented prose description for **every** section profile
defined by the IPS, starting with the under-described ones (Social
History is named in the ticket; Physical Findings and Anthropometry
are named by sibling ticket FHIR-52097). For each affected
section profile (e.g., `Composition-section-socialHistory-uv-ips` and
peers):

1. Tighten the **Definition** / **Comments** ElementDefinition fields
   on `Composition.section` so they state, in 2–4 sentences: (a)
   what content belongs in the section, (b) what does **not** belong
   (i.e., scope vs. neighbouring sections such as Vital Signs vs.
   Anthropometry), and (c) which IPS profile(s) are expected entries.
2. Mirror that prose in the corresponding section narrative page in
   the IG so both the StructureDefinition view and the rendered
   guidance page agree.
3. Cite the bound LOINC section code and link to the entry profiles
   the section may reference, so the description is a navigation aid
   as well as a definition.

#### Justification

This is what the ticket literally asks for and what the 2022 re-open
implies the WG now considers in scope. The fix is documentation-only
(no StructureDefinition cardinality, slicing, or terminology binding
changes), so it is low risk for implementers and backwards-compatible.
The existence of the keyword-sibling FHIR-52097 against EU Hospital
Discharge Report shows the same gap is being noticed in other Patient
Summary–family IGs; addressing it in IPS first establishes a pattern
the sibling IGs can adopt. Multiple historical Social History tickets
(FHIR-10327/10328/10329/10330, FHIR-11883, FHIR-46923) confirm Social
History's scope is a persistent source of implementer confusion that
better prose would help resolve.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than rewriting per-section prose ad hoc, adopt a **structured
section-description template** that every IPS section profile must
populate, then back-fill all sections (including Social History)
through that template. Concretely:

1. Define a short authoring template — e.g., **Purpose**, **Typical
   Content**, **Out of Scope / Use Other Section**, **Required and
   Recommended Entries**, **LOINC Section Code** — and document it
   once in the IPS authoring guide.
2. Apply the template to all `Composition.section.*` slices in IPS
   (`socialHistory`, `vitalSigns`, `physicalExam`/`anthropometry` if
   applicable in the current IPS, `medicationSummary`, `allergies`,
   `problems`, `immunizations`, `procedures`, `medicalDevices`, etc.),
   even where the existing prose is already adequate, so the IG is
   uniform.
3. Coordinate with the EU Hospital Discharge Report editors
   (FHIR-52097) and the IPA / US-Core authors so the same template
   shape is reusable across the Patient Summary family.

#### Justification

The ticket reports a *symptom* (Social History description is weak);
the underlying cause is that there is no contract for what a section
description must contain, so editors writing different sections
produce uneven output. A template fix is more durable and prevents
this class of ticket from recurring in the next ballot cycle. The
trade-off vs. Disposition A is scope: B touches sections that aren't
currently complained about and requires editor coordination, which
delays delivery. It also creates a small migration cost for any
external IGs that re-use IPS section text verbatim.

---

### Disposition C: Decline

#### Proposal

Close the ticket as **Not Persuasive – Insufficient Detail** on the
grounds that the ticket does not propose specific replacement text,
does not enumerate which sections beyond Social History need work,
and does not name a concrete defect in the current descriptions.
Invite the reporter (or any commenter) to file targeted, per-section
tickets that include proposed wording, and re-resolve those.

#### Justification

The ticket is effectively a TODO — it identifies a perceived
weakness without proposing a fix, and the WG already considered and
deferred it once in 2018 ("Considered for Future Use", 16-0-3). With
no assignee, no proposed wording, and no Zulip discussion to anchor
the change, accepting it commits editor time against an unscoped
problem. Declining and asking for per-section tickets converts a
diffuse request into actionable work items.

---

### Recommendation

**Recommended disposition:** A

The 2022-11-22 re-open by the WG chair (`lynn_laakso`) signals that
the WG has, in fact, decided this work *is* in scope post-STU. The
ticket is also documentation-only and therefore very cheap to ship in
the next IPS publication: there is no normative impact, no breaking
change, and no terminology coordination required. Disposition B is
attractive in principle but materially expands scope and pulls in
sibling IG editors, which is more than this ticket bargained for and
risks stalling again. Disposition C is defensible on process grounds
but conflicts with the WG's own 2022 decision to re-open. The
pragmatic path is Disposition A, executed as a focused editorial
sweep starting with Social History and the other sections most often
flagged in adjacent tickets (FHIR-52097's Physical Findings and
Anthropometry, plus the Social History scope confusion documented in
FHIR-10327 et al.); the template idea from Disposition B can be
adopted *informally* by the editors during that sweep without being
made a precondition.
