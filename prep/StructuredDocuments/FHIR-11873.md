## Summary

This ticket asks the C-CDA on FHIR implementation guide to convert the
narrative reference to "more specific use cases" — namely Discharge Summary,
Transfer Summary, Referral Note, Consultation Note, and Progress Note — into
actual hyperlinks pointing at those sibling profiles (or their own IGs). The
existing wording calls these alternative profiles by name but provides no way
for a reader of the IG to navigate to them. The reporter (Chris Hills, 2016)
proposes adding the missing links so that the prose is functional rather than
only descriptive. The ticket was originally dispositioned in November 2016 as
*Considered for Future Use* with the rationale "consider this for final
publication after the URLs for IGs stabilize," and was re-opened by Lynn
Laakso in March 2024 post STU 1.2.0 publication so that the workgroup can
revisit it now that the relevant IG URLs are stable. No cross-referenced
Jira tickets were surfaced.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Edit the narrative on the affected page of the US C-CDA on FHIR IG (the
generic "document" or General Header / Composition profile page that
currently reads *"More specific use cases, such as a Discharge Summary,
Transfer Summary, Referral Note, Consultation Note, or Progress Note, are
available as alternative profiles"*) to wrap each profile name in a
hyperlink to the corresponding C-CDA on FHIR profile page (e.g.,
`StructureDefinition-DischargeSummary.html`,
`StructureDefinition-ReferralNote.html`, etc., as published in the latest
STU). Apply the same treatment anywhere else in the IG narrative where
those five profile names appear unlinked. No structural / conformance
content changes — strictly an editorial / authoring update to the prose.

#### Justification

This is exactly what the reporter asked for and what the original 2016
disposition deferred until "the URLs for IGs stabilize." Post-STU 1.2.0,
those URLs are now stable, which is precisely why Lynn Laakso re-opened the
ticket in March 2024. Linking sibling profiles by name is consistent with
standard IG authoring practice across HL7 publications and improves
navigability without affecting conformance. Since no implementer behavior
changes, this is a low-risk editorial fix that closes a long-standing piece
of unfinished work.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than only linking the inline names, replace the prose paragraph with
a small inline list (or table) of the specific use-case profiles, each item
being a link with a one-line description (e.g., "Discharge Summary — used
at the conclusion of an inpatient stay"). Additionally, audit the broader
"alternative profiles" narrative across the IG so that every place a
sibling profile is mentioned by name is consistently linked, and add a
short navigational stub at the top of the General Header / Composition
profile pointing to the index of all C-CDA on FHIR document profiles.

#### Justification

The literal request ("add links") fixes only the one sentence. An IG-wide
sweep produces a more durable improvement and prevents the same complaint
from being reopened against a different page. Presenting the profiles as a
list/table also makes the relationship between the generic Composition
profile and the more specific document profiles easier to scan than running
prose. The trade-off is more editorial scope and slightly more review
effort for the workgroup, but the change is still purely editorial and
non-breaking.

---

### Disposition C: Decline

#### Proposal

Close as *Persuasive with Modification — already addressed* (or *Not
Persuasive* if no equivalent navigation exists), on the basis that the IG
already provides a profiles index / TOC from which readers can locate
Discharge Summary, Transfer Summary, Referral Note, Consultation Note, and
Progress Note, and inline cross-links are an authoring preference rather
than a conformance gap.

#### Justification

If the workgroup considers the IG TOC / profile index sufficient for
discoverability, then this ticket is purely cosmetic. Declining keeps
authoring effort focused on substantive ballot work and avoids re-touching
narrative pages that have just been published in STU 1.2.0. The downside
is that the original reporter's concern — that readers shouldn't have to
hunt through the TOC to find named sibling profiles — would remain
unaddressed, and the ticket would likely be re-opened a third time.

---

### Recommendation

**Recommended disposition:** A

The request is small, concrete, and entirely editorial; the only reason it
was previously deferred ("wait until IG URLs stabilize") no longer applies
post STU 1.2.0, which is exactly why Lynn Laakso re-opened it. There is no
conformance impact, no risk to implementers, and no community disagreement
to weigh — the only Zulip "discussion" is an automated bot post.
Disposition B is attractive as a follow-on but expanding scope to an
IG-wide audit risks turning a trivial fix into a multi-cycle effort and
shouldn't block landing the literal change. Disposition C leaves a
long-standing reporter complaint unresolved for no real benefit. Accept as
requested, link the five named profiles in the cited sentence, and close
the ticket.
