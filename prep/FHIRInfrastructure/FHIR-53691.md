## Summary

In the FHIR R6 ballot 4 introductory text, under the bullet *"Immature
resources have been moved out of this specification into other IGs,
often labelled as 'incubator IGs'. Here's the list:"*, the entry for the
**Citation** resource links to
`https://build.fhir.org/ig/HL7/ebm/StructureDefinition-Citation.html`,
which does not resolve. Citation was removed from the FHIR core
specification and is now expected to live in an HL7 incubator IG;
however, the URL currently embedded in the published ballot points to a
location that is not (or no longer) served by the HL7 build server. The
reporter — Brian Alper, who has historically driven Citation-related
work — is asking that the link be repaired so that readers of the R6
ballot can navigate from the core spec's "moved resources" list to the
canonical home of Citation in its incubator IG.

There are no cross-referenced Jira tickets, no linked Zulip threads,
and no associated GitHub items in the Augury index for this ticket.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Treat this as a pure link-correction in the FHIR Core ballot intro.
Edit `source/ballot-intro.html` in `HL7/fhir` and replace the broken
`href` for the Citation list item (currently
`https://build.fhir.org/ig/HL7/ebm/StructureDefinition-Citation.html`)
with the live canonical URL for the Citation StructureDefinition in its
new home. The replacement URL must be obtained by checking the current
publish location of the EBM / Citation incubator IG (e.g., on
`https://build.fhir.org/ig/HL7/<ebm-or-successor>/`) and using whatever
slug that IG actually publishes for `StructureDefinition-Citation`.
Leave the surrounding bullet list, anchor structure, and visible link
text ("Citation Resource") unchanged. While the file is open, sanity-
check the other "moved resource" entries in the same `<ul>` (e.g., the
`admin-incubator`, `pc-incubator` links on adjacent lines) to confirm
they still resolve, and fix any others that are similarly stale in a
single editorial pass.

#### Justification

This is exactly what the reporter asked for and matches the briefing's
**Update narrative / prose** recipe for FHIR Core: a direct edit to a
top-level `source/*.html` authored page, with no StructureDefinition,
schema, or generated artifact involved. The change is low-risk
(navigation only), affects only a single line of HTML in
`ballot-intro.html`, and is appropriate as a Technical Correction at
ballot-cycle scope. Doing the one-page sweep of sibling links at the
same time avoids a near-term repeat of this same ticket against the
other incubator entries.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than only patching the URL, repair the link **and** stop
hard-coding `build.fhir.org/ig/HL7/<ig>/StructureDefinition-*.html`
URLs in `ballot-intro.html`. Replace the Citation entry (and, by the
same pattern, the other incubator entries in that bullet list) with
references that resolve via the registry / canonical URL of each moved
resource — e.g., link to the resource's canonical
(`http://hl7.org/fhir/StructureDefinition/Citation` or the IG's
canonical) and let the registry redirect to the current published
location, or use a small indirection page in `source/` that the
publisher maintains. Coordinate with the EBM workgroup so that the
Citation incubator's canonical and publish URL are documented
authoritatively in one place.

#### Justification

The underlying defect is a structural one: `ballot-intro.html`
hard-codes ephemeral `build.fhir.org` continuous-build URLs for
resources whose hosting IGs are themselves still moving. Each time an
incubator IG renames itself or re-organises its pages, this list goes
stale and a new ticket like FHIR-53691 will be filed. Routing through
canonical URLs (or a curated indirection table) decouples the FHIR
core ballot from the build-server layout of every incubator IG and
makes future link-rot a configuration change in one place rather than
an HTML edit in core. The trade-off versus Disposition A is greater
scope and the need for cross-WG coordination (FHIR-I + EBM and the
other incubator owners), which is heavier than a Technical Correction
typically warrants.

---

### Disposition C: Decline

#### Proposal

Decline the ticket as **already addressed by existing mechanisms** /
out of scope for FHIR Infrastructure: the Citation resource is now
owned by the EBM workgroup in an incubator IG, and the responsibility
for publishing a working URL for `StructureDefinition-Citation` lies
with that IG, not with FHIR core. Recommend the reporter raise the
issue against the EBM incubator IG (or its tracker) so that the
incubator IG starts publishing at the URL that core already links to.

#### Justification

Defensible only on a strict reading of ownership boundaries: FHIR core
correctly states that Citation has moved, and the link points at the
expected `HL7/ebm` build location; if that location is not being
served, that is an EBM/incubator-publishing failure, not a FHIR-I
documentation error. However, this places the burden on the reader to
discover and follow a redirect that doesn't exist today, and it leaves
a known-broken link in a balloted artifact, which is hard to justify
publicly during an open ballot.

---

### Recommendation

**Recommended disposition:** A

This is a Medium-priority Technical Correction filed during an active
ballot against a single, well-localised line in `source/ballot-intro.html`.
The Update-narrative recipe in the FhirCore briefing covers exactly this
kind of edit, the change is low-risk, and the reporter's request is
literal and easy to honour. Disposition B is the right *long-term*
direction but is disproportionate to a ballot-cycle technical
correction and would need cross-WG coordination that is unlikely to
land in time. Disposition C is technically defensible but leaves a
broken link in a balloted document, which the workgroup should not
accept. The pragmatic path is: (1) fix the Citation URL now per
Disposition A, (2) audit the sibling incubator links in the same `<ul>`
as a one-pass cleanup, and (3) record Disposition B as a follow-on
item for FHIR-I + EBM to consider for the next ballot rather than
blocking this correction on it.
