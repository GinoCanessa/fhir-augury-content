## Summary

The ticket asks the Da Vinci PCT IG to anchor the `PCTAdjudication` ValueSet
(and its underlying CodeSystem) in HL7 Terminology (THO) at
`terminology.hl7.org` rather than carrying it as an IG-local artifact. The
reporter notes that one of the underlying code systems is already in THO,
so consolidating the adjudication concepts into the shared THO
`http://terminology.hl7.org/CodeSystem/adjudication` (AdjudicationValueCodes)
would align the IG with the rest of the FHIR ecosystem and let other
financial IGs reuse the same codes.

The ticket was originally dispositioned `Considered for Future Use` in the
Block-2025-Jul block vote (9-0-2, 2025-07-08) on the basis that the code
system should eventually move to THO but only after cross-IG coordination
and further definition work. On 2025-10-08 Lynn Laakso reopened the ticket
post-STU2 publication, so it is once again in `Triaged` and back on the
work group's plate. There are no explicitly linked sibling tickets in the
Jira cross-reference index, but the Zulip block-vote thread groups this
issue with a family of related "anchor in THO" requests against PCT
(FHIR-45812, FHIR-45815, FHIR-45816, FHIR-45817, and the older FHIR-3553x
series), all of which received the same `Considered for Future Use`
disposition in the same block vote.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Adopt the concrete change set Corey Spears recorded on the ticket:

- Submit the four PCT-local concepts (`coinsurance`, `noncovered`,
  `memberliability`, `discount`) from
  `http://hl7.org/fhir/us/davinci-pct/CodeSystem/PCTAdjudication` to THO
  for inclusion in `http://terminology.hl7.org/CodeSystem/adjudication`
  (AdjudicationValueCodes), with definitions reviewed against the
  semantics already in THO.
- Remove the IG-local CodeSystem
  `http://hl7.org/fhir/us/davinci-pct/CodeSystem/PCTAdjudication` and the
  IG-local ValueSet
  `http://hl7.org/fhir/us/davinci-pct/ValueSet/PCTAdjudication` from the
  PCT IG once the THO additions are accepted and a THO release containing
  them is referenceable.
- Repoint
  `http://hl7.org/fhir/us/davinci-pct/StructureDefinition/davinci-pct-aeob`
  (PCTAdvancedEOB) so its `adjudication.category` fixed-value
  `system` slices reference
  `http://terminology.hl7.org/CodeSystem/adjudication` instead of the
  PCT-local URL.
- Recompose `http://hl7.org/fhir/us/davinci-pct/ValueSet/PCTTotal` to
  include codes from
  `http://terminology.hl7.org/ValueSet/adjudication`
  (AdjudicationValueCodes) instead of the local PCTAdjudication ValueSet.
- Add IG narrative + change-history entries explaining the migration and
  how implementers should map any existing instances using the old
  CodeSystem URL.

#### Justification

This is what the reporter asked for and what the FM-aligned terminology
guidance generally favors: codes that are useful beyond a single IG
belong in THO rather than being trapped in IG-local code systems. THO
already hosts `http://terminology.hl7.org/CodeSystem/adjudication`, so
the destination is well-defined and the four PCT additions are small and
narrowly scoped. Doing the work now also lines this ticket up with the
sibling THO-anchoring tickets (FHIR-45812/45815/45816/45817 and the
FHIR-3553x series) that were grouped with it in Block-2025-Jul, which
makes a coordinated cross-ticket fix more efficient than handling each
one individually later. The original 9-0-2 block vote endorsed the
*direction* of moving to THO; reopening it post-STU2 is a natural
opportunity to convert that direction into action for the next PCT
release.

---

### Disposition B: Alternative Approach

#### Proposal

Take a staged approach that decouples the THO additions from the IG-local
removals:

1. In the next PCT release, deprecate (do not remove) the IG-local
   `PCTAdjudication` CodeSystem and ValueSet, marking each concept with a
   `deprecated` status and a `replacedBy` (or equivalent) note pointing
   at the corresponding code in
   `http://terminology.hl7.org/CodeSystem/adjudication` once the THO
   submission is accepted.
2. Add an extensible binding (or composition) on the affected
   PCTAdvancedEOB and PCTTotal artifacts that *includes* both the
   IG-local codes and the THO `adjudication` codes for one publication
   cycle.
3. Submit the four PCT-local concepts to THO via the standard
   Vocabulary/UTG process and coordinate naming/definition with adjacent
   IGs (CARIN BB, Da Vinci PDex, US Drug Formulary, etc.) that use
   adjudication semantics, capturing the cross-IG review Corey flagged
   in his comment.
4. Once the THO release containing the new concepts is published and
   adopted by the sibling IGs, remove the IG-local CodeSystem/ValueSet
   in the *following* PCT release and switch the fixed-value bindings
   over.

#### Justification

The Corey Spears / Lynn Laakso comment trail explicitly flags that the
adjudication codes "represent different types of concepts that may need
to be included in different CodeSystems" and that cross-IG coordination
is required before encoding the definitions into THO. A hard cut-over in
a single release risks (a) breaking existing PCT implementations that
have wired up the
`http://hl7.org/fhir/us/davinci-pct/CodeSystem/PCTAdjudication` URL into
their AEOB pipelines, and (b) committing THO to definitions that adjacent
IGs have not yet reviewed. Staging the change preserves implementer
investment, gives the Vocabulary work group time to review the four
concept definitions, and still gets the IG to the requested end state -
just over two publication cycles instead of one.

---

### Disposition C: Decline

#### Proposal

Resolve the ticket as `Considered for Future Use` (the original
2025-07-08 block-vote disposition) and explicitly defer the actual
relocation to a future PCT release once the cross-IG / cross-WG work
described by Corey Spears has happened. Concretely: leave the PCT-local
`PCTAdjudication` CodeSystem and ValueSet in place for STU2.x, and add a
non-normative note in the IG narrative that a future release intends to
move these concepts to
`http://terminology.hl7.org/CodeSystem/adjudication` once cross-IG
coordination is complete.

#### Justification

This is the disposition the work group already arrived at by a 9-0-2
vote, and the recorded rationale - that the codes "may need to be
included in different CodeSystems" and require coordination across
numerous IGs - has not changed since July. Reopening the ticket
post-STU2 is procedural (Lynn's note simply re-opens it after
publication); it does not represent new evidence that the underlying
trade-off has shifted. Declining for now keeps PCT STU2.x stable, avoids
churning published instances and downstream implementations, and lets
the Vocabulary WG / UTG do the cross-IG harmonization properly before
any code system surgery touches the published IG.

---

### Recommendation

**Recommended disposition:** B

The reporter's underlying request (codes belong in THO) is sound and the
work group already endorsed that direction in the original block vote.
Disposition A captures the right end state but would force the cut-over
into a single PCT release, which collides with the explicit caveats
captured by Corey Spears and Lynn Laakso about cross-IG coordination and
"adjudication codes representing different types of concepts." Dispostion
C honours those caveats but indefinitely defers the change and leaves the
sibling THO-anchoring tickets in the same family stuck in the same
holding pattern, which is what motivated reopening the ticket
post-STU2 in the first place. Disposition B threads the needle: it
commits the work group to actually doing the migration, lines up with the
THO/UTG submission and cross-IG review steps that Corey said are needed,
and gives implementers a deprecation window before the IG-local
CodeSystem disappears. It is also the disposition that scales to the rest
of the FHIR-45812/45815/45816/45817 sibling tickets if the work group
wants to handle them as a coordinated tranche.
