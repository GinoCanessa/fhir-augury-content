## Summary

The ticket requests that the canonical defining URL for the
`C4BBClaimDiagnosisType` code system in the US CARIN Blue Button IG be
anchored under `terminology.hl7.org` (THO) rather than under
`hl7.org/fhir`. The reporter also calls for a companion UTG ticket to
be opened so the code system can be registered in THO as a properly
maintained (non-example) terminology artifact. It is part of the
`Block-Vote-6` set for CARIN BB STU2, and was originally resolved
"Considered for Future Use" in the 2022-07-26 block vote, then
re-opened after STU2 publication pending completion of
[UP-309](https://jira.hl7.org/browse/UP-309) ("Create Diagnosis Type
Code Systems that is not an example"). UP-309 is the UTG/THO-side
work the reporter anticipated: a replacement code system (the current
one is flagged as an "example" code system) that CARIN BB and Da Vinci
PCT would both consume. UP-309 is still `Tabled` in UTG/Terminology
Infrastructure, with open questions about scope of use, code system
naming, and whether some concepts even belong under "diagnosis type"
at all — meaning the THO artifact this ticket depends on is not yet
available.

## Related Jira Tickets

### [UP-309: Create Diagnosis Type Code Systems that is not an example](https://jira.hl7.org/browse/UP-309)

UP-309 is the UTG ticket that FHIR-35392 explicitly depends on. It
proposes replacing the existing "example" diagnosis type code system
with a properly maintained THO code system that can serve both CARIN
BB (`C4BBClaimInpatientInstitutionalDiagnosisType`,
`C4BBClaimOutpatientInstitutionalDiagnosisType`,
`C4BBClaimProfessionalAndNonClinicianDiagnosisType`) and Da Vinci PCT
(`PCTDiagnosisTypeVS`). The UTG/Terminology Infrastructure call on
2023-04-13 tabled the ticket, flagging that the replacement code
system should have a distinguishable name, that some proposed concepts
may not fit "diagnosis type", and that the relationship to
`http://hl7.org/fhir/R4/valueset-diagnosis-role.html` (used by
`EpisodeOfCare.diagnosis.role`) needs to be worked out. Until UP-309
lands and is published in THO, the canonical re-anchoring requested
here cannot be performed.

Involved users: Corey Spears (reporter/assignee), Robert Hausam,
Carmela A. Couderc, Mary Kay McDaniel, Kathleen Connor, Rob McClure,
Julie Snell.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Move `C4BBClaimDiagnosisType` out of the CARIN BB IG as a
locally-defined code system and re-anchor its canonical URL under
`terminology.hl7.org` (e.g.,
`http://terminology.hl7.org/CodeSystem/claim-diagnosistype` or the
successor name chosen by UTG), then update every reference in the
CARIN BB IG — the `CodeSystem-C4BBClaimDiagnosisType` definition, the
bound value sets (`C4BBClaimInpatientInstitutionalDiagnosisType`,
`C4BBClaimOutpatientInstitutionalDiagnosisType`,
`C4BBClaimProfessionalAndNonClinicianDiagnosisType`), and any
profiles/examples that cite the old URL — to use the new THO canonical.
Gate the change on publication of the THO code system delivered by
[UP-309](https://jira.hl7.org/browse/UP-309); include a persistent
redirect/deprecation note on the old IG-local canonical for existing
implementers.

#### Justification

This is exactly what the ticket asks for and matches HL7 terminology
policy that non-IG-specific code systems should live in THO rather
than under an IG's canonical root. The sibling Block-Vote-6 tickets
(FHIR-35391, FHIR-35393, FHIR-35395) were dispositioned the same way,
so acting consistently across them keeps the STU2+ package coherent.
The FM WG block vote (13-0-0) indicated no substantive objection to
the move — the only reason it was parked was tooling readiness in THO.

---

### Disposition B: Alternative Approach

#### Proposal

Instead of introducing a new CARIN-specific THO code system, reuse
and, if necessary, extend an existing published terminology — most
naturally `http://hl7.org/fhir/diagnosis-role` (FHIR
`DiagnosisRole`, used by `EpisodeOfCare.diagnosis.role` and
`Encounter.diagnosis.use`). Retire the example
`C4BBClaimDiagnosisType` code system in CARIN BB, rebind the three
`C4BBClaim*DiagnosisType` value sets to `DiagnosisRole` (plus any
additional concepts submitted upstream to the FHIR core value set or
supplemented via a small THO code system for the genuinely
claims-specific codes), and update dependent profiles/examples
accordingly.

#### Justification

This matches the alternative raised by Rob McClure on UP-309
("Perhaps this could be additions to DiagnosisRole used for
EpisodeOfCare.diagnosis.role") and by Robert Hausam ("some of the
codes do not seem consistent with 'diagnosis type'"). It avoids
standing up yet another overlapping code system, improves
harmonization with Da Vinci PCT (which UP-309 also serves), and
addresses the UTG concern that the proposed new code system mixes
concepts that may not all be "diagnosis type". Cost is higher
short-term coordination with Vocabulary/Terminology Infrastructure and
possible core-spec change requests, but it reduces long-term
maintenance surface.

---

### Disposition C: Decline

#### Proposal

Close FHIR-35392 as **Not Persuasive with Mod / Considered for Future
Use** (restoring the 2022-07-26 resolution) on the basis that CARIN BB
STU2 has already shipped with the IG-local canonical and that
re-anchoring is blocked indefinitely by an un-published THO
prerequisite ([UP-309](https://jira.hl7.org/browse/UP-309), currently
`Tabled`). Document that the re-anchor will be reconsidered only
once UP-309 delivers a published THO code system, and do not carry
FHIR-35392 open against CARIN BB.

#### Justification

The change cannot be implemented in CARIN BB until its THO dependency
exists, and UP-309 has stalled (`Tabled`) over unresolved naming and
scope questions in Terminology Infrastructure. Keeping FHIR-35392 open
and re-opened across multiple publication cycles creates churn without
delivering change; tracking the substantive work on the UTG ticket is
more honest. Existing implementers already consume the current
canonical, and breaking that URL without a ready THO replacement would
cause harm.

---

### Recommendation

**Recommended disposition:** A (gated on UP-309), with B retained as
a fallback if UTG declines to publish a standalone code system.

Disposition A preserves the intent of the FM WG's prior unanimous
vote, aligns FHIR-35392 with the parallel fates of FHIR-35391 /
FHIR-35393 / FHIR-35395, and gives CARIN BB implementers a stable
migration story: the IG-local URL stays valid until the THO canonical
is published, after which a single coordinated re-anchor lands in the
next CARIN BB publication. The risk on Disposition A is the UP-309
dependency — which is real, since UP-309 is currently tabled — so the
FM WG should (i) formally tie FHIR-35392's resolution plan to UP-309
status in the ticket, and (ii) empower the CARIN BB editors, in
coordination with Vocabulary, to switch to Disposition B (reuse
`DiagnosisRole` plus minimal supplementation) if UP-309 remains
stalled through the next STU cycle. Disposition C (pure decline)
understates community consensus that the URL *should* move and would
need to be revisited the next time an STU ballot opens.
