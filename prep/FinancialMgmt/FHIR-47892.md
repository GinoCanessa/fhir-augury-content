## Summary

This ballot comment against the **US Da Vinci PDex (FHIR)** implementation
guide reports a single typographical error in the narrative text — the word
"iis" appears where "is" was intended ("This information iis added"). The
reporter, Jenny Couse, asks that the wording be corrected to read simply
"is". The ticket has been triaged as a Technical Correction at Highest
priority, resolved as **Persuasive**, and the labels (`Pre-Applied`,
`PDex-Vote-1`, `TechnicalCorrection`) indicate the spelling fix has already
been applied in the source for the next PDex publication. No cross-referenced
Jira tickets, Zulip threads, or GitHub items were surfaced for this issue,
which is consistent with its narrow editorial scope.

## Proposed Dispositions

The dispositions below all concern a single narrative-text typo in the US Da
Vinci PDex IG. Because no GitHub cross-references were returned, the specific
authoring file is not pinpointed by tooling here; the change would be made in
the PDex IG authoring source (the markdown/XML page that contains the phrase
"This information iis added"), and the workgroup editor should locate the
exact page in the PDex IG repository before applying.

### Disposition A: Accept As Requested

#### Proposal

Apply the spelling correction exactly as requested: in the affected PDex IG
narrative page, replace the string "This information iis added" with "This
information is added" (i.e., remove the duplicated leading `i` from `iis`).
No structural, conformance, or normative content changes are involved — this
is a pure editorial fix to a single word. Re-publish the PDex IG so the
corrected wording appears in the next ballot/publication build.

#### Justification

The request is unambiguously a typo ("iis" is not a valid English word in
this context and is clearly intended to be "is"). The fix has zero impact on
conformance, profiles, value sets, examples, or any machine-readable
artifact. The ticket is already labelled `Pre-Applied` and resolved
`Persuasive`, and the resolution description ("Spelling is corrected")
indicates the editor has already made this change. Accepting as requested
formalises what has already happened and keeps the ballot record consistent
with the source.

---

### Disposition B: Alternative Approach

#### Proposal

Accept the spirit of the request but, while the editor is in the affected
PDex page, do a light editorial sweep of the immediately surrounding
sentence(s) to ensure the corrected sentence reads naturally in context.
Specifically: (1) fix `iis` → `is` as requested; (2) verify the sentence is
grammatical and unambiguous after the fix (for example, confirm that "This
information is added" has a clear referent — what information, and where it
is added); (3) if the surrounding text is genuinely confusing, file a
follow-up technical correction ticket rather than expanding scope here.

#### Justification

Single-word typo tickets are an opportune moment to catch nearby wording
problems with negligible additional cost, and reviewers occasionally find
that a typo is a symptom of a sentence that was never carefully read.
However, scope creep on a `Technical Correction` is a real risk and would
violate the ticket's own boundaries, so any deeper edits should be split out
into a separate ticket. This option preserves the requested fix while
encouraging a one-paragraph sanity check.

---

### Disposition C: Decline

#### Proposal

Decline the change and leave the existing wording ("This information iis
added") in place.

#### Justification

There is no defensible reason to decline. The current text is a clear
typographical error, the proposed correction is unambiguous, the change has
no normative impact, no community member has objected on Zulip or in linked
tickets, and the work has in fact already been applied (`Pre-Applied`
label, `Persuasive` resolution). Declining would knowingly leave a typo in a
published HL7 implementation guide and contradict the workgroup's own
prior action on this ticket. This option is included only for completeness.

---

### Recommendation

**Recommended disposition:** A

This is a textbook `Persuasive` / `Pre-Applied` technical correction: a
single-character typo in narrative text with no normative or
machine-readable consequence. The workgroup has already applied the fix and
recorded the resolution as "Spelling is corrected", and there is no
community discussion or related work that complicates the picture.
Accepting as requested simply ratifies what has already occurred and
finalises the ticket. Disposition B is reasonable as an editorial habit but
should not block closure; Disposition C is not defensible.
