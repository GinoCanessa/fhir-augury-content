## Summary

The ticket requests that the canonical defining URL for the CARIN Blue Button
ValueSet `NCPDPCompoundCode` be re-anchored under terminology.hl7.org (THO)
rather than hl7.org/fhir, on the principle that ValueSets bound as `required`
to a `code` data type belong in THO. The reporter also asks that a UTG ticket
be opened to register the ValueSet in THO. The ticket is one of a large batch
of CARIN BB STU2 ValueSet/CodeSystem URL re-anchoring tickets reviewed
together as **Block-Vote-6** by the Financial Management work group on
2022-07-26 (Corey Spears / Kathleen Conner, 13-0-0). It was originally
resolved as **Considered for Future Use** with the disposition "Plan on
making change when external CodeSystem 'all codes' ValueSet is published
(UP-316)", but the resolution was reverted on 2022-11-28 by Lynn Laakso
("re-open post STU2 publication") and the ticket is now back in **Triaged**
status awaiting fresh disposition. No directly-linked Jira tickets are
recorded; the only cross-reference is the Zulip block-vote announcement that
groups it with its sibling NCPDP/AHA-NUBC/etc. URL-rehoming tickets and with
the upstream UP-316 ("all codes" ValueSet) blocker.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Re-anchor `ValueSet-NCPDPCompoundCode` so its canonical `url` is
`http://terminology.hl7.org/ValueSet/NCPDPCompoundCode` instead of an
`http://hl7.org/fhir/...` URL under the CARIN BB IG, and file (or link to)
the corresponding UTG ticket to register the ValueSet in THO. As a
precondition, ensure that UP-316 (or an equivalent UTG mechanism) has
delivered an "all codes" wrapper ValueSet for the external NCPDP Compound
Code code system, or accept that this IG will define and maintain the
wrapper ValueSet in THO itself. Update every binding in the CARIN BB IG that
references the old URL (profiles, slices, narrative) to the new THO URL,
and either retire the IG-local ValueSet or leave a deprecated alias for one
release with a `replacedBy` pointer.

#### Justification

This is consistent with the long-standing HL7 stewardship principle invoked
across the entire Block-Vote-6 slate: ValueSets bound `required` against a
`code` data type, especially those that simply enumerate an external
CodeSystem, are properly stewarded in THO rather than inside an IG. The
2022-11-28 reopening comment by Lynn Laakso explicitly endorses this path —
"This is a ValueSet including all codes in a complete external Code system…
the ValueSet in this IG can be removed and the bindings point to the THO
ValueSet" — and the only reason the workgroup originally deferred was the
lack of a published "all codes" mechanism. Aligning now makes FHIR-35442
move in lockstep with its NCPDP/FDA/CMS siblings (FHIR-35438–35441,
35443–35445) and avoids future divergence between the IG and the broader
CARIN/Da Vinci/US Core terminology stack.

---

### Disposition B: Alternative Approach

#### Proposal

Resolve the ticket as **Persuasive with Modification**: keep the canonical
URL change in scope but do *not* hand-author a new IG-local THO entry.
Instead, (1) wait on, or actively co-sponsor, [UP-316](https://jira.hl7.org/browse/UP-316)
so a generic "all codes" ValueSet for the NCPDP Compound Code CodeSystem is
published in THO, (2) once available, re-bind the affected CARIN BB
profiles/elements directly to that THO ValueSet, and (3) delete
`ValueSet-NCPDPCompoundCode` from the CARIN BB IG entirely (rather than
re-homing it). Track the cut-over with a single coordination ticket that
covers all NCPDP siblings (FHIR-35441, 35443, 35444, 35445) so they ship in
the same publication.

#### Justification

The 2022-07-26 vote already concluded that the right long-term fix is to
let UP-316 land first; the 2022-11-28 reopen does not contradict that, it
just observes that STU2 publication has happened and the workgroup can now
re-plan. Eliminating the IG-local ValueSet (rather than moving it) reduces
maintenance: NCPDP Compound Code is an externally-governed code system, so
the IG should not own the enumeration. Bundling FHIR-35442 with its NCPDP
peers also avoids a churn of canonical-URL changes spread across releases,
each of which is a breaking change for downstream implementers and
validators that have memoized the old hl7.org/fhir URL.

---

### Disposition C: Decline

#### Proposal

Resolve as **Not Persuasive** on the grounds that, for STU2 as published,
re-anchoring the canonical URL of `ValueSet-NCPDPCompoundCode` is a
breaking change for conformant implementations and validators, and the
underlying need (using NCPDP Compound Codes in CARIN BB) is already met by
the existing IG-local ValueSet. Defer all THO migration to a future major
version and close FHIR-35442 with a pointer to UP-316 as the gating
prerequisite.

#### Justification

A change to a ValueSet's canonical URL is, in practice, a breaking change:
profiles, validators, terminology servers, payer-side mappings, and prior
artifacts all key off the URL string. Because NCPDP Compound Code is
stable, narrowly-scoped, and already operational under its current URL, the
cost/benefit of churning the URL inside an STU release is low. The
parallel "ValueSet is not complex, fairly specific to this IG" tickets in
the same Block-Vote-6 (e.g., FHIR-35432, FHIR-35446, FHIR-35447) were
declined on essentially the same reasoning, so there is workgroup
precedent for declining individual entries in this family.

---

### Recommendation

**Recommended disposition:** B

The reopening comment by Lynn Laakso effectively asks the workgroup to
re-decide now that STU2 is out the door, and the substance of her note
("the ValueSet in this IG can be removed and the bindings point to the THO
ValueSet") aligns closely with Disposition B. Disposition A delivers the
requested URL change but risks creating a second IG-stewarded copy of an
externally-governed code system, which is the very pattern THO is meant to
discourage; Disposition C respects the URL-stability concern but leaves the
ticket — and its NCPDP siblings — permanently parked behind UP-316 with no
exit plan. Disposition B threads both needles: it commits the workgroup to
the THO destination the reporter asked for, makes UP-316 the explicit gate
(matching the original 2022-07-26 reasoning), retires the IG-local
ValueSet rather than duplicating it in THO, and groups the NCPDP family so
the unavoidable URL break happens once, in a coordinated release.
