## Summary

The reporter notes a spelling error in the definition of the `balanced` code
in the `invoice-status` CodeSystem (used by the `Invoice.status` element).
The definition currently reads "the invoice has been balaced / completely
paid." and should read "the invoice has been **balanced** / completely
paid." The fix is purely a typo correction in the human-readable definition
text; no code value, display, URL, identifier, or value-set membership is
affected. No related Jira tickets were surfaced via cross-references, and no
Zulip discussion of the typo was found.
