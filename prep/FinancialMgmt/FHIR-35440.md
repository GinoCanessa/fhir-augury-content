## Summary

The ticket asks the CARIN IG for Blue Button® (C4BB) work group to move the
canonical / defining URL of the `ValueSet-FDANationalDrugCode` value set from
the `hl7.org/fhir` namespace into the HL7 Terminology (THO) namespace
(`terminology.hl7.org`). Because the underlying code system is the external FDA
NDC code system (not an IG-local code system), the value set is essentially a
thin "all codes" wrapper and is a natural candidate to live in THO so that
multiple IGs can bind to a single canonical artifact. The reporter explicitly
flags that a corresponding UTG (Unified Terminology Governance) ticket should
be filed to register the value set in THO. The ticket was bundled into
**Block-Vote-6** for the 2022-07-26 FM call alongside more than 25 sibling
tickets (FHIR-35390 … FHIR-35447 plus a few related Corey Spears items) all
asking the same kind of "anchor in THO not hl7.org/fhir" cleanup against C4BB
CodeSystems and ValueSets. The block vote initially resolved this ticket as
**Considered for Future Use** with the disposition *"Plan on making change when
external CodeSystem 'all codes' ValueSet is published (UP-316)"*, but Lynn
Laakso reverted the resolution post-STU2 publication on 2022-11-28, returning
the ticket to **Triaged** — that is the state the work group is being asked to
re-dispose of now.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Re-affirm the original Block-Vote-6 outcome of **Considered for Future Use**,
but make the dependency explicit and concrete: keep
`ValueSet-FDANationalDrugCode` in C4BB unchanged for STU2.x and commit to
removing it from the IG and re-pointing every binding to a THO-hosted
equivalent value set as soon as the THO "all codes" value-set machinery
delivered by [UP-316](https://jira.hl7.org/browse/UP-316) is published. The
specific actions are:

1. File (or link to) a UTG/THO ticket that creates an "all codes" value set
   over the FDA NDC code system at a `terminology.hl7.org/ValueSet/...`
   canonical URL.
2. When that THO value set exists, in the next CARIN BB version delete
   `ValueSet-FDANationalDrugCode` from the IG sources and update every
   `ElementDefinition.binding.valueSet` reference (most notably on the
   prescribed-drug / NDC-coded slices in the CARIN BB Pharmacy and EOB
   profiles) to the THO canonical URL.
3. Mark FHIR-35440 as `Considered for Future Use` with a tracker
   forward-link to the UP-316 follow-up ticket so the dependency is
   discoverable when UP-316 closes.

#### Justification

This matches both the reporter's stated goal (anchor in THO, register via
UTG) and the FM/FMG terminology pattern: external, complete code systems
should be exposed via THO-hosted "all codes" value sets so that multiple IGs
share one canonical, governed artifact rather than each IG inventing its own
wrapper. The original 13-0-0 vote on Block-Vote-6 already endorsed exactly
this trajectory, and the only reason the ticket was re-opened was
post-publication housekeeping (Lynn Laakso, 2022-11-28), not a substantive
disagreement with the rationale. Doing the work now, before UP-316 lands,
would force CARIN BB to either invent a parallel canonical URL in THO that
UTG would later have to reconcile, or to ship a breaking change to value-set
canonicals twice in quick succession.

---

### Disposition B: Alternative Approach

#### Proposal

**Persuasive with Modification — relocate immediately under THO without
waiting for UP-316.** Rather than deferring to the UP-316 "all codes"
value-set tooling, work with the Vocabulary work group to register a
hand-authored intensional value set at a stable THO canonical URL now (e.g.,
`http://terminology.hl7.org/ValueSet/v3-fda-ndc` or an FM-specific
`http://terminology.hl7.org/ValueSet/c4bb-FDANationalDrugCode`), defined as
"include all codes from `http://hl7.org/fhir/sid/ndc`". Update all CARIN BB
profiles in the next IG version to bind to that THO canonical and mark the
old `hl7.org/fhir/us/carin-bb/ValueSet/FDANationalDrugCode` URL as deprecated
(retire on the next major). Open the corresponding UTG ticket as part of
this disposition rather than as a follow-up.

#### Justification

The substance of the change — bind to a THO-hosted "all codes over NDC"
value set — does not technically require the UP-316 generalised tooling;
the same content can be expressed today as a one-off intensional ValueSet
registered in THO. This unblocks downstream IGs (US Drug Formulary, Da Vinci
PDex, NCPDP-aligned guides) that already want to share a single canonical
NDC value set, and avoids a perpetually-deferred state where the ticket
sits in "Considered for Future Use" indefinitely while UP-316 churns. The
trade-off is increased coordination cost with Vocabulary (one more bespoke
THO artifact to maintain) and a harder migration story for implementers
who pinned to the C4BB canonical URL.

---

### Disposition C: Decline

#### Proposal

Resolve as **Not Persuasive**. Keep `ValueSet-FDANationalDrugCode` defined
in CARIN BB at its current `hl7.org/fhir/us/carin-bb/ValueSet/...` canonical
URL. Document in the IG that, while THO is the long-term home for "all
codes" wrappers around external code systems such as NDC, CARIN BB will
continue to ship the IG-local artifact to preserve binding stability for
existing payer/consumer implementers.

#### Justification

A defensible reading of the precedent in Block-Vote-6 itself: a number of
sibling tickets in the same vote (FHIR-35403, FHIR-35413, FHIR-35414,
FHIR-35415, FHIR-35432, FHIR-35434/-35439/-35446/-35447, etc.) were closed
**Not Persuasive** with the rationale "ValueSet is not complex, fairly
specific to this IG, and does not warrant the maintenance of moving to
THO." NDC is a stable, well-known external code system but the *wrapper*
ValueSet itself is trivial and adds little value if implementers are
already binding by code system rather than canonical. Declining avoids the
breaking-change burden on existing CARIN BB consumers who have hard-coded
the IG canonical URL into validators and translation maps, and defers any
THO registration cost.

---

### Recommendation

**Recommended disposition:** A — Accept As Requested (Considered for Future
Use, contingent on UP-316).

This recommendation aligns with the work group's already-recorded 13-0-0
vote on Block-Vote-6 and with the reporter's stated intent (move the
canonical to THO, file a UTG ticket). FDA NDC is a poster-child case for
the UP-316 mechanism — a complete external code system whose IG-local
"all codes" wrapper is inherently duplicative — so chasing a bespoke
intensional ValueSet in THO now (Disposition B) would burn Vocabulary
cycles on an artifact that UP-316 is explicitly designed to obviate, and
would force a second canonical-URL change once UP-316 ships. Disposition
C contradicts the reporter and the original block vote, and does not
hold up well against sibling tickets in the same block that *were*
accepted on the same "external code system → THO" basis (e.g.,
FHIR-35401/-35402/-35404 …). The work group should formally close
FHIR-35440 as Considered for Future Use with an explicit forward link to
UP-316 (and to whatever UTG ticket is opened under it for the NDC value
set) so the dependency is auditable when UP-316 closes.
