## Summary

The reporter, Ron G. Parker, asks the PQ-CMC work group to consider anchoring
the value set(s) defined in the US PQ-CMC (FHIR) IG in the HL7 Terminology
service (THO, `terminology.hl7.org`) rather than maintaining them only inside
the IG. The motivating argument is reuse: the value set — based on the comment
history, this is specifically the **Route of Administration** value set sourced
from NCI Thesaurus and shared with the FDA SPL-on-FHIR project — could be
useful beyond PQ-CMC, and anchoring it in THO would make that reuse
discoverable and version-managed.

The ticket has a long history. It was twice resolved as **"Considered for
Future Use"** (vote Smita Hastak / Norman Gregory, 14-0-0) on the grounds
that PQ-CMC is leveraging the route-of-administration list from another FDA
project (SPL on FHIR) and that NCI Thesaurus would remain the source of truth;
either SPL on FHIR or PQ-CMC would consider registering it with THO at a later
date. Both resolutions were reverted after publication (post-STU1 in Dec 2024,
post-STU2 in Jul 2025) — per shastak's note, GitHub branching between the STU
cycles caused some resolved issues to revert, so this is being re-opened to be
re-decided. There are no cross-references to other Jira tickets, Zulip
threads, or GitHub items in the FHIR Augury index.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Move the PQ-CMC Route-of-Administration value set (and any other PQ-CMC value
sets that draw on broadly-applicable code systems such as NCI Thesaurus) out
of the PQ-CMC IG and define them with `terminology.hl7.org`-anchored
canonicals (e.g. `http://terminology.hl7.org/ValueSet/route-of-administration`
or a PQ-CMC-prefixed variant under that namespace). Coordinate the move with
the HL7 Vocabulary work group and submit the value set(s) via the standard
THO submission/UTG process. Update the PQ-CMC IG profiles and bindings to
point at the new THO canonical URLs, and retain a redirecting/deprecated
alias from the previous IG-scoped canonical for one publication cycle so
existing instances continue to validate.

#### Justification

Anchoring shared, cross-domain code lists (route of administration is plainly
not PQ-CMC-specific — it is consumed by SPL, by Medication* resources, by
many pharmacy and prescribing IGs) in THO is the long-standing HL7 pattern
and is exactly what the THO/UTG process exists to enable. The reporter's
underlying concern is reuse and discoverability; anchoring in THO addresses
both directly. The previous "Considered for Future Use" resolutions
explicitly acknowledged that PQ-CMC or SPL on FHIR "will consider registering
it with THO for broader use" — Disposition A simply commits to that step
now rather than deferring it again.

---

### Disposition B: Alternative Approach

#### Proposal

Keep the value set definition inside the PQ-CMC IG for the current STU cycle,
but (1) have PQ-CMC formally hand off ownership of the Route-of-Administration
value set to a cross-cutting steward — either the SPL on FHIR project or the
HL7 Vocabulary WG — with a documented timeline to migrate the canonical to
THO in the next ballot cycle, and (2) in the meantime, publish the value set
expansion at a stable, citable URL and add an explicit "intended for
migration to THO" note in the IG so downstream implementers know the
canonical is provisional. PQ-CMC profiles continue to bind to the IG-scoped
canonical for STU2; a follow-up ticket tracks the migration for STU3.

#### Justification

The historical resolutions captured a real coordination problem: PQ-CMC
inherited the list from the FDA SPL-on-FHIR project, NCI Thesaurus is
asserted as the source of truth, and ownership of a THO-published value set
needs a steward outside PQ-CMC. Disposition B respects that constraint —
it doesn't force PQ-CMC to take on cross-IG terminology stewardship it
isn't resourced for — while still giving the reporter the reuse path he is
asking for, just on a slower schedule. It also avoids a late-cycle binding
churn in profiles that have already been balloted under STU2.

---

### Disposition C: Decline

#### Proposal

Re-affirm the prior "Considered for Future Use" disposition: PQ-CMC will
**not** anchor the Route-of-Administration value set in THO at this time.
NCI Thesaurus remains the authoritative source for the codes; the value set
remains defined within the PQ-CMC IG; reuse by other IGs can be achieved by
those IGs binding to the same NCIt code system or by re-defining their own
value sets against NCIt.

#### Justification

The work group has twice voted 14-0-0 to defer this. PQ-CMC does not own the
underlying code system, anchoring in THO requires cross-WG coordination and
ongoing stewardship that PQ-CMC cannot unilaterally commit, and the
underlying need (reuse) can already be met by any consumer binding directly
to NCI Thesaurus codes. Reverting only because a Git branch operation
clobbered the prior resolution is not, on its own, a reason to change the
substantive answer.

---

### Recommendation

**Recommended disposition:** B

The unanimous prior votes were not really votes against the *idea* of THO
anchoring — they were votes against PQ-CMC carrying the work alone, while
NCI Thesaurus owned the codes and SPL on FHIR was the parallel consumer.
That coordination obstacle has not gone away, so a straight Disposition A
is likely to stall the same way it did in 2023 and 2024. Disposition C, by
contrast, cements the deferral indefinitely and ignores that the reporter
(and the broader Medication*-using community) has a legitimate reuse case.

Disposition B threads the needle: PQ-CMC keeps its STU2 bindings stable,
but the work group commits — in writing in the IG and via a follow-up
ticket on the HL7 Vocabulary WG — to migrating the canonical to THO in the
next cycle, with a named steward (SPL on FHIR or Vocabulary). That
preserves the prior work group consensus on *who* should own the value
set while still giving the reporter the path to broader reuse he is
asking for. The work group should expect to revisit this ticket in STU3
prep as the actual migration trigger.
