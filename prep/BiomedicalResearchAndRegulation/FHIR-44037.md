## Summary

FHIR-44037 is a Highest-priority change request against the Pharmaceutical
Quality (Industry) FHIR Implementation Guide, asking the Biomedical Research
& Regulation work group to perform a sweep of the IG to (a) identify
additional standard terminologies that should be referenced and (b) add or
tighten terminology bindings on profile elements where bindings are missing
or under-specified. The ticket was originally voted "Considered for Future
Use" on 2024-02-21 (Diana Wright / Rob Hausam, 15-0-0) but Lynn Laakso
reverted that resolution and re-opened the ticket on 2024-05-08 to be
addressed post-STU1 publication. No related Jira tickets, Zulip threads, or
GitHub items were surfaced via cross-references or search, so the ticket
stands on its own as a broad terminology-coverage review task scoped to the
PQ-Industry IG profiles.

## Proposed Dispositions

No GitHub repositories were surfaced for this ticket, so the dispositions
below cannot name a specific authoring root or file path from a persisted
repo briefing. They are framed at the IG-content level (PQ-Industry IG
profiles, value sets, and bindings) and assume the work group will resolve
the change against the PQ-Industry IG source repository at execution time.

### Disposition A: Accept As Requested

#### Proposal

Conduct a structured terminology review of the Pharmaceutical Quality
(Industry) IG as a single tracked work item:

1. Enumerate every coded element across the IG's profiles (every element
   with type `code`, `Coding`, `CodeableConcept`, or
   `CodeableReference`), grouped by profile.
2. For each coded element, record the current binding (value set + binding
   strength) and identify gaps: missing bindings, `example`-strength
   bindings that should be at least `preferred`, and elements bound to
   ad-hoc IG-local value sets where a recognized standard terminology
   (e.g., SNOMED CT, LOINC, UCUM, EDQM Standard Terms, ISO 11238 / IDMP
   substance terminologies, NCI Thesaurus, ICH-defined code lists) would
   be more appropriate.
3. Propose, per element, a target binding (value set URI, strength,
   rationale) and either (a) bind to an existing standard value set,
   (b) compose a new IG-local value set that includes the chosen standard
   code system, or (c) leave as-is with documented rationale.
4. Land the changes as profile and value-set edits in the PQ-Industry IG
   source, regenerate the IG, and ballot the resulting binding changes
   post-STU1 as targeted for STU2.

#### Justification

This is what the ticket literally asks for and what the 2024-05-08 re-open
note implies: a post-STU1 sweep that the WG deferred during STU1 freeze.
A single co-ordinated review is more defensible than ad-hoc per-element
tickets because terminology choices in PQ are highly inter-dependent
(IDMP-aligned substance, dose form, and route terminologies in particular
need to be chosen together to stay internally consistent). Performing the
review once, with explicit rationale per element, also produces an audit
trail the work group can reuse when ballot comments arrive against STU2.

---

### Disposition B: Alternative Approach

#### Proposal

Decompose this umbrella request into per-domain follow-up tickets and
close FHIR-44037 as a tracking ticket once the children exist. Suggested
decomposition, each as its own change request against the PQ-Industry IG:

- **Substance / ingredient terminology:** alignment with IDMP / ISO 11238
  substance identifiers and EMA/FDA substance code systems.
- **Dose form, presentation, container, and route:** alignment with EDQM
  Standard Terms.
- **Units of measure:** UCUM coverage and `Quantity.code` bindings.
- **Manufacturing / process terminology:** evaluate NCI Thesaurus and
  ICH-defined code lists.
- **Regulatory status / lifecycle codes:** evaluate existing FHIR base
  value sets vs. PQ-specific lists.

Each child ticket has a concrete scope, an owner, and a binding-strength
target. The umbrella ticket itself is resolved as "Persuasive with
Modification" pointing at the child set.

#### Justification

The current ticket is effectively an open-ended program of work, not a
single change. Treating it as an umbrella and producing scoped children is
how the WG normally moves a broad review through ballot: each child can be
voted, dispositioned, and applied independently, and ballot reviewers can
comment on a specific terminology decision without having to engage with
the full sweep. This also avoids the risk of the umbrella ticket sitting
in Triaged for another publication cycle because no single owner is
willing to take the entire scope.

---

### Disposition C: Decline

#### Proposal

Resolve as **Not Persuasive** on the grounds that "review the IG for
additional standard terminologies" is not actionable as a change request:
the ticket proposes no specific element, no specific value set, and no
specific binding strength. Close FHIR-44037 and require any actual
terminology gap to be filed as its own ticket naming the profile, the
element path, the proposed value set URI, and the proposed binding
strength.

#### Justification

HL7 ballot tooling expects change requests to identify the artifact and
the specific change. An umbrella "do a review" ticket bypasses that
discipline and tends to either sit indefinitely (as this one already has,
through one STU cycle) or to be resolved with vague language that doesn't
produce traceable spec deltas. Declining on procedural grounds pushes the
work back into the normal per-change pipeline where each binding decision
gets its own discussion and vote, which is the WG's stated preference for
binding changes elsewhere.

---

### Recommendation

**Recommended disposition:** B

Disposition B preserves the intent of the original re-open (a real
post-STU1 terminology pass is needed) while making the work tractable.
Disposition A is the "right" answer in principle, but a single ticket
covering substance, dose form, units, process, and regulatory
terminologies is too coarse to assign and too coarse to ballot — it has
already been deferred once and would likely be deferred again at the same
granularity. Disposition C is procedurally clean but throws away the
useful signal that the reporter and the WG actually want this sweep
performed; it leaves no forcing function. Decomposing into per-domain
child tickets (B) gives each terminology decision an owner, a scope, and
a ballotable disposition, and lets the WG land the high-confidence pieces
(e.g., UCUM on `Quantity.code`, EDQM on dose form / route) in STU2 even
if the harder substance/IDMP alignment slips to a later cycle.
