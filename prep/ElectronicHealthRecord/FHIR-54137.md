## Summary

PatrickJoyce reported that the AI Transparency on FHIR IG (January 2026
ballot) shipped with placeholder author-instruction text — literally
"Insert example scenarios for using data source documentation" — that
was never replaced with the intended content. The ticket asks the
authors either to (a) substitute real example scenarios that
demonstrate how to document data sources for an AI model, or (b) drop
the subsection if no examples are yet available, so the published IG
no longer exposes drafting boilerplate to readers. There are no
cross-referenced Jira tickets; the ticket has already been resolved
"Persuasive with Modification" by restructuring the page so that
"data sources" is reframed as "context" (with prompts as the canonical
example), all delivered in `HL7/aitransparency-ig#22`.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Confirm the existing resolution: in `input/pagecontent/general_guidance.md`
of `HL7/aitransparency-ig`, the placeholder string "Insert example
scenarios for using data source documentation" is replaced with concrete
example narrative. As implemented in PR #22, the "data sources"
subsection is renamed "Context of AI Usage" (or equivalent), prompts
are identified as the representative context, and worked example
prompts are supplied inline. No `sushi-config.yaml`, FSH, or example
instance changes are required. Post-merge, audit `index.md`,
`use_cases.md`, and `explainer.md` for any
`general_guidance.html#data-sources` anchor links and update them to
the new anchor.

#### Justification

This matches both the literal request in the ticket ("Replace with
actual example scenarios") and the merged code change. The IG must
not ship to ballot or publication with author-instruction boilerplate
visible to readers — that is a documentation-quality defect, not a
design choice. The reframing from "data sources" to "context (prompts)"
is also internally consistent with the rest of the IG, which already
treats prompts as a first-class subject (see `AI_Prompts.fsh` in the
authoring root). Closing the ticket as Persuasive with Modification on
the basis of PR #22 is the proportionate, low-risk response.

---

### Disposition B: Alternative Approach

#### Proposal

Apply the *fallback* offered by the reporter: if the workgroup judges
the merged "context / prompts" example set to be still too thin or
not yet ballot-ready, remove the subsection entirely from
`input/pagecontent/general_guidance.md` and drop any links that
pointed at it, deferring real example authoring to the next ballot
cycle. Concretely: delete the now-renamed subsection (and its
heading), update `input/pagecontent/index.md` /
`use_cases.md` / `explainer.md` to remove dangling anchor links, and
optionally add the now-missing fragments to `input/ignoreWarnings.txt`
to keep the IG-publisher quiet. Track the deferred example work as a
new ticket against the next AI Transparency IG ballot.

#### Justification

The ticket explicitly accepts removal as an acceptable outcome ("or
remove this subsection if examples are not yet available"). If the
prompt examples added in PR #22 are felt to under-deliver against the
"example scenarios" the ticket envisaged, omission is preferable to
shipping shallow content that implementers may treat as authoritative
guidance. This costs the IG some narrative completeness in the short
term but protects implementer confidence and avoids an October-ballot
cycle of follow-up corrections. The repo briefing flags the anchor /
`ignoreWarnings.txt` housekeeping that this option requires, so the
mechanics are well understood.

---

### Disposition C: Decline

#### Proposal

Decline the ticket on the grounds that it is already obsolete: PR #22
has been merged into `HL7/aitransparency-ig` and the placeholder string
no longer appears in the ballot snapshot referenced from the Zulip
triage thread (https://hl7.org/fhir/uv/aitransparency/2026Jan/index.html).
Mark the ticket as Not Persuasive / Already Addressed with a pointer to
the PR.

#### Justification

A "Decline" disposition is defensible only as a procedural close — the
substantive request has already been implemented, so re-opening a
design decision around it would be wasted workgroup cycles. This is the
weakest of the three options because the resolution as filed in Jira
("Persuasive with Modification") more honestly describes what
happened: the workgroup acted on the ticket, just not with the exact
literal text the reporter proposed.

---

### Recommendation

**Recommended disposition:** A

The merged change in `HL7/aitransparency-ig#22` directly solves the
defect the reporter raised — the placeholder boilerplate is gone — and
goes a step further by reframing the surrounding narrative around
"context / prompts", which is consistent with the IG's existing prompt
artifacts (`AI_Prompts.fsh`). The Zulip triage explicitly bucketed
FHIR-54137 as a non-bolded "Technical & Formatting" item, signalling
that the workgroup did not see substantive design risk here. Disposition
B (remove the subsection) gives up real reader value that PR #22
already delivers, and Disposition C (Decline) misrepresents what the
workgroup actually did. Confirming the existing resolution as
Persuasive with Modification, with a brief follow-up to audit any
anchor links that pointed at the old "data sources" heading, is the
cleanest close.
