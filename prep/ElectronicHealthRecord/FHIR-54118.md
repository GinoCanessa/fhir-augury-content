## Summary

The reporter is asking the AI Transparency on FHIR IG editors to restructure
the opening paragraph of the IG so it reads more clearly. As currently
worded, the intro jumps directly from a high-level mission statement
("provides guidance for representing the …") into a list of FHIR
mechanics ("how to tag FHIR resources, and expanding into how to use
Provenance, Device, and other data elements") without first telling the
reader *what real-world things* the IG is actually trying to model. The
suggestion is to invert that order: first name the AI-technology
elements being described (e.g., model card, prompt, device,
ai-generated-data), and only then explain the FHIR resources used to
encode each of them. The ticket also implicitly calls out a small
copy-edit issue — the sentence "provides guidance for representing the ."
is incomplete (a missing noun after "the"). No related Jira tickets are
linked, and the only cross-reference is the work group's January-ballot
triage roll-up on Zulip, which lists this item under
"Documentation & Clarity" but contains no substantive discussion of it.

## Proposed Dispositions

The ticket is a narrative / editorial change to the IG's introductory
page. None of the dispositions below imply a profile, value-set, or
schema change; all changes would land in the IG's authoring narrative
(typically `input/pagecontent/index.md` or an equivalently-named intro
page in the AI Transparency IG source repo). Because no GitHub
cross-references were returned, the specific repo and file path were
not auto-resolved by the briefing pipeline; the WG editor should
confirm the intro page path before applying any of the proposals.

### Disposition A: Accept As Requested

#### Proposal

Rewrite the opening paragraph of the AI Transparency on FHIR IG intro
page so that it (a) completes the truncated "representing the ."
sentence, and (b) leads with the *real-world AI elements* the IG
models, then maps each to its FHIR encoding. Concretely, replace the
current opening with text along the lines of:

> "This Implementation Guide describes how to represent the key
> artefacts of AI-augmented care — the **AI model** itself (including
> its model card / metadata), the **prompt** or input given to it, the
> **device / software** that hosts it, and the **AI-generated data**
> it produces — in FHIR. Starting with how to **tag** resources whose
> content was produced or influenced by an AI, the guide then shows
> how `Provenance`, `Device`, `DeviceDefinition`, and related
> resources are used to record each of those artefacts so that
> downstream systems can identify, audit, and make informed use of
> AI-inferred health data."

The order in the new wording is: (1) name the real-world AI elements,
(2) then enumerate the FHIR resources used to encode them.

#### Justification

This is exactly what the reporter asked for and is consistent with the
HL7 IG editorial convention of stating *what* a guide describes before
*how* it is encoded. It also fixes the obvious copy-edit defect ("the
.") that the current text contains, and it puts model card / prompt /
device / ai-generated-data — the four concepts the rest of the IG
revolves around — front-and-centre, which lowers the cognitive cost
for first-time readers. No normative content changes; resolution
should be `Persuasive`.

---

### Disposition B: Alternative Approach

#### Proposal

Accept the *intent* but rather than rewriting only the opening
paragraph, restructure the intro page into two short, explicitly
labelled subsections:

1. **"What we model"** — a bulleted list of the real-world AI
   artefacts in scope (model / model card, prompt, device hosting the
   model, AI-generated data, and provenance of any of the above), each
   with one sentence of plain-English description.
2. **"How we model it in FHIR"** — a parallel list mapping each
   bullet above to the FHIR resource(s) used to encode it (e.g.,
   *model card → `Device` + `DeviceDefinition` + extensions*,
   *AI-generated data → tagged resources + `Provenance`*, etc.).

Also fix the truncated "representing the ." sentence as part of the
edit. Apply the same restructure to the IG's table-of-contents teaser
so the home page and the intro stay in sync.

#### Justification

The reporter's diagnosis (concept-before-encoding) is sound, but
"rewrite the paragraph" only fixes the symptom on one page. Splitting
the intro into a "what / how" pair makes the mapping inspectable at a
glance and gives every later page in the IG a stable anchor to link
back to ("see *What we model* → *Prompt*"). It also makes the IG's
boundaries easier to audit: if a future reviewer asks "is X in scope?"
they can check the "What we model" list directly. Slightly more work
than Disposition A, but the structural payoff is durable across the
remaining ballot cycles and pairs naturally with several of the other
"Documentation & Clarity" tickets sitting in the same Zulip triage
bucket.

---

### Disposition C: Decline

#### Proposal

Mark the ticket `Not Persuasive` on the grounds that the existing
paragraph already names the FHIR mechanics (tagging, `Provenance`,
`Device`) and that the reader is expected to consult the IG's
dedicated "Concepts" / "Scope" pages for the real-world definitions.
No change to the IG narrative.

#### Justification

Defensible only on a strict reading that "the introduction is a
summary, not a primer." Declining keeps the WG's editorial backlog
small and avoids re-balloting introductory prose. The clear weakness
is that the current sentence is grammatically incomplete ("the .")
which is not defensible to leave unchanged at ballot, so this
disposition would at minimum still need an editorial fix and is
therefore the weakest of the three.

---

### Recommendation

**Recommended disposition:** A

Disposition A is the smallest change that fully satisfies the
reporter, repairs an obvious grammatical defect in the published
ballot text, and aligns the intro with the IG's actual content
(`Device`, `Provenance`, AI-generated-data tagging). Disposition B is
attractive and the WG should keep it on the table if other
"Documentation & Clarity" tickets in the January roll-up converge on
the same restructuring need — in that case, A and B can be merged
into a single editorial PR. Disposition C is not recommended because
the existing sentence is incomplete and shipping it as-is would itself
attract negative ballot. Resolve as **Persuasive**, assign to the IG
editor, and apply the wording change to the IG intro page in the AI
Transparency on FHIR IG authoring source.
