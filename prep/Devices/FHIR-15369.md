## Summary

FHIR-15369 is a non-substantive, clarification-class change request against the
Point-of-Care Devices (PoCD) FHIR Implementation Guide, originally filed by
Swapna Abhyankar in February 2018 as part of the 2018-Jan PoCD ballot
(comment #36). The reporter proposes rewording the IG's Overview paragraph
that describes how the guide enriches numeric/waveform observations using the
IEEE 11073-10201 Domain Information Model (DIM). The current sentence is
considered awkward; the reporter supplied a proposed rewrite that conveys the
same intent more clearly (frame the IG as providing a *framework for
enriching* observations rather than as *organizing device data according to*
the DIM, and clarify that the added context lets receiving systems determine
attributes of the device system and its subsystems that bear on
interpretation).

The ticket has a complicated history: it was originally dispositioned in
2018-03-28 as **Considered for Future Use** (Reinhold/Faughn 9-0-0,
non-substantive clarification, "Wording to be changed as suggested"). On
2026-03-26 Erik Moll commented that the Overview section has since been fully
rewritten and "no further action can be taken." On 2026-04-16 the Devices WG
(per the [2026-04-08 Devices Main Call](https://confluence.hl7.org/spaces/HCD/pages/453903324/2026-04-08+Devices+Main+Call+Agenda+and+Minutes))
**reverted the previous resolution** and put the ticket back on the queue to
be addressed in preparation for the September 2026 ballot. The substantive
question for the WG is therefore not whether to make the original 2018 edit,
but whether the *current* Overview text already satisfies the reporter's
clarity concern, or whether a further rewording pass is warranted before the
September ballot.

There are no explicitly linked Jira tickets and no related GitHub items
surfaced through cross-references. The only Zulip cross-reference is the
automated FHIR Bot tracker post from the day the ticket was filed.

## Details

**Description:**

> Existing Wording: This implementation Guide provides point-of-care device
> data organized according to the IEEE 11073-10201 Domain Information Model
> (DIM), to provide a framework for enriching the numeric and waveform
> observations with context not provided by the FHIR Observation Resource and
> the Device Resource by themselves. The context is to allow a receiving
> system to determine attributes of the device system and its subsystems that

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Apply the reporter's proposed rewrite to the Overview section of the
Point-of-Care Devices (FHIR) IG narrative, replacing the current opening
paragraph with:

> "This Implementation Guide provides a framework for enriching the numeric
> and waveform observations from point-of-care devices based on the IEEE
> 11073-10201 Domain Information Model (DIM), in order to provide context
> that is not included in the primary FHIR Observation or Device Resources.
> This will allow receiving systems to determine the attributes of the
> device system and its subsystems that bear on the interpretation of the
> observations."

Mark the ticket Persuasive (or Persuasive with Mod if minor copy edits are
needed to reconcile the proposed text with the *current* Overview wording),
Non-Substantive / Clarification, with the change targeted at the September
2026 ballot. Apply the change in the PoCD IG's narrative source for the
Overview page in the relevant authoring repository (per the WG's PoCD IG
publication source).

#### Justification

The original 2018-03 vote (9-0-0) already approved the requested rewording as
a non-substantive clarification, and the WG has now formally reverted Erik
Moll's "no further action" close in order to revisit the wording for the
upcoming ballot. The proposed wording is materially clearer than the original
2018 wording: it lifts the awkward "data organized according to … DIM, to
provide a framework" construction and re-centers the sentence on the IG's
actual purpose (enrichment with DIM-derived context). Because the change is
narrative-only and has no impact on conformance, profiles, value sets, or
generated artifacts, the implementer cost is essentially zero.

---

### Disposition B: Alternative Approach

#### Proposal

Treat FHIR-15369 as a trigger for a **scoped Overview pass** rather than a
literal one-paragraph swap. Compare the reporter's 2018 proposal against the
*current* (post-rewrite) Overview text that Erik Moll referenced, and apply
only the deltas that genuinely improve clarity — likely:

1. Frame the IG as a "framework for enriching numeric and waveform
   observations" rather than as "organizing device data."
2. Make explicit that the enrichment is *DIM-derived context* layered on top
   of `Observation` and `Device`.
3. Tighten the second sentence to "…allow receiving systems to determine the
   attributes of the device system and its subsystems that bear on the
   interpretation of the observations."

Capture the resulting paragraph in the PoCD IG's Overview narrative source
for the September 2026 ballot, again as Persuasive with Mod /
Non-Substantive / Clarification.

#### Justification

Erik Moll's comment is a real signal: the Overview has already been
rewritten once in the eight years since this ticket was filed, and blindly
pasting the 2018 proposed paragraph risks regressing improvements that were
made in the interim or duplicating points the new Overview already covers
elsewhere. A targeted reconciliation pass honours both the original
ballot-comment author's intent (clarity) and the current editorial state of
the IG, and it gives the WG a defensible audit trail ("we considered the
2018 proposal against the current text and adopted X but not Y").

---

### Disposition C: Decline

#### Proposal

Mark the ticket **Not Persuasive — Already Addressed**, on the grounds that
the Overview section has been fully rewritten since the ticket was filed
(per Erik Moll's 2026-03-26 comment) and the current text already conveys
the clarification the reporter sought. Close without further narrative
changes for the September 2026 ballot.

#### Justification

The 2018 proposed rewrite was a clarity edit against a paragraph that no
longer exists verbatim in the IG. If the *current* Overview already states
that the IG provides a framework for enriching point-of-care observations
with DIM-derived context — which is what Erik Moll's close implied — then
re-opening the ticket consumes ballot-prep cycles for no substantive gain.
Declining preserves the WG's ability to focus September-ballot effort on
substantive PoCD issues rather than on a 2018 wording suggestion that has
been overtaken by editorial work.

---

### Recommendation

**Recommended disposition:** B

The Devices WG explicitly reverted the prior close and queued this ticket
for the September 2026 ballot, so Disposition C ("decline as already
addressed") is in tension with the WG's own 2026-04-08 decision and would
need to be re-litigated on the call. Disposition A is also weak: the IG
Overview has demonstrably been rewritten since 2018 (per Erik Moll), so
literally pasting in the reporter's 2018 paragraph could regress later
editorial improvements. Disposition B threads the needle — it respects the
WG's decision to act on this ticket for September, honours the reporter's
clarity intent, and explicitly reconciles the 2018 proposal against the
current Overview text so the ballot edit is defensible. The resulting change
remains Non-Substantive / Clarification, so disposition cost to implementers
is zero. The WG should assign an editor (likely the commenter who reverted
the resolution, mcros, in coordination with erikmoll who knows the current
Overview text) to do the reconciliation pass and bring the proposed
paragraph back to the call for sign-off ahead of the September freeze.
