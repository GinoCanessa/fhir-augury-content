## Summary

The ticket asks the Patient Care / Ophthalmology Retinal (Eyes on FHIR) work
group to re-root the canonical URL of the `observation-visual-acuity` Value
Set so that it lives under `terminology.hl7.org` (THO) instead of under
`hl7.org/fhir`. The reporter (Ron G. Parker, originally surfaced as
"Comment 10" during ticket import) further notes that a companion UTG
(Unified Terminology Governance) ticket should be opened to register the
Value Set in THO so the migration is governed properly. No related Jira
tickets were found via cross-references, and no UTG follow-up ticket
appears to be linked. The request is purely a URL/governance change for
a single Value Set artifact in the Eye Care IG; it does not propose any
change to the underlying codes, the binding strength, or the profiles
that consume the Value Set.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Re-root the `observation-visual-acuity` Value Set canonical URL from
under `http://hl7.org/fhir/...` to under
`http://terminology.hl7.org/ValueSet/observation-visual-acuity` (or the
equivalent THO-conformant URL) in the Ophthalmology Retinal /
Eyes on FHIR IG. Concretely:

1. File a companion UTG ticket to register the Value Set (and any
   underlying CodeSystem fragment that is also currently rooted under
   `hl7.org/fhir`) in THO, following the standard UTG submission
   workflow.
2. In the Eye Care IG source, change the `ValueSet.url` of
   `observation-visual-acuity` to the new THO canonical and remove the
   value-set definition itself from the IG once UTG publishes it.
3. Update every binding that references the old URL — at minimum the
   `observation-visual-acuity` StructureDefinition's `value[x]` /
   component bindings — to point at the new THO canonical.
4. Publish a redirect / deprecation note from the old URL so existing
   instances are not silently invalidated.

#### Justification

This is exactly the migration pattern HL7 has applied to other
non-core value sets: terminology artifacts whose lifecycle is
independent of any single specification belong in THO so they can be
reused, versioned and maintained outside an IG release cycle. The
visual-acuity Value Set is a clinical-terminology artifact, not a
FHIR-core artifact, so its current `hl7.org/fhir`-rooted URL is
inconsistent with HL7's terminology-governance policy and with how
peer value sets in other IGs are now published. The reporter explicitly
calls out the need for the UTG companion ticket, which acknowledges
the governance dependency.

---

### Disposition B: Alternative Approach

#### Proposal

Keep the artifact but move it under the IG's own canonical namespace
(e.g., `http://hl7.org/fhir/uv/eyecare/ValueSet/observation-visual-acuity`
or `http://hl7.org/fhir/ig/eye-care/...`) rather than into THO.
Coordinate the rename with the next Eye Care IG publication, deprecate
the old URL with a `replacedBy` extension, and leave THO out of the
loop for now. Revisit a THO migration only once the Value Set is
stable and demonstrably reused by IGs other than Eye Care.

#### Justification

THO registration imposes real overhead: a UTG ticket, vocabulary-WG
review, and a publication cadence the Eye Care project does not
control. The Zulip "Reviving the Visual Acuity Observation Profile
development" thread shows the Value Set's content is still actively
being reshaped (units, optotype codes, lens-type codes are all being
proposed and are not yet in LOINC/SNOMED/UCUM). Locking that
still-evolving content into THO is premature; a namespace move that
keeps the artifact close to its IG lets the Eye Care WG iterate
quickly while still removing the incorrect `hl7.org/fhir` rooting that
the reporter objected to.

---

### Disposition C: Decline

#### Proposal

Decline the rename. Leave the canonical URL of
`observation-visual-acuity` where it is and instead document, in the
Eye Care IG's terminology section, that the Value Set is IG-scoped and
not intended for cross-IG reuse. No UTG ticket is opened.

#### Justification

The ticket was filed in 2021 and has sat in `Triaged` with no
assignee, no comments, no linked tickets and no UTG follow-up for
roughly four years. There is no evidence of an implementer being
broken by the current URL, and the only Zulip activity around the
artifact has been about its clinical model, not its namespace. The
cost (UTG submission, binding updates in every consumer, redirect /
deprecation handling, and an instance-data migration story) would
fall on a small WG with limited bandwidth and would yield no
end-user-visible improvement. Declining preserves stability for the
small but real implementer base that has begun to consume the
profile (per the implementers-stream thread).

---

### Recommendation

**Recommended disposition:** A

The reporter's underlying point is correct on policy grounds: a clinical
Value Set whose lifecycle is independent of FHIR-core does not belong in
the `hl7.org/fhir` canonical namespace, and the longer the artifact is
left under the wrong root the more implementers will bind to it there
(the implementers-stream thread is early evidence that this is starting
to happen). Disposition C effectively bakes in a known-incorrect
canonical and pushes the cleanup cost onto whoever eventually has to
do it, plus a future deprecation. Disposition B is reasonable as a
fallback if Vocabulary / UTG capacity is the blocker, but it spends the
"namespace-rename" budget twice — once now for the IG-local move and
again later for the THO move that the reporter is asking for. The
right path is to do it once, correctly: open the UTG ticket the
reporter asks for, register the Value Set in THO, repoint the
`observation-visual-acuity` profile's bindings, and publish a
deprecation/redirect note from the old URL. The work group should
coordinate with the Eye Care IG's current maintainers (Lars Fuhrmann
et al.) so the namespace move lands together with the next profile
revision rather than as a separate breaking change.
