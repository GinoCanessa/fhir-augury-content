## Summary

John Silva reports a typographical error in the International Birth And
Child Model (IBCM) FHIR Implementation Guide: the word **"subjectt"**
appears (with a doubled "t") on what the reporter describes as the
"3rd bullet" of a page in the IG and should be corrected to
**"subject"**. The ticket is a Technical Correction with no proposed
change to semantics, structure, conformance, or terminology — purely an
editorial fix to the narrative content.

The ticket carries no cross-referenced Jira tickets, no comments, and
the FHIR Augury cross-reference index returned zero hits for
`FHIR-41659`. A targeted search for the literal token `subjectt` across
Zulip and GitHub also returned no results, so the exact location of the
misspelling has not been confirmed from external context — the IG
maintainers will need to grep the IBCM source for the typo to find the
page and bullet that the reporter saw.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Apply the editorial fix exactly as reported: locate the page in the
IBCM IG narrative content where the third bullet of a list contains
the word `subjectt` and change it to `subject`. The change is
narrative-only — no StructureDefinition, ValueSet, CodeSystem, profile,
extension, or example resource is affected, and no IG metadata
(canonical, version, dependencies) needs to change. The fix would be
made in the IBCM IG's authoring source (the `input/pagecontent/`
markdown / `input/images/` narrative pages) rather than in any
generated output, and validated by re-running the IG publisher to
confirm the rendered HTML reads "subject".

#### Justification

This is a textbook Technical Correction: a single-character typo in
narrative prose with no conformance impact, no breaking change risk,
and no normative content. Accepting it as requested is the cheapest,
lowest-risk path, removes a small but real distraction for readers,
and matches how Patient Care has historically dispositioned analogous
spelling fixes. Because the reporter only said "3rd bullet" without
naming a page, the editor applying the fix should grep the IBCM IG
authoring source for the literal string `subjectt` to confirm the
location before committing — this guards against the (small) chance
that the misspelling appears in more than one place.

---

### Disposition B: Alternative Approach

#### Proposal

Accept the fix, but expand its scope to a one-time editorial sweep of
the IBCM IG narrative pages for the same class of error. Concretely,
in addition to changing `subjectt` → `subject`, run a spell-check pass
(e.g., `cspell` or the IG publisher's lint output) over
`input/pagecontent/**/*.md` and fix any other obvious narrative
typos found in the same commit, with each correction listed in the
commit message so the workgroup can review them as a batch. Optionally,
add a CI step (or document a maintainer checklist item) so that future
PRs to the IG run a spell-check over narrative content.

#### Justification

The cost of a typo-only PR is dominated by review and merge overhead,
not by the edit itself. Bundling a broader narrative spell-check pass
amortises that overhead and reduces the chance that the next reader
files another single-typo ticket against the IG. The downside is a
slightly larger diff, which the reviewer must skim carefully to ensure
no proper noun, code element, or intentional spelling (e.g., a
URL fragment or a person's name) is "corrected" by mistake — hence
listing each change explicitly in the commit message.

---

### Disposition C: Decline

#### Proposal

Close the ticket as **Not Persuasive** on the grounds that the
description does not provide enough context to act on safely: the
reporter names neither the page nor the section that contains
`subjectt`, and the IBCM IG has many pages with bulleted lists. Ask
the reporter to reopen with a page URL or, ideally, a pull request
against the IBCM IG repository that points to the exact location.

#### Justification

This is a defensible disposition only if the editor cannot reproduce
the typo by searching the IG source — i.e., if `grep -r "subjectt"`
over the authoring content returns no hits, the bug as written is
unreproducible and the workgroup is right to ask for more information
rather than guess. It is, however, a poor outcome for a one-character
narrative fix that is almost certainly easy to locate, and it pushes
back on a reporter who has already given the workgroup enough to act
on. Decline should only be used as a fallback if the typo genuinely
cannot be found.

---

### Recommendation

**Recommended disposition:** A

This is the cleanest fit for a Technical Correction of this kind: the
change is narrative-only, has no normative or structural impact, and
the reporter's description — while terse — is precise enough
(`subjectt` → `subject`, on a bulleted list) that an editor can locate
and fix it with a single grep over the IBCM IG authoring source.
Disposition B is reasonable as a stretch goal if the editor has the
appetite for a broader sweep, but should not be a precondition for
closing this ticket; the workgroup can always file a follow-up if it
wants a systematic spell-check pass. Disposition C should only be
invoked if the typo cannot in fact be found in the current IBCM IG
source, which is unlikely given the specific spelling reported.
