## Summary

The ticket asserts that the canonical URL for the `body-site-eye` value set,
used by the Ophthalmology Retinal (FHIR) specification, is incorrectly rooted
under `hl7.org/fhir` and should instead be rooted under
`terminology.hl7.org` (THO). The reporter further requests that a
corresponding UTG (Unified Terminology Governance) ticket be opened to
register the value set in THO so that it is governed and published there
alongside other HL7-wide terminology artifacts. No cross-referenced Jira
tickets, Zulip threads, or GitHub items were surfaced for this ticket.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Move the `body-site-eye` ValueSet out of the Ophthalmology Retinal (FHIR)
IG namespace and into THO:

1. Change the ValueSet's canonical URL from
   `http://hl7.org/fhir/ValueSet/body-site-eye` to
   `http://terminology.hl7.org/ValueSet/body-site-eye`.
2. File a UTG proposal to register the value set in THO (new ValueSet
   resource published under the THO IG, with appropriate steward and
   scope metadata).
3. In the Ophthalmology Retinal IG, remove the local ValueSet definition
   and bind the relevant elements (e.g., `bodySite` slices/extensions
   that reference eye anatomy) to the new THO canonical.
4. Add a deprecation/redirect note in the IG's change log indicating the
   canonical URL move, and — if the IG has already balloted/published
   under the old URL — publish a retirement/equivalence note at the old
   canonical.

#### Justification

The reporter is applying a well-established HL7 convention: value sets
and code systems that are reusable across specifications are stewarded
by Vocabulary in THO rather than embedded in a domain IG. The existing
`hl7.org/fhir/ValueSet/body-site-eye` URL implies the value set is
maintained as part of FHIR core, which it is not — it is IG content.
Moving it to THO gives it a stable governance home (UTG), makes it
discoverable to other ophthalmology / body-site work (e.g., eyeCare,
future Observation profiles), and aligns with HL7 URL-namespace policy
for non-core terminology. This is consistent with how other
body-site / anatomy value sets are handled.

---

### Disposition B: Alternative Approach

#### Proposal

Keep the value set's definition within the Ophthalmology Retinal IG but
rehome its canonical URL under the IG's own namespace (e.g.,
`http://hl7.org/fhir/uv/ophthalmology-retinal/ValueSet/body-site-eye`
or the equivalent path for this IG), rather than migrating it into
THO. Do not open a UTG ticket. Only move it to THO later if another IG
(or FHIR core) needs to reuse the value set.

#### Justification

Migrating a value set to THO is a non-trivial governance step: it
transfers stewardship to Vocabulary, requires UTG review, and commits
HL7 to maintaining the value set independently of the IG that needs it.
If the Ophthalmology Retinal IG is the sole consumer and the value set
is tightly coupled to that IG's clinical model, keeping it IG-local
under a correct (non-core) canonical is a lower-cost fix that still
resolves the reporter's core complaint — that the current URL
incorrectly suggests FHIR-core ownership. Promotion to THO can be done
later if/when reuse materializes, without losing any work done here.

---

### Disposition C: Decline

#### Proposal

Decline the request on the basis that the current canonical URL is a
published artifact of the Ophthalmology Retinal (FHIR) specification
and that changing it after publication would be a breaking change for
any implementer already binding to the existing URL. Keep the value
set and canonical URL as-is.

#### Justification

Canonical URLs are intentionally stable identifiers; consumers bind to
them in conformance resources (profiles, questionnaires, validators).
Moving the canonical mid-life causes silent validation regressions for
existing implementations and creates a dangling identifier unless
carefully retired. If no implementers have complained and no reuse
outside the IG is planned, the cost of the move (UTG ticket, THO
publication, IG rebinding, change-log plumbing, implementer
communication) is not clearly justified by the benefit of a tidier URL
namespace.

---

### Recommendation

**Recommended disposition:** A

The reporter — a long-standing participant in HL7 governance — is
calling out an actual convention violation: a reusable body-site value
set is currently published under a URL namespace
(`hl7.org/fhir/ValueSet/...`) reserved for FHIR core. That mislabels
ownership and stewardship regardless of whether the value set is
reused today. The ticket has been `Triaged` but unresolved since 2021,
which suggests it is low-traffic rather than contested, and no Zulip
or GitHub activity has contradicted the ask. Disposition B partially
addresses the URL-namespace complaint but leaves body-site eye anatomy
stewardship inside a single clinical IG, which is not where anatomy
terminology belongs long-term; Disposition C protects a very small
implementer base (this IG is narrow in scope and low in adoption) at
the cost of perpetuating the governance error. Accepting as requested,
with a companion UTG ticket and a clear IG change-log/redirect for the
old canonical, is the cleanest path and is consistent with how HL7
routinely resolves "this belongs in THO" findings.
