## Summary

The reporter notes that the `observation-final-status` Value Set, defined
within the Ophthalmology Retinal FHIR IG, has its canonical URL rooted at
`http://hl7.org/fhir/...`. Per HL7 terminology governance, value sets that
are intended to be reusable terminology artifacts should instead be rooted
at `http://terminology.hl7.org/` (THO). The ticket asks that the Value Set's
canonical URL be re-rooted in THO and that a corresponding UTG (Unified
Terminology Governance) ticket be filed to formally register the Value Set
in THO.

No cross-referenced Jira tickets, Zulip threads, or GitHub items were
returned by the FHIR Augury indexes for this ticket, so the request stands
on its own as a terminology-governance cleanup item against the
Ophthalmology Retinal IG.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

In the Ophthalmology Retinal IG, change the canonical URL of the
`observation-final-status` ValueSet from its current `http://hl7.org/fhir/...`
form to `http://terminology.hl7.org/ValueSet/observation-final-status` (or a
URL chosen by the Vocabulary WG when the UTG ticket is processed). Update
all in-IG references (profiles, structure definitions, narrative,
search-parameter bindings, examples) to point at the new canonical. Open a
companion UTG ticket against the Vocabulary WG to formally register the
Value Set in THO, including its identifier, name, definition, intended use,
and steward (Patient Care). Coordinate publication so that the new THO
artifact is available before the IG re-publishes with the updated URL, and
add a note in the IG history page documenting the move.

#### Justification

This is consistent with current HL7 terminology governance: code systems
and value sets that have any cross-IG / cross-product reusability should
live in THO under `http://terminology.hl7.org/`, not under
`http://hl7.org/fhir/` (which is reserved for artifacts owned by the FHIR
core spec itself). Re-homing the Value Set in THO removes the implicit
ownership claim by FHIR core, makes the Value Set discoverable through the
standard THO publication channel, and lets it be versioned independently
of the Ophthalmology Retinal IG's release cadence. Filing the
accompanying UTG ticket is the documented mechanism Vocabulary WG uses to
take stewardship of such artifacts, and the ticket text explicitly asks
for it.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than relocating the existing Value Set wholesale, deprecate the
current `http://hl7.org/fhir/...` canonical in the Ophthalmology Retinal
IG and publish a *new* THO Value Set
(`http://terminology.hl7.org/ValueSet/observation-final-status`) whose
content is identical. Mark the old canonical as `status = retired` with a
narrative pointer to the THO replacement, and update IG bindings to use
the THO URL going forward. Existing implementers who consumed the old URL
continue to resolve the retired artifact (with a clear pointer) instead of
getting a 404 or a silently-changed identifier.

#### Justification

Canonical URLs are identifiers that implementers may have already wired
into validators, terminology servers, and conformance tooling. A pure
"rename" (Disposition A) silently breaks any consumer that pinned the old
URL. Publishing-then-retiring preserves traceability and follows the
THO/UTG pattern used for other re-homed value sets, while still meeting
the governance goal of moving the authoritative artifact into THO. The
trade-off is that two artifacts must be maintained briefly and the IG
narrative must clearly point readers at the new one.

---

### Disposition C: Decline

#### Proposal

Decline the request and leave the Value Set canonical at
`http://hl7.org/fhir/ValueSet/observation-final-status` within the
Ophthalmology Retinal IG. Rationale category: out of scope — the Value
Set is IG-specific and there is no demonstrated cross-IG reuse that would
justify the cost (UTG ticket processing, dual publication, consumer
migration) of moving it into THO.

#### Justification

If the Value Set is genuinely scoped to Ophthalmology Retinal observation
finality and is not consumed outside that IG, hosting it under the IG's
own canonical root is acceptable per current HL7 practice; not every IG
value set needs to live in THO. Declining avoids breaking implementers
who already consume the existing URL and avoids loading additional work
onto Vocabulary WG / UTG for a single-IG artifact. This disposition is
only defensible if Patient Care can confirm that the Value Set has no
reuse intent beyond the Ophthalmology Retinal IG.

---

### Recommendation

**Recommended disposition:** B

Disposition B best balances the governance goal (the artifact belongs in
THO and the request is well-founded) with implementer stability (canonical
URLs are identifiers, and silently re-rooting them via Disposition A
risks breaking downstream tooling). The work is the same as Disposition A
plus a small additional cost — retiring the old artifact with a pointer —
and that small cost is what preserves traceability for anyone who already
consumed the original URL. Disposition C is hard to defend without
explicit evidence that the Value Set is single-IG-only; given the request
came from a senior reviewer (Ron G. Parker) and explicitly asked for UTG
registration, the assumption should be that broader reuse is intended.
Patient Care should accept the move and coordinate with Vocabulary WG via
the requested UTG ticket, choosing the publish-then-retire path.
