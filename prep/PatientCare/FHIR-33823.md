## Summary

The ticket asks that the canonical URL for the `iop-methods` ValueSet,
which is part of the **Ophthalmology Retinal (FHIR)** Implementation
Guide, be rooted in `terminology.hl7.org` (THO) rather than under
`hl7.org/fhir`. The reporter (Ron G. Parker) further requests that a
companion UTG ticket be opened to register the ValueSet in THO so the
relocation is durable and follows HL7's terminology-stewardship
process. The change is editorial/normative for the IG itself but has a
cross-product dependency on UTG.

This ticket is not explicitly cross-referenced to any other Jira,
Zulip, or GitHub item in Augury. However, keyword similarity surfaces a
large family of effectively-identical tickets filed by the same
reporter against other Ophthalmology Retinal value sets (FHIR-33822,
FHIR-33824, FHIR-33826, FHIR-33830, FHIR-33831, FHIR-33832, FHIR-33833,
FHIR-33750, FHIR-33753, FHIR-33761, …). They share the same fix
pattern: move the canonical URL from `hl7.org/fhir/...` to
`terminology.hl7.org/...` and lodge a UTG registration. They should
almost certainly be triaged and dispositioned together.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Re-root the `iop-methods` ValueSet's canonical URL from
`http://hl7.org/fhir/.../ValueSet/iop-methods` to
`http://terminology.hl7.org/ValueSet/iop-methods` in the **Ophthalmology
Retinal (FHIR)** IG sources. Update every internal reference (binding
`valueSet`, narrative, IG TOC entries, dependency declarations) so
nothing still points at the old URL. In parallel, file the companion
**UTG** ticket to register the ValueSet (and, if not already present,
its CodeSystem) under `terminology.hl7.org`, following UTG's value-set
intake process. Coordinate the IG publication cycle so the IG only
adopts the THO URL once UTG has accepted the registration (or via an
agreed pre-publication URL reservation), to avoid a window where the IG
references a URL UTG has not yet minted. Apply the same treatment, as
one batch, to the keyword-related sibling tickets (FHIR-33822,
FHIR-33824, FHIR-33826, FHIR-33830–33833, FHIR-33750, FHIR-33753,
FHIR-33761, …).

#### Justification

This matches HL7's terminology-stewardship policy: value sets that are
not intrinsic to a single specification's structure belong in THO so
they can be reused across products and versioned independently of any
one IG's release cadence. The reporter explicitly anticipates the UTG
hand-off, which is the mechanism HL7 has standardised for exactly this
kind of relocation. Doing the moves as a batch (one disposition applied
to the whole sibling family) avoids fragmenting the URL rewrite across
many IG publications and keeps narrative cross-references coherent.

---

### Disposition B: Alternative Approach

#### Proposal

Keep the ValueSet **defined in the Ophthalmology Retinal IG** but lodge
a UTG ticket only to **register a stable THO alias / canonical
redirect** for `iop-methods`. The IG would publish a deprecation note
on the `hl7.org/fhir`-rooted URL pointing to the THO URL as the
preferred identifier, and any new bindings would use the THO URL while
existing instances and downstream IGs continue to resolve. Once the
THO-rooted URL is established and adoption is verified, schedule the
authoritative move in a future, coordinated IG release.

#### Justification

The ticket is from 2021 and the IG (and any conformance resources or
instances built against it) has had several years to bake in the
`hl7.org/fhir`-rooted URL. A hard rename is a breaking change for any
downstream artifact that imports or expands the value set by canonical
URL. A staged approach honours the reporter's underlying goal —
THO-stewarded identity — while limiting churn for existing implementers
and giving Patient Care + Vocabulary time to validate that the THO
registration is complete before retiring the old URL.

---

### Disposition C: Decline

#### Proposal

Decline the change. The ValueSet is intrinsic to the **Ophthalmology
Retinal (FHIR)** IG, was published with its current canonical URL, and
the IG is mature enough that renaming the canonical URL is a breaking
change for any conformance resources, expansions, or instances that
already bind to it. Mark this ticket — and the sibling family — as
"Persuasive with Mod / Not Persuasive" depending on workgroup
preference, with a note that future revisions of the IG may revisit
terminology hosting if and when a coordinated UTG migration is funded.

#### Justification

Canonical URL renames are breaking. The ticket does not document
implementer demand, downstream harm caused by the current location, or
a migration plan for existing artifacts. Moving every value set to THO
purely on stylistic grounds, without coordinated UTG capacity and an
implementer-facing transition plan, risks orphan references and
publication churn across many IGs that may consume these value sets.

---

### Recommendation

**Recommended disposition:** A

The reporter's request aligns squarely with HL7's standing direction
that reusable, terminology-style value sets live in THO rather than in
spec-rooted URL space, and the request explicitly anticipates the UTG
hand-off rather than asking the IG to host the value set in
perpetuity. The keyword-similar sibling tickets show this is a
*systemic* request across the Ophthalmology Retinal (FHIR) IG, which
strongly favours a single, coordinated disposition rather than ten
near-identical individual decisions. Disposition B's "alias / deprecate
later" path is a reasonable safety valve only if Patient Care has
evidence of installed-base usage of the current URL — none was
surfaced in Augury, and the IG's maturity and audience make widespread
breakage unlikely. Disposition C protects the status quo at the cost
of HL7's stated terminology policy and would leave the entire family
of sibling tickets unresolved. Patient Care should accept, batch the
work with the sibling tickets, and coordinate publication with UTG.
