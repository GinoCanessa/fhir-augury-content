## Summary

The ticket asserts that the canonical (defining) URL for the
`iop-methods` CodeSystem — currently rooted under `hl7.org/fhir` in the
Ophthalmology Retinal (FHIR) IG — should instead be rooted under the
HL7 Terminology (THO) namespace at `terminology.hl7.org`. The reporter
also notes that a companion UTG ticket should be opened so that the
CodeSystem is properly registered in THO. No related Jira tickets,
Zulip discussions, or GitHub items were surfaced via cross-references
or content search, so the ticket stands on its own as a request to
relocate the CodeSystem's canonical identifier into the appropriate
terminology namespace.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Re-root the `iop-methods` CodeSystem's canonical URL from the
`hl7.org/fhir` namespace into the HL7 Terminology (THO) namespace at
`terminology.hl7.org/CodeSystem/iop-methods` (or the equivalent path
chosen by the UTG editors). Concretely:

1. File a companion UTG (Unified Terminology Governance) ticket
   requesting registration of the `iop-methods` CodeSystem in THO with
   the new canonical URL, including the existing concept set, display
   strings, definitions, and any properties currently published in the
   Ophthalmology Retinal IG.
2. Once UTG accepts and the CodeSystem is published from THO, update
   the Ophthalmology Retinal (FHIR) IG so that the in-IG CodeSystem
   resource either (a) is removed in favour of a reference to the THO
   canonical, or (b) carries the new THO canonical URL and is
   marked/redirected accordingly.
3. Update every binding, ValueSet, profile, narrative, and example in
   the Ophthalmology Retinal IG that currently references the old
   `hl7.org/fhir` canonical so that it points to the new THO canonical.
4. Add a change note / migration guide in the IG release notes
   pointing implementers at the new canonical URL.

#### Justification

HL7's terminology governance pattern is that CodeSystems and
ValueSets defined by HL7 belong under `terminology.hl7.org` (THO) and
are managed via UTG, while the `hl7.org/fhir` namespace is reserved
for the FHIR core specification's own artifacts. Hosting an IG-level
CodeSystem under `hl7.org/fhir` mis-classifies it as core-FHIR content
and bypasses UTG's stewardship process (versioning, deprecation,
review). The reporter (Ron G. Parker) is explicit about this and
explicitly calls for the UTG companion ticket, which is consistent
with how analogous CodeSystems in other IGs have been migrated.

---

### Disposition B: Alternative Approach

#### Proposal

Keep the `iop-methods` CodeSystem's canonical URL in its current
location for the already-published version of the Ophthalmology
Retinal IG (to avoid breaking any implementations that have started
using it), but:

1. Mark the existing `hl7.org/fhir`-rooted CodeSystem as deprecated in
   the next IG release.
2. Open the UTG ticket to register a new THO-rooted canonical
   (`terminology.hl7.org/CodeSystem/iop-methods`) carrying identical
   content.
3. Republish the IG against the THO canonical going forward, and
   include a mapping/redirect note that points implementers from the
   old URL to the new one.

This is effectively "Disposition A with a deprecation window" rather
than an in-place URL swap.

#### Justification

Canonical URLs are identity in FHIR — silently moving one is a
breaking change for any implementer that has bound to the existing
URL. A deprecation-and-replace path honours the governance objective
(get the CodeSystem under THO/UTG) while protecting existing
implementations. Given that the ticket has been in `Triaged` state
since 2021 with no recorded community discussion, it is plausible that
some implementations have already adopted the existing URL, and a
soft-migration is lower risk than a hard rename.

---

### Disposition C: Decline

#### Proposal

Leave the `iop-methods` CodeSystem's canonical URL rooted under
`hl7.org/fhir` and do not open a UTG ticket. Document in the
Ophthalmology Retinal IG that this CodeSystem is IG-scoped and
intentionally published from the IG's own namespace.

#### Justification

This disposition would only be defensible if (a) the Ophthalmology
Retinal IG were judged to be the sole and permanent steward of the
`iop-methods` concepts, with no expectation of cross-IG reuse, and (b)
the project chose to formally exempt this CodeSystem from the THO/UTG
governance pattern. Neither condition is supported by the ticket or
by HL7's general terminology policy, so declining would essentially
require Patient Care to argue that the standard governance pattern
should not apply here. That argument is weak on its face.

---

### Recommendation

**Recommended disposition:** A

The request aligns directly with HL7's established governance pattern
that HL7-defined CodeSystems live under `terminology.hl7.org` and are
managed through UTG, and the reporter has already called out the
required UTG companion ticket. The ticket is from 2021 and remains in
`Triaged` with no Zulip, GitHub, or cross-reference activity, which
suggests there is no known implementer constituency that would be
materially harmed by a clean re-rooting; if Patient Care discovers
during disposition that there *are* downstream consumers of the
existing URL, the work can be downgraded to Disposition B (deprecate
the old canonical and publish a new THO-rooted one in parallel)
without re-opening the underlying question. Disposition C should only
be entertained if Patient Care has a specific reason to keep this
CodeSystem out of THO, which is not evident from the available
material.
