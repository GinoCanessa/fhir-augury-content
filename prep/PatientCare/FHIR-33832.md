## Summary

The ticket asks the Patient Care work group (Ophthalmology Retinal IG) to
relocate the canonical URL of the `observation-visual-acuity-methods`
ValueSet from the `hl7.org/fhir` namespace to the `terminology.hl7.org`
(THO) namespace, on the basis that ValueSets used across specifications
should be governed centrally in THO rather than carried inline in an
individual FHIR specification. The reporter further asks that a
companion UTG (Unified Terminology Governance) ticket be opened to
register the ValueSet in THO so the rehoming can be tracked through the
terminology change-control process. No cross-referenced Jira tickets,
Zulip threads, or GitHub items were found in the augury index for this
key, so the request stands on its own as a small terminology-governance
clean-up against the Ophthalmology Retinal IG.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Rehome the `observation-visual-acuity-methods` ValueSet from the
Ophthalmology Retinal IG into THO. Concretely:

1. File a UTG proposal to register the ValueSet (and any new code
   system content it requires) in `terminology.hl7.org`, under an
   appropriate Patient Care / Ophthalmology folder.
2. Once the UTG proposal is accepted and published, update the
   Ophthalmology Retinal IG so the ValueSet definition either lives
   only in THO or is republished from THO with the new canonical
   `http://terminology.hl7.org/ValueSet/observation-visual-acuity-methods`
   (final slug per UTG editor guidance).
3. Update every binding in the IG (Observation profiles, extensions,
   examples, narrative) that currently points at the old
   `hl7.org/fhir`-rooted canonical to the new THO canonical.
4. Provide a `ConceptMap` / release-notes entry recording the canonical
   change so downstream implementers can migrate.

#### Justification

The HL7 governance pattern is that reusable terminology — especially
ValueSets that are not tightly bound to a single resource definition —
belongs in THO so it can be versioned and maintained independently of
any one specification's release cadence. Rehoming
`observation-visual-acuity-methods` aligns the Ophthalmology Retinal IG
with that pattern, makes the ValueSet discoverable to other care
domains that may want to reuse it, and removes the IG from the
business of being its own terminology authority. The reporter, Ron G.
Parker, is a long-standing voice for this governance pattern, and the
request is consistent with how Patient Care has handled comparable
ValueSets in other IGs.

---

### Disposition B: Alternative Approach

#### Proposal

Keep the ValueSet defined in the Ophthalmology Retinal IG for now, but
mint the canonical URL under the `terminology.hl7.org` namespace
(`http://terminology.hl7.org/ValueSet/observation-visual-acuity-methods`)
and have the IG publish under that canonical with HL7 Terminology
Authority co-stewardship. File the UTG ticket as requested, but scope it
to "claim the canonical and register stewardship" rather than "fully
move authoring into THO". Authoring stays with the Ophthalmology
Retinal IG team; THO simply owns the namespace and the publication
endpoint.

#### Justification

This satisfies the substantive concern in the ticket — that the
canonical URL is in the wrong namespace — without forcing the
Ophthalmology Retinal IG team to coordinate every future content
change through the UTG release train. For a niche, domain-specific
ValueSet whose only consumer today is this IG, full migration into THO
adds process overhead that may not pay off. Splitting "where the
canonical lives" from "who authors the content" is a recognised THO
pattern and gives the IG a path forward that is incremental and
reversible, while still fixing the canonical-URL defect that the ticket
flags.

---

### Disposition C: Decline

#### Proposal

Decline the request and leave the `observation-visual-acuity-methods`
ValueSet defined under its current `hl7.org/fhir`-rooted canonical
inside the Ophthalmology Retinal IG. Document in the resolution that
the canonical URL is intentionally scoped to the IG that authors and
ships the ValueSet, and that no migration to THO is planned at this
time.

#### Justification

The ValueSet is currently used only by the Ophthalmology Retinal IG
and there is no recorded cross-IG reuse, no Zulip discussion, and no
GitHub PRs in the augury index demanding the move. Changing a
canonical URL is a breaking change for any implementer that has
already bound to it, and unilaterally rehoming a single ValueSet
without a broader Patient Care / Ophthalmology terminology cleanup
trades a small governance improvement for real implementer churn.
Declining keeps the change cost contained until a more comprehensive
THO migration plan is on the table.

---

### Recommendation

**Recommended disposition:** A

The ticket is small in scope, the underlying governance argument is
correct, and the remediation path is well-trodden: file a UTG ticket,
register the ValueSet in THO, and re-point bindings in the
Ophthalmology Retinal IG. Disposition B's "claim the namespace but
keep authoring local" is reasonable, but in practice it tends to leave
the ValueSet in a half-migrated state that future maintainers have to
finish anyway. Disposition C ignores the legitimate point that
cross-domain terminology should not live under `hl7.org/fhir`. The
canonical-URL change is breaking by definition, but for a
Standard-for-Trial-Use Ophthalmology Retinal artifact that is the
appropriate moment to make exactly this kind of correction, with a
release-note entry to guide implementers. The work group should adopt
Disposition A and ask the reporter to file (or co-sign) the
corresponding UTG ticket.
