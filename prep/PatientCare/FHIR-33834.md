## Summary

The reporter asks the Patient Care work group (owners of the Ophthalmology
Retinal FHIR specification) to relocate the canonical URL of the
specification's `qualifiers` Code System from a `hl7.org/fhir`-rooted URL
into the `terminology.hl7.org` (THO) namespace, and to file a corresponding
UTG ticket so the Code System is registered and maintained in THO. The
request is motivated by HL7's standing convention that all HL7-published
code systems are owned by Vocabulary and live under THO, with FHIR
specifications referencing them by their THO canonical URL rather than
defining them locally under `hl7.org/fhir/...`. No cross-referenced Jira
tickets, Zulip threads, or GitHub items were returned for this ticket, so
the scope is limited to the single Code System called out in the
description (the comment-16 imported note attributes the original
observation to Ron G. Parker on the predecessor tracker).

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Re-host the Ophthalmology Retinal IG `qualifiers` CodeSystem under THO and
update the IG to reference it by its new canonical URL. Concretely:

1. File a UTG (Unified Terminology Governance) ticket against the
   Vocabulary work group requesting that the `qualifiers` CodeSystem be
   adopted into THO. Provide the current CodeSystem resource (concepts,
   definitions, properties), the requested canonical URL
   (`http://terminology.hl7.org/CodeSystem/<id>` — exact id to be
   coordinated with Vocabulary), and a statement that ongoing maintenance
   moves to UTG.
2. Once Vocabulary accepts the UTG ticket and publishes the THO
   CodeSystem, change the Ophthalmology Retinal IG so that:
   - The CodeSystem is no longer defined locally under `hl7.org/fhir/...`.
   - All ValueSets, profiles, examples, and narrative references in the
     IG that previously pointed at the `hl7.org/fhir` canonical now
     reference the THO canonical instead.
   - The IG's "Terminology" / dependencies section declares THO as the
     source for that CodeSystem.
3. Coordinate publication so the THO release containing the new
   CodeSystem precedes (or matches) the next Ophthalmology Retinal IG
   publication, avoiding broken references.

#### Justification

This is the literal HL7 governance pattern: HL7-defined code systems are
owned by Vocabulary in THO; FHIR specifications and IGs reference, but do
not (re)define, those code systems. The reporter's request is a direct
application of that policy. Doing this also yields the standard
secondary benefits — a single authoritative canonical URL, lifecycle
managed under UTG, easier reuse by other IGs, and avoidance of the
`hl7.org/fhir`-rooted URL implying core-FHIR ownership of an
IG-specific code system. The request is small, well-scoped, and
aligned with how recent IGs already structure their terminology.

---

### Disposition B: Alternative Approach

#### Proposal

Treat the reassignment as a versioned, backward-compatible migration
rather than a hard switch:

1. Still file the UTG ticket and publish the CodeSystem under THO at
   `http://terminology.hl7.org/CodeSystem/<id>` (Disposition A, step 1).
2. In the next Ophthalmology Retinal IG publication, add the THO
   canonical as the **preferred** reference, but retain the existing
   `hl7.org/fhir`-rooted CodeSystem definition as a deprecated alias for
   one publication cycle. Mark the local CodeSystem with
   `status = retired` (or an equivalent narrative deprecation), point
   readers and validators at the THO URL, and provide a ConceptMap (or
   simple narrative table) that documents the URL change.
3. Remove the deprecated `hl7.org/fhir`-rooted CodeSystem in the
   following IG cycle, after implementers have had a publication window
   to update.

#### Justification

The Ophthalmology Retinal IG has been published with the original
canonical URL since at least 2021; existing implementations, profiles,
and downstream artifacts may have hard-coded the `hl7.org/fhir`-rooted
URL. A two-step migration honours the governance objective (CodeSystem
owned in THO) while giving implementers a documented deprecation
window, which is the way other IGs have handled URL migrations of
already-published code systems. The trade-off vs. Disposition A is one
extra publication cycle of dual definitions, in exchange for not
silently breaking validators/clients pinned to the old URL.

---

### Disposition C: Decline

#### Proposal

Close as "will not fix" on the grounds that the IG-defined `qualifiers`
CodeSystem is intrinsically tied to the Ophthalmology Retinal IG (its
concepts model retinal-image qualifiers used by the IG's own profiles
and have no use outside that IG), and therefore should remain defined
within the IG under its `hl7.org/fhir`-rooted canonical URL. Note in
the resolution that HL7 policy permits IG-local CodeSystems where the
content is genuinely IG-scoped, and that the Patient Care WG does not
believe relocating it to THO would benefit other consumers.

#### Justification

THO is the right home for code systems with cross-IG / cross-product
reach. For an IG-private vocabulary that is unlikely to be reused, the
overhead of UTG governance, THO publication cadence coordination, and
canonical URL churn for downstream implementers can outweigh the
governance benefit. Declining is defensible **only** if Patient Care
can affirm both (a) that the `qualifiers` CodeSystem really is
IG-private with no plausible reuse, and (b) that no Vocabulary policy
currently requires HL7-published IG code systems to live in THO.

---

### Recommendation

**Recommended disposition:** B

The governance objective the reporter raises is correct and should be
honoured — HL7's direction is for HL7-published code systems to live in
THO under UTG governance, regardless of whether the immediate consumer
is core FHIR or an IG. Disposition C ignores that policy and trades
short-term IG editor convenience for long-term inconsistency with the
rest of HL7's terminology landscape, so it is hard to defend.
Disposition A is the right end state, but the Ophthalmology Retinal IG
has been published with the original `hl7.org/fhir`-rooted canonical
since 2021; flipping the URL in a single cycle risks silently breaking
validators, profiles, and implementations that pinned the old
canonical. Disposition B reaches the same destination as A while
giving implementers one publication cycle of overlap and an explicit
deprecation, which is the pattern HL7 has used for comparable
canonical-URL migrations. Patient Care should pair this with the UTG
ticket up front so Vocabulary can sequence the THO release ahead of
the next Ophthalmology Retinal IG publication.
