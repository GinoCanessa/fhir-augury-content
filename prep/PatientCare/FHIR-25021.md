## Summary

The reporter requests that **Nutrition** be added as an *optional* section
of the International Patient Summary (IPS). The rationale is patient-safety
oriented: certain diet regimens — most notably enteral feeding and
dysphagia diets — are highly relevant during unplanned/emergency care, and
without a dedicated IPS section these clinically critical facts can be lost
when a patient presents at a site that does not have access to their
home-record nutrition plan. The original ticket was resolved on
2020-02-04 as "Considered for Future Use" (vote 12-0-1, Hausam/Zautke),
with a recommendation to share the comment with the ISO 27269 (IPS Dataset)
team. It was reverted/re-opened on 2022-11-22 by Lynn Laakso post the IPS
STU update publication so it can be reconsidered in a subsequent IPS cycle.
No cross-referenced Jira tickets, Zulip threads, or GitHub items were
returned by Augury for this ticket.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Add a new **optional** IPS section titled **"Nutrition"** to the IPS
Composition profile (HL7 IPS IG, Patient Care). Concretely:

- Extend the IPS `Composition` profile (`Composition-uv-ips`) with an
  additional `Composition.section` slice named `sectionNutrition`,
  cardinality `0..1`, with a fixed LOINC `Composition.section.code`
  selected from the IPS terminology process (candidates include the LOINC
  panel codes already used for nutrition/diet narratives, e.g., a
  diet/nutrition LOINC panel code coordinated with the LOINC IPS
  document panel committee).
- Allow `Composition.section.entry` to reference `NutritionOrder`
  and (where available) `NutritionIntake` resources, plus
  `Observation` (e.g., dietary restrictions, swallowing assessments) to
  cover dysphagia/enteral feeding scenarios. Profile each referenced
  resource with `*-uv-ips` profiles as needed (e.g.,
  `NutritionOrder-uv-ips`).
- Add the new section to the IPS narrative-generation guidance and the
  document-bundle examples, and update the IPS section-table in the
  Implementation Guide.
- Coordinate the change with the ISO 27269 (IPS Dataset) team so the
  HL7 FHIR IPS section aligns with the ISO dataset.

#### Justification

Nutrition information — specifically enteral feeding regimens, NPO/NBM
status, and dysphagia/texture-modified diets — is recognised
clinical-safety content during unplanned care. The IPS already accepts
optional sections (e.g., advance directives, functional status), so
adding **Nutrition** as `0..1` is consistent with existing IPS patterns
and imposes no burden on senders that don't have the data. FHIR has
mature resources (`NutritionOrder`, and `NutritionIntake` in R5) that fit
the use case without inventing new modelling. The 2022-11-22 re-open by
Lynn Laakso explicitly anticipated reconsideration in the next IPS cycle,
and the Hausam/Zautke 12-0-1 vote on 2020-02-04 already endorsed
"considered for future use" — i.e., the workgroup was favourable, just
not in-cycle. ISO 27269 alignment is the only remaining coordination
risk.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than introducing a new top-level IPS section, extend the existing
**Functional Status** / Plan of Care content path:

- Document a profiled use of `Observation` (and optionally
  `NutritionOrder`) within the existing IPS sections — specifically the
  Plan of Care section and/or Functional Status — to convey diet
  restrictions, dysphagia status, and enteral feeding orders, using
  IPS-profiled LOINC codes for diet/nutrition observations.
- Provide implementation guidance and examples in the IPS IG showing how
  enteral feeding and dysphagia information surface today without a
  dedicated section, and explicitly cross-reference ISO 27269.
- Defer a dedicated `sectionNutrition` until the ISO IPS Dataset team
  formally adds Nutrition to the dataset, at which point HL7 IPS adds
  the section in lockstep.

#### Justification

Adding a new IPS section is a meaningful structural change that affects
every IPS producer/consumer's section table, narrative generators, and
conformance assertions. If the underlying clinical content (enteral
feeds, dysphagia, NPO) can already be expressed via existing sections
plus profiled Observations / NutritionOrder, the patient-safety need is
met without churn. This also avoids racing ahead of ISO 27269: the
2019-11-06 suggested resolution and the 2022-11-22 re-open both flagged
ISO IPS Dataset coordination as a prerequisite. A guidance-only update
keeps HL7 IPS aligned with the ISO dataset until ISO formally adds
Nutrition.

---

### Disposition C: Decline

#### Proposal

Decline the change for the IPS specification on the grounds that (a) IPS
is intentionally a **minimum** summary, not a comprehensive nutrition
record; (b) detailed nutrition / diet data is the natural scope of more
specialised IGs (e.g., a future Nutrition IG or the existing
`NutritionOrder` / `NutritionIntake` resources used directly by senders);
and (c) the ISO 27269 (IPS Dataset) does not currently include
Nutrition, so a unilateral HL7 addition would diverge from the aligned
international dataset.

#### Justification

The IPS scope statement deliberately restricts the document to a
minimum, broadly-applicable summary; expanding it ad hoc on a
section-by-section basis erodes that scope discipline. The clinical
patient-safety concern is real but is better served by either (i) the
sending system attaching the relevant `NutritionOrder` / `Observation`
resources outside of IPS, or (ii) waiting for ISO 27269 to add Nutrition
so HL7 IPS can adopt it consistently with the international dataset.

---

### Recommendation

**Recommended disposition:** A

The original 2020-02-04 vote (12-0-1) and the 2022-11-22 re-open both
indicate the workgroup is favourable in principle and was simply timing
the change to a future IPS cycle. The clinical safety case for
enteral-feeding and dysphagia/texture-modified diets in unplanned care
is well-established and not adequately served by Plan of Care or
Functional Status sections, which are not where consumers expect to find
diet orders. Adding `Composition.section:sectionNutrition` at `0..1`
preserves backward compatibility for senders that lack the data while
giving senders that do have it a discoverable, conformant home for it.
The ISO 27269 coordination called out in both the 2019 suggested
resolution and the 2022 re-open should be executed as part of the
disposition (parallel notification + alignment review), not used as a
reason to defer indefinitely. Disposition B is a reasonable fallback if
Patient Care and the IPS team conclude that ISO alignment must precede
any HL7-side section addition; Disposition C is not recommended because
the clinical-safety rationale is sound and the requested change is
narrow (optional, single section).
