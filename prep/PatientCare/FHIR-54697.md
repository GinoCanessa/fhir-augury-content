## Summary

The reporter (Rob McClure) flags a publication bug in the **US SDOH Clinical
Care** IG: the Hunger Vital Signs example section contains links that, instead
of resolving to the actual example instances (e.g., the `Observation`,
`QuestionnaireResponse`, and `Bundle` instances illustrating Hunger Vital
Signs), resolve to the underlying FHIR standard artifact pages (the resource
or profile definitions). The end result is that a reader who clicks an
"example" link is taken to a definitional page rather than the populated
example instance, defeating the intent of the example section. No
cross-referenced Jira tickets were returned by the orchestrator for this
issue, so the request stands on its own as a publication / IG-rendering
correction rather than as part of a broader change set.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Repair the Hunger Vital Signs example section in the US SDOH Clinical Care
IG so that each link in the "Examples" listing resolves to the rendered
example instance page (e.g., `Observation-hungervitalsign-NoRiskExample.html`,
`Observation-hungervitalsign-RiskExample.html`, the corresponding
`QuestionnaireResponse-…` and `Bundle-…` pages) rather than to the underlying
profile or base resource definition page. Concretely:

- Audit the intro / pagecontent fragment that renders the Hunger Vital Signs
  example list (likely `input/pagecontent/HungerVitalSign-*.md` or the
  StructureDefinition intro/notes for `SDOHCC-ObservationScreeningResponse`
  scoped to the HVS examples).
- Replace any `[label](StructureDefinition-…html)` /
  `[label](Observation-…html)`-pointing-at-profile links with the matching
  `[label](Observation-<example-id>.html)` (and equivalent for
  QuestionnaireResponse / Bundle) anchors generated by the IG Publisher.
- Where the intent is "see the profile," keep the profile link but label it
  unambiguously as the profile, and add a separate "Examples" sub-list
  pointing at the example instance pages.
- Re-publish the IG and confirm the rendered links now land on the example
  instance pages.

#### Justification

The ticket is filed as a `Technical Correction` at `Highest` priority by the
Patient Care WG co-chair (Rob McClure), which is appropriate: an "Examples"
section that links to definitional pages is a straightforward publication
defect that breaks reader expectations and undermines the IG's value as
implementer guidance. Fixing the links matches FHIR IG conventions (every
other comparable IG — US Core, mCODE, QI-Core — links example sections to the
example instance pages) and the change is local to intro/notes content; it
does not affect conformance, terminology, or the StructureDefinitions
themselves.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than hand-editing individual links, regenerate the Hunger Vital Signs
example listing using the IG Publisher's automatic example list/`{% include
list-simple-examples.xhtml %}` (or equivalent IG-template) mechanism scoped to
the HVS profile(s). Specifically:

- Tag every Hunger Vital Signs example resource (Observation,
  QuestionnaireResponse, Bundle) with the HVS profile in `meta.profile` and/or
  add them to an `ImplementationGuide` grouping for HVS.
- Replace the manually-authored link list in the HVS intro/notes content with
  a Publisher-generated examples block that enumerates everything claiming
  conformance to the HVS profiles, with links to the rendered example pages.
- Add a CI check (or a note in `qa.html` review) that the HVS section's
  example count matches the count of resources with `meta.profile` pointing
  at HVS profiles, so this regression does not silently recur.

#### Justification

The reported defect is a symptom of hand-maintained link lists drifting from
the actual example set. Regenerating the list from metadata removes the
class of bug rather than the single instance, is consistent with how the rest
of the IG already handles "Examples" tabs on StructureDefinition pages, and
makes future example additions zero-touch. The trade-off vs. Disposition A is
a slightly larger edit (template + metadata change) and a one-time
verification that every existing HVS example carries the right
`meta.profile`; this is more work up front but eliminates the maintenance
burden going forward. It is also a friendlier change for downstream IG
template users.

---

### Disposition C: Decline

#### Proposal

Decline the ticket on the grounds that the "Examples" section already
provides links and the reader can navigate from the profile page's
auto-generated Examples tab to reach the instances; mark as "Not Persuasive"
or "Already Addressed by Existing Mechanism."

#### Justification

This rationale would only be defensible if the existing links were in fact
the intended target (i.e., the IG editors deliberately link to the profile
because the profile page itself surfaces the examples in a tab). Even then,
the labeling is misleading and is reported by the WG co-chair as broken from
a reader's perspective, so declining is hard to defend; it would leave a
visible publication defect in a `Highest`-priority Technical Correction. This
disposition is included for completeness but is **not** recommended.

---

### Recommendation

**Recommended disposition:** A (with B as a follow-up improvement)

The ticket is a `Highest`-priority `Technical Correction` filed by the Patient
Care co-chair against an SDOH-CC publication defect that is plainly visible
to readers. Disposition A is the smallest correct change, fits the ticket's
"Technical Correction" type, and can ship in the next SDOH Clinical Care
errata / republish without touching conformance artifacts. Disposition B
(regenerating the example list from `meta.profile`) is the more robust
long-term fix and addresses the root cause, but it is a template/tooling
change that should not block the immediate link repair; the WG should adopt
A now and open a follow-up to do B in the next normative cycle. Disposition C
is not viable — the WG co-chair has reported the links as wrong, and the
example section is one of the primary entry points implementers use to pattern
HVS Observations and the related Questionnaire/Bundle, as evidenced by the
recent `#united states > Hunger Vital Signs` discussion about how to
represent the HVS panel total score.
