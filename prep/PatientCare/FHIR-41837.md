## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Accept the technical correction as filed. Locate the page in the
"International Birth And Child Model (FHIR)" IG that contains the
referenced second bullet point ending in `..`, and delete the redundant
trailing period so the sentence ends with a single `.`. Commit the
single-character edit, regenerate the IG, and resolve the ticket with a
"Persuasive — fixed" disposition.

#### Justification

This is a "Technical Correction" by type and was filed at "Highest"
priority. The change is a one-character editorial fix with zero
semantic, structural, or conformance impact, and no cost to
implementers. There is no design question to weigh, no community
disagreement on Zulip, and no cross-referenced ticket suggesting a
broader concern. Accepting the request as stated is the minimum
defensible outcome for an obvious typo.

---

### Disposition B: Accept With Editorial Sweep

#### Proposal

Accept the specific fix as in Disposition A, and additionally have the
IG editor do a quick proofreading pass on the same page (and ideally the
sibling pages in the same authoring directory) to catch any other
adjacent typos — duplicated punctuation, trailing whitespace, broken
markdown links, or stale cross-references — while the file is open.
Bundle all editorial fixes under this ticket's resolution comment so the
fix history stays discoverable.

#### Justification

The cost of opening, editing, and republishing an IG page is much higher
than the cost of a one-character edit, so it is generally efficient to
clean adjacent rough edges in the same change. This avoids a future
follow-up ticket for the next typo a reader notices on the same page,
and gives the workgroup a slightly higher-leverage outcome from the
review cycle. The trade-off is a marginally larger diff to review, but
since the IG is narrative-only here, review burden remains trivial.

---

### Disposition C: Decline (Not Persuasive — Insufficient Detail)

#### Proposal

Decline the ticket as filed on the grounds that the report does not
identify which page contains the duplicated period. Resolve as "Not
Persuasive — insufficient information" and ask the reporter, via the
resolution comment, to refile with the specific page URL (and ideally a
copy of the offending sentence) so the fix can be applied without
guesswork.

#### Justification

The description references "the second bullet point" without naming a
page, section, or URL. An editor working from this ticket alone cannot
deterministically locate the typo and could plausibly fix the wrong
bullet on the wrong page. Declining-pending-clarification is a
defensible posture when a technical-correction ticket lacks an
unambiguous pointer to the artifact under correction.

---

### Recommendation

**Recommended disposition:** A

A one-character punctuation fix in an IG narrative page is the textbook
case for accepting a technical correction without ceremony. The
ambiguity flagged in Disposition C is real but easily resolved by the
editor doing a 30-second scan of the IG's narrative pages for a bullet
ending in `..`; that is much cheaper than a round-trip back to the
reporter, especially given the ticket has been sitting at "Highest /
Triaged" since August 2023. Disposition B (the editorial sweep) is
attractive but should be left to the editor's discretion rather than
mandated by the disposition — if they spot adjacent issues while in the
file, they should fix them, but the workgroup vote should be on the
specific change requested. Recommend Patient Care close this ticket as
Persuasive — fixed once the duplicate period is removed.
