## Summary

This ticket, filed against the Ophthalmology Retinal FHIR Implementation
Guide, asserts that the canonical URL for the `procedure-cpt` ValueSet is
incorrectly rooted in `http://hl7.org/fhir/...` and should instead be
rooted in the terminology server namespace `http://terminology.hl7.org/...`
(THO). The reporter (Ron G. Parker, imported as "Comment 12") further notes
that a companion UTG (Unified Terminology Governance) ticket should be
created so that the ValueSet is formally registered in THO. No
cross-referenced Jira tickets, Zulip discussions, or GitHub items were
surfaced for this ticket, so the scope is confined to what is stated in
the description.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Change the canonical URL of the `procedure-cpt` ValueSet in the
Ophthalmology Retinal FHIR IG from its current `http://hl7.org/fhir/...`
form to an equivalent URL rooted under
`http://terminology.hl7.org/ValueSet/procedure-cpt` (exact slug to match
the IG's existing artifact identifier). File a companion UTG ticket to
register the ValueSet in THO so that the new canonical is resolvable
and governed through the terminology infrastructure. Update every
reference to the ValueSet inside the IG (bindings on profiles,
narratives, artifact indices, and any examples) to the new canonical,
and leave a redirect / deprecation note where feasible so that
implementers tracking the previous URL are pointed at the THO-hosted
artifact.

#### Justification

HL7 publication policy directs that value sets not tied to a specific
base specification or IG namespace belong in THO so they can be
centrally governed, versioned, and reused. A `procedure-cpt` value set
is clearly reusable across domains — it is not Ophthalmology-specific
content — so rooting it in THO is consistent with how similar
cross-domain vocabularies are treated. Filing the UTG ticket ensures the
change is tracked through the terminology governance process rather than
being done ad-hoc at publication time.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than migrating the existing `procedure-cpt` ValueSet wholesale,
define the authoritative artifact in THO (via a UTG ticket) and then
retire the Ophthalmology Retinal IG's local copy by replacing it with a
reference to the THO ValueSet. The IG's profiles would bind directly to
the THO canonical URL, and the IG would no longer ship its own
`ValueSet/procedure-cpt` resource. If there are ophthalmology-specific
subset semantics, model them in a distinct IG-scoped ValueSet that
*includes* the THO ValueSet, rather than duplicating its definition.

#### Justification

This approach addresses the same underlying concern (CPT procedure codes
belong in THO, not a spec namespace) while avoiding the long-term
maintenance cost of two artifacts that look the same but differ in
canonical URL. It also cleanly separates reusable terminology from
IG-scoped refinements, which is the pattern HL7 prefers for CPT-based
value sets given the licensing and governance sensitivities around CPT.
The trade-off is that implementers who have already bound to the IG's
current canonical must be updated, but since the ticket already demands
a canonical-URL change, any existing bindings will be invalidated under
Disposition A as well.

---

### Disposition C: Decline

#### Proposal

Close the ticket as "Not Persuasive" / out of scope, on the grounds that
the Ophthalmology Retinal IG is entitled to publish its own
IG-scoped ValueSets under its own canonical namespace, and that moving
the artifact to THO is a terminology-governance decision that belongs
to the Vocabulary WG via a UTG ticket — not a correction to the
Ophthalmology Retinal IG. Retain the existing `hl7.org/fhir`-rooted
canonical URL and direct any request to register the content in THO to
the UTG process.

#### Justification

Many published IGs define ValueSets inside their own namespace as an
accepted pattern; there is no invariant that every reusable ValueSet
must live in THO. If the true concern is discoverability and reuse, that
is resolvable by UTG registration independently of whether the IG's
current canonical URL is "wrong". Declining here avoids a breaking
canonical-URL change to an already-published IG and defers the
registration question to the appropriate governance body.

---

### Recommendation

**Recommended disposition:** A

Disposition A most faithfully honors the reporter's request and aligns
with current HL7 publication practice for non-IG-specific vocabulary: a
cross-domain CPT procedure ValueSet belongs in THO, both for governance
and for reuse outside the Ophthalmology Retinal IG. Disposition B is
attractive in principle, but because the ticket itself already implies a
canonical-URL change (and therefore a breaking change for any existing
bindings), the marginal cost of going further and collapsing the IG-local
copy is not small enough to justify deferring action here — that cleanup
can be pursued as a follow-up once the THO artifact exists. Disposition
C understates the governance concern: CPT-based value sets have specific
licensing and stewardship considerations that THO is designed to handle
centrally, so leaving the artifact under `hl7.org/fhir` indefinitely is
not a neutral choice. The workgroup should accept the change and file
the companion UTG ticket as the reporter requested.
