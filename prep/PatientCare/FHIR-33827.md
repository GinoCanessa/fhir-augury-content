## Summary

The ticket reports that the `observation-oct-rnfl` ValueSet defined in
the Ophthalmology Retinal (FHIR) implementation guide has its canonical
URL rooted under `hl7.org/fhir/...`, when by HL7 policy it should be
rooted under `terminology.hl7.org` (THO). The reporter further notes
that a corresponding UTG (Unified Terminology Governance) ticket should
be opened so the ValueSet can be properly registered and maintained in
THO. No related Jira tickets, Zulip discussions, or GitHub items were
surfaced through the cross-reference index for this ticket.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Move the `observation-oct-rnfl` ValueSet's canonical URL from the
`http://hl7.org/fhir/...` namespace into the THO namespace
(`http://terminology.hl7.org/ValueSet/observation-oct-rnfl`). Concretely:

1. File a UTG (Unified Terminology Governance) proposal/ticket against
   the `HL7/UTG` repository requesting registration of
   `observation-oct-rnfl` (and any associated CodeSystem(s)) under
   `terminology.hl7.org`.
2. Once UTG accepts and publishes the ValueSet under THO, update the
   Ophthalmology Retinal IG so that:
   - The IG no longer defines `observation-oct-rnfl` locally.
   - All references in profiles, examples, and narrative point to the
     THO canonical URL.
3. Add a redirect / deprecation note from the old `hl7.org/fhir`
   canonical for one publication cycle so existing implementations have
   a clear migration path.

#### Justification

HL7 policy is that cross-IG, reusable terminology artifacts (ValueSets
and CodeSystems that are not tightly bound to a single IG's structural
content) should live in THO so they can be governed by the Vocabulary
WG, versioned independently of any one IG, and reused without forcing a
dependency on a specific FHIR core or IG release. A ValueSet describing
OCT RNFL observations is exactly that kind of cross-cutting clinical
terminology — it could plausibly be referenced from other ophthalmology
or imaging IGs in the future, and rooting it in THO removes the
implicit "owned by Ophthalmology Retinal IG" coupling.

---

### Disposition B: Alternative Approach

#### Proposal

Keep the ValueSet defined and published from the Ophthalmology Retinal
IG (its current `hl7.org/fhir/...`-rooted canonical URL), but:

- Document explicitly in the IG that `observation-oct-rnfl` is
  IG-scoped terminology, owned by Patient Care / the Ophthalmology
  Retinal project.
- Defer THO migration until there is a concrete second consumer (another
  IG or profile) that needs to reference the ValueSet, at which point a
  UTG ticket would be raised and the canonical relocated.

#### Justification

Migrating a ValueSet to THO is not free: it requires a UTG submission,
Vocabulary WG review, and coordinated updates to every reference in the
IG, plus a migration window for implementers already binding to the
existing canonical. If `observation-oct-rnfl` is only ever consumed
within the Ophthalmology Retinal IG, leaving it under the IG's
namespace is consistent with how many narrowly-scoped IG-internal
ValueSets are managed and avoids churn for existing consumers. The
trade-off vs. Disposition A is that the canonical URL no longer
reflects HL7's preferred long-term hosting policy and a future
migration would still cost a deprecation cycle.

---

### Disposition C: Decline

#### Proposal

Decline the request and retain the existing `hl7.org/fhir`-rooted
canonical URL for `observation-oct-rnfl`, on the basis that the
Ophthalmology Retinal IG is the sole defining and consuming context
for this ValueSet and changing the canonical URL of an already-published
artifact would be a non-trivial breaking change for any implementer
that has bound to it.

#### Justification

Canonical URLs are identity for terminology artifacts. Changing them
breaks `ValueSet.url`-based binding lookups, validators that have
cached the artifact, and any Implementation Guide that references the
ValueSet by canonical. If there is no concrete pain point reported by
implementers and no second consumer asking for the artifact in THO,
the policy-purity argument may not justify the breakage. This is the
weakest of the three options because it explicitly accepts a deviation
from HL7's preferred terminology hosting model.

---

### Recommendation

**Recommended disposition:** A

Accepting the request aligns the artifact with HL7's standing policy
that broadly-applicable clinical terminology is hosted in THO and
governed by Vocabulary WG, not embedded in an IG namespace. The
reporter (Ron G. Parker) has correctly identified both the policy
issue and the operational next step (raise a UTG ticket), which means
the path forward is well-understood and inexpensive to scope. The
backward-compatibility concern raised in Disposition C is real but
manageable: the Ophthalmology Retinal IG is still maturing and the
deprecation/redirect window proposed in Disposition A gives existing
implementers a clean migration path. Disposition B effectively defers
the same work to a later — and more disruptive — moment, after more
implementers may have bound to the IG-rooted URL. Patient Care should
coordinate with Vocabulary WG on the UTG ticket and schedule the
canonical relocation for the next IG publication.
