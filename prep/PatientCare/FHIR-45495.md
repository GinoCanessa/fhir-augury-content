## Summary

Jay Lyle requests clarifying edits to the US SDOH Clinical Care IG so the
word "assessment" is used unambiguously. He finds Section 1 inconclusive
and proposes two coordinated kinds of change. First, rename the two
bullet headers in Section 1 from "Screening" to "Screening Observation"
and from "Assessment/Diagnosis" to "Assessment Observation"; if
"Assessment" is intended to cover "Assessment/Diagnosis", that needs to
be stated somewhere explicitly. Second, rename the profiles themselves
so the noun order matches the conceptual pattern: **SDOHCC Observation
Assessment → SDOHCC Assessment Observation** and **SDOHCC Observation
Screening Response → SDOHCC Screening Response Observation**. Finally,
rewrite the Assessment narrative in Section 3.4 to describe a cleaner
flow — information gathered from a patient (typically via a
standardized Questionnaire producing one or more SDOHCC Observation
Screening Responses) is used to inform the creation of one or more
SDOHCC Observation Assessments and possibly one or more SDOHCC
Conditions — dropping the previous awkward dual-path wording about
"additional" assessments and separately tracked concerns on a problem
list.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Implement all three edits verbatim in the US SDOH Clinical Care (FHIR)
IG:

1. **Section 1 bullet headers.** Rename "Screening" to "Screening
   Observation" and "Assessment/Diagnosis" to "Assessment Observation".
   Add a one-line clarifier in Section 1 stating that, in this IG,
   "Assessment" encompasses "Assessment/Diagnosis".
2. **Profile renames.**
   - `SDOHCC Observation Assessment` → `SDOHCC Assessment Observation`
     (profile title + id + canonical URL as the IG's naming rules
     dictate).
   - `SDOHCC Observation Screening Response` → `SDOHCC Screening
     Response Observation`.
3. **Section 3.4 narrative.** Replace the current Assessment paragraph
   with: *"Assessment — where information that is gathered from a
   patient (typically via a standardized Questionnaire resulting in one
   or more SDOHCC Observation Screening Responses) is used to inform
   the creation of one or more SDOHCC Observation Assessments and
   possibly one or more SDOHCC Conditions."*

Regenerate derived artifacts (Assessment Instrument Support page,
profile index, any Observation.profile references in examples and
search-parameter sections).

#### Justification

The reporter's diagnosis is accurate: the current Section 1 wording and
3.4 narrative describe two parallel paths ("these may be accompanied by
additional SDOHCC Observation Assessments" / "a SDOHCC Condition will
be defined…") in a way that reads as an enumeration of independent
events rather than a derivation flow. The proposed 3.4 rewrite states
the actual intent (screening responses inform assessments and
optionally conditions) and matches how other FHIR narratives describe
observation-derivation chains. The "<Noun> <Modifier>" profile order
("Assessment Observation", "Screening Response Observation") is also
more consistent with the IG's own prose, which always says "an
observation that is an assessment" rather than "an assessment that is
an observation".

### Disposition B: Alternative Approach — Prose & Narrative Only (Keep Profile Names)

#### Proposal

Accept the **narrative** changes but **do not rename the profiles**:

1. Apply the Section 1 header change and the explicit Section 1
   statement that "Assessment" includes "Assessment/Diagnosis" in this
   IG.
2. Apply the Section 3.4 rewrite exactly as proposed.
3. Leave `SDOHCC Observation Assessment` and `SDOHCC Observation
   Screening Response` unchanged. Instead, add a glossary entry (or a
   sentence at the first use in Section 1) explicitly mapping the
   informal phrases "Assessment Observation" / "Screening Response
   Observation" to the existing profile names.
4. Open a companion follow-up coordinating with FHIR-49197, FHIR-46247,
   FHIR-40161, FHIR-49645, and FHIR-51516 so the terminology
   clarification is applied consistently wherever "assessment" appears.

#### Justification

Renaming a published profile changes its title, id, and (depending on
IG naming rules) its canonical URL and `StructureDefinition` filename.
That breaks inbound references from US Core (see FHIR-49645, where the
SDOHCC profile is described as *derived from US Core Observation
Screening Assessment 7.0.0* — the parent keeps "Observation" first) and
from any implementer that pinned the current URL. The underlying
problem the reporter is solving is readability, and the Section 1 +
Section 3.4 changes plus a glossary entry solve the readability
problem at a fraction of the blast radius. If the Patient Care WG and
US Core ever agree to flip the noun order across the family, that
should be a single coordinated editorial change, not a one-IG rename.

### Disposition C: Decline

#### Proposal

Decline the profile-rename portion and the Section 1 header rename.
Close as "Not Persuasive" with a response noting that (a) SDOHCC's
profile naming deliberately parallels US Core (`Observation Screening
Assessment`, etc.) and (b) the requested Section 1 headers
("Screening Observation" / "Assessment Observation") introduce two new
terms that are not used elsewhere in the IG, which would create more
confusion than it removes.

#### Justification

The argument for declining rests on two points. First, SDOHCC's profile
family intentionally mirrors the US Core parent `Observation Screening
Assessment` — see FHIR-49645. Flipping the word order only in SDOHCC
would make the "derived from" chain look like a different concept.
Second, introducing "Screening Observation" and "Assessment
Observation" in Section 1 creates terms that are not the profile
titles, not US Core's terms, and not community-adopted phrases, which
risks making Section 1 **more** ambiguous, not less. Declining here
doesn't preclude future editorial cleanup under a broader harmonisation
ticket.

---

### Recommendation

**Recommended disposition:** B

The reporter has correctly identified a real readability problem, and
the Section 3.4 rewrite in particular is a clear, scope-contained
improvement that should be adopted. The profile renames, however, are
disproportionate to the underlying clarity issue: they affect the
profile id, canonical URL, and the derivation relationship with US
Core's `Observation Screening Assessment` (FHIR-49645), and they
collide with active work on the same profile family
(FHIR-51516, FHIR-46247). Disposition B keeps the high-value editorial
fixes — Section 1 header clarifier, Section 3.4 narrative, and an
explicit glossary mapping — while avoiding a breaking rename that would
need coordinated republication with US Core and inbound implementers.
If the WG later decides the "<Type> Observation" word order is worth
adopting globally, that decision should be taken once, across
SDOHCC + US Core + related PC IGs, under a dedicated harmonisation
ticket rather than as a side effect of this one.
