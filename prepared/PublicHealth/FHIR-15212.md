## Details

This ticket was submitted as part of the 2018-Jan eCR ballot cycle (comment #28) against the Electronic Case Reporting (eCR) Implementation Guide. The commenter (Catherine Staes) raises two concerns about the occupational data value sets used in the eCR specification:

1. **Scope of occupation value sets**: The commenter questions why only the occupational value set is included, implying that additional occupation-related value sets (e.g., industry codes) should also be present.
2. **Incorrect code system for occupation**: The commenter notes that **SOC (Standard Occupational Classification) codes are not the correct codes** used for occupation in public health reporting. Instead, the **CDC Census occupation codes** should be used.

The resolution is marked **Persuasive with Modification**, indicating the workgroup agreed the concern was valid but applied a modified approach to address it. The specification this was filed against (Electronic Case Reporting IG) is now deprecated, and subsequent versions of the eCR IG have incorporated occupation and industry data handling through the Occupational Data for Health (ODH) profiles.

## Proposed Dispositions

### Disposition A: Accept as Persuasive with Modification (Apply the Existing Resolution)

**Rationale:** The original resolution of "Persuasive with Modification" is appropriate. The commenter correctly identified that SOC codes are not the standard for occupation coding in public health case reporting—CDC Census occupation codes are the accepted standard. The eCR IG should reference the correct code system. Since the original eCR specification is now deprecated and occupation data is handled through the Occupational Data for Health (ODH) profiles in current eCR IGs, the modification has effectively been addressed in successor specifications.

**Action:** Confirm the Persuasive with Modification resolution. Verify that the current eCR IG (or its successor) properly references CDC Census occupation codes (via ODH profiles) rather than SOC codes. Mark as Applied if the successor specification has addressed this.

### Disposition B: Accept as Persuasive (Full Agreement)

**Rationale:** The commenter's points are technically correct on both counts: (1) additional occupation-related value sets beyond just the occupational value set should be included, and (2) CDC Census codes are the correct code system for occupation, not SOC. Upgrading to full Persuasive acknowledges the comment was entirely valid without modification.

**Action:** Accept as Persuasive. Document that the concern has been addressed in successor eCR IG versions through adoption of ODH profiles which use the correct CDC Census occupation and industry code systems. Transition status to Applied.

### Disposition C: Decline (Not Persuasive)

**Rationale:** The specification this was filed against is deprecated. The concern, while technically valid at the time, is moot because the eCR IG has been superseded by newer versions that handle occupation data differently through ODH. No further action is needed on a deprecated specification.

**Action:** Close as Not Persuasive due to the deprecated status of the target specification. Note that the concern has been organically resolved in successor specifications.

## Recommendation

**Disposition A: Accept as Persuasive with Modification** is recommended. This aligns with the existing resolution already assigned to the ticket and respects the original workgroup decision. The commenter's concern was valid—CDC Census codes, not SOC codes, are the correct code system for occupation in public health reporting—but the "with Modification" qualifier appropriately reflects that the fix may have been applied in a broader or different manner than the commenter originally envisioned (i.e., through adoption of ODH profiles in successor IGs rather than a simple value set swap). The ticket should be transitioned to **Applied** status, confirming the resolution has been implemented in the current eCR specification lineage.
