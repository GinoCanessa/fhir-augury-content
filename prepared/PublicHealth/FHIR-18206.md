## Description

The submitter raises two related concerns about the BSeR IG:

1. **Observation-to-DAM mapping gap:** The FHIR Observations listed in the Structure Definitions do not clearly map to the Domain Analysis Model (DAM). The DAM's final pages list observations by referral program type, but there is no "Arthritis Feedback" observation. An arthritis grouper exists, but the program-specific feedback data types are more granular than what the profiles capture.

2. **Missing terminology bindings:** The observations should be linked to specific LOINC codes. The submitter asks whether the terminology has not been fully fleshed out, or whether SNOMED code 3723001 (from a follow-up comment) is intended as the code for the arthritis profile.

## Analysis

This ticket identifies a foundational gap in the BSeR IG: the disconnect between the DAM's observation taxonomy and the FHIR profiles, compounded by absent or inaccessible terminology bindings. Several related tickets (FHIR-20832, FHIR-20877, FHIR-42479) confirm this is a recurring theme — implementers cannot determine which codes to use for BSeR observations because the IG does not fix or even suggest LOINC codes, answer types, or value sets.

Notable context:
- **FHIR-20832** (Lack of BSeR Terminology) and **FHIR-20877** (Referral Feedback Profile Clarifications), both by the same reporter, were resolved as **Persuasive** and **Applied**, indicating the workgroup agreed with the underlying concerns and made changes in subsequent IG versions.
- **FHIR-42479** (filed later by Lloyd McKenzie) is still **Triaged/Unresolved** and re-raises the same observation underspecification issue with greater detail, suggesting the problem was not fully resolved by earlier fixes.
- The ticket is now over 6 years old and still Triaged, suggesting it may have been partially addressed by the related Applied tickets but never formally dispositioned.

## Proposed Dispositions

### Disposition A: Persuasive (Recommend)
**Rationale:** The concerns are valid and well-documented. The DAM-to-profile mapping gap and missing terminology bindings were real issues at the time of filing. Related tickets (FHIR-20832, FHIR-20877) were resolved as Persuasive, confirming the workgroup agreed. Accept this ticket as Persuasive, document the DAM-to-Observation mapping for arthritis feedback, and ensure the relevant LOINC/SNOMED bindings are specified in the profiles. If subsequent IG versions have already addressed portions of this, note the specific changes made.

**Action:** Add explicit DAM-to-profile traceability for arthritis observations. Specify LOINC or SNOMED bindings for the Arthritis Feedback observation profile. Update the IG narrative to explain how observations map to DAM program types.

### Disposition B: Persuasive with Modification
**Rationale:** The core concern is valid, but the scope should be narrowed. Rather than requiring a full DAM-to-profile mapping exercise (which is partly addressed by FHIR-20832 and FHIR-20877), resolve this ticket by adding a clarifying note or cross-reference in the IG that explains the relationship between the DAM observation taxonomy and the FHIR profiles, and point implementers to the appropriate terminology resources. The specific SNOMED code question (3723001) from the comment should be answered directly.

**Action:** Add narrative clarification in the IG explaining how DAM observations relate to FHIR profiles. Respond to the SNOMED code 3723001 question. Cross-reference related applied changes from FHIR-20832 and FHIR-20877.

### Disposition C: Duplicate / Applied by Other Tickets
**Rationale:** The substantive concerns raised here were subsequently addressed through FHIR-20832 (Lack of BSeR Terminology — Persuasive, Applied) and FHIR-20877 (Referral Feedback Profile Clarifications — Persuasive, Applied). If the workgroup confirms that changes from those tickets adequately cover the DAM mapping gap and terminology binding questions raised here, this ticket can be closed as addressed by prior work. However, FHIR-42479 suggests gaps remain.

**Action:** Review changes applied for FHIR-20832 and FHIR-20877. If they sufficiently address the DAM mapping and terminology binding concerns, close as duplicate. If gaps remain (as suggested by FHIR-42479), escalate or merge with FHIR-42479.

## Recommendation

**Disposition B: Persuasive with Modification** — The underlying concerns are valid and shared by multiple other tickets. However, significant related work has already been Applied (FHIR-20832, FHIR-20877), and this ticket is over 6 years old. The most practical path is to accept the ticket with modification: acknowledge the valid concerns, answer the specific SNOMED code question, add narrative clarification about DAM-to-profile mapping, and cross-reference the related applied tickets. Any remaining unresolved observation specification gaps should be tracked through FHIR-42479 rather than this ticket.
