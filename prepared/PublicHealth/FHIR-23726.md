## Description

The reporter notes that all value sets defined in the HAI-LTCF (Healthcare Associated Infection Reports – Long Term Care Facilities) IG display the message **"This value set is not used."** The reporter asks:

1. If this is true, why do the value sets exist in the IG?
2. If it is not true, how are they actually used?
3. Please clarify the situation.

This is a documentation/usability concern: value sets that claim to be unused create confusion for implementers trying to understand the IG.

## Proposed Dispositions

### Disposition A: Persuasive – Remove Unused Value Sets

**Resolution:** Accept the comment as persuasive. If the value sets are genuinely not used by any profile, Questionnaire, or QuestionnaireResponse in the HAI-LTCF IG, remove them from the IG entirely. Value sets that serve no binding or reference purpose should not be published, as they create confusion for implementers.

**Rationale:** The reporter raises a valid concern. Publishing value sets labelled "not used" adds noise to the specification and confuses implementers about whether the value sets are needed for conformance. If they are truly unused, removing them simplifies the IG.

**Impact:** Removal of value set resources from the IG. QA review needed to confirm no hidden bindings or indirect references exist before deletion.

### Disposition B: Persuasive with Modification – Correct the Documentation

**Resolution:** Accept the comment as persuasive with modification. The value sets _are_ used (e.g., bound within Questionnaire items or referenced from CodeSystem definitions), but the generated documentation text "This value set is not used" is a tooling artifact or metadata oversight. Update the value set metadata so that the IG publisher correctly reflects their usage, and add narrative text clarifying where each value set is bound.

**Rationale:** It is possible the value sets are in fact used but the IG publisher rendering does not detect the usage (e.g., if bindings are in Questionnaire resources rather than StructureDefinitions). Correcting the metadata and adding explicit usage notes would address the implementer confusion without removing potentially necessary terminology resources.

**Impact:** Metadata and narrative updates to value set resources. May require tooling investigation to understand why "not used" is displayed.

### Disposition C: Not Persuasive – Value Sets Are Intentionally Published for Future Use

**Resolution:** Reject the comment as not persuasive. The value sets are intentionally included for forward compatibility or alignment with the parent HAI IG, even though they are not currently bound in the HAI-LTCF IG. The "not used" label is accurate for the current IG version.

**Rationale:** Some IGs publish value sets that align with a parent specification or are anticipated for future use. The existing label is technically correct.

**Impact:** No change to the IG. However, this does not address the implementer confusion raised by the reporter.

## Recommendation

**Disposition B: Persuasive with Modification** is recommended.

The HAI-LTCF IG has been published through multiple STU versions since this ticket was filed (STU 1.0 in Jan 2023, STU 1.1 in Apr 2023), yet this ticket remains unresolved. The most likely explanation is that the value sets _are_ used within Questionnaire items but the IG publisher did not detect those bindings, resulting in the misleading "not used" label. The work group should:

1. Audit each HAI-LTCF value set to determine if it is bound in any Questionnaire, QuestionnaireResponse, or profile.
2. For value sets that are used, correct the metadata so the IG publisher reflects the binding correctly, and add narrative clarifying usage.
3. For any value sets that are genuinely unused, remove them per Disposition A.

This approach addresses the reporter's core concern (implementer confusion) while preserving legitimately needed terminology resources. The related ticket FHIR-23727 (unclear "other resources") suggests a broader documentation clarity effort may be warranted for this IG.
