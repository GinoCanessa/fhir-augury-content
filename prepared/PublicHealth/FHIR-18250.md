**Status:** Triaged | **Type:** Change Request | **Priority:** Medium
**Specification:** US Bidirectional Services eReferrals (BSeR) (FHIR)
**Work Group:** Public Health
**Reporter:** Craig Newman | **Assignee:** Unassigned
**Created:** 2018-09-24 | **Updated:** 2019-02-23

All links to the common transaction profiles in the BSeR IG (September 2018 ballot) are broken, returning 404 File Not Found errors. The reporter identified specific broken URLs pointing to pages within the `http://hl7.org/fhir/us/bser/2018Sep/` path, including:

- `ReferralRequestTransactionProfiles.html`
- `ReferralFeedbackTransactionProfiles.html`
- `ImplementationProfileBundles.html`

Additionally, links on the `CommonTransactionProfiles.html` page are also reported as returning 404 errors.

## Proposed Dispositions

### Disposition A: Persuasive (Fix the broken links)

**Resolution:** Accept the ticket as Persuasive and fix all broken links to common transaction profiles in the current published version of the BSeR IG.

**Rationale:** The broken links prevent reviewers and implementers from navigating to critical content about referral request and feedback transaction profiles. This is a clear defect in the published IG. Related tickets (FHIR-22527, FHIR-22565) addressing similar broken link issues in BSeR were already resolved as Persuasive, establishing precedent.

**Actions required:**
1. Audit all links on the Common Transaction Profiles page and related pages in the current BSeR IG build.
2. Update broken URLs to point to the correct pages or rename/restructure pages to match existing links.
3. Verify links resolve correctly in the next IG build.

### Disposition B: Applied — Already Resolved in Subsequent Versions

**Resolution:** Mark as Applied, noting that the broken links from the September 2018 ballot have been addressed through subsequent BSeR IG publications and related ticket fixes (FHIR-22527, FHIR-22565).

**Rationale:** This ticket was filed against the September 2018 ballot version. The BSeR IG has gone through multiple subsequent publication cycles. The related tickets FHIR-22527 and FHIR-22565, which addressed broken links in the same IG, have already been Applied. It is likely that the page structure and links were corrected as part of those or other IG updates.

**Actions required:**
1. Verify that the current BSeR IG build has working links to the transaction profile content.
2. If confirmed resolved, close as Applied with a note referencing the subsequent fixes.

### Disposition C: Not Persuasive — Historical Ballot Artifact

**Resolution:** Close as Not Persuasive, noting this applies only to an archived ballot version that is no longer the current published IG.

**Rationale:** The broken links are specific to the `2018Sep` ballot path, which is a historical snapshot. The current BSeR IG has been restructured in subsequent versions. Fixing links in archived ballot versions is not standard practice.

**Actions required:** None.

## Recommendation

**Disposition B: Applied — Already Resolved in Subsequent Versions** is recommended.

This ticket was filed in September 2018 against the BSeR September 2018 ballot. Two closely related broken-link tickets in the same BSeR IG (FHIR-22527 and FHIR-22565) have already been resolved as Persuasive and Applied. Given that the BSeR IG has undergone multiple publication cycles since 2018, and the page structure has been significantly revised, the specific broken links reported here have almost certainly been addressed. The workgroup should verify the current IG build has working transaction profile links and close this long-standing ticket accordingly.
