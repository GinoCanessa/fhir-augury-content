## Proposed Dispositions

### Disposition A: Persuasive (Remove the class diagram section)

**Action:** Accept the request as submitted. Remove the class diagram section from the BSeR IG entirely.

**Rationale:** The submitter's concern is straightforward — if the class diagram does not connect to the profiles or add implementer understanding, it is extraneous content that may cause confusion. Removing it simplifies the IG and reduces maintenance burden. The companion ticket FHIR-18255 reinforces this concern by noting the IG lacks narrative tying diagrams to artifacts, suggesting the diagram was not serving its intended purpose.

**Applied Change:** Delete the class diagram section from the relevant page of the BSeR IG.

### Disposition B: Persuasive with Modification (Replace with a meaningful diagram or narrative)

**Action:** Rather than simply removing the class diagram, replace it with a diagram or narrative that meaningfully connects to the BSeR profiles and workflow.

**Rationale:** A class diagram *could* be valuable if it accurately reflects the relationships between BSeR profiles (e.g., ServiceRequest, Task, Composition, etc.). The issue is not that a diagram exists, but that the current one is disconnected from the profiled resources. Replacing it with a profile-centric diagram or a brief narrative overview would address the concern while retaining visual guidance for implementers.

**Applied Change:** Replace the existing class diagram section with a revised diagram showing BSeR profile relationships, or with narrative text describing how the profiles interrelate.

### Disposition C: Not Persuasive (Retain the class diagram)

**Action:** Keep the class diagram section as-is.

**Rationale:** The class diagram may provide useful context for readers coming from a UML or systems-modeling background. The connection to profiles may not be immediately obvious but could still be valuable as background context. However, this disposition is weak given that the submitter (an experienced IG reviewer) found the diagram unhelpful and no comments defend its inclusion.

**Applied Change:** No change. Optionally add a brief introductory sentence explaining the diagram's purpose.

## Recommendation

**Disposition A: Persuasive** — Remove the class diagram section.

This is a 2018 ballot comment on the BSeR IG that has remained triaged and unresolved for over six years. The request is clear, narrowly scoped, and low-risk. The class diagram is reported as disconnected from the IG's profiles, and removing unnecessary content simplifies the specification. The companion ticket FHIR-18255 (also from Eric Haas, same ballot cycle) corroborates that diagrams in this IG lacked meaningful ties to the artifacts. If the work group later determines a diagram is valuable, it can be reintroduced with proper profile linkage (per Disposition B), but the immediate action should be removal of the unhelpful content.
