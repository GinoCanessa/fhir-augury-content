## Summary

The submitter (Genevieve Luensman, NIOSH-NPPTL) requests the addition of use cases demonstrating hospital PPE (Personal Protective Equipment) data collection, reporting, and automated computation within the Situation Awareness for Novel Epidemic Response (SANER) Implementation Guide. The proposed use cases cover collecting hospital PPE data measures through an automated central platform (section 2.3.1), defining the PPE collection process flow (2.3.1.1), accessing PPE measure reports via dashboards and queries (2.3.2), and automating measure computation to reduce manual data entry burden (2.3.4). The ticket was originally resolved as "Considered for Future Use" during the STU1 ballot cycle in April 2021, then reopened post-publication of STU1 in September 2021, and subsequently resolved as "Not Persuasive" with the rationale that the content is not within the scope of the SANER IG or the Helios project.

## Details

**Description:**

It is important to provide a Use Case for Collecting Hospital PPE Measures including the steps for collecting PPE data, measuring reports, and automating the data received. NIOSH-NPPTL has an example of a use case that can be provided as an example of a supply use case to complement the existing patient use cases.

- **2.3.1. Collecting Hospital PPE Data Measures:** This use case addresses the collection of PPE from several hospital systems in an automated system that communicates supply back to hospitals and helps federal, state, and local stockpiles anticipate shortages and demand. A central, automated system is able to collect multiple PPE data elements such as N95 respirators, surgical masks, and face shields by examining the "eaches" received by model number, manufacturer, and NIOSH approval number, and then reports these to a trust data platform for hospital systems. This use case is supported by PPE current inventory reported.

- **2.3.1.1 - Collecting Hospital PPE and Measures Process Flow:** PPE report data can come from multiple applications or tools across hospitals, but reported to one trust data platform for normalization and standardization. Provides PPE Capacity including total number of PPE available, PPE being replenished, and warning levels. In order to automate this PPE inventory information, a central reporting system should be made available to accept data, which can then be aggregated. NIOSH-NPPTL can provide a detailed overview of such a process flow.

- **2.3.2. Accessing PPE Measure Reports:** Any federal, state, or local hospital or stockpile group should be able to query a reporting system to report on PPE supply and shortages for a particular hospital system, region, or state. Dashboards that are accessible can be provided and updated regularly using validated algorithms for current and replenished PPE supplies.

- **2.3.4. Automating Measure Computation:** A neutral trust data platform through an automated procedure can compute and report PPE measure data. An automated measure reduces burden on individual hospitals and manual data entry steps. Additionally, a PPE data standard will make PPE data computation standardized and easier to automate and communicate supply and demand.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal
Add new use case sections to the SANER IG covering hospital PPE data collection (2.3.1), PPE collection process flow (2.3.1.1), PPE measure report access (2.3.2), and automated PPE measure computation (2.3.4), drawing on NIOSH-NPPTL expertise and examples for supply-side inventory use cases.

#### Justification
The request provides detailed, actionable use case descriptions that address a genuine gap in the SANER IG — the specification currently focuses primarily on patient-centric use cases and does not comprehensively cover supply-chain scenarios for PPE. Hospital PPE tracking was a critical need exposed during the COVID-19 pandemic, and NIOSH-NPPTL is the authoritative federal body on this topic. Including these use cases would strengthen the IG's practical applicability and align with the original STU1 ballot cycle intent (the 22-0-0 vote to consider for future use suggests broad agreement on the value of the content).

---

### Disposition B: Alternative Approach — Reference External PPE Standards Without Expanding IG Scope

#### Proposal
Rather than adding full PPE data collection use cases to the SANER IG, add an informative section or appendix that references NIOSH-NPPTL PPE data standards and external supply-chain reporting frameworks. Include a brief description of how the SANER Measure and MeasureReport resources could be adapted for PPE supply tracking, without defining new normative use case sections.

#### Justification
The SANER IG's primary scope is situation awareness for epidemic response (bed capacity, ventilator availability, etc.), not supply-chain logistics. Full PPE inventory tracking — covering model numbers, manufacturer details, NIOSH approval numbers, and multi-hospital aggregation platforms — may be better addressed by a dedicated supply-chain IG or a separate track of work (e.g., within the Public Health work group or a NIOSH-led initiative). An informative reference keeps the SANER IG focused while acknowledging the need and pointing implementers to the right resources. This approach also avoids the scope concerns raised in the "Not Persuasive" resolution.

---

### Disposition C: Decline

#### Proposal
Decline the request, affirming the "Not Persuasive" resolution. The PPE supply-chain use cases are out of scope for the SANER IG and the Helios project, and no subsequent work has been undertaken to bring this content into scope.

#### Justification
The ticket was resolved as "Not Persuasive" with the rationale that PPE supply-chain data collection is outside the scope of both the SANER IG and the Helios project. The SANER IG is focused on situational awareness measures (e.g., bed and ventilator capacity), not on detailed supply-chain inventory management. Since the original "Considered for Future Use" resolution in April 2021 and the subsequent reopening, no concrete work has materialized to incorporate these use cases, suggesting the work group has not prioritized this direction. The request, while valuable, would be better served by a dedicated effort outside the SANER IG.

---

### Recommendation

**Recommended disposition:** C (Decline)

The "Not Persuasive" resolution is well-founded. While the submitter raises a legitimate need for PPE supply-chain tracking — one that was particularly acute during the COVID-19 pandemic — the SANER Implementation Guide is scoped to situational awareness measures such as bed capacity, ventilator availability, and patient throughput. Detailed PPE inventory management, including tracking by model number, manufacturer, and NIOSH approval number across multi-hospital aggregation platforms, represents a fundamentally different domain (supply-chain logistics) that warrants its own dedicated specification effort. The work group previously acknowledged the value of the idea with a 22-0-0 "Considered for Future Use" vote, but no follow-through occurred over multiple years, reinforcing that this content does not fit naturally within the current IG's scope. Affirming the decline is the most defensible path, while the work group could separately encourage NIOSH-NPPTL to propose a new IG or project focused on PPE supply-chain interoperability.
