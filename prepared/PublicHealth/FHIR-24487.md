## Description

The submitter requests that the IG Publisher suppress the "definition" column in rendered structure/element tables when none of the elements in the profile or resource have definitions populated. This is identified as a **tooling issue** — the HAI-LTCF IG displays empty "definition" columns that add visual clutter without conveying information.

## Analysis

This is a longstanding tooling request (nearly 6 years old) that was originally deferred because it requires an IG Publisher change rather than an IG content change. The core ask is simple and well-defined: when a structure table has no definitions for any element, the "definition" column should be omitted from the rendered output.

**Key considerations:**
1. **Scope**: This is purely a rendering/tooling issue. It requires no changes to FHIR resource definitions, profiles, or conformance artifacts.
2. **IG Publisher dependency**: The fix must come from the FHIR IG Publisher (maintained by Grahame Grieve / HL7). The HAI-LTCF IG cannot independently resolve this.
3. **Recent IG Publisher activity**: The March 2025 Zulip thread shows Grahame actively fixing similar column rendering bugs in the IG Publisher, suggesting the tooling landscape may have evolved since the original 2019 deferral.
4. **Reopened post-STU1.1**: The 2023 reopening indicates the WG still considers this relevant and wants it tracked.

## Proposed Dispositions

### Disposition A: Refer to FHIR Infrastructure (Recommended)
**Resolution: Referred**

Refer this ticket to the **FHIR Infrastructure** work group (or directly to the IG Publisher maintainer) as a tooling enhancement request. The HAI-LTCF IG cannot resolve this independently — it requires the IG Publisher to detect when all definition cells in a table are empty and suppress the column. The Public Health WG should file or link to a specific IG Publisher GitHub issue (e.g., in `HL7/fhir-ig-publisher`) to track implementation.

**Rationale:** This is explicitly a tooling issue, as acknowledged in the original 2019 vote. The appropriate path is referral to the team that maintains the IG Publisher, especially given recent evidence of active column rendering fixes.

### Disposition B: Considered for Future Use
**Resolution: Considered for Future Use**

Retain the original disposition. Acknowledge that this remains desirable but is dependent on IG Publisher tooling changes that are outside the WG's control. Re-evaluate when the IG Publisher adds support for conditional column suppression.

**Rationale:** The WG has no mechanism to force a tooling change. Deferring until the capability exists avoids leaving the ticket in an indefinite "Triaged" state with no actionable next step within the WG's scope.

### Disposition C: Not Persuasive — Resolved via Workaround
**Resolution: Not Persuasive**

Close as not persuasive, noting that IG authors can use custom page templates or narrative overrides to control rendered output. The IG Publisher's default rendering is a convenience, and individual IGs can customize their presentation layer.

**Rationale:** While valid, this disposition is the weakest option — the original submitter specifically identified this as a tooling gap, and workarounds impose maintenance burden on individual IG authors rather than solving the systemic issue.

## Recommendation

**→ Disposition A: Refer to FHIR Infrastructure**

This is unambiguously a tooling issue that the Public Health WG cannot resolve within the HAI-LTCF IG. The most productive path is referral to the IG Publisher maintainers, ideally accompanied by filing a GitHub issue in `HL7/fhir-ig-publisher` so the request has a concrete tracking artifact. Recent IG Publisher activity on column rendering bugs (March 2025) suggests the timing may be favorable.
