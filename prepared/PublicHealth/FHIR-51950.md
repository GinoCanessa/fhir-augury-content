## Summary

This ticket requests a review of the modeling approach for the **US PH Characteristics of Home Environment** profile, specifically whether it should use **Condition vs Observation** as its base resource. The profile is mapped to USCDI+ data elements for **Congregate Living** and **Housing Instability**. The submitter notes that housing insecurity is well defined and coded within the **Gravity SDOH** project and suggests exploring alternate modeling approaches based on Gravity and related work.

The ticket was previously resolved as **"Considered for Future Use"** (vote: 32-0-6, movers Sarah Gaunt / Dan Rutz) on 2025-12-04 with the note *"We will consider looking into this modeling in the future."* It was subsequently **reopened** post-STU2 publication by lynn_laakso on 2026-02-17.

## Details

### Description
The core question is whether the Characteristics of Home Environment profile should be modeled as a **Condition** or an **Observation**, given its mapping to USCDI+ Congregate Living and Housing Instability data elements. The submitter points out that:

- Alternate modeling based on Gravity and related efforts should be explored

## Proposed Dispositions

### Disposition A: Accept — Align with Gravity SDOH Modeling (Persuasive)

**Resolution:** Persuasive
**Action:** Refactor the US PH Characteristics of Home Environment profile to align with the Gravity SDOH modeling approach for housing-related social determinants. This would involve:
- Evaluating whether an Observation-based model (consistent with Gravity's SDOH Clinical Care IG screening/assessment pattern) better fits housing instability and congregate living data
- Adopting Gravity-defined value sets for housing insecurity coding
- Ensuring the profile remains compatible with USCDI+ Congregate Living and Housing Instability mappings

**Rationale:** The Gravity project has established well-defined, consensus-based models for SDOH domains including housing. Aligning with Gravity reduces duplication, improves interoperability, and leverages existing coded terminology. The STU2 publication provides a natural opportunity to reassess the modeling approach before broader adoption.

### Disposition B: Accept with Scoped Investigation — Comparative Analysis First (Persuasive with Modification)

**Resolution:** Persuasive with Modification
**Action:** Accept the change request but scope it to first produce a comparative analysis of the current Condition-based model vs an Observation-based model (aligned with Gravity SDOH). The analysis should evaluate:
- Data capture fidelity for housing instability and congregate living
- Compatibility with existing implementations and USCDI+ mappings
- Impact on downstream consumers (e.g., case reporting, public health surveillance)
- Alignment with how FHIR-51949 (Chief Complaint) resolves its own Condition vs Observation modeling question

The actual profile changes would be deferred to a subsequent ballot cycle based on the analysis findings.

**Rationale:** A formal analysis reduces risk by ensuring the modeling change is well-justified before implementation. The Condition vs Observation question affects multiple profiles (see FHIR-51949), so a coordinated approach may be beneficial.

### Disposition C: Decline — Defer to Future Cycle (Not Persuasive)

**Resolution:** Not Persuasive
**Action:** Retain the current modeling and revisit in a future cycle. The ticket was previously resolved as "Considered for Future Use" with strong support (32-0-6). While it has been reopened post-STU2, the current modeling is functional and changing it introduces breaking changes for implementers.

**Rationale:** The current profile is published and in use. Changing from Condition to Observation (or vice versa) is a significant breaking change that requires careful coordination. The Gravity SDOH project is still evolving, and waiting for further maturation would reduce the risk of needing additional changes later.

## Recommendation

**Disposition B: Persuasive with Modification** — A comparative analysis is the most prudent path forward. The reopening post-STU2 signals renewed interest, and the Gravity alignment argument has merit. However, changing the base resource type (Condition vs Observation) is a significant structural decision that warrants formal evaluation rather than immediate refactoring. This approach respects the original ballot sentiment (which deferred, not rejected, the change), provides a framework for coordination with the parallel FHIR-51949 modeling question, and positions the work group to make a well-informed decision in the next ballot cycle.
