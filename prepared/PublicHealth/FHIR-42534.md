## Summary

This ticket poses a foundational feasibility question for the BSeR IG: Is it realistic for community-based organizations (CBOs) to offer secure FHIR Subscriptions to data in their systems? It further asks how subscriptions would work when an intermediary (such as a CBO hub or HIE) is involved, and specifically when that intermediary acts as a pure pass-through. The question was raised during ballot review and has been extensively discussed across several related tickets.

## Details

### Description

> Is it realistic for community-based organizations (CBOs) to offer secure subscriptions to data in their systems? How would that work when there is an intermediary? When the intermediary is a pure pass through?

### Comments

### Related Ticket Context

The BSeR_SUBSCRIPTIONS label group contains several closely related tickets that provide substantial context:

- **FHIR-42127** (Triaged, Highest priority) — Contains the workgroup's direct response to this question: *"It is not realistic at this point to expect CBOs to offer secure subscriptions to data in their systems EXCEPT if the CBO is served by a CBO Network service (such as Unite Us or FindHelp/AuntBertha). In those cases, the CBO Network could get the information from the CBO and offer it to others."* This ticket also covers intermediary behavior, noting that both "act on behalf of CBO" and "pass-through" modes should be represented in the IG.

- **FHIR-41811** (Resolved – No Change) — Brett Marquard noted that Subscriptions are relatively new, IHE 360X works without them, and pilot exchanges should be conducted before defining a subscription-based approach. A 2025 comment notes this work is no longer part of the eCR contract.

- **FHIR-42450** (Resolved – No Change, Highest priority) — Lloyd McKenzie provided a detailed technical analysis: CBOs that can expose a FHIR server interface could support subscriptions without major additional effort. Security is needed during subscription establishment and subsequent queries, but notifications themselves (which should only say "something changed" rather than carrying data) don't need to be secured. For intermediaries, a "fat" intermediary would relay subscriptions/notifications/queries, while a pass-through just does routing and URL adjustment.

- **FHIR-42148** (Resolved – Persuasive) — Identified that the transport options page was missing a reference to Subscriptions, which was accepted as a change.

- **FHIR-42471** (Triaged, Highest priority) — Lloyd McKenzie noted the IG needs to define or reference SubscriptionTopic(s) that would be used, and these should not vary across implementations.

## Proposed Dispositions

### Disposition A: Resolve as Considered – No Action Required

**Rationale:** This ticket is a question, not a change request. The question has already been substantively answered in related ticket FHIR-42127, where the workgroup responded that it is not realistic for CBOs to offer subscriptions unless they are served by a CBO Network (e.g., Unite Us, FindHelp). Additionally, FHIR-42450 (also resolved – no change) provided Lloyd McKenzie's detailed technical analysis of how subscriptions could work with intermediaries. The question has been fully addressed through these parallel tickets, and no specification change is needed for a question-type ticket. Resolving with "Considered – No action required" acknowledges the question was reviewed and addressed through existing workgroup responses.

### Disposition B: Resolve as Considered with Informative Text Addition

**Rationale:** While the question itself does not require a normative change, the workgroup could add an informative note or guidance section to the BSeR IG that addresses CBO subscription readiness. This note would incorporate the consensus view from FHIR-42127 (CBOs generally cannot offer subscriptions except through CBO Network intermediaries) and FHIR-42450's technical guidance (security model for notifications vs. queries). This would make the IG self-contained on this topic and reduce future ballot questions. This could be resolved as "Persuasive with Mod" with a note to add informative guidance text to the transport options or implementation guidance section.

### Disposition C: Defer Pending SubscriptionTopic Definition

**Rationale:** The question of CBO subscription feasibility is closely tied to the unresolved FHIR-42471 (Define SubscriptionTopic(s)), which remains triaged at Highest priority. Rather than resolving this question in isolation, defer it until the SubscriptionTopic work is complete, at which point the subscription architecture for CBOs and intermediaries will be better defined. This ensures the answer to the feasibility question is grounded in the actual subscription design. However, this risks keeping the ticket open indefinitely given that FHIR-41811 notes the subscription work is no longer part of the eCR contract.

## Recommendation

**Disposition A: Resolve as Considered – No Action Required.**

The question has been thoroughly answered across the related BSeR_SUBSCRIPTIONS tickets. FHIR-42127 provides the workgroup's definitive response (CBOs cannot realistically offer subscriptions except through CBO Networks), and FHIR-42450 provides the technical analysis of how intermediaries would handle subscriptions. Both of those tickets have already been resolved with "No Change." Since FHIR-42534 is a Question-type ticket and does not request a specification change, the most appropriate resolution is to acknowledge that the question has been considered and addressed. The 2025 comment on FHIR-41811 that this work is no longer part of the eCR contract further supports closing this ticket without additional action.
