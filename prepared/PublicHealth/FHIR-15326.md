## Description

The commenter notes the following existing wording in the Electronic Case Reporting (eCR) IG:

> "Health Data Networks (HDNs) like the eHealth Exchange, Carequality, CommonWell, and DirectTrust also may play important roles."

The commenter asks whether this text needs to be updated to reflect the Trusted Exchange Framework and Common Agreement (TEFCA) — a proposed interoperability framework that was recently announced at the time of filing (early 2018). No specific replacement wording was provided.

This was filed as part of the **2018-Jan eCR ballot** (item #120).

## Analysis

This ticket was filed during the January 2018 eCR ballot cycle against the **Electronic Case Reporting** specification, which is now marked as **[deprecated]**. The commenter raised a forward-looking concern: whether the IG's reference to specific Health Data Networks should be updated to mention TEFCA, which had just been proposed at the time.

Key observations:

1. **Specification is deprecated.** The target specification ("Electronic Case Reporting [deprecated]") is no longer the active eCR IG. Any updates to eCR content would need to be applied to the successor specification, if applicable.

2. **Resolution was "Considered for Future Use."** The ticket was resolved in August 2018 with this disposition, meaning the workgroup acknowledged the concern but deferred action.

3. **TEFCA has since matured.** Since 2018, TEFCA has become a major reality in US health data exchange. The Sequoia Project was designated as the Recognized Coordinating Entity (RCE). Active FHIR-based TEFCA connectathon tracks exist (as of 2025), and a dedicated `TEFCA FHIR` Zulip stream is active. The landscape described in the original eCR wording is now significantly outdated.

4. **Related work in other IGs.** FHIR-42646 (National Directory IG) addresses TEFCA and the same networks (eHealth Exchange, Carequality, CommonWell, DirectTrust) in a more current context, suggesting that TEFCA references are being handled in active specifications.

## Proposed Dispositions

### Disposition A: Not Persuasive — Deprecated Specification
**Rationale:** The Electronic Case Reporting specification targeted by this ticket is deprecated. The request to update wording about Health Data Networks and TEFCA is no longer actionable against this specification. Any TEFCA-related updates should be pursued in the successor eCR IG or other active specifications (e.g., the US Public Health Profiles Library). Close with no change.

### Disposition B: Persuasive — Apply to Successor Specification
**Rationale:** The concern is valid and has become more relevant since 2018 as TEFCA has matured. The workgroup should identify the successor eCR specification and update the Health Data Networks wording to reflect TEFCA's role. Specific wording could reference TEFCA alongside or in place of the current list of individual networks, acknowledging TEFCA as the overarching framework.

### Disposition C: Not Persuasive — Already Addressed Elsewhere
**Rationale:** TEFCA is now referenced in active IGs such as the National Directory of Healthcare Providers and Services (FHIR-42646, FHIR-42644). The concern about updating references to reflect the TEFCA framework has been effectively addressed through the evolution of other specifications. No further action is needed on this deprecated ticket.

## Recommendation

**Disposition A: Not Persuasive — Deprecated Specification**

The target specification is deprecated and the original resolution of "Considered for Future Use" was appropriate at the time. Since the eCR specification has been superseded, changes cannot be applied to the original target. TEFCA is now being addressed in active specifications (e.g., the National Directory IG). If the workgroup believes the successor eCR IG needs updated TEFCA language, that should be filed as a new ticket against the current specification rather than reopening this deprecated-target ticket.
