## Summary

This ticket requests that the US Public Health Profiles Library IG adopt the standard `cqf-expansionParameters` extension from the FHIR Extension Pack instead of maintaining a US-Public-Health-specific extension for the same purpose. The `cqf-expansionParameters` extension ([StructureDefinition](https://hl7.org/fhir/extensions/StructureDefinition-cqf-expansionParameters.html)) supports specifying expansion parameters by reference from a Library and is available for general use. The resolution notes that the profile has been moved into the US Public Health Profiles Library IG and that the work is now being tracked under [FHIR-52764](https://jira.hl7.org/browse/FHIR-52764).

## Details

### Description

The FHIR Extension Pack contains an extension (`cqf-expansionParameters`) that supports the use case of specifying ValueSet expansion parameters generally. The request is to consider adopting this standard extension and removing the US-Public-Health-specific one, reducing duplication and promoting cross-IG consistency.

### Resolution Description

The resolution states: _"This profile is now in the US Public Health Profiles Library IG and this has been addressed by this ticket on that specification: [FHIR-52764](https://jira.hl7.org/browse/FHIR-52764)."_ FHIR-52764 (status: Published, resolution: Persuasive) directly tackles the same request within the US Public Health Profiles Library specification and has been resolved.

## Proposed Dispositions

### Disposition A: Accept (Persuasive with Modification)

**Action:** Accept the ticket as Persuasive with Modification, acknowledging that the work has been redirected to FHIR-52764, which has already been resolved (Persuasive) and published. No additional changes are needed in this ticket's original specification context since the profile has moved to the US Public Health Profiles Library IG.

**Rationale:** The intent of FHIR-49700 — adopting `cqf-expansionParameters` and removing the USPH-specific extension — has been fully addressed by FHIR-52764. The resolution description on this ticket already documents the handoff. Accepting confirms the work group agrees with the approach and the outcome.

### Disposition B: Defer to FHIR-52764 (Not Persuasive with Modification)

**Action:** Mark as Not Persuasive with Modification, noting that the original specification context no longer applies because the profile has moved. Direct reviewers to FHIR-52764 for the authoritative resolution.

**Rationale:** Since the profile has moved out of the specification this ticket was originally filed against, one could argue this ticket is moot in its current context. Closing it with a redirect avoids duplicated tracking while still honoring the submitter's intent.

### Disposition C: Decline (Not Persuasive)

**Action:** Close as Not Persuasive, reasoning that maintaining a US-Public-Health-specific expansion parameters extension could offer benefits such as tighter control over USPH-specific expansion semantics.

**Rationale:** Some implementers may prefer realm-specific extensions for governance or versioning independence. However, this conflicts with the broader HL7 direction toward Extension Pack consolidation (as seen in FHIR-43969) and the submitter's clear intent, and the successor ticket FHIR-52764 has already been accepted.

## Recommendation

**Disposition A: Accept (Persuasive with Modification)** is recommended. The request is well-founded — adopting the standard `cqf-expansionParameters` extension reduces duplication and aligns with the HL7 trend of consolidating extensions into the Extension Pack (see FHIR-43969 precedent). The work has already been completed via FHIR-52764 (Published, Persuasive), confirming that the Public Health WG agrees with this direction. Accepting this ticket with the existing resolution description cleanly closes the loop.
