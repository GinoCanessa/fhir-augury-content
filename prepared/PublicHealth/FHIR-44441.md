## Description

The submitter requests that code system and value set URLs in the Vital Records Common Profiles Library be anchored in terminology.hl7.org (THO), per the TSC-approved policy for Terminology Guidance for IG Authors ([Confluence: Terminology Expectations for IG Developers](https://confluence.hl7.org/display/TSMG/Terminology+Expectations+for+IG+Developers)). The request specifically asks to start the UTG process to add the value set (comment 31) to THO.

## Proposed Dispositions

### Disposition A: Persuasive — Begin UTG Process for THO Migration

**Rationale:** The TSC-approved policy ([Terminology Expectations for IG Developers](https://confluence.hl7.org/display/TSMG/Terminology+Expectations+for+IG+Developers)) directs that code systems and value sets should be anchored in THO rather than defined locally within IGs. The previous resolution already acknowledged that the value set "has nothing to do with vital records and is of general utility" and committed to beginning the THO migration process. Now that STU2 has been published, the workgroup should follow through on that commitment and formally initiate the UTG process. This aligns with HL7 terminology governance goals and improves interoperability by centralizing shared terminology resources.

**Vote recommendation:** Persuasive

### Disposition B: Not Persuasive with Modification — Defer Until THO Migration Coordination Is Complete

**Rationale:** While the policy direction is agreed upon, migrating individual value sets to THO piecemeal creates coordination challenges. The workgroup could resolve this as Not Persuasive with Modification, acknowledging the goal but deferring action until a coordinated THO migration plan covering all the related tickets (FHIR-44434, FHIR-44447, FHIR-44457, etc.) is developed. The modification would note that the IG will reference THO-hosted versions of these value sets once the UTG process is completed for the full set, avoiding partial migrations.

**Vote recommendation:** Not Persuasive with Modification

### Disposition C: Considered for Future Use — Reinstate Prior Resolution

**Rationale:** The original 2024-02-22 resolution (22-0-2) as Considered for Future Use remains valid in substance. The workgroup acknowledged the need to migrate to THO and committed to referencing THO-hosted value sets in a future version. Re-opening post STU2 does not change the underlying reality that the UTG process has not yet been initiated or completed. Reinstating this resolution keeps the ticket visible for future action while recognizing that no IG change can be made until THO hosting is available.

**Vote recommendation:** Considered for Future Use

## Recommendation

**Disposition A: Persuasive** is recommended.

The TSC policy is clear, the previous vote (22-0-2) demonstrated near-unanimous agreement that this value set belongs in THO, and the workgroup already committed to beginning the UTG process. With STU2 now published, this is the appropriate time to follow through. Resolving as Persuasive formally commits the workgroup to initiating the UTG submission, which is the necessary next step regardless of which future IG version will reference the THO-hosted value set. The sibling ticket FHIR-44457 (also re-opened post STU2) should be handled consistently.
