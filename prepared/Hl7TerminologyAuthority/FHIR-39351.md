HL7 Germany requests that the canonical URL for ICD-10 GM be updated from `http://fhir.de/CodeSystem/dimdi/icd-10-gm` to `http://fhir.de/CodeSystem/bfarm/icd-10-gm`, reflecting that DIMDI (Deutsches Institut für Medizinische Dokumentation und Information) was reorganized into BfArM (Bundesinstitut für Arzneimittel und Medizinprodukte). The ticket also asks to update the reference link from the old wiki (`http://wiki.hl7.de/index.php?title=Kodesysteme`) to the new IG page (`https://ig.fhir.de/basisprofile-de/stable/Ressourcen-Condition-ICD-10-GM.html`), and to update examples on the "Using ICD with FHIR" page.

## Details

### Background
The original canonical URL for ICD-10 GM (`http://fhir.de/CodeSystem/dimdi/icd-10-gm`) was established via FHIR-13723 in 2017, when ICD-10-GM was managed by DIMDI. HL7 Germany subsequently updated their base profiles (v1.0.0) with a breaking change to how post-coordinated ICD-10 GM codes (asterisk, dagger, "!" codes) are represented. As part of that breaking change, they also updated the canonical to reference BfArM, the successor organization.

### Key Discussion Points
1. **Canonical stability concern (Michael Lawley, 2022-10-30):** Embedding organization names in canonicals creates risk of future breakage. Suggested using `http://fhir.de/CodeSystem/icd-10-gm` instead, without an organization in the path.
2. **UTG process requirement (Carmela Couderc, 2023-01-09):** The "Using ICD with FHIR" page has moved to terminology.hl7.org (THO). Changes must go through the UTG Jira process (UP project), not the FHIR tracker.
3. **Joint Vocab/TSMG session (2023-01-18):** The group discussed versioning concerns (Germany issues a new ICD-10 code system annually) and proposed a three-part resolution: (a) update the ICD page to include the new URL and deprecate the old one, (b) update links as requested, (c) create a THO metadata record.
4. **Implementation blocker (Carmela Couderc, 2023-03-20):** Terminology Infrastructure cannot implement directly because the page now lives in THO. Reporter needs to file a UP ticket.
5. **Planned closure (jsnell, 2025-10-20):** The ticket is being planned for closure. The new URL cannot be added as an official URL in FHIR Core, but an alternate identifier can be requested through the UTG Jira process at https://jira.hl7.org/browse/UP.

### Current State
The ticket has been open for nearly three years without resolution. The consensus is that HL7 Terminology Authority / FHIR Core is not the right venue for this change — it must go through the UTG (UP project) process. No UP ticket appears to have been filed.

## Proposed Dispositions

### Disposition A: Redirect to UTG Process (Not Persuasive — Close)
**Action:** Close this ticket as **Not Persuasive** and direct the submitter to the UTG (UP) project.

**Rationale:** Multiple commenters (Carmela Couderc, jsnell) have confirmed that the "Using ICD with FHIR" page now resides in THO (terminology.hl7.org), not in FHIR Core. Changes to terminology registration, canonical URLs, and ICD page content must go through the UTG Jira process (UP project). This ticket is in the wrong tracker. The submitter should file a UP ticket requesting: (1) addition of the new canonical as an alternate identifier, (2) deprecation of the old `dimdi`-based canonical, and (3) updated links/examples. Closing this ticket aligns with the 2025-10-20 comment from jsnell indicating planned closure.

### Disposition B: Accept with Modification — Partial Update
**Action:** Mark as **Persuasive with Modification**. Update only the reference link in FHIR Core (from the old wiki URL to the new IG URL) and add a note pointing to THO for the canonical definition. File a companion UP ticket for the canonical change itself.

**Rationale:** While the canonical URL change must go through UTG, the outdated reference link (`wiki.hl7.de`) in FHIR Core can be fixed directly since it's just a documentation link update, not a terminology registration change. This addresses part of the request while properly routing the canonical change to UTG. However, this disposition may be moot if the ICD page content has fully migrated to THO and no longer exists in FHIR Core.

### Disposition C: Accept as Submitted
**Action:** Mark as **Persuasive** and update the canonical URL, links, and examples in the ICD page as requested.

**Rationale:** HL7 Germany owns the `fhir.de` namespace and has already made the change in their base profiles. The FHIR spec should reflect reality. However, this disposition conflicts with the established UTG governance process and ignores Michael Lawley's valid concern about embedding organization names in canonicals (BfArM could be reorganized in the future, just as DIMDI was).

## Recommendation

**Disposition A: Redirect to UTG Process (Not Persuasive — Close)**

This is the recommended disposition. The evidence from the comment history is clear and consistent: the ICD terminology content has migrated to THO, and canonical URL changes must follow the UTG process via the UP project. Both Carmela Couderc (2023) and jsnell (2025) have stated this explicitly, and the ticket has been open for nearly three years without resolution precisely because it's in the wrong tracker. Closing with clear guidance to file a UP ticket is the most appropriate action. Additionally, when filing the UP ticket, the submitter should consider Michael Lawley's suggestion to use an organization-neutral canonical (`http://fhir.de/CodeSystem/icd-10-gm`) to avoid future breakage if BfArM is ever reorganized.
