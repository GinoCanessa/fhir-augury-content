## Summary

The reporter (Anthony Julian) identified that the link to the **genderIdentity** value set on the **data-absent-reason** code system page is broken. The "This Code system is referenced in the content logical definition of the following value sets" section includes a link to `https://build.fhir.org/ig/HL7/fhir-extensions/valueset-gender-identity.html`, which is invalid. The issue was reported as broken in both R5 and the CI build.

A comment from cooper.thompson (PA WGM, May 2024) clarifies that the **Patient Administration** work group moved the gender identity value set out of the FHIR Extensions Pack into **HL7 Terminology (THO)**. The correct current location is `https://terminology.hl7.org/ValueSet-gender-identity.html`. The data-absent-reason code system page was not updated to reflect this move, resulting in the broken link.

## Details

- The broken link points to the **FHIR Extensions Pack** build location for the gender identity value set: `https://build.fhir.org/ig/HL7/fhir-extensions/valueset-gender-identity.html`
- The gender identity value set was migrated from the FHIR Extensions Pack to **THO** (HL7 Terminology). The correct URL is: `https://terminology.hl7.org/5.5.0/ValueSet-gender-identity.html`
- The "used by" / cross-reference section on the data-absent-reason code system page needs to be updated to point to the THO location.
- This is a publication/tooling issue — the automated cross-reference generation did not track the value set's migration to a different IG.
- The issue affects both R5 and CI build outputs.

## Proposed Dispositions

### Disposition A: Accept as Technical Correction (Persuasive)

**Action:** Update the cross-reference link on the data-absent-reason code system page to point to the gender identity value set at its current THO location (`https://terminology.hl7.org/ValueSet-gender-identity.html`) instead of the stale FHIR Extensions Pack URL.

**Rationale:** The gender identity value set has been officially moved to THO, as confirmed by the PA work group. The broken link is a straightforward publication error — the cross-reference metadata was not updated when the value set migrated. This is a non-controversial technical correction with no semantic or normative impact.

### Disposition B: Accept and Address Systemically (Persuasive with Modification)

**Action:** Fix the broken link as in Disposition A, and additionally file a follow-up item to audit the publication tooling so that cross-reference links are automatically updated when value sets or code systems move between IGs (e.g., from FHIR Extensions Pack to THO). This would prevent similar broken links from recurring.

**Rationale:** While fixing the immediate link is necessary, the root cause is that the publication pipeline does not track value set migrations across IGs. A systemic fix would reduce the risk of future broken links as terminology assets continue to be reorganized.

### Disposition C: Decline (Not Persuasive)

**Action:** Close with no change, noting that the link will be corrected naturally when the next publication cycle runs and the tooling picks up the correct location.

**Rationale:** If the publication tooling has already been updated to resolve cross-references to THO and the fix will appear in the next build, no manual intervention may be needed.

## Recommendation

**Disposition A: Accept as Technical Correction (Persuasive)**

This is a clear-cut broken link caused by the gender identity value set's migration from the FHIR Extensions Pack to THO. The fix is simple, non-controversial, and has no normative impact. The PA work group has already confirmed the correct location. Fixing this promptly improves the usability of the published specification.
