## Description

The submitter requests that four dedicated sections be excluded from the first version of the EU Imaging Study Report IG to reduce complexity and focus on core imaging report interoperability across the EU:

1. **AdverseEvents** — Tracking workflows vary significantly; completed cross-border procedures rarely need this information for interpreting results.
2. **Radiation Dose** — Data should originate from PACS, not EHRs; EHR adoption is low. A European Radiology survey recommends separate dose tracking systems. The [Radiation Dose Summary IG](https://build.fhir.org/ig/HL7/fhir-radiation-dose-summary-ig/) is better suited for FHIR mapping of dose data.
3. **Contrast Medications** — Whether contrast was used is typically reflected in the procedure code and radiologist narrative; discrete medication sharing adds scope for minimal benefit.
4. **Procedure Phases** — Results are generally produced for imaging as a whole; phase data already exists in DICOM data and is not typically queried independently.

The rationale is that including these sections increases implementation complexity and cost while distracting from the immediate goal of making imaging reports available across EU borders.

## Proposed Dispositions

### Disposition A: Persuasive with Modification

**Resolution:** Acknowledge that the submitter's core concern about scope and complexity is valid, and note that the WG has already substantially addressed it through related applied tickets. Close this ticket recognizing that the spirit of the request has been fulfilled.

**Rationale:** The WG has already acted on three of the four areas raised:
- **Radiation Dose:** FHIR-51279 (Applied/Persuasive) removed detailed observation components and redirected to DICOM source data and the Radiation Dose Summary IG — directly aligning with the submitter's recommendation.
- **Procedure Phases:** FHIR-51278 (Applied/Persuasive with Modification) clarified that the concept varies across modalities and improved mapping to ImagingSelection rather than removing it entirely.
- **AdverseEvents:** FHIR-53155 (Applied/Persuasive) fixed document tree references, and FHIR-51330 (Triaged) continues to address complications modeling.

The modification is that rather than removing these sections outright, the WG has simplified and clarified them — reducing the implementation burden while preserving clinically useful optional content. The WG should confirm that all four section types are optional (not required) to allow implementers to focus on core reporting.

**Specific Changes:** Verify and document that the AdverseEvents, Radiation Dose, Contrast Medications, and Procedure Phases sections have 0..* cardinality in the Composition profile, and add implementation guidance stating that systems may omit these sections in early implementations to focus on core imaging report content.

### Disposition B: Persuasive

**Resolution:** Accept the request and remove AdverseEvents, Radiation Dose, Contrast Medications, and Procedure Phases sections from the STU1 Composition profile.

**Rationale:** For a first STU1 ballot focused on cross-border interoperability, scope discipline is critical. Including these sections invites inconsistent implementations across EU member states, where radiology workflows differ significantly. The submitter correctly notes that radiation dose is better handled by PACS/dedicated systems, adverse events are rarely relevant in completed cross-border reports, contrast information is captured in procedure codes and narrative, and procedure phase data resides in DICOM. Removing them from STU1 lets the community build experience with core reporting before expanding in STU2.

**Specific Changes:** Remove the dedicated Composition sections for AdverseEvents, Radiation Dose, Contrast Medications, and Procedure Phases. Add a "Future Considerations" section to the IG documenting these as candidates for STU2 expansion once implementer experience is gained.

### Disposition C: Not Persuasive

**Resolution:** Retain the current approach. The concerns about complexity have been addressed by simplifications made through related tickets.

**Rationale:** The WG has already simplified radiation dose handling (FHIR-51279, FHIR-51332), improved procedure phase documentation (FHIR-51278), and fixed adverse event inclusion (FHIR-53155). These sections provide clinically important optional content — adverse events are needed for patient safety (the Czech Republic IG also includes a complications section per FHIR-51330 comments), radiation dose information is required by some EU member state regulations, and contrast information is relevant for future imaging orders and allergy management. Removing them would reduce the IG's clinical completeness and potentially require reintroduction later, disrupting implementers who have already built support. The sections are optional; implementers are free to omit them if not relevant to their workflows.

## Recommendation

**Disposition A: Persuasive with Modification** is recommended.

The submitter raises a legitimate concern about STU1 scope complexity, and the WG has already responded substantively by simplifying radiation dose (FHIR-51279), clarifying procedure phases (FHIR-51278), and improving adverse event handling (FHIR-53155). Completely removing these sections (Disposition B) would be overly aggressive given the work already applied and the clinical value they provide. Rejecting outright (Disposition C) does not fully acknowledge the submitter's valid point about implementation burden. The best path is Disposition A: recognize that the WG has addressed the spirit of this request through incremental improvements, confirm that all four sections are optional for implementers, and add guidance that early adopters may focus on core reporting sections first. This preserves clinical completeness while explicitly reducing the perceived implementation barrier.
