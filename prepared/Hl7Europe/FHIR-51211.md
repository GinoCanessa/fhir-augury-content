## Description

Giorgio Cangioli questions the **Group 6** mapping in the ConceptMap that maps from the Imaging report model to the **IM ImagingStudy Manifest**. He states this mapping is unclear because the context is the *report* space, and he would expect to see a mapping to the **report bundle** rather than the **manifest bundle**.

## Analysis

This ballot comment is one of a series filed by Giorgio Cangioli (FHIR-51152, FHIR-51210, FHIR-51211, FHIR-51215) reviewing the ConceptMap-based model-to-FHIR mappings in the EU Imaging Study Report IG. The core concern is that a Group 6 mapping appears to direct report-context elements to the ImagingStudy Manifest bundle rather than the report bundle, which is unexpected.

The responder (JIRAUSER28670) was unable to identify the specific Group 6 mapping Giorgio described in the current ConceptMaps, suggesting the artifact may have been modified or the reference was to an earlier version.

**Critically**, per the comment on FHIR-51152 (bvdh, 2026-03-03), the workgroup has **decided to abandon ConceptMaps** in favor of markdown table–based mappings. This architectural decision fundamentally changes the context of this ticket: the specific ConceptMap group in question no longer exists in the current approach.

## Proposed Dispositions

### Disposition A: Persuasive
**Action:** Acknowledge the concern that report-context elements should map to the report bundle, not the manifest bundle. Ensure the new markdown table–based mappings (which replaced ConceptMaps) correctly distinguish between report and manifest mapping targets, and that no report-model elements are erroneously directed to the manifest bundle.

**Rationale:** The underlying concern about correct mapping targets is valid regardless of the mapping representation format. The workgroup should verify that the transition from ConceptMaps to markdown tables addressed this separation.

### Disposition B: Not Persuasive
**Action:** No change. The responder could not reproduce the described Group 6 mapping to the ImagingStudy Manifest in the current ConceptMaps. The submitter did not provide a follow-up clarification despite being asked.

**Rationale:** The specific artifact described in the comment could not be confirmed. Without further clarification from the submitter, there is insufficient information to act on.

### Disposition C: Not Persuasive (Superseded)
**Action:** Close as superseded. The workgroup's decision (documented in FHIR-51152) to replace ConceptMaps with markdown table mappings eliminates the specific artifact this comment addresses. The workgroup should confirm that the new markdown-based mappings correctly handle the report-versus-manifest distinction.

**Rationale:** The ConceptMap that contained the questioned Group 6 mapping has been deprecated. The broader approach change resolves this specific concern. However, the underlying principle (report-context mappings should target the report bundle) should be validated in the new mapping tables as part of the mapping migration effort.

## Recommendation

**→ Disposition C: Not Persuasive (Superseded)**

The decision to move from ConceptMaps to markdown table mappings (per FHIR-51152) eliminates the specific artifact Giorgio questioned. The responder's inability to locate the described Group 6 mapping further supports that the content has evolved since the ballot comment was filed. The workgroup should confirm during the mapping migration review that report-model elements are correctly targeted to the report bundle (not the manifest bundle) in the new markdown tables, but no separate change is required for this ticket.
