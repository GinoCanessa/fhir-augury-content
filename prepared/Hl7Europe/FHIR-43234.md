## Description

Ulrike Merrick raises a question about the obligation extension usage in the EU Laboratory Report IG, drawing on US experience with V2-based Laboratory Results Interface (LRI) conformance declarations used for EHR certification under Meaningful Use.

Key points raised:
1. **US precedent**: The US developed an [LRI Conformance Declaration specification](https://www.hl7.org/implement/standards/product_brief.cfm?product_id=433) to enable certification and system functionality testing of EHR laboratory interfaces. Without such a declaration, certification cannot be achieved.
2. **Clarity of obligation levels**: While the EU Laboratory Report IG examples define action verbs for each actor, the **level of expectation (MAY, SHOULD, SHALL) is not clear** from the current notation. Without clear indication of obligation level, the specification is "not as useful as expected."
3. **European constraint philosophy**: The reporter recommends that the European Profile should set obligation levels **as low as reasonable**, allowing national profiles to apply further constraints.

## Analysis

This ticket asks a legitimate and practical question about a gap in the EU Laboratory Report IG: the obligation extension examples show action verbs per actor, but **do not clearly indicate the conformance level** (SHALL, SHOULD, MAY) for each obligation. The reporter draws on established US practice (LRI Conformance Declaration) to argue this clarity is essential for certification and testing.

The concern is well-supported by community discussion — the obligation extension and its interaction with mustSupport has been an active topic across multiple work groups (FHIR-I, AU Core, HL7 Europe) and Zulip streams. The base FHIR obligation mechanism does support conformance level codes (e.g., `SHALL:populate`, `SHOULD:populate`), but the EU Lab Report IG may not be consistently applying them.

The secondary recommendation — that European profiles should set obligation levels as low as reasonable to allow national-level tightening — aligns with the layered constraint philosophy endorsed across the FHIR community (as confirmed in the Zulip thread about derived IGs creating new ActorDefinitions).

---

## Proposed Dispositions

### Disposition A: Persuasive
**Action:** Update the EU Laboratory Report IG to explicitly specify conformance levels (SHALL, SHOULD, MAY) on all obligation extension instances, and add narrative text explaining the obligation levels chosen and the rationale for keeping them as permissive as practical for national derivation.

**Rationale:** The question identifies a genuine gap. The obligation extension supports conformance level codes, and failing to use them leaves implementers guessing. Explicit levels are necessary for testability and certification, as US experience with LRI demonstrates. This aligns with FHIR-I guidance that obligations should carry clear conformance expectations.

### Disposition B: Persuasive with Modification
**Action:** Add a clarifying section to the EU Laboratory Report IG explaining how obligation levels should be interpreted, and commit to reviewing and annotating obligation instances with explicit conformance levels in a future ballot cycle. Address the immediate concern with narrative guidance now, with structured obligation codes to follow.

**Rationale:** A phased approach is practical given that the obligation extension was still evolving at the time of the ballot (as noted in Zulip conformance discussions). The IG can provide immediate clarity through narrative while aligning with the maturing obligation infrastructure for structured codes in a subsequent cycle.

### Disposition C: Not Persuasive
**Action:** No change. Note that the obligation extension and its code system are defined in base FHIR/FHIR Extensions Pack and that the EU Laboratory Report IG uses them as specified. Direct the commenter to the obligation code system documentation for interpretation guidance.

**Rationale:** The obligation levels are inherent in the obligation codes used. The IG follows the base specification, and adding redundant narrative about conformance levels may diverge from base FHIR guidance. Implementers should refer to the extension definition for semantics.

---

## Recommendation

**Disposition A: Persuasive**

The reporter's concern is well-founded and practically important. The obligation extension mechanism explicitly supports conformance levels via codes like `SHALL:populate` and `SHOULD:populate`, and the EU Laboratory Report IG should use them consistently. Without explicit levels, the obligations are ambiguous and cannot support certification or systematic conformance testing — the very purpose for which obligations exist. The recommendation to keep European-level obligations as permissive as practical (to enable national tightening) is sound and aligns with the layered profiling philosophy already endorsed in FHIR community discussions. This is a concrete, actionable improvement that strengthens the IG.
