# Ticket Review: FHIR-43193

**Title:** Representation of narrative
**Status:** Triaged **Priority:** Medium **Type:** Question

## Summary

Gabriel Kleinoscheg raises a question regarding the EU Laboratory Report (FHIR) specification's mapping of narrative data elements from the European eHealth Network (eHN) logical model to FHIR Composition resources. Specifically, the ticket asks how `Result.narrative.report` (A.5.1.1 Narrative report) and `Result.narrative.notes` (A.5.1.2 Comments, interpretation and recommendations) should be represented across the available text fields in the FHIR Composition, namely `Composition.text`, `Composition.section:lab-no-subsections.text`, and `Composition.section:lab-subsections.section.text`. The submitter seeks clarification on how to split or map these narrative elements into the correct FHIR fields and, conversely, how to interpret the FHIR text fields when reading data back into the eHN model categories.

## Details

**Description:**

Within Group 3 Mapping from A.5 – Results data elements to Composition: Laboratory Report, the `Result.narrative.report` (A.5.1.1 Narrative report) may be mapped to:

- `Composition.text`
- `Composition.section:lab-no-subsections.text` or `Composition.section:lab-subsections.section.text`

Likewise, the `Result.narrative.notes` (A.5.1.2 Comments, interpretation and recommendations) may be mapped to:

- `Composition.section:lab-no-subsections.text` or `Composition.section:lab-subsections.section.text`

The submitter asks how `Result.narrative.report` and/or `Result.narrative.notes` should be split up in order to be correctly represented in FHIR. Alternatively, how the `*.text` fields in FHIR should be interpreted to understand them correctly as narrative report versus comments/interpretation/recommendations.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Clarify the EU Laboratory Report specification's mapping guidance to explicitly state that `Result.narrative.report` (A.5.1.1) maps to `Composition.section.text` at the appropriate section level (i.e., `Composition.section:lab-no-subsections.text` for flat reports or `Composition.section:lab-subsections.section.text` for hierarchical reports), and that `Result.narrative.notes` (A.5.1.2) is carried as a distinct narrative block within the same section text or in a dedicated subsection. Add normative or informative text in the specification clarifying the distinction, including guidance that `Composition.text` is reserved for the overall document-level narrative summary and should not be used for individual result narratives.

#### Justification

The question highlights a genuine ambiguity in the current mapping table where both data elements appear to map to the same FHIR fields without clear demarcation. Providing explicit guidance reduces implementer confusion and improves interoperability by ensuring consistent representation of narrative content across EU Laboratory Report implementations.

---

### Disposition B: Alternative Approach

#### Proposal

Rather than updating the mapping table alone, introduce a dedicated `Composition.section` slice (e.g., `section:lab-comments`) specifically for comments, interpretations, and recommendations (A.5.1.2), distinct from the section carrying the narrative report (A.5.1.1). Update the mapping table accordingly so that `Result.narrative.report` maps to the results section text and `Result.narrative.notes` maps to the new comments section. Provide an accompanying example in the implementation guide.

#### Justification

Using separate sections for distinct narrative purposes aligns with FHIR document best practices, where each section should carry a coherent narrative with a defined purpose. This approach eliminates the ambiguity entirely by giving each eHN data element its own structural home, making round-tripping between the eHN logical model and FHIR straightforward. It also improves readability for clinical consumers viewing the document.

---

### Disposition C: Decline

#### Proposal

Close the ticket as a question that does not require a specification change. Respond to the submitter with informal guidance explaining that the current mapping is intentionally flexible: implementers may choose how to distribute narrative content across the available text fields based on their use case, and the specification does not mandate a single approach.

#### Justification

The current specification may intentionally allow flexibility in how narrative text is represented. The eHN logical model elements and the FHIR Composition text fields do not necessarily require a strict one-to-one mapping. Declining avoids over-constraining the specification for a design decision that may vary legitimately across national implementations and laboratory workflows.

---

### Recommendation

**Recommended disposition:** B

Disposition B (Alternative Approach) is recommended because it resolves the ambiguity at a structural level rather than relying solely on textual guidance. The current mapping table allows two semantically different data elements—narrative reports and interpretive comments—to land in the same FHIR field, which makes reliable round-tripping between the eHN model and FHIR difficult. Introducing a dedicated section slice for comments and recommendations gives each element a clear, unambiguous home in the FHIR document structure. This approach follows FHIR document composition best practices, provides better separation of concerns for clinical consumers, and supports cleaner automated processing. It is more robust than Disposition A's textual clarification alone, which still leaves room for inconsistent interpretation, yet avoids the rigidity concerns of Disposition C's decline, which would leave implementers without actionable guidance.
