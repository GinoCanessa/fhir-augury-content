## Description

The submitter reviewed both PACIO IGs (Functional Status and Cognitive Status) and found it unclear why two separate IGs were needed. Key observations:

- **>80% narrative overlap** between the two IGs with no discernable differences in driving use cases.
- **Profile/extension differences are minor** — element bindings and profile details are essentially the same.
- **Dual IGs impose implementer burden** — different profile names for subsections of content drawn from the same assessment instruments (e.g., CMS-required MDS, OASIS) create unnecessary complexity for implementers and receivers.
- **No rationale provided** in the IGs for why they were split.
- **Proposal:** Merge into a common core for functioning data, with domain-specific profiles organized into subsections. This would provide a more robust foundation for additional PACIO content areas.

## Analysis

### Key Observations

1. **The proposed merge has effectively been accomplished.** FHIR-49422 and FHIR-49425 (January 2025) confirm that PACIO has replaced the Functional Status and Cognitive Status IGs with the consolidated **PACIO Personal Functioning and Engagement (PFE) IG** (https://build.fhir.org/ig/HL7/fhir-pacio-pfe/). The old IGs are being archived.

2. **This ticket has been open for over 4 years.** Originally filed 2020-12-15, initially resolved as "Considered for Future Use" (6-0-0 vote, 2021-06-28), then reopened 2021-11-04 for post-STU1 consideration. No further activity since.

3. **FHIR-30014 is a duplicate.** It contains identical text filed against the Functional Status specification and should be resolved in tandem.

4. **The original concern was well-founded.** The PACIO community arrived at the same conclusion and created the PFE IG, vindicating the submitter's recommendation.

5. **Companion tickets FHIR-29978 and FHIR-30015** about scope clarification and ICF alignment should also be evaluated in light of the PFE IG's current scope.

## Proposed Dispositions

### Disposition A: Persuasive — Close as Implemented
**Resolution:** Persuasive
**Action:** Close this ticket as the requested change has been implemented. The PACIO Functional Status and Cognitive Status IGs have been merged into the PACIO Personal Functioning and Engagement (PFE) IG, which is the exact outcome requested. Also resolve FHIR-30014 as duplicate. Note in the resolution that the PFE IG addresses this feedback.

**Rationale:** The submitter's core request — merging the two overlapping IGs into a unified foundation — has been fulfilled by the PFE IG. The ticket can be closed with acknowledgment that the community acted on this feedback.

### Disposition B: Considered — No Action Required
**Resolution:** Considered - No action required
**Action:** Close with a note that the PACIO project has independently consolidated the Functional Status and Cognitive Status IGs into the PFE IG, making this request moot. The Cognitive Status IG (against which this ticket is filed) is being archived.

**Rationale:** Since the specification this ticket was filed against is being archived, the ticket is effectively obsolete. No explicit action is required by the Patient Care workgroup because the PACIO project has already addressed the concern.

### Disposition C: Not Persuasive — Defer to PACIO Project
**Resolution:** Not Persuasive
**Action:** Close with a note that IG architecture decisions are within the purview of the PACIO project community, which has already made the determination to consolidate.

**Rationale:** This would assert that the workgroup does not need to take a formal position since the PACIO project owns the IG decisions. However, this framing undervalues the submitter's contribution since the merge was ultimately adopted.

## Recommendation

**→ Disposition A: Persuasive — Close as Implemented**

This is the most appropriate resolution because:
1. The requested change has been definitively implemented — the two IGs were merged into the PFE IG.
2. A "Persuasive" resolution properly credits the submitter's feedback as having influenced the outcome.
3. It provides a clean resolution narrative that ties this long-open ticket to the concrete result (PFE IG).
4. FHIR-30014 should be resolved simultaneously as a duplicate pointing to this ticket's resolution.
5. The companion scope-clarification tickets (FHIR-29978, FHIR-30015) should be reviewed separately against the PFE IG's current narrative.
