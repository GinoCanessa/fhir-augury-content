# FHIR-12993: Please Create a NonMedicationAdministration object or an Administration object

## Summary

**Status:** Triaged | **Type:** Change Request | **Priority:** Medium | **Resolution:** Unresolved
**Work Group:** Patient Care | **Specification:** FHIR Core (FHIR)
**Reporter:** Jamie Hignite | **Assignee:** Unassigned
**Labels:** Discuss-at-WGM
**Created:** 2017-03-08 | **Updated:** 2025-09-17
**URL:** https://jira.hl7.org/browse/FHIR-12993

The submitter requests a generic Administration resource (or NonMedicationAdministration) to cover clinical activities that are administered like medications (with dose, rate, quantity) but are **not** medications. The core problem is that MedicationAdministration captures dosage/rate semantics that Procedure lacks, but using MedicationAdministration for non-medications is semantically incorrect.

**Use cases cited:**
1. Charting against physician orders that are not diagnostic tests or medications (e.g., placement of lines, tubes, drains such as foley catheters, IVs)
2. Respiratory therapy charting against oxygen or ventilator orders
3. Rehab staff charting against tasks (gait training)
4. Hemodialysis, peritoneal dialysis
5. Blood product transfusions
6. Additional: DVT prophylaxis (anti-emboli stockings), removal of staples/sutures, restraints

## Analysis

This is a long-standing request (filed 2017, still Triaged in 2025) that identifies a genuine gap in FHIR's resource model. The core issue is that several clinical activities require **dose/rate/quantity semantics** during administration, but are not medications:

- **Blood products** — require dose (units/mL), rate, and have their own product resource (BiologicallyDerivedProduct) but no administration resource
- **Radiation therapy** — requires dose (grays), fractions, and has no suitable home
- **Oxygen/respiratory gases** — administered with rates and concentrations
- **Nutrition/tube feedings** — rate-based administration

The FHIR spec currently offers two suboptimal paths:
1. **Procedure** — lacks dosage, rate, and quantity elements; its own definition explicitly excludes dose-based therapies
2. **MedicationAdministration** — has the right structure but the wrong semantics; blood is not a Medication

The community has discussed three approaches (documented in the 2018 Pharmacy comment by ehaas):
- **Option A:** Reuse MedicationRequest/MedicationAdministration for blood products (semantically strained)
- **Option B:** Add Dosage capacity to ServiceRequest/Procedure (FHIR-13047)
- **Option C:** Create new resource(s) for these use cases

The 2025 discussion on FHIR-51207 highlights that the BiologicallyDerivedProduct resource was created but the surrounding lifecycle resources (Request, Administration, Statement) remain incomplete — exactly the gap this ticket identified in 2017. The problem has grown in visibility as more implementers encounter it across radiation therapy, blood banking, and oncology use cases (mCODE).

## Proposed Dispositions

### Disposition A: Create an Administration Pattern or Abstract Administration Resource
**Action:** Persuasive — create an "Administration" pattern (similar to the Event pattern) that can be applied across domains, with MedicationAdministration, BiologicallyDerivedProductAdministration, and potentially RadiationAdministration as conforming resources. This mirrors how Request/Event patterns already work in FHIR.

**Rationale:** This addresses the full scope of the request — blood products, radiation, respiratory gases, nutrition — in a consistent and extensible way. The administration "pattern" concept was explicitly suggested by Pharmacy in the 2018 WGM discussion. It aligns with the resource lifecycle gaps identified in FHIR-51207 (BDP lacks Request, Administration, and Statement resources paralleling Medication and Device).

**Risks:** High design effort; requires coordination across Patient Care, Orders & Observations, and Pharmacy work groups. Pattern-based approaches are architecturally sound but take multiple ballot cycles.

### Disposition B: Add Dosage/Quantity Elements to Procedure
**Action:** Persuasive with modification — add optional dosage, quantity, and rate elements to the Procedure resource (as requested in FHIR-13047), and update Procedure's scope definition to include dose-based therapies.

**Rationale:** This is the lowest-effort solution that addresses the immediate gap. Many of the listed use cases (dialysis, line placement, gait training) are already properly modeled as Procedures. Adding dosage capacity would allow the remaining use cases (blood products, radiation, respiratory therapy) to also use Procedure. Grahame Grieve expressed support for broadening Procedure's scope.

**Risks:** May make Procedure too broad and overloaded. Does not address the semantic distinction between "something done to a patient" and "something administered to a patient." Does not address the BDP lifecycle gap (no BDP-specific Statement or Request resources).

### Disposition C: Defer Pending Resolution of FHIR-51207
**Action:** Defer — mark this ticket as dependent on FHIR-51207 (Resources for tracking BDP are possibly incomplete), which is owned by Orders & Observations and addresses the broader resource lifecycle question for BiologicallyDerivedProducts.

**Rationale:** FHIR-51207 (filed 2025) is a more recent and comprehensive framing of the same problem space, specifically asking the community to evaluate whether new BDP lifecycle resources are needed. Resolving FHIR-51207 first would inform whether a new Administration resource is the right approach. This avoids duplicating effort.

**Risks:** Further delays resolution of an already 8-year-old ticket. Does not address non-BDP use cases (radiation therapy, respiratory gases). FHIR-51207 itself is also Triaged/Unresolved with no clear timeline.

## Recommendation

**Disposition A** (Create an Administration Pattern or Abstract Administration Resource) is recommended. This ticket has been open for over 8 years and the community has repeatedly confirmed the resource gap across multiple domains (blood products, radiation therapy, respiratory therapy, nutrition). The problem is not limited to BiologicallyDerivedProducts — it spans any clinical activity that involves dose/rate/quantity but is not a medication. A pattern-based approach provides the right level of abstraction to address all these use cases consistently, while allowing domain-specific resources to be developed incrementally. The 2018 Pharmacy suggestion to investigate an "administration pattern" was never followed through and remains the most architecturally sound approach. Patient Care should coordinate with Orders & Observations (FHIR-51207) and Pharmacy to advance this work, potentially as a joint project at an upcoming WGM.
