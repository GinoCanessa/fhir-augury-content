## Summary

This ticket requests adding vaccination/immunization request support to `CarePlan.activity.plannedActivityReference`. Currently the element's reference list includes Appointment, CommunicationRequest, DeviceRequest, MedicationRequest, NutritionOrder, Task, ServiceRequest, VisionPrescription, and RequestOrchestration — but no resource type covers immunization/vaccination requests. The reporter argues that patients with chronic or multiple chronic conditions need vaccination requests as planned activities in their care plans, and notes that neither MedicationRequest nor ServiceRequest appear to encompass vaccination requests in their scope. The reporter suggests coordination with Pharmacy and Public Health work groups.

---

## Details

### Description

`CarePlan.activity.plannedActivityReference` is currently typed as:

> `0..1 Reference(Appointment | CommunicationRequest | DeviceRequest | MedicationRequest | NutritionOrder | Task | ServiceRequest | VisionPrescription | RequestOrchestration)`

The reporter identifies a gap: patients with chronic or multiple chronic conditions need vaccination requests included as planned activities. There does not appear to be a dedicated FHIR resource for immunization (vaccination) requests, and neither MedicationRequest nor ServiceRequest explicitly include vaccination requests in their scope. The reporter suggests discussions with the Pharmacy and Public Health work groups on how to support this requirement.

## Proposed Dispositions

### Disposition A: Accept — Add ImmunizationRecommendation to plannedActivityReference

**Action:** Add `ImmunizationRecommendation` to the reference type list for `CarePlan.activity.plannedActivityReference`.

**Rationale:** ImmunizationRecommendation is a request-pattern resource (confirmed by Lloyd McKenzie in Zulip, Mar 2022). It is the most appropriate existing resource for representing vaccination recommendations in care plans. There is clear precedent: FHIR-34477 successfully added SupplyRequest to this same reference list with a "Persuasive" resolution. Real-world implementations (e.g., New Zealand AIR project) are already using CarePlan with ImmunizationRecommendation in `plannedActivityReference`. The R5 build already references ImmunizationRecommendation in this context. This is a targeted, low-risk change that fills a documented gap for chronic disease management requiring immunization scheduling.

**Updated reference list would be:**
> `Reference(Appointment | CommunicationRequest | DeviceRequest | ImmunizationRecommendation | MedicationRequest | NutritionOrder | Task | ServiceRequest | VisionPrescription | RequestOrchestration)`

### Disposition B: Accept with Broader Scope — Add ImmunizationRecommendation and coordinate with Pharmacy/Public Health WGs

**Action:** Add `ImmunizationRecommendation` to the reference type list and initiate cross-WG discussion on whether a more general vaccination/immunization request resource is needed.

**Rationale:** While adding ImmunizationRecommendation addresses the immediate gap, the reporter's observation that "there doesn't appear to be a FHIR resource to support vaccination (immunization) request" raises a deeper question. ImmunizationRecommendation was designed as a CDS output resource (per Nathan Bunker), not a clinical order. MedicationRequest lacks fields like `targetDisease` needed for immunization ordering. This disposition addresses the immediate need while flagging the broader design question for cross-WG collaboration. However, the broader scope may delay resolution.

### Disposition C: Decline — Existing resources are sufficient

**Action:** Resolve as Not Persuasive, noting that MedicationRequest or ServiceRequest can be used for vaccination requests, or that Task can wrap immunization-related activities.

**Rationale:** The current reference list already includes MedicationRequest (vaccines are medications), ServiceRequest (vaccination is a service), and Task (can represent any workflow activity). Implementers can use these existing types to represent vaccination requests in care plans without expanding the reference list. Adding ImmunizationRecommendation could blur the boundary between CDS recommendations and clinical orders within CarePlan.

---

## Recommendation

**Disposition A: Accept — Add ImmunizationRecommendation to plannedActivityReference.**

This is the strongest disposition because:

1. **Clear precedent exists.** FHIR-34477 (adding SupplyRequest) was resolved as Persuasive, establishing the pattern for expanding this reference list when a legitimate request-pattern resource is missing.
2. **ImmunizationRecommendation is confirmed as a request-pattern resource** by Lloyd McKenzie, making it architecturally consistent with the other referenced types.
3. **Real-world implementations already depend on this.** The New Zealand AIR project and other immunization-focused implementations are using CarePlan with ImmunizationRecommendation for this purpose.
4. **The reporter's use case is compelling.** Chronic disease patients frequently need vaccination planning integrated into their care plans (e.g., flu vaccines for COPD patients, pneumococcal vaccines for diabetics).
5. **Low risk.** This is an additive change to a reference type list with no breaking impact on existing implementations.

The work group should add ImmunizationRecommendation to `CarePlan.activity.plannedActivityReference` and consider adding a documentation note recommending coordination with Pharmacy and Public Health WGs for vaccination-related care planning workflows.
