## Summary

This ticket requests adding a hyperlink to the C-CDA on FHIR Participant Extension StructureDefinition page within the IPS specification text. The existing wording states "This extension is based on the C-CDA on FHIR Participant Extension" without providing a clickable link. The proposed change adds the URL (`http://www.hl7.org/fhir/us/ccda/StructureDefinition-CCDA-on-FHIR-Participant.html`) inline so readers can navigate directly to the referenced extension definition.

## Proposed Dispositions

### Option A: Persuasive (Apply the Change)

**Rationale:** The ticket was re-opened post-STU-update, suggesting the participant extension reference to C-CDA may still exist or has been reintroduced in the current IPS build. If the IPS specification still references the C-CDA on FHIR Participant Extension, adding the hyperlink is a straightforward editorial improvement that aids reader navigation. The original request is simple and non-controversial — it adds a URL where one is currently missing.

**Action:** Add the link `http://www.hl7.org/fhir/us/ccda/StructureDefinition-CCDA-on-FHIR-Participant.html` (or updated canonical URL) inline where the IPS specification mentions the C-CDA on FHIR Participant Extension. Verify the URL is still valid and points to the current C-CDA on FHIR IG.

---

### Option B: Not Persuasive (No Change Needed)

**Rationale:** FHIR-23887 revised the participant element and was resolved as Persuasive with Modification. If the participant extension was removed or substantially reworked so that the text referencing C-CDA on FHIR Participant Extension no longer exists in the current IPS specification, this ticket's request is moot. The original 2019 suggested resolution and 2020 WG vote (12-0-1) both concluded it no longer applies.

**Action:** Close the ticket, noting that the text referencing C-CDA Participant Extension no longer exists in the current IPS build (if confirmed).

---

### Option C: Considered for Future Use

**Rationale:** If the current IPS specification does not reference the C-CDA Participant Extension but there is intent to reintroduce such a reference in a future version, the request to include a hyperlink can be deferred. This preserves the editorial improvement for when it becomes applicable again.

**Action:** Defer to a future IPS ballot cycle when the participant extension reference is re-evaluated.

---

## Recommendation

**Option A: Persuasive** — The ticket was deliberately re-opened by Lynn Laakso in November 2022 following STU update publication, indicating the IPS specification still contains text referencing the C-CDA on FHIR Participant Extension that would benefit from an inline hyperlink. The change is a minor editorial improvement with no technical or conformance impact. The WG should verify the current IPS spec text and the validity of the target URL, then apply the link. If the participant extension text has been fully removed, fall back to Option B.
