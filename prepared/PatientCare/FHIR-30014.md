## Description

Daniel Vreeman proposes merging the PACIO Functional Status IG and Cognitive Status IG into a single combined IG. The rationale:

- **>80% narrative overlap** between the two IGs with no discernable differences in driving use cases.
- Profile structures and element bindings are essentially the same; only a few parts (e.g., extensions) differ.
- Splitting into two IGs with different profile names for content drawn from the same instruments (e.g., CMS-required MDS, OASIS) **imposes unnecessary burden** on implementers and receivers.
- The rationale for the split was not described in either IG.
- A merged IG with a common core and domain-specific subsections would provide a **more robust foundation** for future PACIO project work.

## Analysis

This ticket requests merging the PACIO Functional Status and Cognitive Status IGs. Critically, **the merge has already occurred**. The PACIO project created the **Personal Functioning and Engagement (PFE) IG**, which combines both IGs into a unified specification. Evidence:

1. **PFE STU1** was published in January 2024 (http://hl7.org/fhir/us/pacio-pfe/STU1).
2. **PFE STU2** was published in July 2025 (http://hl7.org/fhir/us/pacio-pfe/STU2).
3. FHIR-49422 and FHIR-49425 (January 2025) explicitly state: *"PACIO is in the process of archiving the Functional and Cognitive Status IGs as they have been replaced with the PACIO Personal Functioning and Engagement IG."*
4. A dedicated Zulip channel was created for PACIO PFE in May 2024.

The original 2021 vote of "Considered for Future Use" (6-0-0) deferred action while PACIO/Gravity Project investigated runtime value set differentiation. The ticket was re-opened post-STU1 for reconsideration but was never formally resolved despite the merge having been completed.

The companion ticket FHIR-29977 (filed against the Cognitive Status IG) has the identical text and history, and should be resolved in tandem.

## Proposed Dispositions

### Disposition A: Applied — Resolved by Existing Work
Close this ticket as **Applied** (or equivalent satisfied resolution), noting that the requested merge has been accomplished through the creation of the PACIO Personal Functioning and Engagement (PFE) IG, now at STU2. The original Functional Status and Cognitive Status IGs are being archived. Companion ticket FHIR-29977 should be resolved identically.

- **Pros:** Accurately reflects that the requested change was implemented. Cleans up a stale ticket.
- **Cons:** The ticket was never formally acted upon by the work group — the merge was driven by the PACIO project independently.

### Disposition B: Persuasive — Formally Acknowledge the Merge
Resolve as **Persuasive**, explicitly acknowledging that the submitter's request was valid and has been addressed by the PFE IG. Document the PFE IG URL and archival status of the predecessor IGs in the resolution notes.

- **Pros:** Gives formal credit to the ballot comment as having driven a concrete outcome. Creates a clear audit trail.
- **Cons:** "Persuasive" typically implies the work group made a change in response; the timeline suggests the merge may have been independently motivated by the PACIO project.

### Disposition C: Considered for Future Use — No Further Action Needed
Restore the original "Considered for Future Use" resolution, updating the notes to indicate the issue has been overtaken by events (the PFE IG merge).

- **Pros:** Consistent with the original ballot disposition.
- **Cons:** Misleading — the issue is not being deferred to the future; it has already been resolved. Leaves the ticket in an ambiguous state.

## Recommendation

**Disposition A: Applied — Resolved by Existing Work** is recommended. The merge requested in this ticket has been fully accomplished through the PACIO Personal Functioning and Engagement (PFE) IG, which has progressed through STU1 (2024) and STU2 (2025). The original Functional Status and Cognitive Status IGs are being archived. Closing as "Applied" accurately reflects the outcome and cleans up both this ticket and its companion FHIR-29977. The resolution notes should reference the PFE IG (http://hl7.org/fhir/us/pacio-pfe/STU2) and the archival of the predecessor IGs.
