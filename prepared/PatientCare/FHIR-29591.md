## Description

The submitter observes that the UseOfDevice profiles between the PACIO Cognitive Status and Functional Status IGs appear identical except for some narrative text. They request that one IG define the profile and have it used by both. Similarly, the assistance and device extensions look the same other than narrative text and may be able to be reused across both IGs rather than being duplicated.

## Analysis

This ticket addresses a real and persistent concern about unnecessary duplication between the PACIO Functional Status and Cognitive Status IGs. The core issue is that UseOfDevice profiles and the AssistanceRequired/DevicePatientUsed extensions are substantively identical across both IGs, differing only in narrative text. This creates maintenance burden, increases the risk of divergence, and confuses implementers.

The issue was previously voted on (6-0-1) and resolved as "Considered for Future Use" in August 2021, but was then re-opened in November 2021 for post-STU1 consideration. It remains unresolved. Notably, FHIR-43145 (filed by Lloyd McKenzie in 2023) raises the same concern, indicating this remains an active pain point for the community.

The broader question of whether the two IGs should be merged entirely (FHIR-30014, FHIR-29977) is closely related but represents a larger scope change. This ticket could be addressed independently by consolidating shared artifacts without requiring a full IG merge.

## Proposed Dispositions

### Disposition A: Persuasive with Modification
**Action:** Accept the request and consolidate shared profiles/extensions into a single definition. Either designate one IG as the canonical source for UseOfDevice, AssistanceRequired, and DevicePatientUsed (with the other IG declaring a dependency), or extract these shared artifacts into a common PACIO base IG or the HL7 core extensions IG. Update narrative text in both IGs to reference the single canonical definition.

**Rationale:** The duplication is acknowledged by the PACIO team and by independent reviewers (Lloyd McKenzie in FHIR-43145). Consolidation reduces maintenance burden, prevents divergence, and follows HL7 best practices for extension definition. This is a narrower, more tractable change than merging the two IGs entirely.

### Disposition B: Considered for Future Use
**Action:** Defer resolution pending the outcome of the broader IG consolidation discussion (FHIR-30014/FHIR-29977) and any PACIO restructuring efforts. Revisit when the PACIO project has a clearer direction on IG architecture.

**Rationale:** This was the previous disposition. If the PACIO project is planning to merge the two IGs or restructure them significantly, addressing profile/extension reuse in isolation may result in rework. Deferral avoids premature action.

### Disposition C: Not Persuasive
**Action:** Decline the request, maintaining separate definitions in each IG.

**Rationale:** The two IGs may evolve to use different value sets for their respective domains (functional vs. cognitive), which could eventually justify separate extension definitions. The original deferral comment noted that PACIO was investigating mechanisms for domain-specific value set differentiation. However, this rationale is weak since the extensions themselves are structural, not value-set-dependent, and the differentiation concern applies to observation profiles rather than device/assistance extensions.

## Recommendation

**Disposition A: Persuasive with Modification** is recommended. The duplication has been acknowledged by multiple reviewers over a three-year period (2020–2023), the previous deferral has not yielded resolution, and FHIR-43145 confirms this remains an active issue. Consolidating shared profiles and extensions into a single canonical source is a well-scoped, low-risk change that aligns with HL7 best practices and can proceed independently of the broader IG merge discussion.
