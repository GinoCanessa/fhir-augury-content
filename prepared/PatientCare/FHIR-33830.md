## Description

The reporter requests that the **procedure-cpt** Value Set canonical URL be moved from its current root at `hl7.org/fhir` to `terminology.hl7.org` (THO — the HL7 Terminology server). A UTG (Unified Terminology Governance) ticket should be created to register the Value Set in THO.

This ticket was imported from a ballot comment (Comment 12, imported by Ron G. Parker).

## Proposed Dispositions

### Disposition A: Persuasive

**Action:** Accept the request as-is. Move the `procedure-cpt` Value Set canonical URL from `hl7.org/fhir` to `terminology.hl7.org` and create a UTG ticket to register it in THO.

**Rationale:** The request aligns with HL7's established direction of consolidating terminology artifacts under THO. The sibling ticket FHIR-33768 (Devices WG) was resolved as "Persuasive with Modification" and applied, establishing precedent. Canonical URLs for Value Sets should be rooted in the terminology server to ensure consistent discoverability and governance.

**Impact:** Requires updating the canonical URL in the Ophthalmology Retinal IG for the procedure-cpt Value Set and submitting a UTG request. Implementers referencing the old URL will need to update their systems, though URL redirects can ease the transition.

### Disposition B: Persuasive with Modification

**Action:** Accept the intent of the request but coordinate the migration of the `procedure-cpt` Value Set canonical URL to THO as part of a batch effort covering all related Ophthalmology Retinal Value Sets (FHIR-33822 through FHIR-33831). File a single UTG ticket to register all affected Value Sets rather than individual UTG tickets per Value Set.

**Rationale:** There are at least 8 sibling tickets requesting the same change for different Value Sets within the same specification. Batching these into a coordinated migration is more efficient, reduces the risk of partial migrations, and ensures consistent handling across the IG. This also follows the approach taken by the Devices WG with FHIR-33768.

**Impact:** Same as Disposition A, but applied across all affected Value Sets simultaneously, which reduces overall effort and ensures consistency. Requires coordination to create one comprehensive UTG submission.

### Disposition C: Not Persuasive

**Action:** Reject the request. Keep the `procedure-cpt` Value Set canonical URL rooted at `hl7.org/fhir`.

**Rationale:** The Ophthalmology Retinal IG has been published with the current canonical URLs. Changing them introduces breaking changes for implementers. The value of the URL migration may not justify the disruption, particularly if the IG has limited active implementation. Additionally, the IG itself may need broader review before individual URL changes are applied.

**Impact:** No changes required. Existing implementations remain unaffected. However, this creates inconsistency with the Devices WG precedent (FHIR-33768) and the broader HL7 direction toward THO.

---

## Recommendation

**Disposition B: Persuasive with Modification** is recommended.

The request is valid and consistent with HL7 terminology governance direction, as demonstrated by the Devices WG resolving the analogous FHIR-33768 as "Persuasive with Modification." However, given that there are at least 8 nearly identical sibling tickets affecting Value Sets in the same Ophthalmology Retinal specification, the most practical approach is to batch-process these changes together. This avoids piecemeal migrations, ensures consistency across the IG, and reduces the total number of UTG tickets. The work group should resolve all related tickets (FHIR-33822 through FHIR-33831) as a coordinated set with a single UTG submission.
