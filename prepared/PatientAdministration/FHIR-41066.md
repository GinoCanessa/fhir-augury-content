## Description

The submitter asks whether the official URL for a terminology artifact (Comment 4 in a series of imported ballot comments) should be anchored under `terminology.hl7.org` rather than `hl7.org/fhir`. This is part of a broader pattern of ballot comments by Ron G. Parker questioning whether code systems and value sets defined in FHIR IGs should have their canonical URLs rooted in the HL7 Terminology (THO) namespace rather than the FHIR IG namespace.

## Resolution Description

> Based on other more substantive changes to the spec, we are removing the Subscription Events value set and code system. So the question of whether it belongs in THO or in this IG is moot.

## Proposed Dispositions

### Disposition A: Not Persuasive (Agree with Current Resolution)

**Rationale:** The resolution correctly identifies that the question is moot because the Subscription Events value set and code system are being removed from the RTLS specification due to other substantive changes. Since the artifacts in question will no longer exist in the IG, there is no URL to anchor anywhere. This is consistent with the resolution of sibling ticket FHIR-41064 (also Not Persuasive) and the analogous mCODE ticket FHIR-41049.

**Suggested response text:** "The Subscription Events value set and code system have been removed from the RTLS specification as part of broader substantive changes. Since these artifacts no longer exist in the IG, the question of URL anchoring is moot. No change required."

### Disposition B: Not Persuasive with Modification

**Rationale:** While the specific artifacts are being removed (making the immediate ask moot), the workgroup could acknowledge the commenter's broader point that IG-defined terminology should follow THO URL conventions where applicable, and commit to applying this principle to any future terminology artifacts added to the RTLS IG. This mirrors the resolution given to sibling ticket FHIR-41065.

**Suggested response text:** "The Subscription Events value set and code system are being removed, making this specific request moot. However, we acknowledge the principle that terminology artifacts should use terminology.hl7.org URLs where appropriate, and will apply this convention to any future terminology artifacts in the specification."

### Disposition C: Considered for Future Use

**Rationale:** If the workgroup anticipates that new terminology artifacts may be introduced to the RTLS IG in the future, this disposition defers action while acknowledging the commenter's underlying concern. This mirrors the resolution given to sibling ticket FHIR-41063 (Comment 1).

**Suggested response text:** "The specific artifacts referenced have been removed. Should new code systems or value sets be introduced in future versions of the RTLS IG, the workgroup will consider anchoring their canonical URLs in terminology.hl7.org as appropriate."

## Recommendation

**Disposition A: Not Persuasive** — This is the most appropriate disposition. The resolution rationale is clear and accurate: the artifacts in question are being removed, rendering the request moot. This is consistent with the resolution applied to the majority of sibling tickets (FHIR-41064, FHIR-41049) and provides a straightforward, defensible answer. There is no actionable change to make.
