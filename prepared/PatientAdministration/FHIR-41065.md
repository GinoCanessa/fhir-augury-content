## Summary

This ticket is Comment 3 from a series of imported ballot comments (FHIR-41063 through FHIR-41066) filed by Ron G. Parker, all asking the same fundamental question: should the official URLs for RTLS terminology artifacts be anchored in `terminology.hl7.org` (HL7 Terminology, THO) rather than `hl7.org/fhir`?

The submitter's concern is that terminology artifacts defined in the RTLS Implementation Guide use URLs under the `hl7.org/fhir` namespace, when HL7's current terminology governance guidance recommends that code systems and value sets be published through `terminology.hl7.org`.

## Resolution Description

> In FHIR-41063, we plan to eventually move the code system to THO. However, the value set feels very RTLS specific, so could remain in the RTLS IG long term. So we propose to keep the value set in the RTLS IG, but note that the CLD (Code List Definition) will be updated once the code system moves to THO.

The resolution distinguishes between the **code system** (planned for eventual migration to THO) and the **value set** (RTLS-specific, remaining in the RTLS IG). This acknowledges the submitter's concern in part while deferring the code system migration and retaining IG-specific value sets locally.

## Proposed Dispositions

### Disposition A: Accept as "Not Persuasive with Modification" (Affirm Current Resolution)

**Rationale:** The existing resolution strikes a practical balance. The value set is RTLS-specific and appropriately remains in the RTLS IG. The resolution acknowledges the broader concern by noting the code system will eventually move to THO (as tracked in FHIR-41063). This aligns with the pattern where IG-specific value sets stay in the IG while shared code systems migrate to THO. The CLD update commitment ensures the value set will properly reference the code system once it is in THO.

**Concrete changes:** No specification changes needed for this ticket. The value set URL remains in the RTLS IG. A note should be added (or confirmed present) indicating the CLD will be updated when the code system moves to THO per FHIR-41063.

### Disposition B: Resolve as "Not Persuasive"

**Rationale:** The RTLS IG's terminology artifacts—both code systems and value sets—are tightly coupled to the RTLS domain and have limited reuse outside that context. IG-specific terminology is conventionally published under the IG's own URL namespace. The modification aspect (promising a future CLD update) creates a maintenance obligation that may not be necessary if the code system also remains in the IG. Several similar requests across other IGs (FHIR-41049, FHIR-41064, FHIR-41066) were resolved as "Not Persuasive."

**Concrete changes:** No specification changes. Value set URL stays in the RTLS IG with no commitment to future CLD updates tied to code system migration.

### Disposition C: Resolve as "Persuasive" — Move Value Set to THO

**Rationale:** HL7's terminology governance increasingly directs that terminology artifacts be published through THO for discoverability, governance, and consistent URL management. If the code system is already planned for THO migration (per FHIR-41063), moving the value set alongside it would maintain cohesion between the code system and its value sets. FHIR-42258 (NDH) was resolved as "Persuasive with Modification" for a similar request, demonstrating precedent for moving IG value sets to THO.

**Concrete changes:** Migrate the RTLS value set to `terminology.hl7.org`, update the RTLS IG to reference the new THO-based value set URL, and coordinate timing with the code system migration planned in FHIR-41063.

## Recommendation

**Disposition A: Accept as "Not Persuasive with Modification" (Affirm Current Resolution)**

This disposition is recommended because it correctly differentiates between the code system (shared, THO-bound) and the value set (RTLS-specific, IG-appropriate). The approach is consistent with HL7 terminology governance principles while being pragmatic about IG-specific artifacts. The commitment to update the CLD when the code system moves to THO ensures future alignment without premature migration of a domain-specific value set. The sibling tickets' resolutions form a coherent strategy: the code system will eventually move to THO (FHIR-41063), while IG-specific value sets and other artifacts remain in the RTLS IG.
