## Problem Statement

The examples for the **vhdir-restriction** profile (based on the Consent resource in the Validated Healthcare Directory IG) lack proper terminology as defined in the core FHIR specification. The ticket proposes updating the Restriction examples to use correct, standard code systems from HL7 Terminology (THO).

## Proposed Change

The reporter (Eric Haas) provides a complete replacement example for the Restriction profile (Consent resource with id "restrict"), sourced from the VhDir demo server. The proposed example uses proper terminology bindings including:

- **scope**: `http://terminology.hl7.org/CodeSystem/consentscope` → `patient-privacy`
- **category**: `http://terminology.hl7.org/CodeSystem/v3-ActCode` → `INFA` (information access)
- **provision.actor.role**: `http://terminology.hl7.org/CodeSystem/v3-ParticipationType` → `IRCP` (information recipient)
- **provision.action**: `http://terminology.hl7.org/CodeSystem/v3-ActReason` → `use`
- **provision.securityLabel**: `http://terminology.hl7.org/CodeSystem/v3-ActCode` → `SDV` (sexual assault, abuse, or domestic violence information sensitivity)
- **provision.purpose**: `http://terminology.hl7.org/CodeSystem/v3-ActReason` → `TREAT` (treatment)
- **meta.profile**: `http://hl7.org/fhir/uv/vhdir/StructureDefinition/vhdir-restriction`

## Proposed Dispositions

### Disposition A: Persuasive (Accept as-is)
**Rationale:** The proposed example is well-formed, uses correct HL7 Terminology code systems (`consentscope`, `v3-ActCode`, `v3-ParticipationType`, `v3-ActReason`), and was sourced from the VhDir demo server. The Zulip discussion confirms that Consent examples are known to be broken and need fixing. Prior tickets (FHIR-13420, FHIR-10610) for similar Consent example improvements were accepted. Accept the proposed example replacement directly.

**Action:** Replace the existing Restriction profile examples in the VhDir IG with the proposed Consent resource example, incorporating proper THO terminology bindings as specified.

### Disposition B: Persuasive with Modification
**Rationale:** The proposed example is directionally correct, but should be reviewed and potentially updated for alignment with more recent FHIR versions (the ticket dates from 2019). The example may need adjustments for R5 changes to the Consent resource structure (e.g., `scope` was removed in R5, `provision` structure changed). Accept the intent but have the editor update the example to match the current VhDir IG version and FHIR version in use.

**Action:** Accept the intent to fix Restriction examples with proper terminology. Have the VhDir IG editor update the example to align with the current FHIR version and VhDir IG structure, using the proposed terminology bindings as a starting point.

### Disposition C: Not Persuasive (Reject)
**Rationale:** The VhDir IG is a Universal Realm IG that has seen limited recent activity. Example updates, while useful, may not be worth the effort if the IG is not actively being maintained or balloted. The current examples, while not using ideal terminology, are still functional for demonstrating the profile's intent.

**Action:** Close without change. Revisit if the VhDir IG enters active development or a new ballot cycle.

## Recommendation

**Disposition B: Persuasive with Modification**

The core request is valid and well-supported — the Restriction examples genuinely need proper terminology, and the Zulip discussions confirm this is a known issue. However, given that this ticket is from 2019 and FHIR has evolved (particularly the Consent resource in R5), accepting the example verbatim may introduce outdated patterns. The safest approach is to accept the intent and have the editor produce an updated example that uses the correct terminology bindings while ensuring alignment with the current VhDir IG version and target FHIR version. This follows the precedent set by FHIR-13420 and FHIR-10610, both of which were accepted with similar Consent example improvements.
