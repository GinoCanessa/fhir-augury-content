## Summary

The submitter (Bas van den Heuvel) requests that several extensions defined in the NDH IG be moved to the FHIR core extension set, arguing they represent general-purpose mechanisms not specific to national directories. The extensions in question are:

- **Logo** — used in NDH HealthcareService, Organization, and OrganizationAffiliation profiles
- **Endpoint Rank** — used in NDH Endpoint profile
- **Endpoint Access Control Mechanism** — used in NDH Endpoint profile
- **Associated Servers** — used in NDH Endpoint profile

The ticket was originally resolved as "Considered for Future Use" (vote 5-0-1, 2023-12-20) with direction to submit a PA ticket for core extensions in R6. It was re-opened post-STU publication in April 2025 for further review, and subsequently resolved as **Not Persuasive** during the January 2026 WGM.

### Resolution Description

> The maturity of these extensions has not yet been proven. The IG is the right place for that maturity to be shown. Further, when an extension is sufficiently ready to be added to the extension pack, that would be a new Jira ticket first to add the extension to the extension pack, and then when that is done another Jira ticket to adjust NDH to the extension pack extension.

## Proposed Dispositions

### Disposition A: Not Persuasive ⭐ RECOMMENDED

**Rationale:** The extensions in question (Logo, Rank, Access Control Mechanism, Associated Servers) have not yet demonstrated sufficient maturity or cross-implementation adoption to warrant promotion to the FHIR core extension set. John Moehrke's December 2025 analysis confirms that only Logo has a partial equivalent in R6 (on HealthcareService only), while Rank, Access Control, and Associated Servers have no R6 counterparts or clear extension candidates. The NDH IG is the appropriate venue for these extensions to mature. The established pathway — file a Jira ticket to add a mature extension to the extension pack, then file another to adjust NDH — provides a clear promotion mechanism when readiness is demonstrated. This aligns with the resolution of the related FHIR-47300 (also Not Persuasive) which sought to add similar access control concepts to core Endpoint.

**Suggested wording:** "Not Persuasive. The extensions have not yet demonstrated sufficient maturity for core inclusion. The NDH IG is the appropriate maturation environment. When individual extensions are ready, separate tickets should be filed to add them to the extension pack, followed by NDH alignment."

### Disposition B: Persuasive with Modification — Defer with tracked follow-up tickets

**Rationale:** The submitter's underlying concern has merit — FHIR-42659 (Persuasive) already acknowledged that NDH-specific code systems for access control should be universal, and the extension pack's Organization Brand extension already partially overlaps with NDH's Logo extension. Rather than a blanket rejection, the work group could create individual tracker tickets for each extension (Logo, Rank, Access Control, Associated Servers) to evaluate promotion readiness independently as NDH matures. This provides a structured path forward without requiring immediate action.

**Suggested wording:** "Persuasive with Modification. While the extensions are not yet mature enough for immediate core inclusion, create individual tracker tickets for each candidate extension (Logo, Endpoint Rank, Endpoint Access Control Mechanism, Associated Servers) to monitor readiness for future promotion to the extension pack."

### Disposition C: Considered for Future Use

**Rationale:** This was the original 2023 resolution (5-0-1 vote) before re-opening. The concept remains valid: the extensions may eventually warrant core inclusion but need further maturation in NDH first. However, this disposition was already tried and subsequently overturned by re-opening, suggesting the work group wanted a more definitive resolution. Additionally, the direction from the original vote ("submit a ticket to PA to add core extensions to R6") was not acted upon in over a year, indicating the extensions were not prioritized for promotion.

**Suggested wording:** "Considered for Future Use. The extensions may warrant core inclusion once maturity is demonstrated through broader adoption. The NDH IG remains the primary maturation venue."

---

## Recommendation

**Disposition A: Not Persuasive** is recommended. The technical analysis by john_moehrke demonstrates that most of these extensions lack R6 equivalents and have not yet proven general-purpose applicability beyond NDH. The resolution provides a clear, actionable pathway (extension pack → NDH alignment) for future promotion when maturity is achieved, without prematurely elevating IG-specific concepts to core. This is consistent with the outcome of FHIR-47300 and reflects the work group's direction during the January 2026 WGM.
