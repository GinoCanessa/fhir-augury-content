## Description

The submitter requests that the NDH-specific extensions indicating IHE actors and transactions supported by an endpoint be promoted to FHIR core extensions rather than remaining NDH-specific. The argument is that this functionality is broadly useful and should be more readily available across implementations.

## Resolution Description

> The extension needs to be added to the extension pack by a Jira ticket for the global PA committee. Once that extension exists then NDH can consider that extension in a new Jira ticket.

## Analysis

The core request is valid: NDH-defined extensions for IHE actor/transaction indication on Endpoint have broad utility beyond NDH. However, the ecosystem has evolved since the original filing:

1. **FHIR-48540** (Applied) already consolidated the NDH extensions and dropped the "IHE" prefix, making them more general within NDH.
2. **FHIR-53711** (Persuasive with Modification) targets the root problem in FHIR Core R6 by proposing richer Endpoint.connectionType structure (canonical URI, version, actor, options).
3. The "Not Persuasive" resolution directs the submitter to pursue this via the global PA extension pack process, which is exactly what FHIR-53711 does at the core level.

The ticket is currently in **Triaged** status with a **Not Persuasive** resolution, and the reporter has been asked to respond but has not yet done so (as of the available data).

## Proposed Dispositions

### Disposition A: Affirm Not Persuasive (Recommended)
**Resolution:** Not Persuasive

**Rationale:** The underlying need is valid and is being addressed through better-scoped mechanisms. FHIR-53711 proposes the necessary structural changes to Endpoint.connectionType in FHIR Core R6 (resolved as Persuasive with Modification). FHIR-48540 has already been applied to NDH, consolidating and generalizing the extensions. This ticket asks NDH to promote its extensions to core, but the correct approach — filing against FHIR Core — has already been done via FHIR-53711. The "Not Persuasive" resolution is appropriate because the *mechanism* requested (NDH promoting its own extensions) is not the right path; the *goal* is being achieved through the proper channel.

### Disposition B: Resolve as Duplicate of FHIR-53711
**Resolution:** Duplicate

**Rationale:** FHIR-53711 directly addresses the same fundamental need — making IHE actor/transaction indication available beyond NDH — but targets FHIR Core R6 where it belongs. Since FHIR-53711 is already resolved as Persuasive with Modification and is being tracked by Patient Administration, this ticket can be closed as a duplicate. This acknowledges the submitter's valid concern while pointing to the active ticket.

### Disposition C: Defer Pending FHIR-53711 Outcome
**Resolution:** Deferred

**Rationale:** While FHIR-53711 has been resolved as Persuasive with Modification, it has not yet been applied to the spec. If R6 changes are delayed or modified, NDH may still need to maintain its own extensions. Deferring preserves the option to revisit after R6 publication. However, this adds tracking overhead for a ticket that is effectively superseded.

## Recommendation

**Disposition A: Affirm Not Persuasive.** The resolution rationale is sound — the extension needs to come from FHIR Core, not NDH. FHIR-53711 is the correct vehicle and has already been accepted. john_moehrke's comment explicitly recommends this path. The reporter has been asked to respond but has not objected. Affirming the current resolution is the cleanest outcome that respects both the submitter's intent and the proper standards process.
