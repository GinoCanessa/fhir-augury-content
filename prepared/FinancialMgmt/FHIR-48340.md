## Summary

This change request asks that multiple value sets (and related code systems where applicable) currently defined in the CARIN Blue Button Implementation Guide (C4BB) be submitted to the HL7 Terminology (THO) server for broader community use. Nine value sets are enumerated, spanning institutional claim subtypes, care team roles, NUBC admission/discharge codes, diagnosis codes, diagnosis types (inpatient and outpatient EOB), procedure types, and present-on-admission indicators. The reporter notes that some of these value sets already partially exist in THO (e.g., PresentOnAdmission) and that the CareTeamRole value set may need discussion regarding scope beyond physicians.

## Details

### Description

The ticket requests submission to THO of the following value sets from the CARIN Blue Button IG:

   - *Note:* CareTeamRole should potentially be more open to non-physicians, but the reporter observes that only a physician can submit a claim. May need discussion.
3. **NUBCPointOfOriginForAdmission** — [AHANUBCPointOfOriginForAdmissionOrVisit](https://hl7.org/fhir/us/carin-bb/ValueSet-AHANUBCPointOfOriginForAdmissionOrVisit.html)

5. **Diagnosis** — [CDCICD910CMDiagnosisCodes](https://hl7.org/fhir/us/carin-bb/ValueSet-CDCICD910CMDiagnosisCodes.html)
6. **DiagnosisType (Inpatient EOB)** — [C4BBClaimInpatientInstitutionalDiagnosisType](https://hl7.org/fhir/us/carin-bb/ValueSet-C4BBClaimInpatientInstitutionalDiagnosisType.html)
7. **Procedure.type (Inpatient EOB)** — [C4BBClaimProcedureType](https://hl7.org/fhir/us/carin-bb/ValueSet-C4BBClaimProcedureType.html)
8. **DiagnosisType (Outpatient EOB)** — [C4BBClaimOutpatientInstitutionalDiagnosisType](https://hl7.org/fhir/us/carin-bb/ValueSet-C4BBClaimOutpatientInstitutionalDiagnosisType.html)
9. **PresentOnAdmission** — [AHANUBCPresentOnAdmission](https://hl7.org/fhir/us/carin-bb/ValueSet-AHANUBCPresentOnAdmission.html)
   - *Note:* Already partially exists in THO. The THO v2-0895 value set ([v2-0895](https://terminology.hl7.org/3.1.0/ValueSet-v2-0895.html)) and the POAIndicators value set exist in two versions — [3.1.0](https://terminology.hl7.org/3.1.0/ValueSet-POAIndicators.html) (missing code "1") and [6.0.2](https://terminology.hl7.org/6.0.2/ValueSet-POAIndicators.html) (includes all codes).

## Proposed Dispositions

### Disposition A: Accept — Submit All Listed Value Sets to THO

**Action:** Accept the change request and submit all nine listed C4BB value sets (and their related code systems) to THO. For the PresentOnAdmission value set, coordinate with THO maintainers to ensure the existing POAIndicators value set is updated (v6.0.2 includes all codes) and the C4BB-specific value set references or aligns with it. For the Inpatient DiagnosisType value set, expand it per the commenter's suggestion to include concepts from the base Claim.diagnosis.type example value set.

**Rationale:** Moving IG-specific terminology into THO promotes reuse across IGs, reduces duplication, and aligns with HL7's terminology governance strategy. The value sets are mature and actively used in CARIN Blue Button implementations.

### Disposition B: Accept with Phased Approach — Submit Non-Controversial Value Sets First, Defer CareTeamRole

**Action:** Submit the seven value sets that are straightforward (InstitutionalClaimSubType, NUBCPointOfOriginForAdmission, DischargeStatus, Diagnosis, DiagnosisType for inpatient and outpatient EOB, and Procedure.type) to THO immediately. Defer the ClaimInstitutionalCareTeamRole value set pending workgroup discussion about whether it should be scoped more broadly to include non-physician roles. For PresentOnAdmission, harmonize with the existing THO POAIndicators value set rather than creating a new one.

**Rationale:** The reporter explicitly flagged CareTeamRole as potentially needing discussion. A phased approach avoids blocking the majority of value sets while allowing deliberation on the more nuanced one.

### Disposition C: Decline — Keep Value Sets in the CARIN Blue Button IG

**Action:** Decline the request. The value sets are specific to the CARIN Blue Button use case and should remain defined in that IG. Advise the reporter to raise individual THO submission requests through the Vocabulary WG process if broader adoption is demonstrated.

**Rationale:** Some of these value sets reference external code systems (AHA NUBC, CDC ICD-10-CM) with their own licensing and maintenance considerations. Submitting them to THO could create governance complexity. However, this risks continued duplication and inconsistency across IGs.

---

## Recommendation

**Disposition B: Accept with Phased Approach** is recommended.

The bulk of the requested value sets are well-defined, stable, and clearly beneficial to submit to THO for broader community reuse. However, the reporter explicitly noted that ClaimInstitutionalCareTeamRole "may need to discuss" due to the physician-only scoping question. A phased approach lets the workgroup move forward on the uncontroversial value sets immediately while properly deliberating the CareTeamRole scope. Additionally, the PresentOnAdmission situation — where THO already has overlapping value sets — warrants coordination rather than duplication. This disposition balances progress with governance prudence.
