## Details

The reporter identifies discrepancies between the Contract resource's legal state machine diagram and the `contract-legalstate` value set. The Contract introduction page states that the legal state diagram states are formally defined in the value set, but the two are out of sync.

**States in the diagram but missing from the value set:**
- addendum, conditional, counter, draft, effective, exchange of contracts, modified, offer, pended, Request for Information

**States in the value set but missing from the diagram:**
- amended, appended, entered-in-error, offered, policy

The reporter proposes three possible resolutions: (1) update the diagram to match the value set, (2) update the value set to match the diagram, or (3) revise the documentation to clarify how the two are related.

A 2021 comment from **pknapp** notes the issue is "being reviewed in light of recent changes," but no resolution has been applied.

## Proposed Dispositions

### Disposition A: Accept — Reconcile value set and diagram together

**Persuasive.** Accept the issue and perform a comprehensive reconciliation:

1. Audit every state in both the legal state diagram and the `contract-legalstate` value set.
2. Add missing diagram states (`addendum`, `conditional`, `counter`, `draft`, `effective`, `exchange of contracts`, `modified`, `offer`, `pended`, `Request for Information`) to the value set as appropriate, with proper definitions.
3. Add missing value set codes (`amended`, `appended`, `entered-in-error`, `offered`, `policy`) to the diagram, or remove them from the value set if they are not valid legal states.
4. Ensure the introductory text accurately describes the relationship between the diagram and the value set.
5. Coordinate with related tickets FHIR-22704 and FHIR-26966 to ensure the state model is also aligned with the standard FHIR event pattern.

This is the most thorough approach and resolves the core issue of inconsistency while addressing the broader Contract state model concerns raised in related tickets.

### Disposition B: Alternative — Decouple the diagram from the value set

**Persuasive with Modification.** Acknowledge the discrepancy but resolve it by explicitly decoupling the two artifacts:

1. Revise the Contract introduction text to clarify that the legal state diagram is illustrative and does not define normative states.
2. State that `contract-legalstate` is the authoritative source for legal state codes.
3. Relabel the diagram as informational/example workflow rather than a formal state machine.
4. Remove or downgrade the sentence claiming the diagram states are "formally defined" in the value set.

This is a lighter-touch fix that eliminates the inconsistency without requiring a full state model overhaul, but may leave questions about the diagram's purpose and accuracy.

### Disposition C: Decline — Defer to broader Contract state model rework

**Not Persuasive.** Decline this specific ticket in favor of the broader rework being discussed in FHIR-22704 and FHIR-26966, which question the entire Contract state model design. The rationale would be that fixing the diagram/value set alignment in isolation is premature when the fundamental state model may change.

This risks leaving an acknowledged inconsistency unresolved indefinitely, especially since FHIR-22704 and FHIR-26966 have also been triaged without resolution since 2019 and 2020 respectively.

## Recommendation

**Disposition A (Accept — Reconcile value set and diagram together)** is recommended. The discrepancy between the diagram and value set has been open since 2019 and is a clear documentation/specification consistency issue. Reconciling them together is the correct approach because both artifacts should accurately represent the Contract resource's legal states. This work should be coordinated with FHIR-22704 and FHIR-26966 to ensure any reconciliation also addresses the broader concerns about Contract state model alignment with FHIR patterns, rather than creating additional rework later.
