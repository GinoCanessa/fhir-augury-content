## Summary

Joan Harper filed this change request to move the canonical URL for CodeSystem-C4BBClaimDiagnosisType from hl7.org/fhir to terminology.hl7.org (THO). The request calls for creating a UTG ticket to register the Code System in THO. The ticket was previously resolved as "Considered for Future Use" during the CARIN Blue Button STU2 Block-Vote-6 in July 2022, with a vote of 13-0-0 and a plan to move the CodeSystem once the related UTG ticket UP-309 ("Create Diagnosis Type Code Systems that is not an example") was completed and published. The resolution was subsequently reverted by Lynn Laakso post STU2 publication, and the ticket was re-opened with status "Triaged." The ticket remains unresolved and unassigned.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Move the canonical URL for CodeSystem-C4BBClaimDiagnosisType from hl7.org/fhir to terminology.hl7.org (THO). Create a UTG ticket (or verify that UP-309 has been completed) to register the Code System in THO, and update all references in the CARIN Blue Button IG accordingly.

#### Justification

The request aligns with HL7's terminology governance strategy of centralizing Code Systems in THO. The original block vote passed unanimously (13-0-0), indicating strong workgroup consensus that this migration should happen. The ticket was only deferred — not rejected — pending completion of the UTG infrastructure work (UP-309). Re-anchoring the canonical URL in THO improves discoverability, ensures proper terminology governance, and is consistent with similar migrations planned across multiple CARIN BB CodeSystems (FHIR-35390, FHIR-35391, FHIR-35393, FHIR-35396, FHIR-35398).

---

### Disposition B: Defer Pending UTG Readiness

#### Proposal

Resolve as "Considered for Future Use" again, contingent on verifying the current status of UP-309 and the THO infrastructure required to host CodeSystem-C4BBClaimDiagnosisType. Set a specific milestone or target ballot for re-evaluation rather than leaving it open-ended.

#### Justification

The original deferral rationale — that the move depends on completion of UP-309 — may still apply if the UTG ticket has not been completed. Moving the CodeSystem before the THO infrastructure is ready could cause broken references and implementation disruption. A structured deferral with a concrete re-evaluation date avoids indefinite limbo while acknowledging the dependency. This approach was previously supported unanimously by the workgroup.

---

### Disposition C: Decline

#### Proposal

Resolve as "Not Persuasive." The CodeSystem-C4BBClaimDiagnosisType is specific to the CARIN Blue Button IG and does not need to be hosted in THO at this time.

#### Justification

Several related ValueSet tickets in the same block vote (e.g., FHIR-35413, FHIR-35414, FHIR-35415) were resolved as "Not Persuasive" on the grounds that the artifacts are IG-specific and do not warrant the maintenance overhead of THO migration. A similar argument could apply here: the CodeSystem is maintained by the CARIN BB project team and moving it introduces dependency on THO publication timelines without clear immediate benefit to implementers. However, this position is weaker for CodeSystems than for ValueSets, as CodeSystems are more foundational terminology artifacts.

---

### Recommendation

**Recommended disposition:** A (Accept As Requested)

This ticket has been open since January 2022, was unanimously supported for eventual migration in the July 2022 block vote, and was re-opened specifically to be actioned post STU2 publication. The workgroup has already established consensus that CodeSystem-C4BBClaimDiagnosisType should move to THO — the only reason for deferral was infrastructure readiness (UP-309). Given that over two years have passed since the original deferral, it is reasonable to proceed with the migration or, at minimum, verify the current status of UP-309 and act accordingly. Accepting the request fulfills the workgroup's stated intent, aligns with HL7 terminology governance best practices, and is consistent with the treatment of sibling tickets (FHIR-35391, FHIR-35393, FHIR-35396, FHIR-35398) that share the same migration plan. Continued deferral risks this ticket languishing indefinitely without resolution.
