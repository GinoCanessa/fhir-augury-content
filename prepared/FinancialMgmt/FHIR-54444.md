## Summary

The ResourceList page categorizes InsurancePlan and InsuranceProduct under the Financial section, but neither resource is mentioned or described on the Financial Module page (`financial-module.html`). This creates an inconsistency between the specification's resource categorization and the module documentation. The reporter identifies this as a technical correction—the Financial Module page should reference all resources that the ResourceList page assigns to the Financial category.

## Details

### Description

InsurancePlan and InsuranceProduct are listed in the Financial section on the ResourceList page but are not addressed on the Financial Module page. This means implementers navigating via the Financial Module page will not discover these two resources, even though the specification's own categorization places them in the Financial domain.

## Proposed Dispositions

### Disposition A: Accept (Persuasive)

**Action:** Add InsurancePlan and InsuranceProduct to the Financial Module page (`financial-module.html`) with appropriate descriptions, relationship explanations, and diagram references.

**Rationale:** The ResourceList page explicitly categorizes InsurancePlan and InsuranceProduct under Financial. The Financial Module page should be consistent and list all resources in its category. FHIR-47508 (Applied) already set a precedent by adding InsuranceProduct to the administration-module diagram after the InsurancePlan/InsuranceProduct split. The same consistency correction is needed for the Financial Module page.

**Scope of Change:** Update `source/financial-module.html` to include InsurancePlan and InsuranceProduct in the resource listing and index. Add brief descriptions and any relevant relationship context. This is a documentation-only change with no impact on resource definitions.

### Disposition B: Accept with Modification (Persuasive with Modification)

**Action:** Add InsurancePlan and InsuranceProduct to the Financial Module page, but coordinate with FHIR-50478 (which requests a broader update including diagrams and relationship documentation) and FHIR-54121/FHIR-54124 (which discuss whether these resources should be moved to "Additional" status).

**Rationale:** While the inconsistency is clear, there are active parallel discussions about whether InsurancePlan, InsuranceProduct, and Invoice should be reclassified as "Additional Resources" (FHIR-54121, FHIR-54124) and about providing more comprehensive relationship documentation (FHIR-50478). Rather than making a minimal fix now that might be superseded, coordinate the change so that the Financial Module page update reflects whatever categorization decisions are made on those related tickets.

**Scope of Change:** Conditional on outcomes of FHIR-54121 and FHIR-50478. If resources remain in the Financial category, add them to the Financial Module page with full relationship documentation per FHIR-50478. If moved to Additional, update the ResourceList page instead.

### Disposition C: Decline (Not Persuasive)

**Action:** Decline the change as filed and redirect to FHIR-50478, which already covers adding InsurancePlan and InsuranceProduct to the Financial Module page as part of a broader update request.

**Rationale:** FHIR-50478 ("Update the Financial Module Page and related resource intro sections to better explain the relationships between them") already explicitly requests that InsurancePlan, InsuranceProduct, and all Financial resources be added to the Financial Module index. This ticket is a subset of that broader request and would result in duplicate work. Additionally, FHIR-54121 and FHIR-54124 raise questions about whether these resources should remain in the Financial category at all, making a standalone fix premature.

**Risk:** The inconsistency persists until FHIR-50478 is resolved, which could confuse implementers in the interim.

## Recommendation

**Disposition A: Accept (Persuasive)** is recommended. The reported issue is a straightforward, verifiable inconsistency — the ResourceList page lists InsurancePlan and InsuranceProduct under Financial, but the Financial Module page omits them. This is a textbook Technical Correction. FHIR-47508 already established the precedent of updating module pages after the InsurancePlan/InsuranceProduct split, and FHIR-19287 confirmed the Financial categorization was intentional. The fix is a low-risk documentation-only change. While FHIR-50478 requests a broader update, this ticket's narrower scope can be resolved independently — adding the resource references now does not preclude a more comprehensive update later. The questions raised in FHIR-54121/FHIR-54124 about recategorization are separate decisions; until those are resolved, the current categorization should be consistently reflected across all pages.
