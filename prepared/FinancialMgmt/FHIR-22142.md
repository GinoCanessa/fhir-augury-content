## Description

The reporter (Grahame Grieve) identifies that `Contract.term.action.performer` has inappropriate resource types in its reference target list. Specifically:

- **Substance** and **Location** should be **removed** — neither can logically be "asked to perform (or not perform) the action" as described by the element's definition.
- **HealthcareService** should be **added** — it is a valid performer of contract actions.
- A **typo** in the element description: "ction" should read "action".

The current target list for `Contract.term.action.performer` is:
> CareTeam, Device, Location, Organization, Patient, Practitioner, PractitionerRole, RelatedPerson, Substance

## Analysis

This ticket has been open since **May 2019** (over 6 years) and remains unresolved. There is strong consensus across multiple Zulip discussions and from multiple community members (Grahame Grieve, Lloyd McKenzie, Rik Smithies, Thomas Beale) that:

1. **Substance** does not belong in the performer target list — substances cannot perform contract actions.
2. **Location** does not belong — locations cannot perform actions either.
3. **HealthcareService** should be added — it is a valid performer type, and other performer-typed elements (e.g., `CarePlan.activity.detail.performer`, `ServiceRequest.performer`) already include it.

A newer related ticket **FHIR-52996** goes further by proposing that if the intent is to keep a "thing" reference, Substance should become Medication (since medications can be deliberately administered). However, FHIR-22142's original request to simply remove Substance is also valid.

The comparison with similar elements is instructive:
- `CarePlan.activity.detail.performer`: CareTeam, Device, **HealthcareService**, Organization, Patient, Practitioner, PractitionerRole, RelatedPerson — no Location, no Substance
- `ServiceRequest.performer`: same list — no Location, no Substance

The typo ("ction" → "action") is a straightforward editorial fix.

## Proposed Dispositions

### Disposition A: Persuasive (Accept as Submitted)
- **Remove** Substance and Location from `Contract.term.action.performer` target list.
- **Add** HealthcareService to the target list.
- **Fix** the typo ("ction" → "action") in the element description.
- **Resulting target list:** CareTeam, Device, HealthcareService, Organization, Patient, Practitioner, PractitionerRole, RelatedPerson
- **Rationale:** This aligns `Contract.term.action.performer` with other performer-typed elements across FHIR and has broad community support. The element's definition ("entity that will execute the action") clearly excludes substances and locations.
- **Risk:** Low. Existing profiles referencing Substance or Location as performers would need updating, but such usage is unlikely in practice.

### Disposition B: Persuasive with Modification — Replace Substance with Medication
- **Remove** Location from the target list.
- **Replace** Substance with Medication (per FHIR-52996's recommendation).
- **Add** HealthcareService to the target list.
- **Fix** the typo.
- **Resulting target list:** CareTeam, Device, HealthcareService, Medication, Organization, Patient, Practitioner, PractitionerRole, RelatedPerson
- **Rationale:** Incorporates the insight from FHIR-52996 that if a "thing" can perform a contract action, it would be a Medication rather than a raw Substance. Addresses both tickets in one change.
- **Risk:** Low-medium. Adding Medication is a broader change than originally requested and may need coordination with FHIR-52996.

### Disposition C: Not Persuasive — Defer to Broader Pattern Review
- Decline to make changes now; wait for a comprehensive review of all performer/participation type lists across FHIR.
- **Rationale:** The Zulip discussions suggest this is part of a systemic issue with Party/Participation patterns. A piecemeal fix may not align with a future broader cleanup.
- **Risk:** The ticket has already been open 6+ years and was previously deferred. Further delay risks the issue never being addressed. The fix is small and low-risk.

## Recommendation

**Disposition A: Persuasive** — Accept the change as submitted. Remove Substance and Location, add HealthcareService, and fix the typo. This is a well-supported, low-risk change that aligns the element with established patterns for other performer references in FHIR. FHIR-52996 (Substance → Medication) can be addressed separately as its own ticket if the workgroup sees merit in adding Medication.
