## Summary

This ticket requests adding the GS1 Global Location Number (GLN) identifier system to the FHIR identifier registry as a "Known Identifier System." The GLN is identified by OID 1.3.88 and the URI https://www.gs1.org/gln. The reporter notes that the current identifier registry is US-centric and that GLN is a globally recognized identifier system that should be included.

A comment from Lloyd McKenzie (mckenzie) redirects this to the HL7 Terminology Authority (HTA), noting "HTA owns requests for identifier registrations." A related HTA ticket (HTA-40) was filed. Subsequent Zulip discussions indicate the GLN NamingSystem has since been created in HL7 Terminology (THO) at https://terminology.hl7.org/NamingSystem-GLN.html, and a broader effort is underway to migrate identifier registry content from the FHIR spec to THO.

## Details

- **What is GLN?** The Global Location Number (GLN) is a GS1 standard for uniquely identifying physical locations, legal entities, and organizational functions. It is widely used in healthcare supply chain worldwide.
- **OID:** 1.3.88 (endorsed by HTA); also 2.51.1.3 (used in some jurisdictions like Switzerland).
- **URI:** https://www.gs1.org/gln
- **Prior art:** GS1 GTIN (https://www.gs1.org/gtin) was already in the FHIR identifier registry at the time of filing.
- **HTA process:** Lloyd McKenzie directed the reporter to submit through HTA. A linked ticket HTA-40 was created.
- **Current state:** The GLN NamingSystem now exists in THO (https://terminology.hl7.org/NamingSystem-GLN.html). Jessica Bota confirmed in Aug 2022 that the migration of identifier registry entries to THO is complete and coordination with FHIR-I is underway to remove the table from the FHIR spec and point users to THO.

## Proposed Dispositions

### Disposition A: Close as Already Applied / Duplicate (Recommended)

**Resolution: Applied**

The requested change—adding GLN to the set of known identifier systems—has effectively been accomplished. The GLN NamingSystem now exists in the HL7 Terminology (THO) at https://terminology.hl7.org/NamingSystem-GLN.html. Furthermore, the identifier registry itself is migrating from the FHIR Core spec to THO, making this the authoritative location. The HTA process was followed via the linked HTA-40 ticket, and the result was approval of the GLN NamingSystem. This ticket can be closed as the request has been fulfilled through the appropriate HTA channel.

**Suggested wording:** "The GLN identifier system has been registered in THO (https://terminology.hl7.org/NamingSystem-GLN.html) through the HTA process (HTA-40). The FHIR identifier registry is migrating to THO as the single source of truth. No further FHIR Core spec change is needed."

### Disposition B: Refer to Terminology Authority (HTA)

**Resolution: Referred**

Since the comment from Lloyd McKenzie explicitly states "HTA owns requests for identifier registrations," this ticket could be formally referred to HTA. The work may already be complete under HTA-40, but referring ensures HTA confirms the registration is fully resolved and any remaining details (e.g., SSCC, OID alignment) are addressed through the proper process.

**Suggested wording:** "Referred to HTA. Identifier registration requests should be submitted through the HTA process. See HTA-40 for the related GLN registration request."

### Disposition C: Decline — Not in Scope for FHIR Core

**Resolution: Not Persuasive**

The identifier registry is managed by HTA, not FHIR Core. This ticket was filed against the wrong specification/work group. The appropriate process is to file an HTA ticket (which was done as HTA-40). Since GLN is already in THO and the identifier registry is moving out of FHIR Core, no FHIR Core change is warranted.

**Suggested wording:** "Not persuasive for FHIR Core. Identifier registration is managed by HTA. The reporter correctly filed HTA-40, and GLN has been added to THO."

---

## Recommendation

**Disposition A: Close as Already Applied** is recommended. The GLN NamingSystem is now live in THO, the HTA process was followed, and the broader migration of the identifier registry to THO means no FHIR Core spec change is needed. This ticket can be closed with acknowledgement that the request was fulfilled through the proper channel.
