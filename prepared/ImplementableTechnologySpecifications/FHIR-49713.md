## Description

In FHIR-37937, the ITS work group decided to shorten RDF property names that include the type of the target resource, by putting the type in an explicit RDF assertion (i.e., adding a "type arc"). The prose on the `rdf.html` page was updated to reflect this decision, but the yellow Turtle template/schema on that same page was not updated. This ticket corrects that omission by adding the missing `rdf:type` arcs to the Turtle template.

For example, polymorphic properties like `effective[x]` should use a shortened property name (`fhir:effective`) combined with an explicit type assertion (`a fhir:dateTime`, `a fhir:Period`, etc.) instead of embedding the type in the property name (`fhir:effectiveDateTime`, `fhir:effectivePeriod`, etc.).

### PR Diff Summary

The PR corrects the `effective[x]` entries in the yellow Turtle template on the `rdf.html` page:

**Before (type embedded in property name):**
```turtle
fhir:effectiveDateTime [ fhir:v "<dateTime>" ];
fhir:effectivePeriod [ Period ];
fhir:effectiveTiming [ Timing ];
fhir:effectiveInstant [ fhir:v "<instant>" ];
```

**After (shortened property + explicit type arc):**
```turtle
fhir:effective [ a fhir:dateTime ; fhir:v "<dateTime>" ];
fhir:effective [ a fhir:Period ; Period ];
fhir:effective [ a fhir:Timing ; Timing ];
fhir:effective [ a fhir:instant ; fhir:v "<instant>" ];
```

## Proposed Dispositions

### Disposition A: Accept as Persuasive (Technical Correction Applied)
**Action:** Accept the resolution as Persuasive and mark as Applied.
**Rationale:** This is a straightforward technical correction. The yellow Turtle template on the `rdf.html` page was inconsistent with both the prose on the same page and the decision made in FHIR-37937 to use shortened property names with explicit `rdf:type` arcs. The fix (PR #3528) has been reviewed and merged, correcting the template to match the agreed-upon RDF serialization format. The change is minimal (4 lines), non-controversial, and purely editorial in nature — it aligns the example with existing normative decisions.

### Disposition B: Accept as Persuasive with Modification — Verify All Templates
**Action:** Accept as Persuasive but request a broader audit of all yellow Turtle templates across the specification to ensure no other templates still use the old embedded-type-in-property-name pattern.
**Rationale:** While the specific fix in this ticket is correct and has been applied, the fact that this template was missed during the FHIR-37937 implementation suggests other templates may have been missed as well. FHIR-53178 (item #6) covers type assertions more broadly, but an explicit audit step may reduce future errata. This disposition adds verification scope beyond the original ask.

### Disposition C: Not Persuasive — Superseded by FHIR-53178
**Action:** Close as Not Persuasive / Duplicate, deferring to FHIR-53178.
**Rationale:** FHIR-53178 is a comprehensive tracker for all R5-to-R6 RDF/Turtle serialization changes, including type assertions for polymorphic properties (item #6). The work in this ticket could be considered subsumed by FHIR-53178. However, this would ignore that the PR has already been merged and the fix is complete.
