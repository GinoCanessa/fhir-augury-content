## Description

The description for the `xml#narrative` canonicalization on the XML page is truncated. Additionally, it is unclear that this canonicalization has a similar definition to the corresponding JSON narrative canonicalization. The reporter (John Moehrke) included a screenshot showing the truncated text in the canonicalization table.

A related ticket, [FHIR-48937](https://jira.hl7.org/browse/FHIR-48937), identifies a specific typo in the same area—a missing close bracket in the `http://hl7.org/fhir/canonicalization/xml#narrative` description: *"The method only retains the Resource.id and Narrative (Resource.text"* (missing closing parenthesis).

## Proposed Dispositions

### Disposition A: Persuasive

**Action:** Fix the truncated `xml#narrative` canonicalization description on the XML page. Complete the missing text and add a note clarifying that this canonicalization has the same scope as `json#narrative`—both retain only `Resource.id` and `Resource.text` (the narrative).

**Rationale:** The truncation is a clear editorial defect. The lack of clarity about the XML/JSON parallel adds confusion for implementers trying to understand which canonicalization method to use. This is straightforward to fix.

### Disposition B: Persuasive with Modification — Verify and Close as Already Addressed

**Action:** Verify whether the updated signature page (which now defines XML and JSON canonicals together) has already resolved the truncation and clarity issues. If confirmed, close this ticket as addressed by the broader signature page reorganization. If not fully addressed, apply the remaining fix. Also coordinate with [FHIR-48937](https://jira.hl7.org/browse/FHIR-48937) (missing close bracket on the same page) to ensure both issues are resolved together.

**Rationale:** Eric Haas (2025-10-13 Zulip) indicated the updated signature page already defines XML and JSON canonicals together, potentially making this ticket moot. However, the ticket should not be closed without verifying the fix is in place. Coordinating with FHIR-48937 avoids duplicate editorial work on the same page section.

### Disposition C: Not Persuasive — Withdraw

**Action:** Withdraw this ticket on the basis that the signature page has been updated to define XML and JSON canonicals together, resolving both the truncation and the clarity concern.

**Rationale:** Per Eric Haas's suggestion on Zulip, the updated signature page already addresses both issues raised in this ticket. Several related tickets (FHIR-51133, FHIR-41522) have already been retracted as the broader signature guidance work has progressed. If the current build confirms the fix, no further action is needed.

## Recommendation

**Disposition B: Persuasive with Modification** — The ticket raises a valid concern (truncated text and unclear XML/JSON parallel), but the broader signature page reorganization work may have already addressed it. The workgroup should verify the current build of the XML canonicalization page to confirm the description is complete and clear. If already fixed, close as addressed. If not, apply the fix and coordinate with FHIR-48937. This avoids both premature closure (Disposition C) and redundant work (Disposition A) if the fix is already in place.
