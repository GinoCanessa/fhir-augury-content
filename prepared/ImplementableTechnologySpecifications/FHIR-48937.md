## Summary

The ticket reports a missing closing parenthesis in the XML canonicalization page. On the page referenced by `http://hl7.org/fhir/canonicalization/xml#narrative`, the description for the `#narrative` canonicalization method reads:

> *"The method only retains the Resource.id and Narrative (Resource.text"*

The closing `)` after `Resource.text` is missing. The ticket title itself contains a typo — "Type" instead of "Typo" — but the intent is clear.

## Analysis

This is a straightforward editorial fix: a missing closing parenthesis in the description of the `#narrative` XML canonicalization method. The text in the published specification table is truncated.

In the current source (`source/signatures.html`), the `#narrative` row has been reworded to: *"Recursively omits all elements other than Resource.id and elements with type Narrative. I.e. the only content retained is resource identity and narrative."* This reworded version no longer has the parenthetical phrasing, which may indicate partial remediation.

**FHIR-46788 overlap:** FHIR-46788 ("canonical XML for narrative explanation truncated") reported by John Moehrke appears to describe the same or very closely related issue. The workgroup should consider whether these are duplicates.

**Batch resolution context:** Per Zulip discussion, Grahame Grieve has proposed resolving all canonicalization-related tickets as a block, which would include this ticket.

---

## Proposed Dispositions

### Disposition A: Apply Technical Correction (Fix the Typo)
- **Action:** Correct the missing closing parenthesis in the published XML canonicalization table for the `#narrative` method. Verify the rendered output matches the corrected source text.
- **Rationale:** This is a simple, unambiguous editorial fix. The current text is grammatically incomplete and could confuse implementers reading the canonicalization specification.
- **Risk:** None. Pure editorial correction with no semantic change.

### Disposition B: Merge with FHIR-46788 (Duplicate)
- **Action:** Close FHIR-48937 as a duplicate of FHIR-46788, which reports the same truncation issue for the XML #narrative canonicalization description. Apply the fix under FHIR-46788.
- **Rationale:** Both tickets report truncated/incomplete text in the XML #narrative canonicalization description. Consolidating avoids duplicate work and ensures a single, consistent fix.
- **Risk:** Minimal. The fix still gets applied, just tracked under one ticket.

### Disposition C: Defer to Batch Canonicalization Rework
- **Action:** Defer this ticket to be resolved as part of the proposed batch canonicalization rework (per Grahame Grieve's Zulip proposal to handle all canonicalization tickets together via a single collaborative PR).
- **Rationale:** The canonicalization pages are undergoing significant rework. Fixing this in isolation may be overtaken by broader changes.
- **Risk:** The typo remains in the published spec until the batch rework is completed, which has no firm timeline.

---

## Recommendation

**Disposition B: Merge with FHIR-46788** — FHIR-46788 already captures this same truncation issue with higher priority (Highest vs Medium) and broader scope (noting the description is also unclear relative to the JSON equivalent). Closing FHIR-48937 as a duplicate of FHIR-46788 avoids redundant tracking. The actual fix should be applied under FHIR-46788, ideally as part of the proposed batch canonicalization rework. If the batch rework is delayed, the fix is trivial enough to apply independently under FHIR-46788.
