## Details

This change request asks that several references in the BreastAbnormality profile (within the US Breast Cancer Radiology Reporting IG) be refined to use base FHIR resource types instead of custom "obf-" prefixed profiles:

1. **`BreastAbnormality.partOf`** — Should reference the base `MedicationStatement` and `Procedure` resources rather than `obf-MedicationStatement` and `obf-Procedure`. The reporter (Mark Kramer) offered to demonstrate how to use the "No profile" flag to achieve this.
2. **`BreastAbnormality.subject`** — Should be constrained to `Patient` only (removing `Group`, `Location`, etc., which are not meaningful in this clinical context).
3. **`BreastAbnormality.performer`** — Should be constrained to `Practitioner` only (removing `Group`, `Location`, etc.).

The resolution was **Persuasive**, with the resolution description stating: *"We agree. In the next ballot we will be revisiting each of those issues."*

This ticket is part of a broader set of ballot feedback on the September 2019 Breast Cancer Radiology Reporting ballot, addressing profile quality, reference specificity, and overall IG organization.

## Proposed Dispositions

### Disposition A: Accept (Apply as Persuasive)

**Action:** Accept the existing Persuasive resolution and ensure the three reference refinements are applied in the next ballot version of the Breast Cancer Radiology Reporting IG:

1. Change `BreastAbnormality.partOf` to reference base `MedicationStatement` and `Procedure` (without `obf-` profiles), using the "No profile" flag.
2. Constrain `BreastAbnormality.subject` to `Patient` only.
3. Constrain `BreastAbnormality.performer` to `Practitioner` only.

**Rationale:** The reporter's feedback is technically sound — using base resource types for `partOf` avoids unnecessary coupling to custom profiles, and constraining `subject` and `performer` to clinically appropriate actor types improves profile specificity and implementer clarity. The IG authors already agreed with this assessment.

### Disposition B: Accept with Broader Performer Constraint

**Action:** Apply the Persuasive resolution but widen the `performer` constraint to include both `Practitioner` and `PractitionerRole` (rather than `Practitioner` alone), while still applying all other changes as requested.

**Rationale:** Modern FHIR practice increasingly favors `PractitionerRole` references to capture the practitioner's organizational context and specialty. Constraining to `Practitioner` alone may be too narrow for real-world radiology workflows where the performing role (e.g., radiologist at a specific facility) matters. This aligns with US Core trends. The `subject` and `partOf` changes would be applied as originally requested.

### Disposition C: Decline (Not Persuasive)

**Action:** Decline the change request and leave references unconstrained.

**Rationale:** The Breast Cancer Radiology Reporting IG has undergone significant restructuring since this 2019 ballot. The `obf-` (Objective FHIR) framework that generated these profiles has been superseded, and the reference constraints may already be addressed in newer versions. Applying this specific change to a legacy ballot version may not be productive. However, this disposition is weakened by the fact that the IG authors already agreed the changes were needed.

## Recommendation

**Disposition A: Accept (Apply as Persuasive)**

The original resolution is appropriate. The three reference refinements are straightforward, technically correct, and already agreed upon by the IG authors. The changes improve profile specificity by removing irrelevant actor types from `subject` and `performer`, and by avoiding unnecessary coupling to custom `obf-` profiles in `partOf`. Given that multiple related tickets (FHIR-23890, FHIR-23891) raised overlapping concerns about reference constraints and profile quality in this IG, accepting this ticket reinforces a consistent pattern of tightening the specification for implementer clarity.
