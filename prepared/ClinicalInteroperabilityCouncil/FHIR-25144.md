## Description

The commenter (Susan Matney) reports that the elements `summarybiradcode` and `breastexamtechnique` are incorrectly displayed as backbone elements in the specification. They should instead be displayed as extensions. The commenter requests correction.

## Resolution Description

> We agree. We believe this is a tooling error. This will be corrected in the next ballot.

The resolution acknowledges the issue as a tooling error where the IG publisher or profiling tool incorrectly rendered extensions as backbone elements. The work group agreed this is persuasive and committed to fixing it in a subsequent ballot cycle.

## Analysis

This ticket is part of a large batch of ballot comments (BC #83 through #107) filed against the US Breast Cancer Radiology Reporting IG during the September 2019 ballot cycle. The specific issue is a rendering/tooling problem: `summarybiradcode` and `breastexamtechnique` are extensions but were displayed as backbone elements in the published IG. The work group agreed this was a tooling error and marked it persuasive, indicating the fix would be applied in the next ballot.

The ticket has been in "Triaged" status since 2021, while several sibling tickets from the same ballot (e.g., FHIR-25147, FHIR-25146, FHIR-25143) have moved to "Resolved - change required." This suggests FHIR-25144 may have been overlooked during batch disposition processing, or its status was not updated after the tooling fix was applied.

**Action:** Confirm that the tooling error has been corrected in the current published version of the IG (i.e., `summarybiradcode` and `breastexamtechnique` are now correctly rendered as extensions, not backbone elements). If confirmed, mark the ticket as "Resolved - change required" and close it.
**Rationale:** The work group already agreed this was persuasive and committed to fixing it. Given that the IG has undergone subsequent ballot cycles (including STU2 ballot in 2021), the fix has likely been applied. This disposition simply completes the administrative closure.
