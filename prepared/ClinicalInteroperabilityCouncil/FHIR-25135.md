## Description

The commenter identified that a hyperlink for "Clinical Information Modeling and Profiling Language (CIMPL)" in the US Breast Cancer Radiology Reporting IG points to a GitHub URL that has been moved. The reporter requests that the underlying hyperlink be updated with a valid/current value, suggesting `http://standardhealthrecord.org/cimpl-doc/` as a potential replacement.

## Resolution Description

> We agree. In the next cycle we will make sure that all links to existing tooling will be valid.

The resolution was marked **Persuasive**, meaning the work group agreed this is a valid issue and committed to fixing all broken tooling links in the next publication cycle.

## Analysis

This is a straightforward editorial/hyperlink maintenance issue from the 2019 ballot cycle (BC #86). The CIMPL documentation URL referenced in the IG had moved, and the reporter correctly identified it needed updating. The work group agreed in their resolution.

**Key considerations:**
1. **Age**: This ticket was opened in November 2019 and resolved as Persuasive, but it remains in "Triaged" status — it has not been applied. It is now over 5 years old.
2. **CIMPL obsolescence**: The CIMPL tooling (Clinical Information Modeling and Profiling Language) was part of the Standard Health Record (SHR) project. The IG has since been restructured and rebuilt with FSH/SUSHI rather than CIMPL, which may make the original link reference moot in the current version.
3. **Pattern of similar issues**: The same reporter (Mitra Rocca) filed multiple similar broken-link and missing-reference tickets in the same ballot (FHIR-25132, FHIR-25136), suggesting systemic link quality issues in the 2019 ballot version that may have been addressed in subsequent rebuilds.
4. **Current IG state**: The Breast Cancer Radiology Reporting IG has had additional ballot cycles since 2019. If the IG no longer references CIMPL tooling, this issue may already be resolved by the IG restructuring.

## Proposed Dispositions

### Disposition A: Apply — Update or Remove the CIMPL Link
**Action:** Verify whether the current version of the Breast Cancer Radiology Reporting IG still references CIMPL. If so, update the hyperlink to a valid URL or remove the reference if CIMPL is no longer relevant. Mark as Applied.
**Rationale:** Honors the Persuasive resolution. Broken links degrade IG quality and confuse implementers. Even if the IG has been restructured, confirming the fix and closing the ticket maintains good tracker hygiene.

### Disposition B: Resolve as Already Applied — Confirm Fixed by IG Restructuring
**Action:** Verify that the current published version of the IG no longer contains the broken CIMPL link (due to the IG being rebuilt with FSH/SUSHI). If confirmed, mark as Applied with a note that the restructuring resolved the issue.
**Rationale:** The IG has undergone significant changes since 2019. If the CIMPL reference was removed during the tooling migration, the issue is already resolved and the ticket should be closed to reflect that.

### Disposition C: Resolve as No Change — CIMPL Reference Removed
**Action:** If the IG no longer references CIMPL at all and the original context is gone, resolve as No Change with a note that the content the ticket referred to has been removed in subsequent versions.
**Rationale:** The ballot comment targeted content that may no longer exist. The Persuasive resolution committed to fixing links "in the next cycle," and if that content was removed entirely, a No Change disposition with explanation is appropriate.

## Recommendation

**Disposition B: Resolve as Already Applied — Confirm Fixed by IG Restructuring.**

Given the age of this ticket (5+ years), the known migration of the Breast Cancer Radiology IG from CIMPL to FSH/SUSHI tooling, and the pattern of similar link-quality issues from the same ballot, the most likely scenario is that the restructured IG no longer contains the broken CIMPL link. The work group should verify this against the current published version, and if confirmed, close the ticket as Applied with a note documenting that the IG rebuild resolved the issue. This honors the original Persuasive resolution while reflecting the actual state of the specification.
