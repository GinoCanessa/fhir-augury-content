## Description

The submitter notes that in the US Breast Cancer Radiology Reporting IG, the Patient history element references **only** the Observation resource. This is considered too limiting. Patient history should be able to reference other resource types such as DiagnosticReport, H&P (History & Physical) documents, and past pathology documents, among others.

## Resolution Description

> "We agree, and need to discuss this."

The work group accepted the comment as **Persuasive**, acknowledging that the reference constraint is too narrow. The ticket is labeled "Discuss-at-WGM," indicating further discussion is needed before implementation.

## Analysis

This ticket is part of a cluster of ballot comments (BC series) from the 2019 ballot cycle for the US Breast Cancer Radiology Reporting IG. The core issue is a modeling constraint: the Patient history element in the IG only allows references to Observation resources, but clinical practice requires referencing a broader set of resources (DiagnosticReport, DocumentReference for H&P and pathology documents, Condition, etc.).

The work group agreed the constraint is too narrow (resolution: Persuasive) but flagged it for WGM discussion, likely because broadening the reference type has design implications that intersect with related structural decisions (see FHIR-25061 on DiagnosticReport as anchor, FHIR-25140 on adding sections). The ticket has remained in "Triaged" status since 2021 without applied changes.

## Proposed Dispositions

### Disposition A: Apply as Persuasive — Broaden Reference Types
Widen the Patient history reference to support `Reference(Observation | DiagnosticReport | DocumentReference | Condition)` (or similar expanded set). This directly implements the submitter's request and aligns with the accepted "Persuasive" resolution. The implementation should coordinate with the structural decisions from FHIR-25061 and FHIR-25140 to ensure consistency.

### Disposition B: Apply with Modification — Use Reference(Any) with Must Support Guidance
Instead of enumerating specific resource types, broaden the Patient history reference to `Reference(Resource)` with Must Support guidance on the most common types (Observation, DiagnosticReport, DocumentReference). This provides maximum flexibility for future use cases but requires clear implementation guidance to avoid ambiguity.

### Disposition C: Defer — Pending IG Structural Redesign
Defer this change until the broader structural questions raised in FHIR-25061, FHIR-25080, and FHIR-25140 are resolved. The rationale is that the choice of referenceable types depends on whether DiagnosticReport or Composition anchors the report, and which sections exist. However, this risks further delaying resolution of a ticket that has been open since 2019.

## Recommendation

**Disposition A: Apply as Persuasive — Broaden Reference Types.** The work group already agreed the change is persuasive, and the request is clinically well-justified. An explicit, bounded set of reference types (Observation, DiagnosticReport, DocumentReference, Condition) provides clarity for implementers while addressing the submitter's concern. Coordination with related structural tickets should be noted but should not block this discrete improvement.
