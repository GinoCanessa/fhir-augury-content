## Description

The submitter requests multiple structural additions and modeling considerations for the US Breast Cancer Radiology Reporting IG. Specifically:

1. **History section** — Add a section for patient history (symptoms, hormones, family history).
2. **Clinical signs in Observations** — Include clinical signs such as scar, implant, birthmark, etc., which are not currently in the definitions.
3. **Double-reading** — Consider whether the model supports double-reading workflows.
4. **Screening vs. diagnostic reporting** — Consider distinct structures for screening versus diagnostic reports.
5. **Intervention section** — Add a section for interventions (e.g., biopsy).
6. **Dose reporting** — Consider scope of dose reporting; FHIR does not currently provide a resource for this.
7. **Modeling approach** — ACR BI-RADS should be referenced (all or relevant sections), and DICOM Breast Imaging Report Templates (established ~15 years) do not appear to be considered.

## Analysis

This is a broad, multi-part change request from the original STU1 ballot (2019) that raises several important structural and modeling concerns. Key considerations:

1. **Partial overlap with resolved work**: Item #4 (screening vs. diagnostic) has already been addressed by FHIR-31303 (Applied, Persuasive). Item #7's BI-RADS reference concern has been partially addressed by FHIR-31298 (making BI-RADS assessment codes extensible).

2. **Duplicate ticket**: FHIR-25140 is a near-identical duplicate filed one minute later by the same reporter. Both remain triaged and unresolved. Any disposition should consider consolidating these.

3. **Remaining unaddressed items**: History section, clinical signs in observations, double-reading workflow support, intervention section, dose reporting scope, and DICOM Breast Imaging Report Template alignment remain open questions that have not been clearly resolved by subsequent tickets.

4. **Specification maturity**: The IG has evolved through multiple ballot cycles since this was filed in 2019. Some items may have been implicitly addressed through structural changes without explicit tracker resolution.

5. **Scope concern**: This ticket bundles 7+ distinct requests. A workgroup may consider splitting unresolved items into individual tickets for clearer tracking.

## Proposed Dispositions

### Disposition A: Persuasive with Modification — Partially Applied, Remaining Items Deferred
**Rationale:** Acknowledge that items #4 (screening vs. diagnostic, via FHIR-31303) and part of #7 (BI-RADS, via FHIR-31298) have been addressed in subsequent ballot cycles. For the remaining items (history section, clinical signs, double-reading, intervention section, dose reporting, DICOM template alignment), create individual tracker items for future consideration. Close this omnibus ticket as partially addressed. Mark FHIR-25140 as a duplicate.

### Disposition B: Not Persuasive — Scope Too Broad for Single Disposition
**Rationale:** The ticket bundles too many unrelated requests to disposition as a single item. The workgroup could resolve as Not Persuasive on procedural grounds, noting that the submitter should file individual change requests for each distinct concern. Items already addressed (screening/diagnostic, BI-RADS extensibility) can be referenced as evidence that the specification has evolved. The remaining items lack sufficient specificity for actionable implementation.

### Disposition C: Deferred — Pending Next Ballot Cycle
**Rationale:** Defer the unresolved items to a future ballot cycle. The specification has undergone significant evolution since 2019, and some items (dose reporting, DICOM template alignment) require broader FHIR infrastructure changes beyond the scope of this IG alone. The workgroup should revisit when FHIR core provides better support for dose reporting resources and when DICOM-FHIR alignment efforts mature.

## Recommendation

**Disposition A: Persuasive with Modification — Partially Applied, Remaining Items Deferred**

This is the most constructive path forward. It gives proper credit to work already completed (FHIR-31303, FHIR-31298) while acknowledging the valid unresolved concerns. Splitting the remaining items into individual tickets ensures they are not lost and can be prioritized independently. This also allows FHIR-25140 to be formally resolved as a duplicate, cleaning up the tracker. The broad scope of the original comment is better served by targeted, trackable tickets than by leaving a single omnibus item open indefinitely.
