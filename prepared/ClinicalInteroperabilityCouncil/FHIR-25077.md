## Description

The commenter (Claude Nanjo) notes that it is sometimes unclear whether the profiles presented in the US Breast Cancer Radiology Reporting IG are logical model definitions or profiles on actual FHIR resources. As a specific example, `Observation.bodySite` has a number of attributes such as laterality and anatomical location, but in R4, `Observation.bodySite` is a `CodeableConcept`. The recommendation is for the IG to clearly delineate between logical structure definitions and FHIR resource profiles.

## Resolution Statement

> This is an artifact of the CIMPL logical model. For the next ballot we will not be using these logical models and instead adhere more closely to the base FHIR resources.

The resolution acknowledges the confusion stems from the CIMPL (Clinical Information Modeling Profiling Language) logical model approach and commits to moving away from CIMPL logical models toward standard FHIR resource profiles in the next ballot.

## Analysis

This ticket was submitted during the 2019 September ballot of the US Breast Cancer Radiology Reporting IG. The core issue is that the IG used CIMPL (Clinical Information Modeling Profiling Language) to generate its profiles, and this tooling produced StructureDefinitions that blurred the line between logical models and actual FHIR resource profiles. The specific example—`Observation.bodySite` being extended with sub-attributes like laterality and anatomical location when in R4 it is simply a `CodeableConcept`—illustrates how the CIMPL approach created non-conformant structures.

Multiple related ballot comments (FHIR-25078, FHIR-23891, FHIR-25168) from the same ballot cycle raise similar concerns about the IG's reliance on CIMPL abstractions rather than standard FHIR resource patterns. The resolution acknowledges this and commits to abandoning the CIMPL logical models in favor of direct FHIR resource profiling.

This is a well-understood issue with a clear and accepted resolution. The "Persuasive" disposition was appropriate.

## Proposed Dispositions

### Disposition A: Affirm Existing Resolution (Persuasive) — ✅ RECOMMENDED

**Action:** Accept the existing "Persuasive" resolution as-is.

**Rationale:** The resolution correctly identifies the root cause (CIMPL logical model artifacts) and commits to a concrete corrective action (removing CIMPL logical models and adhering more closely to base FHIR resources in the next ballot). This directly addresses the commenter's concern about ambiguity between logical models and FHIR profiles. Multiple related ballot comments received similar dispositions, indicating consensus that the CIMPL approach needed to be replaced.

### Disposition B: Persuasive with Modification — Add Explicit Guidance

**Action:** Accept as "Persuasive with Modification," adding a requirement that the next version of the IG include a clear statement in the introduction distinguishing any remaining logical models from FHIR resource profiles, and include a mapping between them.

**Rationale:** While the resolution commits to moving away from CIMPL, it does not mandate documentation that explains the distinction going forward. If any logical models remain in the IG for conceptual purposes, they should be clearly labeled and mapped to the corresponding FHIR profiles.

### Disposition C: Not Persuasive with Modification — Scope to Documentation Only

**Action:** Change to "Not Persuasive with Modification," acknowledging the concern but treating it as a documentation/labeling issue rather than a structural redesign.

**Rationale:** The IG's content could be preserved as-is if each profile/logical model page clearly indicates whether it represents a logical model or a conformant FHIR profile. This avoids the cost of a full rewrite. However, this is a weaker response given that multiple ballot commenters identified the same fundamental confusion, and simply labeling the problem without fixing the underlying approach may not satisfy implementers.

## Recommendation

**Disposition A: Affirm Existing Resolution (Persuasive)**

The existing resolution is sound. It correctly diagnoses the problem (CIMPL logical model artifacts creating ambiguity) and commits to the appropriate fix (abandoning CIMPL in favor of standard FHIR resource profiling). This was the consensus approach across multiple related ballot items from the same cycle, and affirming it maintains consistency with how the workgroup resolved the broader set of CIMPL-related concerns.
