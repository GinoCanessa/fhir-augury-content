## Summary

The submitter requests that definitions and hyperlinks to specific NCI Thesaurus concepts be added to mCODE value sets. Currently, value set entries referencing NCI Thesaurus codes lack inline definitions and direct hyperlinks to the NCI Thesaurus concept pages. The submitter argues that since the NCI Thesaurus provides definitions, this information should be included to ensure data quality, and that direct hyperlinks (rather than links to the general browser site) would make it easier for end-users to find concepts—consistent with the approach used for SNOMED.

This ticket has a complex voting history. It was initially voted **Persuasive with Modification** (13-0-3, 2024-09-25) with a plan to reorganize staging profiles and add detailed narrative. That resolution was then reverted and re-resolved as **Considered for Future Use** (7-0-0, 2024-10-24) because the IG Publisher relies on terminology/ontology servers for hyperlinks and definitions, and NCI Thesaurus is not currently hosted on such a server. The ticket was subsequently re-opened post STU4.0.0 publication (2025-02-17) and is now back in **Triaged** status.

## Details

### Description

To ensure data quality, definitions for each value should be provided. Since the NCI Thesaurus provides definitions, it is easy to add this information.

Hyperlinks to the specific NCI Thesaurus concepts should be included and not just to the general browser site. This makes it easier for end-users to find the concepts and is consistent with the approach used for other terminologies such as SNOMED.

## Proposed Dispositions

### Disposition A: Considered for Future Use

**Rationale:** The core technical blocker remains: the IG Publisher generates hyperlinks and definitions by querying terminology/ontology servers, and the NCI Thesaurus is not currently hosted on a compatible server. Until NCI Thesaurus is available through an ontology server (e.g., via FHIR-based terminology services like tx.fhir.org), the IG Publisher cannot automatically render these hyperlinks and definitions. This was the prior working group consensus (7-0-0) and the situation has not materially changed since the ticket was re-opened post-publication.

**Action:** Resolve as "Considered for Future Use." Document that the request will be addressed once NCI Thesaurus is available through a FHIR terminology server supported by the IG Publisher. Coordinate with the Terminology Infrastructure work group and NCI to track progress on NCI Thesaurus availability.

### Disposition B: Persuasive with Modification — Manual Workaround

**Rationale:** While the IG Publisher cannot auto-generate NCI Thesaurus hyperlinks, it is technically possible to manually add hyperlinks and definitions to value set narratives or to use CodeSystem supplements. This would require manual maintenance but would address the submitter's concern about data quality and usability. Given that this ticket and its sibling tickets (FHIR-45835, FHIR-45837, FHIR-45805) have been reopened with "Highest" priority, a manual interim solution could be warranted.

**Action:** Resolve as "Persuasive with Modification." Add manually curated hyperlinks to NCI Thesaurus concept pages (e.g., `https://ncit.nci.nih.gov/ncitbrowser/ConceptReport.jsp?dictionary=NCI_Thesaurus&code=CXXXXX`) and include concept definitions in value set narrative text. Apply the same approach across sibling tickets. Document that this manual approach will be replaced by automated IG Publisher support once NCI Thesaurus is available via a terminology server.

### Disposition C: Not Persuasive

**Rationale:** The IG Publisher's approach is to generate hyperlinks and definitions from ontology servers uniformly. Adding manual hyperlinks and definitions for NCI Thesaurus would create an inconsistent maintenance burden and a precedent for manually managing terminology metadata that should be automated. The submitter's request, while valid in intent, asks mCODE authors to work around a tooling/infrastructure gap that is outside their scope.

**Action:** Resolve as "Not Persuasive." Direct the submitter to engage with the Vocabulary work group and NCI to advocate for NCI Thesaurus availability on a FHIR terminology server. Note that the mCODE IG will benefit automatically once infrastructure support exists.

## Recommendation

**Disposition A: Considered for Future Use** is recommended.

The fundamental technical constraint — NCI Thesaurus not being available through an ontology server compatible with the IG Publisher — has not changed since the previous 7-0-0 vote. Re-opening the ticket post-publication does not alter this infrastructure limitation. Manually adding hyperlinks (Disposition B) would create an ongoing maintenance burden across multiple value sets and multiple sibling tickets, and would set a precedent inconsistent with how other terminologies are handled. The most pragmatic path is to acknowledge the request's validity, track NCI Thesaurus terminology server availability as a dependency, and revisit when the infrastructure gap is resolved. The work group should coordinate with the sibling tickets (FHIR-45835, FHIR-45837, FHIR-45805) to apply the same resolution consistently.
