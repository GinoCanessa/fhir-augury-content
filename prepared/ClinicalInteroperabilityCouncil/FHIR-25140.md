## 2. Description & Request

The submitter requests that additional sections be added to the US Breast Cancer Radiology Reporting IG. The ticket enumerates several structural gaps:

1. **History section** – e.g., symptoms, hormones, family history
2. **Clinical signs in Observations** – e.g., scar, implant, birthmark (not currently in definitions)
3. **Double-reading support** – does the model accommodate double-reading workflows?
4. **Screening vs. diagnostic reporting** – consider distinct structure for each
5. **Intervention section** – e.g., biopsy
6. **Dose reporting** – FHIR does not currently provide a resource for this
7. **Modeling approach:**
   - ACR BI-RADS should be referenced (all or relevant sections)
   - DICOM Breast Imaging Report Templates (established ~15 years) do not appear to have been considered

The resolution description field states: *"DiagnosticReport appears sufficient."*

## 7. Analysis

### Context
This ticket is one of several ballot comments (the "BC #90" series) submitted during the initial ballot cycle for the US Breast Cancer Radiology Reporting IG. It raises broad structural feedback spanning multiple areas of the IG.

### Key Observations

1. **Duplicate ticket exists**: FHIR-25139 contains identical content from the same reporter, created one minute prior. One of these should be closed as duplicate.

2. **Screening vs. diagnostic already addressed**: FHIR-31303 (Applied, Persuasive) specifically tackled the differentiation of screening versus diagnostic mammography findings, which resolves bullet point #4 of this ticket.

3. **BI-RADS extensibility partially addressed**: FHIR-31298 (Applied) made BI-RADS assessment category code definitions extensible, which touches on the modeling approach concern (bullet point #7a).

4. **Report structure questions resolved elsewhere**: FHIR-25064 (Persuasive) and FHIR-23885 (Persuasive with Modification) addressed the overall Composition/DiagnosticReport structure, including the hybrid approach and the missing history section in the overview figure.

5. **Remaining open items**: Dose reporting (no FHIR resource exists), explicit DICOM Breast Imaging Report Template alignment, double-reading workflow support, clinical signs in observation definitions, and an intervention section have not been explicitly addressed by other resolved tickets.

6. **Resolution description hint**: The terse note *"DiagnosticReport appears sufficient"* suggests the WG may have informally concluded the existing resource structure is adequate without requiring additional sections, but no formal disposition has been applied.

7. **Age of ticket**: This ticket has been in Triaged status since 2019 with no activity since March 2021, suggesting it may have been overtaken by subsequent ballot cycles and applied changes.

## 8. Proposed Dispositions

### Option A: Not Persuasive
**Rationale:** Several of the requested changes have been addressed by other tickets (FHIR-31303 for screening vs. diagnostic; FHIR-31298 for BI-RADS extensibility; FHIR-25064/FHIR-23885 for report structure). The resolution description already indicates "DiagnosticReport appears sufficient." The remaining suggestions (dose reporting, double-reading, DICOM template alignment) are either out-of-scope for this IG (dose reporting requires a new FHIR resource) or represent future enhancements beyond the current specification's scope. Close FHIR-25140 as Not Persuasive and note the related tickets that addressed individual items.

### Option B: Persuasive with Modification
**Rationale:** Acknowledge that while several individual points have been addressed in other tickets, the ticket raises valid concerns about DICOM Breast Imaging Report Template consideration and clinical signs coverage that could improve the IG. Apply a modification limited to: (1) adding a reference or conformance note regarding DICOM Breast Imaging Report Templates in the IG's modeling approach section, and (2) expanding Observation definitions to include common clinical signs (scar, implant, birthmark). Close the remaining bullet points as either already addressed or out-of-scope.

### Option C: Duplicate
**Rationale:** FHIR-25139 contains identical content from the same reporter and was created one minute before FHIR-25140. Close FHIR-25140 as a duplicate of FHIR-25139, consolidating discussion on the earlier ticket.

## 9. Recommendation

**→ Option C: Duplicate** of FHIR-25139.

FHIR-25140 is identical in content, reporter, and title to FHIR-25139, which was created one minute earlier. The clearest path is to close this ticket as a duplicate and consolidate any remaining disposition work on FHIR-25139. If the work group prefers to dispose of the substance simultaneously, the analysis above (particularly Option A's reasoning that most items are already addressed) can be applied to whichever ticket is retained.
