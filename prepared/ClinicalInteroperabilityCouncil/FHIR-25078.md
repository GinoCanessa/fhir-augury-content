## Description

The commenter notes that the `breastrad-recommendations` section has an entry of `obf-ActionRequest`, which is not a core FHIR resource. It is unclear whether this is intended to be a logical model or a FHIR core resource. The commenter recommends using **PlanDefinition**, which is defined for this purpose and is already part of FHIR Core.

## Resolution Statement

> This is an artifact of the CIMPL logical model. For the next ballot we will not be using ActionRequest and we currently intend to use ServiceRequest and/or Appointment.

The resolution acknowledges that `ActionRequest` was an artifact of the CIMPL (Clinical Information Modeling Profiling Language) logical model, not a FHIR core resource. The team agreed to move away from it and adopt core FHIR resources (ServiceRequest and/or Appointment) in future ballots.

## Proposed Dispositions

### Disposition A: Confirm as Persuasive — Applied (Recommended)

**Action:** Confirm the existing "Persuasive" resolution and mark as Applied.

**Rationale:** The commenter's concern was valid — `obf-ActionRequest` was a CIMPL logical model artifact, not a FHIR core resource, and should not have been used in the recommendations section. The resolution statement commits to replacing ActionRequest with core FHIR resources (ServiceRequest and/or Appointment). This aligns with the commenter's intent to use established FHIR Core resources. Multiple related tickets (FHIR-23885, FHIR-23891, FHIR-25077) raised overlapping concerns about the use of non-core resources and confusion between logical models and FHIR profiles, all of which are addressed by this direction. The IG has since been updated in subsequent ballot cycles.

**Wording:** "Persuasive. ActionRequest was an artifact of the CIMPL logical model and has been replaced with core FHIR resources (ServiceRequest and/or Appointment) in subsequent versions of the IG."

### Disposition B: Persuasive with Modification — Note PlanDefinition Was Not Adopted

**Action:** Change resolution to "Persuasive with Modification" to note that while the concern was accepted, PlanDefinition was not specifically adopted.

**Rationale:** The commenter specifically requested PlanDefinition, but the resolution states the team chose ServiceRequest and/or Appointment instead. While the spirit of the request (use core FHIR resources) was honored, the specific resource suggested was not adopted. A "Persuasive with Modification" disposition would more accurately reflect that the team agreed with the principle but chose a different implementation path.

**Wording:** "Persuasive with Modification. The committee agrees that core FHIR resources should be used instead of ActionRequest. However, ServiceRequest and/or Appointment were selected rather than PlanDefinition, as they better fit the recommendation use case in this IG."

### Disposition C: Not Persuasive — PlanDefinition Not Appropriate for Recommendations

**Action:** Change resolution to "Not Persuasive."

**Rationale:** PlanDefinition is designed for defining protocols, order sets, and clinical decision support rules — not for individual patient-level recommendations in a radiology report. The commenter's suggestion of PlanDefinition may not have been the correct fit. ServiceRequest (for ordering follow-up) or Appointment (for scheduling) are more appropriate resources for patient-specific clinical recommendations arising from a radiology report. However, this disposition would disregard the valid core concern that ActionRequest should not have been used.

**Wording:** "Not Persuasive. While ActionRequest has been removed, PlanDefinition is not the appropriate replacement for patient-level recommendations in radiology reports. ServiceRequest and Appointment better serve this purpose."

## Recommendation

**Disposition A: Confirm as Persuasive — Applied** is recommended.

The commenter's core concern — that a non-FHIR-core resource (ActionRequest) was being used where a standard FHIR resource should be — was valid and accepted. The resolution commits to using core FHIR resources going forward. While the specific resource suggested (PlanDefinition) was not adopted, the committee's choice of ServiceRequest/Appointment is a reasonable alternative that satisfies the intent of the request. The existing "Persuasive" resolution accurately captures the outcome, and the change has been addressed in subsequent ballot cycles of the IG.
