## Description

The commenter identifies that the heading on the extensions page of the US Breast Cancer Radiology Reporting IG reads **"Primary extensions defined as part of this Implementation Guide"**, but the page actually lists extensions that originate from other IGs (e.g., `LandmarkLocation`, `LandmarkToBodyLocationDirection`, `Laterality` from the mCode IG). The heading is therefore misleading.

**Suggested fix (two options):**
1. Change the heading to something like **"Extensions used in this Guide"** and add a column indicating the source/basis for each extension.
2. Only include extensions that were actually created within this IG.

## Resolution Description

> "We agree, and will make these changes in the next ballot."

The resolution is **Persuasive** — the work group accepted the comment and committed to correcting the heading/content discrepancy in a future ballot cycle.

## Proposed Dispositions

### Disposition A: Accept as Persuasive — Rename Heading (Recommended)

**Action:** Accept the existing "Persuasive" resolution. Rename the heading from "Primary extensions defined as part of this Implementation Guide" to **"Extensions Used in This Guide"** and add a "Source" column to the extensions table indicating the originating IG for each extension (e.g., mCode, this IG).

**Rationale:** This approach provides the most informative result for implementers. It preserves the complete list of extensions used in the IG while adding transparency about where each extension originates. The work group already agreed to this direction in their resolution. This is the least disruptive change and adds navigational value.

### Disposition B: Accept as Persuasive — Filter to IG-Only Extensions

**Action:** Accept the "Persuasive" resolution by restricting the extensions page to only those extensions that are actually defined within this IG, removing extensions imported from mCode or other IGs.

**Rationale:** This makes the heading accurate without changing it. However, it reduces discoverability — implementers would need to look elsewhere to find the full set of extensions used in profiles. This is a valid but less informative option.

### Disposition C: Defer

**Action:** Defer the change pending the next ballot cycle, noting that the IG has not been actively balloted recently (last update 2021).

**Rationale:** The IG appears to have limited recent activity. The fix is low complexity but deferred until the next active development cycle when the full extensions page can be reviewed holistically alongside other open tickets like FHIR-38776 (Make Examples) and FHIR-31327 (Device attributes).

## Recommendation

**Disposition A: Accept as Persuasive — Rename Heading.** The work group has already agreed this is persuasive and committed to making the change. Renaming the heading to "Extensions Used in This Guide" and adding a source column is the most transparent, implementer-friendly approach. It aligns the page content with its heading while preserving all useful information.
