## Summary

The submitter identifies a clarity issue in the Observation profiles within the US Breast Cancer Radiology Reporting IG. Several component slices have plural names (e.g., `cystTypes`, `LymphNodeQualifiers`) but are constrained to a cardinality of 0..1, creating confusion about whether these components can repeat. The ticket asks three key questions:

1. **Can components repeat?** If so, how should they be structured for a given patient and laterality?
2. **Can there be multiple observations per side?** Or is a patient limited to two observations total (one per breast)?
3. **How are observations organized?** The IG's home page image suggests multiple observations per side, but this is not explicitly documented.

The resolution (Persuasive) acknowledges the issue: each breast can have many observations of different abnormalities of varying types, and the impressions section can document multiple conditions. The committee agreed to clarify this in the next ballot.

---

## Details

### Reporter's Concern

The core issue is a mismatch between plural-named component slices and their 0..1 cardinality constraints. Implementers cannot determine from the current profiles whether:

- A single Observation resource can carry multiple values for a given component (e.g., multiple cyst types)
- Multiple Observation resources should be created for each abnormality finding per breast
- The relationship between laterality (left/right) and observation multiplicity is constrained

### Resolution Description

> We agree with this item. Each breast can have many observations of different abnormalities of varying types, and the impressions section can document multiple conditions. For the next ballot we will clarify these better.

## Proposed Dispositions

### Disposition A: Accept (Persuasive — as resolved)

**Action:** Accept the existing Persuasive resolution. The IG editors should add clarifying text and examples to the Observation profiles explaining:
- Each breast may have multiple Observation resources for distinct abnormality findings.
- Component slices with plural names but 0..1 cardinality represent a single coded value per component per Observation instance; multiplicity is achieved through multiple Observation resources, not repeating components.
- A narrative or diagram should document the relationship between Patient → laterality → Observation(s) → component structure.

**Rationale:** The resolution description already acknowledges the problem and commits to clarification. This disposition simply affirms that commitment and specifies what the clarification should include.

### Disposition B: Accept with Modification — Adjust Cardinality

**Action:** Accept as Persuasive with Substantive Change. In addition to the clarifying text from Disposition A, review each plural-named component slice and either:
- Change the cardinality from 0..1 to 0..* where the clinical model genuinely supports multiple values (e.g., multiple cyst types for a single observation), or
- Rename the component slice to a singular name if the 0..1 cardinality is intentional, to avoid confusing implementers.

**Rationale:** The original confusion arises from a naming-vs-cardinality mismatch. Clarifying documentation alone may not be sufficient if the cardinality constraints themselves are incorrect. If the clinical model truly allows multiple cyst types per observation, the profile should express that structurally.

### Disposition C: Decline (Not Persuasive)

**Action:** Decline the change request. The existing profiles are technically correct as specified: 0..1 cardinality is the intended constraint, plural names are a stylistic choice, and implementers can create multiple Observation resources to represent multiple findings.

**Rationale:** This approach would argue that the profiles are structurally sound and that naming conventions do not constitute a conformance requirement. However, this ignores the legitimate confusion reported by the implementer and the committee's own agreement that clarification is needed.

---

## Recommendation

**Disposition A: Accept (Persuasive — as resolved)** is recommended.

The committee has already agreed that clarification is needed and committed to providing it. The resolution is sound — the issue is fundamentally about documentation clarity rather than a structural flaw in the profile. While Disposition B raises a valid point about the plural-name/cardinality mismatch, that level of change goes beyond what the submitter requested and could introduce breaking changes. The most pragmatic path is to accept the existing resolution and ensure the next ballot includes clear guidance on how Observation resources and their components are organized per patient and per laterality.
