# Ticket Review: FHIR-44822

**Title:** AU Base Organization: additionalBindings clarification

## Summary

This change request proposes that the AU Base Organisation profile adopt the `additionalBinding` extension (from the FHIR Tools IG) to formally represent value sets that are currently documented only in implementation guidance text. A branch build demonstrating the proposed change to the AU Base Organisation profile is referenced in the ticket description. The request is part of a broader effort across multiple AU Base profiles to replace informal guidance-level value set references with machine-readable additional bindings.

## Details

**Description:**

Now that the [`additionalBinding` extension](https://build.fhir.org/ig/FHIR/fhir-tools-ig/StructureDefinition-additional-binding.html) is available, the value sets currently described in implementation guidance in each of the profiles should be represented using the `additionalBinding` extension. The ticket references a branch build of the [AU Base Organisation profile](https://build.fhir.org/ig/hl7au/au-fhir-base/branches/additionalBindings/StructureDefinition-au-organization.html) that demonstrates the proposed change.

## Proposed Dispositions

### Disposition A: Accept As Requested
#### Proposal
Accept the change request and update the AU Base Organisation profile to use the `additionalBinding` extension to formally express the value sets currently documented in implementation guidance. Apply the changes demonstrated in the referenced branch build.
#### Justification
The `additionalBinding` extension is now a stable, published mechanism from the FHIR Tools IG designed specifically for this purpose. Migrating from prose-based implementation guidance to a machine-readable extension improves tooling support, validation capabilities, and conformance testing. This aligns with parallel efforts across other AU Base profiles (e.g., FHIR-46081 for PractitionerRole, FHIR-46080 for Pathology Result) and ensures consistency across the AU Base IG. A working branch build already demonstrates the feasibility of the change.

---

### Disposition B: Alternative Approach
#### Proposal
Accept the intent of the request but defer implementation until all related AU Base additionalBindings tickets (FHIR-46081, FHIR-44823, FHIR-46080, FHIR-44821) are coordinated and applied together in a single release cycle, ensuring a consistent rollout across all affected profiles.
#### Justification
Applying additionalBindings changes incrementally across individual profiles may create inconsistency within the IG where some profiles use the extension and others still rely on implementation guidance text. A coordinated approach would deliver a uniform authoring pattern across the entire AU Base IG in one release, reducing confusion for implementers and simplifying the review process.

---

### Disposition C: Decline
#### Proposal
Decline the change request on the basis that the current implementation guidance approach is sufficient and the additionalBinding extension adds complexity without clear implementer benefit at this time.
#### Justification
The existing implementation guidance text already communicates the relevant value sets to implementers. The `additionalBinding` extension, while machine-readable, may not yet be widely supported by tooling that Australian implementers rely on. Adding extensions increases profile complexity and may introduce validation issues for implementations that do not process the extension. However, this argument is weakened by the broader community direction toward formal binding representation and the availability of the extension in the FHIR Tools IG.

---

### Recommendation
**Recommended disposition:** A

The `additionalBinding` extension is purpose-built for exactly this use case — formally representing value set bindings that go beyond the base specification's binding mechanism. The extension is published and stable in the FHIR Tools IG, and a working branch build already demonstrates the change applied to the AU Base Organisation profile. Accepting this request brings the profile in line with modern FHIR profiling best practices and improves machine-readability for tooling and validation. The parallel tickets for other AU Base profiles (PractitionerRole, Pathology Result, Medication, Immunization) confirm that this is a deliberate, coordinated direction for the AU Base IG, making it appropriate to proceed with this individual profile change as part of that broader effort.
