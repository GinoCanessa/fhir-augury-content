## Summary

The reporter argues that the wording of AU Base Patient invariants (e.g., `inv-pat-2`) is misleading. The current phrasing — *"Individual pronouns shall be a member of the Australian Pronouns value set if any codes within that value set can apply"* — implies that a value **must** be supplied when one is available, rather than simply constraining the value **if present**. The reporter proposes rewording to *"If present, individual pronouns shall be a member of the Australian Pronouns value set"*, which more accurately reflects the underlying FHIRPath expression (which checks membership only when a value exists). A follow-up comment also notes that the invariant may unintentionally prohibit alternate codings.

The resolution was **Not Persuasive** — co-chairs were satisfied with the current wording.

## Details

- **Current invariant wording (inv-pat-2):** *"Individual pronouns shall be a member of the Australian Pronouns value set if any codes within that value set can apply"*
- **Proposed rewording:** *"If present, individual pronouns shall be a member of the Australian Pronouns value set"*
- The reporter contends the current wording is ambiguous: it could be read as mandating the inclusion of a pronoun code whenever one exists in the value set, rather than simply constraining the code system when a pronoun value is supplied.
- The reporter also flags a potential issue: the invariant may prohibit alternate codings alongside the required Australian Pronouns value set code.
- The same rewording pattern is suggested for other similar AU Base Patient invariants.

## Proposed Dispositions

### Disposition A: Affirm Not Persuasive (Keep Current Wording)

Uphold the existing resolution. The co-chairs reviewed the invariant wording and found it acceptable. The phrase *"if any codes within that value set can apply"* is a standard HL7 conformance pattern that adds nuance beyond a simple "if present" check — it communicates that the binding applies contextually. While the alternative wording is clearer in isolation, the current wording is consistent with established HL7 convention and is well-understood by implementers familiar with AU Base.

### Disposition B: Not Persuasive with Modification (Partial Rewording)

Keep the current resolution category but acknowledge the concern in a non-normative note or implementation guidance. Add clarifying text to the AU Base Patient profile page explaining that the invariant does **not** mandate supplying a pronoun value — it only constrains the code system when a value is present. This addresses the ambiguity concern without changing the normative invariant description. Additionally, investigate whether the invariant inadvertently prohibits alternate codings (as flagged in the reporter's comment).

### Disposition C: Persuasive (Accept Rewording)

Overturn the resolution and accept the proposed rewording. Replace the invariant description with *"If present, individual pronouns shall be a member of the Australian Pronouns value set"* for `inv-pat-2` and apply the same pattern to other similarly worded invariants. This aligns the human-readable description with what the FHIRPath expression actually enforces, reducing confusion for implementers. Note that FHIR-46176 and FHIR-53536 have already reworked the FHIRPath expressions themselves; updating the descriptions would complete the alignment.

## Recommendation

**Disposition A: Affirm Not Persuasive.** The co-chairs already reviewed and accepted the current wording. The underlying FHIRPath expression issues flagged by the reporter's comment have been substantively addressed through related tickets FHIR-46176 (expression traversal fixes, applied as Persuasive) and FHIR-53536 (multi-instance invariant operation fixes, applied as Persuasive). The human-readable invariant description, while arguably less precise than the proposed alternative, follows established HL7 conformance language patterns and does not create conformance ambiguity since implementers validate against the FHIRPath expression, not the description text. Reopening solely for wording preference is not warranted given the related expression fixes already applied.
