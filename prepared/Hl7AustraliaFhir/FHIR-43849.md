## Summary

Queensland Health has suggested that an AU Core–specific value set be defined for the Condition.code binding, rather than relying on the NCTS-published `clinical-condition-1` value set. The rationale is twofold: an AU Core value set could be maintained through HL7 Australia's own standards development process independently of the NCTS, and the current NCTS value set—which includes all members of the Problem/Diagnosis SNOMED CT-AU reference set—may not have been curated with the same goals as AU Core. The ticket was discussed during AU Core R1 and deferred; it is to be reconsidered in AU Core R3.

## Details

**Description:**

This issue was migrated from GitHub issue [hl7au/au-fhir-core#69](https://github.com/hl7au/au-fhir-core/issues/69), originally posted by Reuben Daniels in [AU Core Condition Consensus Discussion #52](https://github.com/hl7au/au-fhir-core/discussions/52#discussioncomment-7309999).

From Queensland Health:

> Code for the Condition/Problem or Diagnosis is currently extensibly bound to the [clinical-condition-1](https://healthterminologies.gov.au/fhir/ValueSet/clinical-condition-1) NCTS value set. It is suggested that an AU Core specific value set is defined for this binding for the following reasons:
>
> - An AU Core value set can be maintained/curated through the HL7 Australia standards development process independent of the NCTS. Note: an AU Core value set may still be published in the NCTS if HL7 Australia and the NCTS operator are open to this.
> - The NCTS value set is itself defined as including all the members of the Problem/Diagnosis SNOMED CT-AU reference set. While many of these concepts are likely to be appropriate for this binding, the reference set may not have been developed with the same goals as AU Core. Further, changes to the reference set (e.g. retirement of member concepts by the Australian NRC or SNOMED International) would impact this value set's expansion and thus the binding.

## Proposed Dispositions

### Disposition A: Accept As Requested
#### Proposal
Create a new AU Core–specific value set for Condition.code that is authored and maintained through the HL7 Australia standards development process. Bind the AU Core Condition profile's `Condition.code` to this new value set using an extensible binding. The value set content may initially mirror the NCTS `clinical-condition-1` set but would be independently curated to align with AU Core scope and goals. The value set could optionally be co-published in the NCTS by agreement.
#### Justification
This approach gives HL7 Australia direct governance over the value set content, ensuring alignment with AU Core's clinical scope. It insulates AU Core from unintended consequences of SNOMED CT-AU reference set changes managed by the NRC. This mirrors the pattern already established for AU Core AllergyIntolerance.code, where a curated value set is used as an Additional Binding.

---

### Disposition B: Alternative Approach
#### Proposal
Rather than replacing the existing binding, add an AU Core–curated value set as an Additional Binding (preferred) on Condition.code, alongside the existing extensible binding to the NCTS `clinical-condition-1` value set. This Additional Binding would contain a curated subset of condition codes deemed most relevant to AU Core use cases. Work with the NCTS to publish this value set within the NCTS infrastructure.
#### Justification
This approach preserves compatibility with the existing NCTS binding while providing AU Core–specific guidance through the Additional Binding mechanism. It avoids duplicating governance by leveraging NCTS publishing infrastructure. A commenter specifically suggested this pattern, noting it follows the precedent set by the AllergyIntolerance code binding. It also reduces risk of breaking existing implementations that rely on the current binding.

---

### Disposition C: Decline
#### Proposal
Decline the request with no change. Continue using the NCTS `clinical-condition-1` value set as the binding for Condition.code in AU Core.
#### Justification
The existing NCTS value set is the established national terminology resource for clinical conditions in Australia. Creating a separate AU Core value set introduces governance complexity and potential divergence between AU Core and NCTS. The current practice—agreed upon between HL7 AU and the NCTS—is that SNOMED CT-based value sets are authored and published via the NCTS. The comment history indicates this issue may be resolved through broader NCTS/HL7 AU governance alignment, and the ticket has already been deferred once. Changes to the SNOMED reference set have not been shown to cause actual problems in practice.

---

### Recommendation

**Recommended disposition:** B

Disposition B (Additional Binding) strikes the best balance between the requester's legitimate concerns and the existing governance model. It allows AU Core to provide curated, clinically focused guidance for Condition.code without disrupting the established NCTS binding or implementations that already depend on it. This approach follows the proven pattern already used for AllergyIntolerance.code in AU Core, as explicitly suggested by a commenter. It also respects the longstanding agreement that SNOMED CT-based value sets are published through the NCTS, while still giving HL7 Australia a mechanism to curate and recommend a preferred subset aligned with AU Core goals. Since the ticket has been deferred to AU Core R3, there is time to coordinate with the NCTS on co-publishing the curated value set.
