## Problem Statement

The ticket requests a standardised identifier type for private health insurers registered in Australia, to be defined in AU Base. Currently there is no unambiguous way to identify a private health insurer organisation in FHIR within the Australian context. This is needed to support ordering and billing workflows — particularly for `Coverage.payor.identifier` and `Coverage.identifier.assigner`.

The requirement originated from the Draft Diagnostic Requests work ([Diagnostic Coverage Profile](http://build.fhir.org/ig/hl7au/draft-diagnostic-orders/branches/master/StructureDefinition-eorders-coverage.html)) and was raised as GitHub issue [hl7au/au-fhir-base#791](https://github.com/hl7au/au-fhir-base/issues/791).

**Note:** AU Base already supports a patient's individual private health insurance *member* number. This ticket is about identifying the *insurer organisation itself*.

## Key Discussion Points

### Identifier Source — ATO vs Services Australia
- **ATO (Australian Tax Office)** assigns identifiers to "complying" private health insurers (e.g. `HCF`, `NTF`). The consumer-facing list is at [privatehealth.gov.au/dynamic/insurer](https://privatehealth.gov.au/dynamic/insurer).
- **Services Australia (SA)** maintains a separate list of private health insurers with different codes for some entities (e.g. Defence Health Limited is `DHF` in SA but `AHB` in ATO).
- The two lists identify the same real-world entities but may use different codes — the chosen system URL must clearly declare which issuing authority's codes are being used.

### System URL — Three Proposals
1. **ATO URL** — a URL reflecting the Australian Tax Office as the identifier issuer.
2. **HL7 AU URL** — a URL under the HL7 Australia namespace.
3. **ADHA-issued URL** — proposed by ADHA (dbojicic-agency): `http://ns.electronichealth.net.au/id/private-health-insurer-id`, since ADHA already registers health-related identifier namespaces (similar to Medicare numbers).

### Insurer-Scoped Member Number (Adjacent Concern)
- gonatf (Oct 2024) proposed a separate insurer-scoped member number system: `http://ns.electronichealth.net.au/id/insurer-scoped/membernumber/1.0/{insurer-code}`, where `{insurer-code}` comes from the privatehealth.gov.au list. This is a related but distinct concern from the insurer identifier itself.

### Real-World Usage
- **Genie** (EMR vendor) currently uses ATO IDs in `Coverage.payor.identifier` with `type = $v2-0203#NII`, `system = "https://privatehealth.gov.au/dynamic/insurer"` — but this consumer website URL is not a proper canonical identifier system URL.
- **ACT Government (Timothy Panoho)** reports using Services Australia's Eclipse 3-letter codes, manually extracted and loaded into billing systems. They do not use ATO lists.
- **heathfrankel** clarified that private insurance is primarily used for theatre bookings rather than diagnostic requests (based on discussions with Sonic and Healius).

### What Is Needed
1. A new **Identifier profile** in AU Base for private health insurer identifiers.
2. An agreed **system URL** that clearly identifies the issuing authority.
3. Update the existing [Coverage/private example](https://build.fhir.org/ig/hl7au/au-fhir-base//Coverage-private.html) to demonstrate use of the new identifier type.

## FHIR Path References

## Proposed Dispositions

### Disposition A: Persuasive with Modification — Define identifier using ADHA namespace
**Resolution:** Accept the request and define a new AU Base Identifier profile for private health insurers using the ADHA-proposed namespace `http://ns.electronichealth.net.au/id/private-health-insurer-id`. Require resolution of which authority's codes (ATO vs Services Australia) are canonical values before publishing. Update the Coverage/private example.

**Rationale:** The ADHA namespace is consistent with existing Australian health identifier patterns (e.g. Medicare, IHI). ADHA has offered to register the namespace. The outstanding question of ATO vs SA code lists must be resolved first, but the approach is sound. This aligns with real-world usage in EMR systems like Genie and ACT Government billing.

### Disposition B: Deferred — Pending identifier authority resolution
**Resolution:** Defer until the ATO vs Services Australia identifier question is conclusively resolved and ADHA confirms the namespace registration and OID allocation (`1.2.36.1.2001.1005.xx`).

**Rationale:** The discussion shows active disagreement about which code list should be canonical. ATO codes cover only "complying" policies while SA has a broader list with different codes for some insurers. Publishing an identifier profile before this is resolved risks creating a standard that doesn't match real-world usage.

### Disposition C: Persuasive — Define two separate identifier systems
**Resolution:** Accept and define two identifier profiles — one for ATO-issued insurer codes and one for SA-issued insurer codes — each with a clearly distinguished system URL. Let implementers choose which is appropriate for their context.

**Rationale:** Both ATO and SA lists are authoritative in their domains. Rather than forcing a single canonical list, defining both reflects the reality that different systems already use different lists. The system URL makes the source unambiguous.

## Recommendation

**Disposition A: Persuasive with Modification** is recommended. The ADHA namespace proposal from dbojicic-agency is well-aligned with existing Australian health identifier conventions, and ADHA is the appropriate authority to register this namespace. The work group should resolve the ATO vs SA code list question as part of the implementation — likely by selecting the privatehealth.gov.au list as the canonical source (since it is publicly accessible and already in use by vendors like Genie) — and then proceed with the identifier profile definition. This unblocks the Coverage payor identification gap that multiple implementers have identified as a real need.
