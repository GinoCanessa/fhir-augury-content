## Description Summary

The submitter raises three concerns about the AU Core Smoking Status Observation profile:

1. **Observation.value — smoking status codes are clinically imprecise.** The current value set includes only broad status codes (Occasional tobacco smoker, Former smoker, Lifetime non-smoker, Current smoker). The submitter argues that for meaningful clinical risk assessment (cardiovascular, neurovascular, respiratory, cancer), duration of smoking and duration since cessation are critical variables. They suggest expanding the value set to include more precise codes reflecting cessation duration and quit attempts.

2. **ObservationInterpretationCodes (extensible) — generic and non-specific to smoking.** The bound value set contains concepts like "significant change up/down," "worse," "off scale low," which are designed for diagnostic test results and are not applicable to smoking status observations.

3. **ObservationReferenceRangeMeaningCodes (preferred) — not applicable to smoking status.** Concepts such as "normal," "recommended," "treatment," "therapeutic" are irrelevant to smoking status. The submitter suggests curating values appropriate for smoking status observations.

## Resolution Description

The proposed resolution is **Not Persuasive**, with the following rationale:

1. **Value set alignment:** The current smoking status concept and value set are aligned with AUCDI (Australian Core Data for Interoperability) and also with the US Core Smoking Status Observation profile. The more granular concepts the submitter requests (overall years of smoking, overall quit date, typical amount) are defined in AUCDI R2 under "Tobacco smoking summary" (3.35) but have been designated by the Sparked CDG as related to the "Chronic condition management" use case rather than "Core," and thus are not automatically included in AU Core yet. The resolution suggests that for inclusion, specific change requests aligned with AUCDI should be raised with implementer support, or new concepts should be introduced into CDG discussions.

2. **Interpretation and referenceRange:** `Observation.interpretation` and `Observation.referenceRange.type` are not supported (not Must Support) in AU Core Smoking Status. To propose these as minimum requirements, specific change requests would need to be raised and introduced to CDG discussions.

## Proposed Dispositions

### Disposition A: Not Persuasive (Affirm Current Resolution)
**Rationale:** The current smoking status value set is intentionally aligned with both AUCDI and US Core. The more granular concepts requested (cessation duration, pack-years, quit attempts) are tracked in AUCDI R2 under the "Chronic condition management" use case, not "Core." The generic `Observation.interpretation` and `Observation.referenceRange.type` elements are inherited from the base Observation resource and are not Must Support in AU Core Smoking Status — they do not need smoking-specific value sets since they are not profiled. The submitter's concerns, while clinically valid, are better addressed through the AUCDI/CDG governance pathway when those concepts are elevated to Core scope. FHIR-52578 (retracted) already explored similar territory.

**Action:** Affirm Not Persuasive. No changes to AU Core Smoking Status profile.

### Disposition B: Not Persuasive with Note
**Rationale:** Affirm Not Persuasive but add an explicit note acknowledging the clinical validity of the request and pointing submitters to the AUCDI R2 roadmap for tobacco smoking summary concepts. This provides a clearer path forward for those who wish to pursue the granular smoking data through the appropriate governance channel (CDG discussions and specific change requests per concept).

**Action:** Affirm Not Persuasive. Add a comment to the ticket documenting the recommended pathway: (1) raise individual change requests for each AUCDI R2 tobacco concept desired in AU Core, (2) gather implementer support, (3) bring to CDG for "Core" designation.

### Disposition C: Persuasive with Modification (Partial)
**Rationale:** While the value set expansion and interpretation/referenceRange concerns are premature for AU Core (pending AUCDI governance), the ticket raises a valid point that could be partially addressed through documentation. AU Core could add guidance text to the Smoking Status profile acknowledging that the current value set captures broad status only and that more detailed tobacco use information (quit date, years of smoking, pack-years) may be represented using additional Observation instances aligned with AUCDI R2 Tobacco smoking summary concepts as they become available.

**Action:** Add informative guidance text to the AU Core Smoking Status profile page noting the relationship to AUCDI R2 tobacco concepts and future expansion path. No structural or value set changes.

---

## Recommendation

**Disposition B: Not Persuasive with Note** is recommended.

The current resolution rationale is sound — the value set is aligned with AUCDI and US Core, and the requested granularity belongs to the "Chronic condition management" scope, not "Core." However, the ticket has generated meaningful community discussion (Brett Esler's AUCDI R2 mapping, the AUCDI/CDG scope assessment, and scheduling for AU Core TDG discussion), and the submitter's clinical reasoning is valid. Adding an explicit note that documents the recommended pathway for future advancement (raise per-concept change requests with implementer support → CDG discussion → potential Core designation) provides constructive closure and preserves institutional knowledge for when these concepts are ready for Core consideration. This also distinguishes the disposition from a flat rejection and acknowledges the work already captured in AUCDI R2 (FHIR-52578).
