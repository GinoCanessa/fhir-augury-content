## Details

This change request proposes that the AU Base PractitionerRole profile adopt the [additionalBinding extension](https://build.fhir.org/ig/FHIR/fhir-tools-ig/StructureDefinition-additional-binding.html) from the FHIR Tools IG to formally represent value sets that are currently described only in the profile's implementation guidance text.

The submitter references a proposal branch demonstrating the change: [AU Base Practitioner Role (additionalBindings branch)](https://build.fhir.org/ig/hl7au/au-fhir-base/branches/additionalBindings/StructureDefinition-au-practitionerrole.html).

This ticket is part of a broader initiative by the same reporter to adopt additionalBindings across multiple AU Base profiles (Organization, Pathology Result, Medication profiles, Immunization). The rationale is that the additionalBinding extension, now available in the FHIR Tools IG, provides a machine-readable, standards-based mechanism for expressing terminology recommendations — replacing informal prose guidance with computable bindings. This improves clarity for implementers and tooling, without altering the primary conformance binding inherited from the FHIR standard.

## Proposed Dispositions

### Disposition A: Persuasive (Accept as Submitted)

**Resolution:** Persuasive

Accept the request to add additionalBinding extensions to the AU Base PractitionerRole profile to represent value sets currently described in implementation guidance.

**Rationale:** This change is consistent with the broader AU Base initiative to adopt additionalBindings across profiles. The sibling tickets for Pathology Result (FHIR-46080), Medication profiles (FHIR-44823), and Immunization (FHIR-44821) have all been accepted and applied — two as Persuasive and one as Persuasive with Modification. The additionalBinding extension is a well-established mechanism in the FHIR Tools IG that converts informal implementation guidance into computable, machine-readable terminology bindings. The proposal branch demonstrates the change is straightforward and does not alter the primary conformance binding.

**Action:** Apply additionalBinding extensions to the AU Base PractitionerRole profile as demonstrated in the [proposal branch](https://build.fhir.org/ig/hl7au/au-fhir-base/branches/additionalBindings/StructureDefinition-au-practitionerrole.html), representing the currently-documented value sets with appropriate binding purposes.

---

### Disposition B: Persuasive with Modification

**Resolution:** Persuasive with Modification

Accept the intent but request the submitter or workgroup to specify the exact value sets and binding purposes (e.g., preferred, component) for the PractitionerRole profile elements before applying.

**Rationale:** The ticket describes the concept but does not enumerate the specific value sets to be added as additionalBindings or their binding purposes. The related Pathology Result ticket (FHIR-46080) was resolved as "Persuasive with Modification" with a detailed table specifying value sets and purposes. Applying the same rigor here would ensure consistency and completeness. The workgroup should review the proposal branch and confirm the specific value sets and purposes.

**Action:** Accept the change in principle. Request the submitter or workgroup to provide a detailed list of value sets and binding purposes for each PractitionerRole element (e.g., `PractitionerRole.code`, `PractitionerRole.specialty`), then apply accordingly.

---

### Disposition C: Not Persuasive (Decline)

**Resolution:** Not Persuasive

Decline the request to add additionalBindings to the AU Base PractitionerRole profile at this time.

**Rationale:** While the additionalBinding extension is a useful mechanism, the existing implementation guidance text already communicates the recommended value sets to implementers. Converting guidance to additionalBindings introduces tooling dependencies and may not provide sufficient benefit for this particular profile if the value sets are already well-understood. However, this rationale is weak given that sibling tickets have been accepted.

**Action:** No change to AU Base PractitionerRole. Retain existing implementation guidance text.

---

## Recommendation

**Recommended Disposition: B — Persuasive with Modification**

The intent of this ticket aligns with an established pattern across AU Base profiles, and sibling tickets have already been resolved and applied. The change is well-motivated and consistent with FHIR tooling evolution. However, the ticket itself lacks the explicit detail (specific value sets and binding purposes per element) that was required for the analogous FHIR-46080 (Pathology Result). Adopting the "Persuasive with Modification" approach — as was done for FHIR-46080 — ensures the workgroup confirms the specific value sets and purposes before implementation, avoiding ambiguity and maintaining consistency with how similar tickets in this series were resolved.
