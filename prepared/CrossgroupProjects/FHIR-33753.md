## Summary

This change request asks that the `military-service-discharge-status-vs` Value Set use a canonical URL rooted in `terminology.hl7.org` (THO) instead of `hl7.org/fhir`. The submitter recommends creating a UTG ticket to register the Value Set in THO so that it follows HL7 terminology governance conventions.

The resolution comment from Sean Muir (2022-02-07) notes that while the direction to use THO-rooted canonical URLs is expected in FHIR R5, it was not expressly mandated in FHIR R4. Work groups agreed to consider such ballot votes "non-persuasive" at that time. The ticket was resolved as **Considered for Future Use**.

## Details

- **Core Issue:** The US Military Service IG defines the `military-service-discharge-status-vs` Value Set with a canonical URL under `hl7.org/fhir` rather than `terminology.hl7.org`. Per emerging HL7 terminology governance policy, value sets that are shared/reusable terminology artifacts should have their canonical URLs rooted in THO (the HL7 Terminology, hosted at `terminology.hl7.org`).
- **Context:** This ticket is one of a batch of nearly identical requests (FHIR-33821 through FHIR-33831) filed by Ron G. Parker against multiple IGs, all requesting the same canonical URL migration to THO. This particular ticket targets the US Military Service (FHIR) specification.
- **Resolution Rationale:** At the time of resolution, FHIR R4 did not explicitly require THO-rooted canonical URLs for IG-defined value sets. The work group deferred the change ("Considered for Future Use"), expecting FHIR R5 to formalize the requirement. The position was that ballot votes requesting this change would be treated as non-persuasive for R4-era IGs.

## Proposed Dispositions

### Disposition A: Accept — Reaffirm "Considered for Future Use"

Agree with the original resolution. The request is valid in principle — THO-rooted canonical URLs are the intended direction — but the US Military Service IG is an R4-era specification. Migrating the canonical URL now would be a breaking change for implementers currently referencing the existing URL. Retain the "Considered for Future Use" resolution and revisit when the IG is updated to target FHIR R5 or later, where THO-rooted canonical URLs are formally expected.

### Disposition B: Persuasive with Modification — Migrate to THO in the next IG release cycle

Accept the request but defer execution to the next planned release of the US Military Service IG. Create a UTG ticket to register `military-service-discharge-status-vs` in THO, establish a redirect from the old canonical URL, and update the IG's value set reference in the subsequent ballot cycle. This aligns with the evolving HL7 policy without forcing an immediate breaking change.

### Disposition C: Decline — Not Persuasive

Decline the request. The canonical URL change is not required by the FHIR R4 specification under which this IG was developed. Changing canonical URLs is a breaking change that would affect all downstream implementers. The value set content and semantics are unchanged regardless of where the canonical URL is rooted. The issue should only be addressed if and when the IG is formally migrated to a FHIR version that mandates THO-rooted URLs.

## Recommendation

**Disposition A: Reaffirm "Considered for Future Use."**

The original resolution correctly identified that FHIR R4 does not mandate THO-rooted canonical URLs. The request reflects sound terminology governance direction that HL7 is moving toward, but applying it retroactively to an R4 IG would create unnecessary breaking changes without clear specification mandate. The "Considered for Future Use" resolution appropriately acknowledges the validity of the request while deferring action to a future IG version targeting a FHIR release where THO canonical URLs are the established convention. This is consistent with the cross-work-group agreement noted in the resolution comment.
