## Summary

The submitter (Bryn Rhodes) notes that although the **US Vital Signs with Qualifying Extensions** implementation guide provides example instances, these examples are **not linked from the profile and extension definition pages**. This makes it difficult for implementers to discover and reference example usages when reviewing a specific profile or extension. The request is to add explicit "Example" links on each profile and extension definition page, pointing to the relevant example instances.

---

## Details

The core issue is one of **discoverability**. Example instances exist within the IG but are only accessible by navigating to a separate examples page or section. Profile and extension definition pages — which are the primary landing pages for implementers — do not include links to those examples. Adding example links directly on definition pages is a common pattern in other IGs and in the FHIR core specification. This is a straightforward IG authoring change that would improve usability without altering conformance semantics.

## Proposed Dispositions

### Disposition A: Accept (Persuasive)

**Action:** Accept the change request and add example links to the profile and extension definition pages in the US Vital Signs with Qualifying Extensions IG.

**Rationale:** This is a straightforward usability improvement with clear precedent. Similar requests in other specifications (FHIR-8543, FHIR-42363) have been resolved as Persuasive. Linking examples from definition pages is a well-established pattern in FHIR IG publishing (the IG Publisher supports `example` metadata on instances that automatically generates these links). The effort required is minimal — the examples already exist; they simply need to be tagged with the appropriate `exampleFor` or `example` metadata in the IG resource definitions so the publisher generates the links. This directly improves implementer experience.

### Disposition B: Alternative — Persuasive with Modification

**Action:** Accept but broaden the scope to also include a consolidated examples listing on a dedicated "Examples" tab or section, in addition to linking from individual profile/extension pages.

**Rationale:** While linking from individual definitions addresses the immediate concern, adding a centralized examples index would further aid discoverability. Some implementers prefer to browse all examples in one place. This is a modest additional effort and follows the pattern used by many mature IGs. However, this goes beyond the original request and the work group may prefer to keep the scope narrow.

### Disposition C: Decline (Not Persuasive)

**Action:** Decline the request, noting that examples are already available within the IG and can be found through the artifacts page.

**Rationale:** The examples are present in the IG; they are simply not cross-linked from definitions. Implementers can find them via the artifacts listing or table of contents. However, this disposition would leave a known usability gap and runs counter to the precedent set by FHIR-8543 and FHIR-42363, both of which were resolved as Persuasive for similar requests.

---

## Recommendation

**Disposition A: Accept (Persuasive)** is recommended. The request is well-founded, low-effort, and aligns with established precedent. The examples already exist — they simply need proper metadata linking from the profile and extension definitions. This is a standard IG authoring practice supported by the IG Publisher tooling, and similar requests across other specifications have consistently been accepted.
