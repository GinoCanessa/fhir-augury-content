## Description

The reporter identifies that the US Military Service IG lacks a mechanism to query for **all** relevant Military Service resources for a given patient. Currently there is no defined way to retrieve a consolidated view of a patient's military service history. Two possible approaches are suggested:

1. **Simple REST query** — A search that returns a Bundle (search set) containing all military-service-related resources.
2. **Composition resource** — Define a Composition profile that aggregates all military service items (service episodes, deployment episodes, occupational data, etc.) into a single summarization document.

## Analysis

This ticket addresses a real implementability gap: the US Military Service IG defines profiles for individual military service episodes, deployment episodes, and occupational data, but provides no mechanism to retrieve or present all of these resources together for a given patient. Without a summarization approach, implementers must independently discover and assemble multiple resources with no guidance on completeness.

The work group previously voted 11-0-1 to defer this as "Considered for Future Use" but explicitly reversed that decision after STU1 publication, signaling intent to address it in the next cycle. The two proposed approaches (search Bundle vs. Composition) each have trade-offs:

- A **defined search interaction** (e.g., a named query or documented search parameters) is simpler but provides no structure to the returned data.
- A **Composition profile** provides a structured, document-style summary but requires more profiling work and may introduce additional conformance requirements.

Several related open tickets (FHIR-33578, FHIR-33353) request additional content types (exposure risks, clinical conditions) that would also benefit from inclusion in a summarization mechanism, suggesting this ticket's scope may grow.

## Proposed Dispositions

### Disposition A: Persuasive — Define a Military Service Summary Composition Profile

Accept the request and define a new Composition profile in the IG that aggregates all military-service-related resources (Military Service Episode, Deployment Episode, Military Occupation, and potentially exposure risk observations) into a structured summarization document. Include guidance on which sections the Composition should contain and how entries reference the existing profiles.

**Rationale:** A Composition provides a well-defined, reusable summarization artifact that aligns with FHIR document patterns. The work group already indicated interest in this approach (seanmuir comment, 2022-08-04). This also creates a natural home for exposure risk data requested in FHIR-33578 and FHIR-33353.

### Disposition B: Persuasive with Modification — Define Search Guidance Rather Than a Composition

Accept the concern but address it by documenting recommended search interactions (e.g., search parameters, _include directives, or a named query) that allow clients to retrieve all military service resources for a patient in a single search set Bundle, rather than defining a Composition profile.

**Rationale:** A search-based approach is lighter-weight, avoids the overhead of a Composition profile, and may be sufficient for systems that simply need to retrieve all relevant resources. This can be implemented sooner and does not preclude adding a Composition in a later cycle.

### Disposition C: Considered for Future Use — Defer Until Related Tickets Are Resolved

Defer this ticket until the related open tickets (FHIR-33578, FHIR-33353, FHIR-33362, FHIR-33442) are resolved, as the scope of what should appear in a summary depends on decisions about exposure risks, occupational coding, and value set bindings.

**Rationale:** The content that a summarization mechanism should include is still being defined. Resolving the dependent tickets first would ensure the summarization approach is comprehensive rather than requiring revision.

---

## Recommendation

**Disposition A: Persuasive — Define a Military Service Summary Composition Profile**

This is the strongest option because: (1) the work group explicitly re-opened this ticket post-STU1 publication, signaling intent to act rather than defer again; (2) the WG previously expressed interest in the Composition approach specifically; (3) a Composition provides a structured, extensible framework that can incorporate future content (exposure risks, clinical conditions) as those related tickets are resolved; and (4) it aligns with established FHIR document patterns used in other IGs (e.g., International Patient Summary). Defining search guidance (Disposition B) could be included as complementary guidance within the same ballot cycle.
