## Details

Bryn Rhodes reports that many links to profiles within the US Vital Signs with Qualifying Extensions IG open in a new browser tab/window instead of navigating within the current tab. This gives users the impression that the linked content belongs to a separate IG rather than the same guide.

The reporter acknowledges that opening a new tab **is** appropriate when linking to an external IG (e.g., the base Vital Signs profile definition in the core FHIR spec). However, for links pointing to profiles and pages **within** the same IG, the links should replace content in the current tab to maintain a cohesive navigation experience.

This is a usability/authoring concern related to how `target="_blank"` (or equivalent) is applied to hyperlinks in the IG publisher output or the IG source content.

## Proposed Dispositions

### Disposition A: Accept — Persuasive (Recommended)

**Action:** Agree that intra-IG links should not open in new tabs/windows; update the US Vital Signs with Qualifying Extensions IG source to remove `target="_blank"` attributes (or equivalent templating directives) from links that point to pages within the same IG. Retain new-tab behavior only for links to external IGs or the base FHIR specification.

**Rationale:** This is a straightforward usability fix. Opening every link in a new tab fragments the user's browsing experience and makes the IG feel disjointed. The reporter correctly distinguishes between internal and external links. Fixing this requires reviewing the IG's page templates or markdown source for unnecessary `target="_blank"` usage — a low-risk, low-effort change that improves navigation coherence.

### Disposition B: Accept — Persuasive with Modification

**Action:** Accept the request but apply it as a broader IG Publisher tooling guideline rather than a one-off fix to this specific IG. Recommend that the IG Publisher default behavior be reviewed to ensure intra-IG links do not open in new tabs, while external links (detected by differing canonical base URLs) do.

**Rationale:** If the root cause is in the IG Publisher's default link rendering rather than in this IG's source content, fixing it at the tooling level would benefit all IGs. However, this broadens the scope and may require coordination with the FHIR Tooling/Publishing work group, potentially delaying resolution.

### Disposition C: Decline — Not Persuasive

**Action:** Decline the change request, noting that link target behavior is a minor usability preference that varies by user expectation, and that the current behavior (opening in new tabs) avoids users losing their place in the IG.

**Rationale:** Some users prefer new-tab behavior as it preserves context. The request is subjective and the current behavior is not incorrect — it is a design choice. However, this argument is weak given that the reporter specifically notes the inconsistency between internal and external links, and standard web UX conventions favor same-tab navigation for internal links.

## Recommendation

**Disposition A: Accept — Persuasive.** The request is well-reasoned, clearly scoped, and identifies a genuine usability issue. Internal IG links should navigate within the current tab to provide a cohesive browsing experience, while external links may continue to open in new tabs. This is a low-risk fix confined to the US Vital Signs with Qualifying Extensions IG source or templates.
