## Analysis

The core question is whether IG-specific terminology artifacts (value sets and code systems) defined within the US Military Service IG should use `terminology.hl7.org` (THO) canonical URLs instead of `hl7.org/fhir`-rooted URLs. The submitter's position is that THO should be the home for these artifacts and that UTG tickets should be filed for registration.

The resolution comment from seanmuir (2022-02-07) indicates the WG concluded that while FHIR R5 would provide clearer guidance on canonical URL placement, FHIR R4 does not mandate the change. The WGs agreed these ballot comments would be treated as "non-persuasive" at the time.

The Zulip discussion from March 2021 reinforces that the FHIR-vs-THO URL governance rules were still being clarified. Lloyd McKenzie articulated the general rule (code-bound = FHIR space; shareable/governable = THO), but acknowledged that the actual migration was incomplete and inconsistent. Since the US Military Service IG artifacts are IG-specific value sets (not core FHIR status codes bound to `code`), the question of whether they belong in THO is less clear-cut and depends on whether they are intended to be shared across IGs and subject to independent terminology governance.

## Proposed Dispositions

### Disposition A: Not Persuasive
**Rationale:** The US Military Service IG value sets are IG-specific artifacts. Per Lloyd McKenzie's guidance, few FHIR value sets should move to THO—only those that are both shareable and amenable to maintenance outside the FHIR ballot process. IG-specific value sets are typically maintained alongside their IG and do not warrant independent THO governance. The current resolution ("Considered for Future Use") already deferred this, and no new guidance has emerged to change the calculus. Moving URLs would also be a breaking change for existing implementations referencing the current canonical URLs.

### Disposition B: Persuasive with Modification
**Rationale:** Accept the principle that terminology artifacts should eventually migrate to THO once clear cross-IG governance rules are established, but defer the actual URL change until: (1) the broader FHIR/THO canonical URL reconciliation effort is completed, and (2) a process exists for coordinated URL migration that includes redirect/alias support. Create a tracking ticket to revisit when THO governance for IG-sourced value sets is formalized. This acknowledges the submitter's valid concern without causing immediate breaking changes.

### Disposition C: Persuasive
**Rationale:** The submitter is correct that terminology artifacts should be governed through THO. Register the `military-service-separation-reason-vs` Value Set in THO with a `terminology.hl7.org`-rooted canonical URL and create the necessary UTG ticket. This aligns with the long-term HL7 strategy of centralizing terminology governance in THO and ensures the value set is discoverable and maintainable through standard UTG processes.

## Recommendation

**Disposition A: Not Persuasive** — The existing resolution of "Considered for Future Use" was appropriate when applied in 2022 and remains so. The WG already determined this was non-persuasive under FHIR R4 rules. IG-specific value sets do not clearly fall under the THO governance mandate per Lloyd McKenzie's articulated criteria. No new developments (e.g., completed FHIR/THO reconciliation, formal R5 URL migration guidance) have emerged that would change the original assessment. Reopening this without a broader, coordinated URL migration strategy would risk breaking changes with no immediate governance benefit.
