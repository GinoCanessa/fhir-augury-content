**Specification:** US Military Service (FHIR)
**Work Group:** Cross-Group Projects

The commenter requests that the `military-service-paygrade-vs` Value Set canonical URL be rooted under `terminology.hl7.org` (THO) rather than `hl7.org/fhir`. A UTG ticket should be created to register the Value Set in THO. This is one of a series of seven identical comments (FHIR-33749 through FHIR-33755), each targeting a different Code System or Value Set in the US Military Service IG.

### Resolution Description (from ticket)

## Related Zulip Discussions

### 1. [shorthand > Change terminology root to THO?](https://chat.fhir.org/#narrow/stream/shorthand/topic/Change%20terminology%20root%20to%20THO%3F) (2023-05–2024-04)

David Pyke received ballot feedback to root IG value set/code system URLs in `terminology.hl7.org`. Chris Moesel explained two approaches: (a) go through the UTG process to publish in THO, or (b) use SUSHI caret rules (`^url = "http://terminology.hl7.org/..."`) plus `special-url` parameters in sushi-config.yaml. The thread confirms that simply changing the URL prefix without the UTG process is technically feasible in SUSHI but creates governance questions. This is directly relevant to the approach the US Military Service IG would need to take.

### 2. [terminology > UTG or FHIR?](https://chat.fhir.org/#narrow/stream/terminology/topic/UTG%20or%20FHIR%3F) (2021-03)

Lloyd McKenzie, Ted Klein, and others discussed the rules for when terminology lives in FHIR vs. THO. Key points:
- **Code-type bindings must stay in FHIR space**; everything else (not example/low-maturity) should go to THO.
- Ted Klein noted that the migration from FHIR to THO during R4 was incomplete and inconsistent, and the reconciliation effort had not been fully undertaken.
- Value sets with canonical URLs rooted at `http://terminology.hl7.org/ValueSet` go into THO via UTG; those rooted in `http://hl7.org/fhir` stay in the FHIR ecosystem.

## Proposed Dispositions

### Disposition A: Non-Persuasive (Retain current resolution)

**Rationale:** The existing resolution of "Considered for Future Use" appropriately defers this work. As noted in the resolution description, explicit direction on canonical URL rooting for IG-local value sets was not clearly established in FHIR R4. The US Military Service IG's value sets are IG-local artifacts, and the ballot comment does not demonstrate that the current `hl7.org/fhir`-rooted URLs cause interoperability issues. The seven sibling tickets were all resolved identically, and re-opening this one without the others would be inconsistent. The work group can revisit when clearer R5+ guidance on IG terminology URL conventions is finalized.

### Disposition B: Persuasive with Modification — Defer to UTG Process

**Rationale:** Accept the principle that THO is the correct home for published HL7 terminology artifacts, but rather than making the URL change directly in the IG, create a UTG proposal to register all seven US Military Service value sets/code systems in THO. This follows the process outlined by Lloyd McKenzie and Ted Klein (Zulip: "UTG or FHIR?" thread) and would give the artifacts proper terminology governance. The modification is that this ticket alone should not drive the change; a coordinated UTG submission covering all seven artifacts (FHIR-33749–33755) should be created.

### Disposition C: Persuasive — Change URLs to THO root

**Rationale:** Accept the comment as-is. Change the canonical URL for `military-service-paygrade-vs` from `http://hl7.org/fhir/...` to `http://terminology.hl7.org/ValueSet/...` in the US Military Service IG, add the URL to the `special-url` parameter (per the SUSHI approach documented in the Zulip shorthand thread), and file a corresponding UTG ticket for THO registration. This applies only to this one value set; the sibling tickets would need to be resolved separately.

---

## Recommendation

**Disposition A: Non-Persuasive** is recommended.

The existing resolution is sound. The core issue — where IG-local terminology canonical URLs should be rooted — was not clearly governed by FHIR R4 rules, and the resolution explicitly acknowledged this as something for future consideration. The Zulip discussions confirm that the FHIR-to-THO migration has been incomplete and that clear direction is being developed for R5+. Re-opening this ticket in isolation would be inconsistent with the coordinated resolution of all seven sibling tickets. If the work group later decides that US Military Service IG terminology should migrate to THO, it should be done as a coordinated effort across all seven tickets with a proper UTG submission, not by re-adjudicating individual ballot comments.
