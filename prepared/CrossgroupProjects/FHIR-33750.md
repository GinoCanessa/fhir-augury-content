## Description

The `military-service-branch-vs` Value Set canonical URL should be rooted in `terminology.hl7.org` (THO), not rooted in `hl7.org/fhir`. A UTG ticket should be created to register the Value Set in THO.

*(Imported as Comment 2 by Ron G. Parker)*

## Related Zulip Discussions

### 1. [terminology > UTG or FHIR?](https://chat.fhir.org/#narrow/stream/terminology/topic/UTG%20or%20FHIR%3F) (2021-03-22 to 2021-03-23)
**Participants:** Melva Peters, Lloyd McKenzie, Rob Hausam, John Moehrke, Forest, Ted Klein, Alexander Henket

Directly relevant background discussion on the broader issue of which terminology artifacts belong in THO vs. FHIR. Key points:
- **Lloyd McKenzie** stated the rule: anything bound to `code` type must live in FHIR space; everything else that is not low-maturity or example content should live in THO. Few FHIR value sets should be in THO—it should be a conscious decision.
- **Ted Klein** explained that during R4 publication, Grahame manually moved some code systems and value sets to THO, but a comprehensive verification of what was moved and what was missed has not been completed. He noted that if a canonical URL is rooted at `http://terminology.hl7.org/ValueSet` then it goes into THO via UTG; if rooted in `http://hl7.org/fhir` or internal to a FHIR IG, it goes into the FHIR ecosystem.
- **Alexander Henket** highlighted inconsistencies between THO and FHIR core spec regarding OID and URI mappings.
- The thread underscores that the boundary between THO and FHIR for value sets was still not fully resolved at the time this ticket was filed.

## Analysis

This ticket requests migrating the `military-service-branch-vs` Value Set canonical URL from the `hl7.org/fhir` namespace to `terminology.hl7.org` (THO). The request is technically sound in principle—HL7's long-term direction is for shareable terminology artifacts to be managed through UTG/THO. However, the resolution notes that explicit direction for this change is part of FHIR R5, not R4, and the workgroup agreed these votes would be considered "non-persuasive" at the time.

The fact that all seven sibling tickets received the identical "Considered for Future Use" resolution suggests this was a batch decision by the work group, likely recognizing that:
1. The US Military Service IG was built against R4 conventions.
2. Migrating canonical URLs is a breaking change for implementers.
3. The THO/FHIR boundary policy was still being clarified (as evidenced by the Zulip discussion).

Since the resolution was applied in February 2022, the question is whether the FHIR R5 direction now provides sufficient guidance to revisit this, or whether the ticket remains appropriately deferred.

## Proposed Dispositions

### Disposition A: Not Persuasive
**Rationale:** The request asks the US Military Service IG to migrate value set canonical URLs to THO, but this IG defines its own value sets for IG-specific use. Per Lloyd McKenzie's guidance, few FHIR value sets should be in THO—it should be a conscious decision for content that is both shareable and amenable to maintenance outside the FHIR ballot process. IG-specific value sets like `military-service-branch-vs` are appropriately rooted in the IG's own namespace under `hl7.org/fhir`. The existing resolution ("Considered for Future Use") already acknowledged the issue is premature. With no new policy mandate requiring the move, the ticket should be closed as not persuasive.

### Disposition B: Persuasive with Modification — Defer to IG-Specific Governance
**Rationale:** The underlying concern is valid—terminology governance best practice favors THO for shared terminology—but the decision of when and how to migrate these artifacts should be made by the US Military Service IG's owning work group in coordination with the Vocabulary WG when the IG next undergoes a major version update. Modify the resolution to explicitly assign responsibility to the IG owners to evaluate canonical URL migration during their next ballot cycle, with guidance that new value sets should use THO URLs if they are intended for cross-IG reuse.

### Disposition C: Not Persuasive — Existing Resolution Stands
**Rationale:** The existing resolution of "Considered for Future Use" is appropriate and already captures the intent. The R5 terminology governance direction has clarified that IG-defined value sets should generally remain in the IG namespace unless they are promoted to cross-IG shared terminology. Since no evidence exists that `military-service-branch-vs` is being reused outside this IG, and since changing the canonical URL would be a breaking change, the current resolution is correct. Close the ticket affirming the existing resolution.

## Recommendation

**Disposition C: Not Persuasive — Existing Resolution Stands**

The existing "Considered for Future Use" resolution correctly captures the state of this request. The `military-service-branch-vs` value set is specific to the US Military Service IG and there is no evidence it is reused across other IGs, which would be the primary driver for THO registration. The Zulip discussions confirm that the THO/FHIR boundary policy focuses on *shared* terminology, not IG-internal artifacts. Changing canonical URLs is a breaking change that should only be undertaken with clear policy mandate and as part of a coordinated IG major version update. The work group's original batch decision to treat these as non-persuasive remains sound. If the IG owners later determine these value sets warrant cross-IG sharing, they can initiate the migration through the UTG process at that time.
