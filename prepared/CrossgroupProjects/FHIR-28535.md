## Summary

Bryn Rhodes requests that the introduction section of the US Vital Signs with Qualifying Extensions IG be updated to clearly explain how its profiles relate to and depend on other IGs. Specifically, the introduction discusses the background for developing the profiles but does not make clear that they derive from and build upon the base FHIR profiles and the current US Core profiles. The ticket requests, at minimum, a clear statement about the dependency on US Core — including which version — and a link to that IG. A diagram showing the profile derivation chain is also suggested.

---

## Details

The US Vital Signs with Qualifying Extensions IG defines profiles for vital signs observations with additional qualifying extensions. The reporter notes that while the introduction provides background on why the profiles were developed, it omits critical information about the profile lineage:

1. **Missing dependency statement**: No explicit statement that the profiles derive from and build upon base FHIR vital signs profiles and US Core vital signs profiles.
2. **Missing US Core version**: No indication of which version of US Core the profiles depend on.
3. **Missing link**: No link to the US Core IG for readers to follow.
4. **Suggested diagram**: A visual diagram showing the derivation chain (base FHIR → US Core → US Vital Signs with Qualifying Extensions) would improve clarity.

This is a documentation/editorial change request rather than a technical or structural change. The ask is straightforward: improve the introduction to help implementers understand how this IG fits into the broader FHIR profile ecosystem.

This ticket has been open since September 2020 and remains unresolved/triaged with no assignee.

## Proposed Dispositions

### Disposition A: Persuasive (Accept)

**Action:** Update the introduction of the US Vital Signs with Qualifying Extensions IG to include:
- A clear statement that the profiles derive from and build upon base FHIR vital signs profiles (Observation) and US Core vital signs profiles.
- Identification of the specific US Core version the IG depends on.
- A hyperlink to the US Core IG.
- Optionally, a diagram illustrating the profile derivation chain (FHIR Observation → US Core Vital Signs → US Vital Signs with Qualifying Extensions).

**Rationale:** This is a straightforward editorial improvement that enhances implementer understanding. Similar requests have been raised for other IGs (FHIR-52163, FHIR-51169), indicating this is a recognized gap. The requested information is factual and already implicit in the IG's profile definitions — making it explicit in the introduction requires minimal effort and adds significant clarity.

### Disposition B: Persuasive with Modification

**Action:** Add a dependency statement and US Core version/link to the introduction, but defer the diagram to a future ballot cycle or make it a non-binding suggestion.

**Rationale:** The textual clarification is essential and low-effort. A diagram, while helpful, requires more design work and may need updating as US Core versions evolve. Splitting the work allows the most impactful change (clear dependency text) to proceed immediately.

### Disposition C: Not Persuasive (Decline)

**Action:** No change. The IG's profile definitions and `ImplementationGuide` resource already declare dependencies on US Core, and implementers can determine the relationship from the profile structure definitions.

**Rationale:** The dependency information is technically available in the IG's machine-readable artifacts. Adding it to the introduction is nice-to-have but not strictly necessary. However, this position is weak given the low effort required and the clear benefit to human readers.

---

## Recommendation

**Disposition A: Persuasive** is recommended. The request is well-justified, low-effort, and addresses a genuine clarity gap. Explicitly documenting IG dependencies and profile derivation in the introduction is a best practice that has been requested across multiple IGs (AU Core, EU Base). Including the version, link, and optionally a diagram would meaningfully improve the reader experience with minimal cost.
