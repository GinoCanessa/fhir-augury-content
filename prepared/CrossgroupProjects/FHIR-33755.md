## Summary

The reporter requests that the `military-service-military-occupation-vs` Value Set canonical URL be rooted in `terminology.hl7.org` (THO) rather than `hl7.org/fhir`. A UTG ticket should be created to register the Value Set in THO. This is one of a series of identical ballot comments (FHIR-33749 through FHIR-33755, FHIR-33822 through FHIR-33825) filed by Ron G. Parker against various value sets in the US Military Service IG and other specifications, all requesting the same canonical URL change.

The resolution description states: *"While specific direction on this is apparently part of FHIR 5 it is not expressly stated in FHIR 4. It was agreed that WGs will consider these votes 'non-persuasive' at this time."*

## Related Zulip Discussions

### [terminology > THO or not in THO](https://chat.fhir.org/#narrow/stream/terminology/topic/THO%20or%20not%20in%20THO) (2024-01-25 to 2024-01-29)
Key discussion initiated by Bret Heale about the nuance of when value sets and code systems should be in THO. **Lloyd McKenzie** clarified the TSMG policy:
- **Code systems** are expected to be in THO (or another external terminology) when FMM 3+, not using data type `code`, and not example-only — unless TSMG grants an exception.
- **Value sets are never required to be in THO.** They may be shared in THO when: (1) they have sufficient complexity to justify sharing, (2) they can be maintained without direct WG oversight, and (3) the implementer community can accommodate changes independent of IG releases.
- Ballot comments arguing for THO registration that violate these expectations **can be found not persuasive**.

### [TSMG > Terminology moved from the base spec to THO](https://chat.fhir.org/#narrow/stream/TSMG/topic/Terminology%20moved%20from%20the%20base%20spec%20to%20THO) (2025-11-02)
Related discussion about moving terminology from the base spec to THO. Notes that changing canonical URLs of code systems is exceptionally problematic, while changing value set canonicals is less disruptive.

### [terminology > UTG or FHIR?](https://chat.fhir.org/#narrow/stream/terminology/topic/UTG%20or%20FHIR%3F) (2021-03-23)
Earlier discussion about the routing rules: if a value set's canonical URL is rooted at `http://terminology.hl7.org/ValueSet` then it goes into THO.

## Proposed Dispositions

### Disposition A: Not Persuasive (Reaffirm Original)
**Rationale:** The original resolution correctly applied the TSMG policy. Value sets are **never required** to be in THO per Lloyd McKenzie's clarification. The military-service-military-occupation-vs value set is tightly coupled to the US Military Service IG and does not clearly meet the THO sharing criteria (sufficient complexity for sharing, maintainable without WG oversight, community tolerance for independent changes). The FHIR 4 specification did not mandate THO placement for value sets. The work group's original "non-persuasive" disposition was consistent with policy.

**Action:** Affirm the existing "Considered for Future Use" resolution. No changes needed.

### Disposition B: Persuasive with Modification — Evaluate for THO Migration
**Rationale:** While value sets are not *required* to be in THO, the military-occupation value set may benefit from THO placement if it has grown in reuse across IGs or if FHIR R5 guidance now recommends it. The sibling ticket FHIR-33763 (Devices WG) was ultimately found Persuasive and applied. The work group should evaluate whether the military-occupation value set meets the THO sharing criteria now, especially considering any evolution since 2022. If it meets criteria, create a UTG ticket to register it.

**Action:** Evaluate the value set against current TSMG criteria. If it qualifies, create a UTG ticket (similar to UP-788 for FHIR-33763). If not, document the rationale and close.

### Disposition C: Persuasive — Move to THO
**Rationale:** The request is reasonable and aligns with the general HL7 direction toward centralizing terminology in THO. Other similar requests (FHIR-33763) have been accepted and applied. Moving the value set to THO improves discoverability, enables cross-IG reuse, and follows the trajectory of FHIR R5 guidance. A UTG ticket should be created.

**Action:** Create a UTG ticket to register `military-service-military-occupation-vs` in THO with a `terminology.hl7.org` canonical URL. Update the US Military Service IG accordingly.

## Recommendation

**Disposition A: Not Persuasive (Reaffirm Original)**

The TSMG policy is clear that value sets are never required to be in THO. The original disposition was correctly applied and consistent with the policy as clarified by Lloyd McKenzie. The military-occupation value set is specific to the US Military Service IG and does not appear to have broad cross-IG reuse that would warrant THO placement. While FHIR-33763 was eventually found Persuasive for the Devices WG, that was a work-group-specific decision based on the 11073MDC code system's broader reusability across 11073-related IGs — a different circumstance. Unless the work group identifies concrete evidence of cross-IG demand for this value set, the original resolution should stand.
