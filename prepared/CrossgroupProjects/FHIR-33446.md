## Issue Description

The reporter raises two concerns with the Pay Grade value set in the US Military Service IG:

1. **Unused value set**: The Pay Grade value set does not appear to be referenced by any profile in the IG. If it is not used, the reporter requests it be removed.
2. **Typo**: The value set description contains a misspelling — "miliatry" should be "military" in the sentence: *"Its scope is limited to miliatry pay grade codes."*

## Resolution Description

> Will update the use of pay grade

The resolution indicates the work group agreed to update how the Pay Grade value set is used (Persuasive disposition), implying it will either be properly integrated into a profile or removed, along with correcting the typo.

## Proposed Dispositions

### Disposition A: Persuasive — Remove unused value set and fix typo

**Action**: Remove the Pay Grade Code value set from the IG since it is not bound to any profile. Correct the "miliatry" typo in the value set description (if retained elsewhere as documentation) or in any remaining references.

**Rationale**: The reporter's observation is valid — an unused value set adds clutter without adding interoperability value. The existing resolution text ("Will update the use of pay grade") supports this direction. The typo fix is straightforward. This aligns with the original Persuasive resolution already assigned to this ticket.

### Disposition B: Persuasive with Modification — Bind value set to a profile rather than removing it

**Action**: Rather than removing the Pay Grade value set, bind it to an appropriate element in the Military Service Episode profile (e.g., as a component of the Observation representing pay grade during service). Fix the "miliatry" typo.

**Rationale**: Pay grade is a meaningful data element for military service history (as noted in FHIR-33363's discussion of pay grades as proxies for skill/managerial levels). Instead of removing the value set, it should be properly integrated. This better serves the IG's goal of capturing comprehensive military service data. However, note that FHIR-33363 (requesting completeness of the CDC pay grade value set) was rejected as Not Persuasive, so the current value set content may be considered adequate.

### Disposition C: Not Persuasive — Retain value set as-is for future use

**Action**: Keep the Pay Grade value set in the IG for anticipated future use. Fix only the typo.

**Rationale**: The value set may be intentionally included for future profiling work or for implementers who wish to use it voluntarily. Removing it now could require re-adding it later. However, this does not address the reporter's valid concern that the IG should not ship artifacts without clear usage, and the work group's own resolution text acknowledges a change is needed.

---

## Recommendation

**Disposition B: Persuasive with Modification** — Bind the Pay Grade value set to an appropriate profile element rather than simply removing it.

The Pay Grade value set represents meaningful military service data. The resolution text ("Will update the use of pay grade") most naturally reads as a commitment to properly integrate the value set, not remove it. Several related tickets (FHIR-33363, FHIR-33361) confirm that pay grade is a recognized data element in the military service domain. Binding it to a profile makes the IG more complete and useful. The typo should be fixed regardless of which disposition is chosen.
