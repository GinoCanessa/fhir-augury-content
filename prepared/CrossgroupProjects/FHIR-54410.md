## Description

The submitter requests that the Detailed Ethnicity value set (OID: 2.16.840.1.114222.4.11.877) be updated to include the two category-level concepts:

- **2135-2** — Hispanic or Latino
- **2186-5** — Not Hispanic or Latino

Currently, the Detailed Ethnicity value set contains only the detailed (granular) ethnicity concepts but omits these top-level category codes. The submitter notes that these categories are necessary to support **computable roll-up** from detailed ethnicity concepts to the categories. This is consistent with how the Detailed Race value set already works — it includes a member value set with all detailed race concepts plus the five category-level race concepts (Race Value Set OID: 2.16.840.1.113883.1.11.14914).

## Analysis

### Context
This ticket is part of the US Core v9 ballot reconciliation cycle. The Detailed Ethnicity value set historically contained only granular ethnicity concepts (descendants of the two CDC ethnicity categories) but omitted the category-level codes themselves. This created an asymmetry with the Detailed Race value set, which already includes its five category-level concepts alongside the detailed codes. Without the category codes in the Detailed Ethnicity set, systems cannot perform a computable roll-up from detailed ethnicity codes to the OMB category level — a common reporting need.

### Resolution Rationale
The resolution (Persuasive) accepted the change as proposed. Gay Dolin (gdolin) confirmed the alignment with the existing race value set pattern. Eric Haas (ehaas) initially objected but retracted. The change was formally approved on the 2026-03-26 CGP call and applied on 2026-04-09, adding concepts 2135-2 (Hispanic or Latino) and 2186-5 (Not Hispanic or Latino) to the Detailed Ethnicity value set in VSAC.

### Impact
- **Change Category:** Compatible, substantive (Enhancement)
- **Scope:** Terminology — value set expansion only. No structural profile changes.
- **Backward Compatibility:** Existing data using only detailed ethnicity codes remains valid. Systems that validate against the expanded value set will now also accept the two category-level codes.

## Proposed Dispositions

### Disposition A: Accept as Applied (Persuasive) ✅ RECOMMENDED
**Rationale:** The change aligns the Detailed Ethnicity value set with the established pattern used by the Detailed Race value set, enabling computable roll-up from granular ethnicity codes to OMB categories. The initial objection was retracted, and the workgroup approved the change unanimously on 2026-03-26. The implementation has been confirmed applied in VSAC. This is a straightforward, well-justified terminology enhancement with no breaking changes.

### Disposition B: Request Deferral for Further Review
**Rationale:** One could argue that the 10-year stability of the existing value set warrants more cautious change management, and that adding category codes to a "detailed" value set blurs the semantic distinction between category and detail levels. However, this concern was raised and retracted by ehaas, the precedent from the race value set is clear, and the change has already been applied.

### Disposition C: Reject and Revert
**Rationale:** If the workgroup believes that a "Detailed" value set should by definition exclude category-level concepts (maintaining a strict hierarchical separation), the change could be reverted and an alternative mechanism for roll-up (e.g., a separate combined value set or ConceptMap) could be proposed. However, this contradicts the existing Detailed Race value set pattern and would leave the asymmetry in place.

## Recommendation

**Accept as Applied (Disposition A).** The change is well-motivated, aligns with existing precedent (Detailed Race value set), was approved by the workgroup after discussion, and has been successfully applied. No objections remain outstanding.
