## Description

The submitter requests that the section 4 heading in the CardX Hypertension Management IG be changed from "Card X" (with a space) to "CardX" (no space). The correct project branding is "CardX" — a single compound word — and the current section title is inconsistent with that branding.

## History & Prior Disposition

This ticket was **previously resolved** as "Considered for Future Use" on 2024-06-27 with a vote of **8-0-1** (Bruce Bray / Jimmy Tcheng). The resolution note stated:

> *"Thank you for catching this. Currently, FHIR Shorthand (FSH) will not permit two letters in the same word to be capitalized, so we are not able to update the header to 'CardX'. We will issue this as a ticket for the FSH developers."*

The ticket was subsequently **re-opened** on 2024-12-12 by Lynn Laakso with the comment: *"re-open post STU1 publication"*, reverting the prior resolution to allow reconsideration in a future cycle.

## Analysis

**Core Issue:** The section 4 heading reads "Card X" instead of the correct branding "CardX". This is a straightforward editorial/branding correction.

**Blocker (Historical):** The original resolution cited a FHIR Shorthand (FSH) tooling limitation that prevented two consecutive uppercase letters in a single word within section headings. This was the sole reason the change was deferred rather than applied.

**Current Context:** The ticket was re-opened post-STU1 publication (December 2024), suggesting the work group believes this can now be addressed — potentially because FSH tooling has been updated, or because a workaround exists (e.g., using a `title:` override in FSH, or editing the generated IG publisher output directly).

**Impact:** Minimal technical impact. This is a display/branding correction in a single section heading. No profile, value set, or conformance changes are involved.

## Proposed Dispositions

### Disposition A: Persuasive
**Action:** Accept the change. Update the section 4 heading from "Card X" to "CardX" to match the correct project branding.

**Rationale:** The request is valid — "CardX" is the official project name. The original blocker was a FSH tooling limitation. If that limitation has been resolved (or can be worked around via title overrides in the IG publisher), there is no reason to keep the incorrect heading. The prior vote was 8-0-1 in favor of the intent, only deferring due to tooling constraints.

### Disposition B: Persuasive with Modification
**Action:** Accept the change, contingent on confirming the FSH tooling limitation has been resolved. If FSH still cannot handle "CardX" natively, apply the fix using an IG publisher title override or equivalent workaround.

**Rationale:** Same as Disposition A, but explicitly acknowledges the need to verify the FSH limitation status before committing to a specific implementation approach.

### Disposition C: Considered for Future Use (Re-defer)
**Action:** Defer again if the FSH limitation has not been resolved and no viable workaround exists.

**Rationale:** If the tooling still cannot produce "CardX" in headings, re-deferring avoids introducing manual workarounds that may break in future IG publisher updates. However, this would be the second deferral, and the re-opening suggests the work group expects resolution is now possible.

## Recommendation

**Disposition A: Persuasive** — The ticket was re-opened specifically because the work group believes the issue can now be addressed post-STU1. The branding correction is unambiguous ("CardX" is the official name), and the prior vote overwhelmingly supported the intent (8-0-1). The only prior objection was tooling-based, not substantive. Accept the change and correct the heading.
