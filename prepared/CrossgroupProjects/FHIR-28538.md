## Description

The links to extensions from the Profiles & Extensions page in the US Vital Signs with Qualifying Extensions IG direct users to the **Detailed Descriptions** page (e.g., `StructureDefinition-AssociatedPreconditionExt-definitions.html`) rather than the **overview page** of the StructureDefinition (e.g., `StructureDefinition-AssociatedPreconditionExt.html`).

This causes two problems:
1. **Wrong landing page:** Users arrive at the detailed element descriptions instead of the overview, with no way to navigate to the rest of the structure definition from that context.
2. **Incorrect external link markup:** The links are marked as external (`class="external-link" target="_blank"`) even though they point to profiles defined within the same implementation guide.

The reporter suggests linking to the overview page and removing the external link designation for internal IG profiles.

## Analysis

This is a straightforward IG publishing/tooling issue within the US Vital Signs with Qualifying Extensions IG. The extension links on the Profiles & Extensions page use the `-definitions.html` suffix instead of linking to the main StructureDefinition page, and they incorrectly use external link markup for internal IG content. The related ticket FHIR-28322 (marked Duplicate/Not Persuasive) reported similar tab/navigation dysfunction on extension pages in the same IG, suggesting a pattern of extension page navigation issues in this ballot cycle.

The fix would involve correcting the link targets in the IG's Profiles & Extensions page (or in the template/publisher that generates those links) to point to the overview pages and removing the `external-link` class and `target="_blank"` attributes for internal references.

This ticket has been in Triaged status since September 2020 with no assignee or resolution activity.

## Proposed Dispositions

### Disposition A: Persuasive (Recommended)
**Action:** Accept the change request and correct extension links on the Profiles & Extensions page to point to the StructureDefinition overview page (without the `-definitions.html` suffix) and remove external link markup for internal IG profiles.

**Rationale:** The reporter identifies a clear usability defect — users clicking extension links land on a page where they cannot navigate to the full StructureDefinition. The fix is well-defined (change link URLs and remove `external-link` class) and directly improves IG navigability. The related FHIR-28322 confirms that multiple reviewers encountered extension page navigation issues in this IG.

### Disposition B: Persuasive with Modification
**Action:** Accept the link target fix (overview page instead of definitions page) but defer the external link markup issue to a broader review of link generation in the IG publisher/template.

**Rationale:** The `-definitions.html` link target is a clear error that should be corrected in this IG. However, the external link markup issue may be a systemic concern across multiple IGs generated by the publisher tooling, and fixing it only in this IG may be insufficient. A broader template/publisher fix would address the root cause.

### Disposition C: Not Persuasive
**Action:** Close without action, noting that this applies to a historical ballot version (2020Sep) that is no longer current.

**Rationale:** The links reference the `2020Sep` ballot version of the IG. If subsequent versions of the IG have already corrected these links (either through template updates or manual fixes), the issue may be moot. However, this disposition risks leaving the defect unfixed if it persists in current versions.

## Recommendation

**Disposition A: Persuasive** — The issue is a clear, well-defined navigability defect with a straightforward fix. Extension links should point to the overview page, and internal IG links should not be marked as external. The fact that a related ticket (FHIR-28322) independently reported extension page navigation problems in the same IG reinforces that this is a real usability issue that should be addressed.
