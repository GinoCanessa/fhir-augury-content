## Summary

This ticket requests that the `military-service-combat-episode-mission-vss` Value Set canonical URL be re-rooted from `hl7.org/fhir` to `terminology.hl7.org` (THO — the HL7 Terminology server). The reporter argues that a UTG (Unified Terminology Governance) ticket should be created to formally register the Value Set in THO, aligning with HL7 terminology governance best practices.

The ticket was resolved as **"Considered for Future Use"** with the resolution noting that while FHIR R5 provides specific direction on canonical URL hosting in THO, FHIR R4 does not expressly state this requirement. The work groups agreed to consider such votes "non-persuasive" at that time.

## Details

**Background:** Ron G. Parker filed a series of nearly identical change requests (at least 10+ tickets, FHIR-33821 through FHIR-33831 and this one) against multiple FHIR IGs, all requesting that Value Set canonical URLs be migrated from `hl7.org/fhir` to `terminology.hl7.org` (THO). This ticket specifically targets the US Military Service IG's `military-service-combat-episode-mission-vss` Value Set.

**Core Issue:** The HL7 Terminology (THO) server at `terminology.hl7.org` is the designated authority for HL7-defined terminologies. Value Sets defined with canonical URLs rooted at `hl7.org/fhir` rather than `terminology.hl7.org` create ambiguity about governance authority and may complicate terminology management.

**Resolution Rationale:** The comment from seanmuir (2022-02-07) marks this as "Non-persuasive." The resolution description explains that while FHIR R5 provides direction on this, FHIR R4 does not expressly require Value Set canonical URLs to be in THO. Work groups collectively agreed to treat these requests as non-persuasive at the time.

## Proposed Dispositions

### Disposition A: Affirm "Considered for Future Use" (Non-Persuasive)

**Action:** Uphold the existing resolution. The request remains non-persuasive for FHIR R4-era IGs, and the Value Set canonical URL stays at `hl7.org/fhir`.

**Rationale:** The original resolution correctly identified that FHIR R4 does not mandate Value Set canonical URLs be hosted in THO. Changing canonical URLs is a breaking change for implementers who already reference the existing URL. Since R5 provides explicit direction on this topic, the appropriate time to address canonical URL migration is during an R5-aligned IG update cycle, not retroactively applied to R4 content.

### Disposition B: Accept and Create UTG Registration Ticket

**Action:** Accept the change request. Create a UTG ticket to register the `military-service-combat-episode-mission-vss` Value Set in THO and update the IG to use the new canonical URL rooted at `terminology.hl7.org`.

**Rationale:** THO is the designated authority for HL7-managed terminologies, and aligning canonical URLs with THO is a recognized best practice. Proactively migrating now — even ahead of R5 adoption — would establish a clean precedent and reduce future migration burden. The Value Set would retain its content; only its canonical URL base would change.

### Disposition C: Defer to Next IG Ballot Cycle

**Action:** Defer the change. When the US Military Service IG undergoes its next ballot cycle or R5 update, include canonical URL migration to THO as part of that work.

**Rationale:** This avoids a breaking change outside of a planned update cycle while acknowledging the validity of the request. Bundling the migration with a broader IG update reduces implementer disruption and aligns with the R5 direction. A tracking item could be created to ensure this is not forgotten.

## Recommendation

**Disposition A: Affirm "Considered for Future Use" (Non-Persuasive)**

The existing resolution is well-reasoned. FHIR R4 did not require THO-rooted canonical URLs, and retroactively changing a canonical URL constitutes a breaking change for existing implementers. The concern raised is valid in principle — THO is the right home for HL7-governed terminologies — but the timing and mechanism were not appropriate. The cross-work-group consensus at the time was to treat these batch-filed requests as non-persuasive, and that decision remains sound. If the IG is updated for R5, canonical URL migration should be addressed at that time as part of the normal update process.
