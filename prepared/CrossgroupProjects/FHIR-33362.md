## Description

The submitter questions whether the IG proposes to match specific military occupational or duty codes (e.g., Military Occupational Specialty — MOS) to NAICS (North American Industry Classification System) codes, and if so, how that mapping would work. The concern is that NAICS codes classify businesses by industry activity, which is fundamentally different from military occupational specialty codes that classify individual job roles and duties within the armed services.

**Existing Wording:** The value set description references "Concepts describing the kind of business that compensates the person for work (e.g., wages, training) or assigns work to a volunteer, as reported by the person. The kind of business is determined based on the primary business activity conducted by the company, organization, or individual (for those who are self-employed), or, for a military position, the self-reported branch of service."

**Proposed Wording:** The proposed wording is identical to the existing wording — the submitter did not provide an alternative, instead raising a question about the appropriateness and methodology of mapping military codes to NAICS codes.

## Analysis

### Core Issue
The ticket raises a valid conceptual concern: NAICS codes classify **industries** (what kind of business an organization conducts), while military occupational specialty (MOS) codes classify **individual job roles** within the military. These are fundamentally different taxonomies:

- **NAICS**: "Manufacturing," "Healthcare," "Defense" — describes the employer's industry.
- **MOS/duty codes**: "Infantry Officer," "Combat Medic," "Aircraft Mechanic" — describes the individual's specific role.

The value set description attempts to handle military positions by substituting "self-reported branch of service" for the NAICS industry code, which is a reasonable but limited approach. The submitter's question — whether there is a proposed mapping between MOS and NAICS — remains unanswered.

### History Context
This ticket was previously resolved as "Considered for Future Use" (11-0-1 vote, July 2022) but was re-opened post-STU1 publication in May 2023 by Lynn Laakso, indicating the workgroup wants to reconsider this for a future ballot cycle.

### Related Pattern
Several sibling tickets from the same STU1 ballot (FHIR-33351 through FHIR-33366, all by Keith Salzman) were similarly handled — some resolved as Not Persuasive, some as Considered for Future Use, and several re-opened post-STU1. This suggests a broader review of military occupational data representation is underway.

## Proposed Dispositions

### Disposition 1: Persuasive with Modification
**Action:** Clarify in the value set description that NAICS codes apply to civilian employment contexts and that military positions use branch of service (not NAICS) as the industry classifier. Add a note in the IG acknowledging that MOS/duty codes represent a different classification dimension (occupational role vs. industry) and that future versions may address MOS-to-occupation mappings using CDC-modified Census occupation codes already referenced in the FHIR occupational data resources.

**Rationale:** The concern is valid — the current text is ambiguous about how military positions relate to NAICS codes. Clarifying that they don't map directly, and that branch of service serves as the military equivalent of industry classification, resolves the ambiguity without requiring structural changes. Referencing CDC-modified Census codes as the path for future MOS mapping provides a concrete direction.

### Disposition 2: Not Persuasive
**Action:** No change. The existing wording already addresses military positions by specifying "for a military position, the self-reported branch of service," which effectively answers the question: NAICS codes are not applied to military positions.

**Rationale:** The value set description already carves out military positions from NAICS classification by specifying that branch of service is used instead. The submitter's question is answered by the existing text, even if indirectly. The prior 11-0-1 vote for "Considered for Future Use" suggests the workgroup already considered this and found the current approach adequate for STU1.

### Disposition 3: Considered for Future Use
**Action:** Defer to a future ballot cycle. Document the need for a comprehensive military occupation classification strategy that addresses the relationship between NAICS industry codes, MOS/duty codes, and CDC-modified Census occupation codes as a future work item.

**Rationale:** Mapping military occupational codes to civilian classification systems is a complex cross-domain challenge that requires input from DoD Occupational Medicine specialists (as noted in related ticket FHIR-33356) and alignment with existing systems like DOEHRS. This work is better suited to a dedicated future effort rather than a quick fix in the current cycle.

---

## Recommendation

**Disposition 1: Persuasive with Modification** is recommended.

The submitter raises a legitimate concern about the conceptual mismatch between NAICS industry codes and military occupational codes. The current text is ambiguous enough to cause confusion. A targeted clarification — stating that military positions use branch of service rather than NAICS, and acknowledging that MOS-level occupation mapping is a future goal — addresses the concern without requiring major structural changes. This approach respects the prior 11-0-1 vote while providing the clarification the re-opening was intended to address. It also aligns with the direction suggested by the FHIR core spec's occupational data resource, which already references CDC-modified Census occupation codes for military occupations.
