## Description

The ticket reports that the NCPDP `<FillStatus>` element does not represent whether a dispense was a partial fill. The existing wording references "NCPDP Data Element for Partial Fill Indicator," indicating the IG's mapping table uses `<FillStatus>` for the partial fill concept, but `<FillStatus>` is actually a numeric field representing which fill in a sequence of fills a dispense is (i.e., the fill number), not whether the fill was partial or complete.

This is a companion issue to [FHIR-29471](https://jira.hl7.org/browse/FHIR-29471), which addresses the same `<FillStatus>` element but from the refill number perspective. Together, these tickets highlight that `<FillStatus>` was incorrectly mapped to two different concepts (refill number and partial fill indicator) in the PDMP NCPDP mapping table, when it only represents one (the fill sequence number).

## Analysis

The core issue is a data mapping error in the PDMP section of the US Medication FHIR IG. The NCPDP `<FillStatus>` element is a numeric field indicating which fill in a sequence (1st fill, 2nd fill, etc.) a dispense represents. The IG's mapping table incorrectly maps this element to the concept of a "Partial Fill Indicator" — whether a dispensed quantity was a partial fill versus a complete fill. These are semantically distinct concepts:

1. **Fill sequence number** (what `<FillStatus>` actually is): e.g., "this is fill #3 of 5 authorized fills"
2. **Partial fill indicator** (what the IG says it represents): e.g., "this dispense was a partial fill — only 15 of 30 tablets were dispensed"

The IG needs to either:
- Identify the correct NCPDP element for partial fill status and map it appropriately, or
- Define a FHIR extension or use an existing element (e.g., `MedicationDispense.type` with an appropriate code) to convey partial fill information.

This issue is part of a larger batch of NCPDP mapping corrections imported from ballot comments. The entire batch was deferred when the IG was withdrawn, then reopened in November 2023 under PSS-2225 PDMP on FHIR.

## Proposed Dispositions

### Disposition A: Persuasive — Correct the mapping and add partial fill guidance

**Resolution:** Accept the change request. Remove the incorrect mapping of `<FillStatus>` to "Partial Fill Indicator" in the NCPDP mapping table. Identify the correct NCPDP element or mechanism for conveying partial fill status (e.g., NCPDP Partial Fill Code or Dispensing Status) and add a proper mapping to the appropriate FHIR element or a new extension on `MedicationDispense`.

**Rationale:** The current mapping is factually incorrect — `<FillStatus>` represents fill sequence number, not partial fill status. Correcting this is essential for interoperability between NCPDP-based pharmacy systems and FHIR-based PDMP consumers. This should be addressed alongside the companion issue FHIR-29471.

### Disposition B: Persuasive with Modification — Correct the mapping, defer partial fill modeling

**Resolution:** Accept the core observation that `<FillStatus>` does not represent partial fill status and correct the mapping table. However, defer the question of how to represent partial fill information in FHIR to a future cycle, as it may require coordination with NCPDP on the correct source element and with the broader MedicationDispense resource modeling.

**Rationale:** The mapping error should be corrected immediately. However, adding a new partial fill indicator mapping requires identifying the correct NCPDP source element and determining the best FHIR representation (existing element vs. extension), which may need additional analysis and cross-SDO coordination.

### Disposition C: Not Persuasive — Defer until IG scope is clarified

**Resolution:** Acknowledge the observation but defer action. The PDMP on FHIR IG is being re-scoped under PSS-2225. Until the IG's NCPDP mapping approach is fully redesigned (addressing FHIR-29471, FHIR-29473, and the broader batch of mapping issues), individual corrections may be premature.

**Rationale:** This is one of many NCPDP mapping issues in the batch. A comprehensive review of all NCPDP-to-FHIR mappings would be more efficient than piecemeal fixes, and PSS-2225 may take a different architectural approach to these mappings.

## Recommendation

**Disposition A: Persuasive — Correct the mapping and add partial fill guidance.**

The mapping is unambiguously incorrect. `<FillStatus>` is a fill sequence number, not a partial fill indicator. This error could cause real interoperability failures where systems misinterpret dispense data. Since the IG has been reopened under PSS-2225 and the full batch of NCPDP mapping issues is being revisited, now is the appropriate time to make this correction. The fix should be coordinated with FHIR-29471 (which addresses the same element from the refill number angle) to ensure `<FillStatus>` is correctly mapped once with a consistent definition.
