## Description

The submitter identifies that the PDMP IG uses wording that conflates "prescriptions" with "dispenses." PDMPs are fundamentally about tracking controlled substance **dispenses**, not prescriptions. The ticket requests a wording correction in an example query description.

**Existing Wording:**
> The above API will fetch all MedicationDispense resources for Patient with a given name of "sherlock" and family name of "holmes" with a birthdate of "1954-01-06" **with a prescription from** January 6, 1954 to December 01, 2019

**Proposed Wording:**
> The above API will fetch all MedicationDispense resources for Patient with a given name of "sherlock" and family name of "holmes" with a birthdate of "1954-01-06" **for medications dispensed from** January 6, 1954 to December 01, 2019

## Resolution Description

> Rewrite of IG addresses references to prescriptions vs dispenses.

## Analysis

This is a straightforward textual correction. The submitter correctly identifies that PDMPs track **dispenses**, not prescriptions, and the example query description in the IG incorrectly refers to "a prescription from [date range]" when the query is actually fetching `MedicationDispense` resources for a dispense date range. The proposed change from "with a prescription from" to "for medications dispensed from" accurately reflects the semantics of the query and the purpose of PDMPs.

The existing resolution ("Persuasive with Modification") notes that the "rewrite of IG addresses references to prescriptions vs dispenses," indicating this concern has already been acknowledged. The ticket was originally resolved as "Considered for Future Use" in January 2023 (when the IG was withdrawn) and then reopened in November 2023 by PSS-2225 when the PDMP on FHIR project was resumed.

This ticket is closely related to FHIR-29453 (same submitter, same theme, same resolution) and should be considered alongside that ticket and FHIR-29531, which addresses a related structural issue with the date range parameter.

## Proposed Dispositions

### Disposition A: Accept as Persuasive with Modification (Confirm Existing Resolution)
**Action:** Confirm the existing "Persuasive with Modification" resolution. Apply the proposed wording change, replacing "with a prescription from" with "for medications dispensed from" in the API example description, as part of the broader IG rewrite that addresses prescription vs. dispense terminology throughout.
**Rationale:** The proposed change is factually correct â€” PDMPs track dispenses, not prescriptions â€” and the resolution description already indicates the IG rewrite addresses this. Confirming this resolution allows the change to proceed as part of the current IG update cycle.

### Disposition B: Accept as Persuasive
**Action:** Apply the submitter's proposed wording change exactly as written: change "with a prescription from January 6, 1954 to December 01, 2019" to "for medications dispensed from January 6, 1954 to December 01, 2019."
**Rationale:** The proposed wording is precise and correct. Accepting as-is avoids scope creep and clearly addresses the specific text identified by the submitter.

### Disposition C: Merge with FHIR-29453
**Action:** Merge this ticket into FHIR-29453 as a duplicate/companion issue, since both address the same underlying problem (misuse of "prescription" terminology in a dispense-focused IG) from the same reporter with the same resolution.
**Rationale:** Both tickets correct the same conceptual error in different sections of the IG. Consolidating them would simplify tracking and ensure the IG rewrite addresses all instances consistently.
