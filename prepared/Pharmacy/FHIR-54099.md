# FHIR-54099: Need for Introductory Explanation of MedicationStatement.adherence

## Summary

**Status:** Triaged | **Type:** Technical Correction | **Priority:** Medium
**Work Group:** Pharmacy | **Specification:** FHIR Core (FHIR)
**Reporter:** Ricardo Quintano | **Assignee:** Unassigned
**Created:** 2026-01-13 | **Updated:** 2026-01-17
**URL:** https://jira.hl7.org/browse/FHIR-54099

The submitter requests adding an introductory explanation of `MedicationStatement.adherence` to the Scope and Usage section of the [MedicationStatement resource](https://hl7.org/fhir/6.0.0-ballot4/medicationstatement.html#scope). Currently, the section includes a paragraph about the adherence element that states it is "specific to that instance of MedicationStatement" and describes tracking adherence over time via `effectivePeriod`. However, the paragraph lacks a preceding sentence that explains **what** the adherence element is and **why** it exists. The submitter argues that an introductory explanation would help readers understand the purpose and intended use of the element before diving into the specifics.

## Analysis

This is a straightforward editorial request for a minor documentation enhancement. The existing scope-and-usage paragraph about `MedicationStatement.adherence` currently reads:

> "MedicationStatement includes an adherence element. Note that this adherence is specific to that instance of MedicationStatement. If MedicationStatement.adherence is being tracked over time, then instances of MedicationStatement would report adherence for the interval noted in effectivePeriod."

The submitter correctly observes that this paragraph jumps into implementation details without first explaining the purpose of the adherence element. A brief introductory sentence (e.g., explaining that adherence captures whether a patient is/was taking the medication as intended, and is represented as a coded value from a defined value set) would improve readability and comprehension.

This request aligns well with the broader theme of the R6 ballot cycle, where multiple ballot comments (FHIR-54315, FHIR-54566) are requesting improved narrative clarity in MedicationStatement. Addressing these tickets together would be efficient.

## Proposed Dispositions

### Disposition A: Persuasive (Recommended ✅)

**Resolution:** Accept the request and add a brief introductory sentence explaining the purpose and nature of `MedicationStatement.adherence` before the existing paragraph.

**Rationale:** The request is well-founded. The adherence element is a backbone element with a coded value that captures whether the patient is taking their medication as intended, and readers encountering it in the scope-and-usage section benefit from a brief statement of purpose before the implementation-level details. This is a low-risk, high-value editorial improvement. Suggested text (to precede the existing paragraph):

> "The adherence element on MedicationStatement captures whether the patient is taking the medication as intended or prescribed. It is represented as a coded value (e.g., taking, not-taking, on-hold) and may optionally include a reason."

This can be coordinated with the closely related tickets FHIR-54315 and FHIR-54566, which also propose narrative improvements to the same resource.

**Vote:** Approve the addition of introductory text for `MedicationStatement.adherence` in the Scope and Usage section.

### Disposition B: Persuasive with Modification

**Resolution:** Accept the intent of the request but expand the scope of the editorial change to cover the adherence element more comprehensively, including a brief description of the value set and relationship to `effectivePeriod`.

**Rationale:** Rather than adding only a single introductory sentence, this approach would restructure the adherence paragraph to include: (1) what adherence represents, (2) the coded values available, (3) the per-instance nature, and (4) how it relates to time tracking via `effectivePeriod`. This provides a more complete explanation but may overlap with work needed for FHIR-54315 and FHIR-54566.

**Vote:** Approve an expanded rewrite of the adherence paragraph in the Scope and Usage section, coordinated with FHIR-54315 and FHIR-54566.

### Disposition C: Not Persuasive

**Resolution:** Reject the request as unnecessary.

**Rationale:** The existing paragraph already opens with "MedicationStatement includes an adherence element," which is itself an introduction. The element definition and comments in the StructureDefinition provide the detailed explanation of purpose. The scope-and-usage section is not intended to replicate element-level documentation, and the current text is sufficient for readers who have reviewed the element definitions.

**Vote:** No change to the specification.

## Recommendation

**Disposition A: Persuasive** is recommended. The request is a reasonable, low-effort editorial improvement that enhances readability of the adherence section. The existing paragraph assumes the reader already knows what adherence means in this context, which is not a safe assumption for all implementers. Adding one or two introductory sentences is a minimal change with clear benefit. This work should be coordinated with the related tickets FHIR-54315 and FHIR-54566 to avoid redundant edits to the same section.
