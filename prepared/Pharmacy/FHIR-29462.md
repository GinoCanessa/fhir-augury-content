### Description

The submitter identifies an incorrect NCPDP element reference in the PDMP Request Data Element mapping table. The current mapping states that the "Requestor Role" maps from the NCPDP `<Qualifier>` element to `PractitionerRole.code`. The submitter asserts that this should instead reference the **Qualifier attribute** of the `<Header>/<From>` element — i.e., the element path is wrong and should be corrected to accurately identify where the requestor role data is sourced from in the NCPDP SCRIPT message.

**Existing Wording (from ticket):**

## 4. Proposed Dispositions

### Disposition A: Applied — Correct the Element Reference

**Action:** Update the NCPDP Data Element column in the PDMP Request Data Element mapping table to change the `<Qualifier>` element reference to the Qualifier attribute of the `<Header>/<From>` element (e.g., `<Header>/<From>/@Qualifier`).

**Rationale:** The submitter (an NCPDP domain expert from the original ballot) has identified that the standalone `<Qualifier>` element reference is inaccurate. The Requestor Role data is actually carried in the Qualifier attribute of the `<Header>/<From>` element in NCPDP SCRIPT messages. This is a factual correction that improves the accuracy of the NCPDP-to-FHIR mapping table. The FHIR target (`PractitionerRole.code`) remains unchanged. This aligns with the pattern of similar corrections across the batch of PDMP-old-issues tickets (e.g., FHIR-29461's request for full element paths, FHIR-29464's correction of non-existent elements).

**Impact:** Low. This is a documentation/mapping table correction only. No profile or structural changes required.

### Disposition B: Considered — Defer for Broader NCPDP Mapping Review

**Action:** Acknowledge the issue but defer resolution to be addressed as part of a comprehensive review of all NCPDP-to-FHIR mappings in the PDMP IG.

**Rationale:** There are approximately 20+ PDMP-old-issues tickets (FHIR-29451 through FHIR-29546) that all address NCPDP mapping problems. Addressing them individually may lead to inconsistent corrections. A holistic review of all NCPDP SCRIPT element references in the mapping table — ideally involving NCPDP domain expertise — would ensure consistency and completeness across the entire mapping set. The PSS-2225 PDMP on FHIR project restart provides a natural opportunity for this.

**Impact:** None immediately. Risk of continued inaccuracy in the mapping table until the broader review is completed.

### Disposition C: Not Persuasive — Insufficient Detail to Confirm Correction

**Action:** Reject the change request due to lack of specificity in the proposed correction.

**Rationale:** While the submitter identifies that `<Qualifier>` is not the correct element, the ticket does not provide the exact corrected wording or specify the precise NCPDP SCRIPT 2017071 element path to use as a replacement. Without a concrete proposed replacement, the workgroup cannot verify the accuracy of the suggested change. The ticket could be returned to the submitter requesting the exact corrected mapping text.

**Impact:** None. The mapping table remains as-is.

## 5. Recommendation

**Disposition A: Applied — Correct the Element Reference**

This is the strongest disposition because:

1. **The correction is factually clear.** The submitter explicitly states that `<Qualifier>` refers to the Qualifier attribute of `<Header>/<From>`, which is a specific, verifiable claim about NCPDP SCRIPT message structure.
2. **It aligns with the batch pattern.** Sister tickets (FHIR-29461, FHIR-29464, FHIR-29466) all involve correcting NCPDP element references to their proper paths, and this ticket follows the same pattern.
3. **The IG has been reactivated.** PSS-2225 PDMP on FHIR project explicitly reopened these issues for resolution, signaling intent to address them.
4. **Low risk.** This is a mapping table text correction with no structural impact on FHIR profiles or conformance resources.
5. **Highest priority.** The ticket is marked Highest priority, and the original ballot comment was submitted by an NCPDP domain expert.

The workgroup should apply the correction, updating the NCPDP Data Element reference for Requestor Role to specify the Qualifier attribute of `<Header>/<From>` (i.e., `<Header>/<From>/@Qualifier` or equivalent notation per IG conventions), while keeping the FHIR mapping target as `PractitionerRole.code`.
