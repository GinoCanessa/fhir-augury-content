## Summary

This ticket requests evaluation of value sets bound to the MedicationStatement resource for potential migration from the FHIR Core specification to HL7 Terminology (THO). Two specific value sets are identified as candidates:

1. **medicationrequest-category** (CodeSystem: `MedicationRequestCategoryCodes`) — bound to `MedicationStatement.category`
2. **medication-statement-adherence** (CodeSystem: `MedicationStatementAdherenceCodes`) — bound to `MedicationStatement.adherence.code`

The ticket notes that migration is not required but should be considered if the value set definition is likely to change, or if the value sets might be useful across CDA, V2, or other specifications. This is one of a batch of similar tickets filed by Lloyd McKenzie across all Medication-related resources (FHIR-55332 through FHIR-55335).

## Details

### Description

The MedicationStatement references several value sets that are potential candidates for migration to THO. While value set migration is not **required**, if it's realistic for the value set definition to need to change in the coming years, or the value sets might be useful in CDA, V2 or other specs, migration should be considered.

**Specifics:**

- **CodeSystem:** `medicationrequest-category` (`MedicationRequestCategoryCodes`) — referenced by ValueSet `medicationrequest-category`, bound to element `MedicationStatement.category`
- **CodeSystem:** `medication-statement-adherence` (`MedicationStatementAdherenceCodes`) — referenced by ValueSet `medication-statement-adherence`, bound to element `MedicationStatement.adherence.code`

## Proposed Dispositions

### Disposition A: Accept — Migrate Both Value Sets to THO

**Resolution:** Applied  
**Action:** Migrate both `medicationrequest-category` and `medication-statement-adherence` CodeSystems and their associated ValueSets from FHIR Core to HL7 Terminology (THO).

**Rationale:**
- The `medicationrequest-category` CodeSystem is already published in THO (as of THO 6.0.2) but remains in draft status. Completing the migration and maturing the code system would resolve implementer confusion (as evidenced by the Zulip committers thread).
- The `medication-statement-adherence` CodeSystem is actively used across FHIR versions (R4 preadoption of R5 elements) and causes cross-version validation failures when not available in THO. Migrating it would improve cross-version interoperability.
- Both value sets are used across multiple medication resources and could be useful in CDA and V2 specifications.
- The `medicationrequest-category` migration should be coordinated with FHIR-55335 (MedicationRequest).

### Disposition B: Accept Partial — Migrate medication-statement-adherence Only

**Resolution:** Applied (partial)  
**Action:** Migrate only the `medication-statement-adherence` CodeSystem/ValueSet to THO. Defer `medicationrequest-category` to be handled as part of FHIR-55335 (MedicationRequest).

**Rationale:**
- The `medication-statement-adherence` CodeSystem has the strongest case for migration: it is new in R5, actively preadopted into R4 IGs, and causes cross-version validation errors because it is not available as a standalone terminology resource.
- The `medicationrequest-category` CodeSystem is already partially in THO (draft status) and is shared with MedicationRequest — its migration decision may be better handled under FHIR-55335 to avoid fragmented decision-making across sibling tickets.

### Disposition C: Decline — No Migration Needed at This Time

**Resolution:** Not Persuasive with Modification  
**Action:** Decline migration. Add a comment noting that both value sets are stable in their current form and do not require THO migration at this time.

**Rationale:**
- Value set migration to THO is explicitly not required per the ticket description.
- The `medicationrequest-category` value set is already available in THO (even if draft); migrating the FHIR Core definition may create unnecessary duplication or versioning confusion.
- The `medication-statement-adherence` value set is relatively new (R5) and may not yet be stable enough for THO — further implementer feedback may be needed before committing to a THO-managed lifecycle.
- The Pharmacy WG could revisit this when there is a concrete need for cross-spec (CDA/V2) use of these code systems.

## Recommendation

**Disposition A: Accept — Migrate Both Value Sets to THO**

The evidence supports migrating both value sets. The `medicationrequest-category` CodeSystem is already in THO but languishing in draft status, causing implementer confusion. The `medication-statement-adherence` CodeSystem has an even stronger case: it is actively being preadopted across FHIR versions, causing real validation failures when implementers try to use it in R4 IGs. Both code systems serve cross-resource and potentially cross-specification purposes. Coordinating this migration with the sibling ticket FHIR-55335 for the shared `medicationrequest-category` value set would ensure consistency. The batch nature of these tickets (FHIR-55332 through FHIR-55337) suggests a systematic approach to THO migration is already underway, and accepting this ticket aligns with that initiative.
