## Description

The submitter notes that current PDMP integration methods allow **both prescribers and pharmacies** to query PDMPs. The ticket asks whether the IG intends to cover both user types as a "PDMP Requestor." If so, the NCPDP mappings in the "PDMP Data Elements and Mappings" section need to be updated to reflect that queries use **different data elements** depending on whether the requestor is a prescriber or a pharmacist.

This is a structural mapping concern: NCPDP SCRIPT represents prescriber and pharmacist data paths differently (e.g., `<Prescriber>/<Name>/<LastName>` vs. `<Pharmacy>/<Pharmacist>/<LastName>`), and the current mappings do not distinguish between these two requestor types.

## Analysis

### Core Issue
The PDMP Data Elements and Mappings section in the IG does not account for the fact that NCPDP SCRIPT uses **different XML element paths** depending on whether the PDMP query is initiated by a prescriber or a pharmacist. The current mappings appear to assume a single requestor type, which is insufficient.

### Key Considerations
1. **IG Scope Clarification**: The IG must explicitly state whether it covers both prescriber and pharmacist requestor types. Related tickets FHIR-29523 and FHIR-29524 independently raised the same concern about pharmacy access.
2. **Mapping Differentiation**: If both requestor types are in scope, mappings need to show distinct NCPDP element paths for each type (as detailed in FHIR-29461).
3. **Specification Evolution**: The original IG (US Medication FHIR IG) was withdrawn and has been superseded by the US Prescription Drug Monitoring Program (PDMP) FHIR IG (per PSS-2225). FHIR-43140 on the newer IG raises similar mapping concerns, suggesting this issue may need to be addressed in the new IG's context.
4. **Cluster of Related Tickets**: FHIR-29459 through FHIR-29463 form a cohesive cluster of NCPDP mapping issues from the same reporter (Daniel Zhang). They should ideally be reviewed together.
5. **Previously Resolved and Reopened**: This ticket was resolved as "Considered for Future Use" in January 2023 when the IG was withdrawn, then reopened in November 2023 when PDMP work resumed under PSS-2225.

## Proposed Dispositions

### Disposition A: Persuasive (Accept and Apply)
**Action**: Update the PDMP Data Elements and Mappings section to include separate NCPDP mapping paths for prescriber-initiated and pharmacist-initiated queries. Explicitly state in the IG that both requestor types are supported.

**Rationale**: The concern is technically valid — NCPDP SCRIPT uses different XML structures for prescriber vs. pharmacist data. The IG must accurately reflect these differences to be implementable. This aligns with related tickets FHIR-29461, FHIR-29462, and FHIR-29463, which provide specific mapping corrections. Addressing this as a group would be efficient.

**Risk**: Moderate effort to create and validate dual-path mappings, but necessary for correctness.

### Disposition B: Persuasive with Modification (Accept with Broader Scope)
**Action**: Accept the concern, but address it in the context of the **new US PDMP FHIR IG** (per PSS-2225) rather than the withdrawn US Medication FHIR IG. Ensure the new IG's mappings section differentiates requestor types, and coordinate resolution with FHIR-29461, FHIR-29462, FHIR-29463, and FHIR-43140.

**Rationale**: The original IG was withdrawn and the PDMP work has moved to a new IG under PSS-2225. Applying changes to the withdrawn IG is not meaningful. The concern should carry forward and be addressed in the new IG where FHIR-43140 already raises similar requestor-type differentiation issues.

**Risk**: Requires coordination across multiple tickets and the new IG timeline.

### Disposition C: Considered - No Action Required
**Action**: Close this ticket as the concern is already being addressed by work on the new US PDMP FHIR IG (PSS-2225), specifically through FHIR-43140, which covers requestor identification and role requirements in the updated mappings context.

**Rationale**: The ticket was filed against the withdrawn US Medication FHIR IG. The new PDMP IG under PSS-2225 is actively addressing mapping concerns including requestor type differentiation. Keeping this ticket open creates duplicate tracking. FHIR-43140 and the new IG's development should subsume this concern.

**Risk**: If FHIR-43140 does not explicitly address the NCPDP element path differentiation by requestor type, this specific concern could be lost.

## Recommendation

**Disposition B: Persuasive with Modification** is recommended. The technical concern is valid and important for implementability — NCPDP SCRIPT genuinely uses different element paths for prescriber vs. pharmacist queries, and the mappings must reflect this. However, since the original IG has been withdrawn and work has moved to the new US PDMP FHIR IG (PSS-2225), the fix should be applied there. This ticket should be resolved with a note that the mapping corrections (including the related cluster of FHIR-29461, FHIR-29462, FHIR-29463) are to be incorporated into the new IG's mappings section, coordinated with FHIR-43140's broader mapping requirements.
