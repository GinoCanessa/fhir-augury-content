## Summary

This change request identifies an incorrect NCPDP SCRIPT element mapping in the US PDMP FHIR Implementation Guide for the "Request Prescription Date Range." The submitter states that when mapping from NCPDP SCRIPT 2017071, the correct elements for the prescription date range should be `<RequestedDates>/<StartDate>` and `<RequestedDates>/<EndDate>`, rather than the generic "NCPDP Data Elements for Request Prescription Date Range" currently referenced in the IG.

The ticket was originally imported from a spreadsheet in October 2020 and was previously resolved as "Considered for Future Use" in January 2023 (vote: 4-0-0, John Hatem / Tim McNeil) because the IG had been withdrawn. It was reopened in November 2023 by smrobertson as part of [PSS-2225 PDMP on FHIR](https://confluence.hl7.org/display/PHAR/US+Prescription+Drug+Monitoring+Program+%28PDMP%29+FHIR+Implementation+Guide), indicating the IG work has resumed.

## Details

### Existing Wording (from ticket)

> NCPDP Data Elements for Request Prescription Date Range

### Proposed Correction

The IG should specify the precise NCPDP SCRIPT 2017071 element paths:

- `<RequestedDates>/<StartDate>` — for the start of the requested date range
- `<RequestedDates>/<EndDate>` — for the end of the requested date range

This correction aligns with a broader pattern of issues in this IG where NCPDP SCRIPT element references lack specificity. Multiple related tickets (see below) request more granular, path-based element references rather than generic descriptions of NCPDP data elements.

## Proposed Dispositions

### Disposition A: Accept — Persuasive (Recommended)

**Resolution:** Persuasive

**Action:** Update the NCPDP-to-FHIR mapping table in the US PDMP FHIR IG to replace the generic "NCPDP Data Elements for Request Prescription Date Range" with the specific NCPDP SCRIPT 2017071 element paths `<RequestedDates>/<StartDate>` and `<RequestedDates>/<EndDate>`.

**Rationale:** The submitter's correction is technically accurate — NCPDP SCRIPT 2017071 defines the `<RequestedDates>` composite element with `<StartDate>` and `<EndDate>` child elements for conveying the requested date range in a PDMP query. Providing explicit element paths rather than generic descriptions improves implementability and reduces ambiguity. This aligns with the pattern established by related tickets (FHIR-29464, FHIR-29463, FHIR-29473) that all request more precise NCPDP element references. The ticket was previously deferred only because the IG was withdrawn, not due to any substantive objection, and the IG has now been reactivated under PSS-2225.

### Disposition B: Accept — Persuasive with Modification

**Resolution:** Persuasive with Modification

**Action:** Update the mapping to reference `<RequestedDates>/<StartDate>` and `<RequestedDates>/<EndDate>`, and additionally review all NCPDP SCRIPT element references in the IG holistically (in coordination with FHIR-29473, FHIR-29464, FHIR-29463, and related tickets) to ensure consistent use of full element paths throughout. This could be done as a batch editorial pass across all PDMP-old-issues tickets.

**Rationale:** While the individual correction is valid, addressing it in isolation risks inconsistency. A coordinated update across all NCPDP mapping references would be more efficient and produce a more coherent IG. The large cluster of related PDMP-old-issues tickets suggests a systematic review is warranted.

### Disposition C: Decline — Not Persuasive

**Resolution:** Not Persuasive

**Action:** No change. Defer to a future ballot cycle or IG revision.

**Rationale:** The IG was previously withdrawn and has been reactivated under PSS-2225, but the PDMP IG content may undergo substantial restructuring. Applying granular mapping fixes to content that may be significantly reworked could result in wasted effort. The workgroup may prefer to address all NCPDP mapping issues as part of the broader IG revision rather than piecemeal.

## Recommendation

**Disposition A: Accept — Persuasive** is recommended.

The correction is factually accurate, technically straightforward, and improves the quality of the NCPDP-to-FHIR mapping guidance. It was previously deferred only due to IG withdrawal (not substantive disagreement), and the IG has since been reactivated. The prior 4-0-0 vote to defer suggests no opposition to the substance of the change. Accepting this correction — along with the cluster of related PDMP-old-issues tickets — would bring the IG's NCPDP element references up to the specificity implementers need.
