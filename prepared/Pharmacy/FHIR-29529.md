## Description

The submitter requests that the NCPDP SCRIPT mappings in the PDMP Request and PDMP Response sections of the US Medication FHIR IG be updated to reference NCPDP SCRIPT 2017071 — the current named version of the standard. The existing mappings reference an older version of the SCRIPT standard.

## Proposed Dispositions

### Disposition A: Persuasive

**Action:** Accept the request and update the NCPDP SCRIPT mapping tables in the PDMP Request and PDMP Response sections to reference NCPDP SCRIPT 2017071.

**Rationale:** Federal regulation now mandates the use of NCPDP SCRIPT 2017071 for electronic prescribing transactions. The closely related ticket FHIR-29528 ("Correct SCRIPT version") has already been resolved as Persuasive, establishing precedent for this update. Multiple related tickets (FHIR-29460, FHIR-29458) further confirm the community consensus that 2017071 should be the referenced version. Updating the mappings aligns the IG with current regulatory requirements and industry practice.

**Impact:** Moderate — requires reviewing and updating the SCRIPT mapping columns/tables in the PDMP Request and PDMP Response pages of the US Medication FHIR IG to ensure all element references correspond to NCPDP SCRIPT 2017071 data elements.

### Disposition B: Persuasive with Modification

**Action:** Accept the request but broaden the scope: update mappings to NCPDP SCRIPT 2017071 and consolidate this work with the overlapping requests in FHIR-29460 (mapping header naming) and FHIR-29528 (version correction). Mark those tickets as resolved in coordination.

**Rationale:** FHIR-29529, FHIR-29460, and FHIR-29528 all address the same underlying issue — the IG references an outdated SCRIPT version. Consolidating these ensures the mapping update is done holistically (headers, body text, and mapping content) rather than piecemeal. This avoids duplicate work and ensures consistency.

**Impact:** Moderate — same scope as Disposition A but explicitly coordinates with FHIR-29460 and FHIR-29528. Editors should process these together.

### Disposition C: Not Persuasive (Defer)

**Action:** Defer action until the PDMP on FHIR IG (PSS-2225) has progressed further in its development cycle.

**Rationale:** While the request is technically valid, the IG was previously withdrawn and only recently re-authorized under PSS-2225. The mapping tables may be substantially restructured as part of the new IG development, making targeted mapping-version fixes premature. Resources may be better spent on overall IG architecture decisions first, such as whether SCRIPT/PIMX mappings should be retained at all (see FHIR-29530).

**Impact:** Low — no immediate changes; the ticket remains open for future ballot cycles.

---

## Recommendation

**Disposition B: Persuasive with Modification** is recommended.

The request is valid — NCPDP SCRIPT 2017071 is the current regulatory standard and the mappings must reflect it. However, three closely related tickets (FHIR-29460, FHIR-29528, and this ticket) all address the same version-update concern from different angles. Coordinating these as a single editorial change ensures consistency, avoids redundant work, and produces a cleaner result. FHIR-29528 has already been resolved as Persuasive, confirming the workgroup's alignment with this direction.
