## Description Summary

The submitter identifies a gap in the PDMP NCPDP data element mappings for **requesting facility identification**. When a prescriber performs a PDMP query, there is no clear mapping path for the facility's DEA Number and NPI:

- The **\<Prescriber\>/\<Identification\>** elements carry the prescriber's DEA and NPI — not the facility's.
- The **\<Pharmacy\>/\<Identification\>** elements are inappropriate because the prescriber is not querying from a pharmacy.
- The NCPDP SCRIPT 2017071 standard **does** support a **\<Facility\>/\<Identification\>** element that could carry the requesting facility's identifiers.

The request is to update the NCPDP data element mappings in the IG to include the \<Facility\>/\<Identification\> path for transmitting requesting facility information (DEA Number, NPI) during PDMP queries.

## Context & Analysis

This ticket belongs to a batch of ~20 ballot comments imported from a spreadsheet in October 2020, all filed by Daniel Zhang against the PDMP section of the US Medication FHIR IG. The batch collectively addresses inaccuracies and gaps in the NCPDP SCRIPT data element mappings.

**Key history:**
1. **2020-10**: Imported as part of ballot feedback.
2. **2023-01**: Resolved as "Considered for Future Use" (4-0-0 vote) because the PDMP IG had been withdrawn.
3. **2023-11**: Reopened when the PDMP on FHIR project (PSS-2225) was restarted under the Pharmacy WG.

**Technical merit:** The issue is technically valid. NCPDP SCRIPT 2017071 does define a \<Facility\> element with \<Identification\> sub-elements. When a prescriber queries a PDMP from a facility (e.g., a hospital or clinic), neither the \<Prescriber\>/\<Identification\> elements (which identify the prescriber personally) nor the \<Pharmacy\>/\<Identification\> elements (which identify a dispensing pharmacy) are appropriate for conveying the requesting facility's DEA Number or NPI. The \<Facility\>/\<Identification\> element is the correct NCPDP mapping target.

**Impact:** This affects FHIR-to-NCPDP interoperability for PDMP query requests originating from non-pharmacy facilities. Without correct mappings, implementers may misidentify the requesting organization or omit facility identification entirely.

## Proposed Dispositions

### Disposition A: Persuasive
**Action:** Update the NCPDP data element mappings in the PDMP section of the IG to include \<Facility\>/\<Identification\>/\<DEANumber\> and \<Facility\>/\<Identification\>/\<NPI\> as the mapping paths for requesting facility identification when the PDMP query originates from a prescriber at a non-pharmacy facility.

**Rationale:** The submitter correctly identifies a gap in the current mappings. The NCPDP SCRIPT 2017071 standard supports the \<Facility\>/\<Identification\> element, and this is the appropriate path for facility-level identifiers. This should be addressed alongside the related cohort of NCPDP mapping corrections (FHIR-29461, FHIR-29464, etc.) for consistency.

### Disposition B: Persuasive with Modification
**Action:** Accept the intent but defer specific mapping details to the active PDMP on FHIR IG project (PSS-2225). The project team should address this as part of a comprehensive review of all NCPDP SCRIPT 2017071 mappings, incorporating this ticket along with the related batch (FHIR-29459, FHIR-29461, FHIR-29464, FHIR-29466, FHIR-29473).

**Rationale:** Since the PDMP on FHIR IG is being actively developed under PSS-2225, and there is an entire cohort of related NCPDP mapping issues, it is more efficient to address all mapping corrections holistically rather than individually. This ensures consistent treatment of \<Facility\>, \<Prescriber\>, and \<Pharmacy\> element paths across the entire mapping table.

### Disposition C: Not Persuasive
**Action:** Close without change.

**Rationale:** The original IG against which this was filed has been withdrawn. The new PDMP on FHIR IG (PSS-2225) may take a fundamentally different approach to NCPDP mappings or may not include NCPDP mappings at all. The ticket's specific mapping suggestion may not be applicable to the new IG's design.

## Recommendation

**Disposition B: Persuasive with Modification** is recommended.

The technical concern is valid and should be addressed, but the most effective approach is to handle it within the context of the active PDMP on FHIR project (PSS-2225) alongside the full cohort of related NCPDP mapping corrections. This avoids piecemeal changes and ensures the entire mapping table is reviewed and corrected consistently. The work group should flag this ticket and its related batch (FHIR-29459, FHIR-29461, FHIR-29464, FHIR-29466, FHIR-29473) for the PSS-2225 project team to address as a group.
