## 7. Context & Analysis

This ticket is one of a cluster of ~20 PDMP-related change requests (labeled `PDMP-old-issues`) imported from a ballot spreadsheet in October 2020, all targeting the US Medication FHIR IG's PDMP section. The entire cluster was initially resolved as "Considered for Future Use" in January 2023 when the IG was withdrawn, then reopened in November 2023 under PSS-2225 (PDMP on FHIR).

The core request is straightforward: the PDMP mapping tables in the IG reference "NCPDP" generically without specifying the SCRIPT version. Since NCPDP SCRIPT 2017071 has been the regulatory standard since January 2020 (replacing SCRIPT 10.6 for electronic prescribing per CMS Part D regulations), the mappings should explicitly state the version. The proposed wording change also adds "RxHistoryRequest" to clarify exactly which SCRIPT message type is being mapped.

This request is closely related to FHIR-29528 (resolved Persuasive, proposing the same version update) and FHIR-29458 (addressing capitalization of "SCRIPT" and version references). A newer IG, the "US Prescription Drug Monitoring Program (PDMP) (FHIR)," has since been developed under PSS-2225, and ticket FHIR-46046 (resolved Persuasive with Modification) addresses how the FHIR PDMP IG fits alongside existing NCPDP integrations.

## 8. Proposed Dispositions

### Disposition A: Persuasive
**Action:** Accept the proposed wording change. Update the PDMP Request mapping table header from "NCPDP Mappings for PDMP Request" to "NCPDP SCRIPT 2017071 RxHistoryRequest Mappings for PDMP Request."
- **Rationale:** The request is factually correct — NCPDP SCRIPT 2017071 has been the regulatory standard since January 2020. Specifying the exact version and message type (RxHistoryRequest) improves clarity and implementer guidance. The closely related ticket FHIR-29528 was already resolved as Persuasive for the same version correction.
- **Considerations:** Should be coordinated with similar version-correction tickets (FHIR-29458, FHIR-29528, FHIR-29529) to ensure consistent terminology across the entire IG.

### Disposition B: Persuasive with Modification
**Action:** Accept the intent but broaden the wording update to cover the entire PDMP mapping section (both Request and Response tables), ensuring all NCPDP references specify "SCRIPT 2017071" and the correct message type names.
- **Rationale:** This ticket only addresses the Request mapping header, but the Response mapping tables have the same ambiguity (as noted by FHIR-29529). Applying a consistent version reference across all PDMP mapping tables in one pass is more efficient and avoids partial fixes.
- **Considerations:** Requires reviewing the full PDMP mapping section in the new PDMP IG (PSS-2225) to confirm the scope of updates. This may partially resolve FHIR-29529 as well.

### Disposition C: Not Persuasive (Duplicate / Consolidated)
**Action:** Close as duplicate of FHIR-29528 (which was resolved Persuasive) and FHIR-29529, consolidating the SCRIPT version updates under those tickets.
- **Rationale:** FHIR-29528 already covers the same correction (SCRIPT 10.6 → SCRIPT 2017071) and was resolved Persuasive. Rather than resolving multiple overlapping tickets independently, consolidation ensures a single clear editorial pass.
- **Considerations:** The submitter's specific suggestion to add "RxHistoryRequest" to the header is incremental value not fully captured by FHIR-29528, so closing as duplicate would lose that detail.

## 9. Recommendation

**Disposition B: Persuasive with Modification** is recommended.

The submitter's core request is valid and aligns with regulatory reality (SCRIPT 2017071 has been mandatory since 2020). However, the fix should be broadened to cover all NCPDP mapping references in the PDMP section — not just the Request table header — ensuring consistency with the full cluster of related version-correction tickets (FHIR-29458, FHIR-29528, FHIR-29529). This approach addresses the submitter's concern while delivering a more complete editorial improvement in a single pass. The workgroup should coordinate this disposition with the other PDMP-old-issues tickets to avoid redundant or conflicting resolutions.
