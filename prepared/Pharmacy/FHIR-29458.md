## Description

The submitter identifies two issues in the US Medication FHIR IG (PDMP section):

1. **Capitalization error:** "script" should be "SCRIPT" (the official product name is "NCPDP SCRIPT" in all caps).
2. **Version update:** NCPDP SCRIPT 2017071 should be referenced as the current named version alongside SCRIPT 10.6.

**Existing Wording:**
> Most of the existing EHR implementations use NCPDP script 10.6 or ASAP web services…

**Proposed Wording:**
> Most of the existing EHR implementations use NCPDP SCRIPT 2017071, NCPDP SCRIPT 10.6, or ASAP web services…

## Resolution Description

The resolution (Not Persuasive with Modification) states:

> Specific reference to MedicationRequest mapping to SCRIPT 10.6 NewRx will be removed. MedicationRequest is not directly related to the PDMP content (minimally used for prescription number, original fills authorized, etc.).
>
> Related tables will be updated to SCRIPT 2017071 RxHistoryRequest and RxHistoryResponse.

This goes beyond the submitter's request by also cleaning up mapping table references to align with SCRIPT 2017071 RxHistory messages rather than the older SCRIPT 10.6 NewRx mappings.

## Analysis

### Context
This ticket is part of a batch of PDMP-old-issues that were originally submitted during the first ballot of the US Medication FHIR IG's PDMP section. The IG was subsequently withdrawn, causing these issues to be deferred ("Considered for Future Use") in January 2023. They were reopened in November 2023 when the PDMP on FHIR project (PSS-2225) was re-initiated.

### Technical Assessment
The request is straightforward and well-justified:
- **Capitalization:** "NCPDP SCRIPT" is the official product name and should be capitalized consistently throughout the IG.
- **Version update:** NCPDP SCRIPT 2017071 became the mandated version for US e-prescribing, superseding SCRIPT 10.6. Multiple sibling tickets (FHIR-29460, FHIR-29528) confirm this same version update need.

The resolution extends the fix appropriately by removing MedicationRequest-to-SCRIPT 10.6 NewRx mappings (which were not directly relevant to PDMP content) and updating mapping tables to reference SCRIPT 2017071 RxHistoryRequest/RxHistoryResponse messages instead.

### Risks and Considerations
- The resolution changes mapping table structure (removing MedicationRequest mapping to NewRx), not just fixing capitalization/version text. Implementers relying on those mappings should be notified.
- Multiple overlapping tickets (FHIR-29460, FHIR-29528) address the same SCRIPT version issue. Disposition should ensure consistency across all of them.
- FHIR-46046 (Published, Persuasive with Modification) questions the overall need for this FHIR-based PDMP IG given existing NCPDP integration — context that may affect the priority of this work.

## Proposed Dispositions

### Disposition A: Accept as Not Persuasive with Modification (Recommended)
**Rationale:** The existing resolution is well-reasoned. It addresses both the capitalization error and the version update while also improving the IG by removing the loosely-related MedicationRequest-to-NewRx mapping and aligning tables to SCRIPT 2017071 RxHistoryRequest/RxHistoryResponse. This is consistent with the regulatory mandate for SCRIPT 2017071 confirmed by sibling tickets (FHIR-29528). The workgroup should confirm the resolution and ensure the related tickets (FHIR-29460, FHIR-29528) are dispositioned consistently.

**Vote Text:** Accept as Not Persuasive with Modification. The capitalization will be corrected to "NCPDP SCRIPT" and version references updated to SCRIPT 2017071. Per the resolution, the MedicationRequest mapping to SCRIPT 10.6 NewRx will be removed and mapping tables updated to SCRIPT 2017071 RxHistoryRequest and RxHistoryResponse.

### Disposition B: Upgrade to Persuasive with Modification
**Rationale:** The submitter's core request (capitalization fix + version update) is entirely correct and substantive. The resolution's additional cleanup of mapping tables is an appropriate enhancement. Given that the submitter accurately identified both a factual error and a needed version update, "Persuasive with Modification" better reflects that the comment was substantially correct while acknowledging the broader scope of the applied fix.

**Vote Text:** Accept as Persuasive with Modification. The submitter correctly identified the capitalization error and the need to reference NCPDP SCRIPT 2017071. Applied changes include correcting capitalization, updating version references, removing the unrelated MedicationRequest-to-NewRx mapping, and updating mapping tables to SCRIPT 2017071 RxHistoryRequest/RxHistoryResponse.

### Disposition C: Defer (Considered for Future Use)
**Rationale:** Given that FHIR-46046 questions the fundamental value proposition of this FHIR-based PDMP IG relative to existing NCPDP integration patterns, and the IG has a history of being withdrawn, the workgroup may choose to defer this editorial/version fix until the IG's overall direction is confirmed. However, this is less ideal given the IG has been actively reopened under PSS-2225.

**Vote Text:** Defer as Considered for Future Use. While the comment is valid, defer until the IG's scope and direction are reconfirmed under PSS-2225.

## Recommendation

**Disposition A: Accept as Not Persuasive with Modification** — The existing resolution is appropriate and well-scoped. The "Not Persuasive with Modification" disposition correctly reflects that while the capitalization fix was accepted, the resolution went further than the submitter's specific wording proposal by restructuring the mapping tables around SCRIPT 2017071 RxHistoryRequest/RxHistoryResponse rather than simply adding "2017071" to the existing text. The workgroup should ensure sibling tickets FHIR-29460 and FHIR-29528 are dispositioned consistently.
