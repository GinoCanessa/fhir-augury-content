## Summary

This change request proposes updating the Genomics Reporting Implementation Guide (GRIG) for FHIR R6 to leverage Molecular Definition (MolDef) profiles instead of the current component-based approach in Observation resources. MolDef contains molecular-level details in non-patient-specific resources/profiles, and the ticket asks how to align GRIG profiles (Variant, Genotype, Haplotype) with MolDef profiles (Variation, etc.).

The workgroup has prior agreement that MolDef integration applies to R6 GRIG only — there are no plans to backport to R4. A new IG repository has been requested for the R6 GRIG, separate from the R4 version. The reporter's intent is to document and track the CG workgroup consensus on this direction from earlier calls and WGMs.

Key discussion points from comments include:
- **Bret Heale** advocates for a formal comparison of DiagnosticReport with and without MolDef (operations, size, search support) before proceeding, and suggests using StructureMap resources to bridge R4→R6 for continuity with CodeX work.
- **Kevin Power (cerkyp)** confirms WG agreement on R6-only scope and notes that concrete implementation proposals will be needed in future JIRAs — this ticket is more directional.
- The reporter (Aly Khalifa / JIRAUSER20910) confirms R6 scope and sees this ticket as documenting the WG agreement for follow-up.

## Proposed Dispositions

### Disposition A: Accept — Approve as Directional Agreement for R6 GRIG
**Action:** Accept the change request as a documented directional agreement that the R6 GRIG will leverage MolDef profiles for Variant, Genotype, and Haplotype. Create follow-up JIRAs for each specific profile alignment (e.g., GRIG Variant → MolDef Variation, GRIG Genotype → MolDef Genotype, GRIG Haplotype → MolDef Haplotype). R4 GRIG remains unchanged.

**Rationale:** The WG has already reached consensus on this direction in prior calls and WGMs. This ticket serves to formally document that agreement. The R6-only scope is clear and uncontested. Concrete implementation details can be tracked in dedicated child tickets.

### Disposition B: Accept with Conditions — Require StructureMap Analysis and Processing Comparison
**Action:** Accept the directional agreement but add two prerequisites before implementation begins: (1) produce StructureMap resources that capture the R4 Observation.component → R6 MolDef mappings computably, and (2) conduct a processing comparison of DiagnosticReport with and without MolDef (covering operations count, payload size, and search support).

**Rationale:** Bret Heale's suggestions for StructureMap deliverables and a comparative analysis are well-founded — they would provide tangible evidence supporting the approach and ensure continuity with R4/CodeX work. This adds rigor before committing to the new architecture.

### Disposition C: Defer — Request Concrete Implementation Proposals First
**Action:** Defer this ticket and ask the reporter to submit specific implementation proposals (one per profile alignment) with detailed design including profile structures, mapping details, and impact on existing GRIG consumers. Revisit once concrete proposals are available.

**Rationale:** As Kevin Power noted, the WG agrees on the general direction, but "it will be the specific implementations that might trigger feedback." Without concrete proposals, this ticket is too high-level for formal disposition. Individual implementation tickets would allow more meaningful review and voting.

---

## Recommendation

**Disposition A: Accept as Directional Agreement** is recommended. The workgroup consensus is clear — MolDef integration applies to R6 GRIG only, with no R4 backport. All three commenters (Aly Khalifa, Kevin Power, Bret Heale) agree on the general direction. The ticket's stated purpose is to document this agreement, not to define specific implementations. Accepting it formalizes the consensus and enables follow-up JIRAs for concrete profile alignments, StructureMap development, and comparative analysis as separate workstreams. Deferring would leave an already-agreed direction without formal record.
