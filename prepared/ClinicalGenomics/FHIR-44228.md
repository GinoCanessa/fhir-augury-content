## Summary

This ticket requests that code systems and value sets defined in the Genomics Reporting IG be anchored in HL7's Unified Terminology Governance (UTG) system at terminology.hl7.org (THO), rather than remaining within the IG itself, in accordance with TSC-approved terminology policy for IG authors. The ticket was originally resolved as "Considered for Future Use" in September 2024 with a 13-0-1 vote, on the rationale that the code system was still tightly coupled with the IG. However, it was re-opened in December 2024 following the STU3 publication cycle. Recent activity indicates active investigation: Kevin Power (cerkyp) evaluated the TSMG terminology expectations checklist and concluded the code system is appropriate for THO (it requires a new code system, is not tightly tied to IG structure, and is used in required bindings). Patrick Werner is investigating whether the relevant concepts—specifically cis and trans—can be found in existing external code systems such as LOINC or NCIT, which could lead to restructuring the value sets or deleting the custom code system entirely.

## Proposed Dispositions

### Disposition A: Accept — Initiate UTG Process to Migrate Code System to THO
#### Proposal
Accept the request and begin the UTG process to migrate the Genomics Reporting IG's code system (containing cis/trans and related concepts) to THO. Update the IG's CodeSystem and ValueSet canonical URLs to reference the new THO-hosted code system. Coordinate with TSMG to register the new code system and ensure the required-strength ValueSet bindings in the IG reference the THO-anchored artifacts.
#### Justification
The TSMG terminology expectations evaluation (cerkyp, 2025-05-26) confirms this code system meets the criteria for THO hosting: it requires a new code system not available in existing external or HL7 code systems, it is not tightly coupled to the IG's structure, and it is used in required bindings (not merely Example strength). The previous deferral rationale—that the CS was tightly coupled—has been revisited and found no longer applicable post-STU3. TSC policy clearly directs IG-defined terminology to THO, and compliance aligns with HL7's terminology governance strategy.

---

### Disposition B: Restructure Using External Terminology Before Migrating
#### Proposal
Before initiating the UTG process, complete Patrick Werner's investigation into whether cis and trans concepts already exist in LOINC, NCIT, or other established code systems. Where suitable external codes exist, restructure the ValueSet(s) to reference those external codes directly. Migrate only the genuinely novel concepts (those not found in any existing code system) to THO as a new, smaller code system. Update the IG bindings accordingly.
#### Justification
Patrick Werner's January 2026 comment indicates active research into whether existing external code systems already define the needed concepts. Restructuring to leverage established terminology (e.g., LOINC, NCIT) before migrating to THO would minimize the proliferation of HL7-specific codes, improve interoperability with systems already using those external code systems, and reduce the long-term maintenance burden on THO. This approach still satisfies the TSC policy requirement while producing a more robust terminological foundation. It may delay resolution slightly but yields a higher-quality outcome.

---

### Disposition C: Defer — Await Maturation of the Code System
#### Proposal
Defer this request again, maintaining the code system within the Genomics Reporting IG until a future ballot cycle when the IG and its terminology have reached higher maturity (FMM 3+). Re-evaluate after the next publication cycle.
#### Justification
The code system is currently used only within low-maturity resources (FMM 2 or less), as acknowledged in the terminology checklist evaluation. The previous vote (13-0-1) to defer reflected consensus that the code system was still maturing alongside the IG. Migrating immature terminology to THO risks premature standardization and the need for breaking changes in THO if concepts are later revised or removed. However, this disposition conflicts with the TSC policy direction, the re-opening of the ticket post-STU3, and the active investigation into external terminology—all of which suggest the work group intends to address this now rather than defer further.

---

### Recommendation
**Recommended disposition:** B

Disposition B (Restructure Using External Terminology Before Migrating) is the most pragmatic path forward. The active investigation by Patrick Werner into whether cis and trans concepts exist in LOINC or NCIT is directly relevant and should be completed before committing to a THO migration. If suitable external codes are found, the IG can simply reference them—eliminating the need for a custom code system entirely—while any novel concepts can be submitted to THO through the UTG process. This satisfies the TSC policy requirement, aligns with the TSMG evaluation that confirmed THO-appropriateness for novel concepts, and produces the most interoperable outcome. The ticket has already been deferred once and re-opened with clear intent to resolve it; Disposition C would likely face resistance. Disposition A is valid but risks creating THO codes that duplicate existing external terminology, which Patrick Werner's research may reveal. Completing that research first is a low-cost, high-value step before finalizing the migration approach.
