## Summary

This ticket requests the establishment of terminology bindings for the Molecular Definition (MolDef) Implementation Guide. The MolDef Datatype resource introduces various profiles for different use cases, and several CodeableConcept and Code elements within these profiles require binding to appropriate ValueSets. The request also raises the possibility of developing new code systems where existing ones are insufficient. A specific example cited is the cytoband Interval elements, which need binding to currently available code systems while preserving the resource and profile semantics. The ticket was discussed at the WGM, with the work group planning to vote on a concrete proposal in the future. A related comment notes that the MolDef core resource may move to the MolDef IG, which could result in some Jira tickets targeting core specifications being marked as duplicates and redirected to this ticket.

## Proposed Dispositions

### Disposition A: Accept As Requested
#### Proposal
Accept the request and proceed with identifying all CodeableConcept and Code elements across MolDef IG profiles that require terminology bindings. For each element, evaluate existing code systems (e.g., ISCN cytoband nomenclature, HGNC, HGVS, LOINC genomics codes) for suitability, define new ValueSets referencing those code systems, and create new code systems only where no adequate external terminology exists (e.g., for cytoband Interval representations). Publish the bindings with appropriate binding strengths (required, extensible, preferred) based on the maturity and coverage of each terminology.
#### Justification
The MolDef IG cannot achieve interoperability without well-defined terminology bindings. Unbound CodeableConcept and Code elements lead to inconsistent data representation across implementations. The work group has already discussed this at the WGM and plans to vote on a concrete proposal, indicating community readiness. Establishing these bindings now, while the IG is being developed, is more efficient than retrofitting them later.

---

### Disposition B: Alternative Approach
#### Proposal
Accept the request in a phased manner. In the first phase, catalog all CodeableConcept and Code elements across MolDef profiles and classify them by priority (critical for interoperability vs. informational). Bind high-priority elements (e.g., cytoband Interval, sequence type, molecular consequence) to existing code systems with extensible bindings. Defer lower-priority elements and the creation of new code systems to subsequent ballot cycles, allowing time for community review and alignment with external terminology standards bodies.
#### Justification
A phased approach reduces the risk of premature binding decisions that may need revision. It allows the work group to focus on the most impactful terminology gaps first while gathering implementer feedback. Given that the MolDef core resource may be moving to the IG (as noted in the comments), the scope of required bindings may shift, and a phased approach provides flexibility to accommodate such structural changes.

---

### Disposition C: Decline
#### Proposal
Decline the request as premature. The ticket does not specify which elements need bindings, what code systems should be used, or what binding strengths are appropriate. Request that the submitter provide a detailed inventory of elements requiring terminology bindings along with candidate code systems before the work group takes action.
#### Justification
The request as stated is broad and lacks the specificity needed for the work group to act on it directly. Without a concrete list of elements and proposed terminologies, accepting the ticket risks open-ended scope. The WGM comment confirms that a concrete proposal is expected in the future, suggesting that this ticket should await that proposal before formal disposition.

---

### Recommendation
**Recommended disposition:** B

A phased approach (Disposition B) is the most pragmatic path forward. The need for terminology bindings in the MolDef IG is real and acknowledged by the work group, as evidenced by the WGM discussion. However, the ticket as currently written is high-level and does not provide the specificity needed for a single comprehensive binding pass. By cataloging elements first and prioritizing bindings for the most critical interoperability gaps — such as cytoband Interval representations — the work group can make meaningful progress while keeping scope manageable. This approach also accommodates the potential structural change of moving the MolDef core resource into the IG, which could alter the set of elements requiring bindings. The phased model aligns well with the comment indicating that a concrete proposal will be voted on in the future, as each phase can correspond to a specific ballot cycle.
