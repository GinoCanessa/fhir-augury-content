## Details

This ticket is a general comment acknowledging the format change in the EHRS Functional Model – Record Lifecycle Events (FHIR) specification. The reporter (Karrie Schwencer) notes that the specification has been updated to move away from a PDF-based format and now uses FHIR IG tooling, giving it the look and feel of a standard FHIR Implementation Guide. The specification aligns with FHIR R5.

This is an informational/supportive comment rather than a change request. It does not propose any specific modifications but rather acknowledges and describes the format migration that has already taken place.

## Proposed Dispositions

### Disposition A: Accept as Informative (Not Persuasive)

**Resolution: Not Persuasive — No Change Required**

Accept the comment as a positive acknowledgment of the format migration from PDF to FHIR IG tooling. Since this is a supportive general comment and does not request any specific change to the specification, no action is required. The work group can note the comment for the ballot record and close the ticket. The format change described (PDF → IG tooling, FHIR R5 alignment) is already implemented.

### Disposition B: Accept as Persuasive with Mod — Add Documentation

**Resolution: Persuasive with Modification**

Treat the comment as an opportunity to add a brief note in the IG's introduction or change log explicitly documenting the format migration from PDF to FHIR IG tooling and the alignment with FHIR R5. This would formalize the change the commenter describes and provide context for future readers encountering the specification for the first time. The note could clarify what changed, why the migration was undertaken, and what version of FHIR is now targeted.

### Disposition C: Decline — Request Resubmission as Specific Feedback

**Resolution: Not Related**

Decline the comment on the basis that it is a general observation rather than actionable ballot feedback. Encourage the commenter to resubmit with specific technical feedback on the IG content, structure, or conformance if they have concerns or suggestions. General comments about format do not typically require work group action.

## Recommendation

**Disposition A: Not Persuasive — No Change Required** is recommended. This ticket is a supportive acknowledgment of the format migration to FHIR IG tooling. It does not identify any issues or request changes. The described migration (PDF → IG tooling, FHIR R5 alignment) is already in place. The work group should acknowledge the comment positively and close it without requiring further action.
