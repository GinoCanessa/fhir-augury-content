## Summary

The AI Transparency IG contained unresolved placeholder text reading *"Insert example scenarios for using data source documentation"* in the General Guidance page. The submitter (PatrickJoyce) proposed that this placeholder be replaced with actual example scenarios demonstrating data source documentation practices, or that the subsection be removed if examples were not yet available. John Moehrke re-classified the ticket from a technical correction to a Change Request, noting that it requires new content creation rather than a simple fix.

## Details

**Current text (as balloted):**
> "Insert example scenarios for using data source documentation"

**Proposed change:** Replace with actual example scenarios or remove the subsection.

**Applied resolution:** The work group restructured the General Guidance page and replaced the concept of "data sources" with "context" for clarity. The resolution further clarified that prompts are the context, and examples of prompts being documented were supplied. This work was implemented in [HL7/aitransparency-ig#22](https://github.com/HL7/aitransparency-ig/pull/22) ("Rewrite of General Guidance Page" by samschifman), which reorganized and rewrote much of `general_guidance.md` to harmonize language and clarify observability factors. The PR was merged on 2026-03-30 with +90/−129 lines changed.

## Proposed Dispositions

### Disposition A: Accept (Persuasive with Modification)

Accept the applied resolution as **Persuasive with Modification**. The placeholder text has been removed and the section restructured with meaningful content. Rather than inserting literal example scenarios for "data source documentation," the work group replaced the concept of "data sources" with "context," clarified that prompts constitute context, and provided examples of documented prompts. This goes beyond the original proposal (which suggested either adding examples or removing the subsection) by reconceptualizing the section entirely, which justifies the "with Modification" qualifier. The implementation is complete in merged PR #22.

### Disposition B: Request Additional Examples

Accept as **Persuasive with Modification** but request that the work group add one or two concrete, end-to-end example scenarios showing how data source (context/prompt) documentation would appear in a real-world AI Transparency use case. The current resolution clarifies terminology and supplies prompt examples, but the original submitter's intent was for worked example scenarios demonstrating the documentation practice in action. Adding such scenarios would improve implementer guidance without undoing the beneficial restructuring already applied.

### Disposition C: Decline (Not Persuasive)

Mark as **Not Persuasive**. The placeholder text was editorial scaffolding that the editors intended to address during normal IG development. Ballot comments about incomplete placeholder text could be considered premature given the IG's maturity level, and the editors were already tracking this work internally. However, this disposition is difficult to justify given that the placeholder was published in the ballot version and the issue has already been substantively addressed.

## Recommendation

**Disposition A: Accept (Persuasive with Modification).**

The resolution directly addresses the reported problem — placeholder text was replaced with substantive content — and goes further by improving the conceptual clarity of the section (replacing "data sources" with "context"). The implementation is complete and merged in PR #22, which also addressed the related page-restructuring concern from FHIR-54240. The "with Modification" qualifier is appropriate since the resolution took a broader approach than the original proposal. No further action is required.
