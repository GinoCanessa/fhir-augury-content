## Summary

The submitter requests improved clarity in the introductory text of the AI Transparency IG. The current introduction jumps straight into how to tag FHIR resources and how to use Provenance, Device, and other data elements, without first describing the real-world AI concepts being modeled. The suggestion is to restructure the narrative so that it first describes the elements of AI technology being represented (model card, prompt, device, ai-generated-data), and then explains which FHIR resources are used to encode them. This is a documentation/clarity improvement classified as a Technical Correction.

## Details

The quoted introductory passage from the IG reads:

> *"This FHIR Implementation Guide (IG) provides guidance for representing the . **Starting with how to tag** FHIR resources, and expanding into how to use Provenance, Device, and other data elements, this FHIR IG provides standards that enable downstream use cases to identify such resources. This allows the informed usage of AI-inferred health data."*

The submitter observes that this text is implementation-centric — it leads with the FHIR encoding mechanics (tagging, Provenance, Device) rather than the real-world concepts being modeled. The proposal is to reorder the narrative to:

1. **First** describe the real-world AI elements: model card, prompt, device (AI system), and AI-generated data.
2. **Then** explain how these map to FHIR resources (Device, Provenance, extensions, etc.).

This aligns with common IG authoring best practice where the conceptual model is introduced before the technical encoding. Several related tickets from the same reporter (FHIR-54112, FHIR-54114, FHIR-54116) also address clarity and scoping of the IG's introductory sections, suggesting a coordinated editorial pass may be warranted.

## Proposed Dispositions

### Disposition A: Accept (Persuasive)

**Resolution:** Persuasive  
**Action:** Restructure the introductory section of the AI Transparency IG to present real-world AI concepts before their FHIR encoding.

Specifically, add a paragraph or subsection before the current introduction that describes:
- **AI Model/System (Model Card):** A description or metadata record of the AI model used (name, version, intended use, known limitations).
- **Prompt / Request:** The input provided to the AI system for a specific invocation.
- **AI Device:** The computational system or service that executed the AI model.
- **AI-Generated Data:** The output produced by the AI, which may be a complete FHIR resource or a data element within one.

Then follow with the existing text (edited for flow) explaining how these map to FHIR resources: Device for the AI system, Provenance for tracking AI involvement, extensions/tags for marking AI-generated content, and model-card profiles for model metadata.

**Rationale:** This is a straightforward documentation improvement that follows standard IG authoring practices (concept-first, then encoding). It requires no structural changes to profiles or extensions and makes the IG more accessible to new readers.

### Disposition B: Accept with Broader Editorial Pass (Persuasive with Modification)

**Resolution:** Persuasive with Modification  
**Action:** Address this ticket as part of a consolidated editorial pass on the IG introduction, coordinated with FHIR-54116 (aim clarification), FHIR-54112 (AI definition scoping), and FHIR-54114 (AI inference vs. FHIR resource distinction).

Rather than adding a paragraph in isolation, rewrite the entire introductory section (Sections 1–2) to:
1. Define what AI means in the context of this IG (addressing FHIR-54112).
2. Describe the real-world elements being modeled (this ticket).
3. Clearly state the IG's aim (addressing FHIR-54116).
4. Distinguish between AI-generated data and AI-generated FHIR resources (addressing FHIR-54114).
5. Then present the FHIR resource mappings.

**Rationale:** Multiple tickets from the same reporter target the same introductory sections. A coordinated rewrite would produce a more coherent result than piecemeal edits. The Model-Card semantics clarification from FHIR-54421 should also be reflected.

### Disposition C: Decline (Not Persuasive)

**Resolution:** Not Persuasive  
**Action:** No change. The current text is adequate for the ballot level.

**Rationale:** The IG is intended for implementers already familiar with FHIR and AI concepts. The current structure — leading with FHIR mechanics — is appropriate for the target audience. Detailed conceptual descriptions of AI elements (model card, prompt, device) are available in the body of the IG and do not need to be duplicated in the introduction. The request is stylistic rather than technically necessary.

## Recommendation

**Disposition B: Accept with Broader Editorial Pass (Persuasive with Modification)**

This ticket is one of four closely related issues (FHIR-54112, FHIR-54114, FHIR-54116, FHIR-54118) from the same reporter, all targeting the introductory sections of the AI Transparency IG. Addressing them individually risks fragmented edits that may introduce new inconsistencies. A coordinated editorial pass on the introduction would produce a more coherent narrative, cover all four tickets efficiently, and incorporate the Model-Card semantics clarification from FHIR-54421. The underlying feedback — that the IG should explain *what* it models before *how* it encodes — is valid and aligns with standard IG authoring practice.
