## Description

The reporter observes that the AI Transparency IG defines "AI algorithm" very broadly as *"any computer-based logic that touches health data in a way that might change the understanding of the data downstream."* This definition encompasses rule-based systems, imaging algorithms, and other non-ML computational logic. However, the IG's actual model, profiles, and use cases appear tailored specifically to machine learning (ML) and large language models (LLMs).

The reporter suggests two alternatives:
1. **Narrow the definition** of "AI system" to better match the IG's actual content (ML/LLM-focused).
2. **Expand the model** to properly support rule-based systems, with harmonization with the CDS Working Group's artifacts.

## Analysis

This ticket sits at the center of a cluster of related scope-definition concerns in the AI Transparency IG ballot (FHIR-54893, FHIR-54726, FHIR-54179, FHIR-54885). The core tension is:

- **The IG's scope definition is intentionally broad**, covering all "computer-based logic that touches health data" — including rule-based systems and simple calculations.
- **The IG's actual artifacts (profiles, examples, use cases) are tailored to ML/LLMs**, creating a gap between what the scope promises and what the IG delivers.
- **The same reporter filed FHIR-54179** specifically asking for rule-based system examples or scope exclusion, making these tickets closely linked.

john_moehrke's comment on FHIR-54893 signals the WG's current position: the broad definition is intentional and meant to be inclusive. However, multiple balloters have independently flagged the disconnect, suggesting the issue is real.

Resolution of this ticket should be coordinated with FHIR-54893, FHIR-54726, and FHIR-54179, as they all address the same scope definition paragraph.

## Proposed Dispositions

### Disposition A: Persuasive with Modification — Refine the definition and acknowledge the gap

**Action:** Revise the scope section to distinguish between the IG's aspirational scope (broad, including all computational logic touching health data) and its current normative scope (focused on ML/AI systems). Add clarifying text that acknowledges rule-based systems are within the broader vision but that the current version primarily addresses ML/AI workflows. Add a note that harmonization with CDS WG artifacts for rule-based systems is a future work item.

**Rationale:** This acknowledges the valid concern without narrowing the definition so much that future extensibility is lost. It sets honest expectations for implementers about what the current IG version covers while preserving the broad vision. Addresses the reporter's concern that the model and use cases don't match the definition.

### Disposition B: Persuasive — Narrow the AI algorithm definition to ML/AI systems

**Action:** Revise the "AI algorithm" definition to explicitly focus on machine learning, deep learning, and statistical models. Remove or reword the claim that rule-based systems and simple calculations are in scope. Coordinate with CDS WG to ensure rule-based system observability is addressed through appropriate CDS artifacts.

**Rationale:** Directly implements the reporter's first suggestion. Aligns the definition with the IG's actual content. However, this is a significant scope reduction that may conflict with the WG's stated intent (per john_moehrke's comment on FHIR-54893) and could leave a gap for non-ML computational logic that still warrants transparency.

### Disposition C: Not Persuasive — The broad definition is intentional and appropriate

**Action:** No change to the definition. Optionally add informative text explaining why the broad definition was chosen and how it enables future extensibility.

**Rationale:** The WG intentionally chose the broad definition to be inclusive (as john_moehrke noted on FHIR-54893). The definition serves as an aspirational scope, and the IG's current examples being ML-focused doesn't mean the definition is wrong — it means the examples will expand over time. The broad definition avoids the difficult task of drawing a line between "AI" and "non-AI" systems.

## Recommendation

**Disposition A: Persuasive with Modification** — The gap between the broad scope definition and the ML-focused content is independently flagged by multiple balloters (FHIR-54112, FHIR-54893, FHIR-54726, FHIR-54179), indicating a genuine usability concern. Rather than narrowing the definition (Disposition B) — which conflicts with the WG's stated intent — or leaving it unchanged (Disposition C) — which ignores valid feedback — the best path is to add clarifying text that distinguishes between the IG's long-term aspirational scope and its current practical coverage. This approach preserves the broad vision, sets accurate implementer expectations, and naturally connects to future CDS WG harmonization for rule-based systems. This ticket should be resolved in coordination with FHIR-54179 (rule-based examples) and FHIR-54893 (AI vs non-AI scope).
