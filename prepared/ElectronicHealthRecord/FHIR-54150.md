## Description

The reporter identifies an inconsistency in the AI Transparency IG between a CodeSystem's human-readable title and its computable name:

- **Current title:** "Added Device.type for AI/LLM"
- **Current computable name:** `AIdeviceTypeCS`

The title reads like a changelog entry ("Added Device.type for AI/LLM") rather than a descriptive resource name. The computable name `AIdeviceTypeCS` uses a different naming pattern altogether. The proposal is to align the title and computable name to use a consistent format and capitalization pattern.

## Resolution Description

The resolved disposition is **Persuasive with Modification**:

- **Current title:** "Added Device.type for AI/LLM"
- **Change to:** "Device Type for Artificial Intelligence"

This replaces the changelog-style title with a proper descriptive name that aligns better with the CodeSystem's purpose and the computable name.

## Proposed Dispositions

### Disposition A: Persuasive with Modification (Accept the Resolution)

**Action:** Accept the proposed resolution — change the CodeSystem title from "Added Device.type for AI/LLM" to "Device Type for Artificial Intelligence."

**Rationale:** The current title "Added Device.type for AI/LLM" reads as a changelog entry and does not follow standard FHIR naming conventions for CodeSystem titles. The proposed new title "Device Type for Artificial Intelligence" is descriptive, professional, and aligns better with the computable name `AIdeviceTypeCS`. This is a straightforward quality improvement that addresses a legitimate naming inconsistency in the IG.

**Impact:** Minimal. Changes only the `title` element of a single CodeSystem resource. No structural or semantic impact on the IG.

### Disposition B: Persuasive (Full Alignment of Title and Name)

**Action:** Accept the concern and go further — align both the title and the computable name to follow a fully consistent naming convention (e.g., title: "AI Device Type Code System", computable name: `AIDeviceTypeCS` with corrected capitalization).

**Rationale:** The resolution addresses the title but does not fully resolve the inconsistency with the computable name. The computable name `AIdeviceTypeCS` uses inconsistent casing (`device` is lowercase while `AI` and `Type` and `CS` are capitalized). A complete fix would also adjust the computable name to follow standard PascalCase conventions (e.g., `AIDeviceTypeCS`).

**Impact:** Slightly broader than Disposition A — changes both title and computable name. Any references to the old computable name in the IG or dependent implementations would need to be updated.

### Disposition C: Not Persuasive

**Action:** Reject the change request.

**Rationale:** The current title and computable name, while not perfectly aligned, are functional and do not impede implementation. Changing them mid-ballot cycle introduces unnecessary churn. The computable name `AIdeviceTypeCS` is already in use and renaming it could break references in dependent systems.

**Impact:** None — no changes to the IG.

## Recommendation

**Disposition A: Persuasive with Modification** — Accept the proposed resolution to change the title to "Device Type for Artificial Intelligence." This is the resolution already agreed upon and it directly addresses the core issue (the changelog-style title) with minimal risk. The title change is a clear quality improvement that aligns with FHIR naming expectations, and since no comments objected to this approach, it should proceed as resolved.
