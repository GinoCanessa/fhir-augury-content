This ticket is an omnibus change request containing **ten distinct editorial and technical clarification items** across the General Guidance page of the AI Transparency IG (2026Jan ballot). The submitter identifies issues in multiple sections — observability factors, AI usage representation, tagging, security labels, and examples — ranging from unclear table structure to ambiguous normative language. A commenter (John Moehrke) upgraded the ticket from technical correction to change request, noting the scope exceeds a simple fix.

### Items Raised

1. **Observability Factors table** — row/column meanings unclear; needs headers or explanatory text.
2. **AI Usage Representation wording** — the phrase "distinguishes where AI was used" should be clarified to "used **to generate or influence**."
3. **Column naming inconsistency** — column 3 is "Context" but section 2.1.3 is "Data Sources"; should be harmonized.
4. **External references** — observability factors should cite external sources if any were used.
5. **Tagging: "completely authored" tip** — "Use when an example is completely authored by an AI" is unclear; suggests rephrasing.
6. **Tagging: informal WG discussion tone** — paragraph about adding codes reads like internal discussion; proposes professional rephrase referencing AddedProvenanceCS.
7. **Tagging: missing tip marker** — "Use when components of an example were authored by AI" should be formatted as a Tip like its sibling.
8. **PROCESSINLINE requirement** — unclear whether using inline-sec-label extension *requires* PROCESSINLINE in meta.security; proposes making it a SHALL requirement.
9. **Hugging Face link** — GitHub URL for huggingface_hub test fixtures should be a clickable external link.
10. **PDF-to-FHIR example** — Provenance targets a transaction Bundle, but Bundle resources aren't tagged AIAST; Bundle doesn't extend DomainResource so can't apply AI Data profile.
