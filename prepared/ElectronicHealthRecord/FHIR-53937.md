## Analysis

### Scope & Impact

This is a **presentation and documentation improvement** request for the AI Transparency IG. The submitter asks that all use cases (currently described in text) be augmented with visual diagrams. The request is broad — it applies to every use case in the IG, not a specific one.

John Moehrke's comment adds an important **practical constraint**: any diagrams must be in an editable format using open-source or widely available tools (e.g., PowerPoint, draw.io/diagrams.net, PlantUML), not formats requiring licensed software.

A near-duplicate ticket (FHIR-54724) also requests use case diagrams, specifically showing FHIR resources used per use case. That ticket was marked as a duplicate of FHIR-54967, though FHIR-54967 concerns the Device module rather than AI Transparency — this may be a mislink.

The request is complementary to FHIR-54278, which asks for more detailed use case descriptions with concrete FHIR examples. Diagrams and detailed descriptions together would significantly improve use case comprehension.

### Considerations

- **Effort:** Adding diagrams to all use cases is a significant authoring effort, especially if the IG has many use cases. The work group should assess whether this should be done incrementally or all at once.
- **Format:** The comment requires editable formats. Options include PlantUML (text-based, version-controllable), draw.io/diagrams.net (free, XML-based), or PowerPoint/SVG. PlantUML or draw.io would be most compatible with IG publisher tooling.
- **Coordination:** If FHIR-54278 (more detailed use case descriptions) is also accepted, the diagram work should be coordinated so diagrams reflect the improved use case content.

## Proposed Dispositions

### Disposition 1: Persuasive

**Action:** Accept the request and add visual diagrams to all use cases in the AI Transparency IG.

**Rationale:** Diagrams demonstrably improve comprehension of workflows and actor interactions, which is especially valuable for a new IG where implementers are learning novel concepts (AI transparency in healthcare). The submitter provides a sample diagram showing the style desired. John Moehrke's constraint (editable, open-source format) is practical and achievable — PlantUML or draw.io/diagrams.net are well-suited for IG development. This aligns with the broader feedback theme in the "Use Cases & Examples" category, where multiple ballot comments request improved use case clarity.

### Disposition 2: Persuasive with Modification

**Action:** Accept the request but scope it to selected use cases rather than all, and specify that diagrams must be in PlantUML or draw.io format for IG publisher compatibility.

**Rationale:** Requiring diagrams for *every* use case may be overly ambitious for a single ballot cycle, especially if use cases are still being refined (per FHIR-54278, FHIR-53814, and others). The work group could prioritize diagrams for the most complex or commonly referenced use cases first, and add remaining diagrams in future cycles. Specifying the diagram format upfront avoids rework and ensures consistency.

### Disposition 3: Not Persuasive

**Action:** Decline the request as out of scope for the current ballot cycle.

**Rationale:** The AI Transparency IG is in its early ballot stages with many substantive content changes still pending (90 issues across 9 categories). Adding diagrams to all use cases is primarily a presentation enhancement and could be deferred until the use case content stabilizes. Resources would be better spent addressing substantive structural and content issues first.

## Recommendation

**Disposition 2: Persuasive with Modification** — Accept the request with a scoped approach: add diagrams for the highest-priority use cases in the current cycle using an open-source editable format (PlantUML or draw.io), and plan to complete the remaining use cases in subsequent ballots. This balances the clear value of visual aids against the practical constraints of an early-stage IG with 90 open issues and evolving use case content. Coordinating with FHIR-54278 (improved use case descriptions) will ensure diagrams reflect finalized content.
