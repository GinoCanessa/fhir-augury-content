## Description

The request is to align the MessageHeader resource's FiveWs (Five Ws) pattern mappings. The FiveWs pattern is a cross-resource analytical framework that maps resource elements to the fundamental questions: Who, What, When, Where, Why, and ancillary categories (identifier, version, status, class, grade, etc.).

Currently, the `messageheader-fivews-mapping-exceptions.xml` file in the FHIR core repository contains **16 unmapped FiveWs pattern elements**, of which **14 have reason "Unknown"** — meaning the mismatch between the pattern and MessageHeader has never been formally analyzed or documented. Only two entries have been addressed:

- `FiveWs.source` → reason: "source"
- `FiveWs.who` → reason: "sender"

The remaining unmapped elements with "Unknown" reasons are:
- `FiveWs.recorded`, `FiveWs.author`, `FiveWs.cause`, `FiveWs.version`, `FiveWs.witness`, `FiveWs.class`, `FiveWs.where`, `FiveWs.context`, `FiveWs.init`, `FiveWs.identifier`, `FiveWs.grade`, `FiveWs.status`, `FiveWs.planned`, `FiveWs.done`

The reporter posted proposed mapping values on a Confluence page: [Message Header five ws](https://confluence.hl7.org/display/INM/Message+Header+five+ws).

## Zulip Discussions

### 1. [committers > 5ws](https://chat.fhir.org/#narrow/stream/committers/topic/5ws) (April 2024)
Anthony Julian raised the issue of receiving INFORMATION build messages about "Unknown" reasons in `messageheader-fivews-mapping-exceptions.xml`. John Moehrke explained three options: (1) correct the model to align with the pattern, (2) enter an excuse/reason in the exceptions file, or (3) do nothing. He noted that the reasons appear on the [FiveWs Detailed Analysis page](https://build.fhir.org/fivews-analysis.html) and warned that pull requests may inadvertently overwrite exception files during local builds. Lloyd McKenzie subsequently updated the [Mapping to Patterns](https://confluence.hl7.org/display/FHIR/Mapping+to+Patterns) Confluence documentation.

## Proposed Dispositions

### Disposition A: Persuasive — Accept and Align All Mappings
**Action:** Accept the ticket. Assign to Anthony Julian (or a committer) to complete the FiveWs alignment for MessageHeader by:
1. Reviewing the proposed mappings on the [Confluence page](https://confluence.hl7.org/display/INM/Message+Header+five+ws), incorporating Lloyd McKenzie's December 2025 feedback.
2. Updating `messageheader-fivews-mapping-exceptions.xml` to replace all "Unknown" reasons with either a proper mapping or a documented rationale for why the pattern element does not apply.
3. If applicable, adding FiveWs mapping entries to the MessageHeader StructureDefinition for elements that do have a natural correspondence.

**Rationale:** The ticket requests a straightforward and well-defined alignment task. Proposed values exist. The work has been in progress since April 2024, and Lloyd McKenzie has provided review feedback. This eliminates noisy "Unknown" build messages and improves conformance documentation.

### Disposition B: Persuasive with Modification — Accept with Scoped Initial Pass
**Action:** Accept, but limit the initial scope to documenting reasons (excuses) in the exceptions file for all "Unknown" entries, without requiring new StructureDefinition mappings. New mappings where MessageHeader elements naturally correspond to FiveWs categories can be tracked as a follow-up.

**Rationale:** The simpler task of documenting reasons is less risky and resolves the build information messages. Adding new mappings to the StructureDefinition requires more analysis and review, and can be done incrementally.
