## Description Summary

This ticket requests a comprehensive set of text updates across the **Terminology Service** page (section 4.7 of the FHIR spec). The changes were entered on behalf of a TI co-chair review documented on the [FHIR R6 Terminology Content Review](https://confluence.hl7.org/spaces/VOC/pages/256510055/FHIR+R6+Terminology+Content+Review) Confluence page. The changes span multiple subsections and fall into several categories:

### Structural Changes
- **Move "Basic Concepts" (§4.7.3)** to be the first section (§4.7.1).
- **Add new links** to the Basic Concepts bulleted list: Operations Framework, Terminology Fundamentals (publication status uncertain), and terminology.hl7.org.
- **Remove content** starting with "In addition..." from Basic Concepts.
- **Remove §4.7.3.1 External Code Systems** entirely (the associated extension is being deprecated via FHIR-53787).

### Terminology Consistency ("concept codes")
- Systematically replace "codes", "codes or concepts", "codes/concepts" with **"concept codes"** across multiple sections (§4.7.4 Value Set Expansion, §4.7.5 Concept Lookup, and others).
- Careful exceptions noted where "code" is appropriate (e.g., "code system", "error code").

### Wording & Accuracy Improvements
- **§4.7.1.1 Restful API:** Remove trailing period from first bullet.
- **§4.7.1.2 Terminology Service Operations:** Rename column from "Required input parameters" to "Input parameters that must be supported".
- **§4.7.1.3 CapabilityStatement/TerminologyCapabilities examples:** Update bullet text.
- **§4.7.3.2 Implementation Note:** Change "terminology" to "code system"; remove "Implementation Note:" section.
- **§4.7.3.3 Terminology Maintenance:** Update sentence to reference "code system, value set, concept map and naming system resources".
- **§4.7.5 Concept Lookup:** Rename from "Concept Lookup / Decomposition" to "Concept Lookup"; update date-related bullet wording.
- **§4.7.6 Value Set Validation:** Clarify CodeableConcept validation behavior; update result description to reference "output parameter = result" and "issues".
- **§4.7.7 Subsumption Testing:** Capitalize "Testing"; add $subsumes operation link; fix grammar ("relationships between the code systems are well defined").
- **§4.7.9 Translations:** Change "the translate operation" to "the $translate operation"; update "coding system at question" phrasing.

## Analysis

### Scope & Impact
This is a **large editorial change request** affecting approximately 10 subsections of the Terminology Service page. The changes are primarily:
1. **Structural reorganization** (moving Basic Concepts, removing External Code Systems)
2. **Terminology standardization** (consistent use of "concept codes")
3. **Wording clarification** (more precise language for operations, validation behavior)

The ticket is well-documented with explicit before/after text for most changes, which should ease implementation. However, the sheer number of changes increases the risk of missed edits or inconsistencies.

### Dependencies
- **FHIR-53787** (deprecating the CapabilityStatement Supported CodeSystem extension) should be resolved before or concurrently with the §4.7.3.1 removal.
- **FHIR-54107** also removes External Code Systems content from a different section; coordination is needed to ensure consistency.
- The reporter notes uncertainty about linking to **Terminology Fundamentals** since it isn't a published spec yet.

### Risks
- Section number references throughout the ticket are based on the *current* numbering and will shift after the Basic Concepts section is moved. The ticket acknowledges this but implementation must be careful.
- The systematic "codes" → "concept codes" replacement requires care to avoid unintended changes in code system, error code, etc. contexts.

## Proposed Dispositions

### Disposition A: Persuasive (Recommended)
**Rationale:** The changes are well-scoped editorial improvements that align the Terminology Service page with current FHIR terminology conventions. The "concept codes" consistency pass addresses genuine reader confusion. The structural reorganization (moving Basic Concepts first) improves discoverability. All changes are clearly documented with specific before/after text. The ticket originates from a formal TI co-chair review, demonstrating organizational endorsement. Accept as-is and implement all proposed changes.

### Disposition B: Persuasive with Modification
**Rationale:** Accept the bulk of the changes but request clarification on two items: (1) The Terminology Fundamentals link — defer adding this link until the spec is published, or add it with a note that it's pre-publication. (2) The "concept codes" replacement — consider whether "concept codes" is the ideal replacement term everywhere, or if some instances would be better served by "concepts" or "codes" depending on context (e.g., §4.7.4 bullet about "concept designations" suggests "concepts" alone may sometimes be more appropriate).

### Disposition C: Not Persuasive
**Rationale:** The changes are editorial in nature but extremely numerous. Given the risk of introducing inconsistencies across the many sections touched and the potential for section-number confusion after the Basic Concepts move, defer this work until other structural tickets (FHIR-53787, FHIR-54107) are resolved, then revisit as a coordinated editorial pass. Not persuasive for this ballot cycle due to scope concerns.

## Recommendation

**→ Disposition A: Persuasive**

The changes are clearly documented, originate from a formal co-chair review, and address genuine readability and consistency issues on the Terminology Service page. The "concept codes" standardization aligns with related efforts in FHIR-54129 (already resolved as Persuasive). The structural changes (moving Basic Concepts, removing deprecated External Code Systems section) are well-justified. Implementation should coordinate with FHIR-53787 and FHIR-54107 to ensure consistency.
