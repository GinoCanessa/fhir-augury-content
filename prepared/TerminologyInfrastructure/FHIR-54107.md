## Summary

This change request proposes removing **Section 4.3.0.1 "External Code Systems"** from the [terminologies-systems.html](https://hl7.org/fhir/6.0.0-ballot4/terminologies-systems.html) page of the FHIR core specification. The rationale is that this section duplicates content now maintained in HL7 Terminology (THO), specifically the "Using xxx with HL7 Standards" pages that have already migrated there. Maintaining the same information in two places creates a maintenance burden and risks inconsistency.

The ticket requests four specific changes:

1. **Remove Section 4.3.0.1** entirely from the terminologies-systems page.
2. **Remove the sentence** "Don't see what you're looking for here: See Registering Code Systems for further advice."
3. **Reword the introductory text** from a statement about a "subset of a much larger list" to a concise redirect pointing readers to [terminology.hl7.org/external_code_systems.html](https://terminology.hl7.org/external_code_systems.html), noting that code system publishers have provided usage information there.
4. **Update "Important Notes"**: remove the first bullet, and change the second bullet's tab name from "Defined Externally" to "External Content".

## Details

### Description

The information in Section 4.3.0.1 repeats content in THO. Now that the "Using xxx with HL7 Standards" pages have moved to THO, there is no need to maintain in two places.

**Suggested text changes:**

**Current introductory text:**
> This is a subset of a much larger list of known code systems, and only details code systems for which FHIR specific usage documentation is provided. For other code systems, see [terminology.hl7.org](https://terminology.hl7.org/external_code_systems.html).

**Proposed replacement:**
> For code systems defined external to HL7, see [terminology.hl7.org](https://terminology.hl7.org/external_code_systems.html). Some code system publishers have provided information about how to use their code system with HL7 standards. You will find that information at the same link.

**Important Notes bullet changes:**
- Remove the first bullet entirely.
- Change the second bullet from: "The external terminology content provided in the **Defined Externally** tab is provided for convenience and should not be considered authoritative nor normative." → To: "The external terminology content provided in the **External Content** tab is provided for convenience and should not be considered authoritative nor normative."

## Proposed Dispositions

### Disposition A: Accept (Persuasive)

Accept the ticket as submitted. Remove Section 4.3.0.1 "External Code Systems" from the `terminologies-systems.html` page. Replace the introductory text with the proposed wording that redirects readers to `terminology.hl7.org/external_code_systems.html`. Remove the sentence about "Registering Code Systems". Update the Important Notes section by removing the first bullet and changing the tab reference from "Defined Externally" to "External Content" in the second bullet.

**Rationale:** This is a natural continuation of the work already accepted in FHIR-37853 and FHIR-35933, which established THO as the single source of truth for external terminology information. The section being removed is acknowledged as duplicative content. Removing it reduces maintenance burden, avoids stale/broken links (as demonstrated by the Zulip discussion about the 404), and aligns with HTA/TSMG/TSC policy decisions. The proposed replacement text is clear and directs users to the authoritative source.

### Disposition B: Accept with Modification (Persuasive with Modification)

Accept the removal of Section 4.3.0.1 and the text changes, but modify the proposed replacement text to also include a brief transitional note for implementers who relied on the previous section. For example, add: "Note: Section 4.3.0.1 (External Code Systems) has been removed. The authoritative and complete listing of external code system URIs and usage guidance is maintained exclusively at [terminology.hl7.org](https://terminology.hl7.org/external_code_systems.html)." Additionally, coordinate with FHIR-53809 to ensure the parallel removal of Section 4.7.3.1 from the Terminology Service page uses consistent wording.

**Rationale:** While the removal is appropriate, a transitional note would help implementers who have bookmarked or referenced Section 4.3.0.1 understand where the content moved. Coordinating language with sibling tickets (FHIR-53809, FHIR-53786) ensures consistency across the spec.

### Disposition C: Decline (Not Persuasive)

Decline the change on the basis that having a summary of external code systems directly in the FHIR core spec provides convenient reference for implementers who may not be aware of THO, and that removing the section could make the FHIR spec less self-contained.

**Rationale:** Some implementers rely on the FHIR spec as their primary reference. However, this argument is weakened by the fact that FHIR-37853 and FHIR-35933 have already been accepted as Persuasive, establishing the policy that THO is the authoritative source. Maintaining duplicate content contradicts that policy and creates the risk of inconsistency.

## Recommendation

**Disposition A: Accept (Persuasive)**

The removal of Section 4.3.0.1 is well-justified and consistent with established HL7 policy (HTA/TSMG/TSC decisions documented in FHIR-37853). The foundational migration work was completed via FHIR-35933, making this section genuinely redundant. The Zulip discussion about broken links to the old `external_terminologies.html` URL concretely demonstrates the maintenance hazard of keeping duplicated content. The proposed replacement text is clear, concise, and correctly points to the current THO URL (`external_code_systems.html`). This ticket is part of a coordinated set of R6 terminology page cleanups (FHIR-53785, FHIR-53786, FHIR-53809) and should be accepted to maintain consistency across that effort.
