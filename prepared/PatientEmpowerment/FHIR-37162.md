## Description

The ticket requests that the **PatientCorrectionBusinessStatusVS** ValueSet canonical URL be moved from the IG namespace to HL7 Terminology (THO). The current canonical URL is:

> `http://hl7.org/fhir/uv/patient-corrections/ValueSet/PatientCorrectionBusinessStatusVS`

The request states the ValueSet should be registered in `terminology.hl7.org` (THO) via a UTG ticket. Only ValueSets with a **required** binding to a `code` data type should remain anchored in `hl7.org/fhir`. This ticket was originally imported as "Comment 5" by Ron G. Parker.

## Proposed Dispositions

### Disposition A: Persuasive — Move ValueSet Canonical URL to THO

**Action:** Accept the request. Create a UTG ticket to register `PatientCorrectionBusinessStatusVS` in terminology.hl7.org (THO). Update the ValueSet canonical URL from `http://hl7.org/fhir/uv/patient-corrections/ValueSet/PatientCorrectionBusinessStatusVS` to a THO-based URL (e.g., `http://terminology.hl7.org/ValueSet/PatientCorrectionBusinessStatusVS`). Coordinate with TSMG for registration.

**Rationale:** This aligns with HL7 best practices for terminology management as documented in the TSMG Code System Identifier Task Force guidance and the R5 specification. The `PatientCorrectionBusinessStatusVS` ValueSet does not have a required binding to a `code` data type, so it should not remain anchored in the IG namespace. The 8 sibling tickets should be resolved consistently. The ticket was previously deferred as "Considered for Future Use" — post-STU1 publication is the appropriate time to make this change.

**Considerations:** This is a breaking change for implementers referencing the current canonical URL. The IG should include a migration note. All 8 sibling tickets (FHIR-37158–37165) should be resolved together to ensure consistency. A UTG ticket must be created as a prerequisite.

### Disposition B: Persuasive with Modification — Batch All 8 THO Anchoring Tickets Together

**Action:** Accept the request, but resolve this ticket as part of a coordinated batch with all 8 sibling tickets (FHIR-37158–37165). Create a single UTG proposal covering all 4 CodeSystems and 4 ValueSets. Evaluate each resource's binding strength to determine which truly need to move to THO versus which (if any) should remain IG-anchored due to required bindings on `code` data types.

**Rationale:** Handling all 8 tickets as a batch ensures consistency, avoids partial migrations, and reduces coordination overhead with TSMG/UTG. The workgroup can evaluate binding strengths for all resources at once and make a single decision about the migration strategy.

**Considerations:** Requires more upfront analysis of binding strengths across all affected profiles. May take longer to resolve but produces a more coherent outcome.

### Disposition C: Not Persuasive — Keep Current Canonical URLs

**Action:** Reject the request. Keep the ValueSet canonical URL anchored in the IG namespace.

**Rationale:** Moving canonical URLs is a breaking change for existing implementers. The Patient Corrections IG is a UV (universal) IG and the current URL structure is functional. The original 7-0-0 vote for "Considered for Future Use" reflected a deliberate decision to defer, and the conditions may not have changed sufficiently to warrant action now.

**Considerations:** This goes against HL7 terminology best practices as documented by TSMG. It would leave the Patient Corrections IG as an outlier compared to other IGs that have adopted THO anchoring. The ticket was explicitly re-opened post-STU1 publication, indicating the workgroup intends to address it now.

## Recommendation

**Disposition B: Persuasive with Modification** — Resolve this ticket as part of a coordinated batch migration of all 8 THO anchoring tickets (FHIR-37158–37165). This approach aligns with HL7 terminology best practices, addresses the original TSMG concern, and ensures a consistent migration strategy across all Patient Corrections CodeSystems and ValueSets. The post-STU1 timing is appropriate for this breaking change, and batching reduces coordination overhead with UTG/TSMG.
