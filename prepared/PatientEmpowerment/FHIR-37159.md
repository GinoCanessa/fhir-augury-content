This ticket requests that the canonical URL for the `PatientCorrectionCommunicationTypes` CodeSystem be moved from its current IG-based location (`http://hl7.org/fhir/uv/patient-corrections/CodeSystem/PatientCorrectionCommunicationTypes`) to the HL7 Terminology (THO) namespace at `terminology.hl7.org`. A corresponding UTG ticket should be created to register the CodeSystem in THO. This aligns with TSMG best practices for CodeSystem identifier management documented in the R5 specification.

The ticket was originally filed by Joan Harper (TSMG) in May 2022 as part of a batch of four related vocabulary tickets (FHIR-37158 through FHIR-37161) covering all four CodeSystems in the Patient Corrections IG. It was previously resolved as "Considered for Future Use" in October 2022 (vote: 7-0-0, Jason Vogt / Rachel Richesson), with the rationale to consider for future implementation once further code changes were no longer expected. It was reopened by Lynn Laakso on 2025-07-10 following STU1 publication.

## Details

### Current State

The `PatientCorrectionCommunicationTypes` CodeSystem currently uses the canonical URL:

```
http://hl7.org/fhir/uv/patient-corrections/CodeSystem/PatientCorrectionCommunicationTypes
```

### Requested Change

Move the CodeSystem canonical URL to be anchored in THO (`terminology.hl7.org`) per TSMG best practices. This requires:

1. Creating a UTG ticket to register the CodeSystem in THO
2. Updating the canonical URL in the Patient Corrections IG to reference the THO-based URL
3. Coordinating with TSMG for proper registration

## Proposed Dispositions

### Disposition A: Accept — Migrate Canonical URL to THO

**Action:** Accept the change request. Create a UTG ticket to register the `PatientCorrectionCommunicationTypes` CodeSystem in THO and update the canonical URL from `http://hl7.org/fhir/uv/patient-corrections/CodeSystem/PatientCorrectionCommunicationTypes` to a THO-based URL (e.g., `http://terminology.hl7.org/CodeSystem/PatientCorrectionCommunicationTypes`).

**Rationale:** This aligns with established TSMG best practices and the R5 specification guidance on CodeSystem identifiers. The ticket was previously deferred until the IG stabilized; now that STU1 has been published, the timing is appropriate. All four sibling tickets (FHIR-37158 through FHIR-37161) should be addressed together to maintain consistency. The previous workgroup vote (7-0-0) endorsed the direction but deferred timing.

**Impact:** Requires coordination with TSMG/UTG. Existing implementations referencing the old canonical URL would need to be updated or a redirect mechanism established. This is a breaking change for systems that have adopted the STU1 URL.

### Disposition B: Accept with Transition Plan — Migrate with Version-Specific Aliasing

**Action:** Accept the change, but implement a transition plan that maintains the old IG-based URL as an alias or supplement identifier for a deprecation period (e.g., one ballot cycle or STU2 publication), while establishing the new THO-based canonical URL as primary.

**Rationale:** This reduces the breaking-change impact on early adopters of the Patient Corrections STU1 IG. It follows the spirit of the TSMG guidance while acknowledging that immediate hard migration may disrupt implementations. The IG can document both URLs during the transition and formally retire the old URL in a subsequent version.

**Impact:** More complex to implement — requires both a UTG registration and IG updates to document the dual-URL transition. Adds editorial overhead but minimizes implementer disruption.

### Disposition C: Decline — Retain Current IG-Based URL

**Action:** Decline the change request. Keep the canonical URL as `http://hl7.org/fhir/uv/patient-corrections/CodeSystem/PatientCorrectionCommunicationTypes`.

**Rationale:** The CodeSystem is specific to the Patient Corrections IG and may not warrant registration in THO. Changing canonical URLs post-STU1 publication introduces breaking changes for adopters. The IG-based URL is functional and unambiguous. Other IGs continue to use IG-based CodeSystem URLs without issue.

**Impact:** No implementation changes required. However, this deviates from TSMG best practices and the direction endorsed by the prior 7-0-0 vote. It may be questioned during future ballot reconciliation.

## Recommendation

**Disposition A: Accept — Migrate Canonical URL to THO.**

The workgroup previously endorsed this direction unanimously (7-0-0) and only deferred it due to timing. With STU1 now published and the ticket explicitly reopened for action, the conditions for deferral no longer apply. Moving the canonical URL to THO aligns with TSMG best practices, R5 specification guidance, and HL7-wide vocabulary governance. The four sibling tickets (FHIR-37158 through FHIR-37161) should be addressed as a coordinated batch to ensure all Patient Corrections CodeSystems are migrated consistently. A UTG ticket should be created as the first step.
