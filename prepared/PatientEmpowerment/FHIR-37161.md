## Description

The CodeSystem canonical URL for **PatientCorrectionTaskTypes** is currently anchored under `hl7.org/fhir`:

> `http://hl7.org/fhir/uv/patient-corrections/CodeSystem/PatientCorrectionTaskTypes`

The request is to re-anchor this URL under `terminology.hl7.org` (THO) per HL7 best practices. A UTG ticket should be created to register the CodeSystem in THO.

The ticket references:
- [TSMG Internal CodeSystem Identifier Task Force presentation (Jan 2022)](https://confluence.hl7.org/download/attachments/81034628/January2022_TSMG_Internal_CodeSystemIdentifierTaskForce.pptx?version=1&modificationDate=1644006505101&api=v2)
- [R5 Terminologies Best Practices](http://hl7.org/fhir/5.0.0-snapshot1/terminologies.html#system)

This is **Comment 4** from a series of vocabulary ballot comments imported by Ron G. Parker.

## Proposed Dispositions

### Disposition A: Applied — Migrate CodeSystem URL to THO

**Action:** Change the canonical URL from `http://hl7.org/fhir/uv/patient-corrections/CodeSystem/PatientCorrectionTaskTypes` to a THO-anchored URL (e.g., `http://terminology.hl7.org/CodeSystem/PatientCorrectionTaskTypes`). Create a UTG ticket to register the CodeSystem in THO. Coordinate with TSMG for the actual URL assignment.

**Rationale:** This aligns with HL7 best practices documented in the R5 specification and the TSMG Code System Identifier Task Force recommendations. CodeSystems defined by HL7 IGs should be registered in THO. The ticket was previously deferred and has now been re-opened post-STU1 specifically to address this. All eight sibling tickets should ideally be handled together as a coordinated batch.

**Impact:** Breaking change for any implementers referencing the current canonical URL. Requires a UTG ticket and TSMG coordination. Should be bundled with FHIR-37158, FHIR-37159, FHIR-37160 (the other three CodeSystem tickets) for consistency.

### Disposition B: Deferred — Defer to next ballot cycle with coordinated plan

**Action:** Defer this ticket and its seven siblings to a future ballot cycle, with an explicit plan to coordinate all eight migrations as a single batch effort including UTG registration.

**Rationale:** Migrating CodeSystem and ValueSet URLs is a breaking change that needs careful planning. A coordinated migration plan covering all eight terminology artifacts, along with implementer migration guidance and a deprecation period, would reduce disruption. The workgroup may want to first assess current implementer adoption before committing to the URL change.

**Impact:** Continues using non-best-practice URLs for another cycle but reduces risk of uncoordinated changes. Delays compliance with THO anchoring policy.

### Disposition C: Applied — Migrate with deprecation and redirect strategy

**Action:** Change the canonical URL to a THO-anchored URL AND define a formal migration strategy: (1) create UTG ticket to register in THO, (2) publish the new URL in the next version, (3) document the old-to-new URL mapping, (4) coordinate with the publisher to support redirects from old URLs. Apply to all eight sibling tickets as a batch.

**Rationale:** This fully addresses the request while minimizing implementer disruption through a managed transition. The old URLs could remain functional as redirects for a deprecation period. This approach was effectively what was envisioned when the ticket was originally deferred — now that STU1 is published, the timing is appropriate.

**Impact:** More work upfront but provides a smoother transition path. Requires coordination with TSMG, UTG, and the IG publisher infrastructure.

---

## Recommendation

**Disposition C: Applied — Migrate with deprecation and redirect strategy**

This is the most thorough approach and the right time to act. The tickets were originally deferred pre-STU1 and have now been specifically re-opened post-publication. The R5 best practices are clear that CodeSystem URLs should be anchored in THO. A managed migration with all eight sibling tickets handled as a batch — including UTG registration, documented URL mappings, and redirect support — provides the best balance of compliance and implementer stability. The workgroup should coordinate with TSMG to begin the UTG registration process.
