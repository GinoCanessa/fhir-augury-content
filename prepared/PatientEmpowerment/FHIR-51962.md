## Description

The submitter requests that the "How to read this guide" section on the PACIO ADI IG home page be updated to reflect the current contents of the IG. Specifically, the General Guidance bullet is missing several pages such as Signatures and Search Parameters. The section no longer accurately represents what is in the IG, making it harder for implementers to navigate.

## Resolution Description

The resolution (Persuasive) agrees that the work group needs to:
- Better align the home page "How to Read" section with the menu format.
- Be more descriptive on the menu sections to guide the implementer.

## Analysis

This is a straightforward editorial/navigation improvement request. The "How to read this guide" section on the PACIO ADI IG home page has fallen out of sync with the actual IG contents — pages like Signatures and Search Parameters under General Guidance are not listed. The resolution acknowledges this gap and commits to aligning the section with the current menu structure and adding more descriptive text.

This ticket is closely related to FHIR-52645 (improve descriptions in the same section) and FHIR-52643 (fix heading capitalization). All three address the same section and could be implemented together. Prior STU1 tickets FHIR-35684 and FHIR-34807 already addressed broader home page improvements, indicating an ongoing pattern of feedback about IG navigability.

The scope is low-risk editorial work with no impact on conformance, profiles, or technical content.

## Proposed Dispositions

### Disposition A: Persuasive (Recommended)
**Action:** Accept as Persuasive, matching the existing resolution. Update the "How to read this guide" section to enumerate all current IG pages and align bullet structure with the navigation menu. Ensure missing pages (Signatures, Search Parameters, and any others added since the last update) are included.

**Rationale:** The request is factually correct — the section is incomplete and does not match the current IG contents. The resolution text already acknowledges the need for alignment and better descriptions. This is a non-controversial editorial fix that improves implementer experience. Coordinating with FHIR-52645 and FHIR-52643 during implementation would be efficient.

### Disposition B: Persuasive with Modification
**Action:** Accept as Persuasive with Modification. In addition to updating the section to reflect current contents, restructure the "How to read this guide" section to group pages by audience (e.g., clinical vs. technical implementers) rather than simply mirroring the menu order.

**Rationale:** While the core request is valid, a more structured approach could provide additional value. However, this adds scope beyond what was requested and may conflict with FHIR-52645 which addresses descriptive improvements separately.

### Disposition C: Not Persuasive
**Action:** Reject the change.

**Rationale:** The IG's navigation menu already serves as the authoritative table of contents, and duplicating it in the "How to read this guide" section creates a maintenance burden — the section will fall out of sync again with future updates. However, this argument is weak given that the section already exists and implementers rely on it for orientation.

---

## Recommendation

**Disposition A: Persuasive** — The request is clear, well-justified, and the resolution already agrees. The "How to read this guide" section should be updated to match the current IG contents. Implementation should be coordinated with FHIR-52645 (richer descriptions) and FHIR-52643 (heading capitalization) to address all three related tickets together efficiently.
