## Description

The submitter (Joan Harper, on behalf of TSMG) requests that the CodeSystem canonical URL for `PatientCorrectionOutputTypes` be moved from its current IG-based URL to the HL7 Terminology (THO) namespace. Specifically:

- **Current URL:** `http://hl7.org/fhir/uv/patient-corrections/CodeSystem/PatientCorrectionOutputTypes`
- **Requested URL:** Should be anchored under `terminology.hl7.org` (THO)

A UTG ticket must be created to register the CodeSystem in THO. The submitter references best practices from:
- The TSMG Internal Code System Identifier Task Force (January 2022 presentation)
- The R5 specification on terminology systems (`http://hl7.org/fhir/5.0.0-snapshot1/terminologies.html#system`)

## Analysis

This is a well-established HL7 best practice issue. The TSMG Code System Identifier Task Force established guidance that HL7-defined CodeSystems should use canonical URLs under `terminology.hl7.org` rather than being embedded under individual IG URLs (`hl7.org/fhir/uv/...`). This ensures CodeSystems are properly registered in UTG (Unified Terminology Governance) and are discoverable and maintainable as shared vocabulary resources.

The ticket was originally deferred ("Considered for Future Use") during the ballot cycle when the IG was still undergoing active development. Now that the Patient Corrections IG has reached STU1 publication, the workgroup has re-opened this and its three sibling tickets to address the terminology governance concern.

**Impact:** Changing the canonical URL for `PatientCorrectionOutputTypes` is a **breaking change** for any system that references this CodeSystem by its current URL. However, this is standard practice during the STU lifecycle and aligns with HL7 terminology best practices. A UTG ticket will need to be filed to register the new URL. All four sibling tickets should ideally be addressed together to avoid piecemeal URL migrations.

## Proposed Dispositions

### Disposition A: Persuasive (Recommended)
**Action:** Accept the change request. Migrate the `PatientCorrectionOutputTypes` CodeSystem canonical URL to `terminology.hl7.org`. File a UTG ticket to register the CodeSystem in THO.

**Rationale:** This aligns with established HL7/TSMG best practices for CodeSystem canonical URL governance. The ticket was previously deferred with unanimous support (7-0-0) only because the IG was still in active development. Now that STU1 has been published, the rationale for deferral no longer applies. Address all four sibling tickets (FHIR-37158, FHIR-37159, FHIR-37160, FHIR-37161) together to migrate all Patient Corrections CodeSystems to THO in a coordinated manner.

### Disposition B: Persuasive with Modification
**Action:** Accept the change in principle but coordinate with TSMG on the exact new canonical URLs before implementation. Defer actual implementation until the next STU update cycle to allow proper migration planning.

**Rationale:** While the direction is correct, changing canonical URLs is a breaking change. A phased approach with explicit migration guidance (e.g., documenting both old and new URLs during a transition period) would reduce implementer disruption. This gives time to file and process the UTG ticket and coordinate with all four sibling tickets.

### Disposition C: Not Persuasive
**Action:** Decline the change. Retain the current IG-based canonical URL.

**Rationale:** The CodeSystem is specific to the Patient Corrections IG and may not warrant registration in THO as a shared terminology resource. The current URL is functional and already in use by implementers. Changing it introduces a breaking change with limited practical benefit for a domain-specific CodeSystem. However, this disposition conflicts with TSMG best practices and the unanimous prior workgroup sentiment.

## Recommendation

**Disposition A: Persuasive** — The change aligns with HL7 terminology governance best practices, has prior unanimous workgroup support, and the deferral rationale (active IG development) no longer applies. The workgroup should coordinate all four sibling tickets together and file the corresponding UTG tickets.
