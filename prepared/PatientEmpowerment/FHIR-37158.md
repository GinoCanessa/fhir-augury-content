## Summary

This ticket requests that the `PatientCorrectionBusinessStatus` CodeSystem canonical URL be migrated from `hl7.org/fhir` to `terminology.hl7.org` (THO — the HL7 Terminology server). Per HL7 best practices established by the TSMG Code System Identifier Task Force, code system canonical URLs should be anchored in THO rather than in the FHIR IG namespace. A corresponding UTG ticket must be created to register the code system in THO.

The current (incorrect) canonical URL is:
`http://hl7.org/fhir/uv/patient-corrections/CodeSystem/PatientCorrectionBusinessStatus`

This ticket was originally resolved as "Considered for Future Use" in October 2022 (voted 7-0-0, Jason Vogt / Rachel Richesson) with the rationale that it would be considered once further code changes were no longer pending. It was re-opened on 2025-07-10 by Lynn Laakso following STU1 publication.

## Proposed Dispositions

### Disposition A: Accept — Migrate canonical URL to THO ✅

**Resolution:** Applied  
**Action:** Change the `PatientCorrectionBusinessStatus` CodeSystem canonical URL from `http://hl7.org/fhir/uv/patient-corrections/CodeSystem/PatientCorrectionBusinessStatus` to a THO-anchored URL (e.g., `http://terminology.hl7.org/CodeSystem/PatientCorrectionBusinessStatus`). Create a UTG ticket to register the code system in THO. Apply the same migration to the three sibling code systems (FHIR-37159, FHIR-37160, FHIR-37161) simultaneously. Include version-mapping / redirect guidance so existing implementations can transition.

**Rationale:** This aligns with HL7 best practices established by the TSMG Code System Identifier Task Force and documented in the R5 specification. The STU1 publication is now complete, removing the original deferral rationale. The previous vote (7-0-0) was unanimously in favor of eventual adoption. Addressing all four code systems together ensures consistency.

### Disposition B: Accept with phased approach — Register in THO, support both URLs during transition

**Resolution:** Applied  
**Action:** Register the code system in THO and update the IG to use the THO-anchored canonical URL as primary, but document the old URL as an alias during a defined transition period (e.g., one ballot cycle). Publish migration guidance for implementers. Coordinate with TSMG on timing and UTG registration.

**Rationale:** This reduces the risk of breaking existing implementations that reference the current canonical URL. It provides a clear migration path while still achieving the goal of THO alignment. Appropriate if the workgroup is concerned about implementer disruption post-STU1.

### Disposition C: Decline — Retain current canonical URL

**Resolution:** Not Persuasive  
**Action:** Keep the current canonical URL `http://hl7.org/fhir/uv/patient-corrections/CodeSystem/PatientCorrectionBusinessStatus` unchanged.

**Rationale:** Changing canonical URLs post-publication is disruptive to implementers who have already adopted the STU1 specification. The current URL is functional and resolvable. However, this contradicts established HL7 terminology best practices, the unanimous prior workgroup sentiment, and the TSMG guidance.

---

## Recommendation

**Disposition A: Accept — Migrate canonical URL to THO**

The request is well-founded, aligns with established HL7 terminology best practices, and was previously supported unanimously (7-0-0). The only reason for deferral was to wait until STU1 publication was complete — which it now is, as confirmed by the 2025-07-10 re-opening. All four sibling code system tickets should be addressed together for consistency. A UTG registration ticket should be created as part of the resolution. This is a straightforward vocabulary governance action with clear institutional backing from TSMG.
