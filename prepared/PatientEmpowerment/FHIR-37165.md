## Description

The ValueSet canonical URL for `PatientCorrectionTaskTypesVS` should be anchored in terminology.hl7.org (THO) instead of `hl7.org/fhir`. A UTG ticket should be created to register the ValueSet in THO. Per TSMG guidance, only those ValueSets with required binding to a `code` data type should retain canonical URLs under `hl7.org/fhir`.

**Existing URL:**
`http://hl7.org/fhir/uv/patient-corrections/ValueSet/PatientCorrectionTaskTypesVS`

This is one of a series of 8 related tickets (FHIR-37158 through FHIR-37165) filed by Joan Harper requesting that all CodeSystem and ValueSet canonical URLs in the Patient Corrections IG be migrated to THO. This ticket specifically addresses the `PatientCorrectionTaskTypesVS` ValueSet.

## Proposed Dispositions

### Disposition A: Applied — Migrate ValueSet URL to THO

**Action:** Move the `PatientCorrectionTaskTypesVS` ValueSet canonical URL from `http://hl7.org/fhir/uv/patient-corrections/ValueSet/PatientCorrectionTaskTypesVS` to a THO-anchored URL under `http://terminology.hl7.org/ValueSet/`. Create a UTG ticket to register the ValueSet in THO. Update all references to this ValueSet throughout the Patient Corrections IG.

**Rationale:** This aligns with HL7 terminology governance best practices as documented in the TSMG Code System Identifier Task Force guidance and the R5 specification. Since this ValueSet does not have a required binding to a `code` data type, it should not be anchored under `hl7.org/fhir`. STU1 has been published and the specification is now stable enough to make this governance change. All 8 sibling tickets should ideally be addressed together for consistency.

**Impact:** Non-compatible (canonical URL change is a breaking change for implementers referencing the existing URL). The IG will need to provide migration guidance and potentially maintain the old URL as an alias during a transition period.

### Disposition B: Deferred — Defer to Next Ballot Cycle

**Action:** Keep the current ValueSet URL unchanged. Defer this work to a future ballot cycle, coordinating with TSMG to plan the migration across all 8 related tickets simultaneously.

**Rationale:** Migrating canonical URLs is a breaking change that affects all implementers currently using the Patient Corrections IG. It may be prudent to batch all 8 THO anchoring tickets together and coordinate with UTG/TSMG to execute the migration in a single coordinated effort during a major version update rather than piecemeal.

**Impact:** No immediate impact. Continues the current non-conformance with THO best practices but avoids disruption to existing implementers.

### Disposition C: Not Persuasive — Retain Current URL

**Action:** Resolve as Not Persuasive. Keep the ValueSet URL under `hl7.org/fhir/uv/patient-corrections`.

**Rationale:** The current URL is functional and already published in STU1. Moving canonical URLs is inherently disruptive and the benefit of THO anchoring for an IG-specific ValueSet may not justify the migration cost. The previous 7-0-0 vote to defer suggests the work group considered this low urgency relative to other work.

**Impact:** No change. Non-conformance with TSMG best practices persists.

## Recommendation

**Disposition A: Applied — Migrate ValueSet URL to THO**

This is the recommended disposition. The ticket has been re-opened specifically because STU1 publication is complete, removing the original reason for deferral. TSMG guidance is clear that ValueSets without required `code` bindings should be anchored in THO, and this is an established best practice in the R5 specification. The work group should coordinate with TSMG/UTG to execute the migration across all 8 sibling tickets (FHIR-37158 through FHIR-37165) together for consistency. The next ballot cycle provides a natural opportunity to introduce this breaking change with proper migration guidance for implementers.
