## Summary

This ticket requests revision of short descriptions in the **CQFM Composite Measure** profile (section 17.46.1.1 of the Quality Measures IG). The reporter identifies that several element short descriptions are not sufficiently descriptive. The affected elements are:

- `Measure.scoringUnit`
- `.relatedArtifact:component.groupId`
- `.relatedArtifact:component.weight`
- `.group.scoringUnit`
- `.group.component`
- `.group.component.groupId`
- `.group.component.weight`
- `.group.population`

The request is to rework these short descriptions to be more helpful and clearly descriptive of what each element represents in the context of composite measure profiles.

## Analysis

**Nature of Change:** This is an editorial/documentation improvement request. It does not propose changes to cardinality, data types, or structural constraints — only to the human-readable short descriptions (i.e., the `short` property on `ElementDefinition`) within the CQFM Composite Measure profile.

**Scope:** Eight elements in the Composite Measure profile are identified. The elements span the root Measure level (`scoringUnit`), related artifact slices (`component.groupId`, `component.weight`), and group-level elements (`scoringUnit`, `component`, `component.groupId`, `component.weight`, `population`).

**Impact:** Low technical risk. Short descriptions appear in IG documentation and tooling (e.g., profile rendering), but do not affect validation, conformance, or data exchange behavior. Improved short descriptions would enhance readability and implementer understanding of the composite measure profile.

**Context:** This is part of a batch of five related tickets from the same reporter, all requesting the same type of editorial improvement across different CQFM measure profiles. A consistent approach across all five tickets would be desirable.

**Gap:** The ticket identifies the problem (short descriptions are not descriptive) but does not propose specific replacement text. The work group may want to request proposed text from the reporter or draft text during review.

---

## Proposed Dispositions

### Disposition A: Persuasive
**Action:** Accept the request. Revise the short descriptions for all eight identified elements in the CQFM Composite Measure profile to be more descriptive of each element's purpose in the context of composite measures. Coordinate with the sibling tickets (FHIR-45681, FHIR-45683, FHIR-45684, FHIR-45685) to apply a consistent editorial style across all CQFM measure profiles.

**Rationale:** The request is straightforward and improves IG usability. Short descriptions should clearly convey the purpose of each element, especially in specialized profiles where the element's role may differ from its base definition. This is a low-risk editorial change with clear benefit to implementers.

### Disposition B: Persuasive with Modification
**Action:** Accept in principle, but request the reporter provide specific proposed short description text for each of the eight elements before applying changes. Consolidate this ticket with the four sibling tickets into a single editorial pass across all CQFM measure profiles.

**Rationale:** While the problem is valid, the ticket lacks concrete replacement text. Asking for proposed text would accelerate editorial work and ensure the changes meet the reporter's expectations. Consolidating the batch of tickets would enable a more efficient and consistent review.

### Disposition C: Not Persuasive
**Action:** Decline the request. The current short descriptions, while not ideal, are functional and the element names themselves provide sufficient context for implementers familiar with composite measures.

**Rationale:** Short descriptions are supplementary documentation. The effort to revise them across multiple profiles may not be justified given other priorities, and the current descriptions are not incorrect — just not optimally descriptive.

---

## Recommendation

**Disposition A: Persuasive** is recommended.

The request is valid, low-risk, and straightforward. Improving short descriptions is a worthwhile editorial improvement that directly benefits implementer experience. The work group should accept the request and, ideally, address all five sibling short description tickets (FHIR-45681 through FHIR-45686) together in a single editorial pass to ensure consistency. If proposed text is not provided, the editor assigned to the work can draft appropriate descriptions based on the element definitions and composite measure semantics.
