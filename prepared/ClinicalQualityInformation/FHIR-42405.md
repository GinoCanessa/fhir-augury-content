## Proposed Dispositions

### Disposition A: Accept As Requested
#### Proposal
Enhance the Value-Based Performance MeasureReport profiles and supporting guidance to specify what information must be included in reports, ensuring that providers can view detailed breakdowns of report content including member/patient-level data. Leverage the ATR Group profile subject reference introduced by FHIR-52950 as a foundation and add additional guidance or required elements as needed to fulfill the submitter's request.
#### Justification
The ticket has been deferred once already with a clear work group consensus to address it in STU2. The related work in FHIR-52950 (using ATR Group as subject) has already been published and partially addresses the concern by enabling drill-down to member/patient lists. Accepting the request aligns with the prior commitment to the submitter and builds on infrastructure that is now in place.

---

### Disposition B: Alternative Approach
#### Proposal
Mark this ticket as resolved by referencing FHIR-52950 (Use ATR Group as subject), which has already been published and addresses the core concern of enabling providers to drill down into report details. If additional information requirements beyond member/patient lists are identified, file new, more specific tickets for each enhancement.
#### Justification
The primary actionable aspect of this request — enabling providers to see which members/patients are included in a report — has been addressed by FHIR-52950. Since the original ticket lacks a detailed description specifying exactly what additional information is desired, it may be more productive to close this broad request and track any remaining gaps as separate, well-defined tickets. This avoids scope creep and ensures each enhancement receives focused review.

---

### Disposition C: Decline
#### Proposal
Decline the request on the grounds that it is insufficiently specified. The ticket title "Information included in reports" and the absence of a description make it unclear what specific changes are being requested beyond what FHIR-52950 has already delivered.
#### Justification
Without a concrete description of what information is missing or what changes are needed, the work group cannot produce a targeted specification change. The submitter has not provided additional detail since the original filing in 2023, and the most recent comment from yanheras suggests the underlying concern may already be addressed by FHIR-52950. Declining encourages re-submission with actionable specifics if gaps remain.

---

### Recommendation
**Recommended disposition:** B (Alternative Approach)

The strongest path forward is to recognize that FHIR-52950 has materially addressed the core concern raised by this ticket. The adoption of the ATR Group profile as the MeasureReport subject directly enables providers to drill down into member and patient lists associated with a report — the very capability highlighted in the most recent comment. Since the original ticket lacks a description and no submitter follow-up has clarified what additional information is desired, treating FHIR-52950 as the resolution provides closure while keeping the door open for new, well-scoped tickets if specific gaps are identified. This approach honors the work group's prior commitment to address the issue in STU2 without leaving an open-ended, under-specified ticket on the backlog.
