## Description

Section 17.0.9 (Example Instances) of the Quality Reporting IG contains several examples that use the generic placeholder description "This is an example only." The reporter requests that these placeholder descriptions be replaced with more in-depth, meaningful descriptions that help the reader understand what each example demonstrates.

## Proposed Dispositions

### Disposition A: Persuasive

**Rationale:** The request is straightforward and well-justified. Placeholder descriptions like "This is an example only" provide no value to readers and are inconsistent with HL7 IG quality expectations. Improving example descriptions enhances the usability of the IG and helps implementers understand what each example demonstrates. This is consistent with FHIR-47427 (resolved Persuasive for Clinical Documents IG) which made an essentially identical request. The work group should update all examples in section 17.0.9 that currently use the generic placeholder text, replacing each with a description specific to what the example illustrates.

### Disposition B: Persuasive with Modification

**Rationale:** Accept the request but scope the change more precisely. Rather than requiring "more in-depth descriptions" for every example, require that each example description: (1) identify the profile being exemplified, (2) summarize the scenario or use case demonstrated, and (3) remain concise (1–2 sentences). This gives authors clear guidance on what constitutes an adequate description and prevents overly verbose or inconsistent results.

### Disposition C: Not Persuasive with Modification

**Rationale:** Acknowledge the concern but note that the IG publisher tooling already provides structural context (profile name, resource type) alongside each example. Request that the reporter identify the specific examples where the placeholder text is most problematic, and limit the change to those instances rather than requiring a blanket update to all examples. This reduces the scope of work while still addressing the most impactful cases.

## Recommendation

**Disposition B: Persuasive with Modification**

This is clearly a valid request — placeholder descriptions add no value and a similar ticket (FHIR-47427) was accepted as Persuasive in another IG. However, adding a modification that defines what a "good" example description should contain (profile being exemplified + scenario/use case + concise length) gives the spec editors actionable guidance and produces more consistent results than the open-ended original request. This approach acknowledges the problem fully while setting a clear quality bar for the fix.
