## Details

The reporter asks about a terminology inconsistency in Figure 1-2 of the Quality Reporting IG. Under "Consumers," the term "payers" is used, but the FHIR Coverage resource uses the data element name `Coverage.payor`. The reporter questions whether these refer to the same concept and whether the IG should align its spelling with the Coverage resource's element name ("payor") or use the more common English spelling ("payer").

This is a well-known, long-standing spelling inconsistency across FHIR specifications and IGs. Both "payer" and "payor" are valid English spellings, but the FHIR community has generally trended toward standardizing on "payer" in IG narrative text, while the base `Coverage.payor` element name remains as-is in the FHIR core spec (renaming was considered but resolved as "Not Persuasive with Modification" in FHIR-27881 to avoid breaking changes).

## Proposed Dispositions

### Disposition A: Persuasive (Accept — Standardize on "payer" in IG narrative)

Update Figure 1-2 and any other narrative text in the Quality Reporting IG to use "payer" consistently. This aligns with the resolution of multiple prior tickets (FHIR-28503, FHIR-20985, FHIR-28823, FHIR-35262) across Da Vinci and other IGs that have standardized on "payer" in prose and diagram text. Note that the FHIR core element `Coverage.payor` retains its current spelling per FHIR-27881, so this change applies only to IG narrative, not to resource element references.

### Disposition B: Persuasive with Modification (Accept with clarification note)

Update Figure 1-2 to use "payer" consistently in narrative text, and add a brief clarifying note (e.g., in a footnote or terminology section) stating: "The term 'payer' is used throughout this IG. Note that the FHIR Coverage resource uses the element name `Coverage.payor` (an accepted alternate spelling); both terms refer to the same concept." This provides explicit guidance for readers who encounter both spellings.

### Disposition C: Not Persuasive (Decline)

Decline the change on the basis that both "payer" and "payor" are accepted English spellings, and the current usage is not technically incorrect. The FHIR core specification itself uses "payor" in the Coverage resource element name, so IG usage of either form is defensible. However, this disposition would leave the IG inconsistent with the prevailing trend across other Da Vinci and FHIR IGs.

## Recommendation

**Disposition A: Persuasive** — Standardize on "payer" in narrative text throughout the Quality Reporting IG, consistent with the well-established community consensus across multiple resolved tickets (FHIR-28503, FHIR-28823, FHIR-35262, FHIR-20985). This is a straightforward editorial correction that aligns with the direction taken by other IGs. No element renaming or structural changes are needed.
