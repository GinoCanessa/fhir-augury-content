## Description

The QI-Core Coverage profile's `type` data element contains incorrect code system steward information. The description currently states the SOP (Source of Payment) code system is defined by the "US Public Health Data Consortium," which is inaccurate. The correct steward of the SOPT code system is the **Centers for Disease Control and Prevention, National Center for Health Statistics (CDC/NCHS)**. The Public Health Data Consortium (PHDSC) was an organization that CDC/NCHS established to work on public health data standards in response to HIPAA of 1996, but it was dissolved years ago.

The reporter references:
- The [Source of Payment Typology - NEW DRAFT](https://confluence.hl7.org/spaces/TA/pages/220698394/Source+of+Payment+Typology+-+NEW+DRAFT) page on the HL7 Terminology Authority Confluence
- UP-631 (SOPT - Additional Identifier and Other Changes) for related upstream work

## Resolution Description

The resolution proposes a multi-step correction approach:

1. **THO (HL7 Terminology) correction** — Already completed. THO v7.1.0 now has the correct owner listed at [External Code Systems](https://terminology.hl7.org/en/external_code_systems.html).
2. **US Core correction** — The correction should roll from THO into US Core.
3. **QI-Core correction** — Remove the incorrect custom description from the QI-Core Coverage.type element. Specifically, remove the statement: *"Categories of types of health care payor entities as defined by the US Public Health Data Consortium SOP code system"*. QI-Core should then inherit the description from US Core, so that when US Core is corrected, QI-Core will automatically reflect the update.

## Proposed Dispositions

### Disposition A: Persuasive (Accept as-is)

**Action:** Accept the resolution as proposed. Remove the incorrect description from QI-Core Coverage.type that names "US Public Health Data Consortium" as the SOP code system steward. Have QI-Core inherit the Coverage.type description from US Core rather than maintaining its own.

**Rationale:** The factual error is unambiguous — the PHDSC no longer exists and CDC/NCHS is the recognized steward of the SOPT code system. THO v7.1.0 has already been corrected. The upstream US Core ticket (FHIR-44122) has been applied. Removing QI-Core's custom description and inheriting from US Core avoids duplicated maintenance and ensures consistency across the IG hierarchy. FHIR-52490 was already closed as a duplicate of this ticket, confirming community agreement on the issue.

**Impact:** Low risk. Removes incorrect text from one data element description. No structural or binding changes to the profile.

### Disposition B: Persuasive with Modification

**Action:** Accept the factual correction, but instead of passively inheriting from US Core, directly update the QI-Core Coverage.type description to name CDC/NCHS as the correct steward. For example, change *"Categories of types of health care payor entities as defined by the US Public Health Data Consortium SOP code system"* to *"Categories of types of health care payor entities as defined by the CDC/NCHS Source of Payment Typology (SOPT) code system."*

**Rationale:** While the inherit-from-US-Core approach is clean long-term, directly correcting the text in QI-Core ensures the fix is applied immediately without depending on the timing of the US Core publication cycle. This may be preferable if there is any delay in US Core publishing the corrected description.

**Impact:** Low risk. Same scope as Disposition A but with explicit text replacement rather than description removal.

### Disposition C: Not Persuasive

**Action:** Reject the proposed change at this time. The steward information correction is already being addressed at the THO and US Core levels. QI-Core should wait for these upstream changes to propagate naturally through the publication pipeline rather than making a targeted edit.

**Rationale:** Since THO v7.1.0 is already corrected and FHIR-44122 (US Core) is applied, the incorrect steward information will be resolved when QI-Core picks up the next US Core baseline. Making a targeted edit now may conflict with or be superseded by the inherited corrections.

**Impact:** The incorrect steward text remains in QI-Core until the next publication cycle picks up US Core changes, potentially misleading implementers in the interim.

## Recommendation

**Disposition A: Persuasive (Accept as-is)**

The correction is factually straightforward and uncontested — the PHDSC is defunct, CDC/NCHS is the recognized SOPT steward, and THO v7.1.0 already reflects this. The proposed approach of removing QI-Core's custom description and inheriting from US Core is the cleanest solution, as it eliminates duplicated maintenance and ensures QI-Core stays aligned with US Core going forward. The upstream US Core fix (FHIR-44122) has already been applied, so the inheritance path is viable. A duplicate ticket (FHIR-52490) confirms independent community recognition of this issue. There is no technical risk or controversy — this is a simple metadata correction.
