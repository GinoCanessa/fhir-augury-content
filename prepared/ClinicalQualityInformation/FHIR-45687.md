## Description

The reporter identifies a dead link to the "CQLLibrary" profile in **Section 7.2 — Library Profile Usage** of the Quality Measures IG. The broken URL is:

> `http://hl7.org/fhir/uv/cql/StructureDefinition/cql-library`

This URL uses a FHIR REST-style canonical path (`/StructureDefinition/cql-library`) rather than the correct IG publication HTML page format. The request is to update the link to the functional URL.

The correct URL is:

> `https://hl7.org/fhir/uv/cql/StructureDefinition-cql-library.html`

## Analysis

This is a straightforward **broken link** issue. The URL `http://hl7.org/fhir/uv/cql/StructureDefinition/cql-library` uses the FHIR canonical URL format (which resolves via a FHIR server endpoint), rather than the IG publication HTML page URL (`https://hl7.org/fhir/uv/cql/StructureDefinition-cql-library.html`). The fix is to update the hyperlink in the IG source to point to the correct HTML page.

**Key consideration:** The near-duplicate ticket FHIR-49735, which reported the same broken link in a different section (Conformance Requirement 3.2) of the US Quality Measures IG, was resolved **Not Persuasive**. This may indicate that the link was corrected in a subsequent publication cycle, or that the work group considered the canonical URL acceptable. The work group should verify whether the link in Section 7.2 has already been fixed in the current build before applying a change.

## Proposed Dispositions

### Disposition A: Persuasive (Recommended)
**Action:** Update the CQLLibrary hyperlink in Section 7.2 (Library Profile Usage) from `http://hl7.org/fhir/uv/cql/StructureDefinition/cql-library` to `https://hl7.org/fhir/uv/cql/StructureDefinition-cql-library.html`.

**Rationale:** This is a valid technical correction. The current link is non-functional for readers navigating the IG in a browser. The fix is minimal, low-risk, and improves usability. Even though the canonical URL is technically correct as a FHIR identifier, within narrative IG text a clickable HTML link should resolve to a readable page. FHIR-45923 (which addressed related Section 7.2 content) was resolved Persuasive with Modification, suggesting the work group is willing to make corrections in this area.

### Disposition B: Not Persuasive
**Action:** No change.

**Rationale:** The canonical URL (`http://hl7.org/fhir/uv/cql/StructureDefinition/cql-library`) is the official FHIR canonical identifier for the CQLLibrary profile. Using canonical URLs in IG narrative is consistent with FHIR conventions. The near-duplicate FHIR-49735 was resolved Not Persuasive, suggesting the work group has already considered and declined this class of change. Readers can locate the profile via the canonical URL in a FHIR registry.

### Disposition C: Persuasive with Modification
**Action:** Update the link to the correct HTML page URL **and** audit all other CQLLibrary/profile links in the Quality Measures IG for the same issue.

**Rationale:** If the link is broken in Section 7.2, the same pattern likely exists in other sections (as evidenced by FHIR-49735 reporting the same issue in Conformance Requirement 3.2). A broader audit and fix would prevent future duplicate tickets and improve overall IG navigation quality.

## Recommendation

**Disposition A: Persuasive** — The broken link is a clear defect that harms reader experience. The fix is trivial (single URL update) and carries no risk of semantic change. While FHIR-49735 was resolved Not Persuasive, this ticket was filed earlier and targets Section 7.2 specifically; the work group should confirm whether the link has already been corrected in the current build before closing without action.
