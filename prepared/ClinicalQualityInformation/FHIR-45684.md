# FHIR-45684: CQFM Ratio Measure Short Description Revision

## Summary

**Ticket:** [FHIR-45684](https://jira.hl7.org/browse/FHIR-45684)
**Status:** Triaged | **Type:** Change Request | **Priority:** Medium
**Specification:** US FHIR Guidance - Quality Reporting (FHIR)
**Work Group:** Clinical Quality Information
**Reporter:** Aaron Nusstein | **Assignee:** Unassigned
**Created:** 2024-05-09 | **Updated:** 2024-06-30

The submitter requests that the short descriptions (slice labels) on `Measure.group.population` slices in the **CQFM Ratio Measure** profile (section 17.54.1.1) be revised. The current short descriptions are described as "not clear or descriptive of what slice means, despite being somewhat intuitive in their meanings." The proposal is to rework them to be more helpful to the reader.

This is a documentation/usability improvement — no structural or conformance changes are involved. The request targets only the human-readable short description text on existing population slices (e.g., initial population, denominator, numerator) within the Ratio Measure profile.

## Analysis

### Context
This ticket is one in a **coordinated set of five change requests** (FHIR-45681 through FHIR-45686, excluding the clone) all filed by Aaron Nusstein on 2024-05-09, each requesting short description improvements for a different CQFM Measure profile. The requests share identical language and rationale, suggesting they should be reviewed and dispositioned consistently.

### Scope of Change
- **Low risk:** Only human-readable short description text on existing slices is affected
- **No structural impact:** No changes to cardinality, data types, constraints, or conformance requirements
- **Usability improvement:** Clearer slice labels would improve readability of the profile for implementers
- **No proposed replacement text:** The ticket identifies the problem but does not provide specific revised wording

### Considerations
1. The submitter does not provide concrete alternative short descriptions, which means the workgroup would need to author the replacement text
2. This should be coordinated with the sibling tickets (FHIR-45683, FHIR-45685, FHIR-45681, FHIR-45686) to ensure consistent naming conventions across all CQFM measure profiles
3. FHIR-45321 (already Published) modified the Ratio Measure profile for stratification support — any population slice changes should be validated against the current state of the profile post-FHIR-45321

## Proposed Dispositions

### Disposition A: Persuasive
**Action:** Accept the request and revise the short descriptions on `Measure.group.population` slices in the CQFM Ratio Measure profile to be more descriptive and reader-friendly.

**Rationale:** The request is reasonable — clearer short descriptions improve implementer experience without any conformance or structural risk. The workgroup would author improved text for each population slice (e.g., "Initial Population for Ratio Numerator" instead of a generic label). This should be coordinated with the sibling tickets to apply a consistent naming pattern across all CQFM measure profiles.

**Trade-offs:** Requires workgroup effort to draft and review replacement text. Must coordinate with 4+ sibling tickets.

### Disposition B: Persuasive with Modification
**Action:** Accept the request but ask the submitter to propose specific revised short description text for each population slice before implementation.

**Rationale:** The ticket correctly identifies a usability gap, but lacks concrete replacement wording. Asking the submitter to propose specific text would reduce workgroup effort and ensure the revisions meet the submitter's expectations. The workgroup can then review and refine the proposed text.

**Trade-offs:** Adds a round-trip with the submitter, potentially delaying resolution. However, it produces better-targeted changes.

### Disposition C: Not Persuasive
**Action:** Reject the request.

**Rationale:** The current short descriptions, while not ideal, are described by the submitter as "somewhat intuitive in their meanings." Short descriptions are supplementary documentation; the formal element definitions and usage notes provide the primary guidance. The effort to revise short descriptions across multiple profiles may not justify the marginal usability improvement.

**Trade-offs:** Misses an opportunity to improve documentation quality. May discourage community feedback on usability issues.

## Recommendation

**Disposition A: Persuasive** — The request is low-risk, clearly beneficial to implementer readability, and part of a coordinated set of tickets that should be addressed together. The workgroup should accept this ticket along with its siblings (FHIR-45683, FHIR-45685, FHIR-45681, FHIR-45686) and establish a consistent naming convention for population slice short descriptions across all CQFM measure profiles. Since the change is purely editorial, the workgroup can author improved text directly without requiring a round-trip to the submitter.
