## Description

The submitter requests the addition of an example Implementation Guide (IG) that demonstrates the use of a **Questionnaire** to provide missing information as part of a Clinical Practice Guideline (CPG). The motivating scenario is a CPG that needs to calculate **BMI** and **eGFR** but is missing a required **Weight** data point. The Questionnaire would be used to solicit that missing data from the clinician during guideline execution.

This addresses a gap in the Clinical Guidelines IG: while the specification describes the concept of collecting information as part of CPG execution (via the Collect Information activity), there is no concrete, end-to-end example IG showing how a Questionnaire integrates into this workflow.

## Analysis

### Context
The Clinical Guidelines (CPG) IG defines a "Collect Information" activity type that uses an ActivityDefinition to generate a Task for gathering data via a Questionnaire. However, the specification currently lacks an end-to-end example IG that walks through this pattern concretely. The reporter (Bryn Rhodes, a primary author of the CPG IG) is requesting an illustrative example with a clinically meaningful scenario (BMI/eGFR calculation requiring a Weight observation).

### Scope
This is a **documentation/example** request — it does not propose changes to profiles, operations, or conformance resources. It asks for an example IG artifact that demonstrates:
1. A CPG PlanDefinition that calculates BMI and eGFR
2. Detection that a required data point (Weight) is missing
3. Use of a Questionnaire to collect the missing Weight
4. Integration of the QuestionnaireResponse back into the CPG workflow

### Relationship to FHIR-50165
FHIR-50165 (Triaged, 2025) goes further by proposing new `$questionnaire` operations on PlanDefinition for interactive CDS. If both tickets are pursued, the example IG from FHIR-44697 could serve as a foundational illustration for the operations proposed in FHIR-50165. They are complementary but can be addressed independently.

### Impact
- **Low risk**: Adding an example IG does not change normative or trial-use content.
- **High educational value**: Fills a significant documentation gap for implementers trying to use Questionnaires within CPG workflows.
- **Implementation effort**: Moderate — requires authoring a complete example IG with PlanDefinition, Library (CQL), ActivityDefinition, Questionnaire, and supporting resources.

## Proposed Dispositions

### Disposition A: Persuasive (Recommended)
**Action:** Accept the request and add an example IG to the Clinical Guidelines specification that demonstrates Questionnaire usage for collecting missing data in a CPG workflow, using the BMI/eGFR scenario described by the submitter.

**Rationale:** The request comes from a primary author of the CPG IG and addresses a concrete documentation gap. The Collect Information activity pattern is already defined in the spec but lacks an illustrative example, which hinders adoption. An example IG using the BMI/eGFR/Weight scenario is clinically intuitive and demonstrates a common real-world need. The effort is bounded (example content only, no spec changes), and the result would benefit implementers significantly.

**Suggested resolution text:** "Agreed. An example Implementation Guide will be added to the Clinical Guidelines specification demonstrating the use of a Questionnaire to collect missing information (Weight for BMI/eGFR calculation) as part of a CPG workflow using the Collect Information activity."

### Disposition B: Persuasive with Modification
**Action:** Accept but broaden the scope to include guidance text (not just an example IG) explaining the pattern of using Questionnaires to collect missing CPG data, and coordinate with FHIR-50165 and FHIR-36652 on the interactive CDS model.

**Rationale:** An example alone may not fully convey the design intent and constraints. Adding a narrative section describing when and how Questionnaires should be used in CPG workflows — alongside the example — would provide more complete implementer guidance. Coordinating with FHIR-50165 ensures the example aligns with the emerging `$questionnaire` operation approach.

**Suggested resolution text:** "Persuasive with modification. An example IG will be added along with narrative guidance describing the pattern of using Questionnaires to collect missing information during CPG execution. This work will be coordinated with FHIR-50165 (Interactive CDS operations) and FHIR-36652 (Questionnaire triggering) to ensure alignment."

### Disposition C: Deferred
**Action:** Defer until FHIR-50165 (Interactive CDS guidance and operations) is resolved, then build the example IG to align with the finalized interactive CDS model.

**Rationale:** FHIR-50165 proposes structural changes (new operations, documentation of questionnaire processing semantics) that would directly shape how such an example IG is authored. Building the example now risks rework if the interactive CDS model changes. Deferral ensures the example reflects the final design.

**Suggested resolution text:** "Deferred. This request will be revisited after FHIR-50165 (Interactive CDS guidance and operations) is resolved to ensure the example IG aligns with the finalized interactive CDS model."

---

## Recommendation

**Disposition A: Persuasive** — The request is well-scoped, low-risk, and addresses a clear documentation gap. The reporter is a primary CPG IG author, the scenario is clinically meaningful, and the work is independent of the broader interactive CDS operations proposed in FHIR-50165. An example IG can be authored now and updated later if needed.
