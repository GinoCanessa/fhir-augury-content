## Description

The submitter requests that the CPG-on-FHIR IG add a description of how to incorporate an expression language (e.g., CQL) into DMN (Decision Model and Notation). The goal is to provide implementers with guidance on bridging DMN-based decision models with CQL expression evaluation within clinical guideline artifacts.

## Analysis

This ticket addresses a documentation gap in the CPG-on-FHIR IG: how to integrate expression languages (specifically CQL) with DMN decision models. DMN is an OMG standard for modeling decisions, and the CPG IG already references decision-support patterns. However, the IG currently lacks explicit guidance on how CQL expressions can be embedded in or invoked from DMN decision tables and business knowledge models.

The ticket was originally deferred unanimously (20-0-0) with explicit intent to revisit post-STU2. STU2 has now been published and the ticket has been formally re-opened, indicating clear work group intent to address it in the next cycle.

Several related tickets (FHIR-24202, FHIR-40486, FHIR-49740) demonstrate a broader pattern of the CDS work group progressively adding expression language guidance across the CPG, CRMI, and Using CQL IGs. This ticket fits naturally into that trajectory.

## Proposed Dispositions

### Disposition A: Persuasive

**Action:** Accept the request. Add a section to the CPG-on-FHIR IG describing how to integrate expression languages (e.g., CQL) with DMN decision models, including how CQL expressions can be referenced from DMN decision tables and how evaluation context flows between the two.

**Rationale:** The ticket was explicitly deferred to post-STU2 with unanimous support (20-0-0) and has been formally re-opened now that STU2 is published. There is a clear documentation gap — implementers working with DMN-based clinical decision models need guidance on how to leverage CQL within that framework. The CPG IG is the appropriate home for this content, and related tickets (FHIR-24202, FHIR-49740) show precedent for adding expression language guidance to HL7 IGs.

### Disposition B: Not Persuasive

**Action:** Decline the request. The integration of CQL with DMN is an implementation concern that belongs in the DMN specification (managed by OMG) or in implementation-specific documentation, not in the CPG-on-FHIR IG.

**Rationale:** DMN is an external standard with its own extension mechanisms for expression languages. Adding guidance in the CPG IG could duplicate or conflict with OMG's DMN specification. Implementers should follow DMN's native FEEL-to-CQL bridging patterns rather than HL7-specific guidance.

### Disposition C: Persuasive with Modification

**Action:** Accept the request but scope the guidance more narrowly. Rather than a full description of CQL-in-DMN integration, add a brief informative section in the CPG-on-FHIR IG that (1) acknowledges DMN as a supported decision modeling notation, (2) points to relevant CQL and DMN specifications for expression language integration, and (3) provides a minimal example or reference to community resources.

**Rationale:** A comprehensive treatment of CQL+DMN integration may be too ambitious for a single IG update and risks going out of date as both CQL and DMN evolve. A scoped, reference-oriented approach addresses the implementer need while keeping maintenance burden low. This aligns with the approach taken in FHIR-24022, which recommended referencing existing guidance rather than duplicating it.

---

## Recommendation

**Disposition C: Persuasive with Modification**

The work group explicitly intended to revisit this post-STU2, and the documentation gap is real. However, a full treatment of CQL-in-DMN integration is a substantial undertaking that spans two independent standards. A scoped approach — acknowledging DMN, providing references, and including a minimal integration example — delivers implementer value while keeping the scope manageable. This can be expanded in future cycles as community experience with CQL+DMN grows.
