## Description

Lloyd McKenzie reports that two extensions in the PACIO Cognitive Status IG have a context of `Element`, meaning they are allowed to appear on any data element on any resource. He requests constraining the extension contexts to only the specific resource(s) and element(s) where these extensions are appropriate.

Based on related tickets (FHIR-29983, FHIR-43145, FHIR-29589), the two extensions in question are **AssistanceRequired** and **DevicePatientUsed**, which are defined in the PACIO Cognitive Status IG and also duplicated in the PACIO Functional Status IG.

## Proposed Dispositions

### Disposition A: Persuasive

**Action:** Accept the request and constrain the extension contexts for AssistanceRequired and DevicePatientUsed to the specific resource(s) and element(s) where they are used (e.g., `Observation` or the specific PACIO Cognitive Status profiles).

**Rationale:** The request is well-founded. Extensions with an `Element` context are declared as appropriate for use anywhere in any resource, which is almost never the intent. Constraining contexts to the actual target resources/elements improves IG quality, aids validation, and is consistent with FHIR best practices. Parallel tickets (FHIR-43143, FHIR-43141) for other IGs have already been resolved as Persuasive and applied/published, establishing clear precedent.

### Disposition B: Persuasive with Modification

**Action:** Accept the request and constrain the extension contexts, but also coordinate with the PACIO Functional Status IG (FHIR-43145) to consolidate the duplicated AssistanceRequired and DevicePatientUsed extension definitions into a single shared location (either one IG declares a dependency on the other, or the extensions are moved to a common/core extensions IG).

**Rationale:** This addresses the context issue while also tackling the duplication problem flagged in FHIR-43145. Maintaining identical extensions in two IGs creates divergence risk and increases maintenance burden. Consolidation is a stronger long-term fix. However, this is a broader scope change than what was requested.

### Disposition C: Not Persuasive

**Action:** Leave the extension contexts as `Element`.

**Rationale:** The extensions technically function correctly regardless of context breadth; context is an informational/validation constraint rather than a functional one. Changing context could break existing implementations that happen to use the extension outside the originally intended scope. However, this contradicts established precedent (FHIR-43143, FHIR-43141) and FHIR community expectations for IG quality.

## Recommendation

**Disposition A: Persuasive** is recommended. The request is straightforward, well-justified, and strongly supported by precedent — identical tickets for other IGs (FHIR-43143, FHIR-43141) have already been accepted and applied. The extension contexts for AssistanceRequired and DevicePatientUsed should be constrained to the resources and elements where they are actually used in the PACIO Cognitive Status IG. The duplication concern raised in FHIR-43145 is valid but is a separate scope item better addressed by that ticket.
