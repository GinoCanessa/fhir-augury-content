## Details

This change request asks the Clinical Guidelines IG to consider adding characterizations in two areas:

1. **Enablement/Usability** — Characterizing how a clinical guideline recommendation is enabled or made usable in a clinical workflow context.
2. **Impact/Outcome** — Characterizing the impact or outcome of guideline implementation, with a reference to "High Touch" as a potential category.

## Proposed Dispositions

### Disposition A: Accept — Define enablement and impact characterization elements

**Action:** Add structured characterization elements to the Clinical Guidelines IG that capture:
- **Enablement/Usability dimensions** (e.g., workflow integration readiness, implementation complexity, required infrastructure) as metadata on guideline recommendations.
- **Impact/Outcome dimensions** (e.g., expected clinical outcomes, resource utilization impact, "High Touch" categorization) to describe the anticipated or measured effects of guideline adoption.

**Rationale:** The ticket was previously deferred with unanimous support (20-0-0) specifically to be pursued after STU2 publication, which has now occurred. The work group has already signaled consensus that this is valuable work. Adding these characterizations would help implementers assess the feasibility and expected value of adopting specific clinical guideline recommendations. This aligns with the IG's goal of making clinical guidelines computable and actionable.

### Disposition B: Accept with narrower scope — Start with impact/outcome characterization only

**Action:** Begin with adding impact/outcome characterization elements (including "High Touch" classification) to the Clinical Guidelines IG, deferring enablement/usability characterization to a subsequent cycle.

**Rationale:** Impact/outcome characterization is more clearly scoped (the "High Touch" reference suggests some initial modeling has been considered). Starting with one dimension allows the work group to validate the approach before tackling the broader enablement/usability characterization, which may require more research into workflow integration patterns. This phased approach reduces risk while still making progress.

### Disposition C: Decline — Defer pending concrete requirements

**Action:** Return to "Considered for Future Use" status until specific characterization dimensions, value sets, and use cases are defined in a more detailed proposal.

**Rationale:** The ticket description is brief and does not specify what specific enablement or impact characterizations should be modeled, how they would be represented (extensions, profiles, new resources), or what clinical use cases they serve. Without a more detailed proposal, the work group risks investing effort in design work that may not meet implementer needs. A concrete proposal with examples would enable more productive discussion.

## Recommendation

**Disposition A: Accept — Define enablement and impact characterization elements.**

The work group has already demonstrated strong consensus (20-0-0) that this is worthwhile work, deferring it only until after STU2 publication—which has now occurred. The re-opening by Lynn Laakso signals readiness to proceed. While the description is brief, the reporter (Bryn Rhodes) is a core contributor to the Clinical Guidelines IG and can elaborate during work group discussion. Accepting the ticket allows the work group to begin scoping both characterization dimensions, and scope can be narrowed during implementation if needed.
