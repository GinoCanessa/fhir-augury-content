## Description

The [CQL IG example library](https://build.fhir.org/ig/HL7/cql-ig/Library-Example.html) and [module definition example library](https://build.fhir.org/ig/HL7/cql-ig/Library-module-example.html#data-requirements) contain references to IPS and QICore artifacts. Because these are only example dependencies (not conformance dependencies), the URLs were changed from `hl7.org` to `example.org` to avoid forcing downstream IGs to take unnecessary dependencies.

The ticket requests two things:

1. **After CQL IG 1.0.0 is published**, change the example URLs back to the real IPS/QICore profile and value set URLs, and add a `dependsOn.reason` to indicate they are example-only dependencies.
2. **Submit a tracker** (done — see FHIR-45519) to propose an extension that allows a dependency to be marked as an "example" dependency.

Bryn Rhodes raised the core issue: the CQL IG has examples referencing IPS/QICore profiles but doesn't want to declare full dependencies. ## Analysis

This ticket is a **post-publication maintenance task** for the CQL IG. The actual workaround (using `example.org` URLs) is already in place and the IG has been publishing successfully. The change requested is to restore the real URLs once the companion extension (FHIR-45519) or the existing `ig-internal-dependency` extension is available and the CQL IG 1.0.0 is published.

**Key considerations:**
- The companion ticket FHIR-45519 has been resolved as "Persuasive with Modification," with Grahame Grieve pointing to the existing `ig-internal-dependency` extension (`http://hl7.org/fhir/tools/StructureDefinition/ig-internal-dependency`) as sufficient for this purpose.
- Multiple IGs (CQL, DEQM, Belgian IGs) share this pattern of example-only dependencies, indicating broad community need.
- The `dependsOn.reason` element already exists in `ImplementationGuide.dependsOn` and can document why a dependency exists.
- The ticket is self-contained within the CQL IG and does not require cross-IG coordination beyond using an agreed-upon mechanism.

## Proposed Dispositions

### Disposition A: Persuasive
**Action:** Accept as-is. Restore the real IPS and QICore URLs in the CQL IG example resources (`Library-Example` and `Library-module-example`) and declare the dependencies in `ImplementationGuide.dependsOn` with the `ig-internal-dependency` extension and a `dependsOn.reason` indicating they are example-only.

**Rationale:** The request is straightforward, the tooling mechanism now exists (`ig-internal-dependency`), and FHIR-45519 has been resolved. This improves example quality by using real URLs instead of `example.org` placeholders.

### Disposition B: Persuasive with Modification
**Action:** Accept, but modify the approach to use the `ig-internal-dependency` extension (as identified in FHIR-45519's resolution) rather than waiting for a new "example dependency" extension. Also add `dependsOn.reason` text clarifying the example-only nature of these dependencies.

**Rationale:** Since FHIR-45519 concluded that `ig-internal-dependency` already serves this purpose, the ticket should be updated to reference this existing mechanism rather than leaving it open-ended. This provides a clearer, immediately actionable path.

### Disposition C: Not Persuasive
**Action:** Reject. The current `example.org` URLs are a functional workaround and the examples still convey the intended concepts. Restoring real URLs adds complexity (dependency management) for minimal benefit.

**Rationale:** The workaround is already in place and publication is clean. The marginal benefit of "real" URLs in examples does not justify the added dependency management overhead.

## Recommendation

**Disposition B: Persuasive with Modification** — The underlying need is valid and well-documented across multiple IGs. The `ig-internal-dependency` extension already exists and has been endorsed by Grahame Grieve on FHIR-45519. The ticket should be accepted with the modification that it explicitly references `ig-internal-dependency` as the mechanism, uses `dependsOn.reason` for documentation, and proceeds to restore the real IPS/QICore URLs in the CQL IG examples. This avoids any ambiguity about waiting for a new extension that is no longer needed.
