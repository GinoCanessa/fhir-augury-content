## Summary

The ticket reports that the CQIF specification page includes narrative text describing the use of the `useContext` element for semantic and topical indexing in knowledge repositories, accompanied by an example fragment said to illustrate usage context for a suicide risk order set. However, the example fragment shown on the page does not actually include a `useContext` element and does not conform to the `UsageContext` data type structure. The reporter initially noted the discrepancy could be either a documentation clarity issue or an outdated example. A follow-up comment from the reporter confirmed that the actual PlanDefinition example in the specification does contain the expected `useContext` structure, meaning the inline fragment on the CQIF page is the component that needs updating to match.

## Details

**Description:**

This page has the following text introducing an example fragment:

> In particular, the useContext and topic elements are intended to provide both semantic and topical indexing functionality for use in knowledge repositories. For example, the following fragment illustrates the use of the useContext element to indicate semantic usage context for an example suicide risk order set:

The example does not include a useContext element nor does it seem to match the UsageContext structure. Either I am misunderstanding, so the text should be made clearer, or the example has become out of date with the resource design.

## Proposed Dispositions

### Disposition A: Accept As Requested
#### Proposal
Update the inline example fragment on the CQIF page to include a valid `useContext` element that conforms to the `UsageContext` data type, matching the structure found in the actual PlanDefinition suicide risk order set example. The updated fragment should demonstrate at least one `useContext` entry with appropriate `code` (from the usage-context-type ValueSet) and `value` (e.g., a `CodeableConcept` referencing a clinical focus condition such as suicide risk assessment).
#### Justification
The reporter confirmed that the full PlanDefinition example already contains the correct `useContext` structure, so the fix is straightforward: replace or augment the outdated inline fragment to align with the existing example. This is a documentation correctness issue—the narrative promises to show `useContext` usage but the fragment does not deliver. Correcting it improves clarity and prevents confusion for implementers referencing the CQIF page.

---

### Disposition B: Alternative Approach
#### Proposal
Rather than updating the fragment in place, remove the outdated inline fragment entirely and replace it with a direct link or reference to the full PlanDefinition suicide risk order set example (e.g., `plandefinition-example-suiciderisk-orderset`), which already demonstrates proper `useContext` usage. Update the surrounding narrative text to point readers to the canonical example rather than maintaining a separate inline snippet.
#### Justification
Maintaining duplicate example fragments risks them falling out of sync again as the specification evolves. Linking to the canonical example ensures ongoing consistency and reduces maintenance burden. This approach also gives implementers access to a richer, complete example rather than an isolated fragment.

---

### Disposition C: Decline
#### Proposal
Decline on the basis that the CQIF specification is deprecated and the content is no longer actively maintained.
#### Justification
The specification listed for this ticket is "Clinical Quality Information Framework (CQIF) [deprecated]." Since CQIF has been superseded by the Clinical Reasoning module and related resources (e.g., PlanDefinition, ActivityDefinition, Library) within the core FHIR specification, investing effort in correcting deprecated documentation may not be the best use of limited editorial resources. The correct `useContext` usage is already demonstrated in the current specification's PlanDefinition examples and Clinical Reasoning module documentation, where implementers should be directed instead.

---

### Recommendation
**Recommended disposition:** C

Given that the CQIF specification is explicitly marked as deprecated and the underlying content has been absorbed into the FHIR Clinical Reasoning module, the most pragmatic course of action is to decline the change. The correct `useContext` example already exists in the current PlanDefinition resource documentation. Updating deprecated content risks creating a false impression that the CQIF page is still authoritative, and any editorial effort would be better directed at ensuring the Clinical Reasoning module's documentation and examples are clear and complete. If the deprecated CQIF page is still publicly accessible, a brief note redirecting readers to the current Clinical Reasoning module would be a minimal and sufficient improvement, but a full example fragment update is not warranted.
