## Summary

Ricardo Quintano submitted review comments on the BPMN-to-FHIR PlanDefinition mapping page in the CPG (Clinical Practice Guidelines) Implementation Guide, specifically the [documentation-methodology page](http://build.fhir.org/ig/HL7/cqf-recommendations/documentation-methodology.html). The comments originated from a FHIR Workflow WG meeting on 2020-03-23 ([meeting minutes](https://confluence.hl7.org/display/FHIRI/FHIR+Workflow+Minutes+CC+20200323)). The ticket description references an attached document containing the review comments, but the actual comments are not visible in the Jira description itself. The ticket was resolved as **"Considered - No action required"**.

## Details

- The ticket concerns the methodology page that documents the mapping between OMG BPMN (Business Process Model and Notation) constructs and FHIR PlanDefinition resources in the CPG IG.
- This mapping enables clinical workflow processes modeled in BPMN to be represented using PlanDefinition resources, supporting automated execution of clinical guidelines.
- The review comments were attached as a file (not inline), so the specific feedback points are not available in the ticket description.
- The resolution ("Considered - No action required") suggests the workgroup reviewed the comments and determined no specification changes were warranted.
- No comments from workgroup members are recorded on the Jira ticket itself.
