## Description

The request asks for a new activity definition in the Clinical Guidelines (CPG) IG to support documenting a clinical assessment using a **ClinicalImpression** resource. The motivating use case is immunization guidance where the recommendation is to "immunize if there is a compelling reason" — the proposed activity would capture the documentation of that compelling reason as a ClinicalImpression.

## Analysis

### Strengths
- The request addresses a real gap: the CPG IG currently lacks a way to represent the activity of documenting a clinical assessment/impression, which is a common step in clinical decision-making workflows.
- The immunization use case is concrete and well-defined — documenting a "compelling reason" to immunize is a tangible clinical action.
- Reporter Bryn Rhodes is a CPG IG author with deep domain knowledge; the related tickets (FHIR-40298, FHIR-23949) show this fits a coherent vision for CPG activity modeling.
- Previous vote was unanimously supportive (20-0-0) but deferred for timing, not substance.

### Risks & Considerations
- **ClinicalImpression maturity:** The resource remains at FMM 1 and is being renamed to ClinicalAssessment in R6. Building a CPG activity around a low-maturity, renaming resource introduces fragility.
- **Adoption uncertainty:** Zulip discussions (2023 and 2025) reveal ongoing debate about whether ClinicalImpression is necessary at all, with some arguing Observation could serve the same purpose.
- **Scope clarity:** The original description is brief. It needs to define what elements of ClinicalImpression the activity would populate (e.g., code, finding, investigation) and how it relates to other CPG activities and the activity/workflow separation established by FHIR-23949.
- **Immunization alignment:** FHIR-40298 changed CPG immunization modeling to use medication-related resources. The immunization use case in this ticket must be reconciled with that change.

## Proposed Dispositions

### Disposition A: Persuasive

**Rationale:** Accept the request and add a ClinicalImpression-based activity definition to the CPG IG. The need to document clinical assessments as a discrete activity in guideline-driven workflows is well-established, and the immunization "compelling reason" use case is a clear illustration. The previous 20-0-0 vote shows strong consensus on the concept. The CPG IG should model the full spectrum of clinical activities, including documentation of clinical reasoning.

**Action:** Define a new CPG activity profile for documenting clinical assessments using ClinicalImpression (or ClinicalAssessment in R6). Include guidance on which ClinicalImpression elements to populate (code, finding, investigation) and how this activity integrates with the activity/workflow separation. Provide the immunization example and at least one other clinical scenario.

### Disposition B: Not Persuasive

**Rationale:** Decline the request. ClinicalImpression is at FMM 1, has limited adoption, is being renamed in R6, and its necessity is actively debated in the community. Adding a CPG activity around a resource whose long-term viability is uncertain would create maintenance burden and implementer confusion. The documentation use case could potentially be served by existing activity types (e.g., a documentation Task or an Observation-based activity). The request should wait until the ClinicalImpression/ClinicalAssessment resource has achieved higher maturity and clearer community consensus.

**Action:** Close as Not Persuasive with a note that the concept can be reconsidered once ClinicalAssessment reaches FMM 2+ and R6 stabilizes the resource.

### Disposition C: Persuasive with Modification

**Rationale:** Accept the concept of a clinical documentation activity but broaden the scope beyond ClinicalImpression alone. Rather than tightly coupling to a single low-maturity resource, define the activity more generically as "document clinical assessment" and allow it to be fulfilled by ClinicalImpression, Observation, or other appropriate resources depending on implementation context. This hedges against ClinicalImpression maturity concerns while still addressing the identified gap.

**Action:** Add a new CPG activity for documenting clinical assessments. Define the activity at a conceptual level with ClinicalImpression as the primary but not exclusive target resource. Include guidance on when ClinicalImpression vs. Observation vs. other resources is appropriate. Provide the immunization use case as the primary example.

## Recommendation

**Disposition C: Persuasive with Modification**

The underlying need — an activity for documenting clinical assessments in guideline-driven workflows — is well-supported by the unanimous prior vote and fits the CPG IG's evolution. However, tightly coupling the activity solely to ClinicalImpression is premature given its FMM 1 status, the R6 rename, and ongoing community debate about the resource's necessity. A broader framing that allows flexibility in the target resource protects against resource-level instability while delivering the requested capability. This also aligns with the activity/workflow separation principle established by the already-accepted FHIR-23949.
