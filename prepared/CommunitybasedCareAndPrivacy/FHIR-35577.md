## Summary

This ticket requests that the Value Set canonical URL defined in the US PACIO Re-Assessment Timepoints IG be re-anchored from `hl7.org/fhir` to `terminology.hl7.org` (THO). The submitter (Joan Harper, via Ron G. Parker) argues that only value sets with a required binding to a `code` data type should have canonical URLs rooted at `hl7.org/fhir`; all others should be registered in THO via a UTG ticket. This is the first of a series of related comments (Comments 1–3) filed against this specification.

The ticket was originally resolved as "Considered for Future Use" in February 2022 (vote: Dave Hill / Dave Pyke, 9-0-0) with the rationale that further experience would inform whether the value set should be absorbed elsewhere, remain IG-specific, or be hosted on THO. That resolution was reverted by Lynn Laakso in September 2022 to re-open the ticket post-STU1 publication, and it currently sits in Triaged/Unresolved status.

---

## Details

**Request:** Move the Value Set canonical URL from `hl7.org/fhir` namespace to `terminology.hl7.org` (THO). Create a UTG ticket to register the Value Set in THO.

**Underlying Principle:** The HL7 terminology governance rule is that value sets bound to the `code` data type with a required binding should remain anchored at `hl7.org/fhir`, while all other value sets should be anchored in THO (`terminology.hl7.org`). This is part of broader HL7 terminology governance to ensure consistent management and public review of terminology artifacts.

**History:**
- **2022-01-10:** Filed by Joan Harper (imported by Ron G. Parker) as Comment 1 against the PACIO Re-Assessment Timepoints IG.
- **2022-02-22:** Resolved as "Considered for Future Use" by vote (9-0-0, Dave Hill / Dave Pyke) — the work group chose to defer, allowing the IG and its terminology to mature before deciding placement.
- **2022-09-28:** Resolution reverted by Lynn Laakso and ticket re-opened post-STU1 publication.

**Sibling Tickets:** FHIR-35578 (Comment 2) and FHIR-35579 (Comment 3) are identical requests against the same specification, each targeting a different value set in the IG.

## Proposed Dispositions

### Disposition A: Persuasive (Accept)

**Action:** Accept the request. Migrate the value set canonical URL from `hl7.org/fhir` to `terminology.hl7.org` and create a UTG ticket to register the value set in THO.

**Rationale:** This aligns with HL7's stated terminology governance policy that non-code-bound value sets should be anchored in THO. The parallel ticket FHIR-35551 (CARIN DIC) was already resolved as Persuasive. The IG has now been published as STU1, providing the maturity that the work group previously wanted before acting. Coordinating with TSMG ensures proper governance. Apply consistently to sibling tickets FHIR-35578 and FHIR-35579.

**Considerations:** Changing the canonical URL is a breaking change for implementers who reference the current URL. This should be coordinated with a new version of the IG and communicated clearly. A UTG proposal must be prepared and submitted.

### Disposition B: Persuasive with Modification (Conditional Accept)

**Action:** Accept in principle, but defer the URL migration to the next major IG version (STU2). In the interim, register the value set in THO with the new canonical URL and provide a mapping/redirect from the old URL.

**Rationale:** The original deferral (9-0-0 vote) acknowledged the need but wanted maturity. A phased approach respects implementers who have already adopted the STU1 URL while moving toward governance compliance. This avoids a disruptive mid-cycle change and aligns the migration with the natural IG update cycle.

**Considerations:** Requires coordination with the PACIO project team to plan the STU2 timeline and with TSMG/UTG for the THO registration process.

### Disposition C: Not Persuasive (Decline)

**Action:** Decline the request. Retain the current canonical URL anchoring at `hl7.org/fhir`.

**Rationale:** The value sets in this IG are specific to the PACIO Re-Assessment Timepoints use case and may not warrant registration in the shared THO namespace. The previous 9-0-0 vote for "Considered for Future Use" reflected legitimate uncertainty about where this value set belongs. The parallel ticket FHIR-38528 (CodeX Radiation Therapy) was resolved as Not Persuasive, showing precedent for declining similar requests when the value set is IG-specific. Moving to THO adds governance overhead without clear benefit for a niche value set.

**Considerations:** This disposition would leave the ticket inconsistent with FHIR-35551 (Persuasive) and the general HL7 terminology governance policy. The work group should document why an exception is warranted.

---

## Recommendation

**Disposition A: Persuasive** is recommended.

The HL7 terminology governance rule is clear: value sets not bound to the `code` data type with a required binding should be anchored in THO. Lloyd McKenzie's authoritative clarification on Zulip confirms this policy. The IG has reached STU1 maturity, which was the condition the work group set when it deferred in 2022. The parallel ticket FHIR-35551 was resolved as Persuasive, providing precedent for acceptance. The work group should accept this ticket, coordinate with TSMG to create a UTG proposal, and apply the same disposition to sibling tickets FHIR-35578 and FHIR-35579 for consistency.
