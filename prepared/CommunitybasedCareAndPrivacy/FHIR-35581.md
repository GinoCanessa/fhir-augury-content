## Analysis

### Context
This ticket is part of a large batch of identical change requests filed by Joan Harper (via Ron G. Parker) across many IGs during the January 2022 ballot cycle. The core request is a terminology governance concern: code systems defined within FHIR IGs should have their canonical URLs rooted in `terminology.hl7.org` (the HL7 Terminology server, THO) rather than under the IG-specific `hl7.org/fhir/...` namespace. This aligns with HL7's terminology management strategy to centralize code system definitions in UTG/THO.

### Prior Disposition
This ticket was previously resolved as "Considered for Future Use" on 2022-02-22 with a 9-0-0 vote (Dave Hill / Dave Pyke). The disposition stated that the proposal should be revisited as the IG and its terminology matures. However, lynn_laakso reverted that resolution on 2022-09-28, re-opening the ticket post-STU1 publication for reconsideration.

### Key Considerations
- The ticket has been open and triaged since September 2022 with no assignee.
- FHIR-35580 (the companion ".1" ticket for the same IG) is in the same state, suggesting both code systems in the PACIO Re-Assessment Timepoints IG need the same treatment.
- Similar tickets in other IGs (e.g., FHIR-35485 for COVID-19 IG) were resolved as "Considered for Future Use" or deferred, suggesting some work groups prefer to defer until the terminology stabilizes.
- The priority is marked "Highest" and labeled "Block-Vote-1."

## Proposed Dispositions

### Disposition A: Applied — Migrate Code System URL to THO
**Action:** Accept the change request. Create a UTG ticket to register the PACIO Re-Assessment Timepoints code system(s) in THO and update the canonical URL from `hl7.org/fhir/...` to `terminology.hl7.org/...`.

**Rationale:** This aligns with HL7's terminology governance strategy. The IG has now reached STU1 publication, and the code system has had time to mature. Migrating to THO ensures consistent terminology management, discoverability, and lifecycle governance. The previous deferral explicitly anticipated revisiting this after further IG maturation. Coordinating with FHIR-35580 ensures both code systems in the IG are addressed together.

**Trade-offs:** Requires a UTG submission and coordination with the Vocabulary work group. The canonical URL change is a breaking change for implementers who reference the current URL, though redirects can mitigate this. Requires careful versioning in the next IG release.

### Disposition B: Deferred — Considered for Future Use (Restore Previous Resolution)
**Action:** Re-apply the previous resolution of "Considered for Future Use," deferring until the IG reaches a more mature publication stage (e.g., STU2 or normative ballot).

**Rationale:** While the principle of THO anchoring is sound, the IG may still be evolving. Changing canonical URLs is a breaking change, and deferring allows the community to determine whether these code systems should be absorbed into existing terminologies, remain IG-specific, or move to THO. The original 9-0-0 vote supporting deferral reflected workgroup consensus that the timing was premature.

**Trade-offs:** Continues the current inconsistency with HL7 terminology governance practices. Risks further entrenchment of the hl7.org/fhir URL as more implementers adopt it.

### Disposition C: Not Persuasive — Keep Code System URL in IG Namespace
**Action:** Reject the change request. The code system canonical URL should remain under the IG's own namespace.

**Rationale:** Not all IG-specific code systems belong in THO. If this code system is tightly coupled to the PACIO Re-Assessment Timepoints IG and unlikely to be reused across other IGs, it may be more appropriate to keep it under the IG's namespace. THO is best suited for broadly applicable terminologies.

**Trade-offs:** Does not align with the general HL7 direction toward centralizing terminology in THO. May need to be revisited if HL7 establishes a firm policy requiring all code systems to be registered in THO.

---

## Recommendation

**Disposition A: Applied — Migrate Code System URL to THO**

The IG has now been published as STU1, and the previous deferral explicitly anticipated revisiting this request after the terminology matured. HL7's direction is to centralize code system management in THO/UTG, and continued delay increases the cost of the eventual breaking change as more implementers adopt the current URL. Coordinating this with the companion ticket FHIR-35580 and submitting a UTG registration would bring the PACIO Re-Assessment Timepoints IG into alignment with HL7 terminology governance practices.
