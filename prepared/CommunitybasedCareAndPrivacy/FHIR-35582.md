## Description

The submitter requests that the Code System canonical URL in the PACIO Re-Assessment Timepoints IG be anchored in `terminology.hl7.org` (THO) rather than `hl7.org/fhir`. A UTG ticket should be created to register the Code System in THO.

This is part of a systematic series of comments (originally imported by Ron G. Parker) filed by Joan Harper across multiple FHIR IGs during ballot review, all requesting that IG-defined code systems be migrated to the HL7 Terminology (THO) infrastructure.

## Analysis

**Core Issue:** The PACIO Re-Assessment Timepoints IG defines one or more code systems with canonical URLs under `hl7.org/fhir`. HL7 terminology governance best practice is that HL7-published code systems should be registered in THO (`terminology.hl7.org`) with canonical URLs anchored there, rather than under the FHIR base URL namespace. This ensures proper terminology governance, discoverability, and lifecycle management.

**Context:** This ticket was originally deferred ("Considered for Future Use") during initial STU1 ballot reconciliation, with the rationale that the IG's terminology was still maturing. After STU1 publication, the ticket was re-opened for reconsideration, suggesting the work group should now evaluate whether the terminology has stabilized enough to warrant migration to THO.

**Complexity Factors:**
- Migration of a code system canonical URL is a **breaking change** for implementers who reference the existing URL.
- The code system may need to be evaluated for whether it is IG-specific or broadly applicable before deciding on THO hosting.
- A UTG proposal process would be required to register the code system in THO.
- Across the HL7 ecosystem, similar tickets have been resolved inconsistently — some as Persuasive, some as Not Persuasive with Modification, and some as Considered for Future Use.

## Proposed Dispositions

### Disposition A: Persuasive

**Resolution:** Accept the request. Migrate the code system canonical URL from `hl7.org/fhir` to `terminology.hl7.org` (THO) and file a UTG proposal to register the code system.

**Rationale:** HL7 terminology governance best practices call for HL7-published code systems to be hosted in THO. The IG has been published as STU1 and the terminology has matured since the original deferral. Aligning with THO ensures proper lifecycle management, discoverability, and governance. The breaking change to the canonical URL is justified as the IG is still in STU (not normative), and implementers should expect such changes.

**Implications:** Requires filing a UTG ticket, coordinating with the Vocabulary work group, and updating all references to the code system's canonical URL in the IG. This is a breaking change for existing implementers referencing the current URL.

### Disposition B: Not Persuasive with Modification

**Resolution:** Acknowledge the concern but defer the migration. Add a note to the IG indicating that the code system is planned for migration to THO in a future release, and establish a redirect or aliasing strategy when THO registration occurs.

**Rationale:** While THO hosting is the long-term goal, a canonical URL change mid-STU lifecycle creates unnecessary disruption for early adopters. The code system can remain IG-hosted for now with a documented plan for migration. This aligns with how FHIR-35558 (CARIN DIC) was resolved. The work group can revisit this when preparing the next major ballot.

**Implications:** Minimal implementation impact in the short term. Requires adding documentation to the IG about the planned migration. The code system remains outside THO governance until a future cycle.

### Disposition C: Not Persuasive

**Resolution:** Decline the request. The code system canonical URL should remain under `hl7.org/fhir` as currently published.

**Rationale:** The code system is specific to the Re-Assessment Timepoints IG and may not warrant THO registration if it serves a narrow, IG-specific purpose. The prior 9-0-0 vote acknowledging uncertainty about whether the code system belongs in THO, is IG-specific, or could be absorbed elsewhere still holds. Until there is clearer guidance on the code system's long-term home, maintaining the status quo avoids unnecessary churn.

**Implications:** No implementation changes required. The code system remains outside THO governance, which may create inconsistency with HL7 terminology best practices over time.

---

## Recommendation

**Disposition B: Not Persuasive with Modification** is recommended.

This balances the legitimate concern about THO governance with the practical reality that a canonical URL change is breaking and the code system's long-term disposition is still unclear. The original deferral vote (9-0-0) explicitly noted uncertainty about whether this code system belongs in THO, is IG-specific, or could be absorbed elsewhere. Documenting a planned migration path in the IG provides a clear signal to implementers while avoiding premature disruption. The work group can re-evaluate for the next major ballot cycle once the terminology has further matured and a UTG proposal can be properly assessed.
