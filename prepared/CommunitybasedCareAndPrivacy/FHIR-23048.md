## Summary

This change request addresses the mapping of Plan Monitor Name and Phone Number within the eLTSS Implementation Guide. The ticket proposes replacing the current use of Encounter and EpisodeOfCare references for Plan Monitor information with a CareTeam-based approach. The ticket was originally resolved as "Considered for Future Use" in July 2020 by a 12-0-1 vote but was subsequently re-opened post-STU2 publication for further discussion and implementer input. Key suggestions include removing the EpisodeOfCare eLTSS Profile, adding a CareTeam eLTSS Profile, updating mapping spreadsheets, adding a SNOMED code for Plan Monitor, and collaborating with the Security Work Group.

## Proposed Dispositions

### Disposition A: Accept As Requested
#### Proposal
Remove the EpisodeOfCare eLTSS Profile and replace it with a new CareTeam eLTSS Profile that maps Plan Monitor name and phone number. Update the eLTSS mapping spreadsheet (XML and HTML) to reflect the new CareTeam-based approach. Request a SNOMED code for the Plan Monitor role (distinct from Case Manager). Coordinate with the Security Work Group on any privacy or access control implications of the CareTeam-based approach.
#### Justification
The EpisodeOfCare resource is not a natural fit for representing Plan Monitor contact information. CareTeam aligns more closely with FHIR's intended resource semantics for representing individuals involved in a patient's care, and provides a straightforward path for mapping provider names, phone numbers, and roles. The original 2020 vote acknowledged the need for this change but deferred it pending implementer input. Re-opening the ticket post-STU2 signals that the work group is ready to address this gap.

---

### Disposition B: Alternative Approach
#### Proposal
Retain the EpisodeOfCare eLTSS Profile but add a new CareTeam eLTSS Profile as the preferred mechanism for representing Plan Monitor name and phone number. Provide guidance in the IG indicating that CareTeam is the recommended approach for new implementations, while EpisodeOfCare remains available for backward compatibility. Evaluate whether other eLTSS provider mappings (e.g., service providers, care coordinators) should also migrate to CareTeam, and document a phased transition plan.
#### Justification
An incremental approach reduces the risk of breaking existing implementations that rely on the EpisodeOfCare profile. By introducing CareTeam as the preferred path while preserving EpisodeOfCare, implementers can migrate at their own pace. This also allows time to resolve the outstanding questions about SNOMED codes and whether other provider mappings should be consolidated through CareTeam.

---

### Disposition C: Decline
#### Proposal
Decline the change request on the basis that the current STU2 specification does not preclude the use of CareTeam for Plan Monitor information, and implementers can already adopt this approach without normative changes to the IG.
#### Justification
As noted by bheale, the current STU2 does not prevent implementers from using CareTeam or PractitionerRole to represent Plan Monitor information. Adding a formal CareTeam profile and removing EpisodeOfCare introduces breaking changes without clear evidence of implementer demand. The outstanding questions about SNOMED codes and broader provider mapping implications remain unresolved, and proceeding without implementer consensus could result in premature normative changes.

---

### Recommendation
**Recommended disposition:** B

Disposition B offers the most balanced path forward. The ticket has a long history — originally filed in 2019, resolved as "Considered for Future Use" in 2020, and re-opened in 2024 — which indicates ongoing uncertainty about the best approach. Introducing a CareTeam eLTSS Profile as the recommended mechanism for Plan Monitor mapping addresses the core concern without forcing a breaking change on existing implementations. It preserves backward compatibility with the EpisodeOfCare profile while signaling a clear direction for new implementations. This incremental approach also provides time to resolve the open questions about SNOMED codes for the Plan Monitor role and whether other eLTSS provider mappings should be consolidated through CareTeam, allowing the work group to gather the implementer feedback that was identified as necessary in the original 2020 vote.
