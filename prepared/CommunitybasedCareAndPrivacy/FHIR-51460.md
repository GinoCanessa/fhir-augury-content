## Summary

This ticket requests that the PACIO Re-Assessment Timepoints IG expand its US Core dependency from US Core 3.1.1 (the only version currently supported) to also include US Core 6.1.0, 7.0.0, and 8.0.0. The goal is to leverage recent IG Publisher infrastructure for multi-version US Core support, maximizing adoption opportunity across implementers on different US Core versions.

This is one of a set of parallel tickets filed by David Hill across multiple PACIO IGs:

- **FHIR-51459** — US Standardized Medication Profile (SMP) — Pharmacy WG — *Resolved: Persuasive*
- **FHIR-51460** — Re-Assessment Timepoints — Community-Based Care and Privacy WG — *(this ticket)*
- **FHIR-51461** — Transitions of Care — Patient Care WG — Submitted
- **FHIR-51462** — Personal Functioning and Engagement — Patient Care WG — Submitted

## Details

The PACIO Re-Assessment Timepoints IG currently depends solely on US Core 3.1.1. Since US Core has progressed significantly (6.1.0, 7.0.0, and 8.0.x are now published), the IG is considerably behind. Supporting multiple versions would allow implementers already on newer US Core versions to adopt this IG without being forced back to 3.1.1, while also not breaking compatibility for those still on 3.1.1.

The sibling ticket FHIR-51459 for the SMP IG has already been resolved as **Persuasive**, establishing a precedent for this approach within the PACIO project.

### Implementation Approach (from Community Discussion)

A Zulip thread ([implementers > IG supports multiple US Core versions](https://chat.fhir.org/#narrow/stream/implementers/topic/IG%20supports%20multiple%20US%20Core%20versions), 2026-04-08) documents the PACIO project's planned approach, discussed by Yunwei Wang and Lloyd McKenzie:

1. **Verify profile compatibility** — confirm no breaking changes across supported US Core versions.
2. **Use `compliesWithProfile` extension** — add `structuredefinition-compliesWithProfile` extensions to each profile's StructureDefinition declaring compatibility with each supported US Core version.
3. **Derive from the latest version** — Da Vinci has taken the approach of deriving from the latest US Core (e.g., 7.0.0 or 8.0.1) to keep the most recent extensions in scope.
4. **Add narrative guidance** — include a section in the IG's general guidance clarifying multi-version support.
5. **Handle US Core 3.1.1 differences** — Lloyd McKenzie noted there are known differences with 3.1.1 that produce errors; patches exist for most (e.g., PractitionerRole requiring both organization and practitioner). The Da Vinci BR guides' `ignoreWarnings` files contain references to needed patches.

### Known Challenges

A related Zulip thread ([IG creation > Odd warnings after updating a dependency version](https://chat.fhir.org/#narrow/stream/IG%20creation/topic/Odd%20warnings%20after%20updating%20a%20dependency%20version), 2025-06-18) documents Brian Meshell encountering 70+ warnings in the PACIO PFE IG when updating a transitive dependency, caused by the IG Publisher resolving canonical references to unexpected US Core versions. This highlights the need for careful version pinning and use of `pin-canonicals` IG parameters.

A prior ticket **FHIR-48414** (Da Vinci HRex) argued that multi-version US Core support in a single IG is "misguided," but it was resolved as **Not Persuasive with Modification**, indicating the community has accepted the multi-version approach when implemented correctly.

## Proposed Dispositions

### Disposition A: Persuasive (Accept as Requested)

Accept the request to add support for US Core 6.1.0, 7.0.0, and 8.0.0 to the Re-Assessment Timepoints IG while retaining US Core 3.1.1 support. The implementation should follow the approach validated by the PACIO project and community:

- Derive profiles from the latest appropriate US Core version (7.0.0 or 8.0.x).
- Add `structuredefinition-compliesWithProfile` extensions for each supported version.
- Add narrative guidance explaining multi-version support.
- Apply known compatibility patches for US Core 3.1.1 differences.

**Rationale:** The sibling ticket FHIR-51459 was resolved as Persuasive, the community has established a working pattern for multi-version support (Da Vinci precedent), and keeping the IG locked to US Core 3.1.1 limits adoption. This is a straightforward alignment with the broader PACIO project direction.

### Disposition B: Persuasive with Modification (Limit to Fewer Versions)

Accept the intent but modify the scope to support only US Core 6.1.0 and 7.0.0 (dropping 8.0.0), deriving from US Core 7.0.0 as the parent. This reduces the compatibility surface while still significantly modernizing the IG from its current 3.1.1-only state.

**Rationale:** US Core 8.0.x may still be relatively new, and supporting fewer versions reduces the testing and maintenance burden. The IG can add 8.0.0 support in a future cycle once adoption is clearer. This mirrors Da Vinci's choice to target 7.0.0 rather than 8.0.x.

### Disposition C: Not Persuasive (Decline)

Decline the request. The Re-Assessment Timepoints IG should focus on updating to a single newer US Core version rather than supporting multiple versions simultaneously, avoiding the complexity of multi-version profile compatibility.

**Rationale:** Multi-version support adds authoring complexity, build warnings, and testing burden. Implementers needing newer US Core versions should update fully rather than relying on backward-compatible multi-version profiles. However, this position contradicts the broader PACIO project direction and the Persuasive resolution of the sibling ticket FHIR-51459.

## Recommendation

**Disposition A: Persuasive** — This ticket should be accepted as requested. The PACIO project has already established this as a cross-IG initiative (FHIR-51459 resolved Persuasive), the community has validated the implementation pattern (compliesWithProfile extensions, deriving from latest version), and practical guidance exists from both the Da Vinci precedent and Zulip discussions. The Re-Assessment Timepoints IG is notably behind at US Core 3.1.1, and expanding support to 6.1.0, 7.0.0, and 8.0.0 directly aligns with maximizing implementer adoption. The work group should coordinate with the other PACIO IG work groups (Patient Care for FHIR-51461/51462) to ensure a consistent approach.
