## Description

This ticket proposes corrections to four FHIRPath invariant expressions that contain errors in logic, context, or type handling. The four invariants addressed are:

1. **ing-1** (Ingredient resource): The current expression `Ingredient.allergenicIndicator.where(value='true')` incorrectly uses `.where(value='true')` to check a boolean field. The proposed fix changes it to `Ingredient.where(allergenicIndicator=true)`, which correctly tests the boolean value.

2. **cmd-4** (ConceptMap resource): The current expression `(noMap.exists() and target.empty()) or noMap.empty()` does not correctly capture the intended logic. The proposed fix uses `(noMap.exists() and noMap=true) implies target.empty()`, which more clearly expresses the business rule that when `noMap` is true, `target` must be empty.

3. **nsd-2** (NamingSystem resource): The current expression runs in `context: uniqueId` using `$this.where(preferred = true).select(type).isDistinct()`. The proposed fix changes the context to `root` and rewrites the expression as `uniqueId.where(preferred = true).select(type).isDistinct()`, which correctly evaluates across all uniqueId entries rather than within each one individually.

4. **nsd-3** (NamingSystem resource): The current expression runs in `context: uniqueId` using `$this.where(authoritative = true).select(type | period).isDistinct()`. The proposed fix changes the context to `root` and rewrites the expression as `uniqueId.where(authoritative = 'true').select(type.toString() & period.start.toString() & period.end.toString()).isDistinct()`. This corrects the context issue and improves the distinctness check by concatenating string representations of type and period boundaries rather than using the union operator `|`.

- **Summary:** Eric Haas revisited nsd-3 and confirmed via FHIRPath-Lab testing that the current invariant expression passes positive unit tests but **cannot detect overlapping periods** — it only checks for exact duplicates of concatenated type/period strings. Brian Postlethwaite developed an alternative FHIRPath expression using `defineVariable` that can properly detect period overlaps by iterating over uniqueId entries and comparing period boundaries. ## Analysis

### Key Observations

1. **Duplicate ticket already resolved.** FHIR-40626 contains identical content and was resolved as Persuasive by the FHIR Infrastructure work group and published. This strongly suggests the corrections proposed here have already been applied (at least partially) to the specification.

2. **Overlapping ticket also resolved.** FHIR-40619 covers nsd-3 and additional invariants, also resolved as Persuasive and published.

3. **nsd-3 proposed fix contains a bug.** The proposed expression uses `authoritative = 'true'` (string comparison against a boolean), which FHIR-43169 identified and corrected. That fix has been applied.

4. **nsd-3 remains fundamentally limited.** Even with the corrections, the nsd-3 invariant cannot detect overlapping periods — it only checks for exact string-concatenation duplicates. FHIR-40608 tracks this deeper issue, and the recent Zulip discussion (April 2026) shows Brian Postlethwaite has developed a `defineVariable`-based FHIRPath approach that properly handles period overlap detection.

5. **Work group mismatch.** This ticket is assigned to Biomedical Research & Regulation, but the invariants it corrects (ing-1, cmd-4, nsd-2, nsd-3) span multiple resources (Ingredient, ConceptMap, NamingSystem) typically managed by other work groups. The duplicate FHIR-40626 was correctly assigned to FHIR Infrastructure.

## Proposed Dispositions

### Disposition A: Persuasive — Accept as Technical Corrections
**Rationale:** The corrections are technically sound improvements to broken or misleading invariant expressions. The changes to ing-1, cmd-4, nsd-2, and nsd-3 all fix genuine errors in FHIRPath logic, context, or type handling. Accept all four proposed changes, noting that the nsd-3 expression should use `authoritative = true` (boolean, not string `'true'`) per the fix already applied via FHIR-43169.

**Risk:** Low for ing-1, cmd-4, nsd-2. The nsd-3 fix is incomplete (does not detect period overlaps per FHIR-40608) but is still an improvement over the status quo.

### Disposition B: Not Persuasive — Duplicate, Already Addressed
**Rationale:** The identical content was already submitted as FHIR-40626 (assigned to FHIR Infrastructure), which was resolved as Persuasive and published. FHIR-40619 also covered the nsd-3 correction and was published. Since these changes have already been applied to the specification through the sibling tickets, this ticket is redundant and should be closed without further action.

**Risk:** None. The desired changes are already in the specification.

### Disposition C: Withdraw — Recommend Reporter Retract
**Rationale:** Given that FHIR-40626 is an exact duplicate that has already been resolved and published, recommend the reporter (Eric Haas) withdraw this ticket. The work group can note that the changes were applied via FHIR-40626 and FHIR-40619, and that the remaining nsd-3 overlap detection issue is tracked separately in FHIR-40608.

**Risk:** None. Leaves the specification unchanged from its current (already corrected) state.

## Recommendation

**Disposition B: Not Persuasive — Duplicate, Already Addressed** is recommended.

The identical corrections were submitted as FHIR-40626 to the FHIR Infrastructure work group and resolved as Persuasive with status Published. The nsd-3 correction was also covered by FHIR-40619 (Published) and the string-vs-boolean bug in the nsd-3 fix was subsequently corrected by FHIR-43169 (Applied). Since all four proposed corrections have already been processed and applied through sibling tickets, this ticket should be closed as a duplicate. The deeper nsd-3 period overlap limitation is tracked separately in FHIR-40608.
