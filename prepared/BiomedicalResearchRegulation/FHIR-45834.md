## Analysis

### Context
This ticket is part of a broad campaign by Ron G. Parker requesting that HL7 IGs anchor their value sets in THO (terminology.hl7.org) for broader reuse. Identical or near-identical tickets exist across multiple IGs (mCODE, QI Core, and others).

### History of Resolution
This ticket has been resolved and re-opened **twice**:
1. Originally resolved as "Considered for Future Use" during STU1 ballot (voted 14-0-0). The workgroup acknowledged the request but deferred, noting that PQ-CMC leverages the Route of Administration value set from the FDA's SPL on FHIR project, and NCI Thesaurus is the authoritative source.
2. Re-opened post-STU1 publication (2024-12-11) due to a GitHub migration issue that reverted resolutions.
3. Re-opened again post-STU2 publication (2025-07-08) for the same reason.

### Key Considerations
- The Route of Administration value set originates from the NCI Thesaurus, maintained by the FDA's SPL on FHIR project — PQ-CMC is a consumer, not the owner.
- The workgroup has consistently deferred THO anchoring, stating that either SPL on FHIR or PQ-CMC would consider registration when appropriate.
- FHIR-38566 (QI Core) was resolved as "Not Persuasive" for a similar request, showing that THO anchoring is not universally required.
- The ticket has been re-opened due to administrative/technical issues (GitHub IG transitions), not because the workgroup changed its position.

---

## Proposed Dispositions

### Disposition A: Considered for Future Use
**Rationale:** Re-affirm the prior 14-0-0 resolution. The PQ-CMC IG appropriately consumes the Route of Administration value set from the NCI Thesaurus via the SPL on FHIR project. Anchoring in THO would require coordination with the SPL on FHIR project team and NCI, which is outside PQ-CMC's direct control. The workgroup should document that THO registration will be revisited when SPL on FHIR progresses to a point where cross-IG value set sharing is practical.

**Vote text:** "PQ-CMC continues to leverage the Route of Administration value set sourced from NCI Thesaurus via the SPL on FHIR project. Anchoring in THO remains a future consideration pending coordination with the SPL on FHIR project. NCI Thesaurus remains the source of truth."

### Disposition B: Not Persuasive
**Rationale:** The value set in question is owned and maintained by NCI Thesaurus, not by the PQ-CMC IG. It is not within the PQ-CMC workgroup's authority to register an externally-owned value set in THO. The appropriate action would be for the requestor to file a THO/UTG ticket directly. This aligns with the precedent set by FHIR-38566 (QI Core).

**Vote text:** "The Route of Administration value set is sourced from NCI Thesaurus and maintained by the FDA's SPL on FHIR project. Registering externally-owned terminology in THO is outside PQ-CMC's scope. The submitter should engage TSMG or the SPL on FHIR project directly for THO registration."

### Disposition C: Persuasive with Modification
**Rationale:** Accept the principle that broadly useful value sets should be in THO, but limit the action to creating a UTG ticket requesting THO registration of the Route of Administration value set, with the SPL on FHIR project as the responsible party. PQ-CMC would then reference the THO-hosted value set once available.

**Vote text:** "Agreed that THO anchoring would benefit cross-IG reuse. A UTG ticket will be created requesting THO registration of the Route of Administration value set, coordinated with the SPL on FHIR project. PQ-CMC will reference the THO-hosted value set once it is available."

---

## Recommendation

**Disposition A: Considered for Future Use**

This is the recommended disposition because:
1. It is consistent with the prior workgroup votes (14-0-0 on two separate occasions), indicating strong consensus.
2. The workgroup has already acknowledged the value of THO anchoring but correctly identifies that timing depends on the SPL on FHIR project's readiness.
3. The ticket was re-opened due to administrative issues (GitHub IG transitions), not because the workgroup's position changed or new information emerged.
4. It avoids overstepping PQ-CMC's authority over externally-owned terminology while keeping the door open for future action.
