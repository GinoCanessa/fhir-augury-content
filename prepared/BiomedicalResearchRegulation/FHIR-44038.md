## Description

## Analysis

This ticket sits at the intersection of two related efforts:
1. **PQ-CMC (FDA) IG** — FHIR-45733 has already been resolved (Persuasive) to update the Test Category hierarchy to align with NCI Thesaurus content.
2. **PQ (Industry) IG** — This ticket (FHIR-44038) requests the same kind of update for the Industry IG's ObservationDefinition and Observation profiles.

The Zulip discussion reveals that active technical work is underway on implementing Test Category hierarchy via CodeSystem supplements for NCIt, with tooling fixes being made in the FHIR publisher to support this. The re-opening of this ticket post-STU1 signals that the work group considers this work appropriate for the next ballot cycle.

The original 15-0-0 vote to defer indicates strong consensus but no opposition — the question was timing, not merit. Now that STU1 is published and FHIR-45733 has addressed the PQ-CMC side, alignment of the Industry IG is a logical next step.

## Proposed Dispositions

### Disposition A: Persuasive
**Action:** Accept the request and update the PQ Test Category modeling, definitions, and hierarchy in the PQ (Industry) IG's ObservationDefinition and Observation profiles to align with current NCI Thesaurus content.

**Rationale:** FHIR-45733 has already resolved the parallel issue for PQ-CMC, establishing the updated Test Category hierarchy aligned with NCIt. The Industry IG should be brought into alignment. The Zulip discussion confirms that the technical approach (CodeSystem supplements for NCIt hierarchy) is viable and tooling support is being added. The original 15-0-0 vote confirmed the change has merit; only timing was the concern, and STU1 publication has now occurred.

### Disposition B: Persuasive with Modification
**Action:** Accept but scope the work to align specifically with the changes already applied in FHIR-45733 (PQ-CMC), ensuring the Industry IG's Test Categories use the same updated Code values, Display Values, and Definitions from NCIt, and adopt the same CodeSystem supplement approach for hierarchy.

**Rationale:** Rather than a broad "consider updating," this narrows the scope to concrete alignment with the already-resolved PQ-CMC changes, reducing ambiguity and ensuring cross-IG consistency. The CodeSystem supplement approach for hierarchy (validated in the Zulip discussion and publisher fixes) should be explicitly adopted.

### Disposition C: Considered for Future Use
**Action:** Defer again until the CodeSystem supplement tooling for hierarchy is fully stable and the PQ-CMC implementation is proven in production.

**Rationale:** While the tooling is improving (Grahame's publisher fixes), the Zulip discussion reveals open questions about supplement scoping, hierarchy invariants, and terminology server support. Deferring until these are fully resolved would reduce implementation risk. However, this was already the previous disposition and the ticket was deliberately re-opened.

---

## Recommendation

**Disposition B: Persuasive with Modification** is recommended.

The original request is broad ("consider updating"), but the work group now has a concrete template: FHIR-45733's resolution in PQ-CMC provides the specific NCIt-aligned Test Category values and hierarchy to adopt. Scoping the work to align with those established changes ensures consistency between the PQ-CMC and PQ (Industry) IGs while avoiding open-ended scope. The CodeSystem supplement approach for hierarchy has been technically validated and tooling fixes are in progress. Re-deferring (Disposition C) would ignore the deliberate re-opening of the ticket and the progress made since the original deferral.
