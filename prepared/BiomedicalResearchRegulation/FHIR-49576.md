## Description

The ticket requests that a "Plain Language Summary about this Guide" be added to the PQ-CMC IG home page. This is part of an HL7-wide initiative encouraging IGs to voluntarily include plain language summaries to improve accessibility for non-technical audiences, including patients and the general public.

The submitter references the MCC IG homepage as an example ([hl7.org/fhir/us/mcc](https://hl7.org/fhir/us/mcc/#plain-language-summary-about-hl7-and-this-guide)) and notes that FHIR leaders recommend excluding the "About HL7" portion for new summaries. Resources for AI-assisted generation of summaries are provided, along with references to peer-reviewed journals that now require Plain Language Summaries (PLS).

## Proposed Dispositions

### Disposition A: Persuasive
**Action:** Accept the request. Add a plain language summary for the PQ-CMC IG to the centralized [HL7/plain-language](https://github.com/HL7/plain-language) repository as `summaries/hl7.fhir.us.pq-cmc-fda.md`.

**Rationale:** The original blocker — that plain language content should wait until all CMC content was included across STUs — has been removed by the publication of STU2. The HL7 plain language infrastructure is now mature, with TSC approval, centralized tooling in the HL7/plain-language repo, and IG publisher support for inline rendering. Numerous other IGs have already resolved identical tickets as Persuasive and applied summaries (e.g., FHIR-47564 for IPS). This aligns with the HL7-wide initiative for accessibility and transparency.

### Disposition B: Persuasive with Modification
**Action:** Accept the request in principle, but note that the summary should be contributed via the centralized HL7/plain-language repository rather than embedded directly in the PQ-CMC IG source, consistent with the FMG-endorsed approach.

**Rationale:** While the ticket text references adding content "to the home page" as if it were an IG-local change, the current HL7 approach (per Grahame Grieve's FMG proposal) centralizes summaries in the HL7/plain-language repo and renders them via the publisher. Accepting with modification clarifies the implementation path and avoids duplicating content that may later need to be migrated.

### Disposition C: Considered for Future Use
**Action:** Defer action, keeping the ticket open for a future ballot cycle.

**Rationale:** Although STU2 has been published, the work group may wish to prioritize remaining CMC content development over plain language summary creation. The summary could be deferred until the IG reaches greater content stability, or until the centralized plain language tooling is further matured. However, this disposition is weaker given that the original stated reason for deferral (STU2 publication) has been satisfied.

## Recommendation

**Disposition A: Persuasive** is recommended.

The original rationale for deferral — waiting until all CMC content was included via STUs — is no longer applicable since STU2 has been published, which is precisely why lynn_laakso re-opened the ticket. The HL7 plain language infrastructure is production-ready with TSC approval, a centralized repository, and publisher integration. Many peer IGs have already applied plain language summaries through this system. The implementation path is straightforward: generate a summary (using the recommended AI prompt with human review), submit it as a PR to [HL7/plain-language](https://github.com/HL7/plain-language) as `summaries/hl7.fhir.us.pq-cmc-fda.md`, and the IG publisher will render it on the home page. The work group's prior 10–0–1 vote in favor of the concept confirms community support.
