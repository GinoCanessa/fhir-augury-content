## Description

The ticket requests that the CDISC Lab implementation guide address two aspects of supplemental variables:

1. **Unmapped FHIR elements as supplemental variables:** Even when a mapping is not present for a particular FHIR element, it is still possible to capture that information as a supplemental variable if the information is relevant to a study.
2. **CDISC supplemental variable registry:** CDISC maintains a registry of "standard" supplemental variables that could potentially be leveraged. The implementation guide does not currently cover mappings to variables from that registry.

This was discussed and created during the Jan 29, 2021 CDISC call.

## Proposed Dispositions

### Disposition A: Persuasive (Accept as-is)

**Action:** Accept the ticket as persuasive and add documentation to the CDISC Lab IG explaining supplemental variable handling.

**Rationale:** The request is straightforward and addresses a genuine documentation gap. The IG should explain that (1) unmapped FHIR elements can still be captured as supplemental variables when relevant to a study, and (2) CDISC's registry of standard supplemental variables exists but is not currently mapped in this IG. This is informational/guidance content that improves implementer understanding without requiring structural changes. The resolution is already marked as "Persuasive" and the request aligns with the broader set of ballot feedback from the same cycle (e.g., FHIR-30528, FHIR-30538) asking for better explanation of mapping scope and variable selection.

### Disposition B: Persuasive with Modification

**Action:** Accept the request but expand the scope to also include a brief discussion of how supplemental variables from the CDISC registry might be mapped in future versions, and provide a reference link to the CDISC controlled terminology/registry.

**Rationale:** While the original request is valid, simply noting the registry exists without providing a reference or forward-looking guidance may leave implementers unclear on next steps. Adding a link to the CDISC supplemental qualifier registry and a brief statement about potential future mapping work would make the guidance more actionable. This minor expansion stays within the spirit of the original request.

### Disposition C: Not Persuasive

**Action:** Decline the request. The IG already implicitly conveys that unmapped elements are outside scope, and the CDISC registry is external content.

**Rationale:** One could argue that the IG's existing mapping tables already convey what is and is not mapped, and that pointing to an external CDISC registry is out of scope for an HL7 implementation guide. However, this disposition is weak because the feedback came from a CDISC coordination call and the guidance would help implementers bridge the gap between FHIR and CDISC submissions.

## Recommendation

**Disposition A: Persuasive** — Accept the ticket as-is. The request is a clear, bounded documentation improvement asking for two specific explanatory paragraphs about supplemental variable handling. The resolution is already marked Persuasive, the reporter (Lloyd McKenzie) created it from an official CDISC coordination call, and similar tickets from the same ballot cycle (FHIR-30528, FHIR-30538) reflect a pattern of feedback requesting better explanation of mapping scope. Adding this guidance is low-risk, high-value, and requires no structural changes to the IG.
