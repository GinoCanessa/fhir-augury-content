## Summary

This ticket requests that the Biomedical Research & Regulation (BR&R) work group evaluate 19 CodeSystem/ValueSet pairs used by the SubstanceDefinition resource for potential migration to HL7 Terminology (THO). These are FHIR-core-defined code systems and value sets that may benefit from independent lifecycle management in THO if they are expected to evolve between FHIR releases or could be re-used by CDA, V2, or other HL7 specifications. This ticket is part of a broader systematic review of THO migration candidates across regulated-product resources, filed by Lloyd McKenzie.

## Details

### Description

The SubstanceDefinition resource references 19 CodeSystem/ValueSet pairs that are candidates for migration to THO. ## Proposed Dispositions

### Disposition A: Accept — Migrate all 19 CodeSystem/ValueSet pairs to THO

**Resolution:** Persuasive

Accept the request and migrate all 19 CodeSystem/ValueSet pairs listed in the ticket to THO. Each of these code systems is narrowly scoped to SubstanceDefinition elements and represents domain-specific terminology (substance grades, stereochemistry, optical activity, source material taxonomy, naming authorities, etc.) that may need to evolve independently of the FHIR release cycle. Moving them to THO enables:
- Independent versioning and updates between FHIR releases
- Potential re-use by CDA, V2, and other HL7 specifications
- Alignment with the broader THO migration effort across sibling BR&R resources (FHIR-55048, FHIR-55042, FHIR-55051, FHIR-55043, etc.)

Before migration, ensure CodeSystems meet shareable profile requirements (notably adding missing "description" elements, as identified in FHIR-37449). The `medicinal-product-domain` CodeSystem is shared with MedicinalProductDefinition (FHIR-55048), so migration should be coordinated across both tickets.

### Disposition B: Accept with selective migration — Migrate only value sets with cross-specification re-use potential

**Resolution:** Persuasive with Modification

Accept the request in principle but migrate only the subset of CodeSystem/ValueSet pairs that have realistic cross-specification re-use potential or are likely to need updates between FHIR releases. Specifically, prioritize:
- **medicinal-product-domain** (shared with MedicinalProductDefinition, cross-resource re-use)
- **substance-name-authority** (naming authorities may evolve as new regulatory bodies emerge)
- **substance-name-type** and **substance-name-domain** (naming conventions may need updates)
- **substance-source-material-type/genus/species/part** (biological taxonomy evolves)
- **substance-structure-technique** (analytical techniques evolve as technology advances)

Retain in FHIR core the highly stable, narrow code systems unlikely to change (e.g., substance-stereochemistry with its limited set of chemistry concepts, substance-optical-activity, substance-weight-type). This reduces THO maintenance burden while capturing the primary benefits.

### Disposition C: Decline — Retain all value sets in FHIR core

**Resolution:** Not Persuasive

Decline the migration request. These 19 CodeSystem/ValueSet pairs are tightly scoped to SubstanceDefinition and are unlikely to be re-used by CDA, V2, or other specifications, as SubstanceDefinition is a FHIR-specific resource without direct counterparts in older HL7 standards. The value sets are small, stable, and domain-specific. Migrating them to THO adds governance overhead (separate publication cycle, THO maintenance) without clear benefit. If individual value sets need updating in the future, they can be migrated to THO at that time on a case-by-case basis.

## Recommendation

**Disposition A: Accept — Migrate all 19 CodeSystem/ValueSet pairs to THO**

This is recommended for the following reasons:

1. **Consistency across the BR&R portfolio:** This ticket is one of a coordinated set of THO migration requests across all regulated-product resources (FHIR-55048, FHIR-55042, FHIR-55051, FHIR-55043, FHIR-55047, FHIR-55052). Migrating SubstanceDefinition value sets in isolation from sibling resources would create an inconsistent governance model. An all-or-nothing approach across the BR&R resource family is cleaner.

2. **Domain terminology evolves:** Substance characterization techniques, source material taxonomy, naming authorities, and regulatory domains are all subject to change as science and regulation advance. THO's independent versioning enables timely updates without waiting for a FHIR release cycle.

3. **Shared CodeSystems require coordination:** The `medicinal-product-domain` CodeSystem is already shared between SubstanceDefinition and MedicinalProductDefinition. THO is the natural home for cross-resource terminology.

4. **Low risk:** Migration to THO is a well-understood process. The value sets are already self-contained with dedicated CodeSystems, making migration straightforward. The shareable profile compliance issue (FHIR-37449) should be resolved as part of migration prep.

5. **Alignment with HL7 strategy:** THO migration for domain-specific terminology is an established HL7 best practice, and these value sets fit the criteria described in the ticket (realistic potential for change, possible cross-spec utility).
