## Description

The ticket requests a comprehensive review of the Pharmaceutical Quality (Industry) IG to identify additional standard terminologies that should be included and to add terminology bindings to profile elements where applicable.

This is a broad improvement request aimed at strengthening the terminology foundation of the PQ Industry IG by ensuring that profile elements reference appropriate standard terminologies (e.g., NCIt, EDQM, SNOMED CT, LOINC) with suitable binding strengths.

## History & Prior Dispositions

- **2024-02-21**: Previously resolved as **"Considered for Future Use"** with a vote of Diana Wright / Rob Hausam: **15-0-0**. The resolution noted the request for consideration in future uses.
- **2024-05-08**: Resolution was **reverted** by Lynn Laakso with the comment "re-open post STU1 publication," indicating the work group decided to revisit this ticket for work beyond STU1.

## Analysis

This ticket represents a **broad, ongoing initiative** to improve terminology coverage across the PQ Industry IG. Key considerations:

1. **Scope**: The request is intentionally open-ended — "review the IG" for all terminology gaps. This could encompass dozens of profile elements across multiple profiles.
2. **STU1 Context**: The ticket was originally deferred as "Considered for Future Use" during STU1 ballot (unanimously, 15-0-0), then deliberately re-opened post-publication, indicating the work group recognizes its importance for STU2.
3. **Related Work**: FHIR-44038 (test category modeling) and the Zulip discussion about NCIt CodeSystem supplements show active parallel work on PQ terminology that should inform or be coordinated with this ticket.
4. **THO Compliance**: Any new code systems or value sets introduced must be anchored in THO per the resolved FHIR-44269 series of tickets.
5. **Binding Strength Precedent**: FHIR-43817 (EDQM binding strength in UV realm) was resolved as Not Persuasive, suggesting the work group has considered and accepted certain binding strength decisions.

## Proposed Dispositions

### Disposition A: Persuasive — Accept and Scope for STU2
**Action:** Accept the ticket and define a concrete work plan for reviewing and enhancing terminology bindings across the PQ Industry IG for the STU2 cycle.
- Systematically review each profile in the IG for elements that could benefit from standard terminology bindings (e.g., NCIt, EDQM, SNOMED CT, LOINC, UCUM)
- Coordinate with FHIR-44038 on test category terminology updates
- Ensure all new code systems/value sets are anchored in THO per established policy
- Assign to a terminology-knowledgeable member (e.g., the reporter, Robert Hausam)
- **Rationale:** The work group already recognized this as important (15-0-0 vote to consider for future) and deliberately re-opened it. The STU2 cycle is the appropriate time to address it. Active community work on NCIt supplements (Zulip thread) demonstrates readiness.

### Disposition B: Persuasive with Modification — Accept with Narrowed Scope
**Action:** Accept but narrow the scope to specific high-priority areas rather than a full IG review.
- Focus on profiles with the most implementer impact (e.g., ObservationDefinition, Observation profiles for test categories)
- Prioritize bindings for elements currently lacking any terminology reference
- Defer comprehensive review of binding strengths on already-bound elements to a future cycle
- **Rationale:** A full review is valuable but potentially too broad for a single ticket. Narrowing scope ensures deliverable progress while the broader goal remains achievable across multiple tickets.

### Disposition C: Deferred — Considered for Future Use (Re-defer)
**Action:** Defer again, keeping the ticket open for a future cycle beyond STU2.
- The current STU2 cycle may already have significant terminology work in flight (FHIR-44038, THO anchoring follow-ups)
- Additional terminology review could be deferred until the IG has more implementation experience to identify real-world terminology gaps
- **Rationale:** While important, the open-ended nature of this request may be premature without more implementer feedback on which specific terminology gaps are causing issues.

## Recommendation

**Disposition A: Persuasive — Accept and Scope for STU2**

This ticket should be accepted for the STU2 cycle. The work group unanimously acknowledged its importance during STU1 ballot and intentionally re-opened it post-publication. The reporter (Robert Hausam) has deep terminology expertise relevant to the PQ domain, and active community work on NCIt terminology supplements demonstrates that the ecosystem is ready for this enhancement. Coordinating with the parallel FHIR-44038 ticket on test category terminology will maximize the value of both efforts. The highest priority designation further supports addressing this now rather than deferring again.
