## Description

The submitter notes that the existing wording in the FHIR to OMOP FHIR IG states:

> "Fast Healthcare Interoperability Resources (FHIR) captures timestamps down to the millisecond to support real-time clinical workflows and system interoperability."

The request is to **add** that FHIR date and datetime are based on the **ISO 8601 standard**. This is a clarification to make the IG's description of FHIR temporal data types more precise and standards-aligned.

## Resolution Description

> "We will add the fact that FHIR date/datetime are based on ISO standards."

The resolution was **Persuasive** — the workgroup agreed the addition is warranted without modification.

## Proposed Dispositions

### Disposition A: Confirm as Applied (Persuasive)

**Rationale:** The ticket has already been resolved as **Persuasive** and implemented. The comments from daverag confirm the change was made in GitHub on January 8, 2026, and the ticket status is "Applied." The request is straightforward — adding that FHIR date and datetime conform to ISO 8601 is factually correct and aligns with the core FHIR specification (see FHIR-41370, which made the same clarification for FHIR Core). No modification is needed to the original resolution.

**Action:** Confirm the applied resolution. No further work required.

### Disposition B: Persuasive with Modification — Verify Wording Alignment

**Rationale:** While the change has been applied, the workgroup may want to verify that the added wording in the OMOP IG aligns precisely with how FHIR Core now documents ISO 8601 conformance (per FHIR-41370). Since both tickets address the same underlying gap, the OMOP IG's language should be consistent with the core spec's approach. The workgroup could review the exact text added and ensure it references the same ISO 8601 standard and uses consistent terminology.

**Action:** Confirm the resolution but request a review of the exact wording for consistency with FHIR Core's ISO 8601 reference.

### Disposition C: Not Persuasive — Close Without Further Action

**Rationale:** The change has already been implemented and the description of FHIR timestamps in the OMOP IG was supplementary context rather than normative mapping guidance. Adding an ISO 8601 reference to introductory text is informational but does not affect the IG's technical mapping content. The workgroup could determine this was unnecessary editorial detail.

**Action:** Close without further action, accepting the change as a minor editorial improvement.

## Recommendation

FHIR date and datetime types are indeed based on ISO 8601, as already documented in FHIRPath and now in the core FHIR specification (FHIR-41370). The change has been implemented in GitHub and the ticket status is "Applied." There is no ambiguity in the resolution and no reason to modify or reject it. Confirming the applied resolution allows the workgroup to move forward without additional overhead.
