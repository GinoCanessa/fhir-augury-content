## Summary

Harold Solbrig reported that hyperlinks to the raw Shape Expressions (ShEx) definitions on FHIR resource pages are broken. The issue affects all artifacts (not just Patient). These links appear in the "Alternate definitions" section of each resource page and are expected to point to the raw `.shex` file for the resource. The ticket has been in **Triaged** status since 2020 and remains unresolved and unassigned.

---

## Details

The FHIR specification includes ShEx (Shape Expressions) schemas as validation artifacts for the RDF/Turtle serialization of FHIR resources. Each resource page provides an "Alternate definitions" section with links to the raw ShEx definition file. The reporter discovered that these hyperlinks no longer resolve to the expected content. Since the issue applies to all artifacts, it is a systemic build/publishing problem rather than an isolated broken link.

The relevant source files in the HL7/fhir GitHub repository include:
- `source/downloads.html` — references `fhir.schema.shex.zip` for bulk ShEx downloads
- `source/rdf.html` — references `fhir.shex` for the complete FHIR RDF schema

This ticket is closely related to other ShEx issues that have been addressed in more recent FHIR versions, particularly around ShEx schema correctness and R5/R6 updates.

## Proposed Dispositions

### Disposition A: Accept (Persuasive)

**Action:** Fix the broken hyperlinks to raw ShEx definitions on all resource pages.

**Rationale:** The links to raw ShEx definitions are documented features of the specification and should work. The build process that generates these pages should be updated to produce correct URLs pointing to the raw `.shex` files for each resource. This is a straightforward publishing/build fix. Given that FHIR-39074 (Applied, Persuasive) already addressed updating ShEx schemas on resource pages for R5, and FHIR-53178 covers R5→R6 ShEx improvements, the link fix can be verified and corrected as part of ongoing build maintenance.

**Scope:** Update the FHIR build tooling to ensure that hyperlinks to raw ShEx definitions resolve correctly for all resource pages.

---

### Disposition B: Persuasive with Modification — Verify Current State First

**Action:** Verify whether the broken links have already been fixed as a side effect of FHIR-39074 or subsequent build changes, and close if resolved; otherwise, fix the links.

**Rationale:** This ticket was filed in March 2020 and has not been updated since August 2020. Significant work on ShEx schemas has occurred since then (FHIR-39074 updated ShEx on resource pages for R5; FHIR-53178 covers R5→R6 changes). It is plausible that the broken links were inadvertently fixed during these updates. The work group should verify the current state of the links in the latest build before committing to additional development work. If the links are now functional, close as already addressed. If still broken, fix them.

**Scope:** Verify current link status in CI build output; fix if still broken.

---

### Disposition C: Decline (Not Persuasive)

**Action:** Close as superseded by FHIR-39074 and FHIR-53178.

**Rationale:** The ShEx landscape has changed substantially since this ticket was filed. FHIR-39074 (Applied) updated ShEx schemas on resource pages for R5, and FHIR-53178 (Resolved, Persuasive) covers comprehensive R5→R6 RDF/Turtle and ShEx changes. Any remaining link issues would be better addressed as part of those broader efforts rather than this standalone 2020 ticket. The ticket is stale (5+ years old, unassigned, no comments) and the original context may no longer apply.

---

## Recommendation

**Disposition B: Persuasive with Modification** is recommended.

This ticket is over 5 years old and the FHIR ShEx infrastructure has undergone significant rework since it was filed. The most efficient path is to verify whether the links are already working in the current build (given FHIR-39074 and FHIR-53178 work). If the links remain broken, fix them as part of ongoing build maintenance. If they are already working, close the ticket as resolved by subsequent work. This approach avoids duplicate effort while ensuring the issue is not silently ignored.
