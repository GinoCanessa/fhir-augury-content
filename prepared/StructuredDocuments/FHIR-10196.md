This is likely a copy-paste error by the reporter, who filed a batch of nearly identical tickets for multiple sections at the same time (see Related Tickets). ## Analysis

### Context
This ticket is one of a batch of four tickets (FHIR-10193 through FHIR-10196) filed by Corey Spears on 2016-06-16, all requesting that various sections in the C-CDA on FHIR IG reference DAF (Data Access Framework) resource profiles in their `Composition.section.entry` elements instead of C-CDA on FHIR's own generic resource references.

### Key Considerations

1. **Title/Description Mismatch:** The title says "Social History Section" but the description body says "Immunization section." Given the sibling ticket FHIR-10195 explicitly covers the Immunization section, this ticket most likely concerns the Social History Section.

2. **DAF Superseded by US Core:** The DAF profiles referenced in this ticket (e.g., `http://hl7.org/fhir/StructureDefinition/daf-*`) were part of the DSTU2-era Data Access Framework. DAF has since been superseded by **US Core** profiles. Any resolution should consider whether the intent is now satisfied by aligning with US Core profiles rather than the obsolete DAF profiles.

3. **Related Resolution (FHIR-11883):** The closely related ticket FHIR-11883 ("All entries for the Social History section seems to be restricted to smoking status") was resolved as **Persuasive** and published, indicating the workgroup agreed the Social History section's entry references were too restrictive. This suggests partial overlap — FHIR-11883 may have already addressed some of the concerns in this ticket.

4. **Age of Ticket:** Filed in 2016 (DSTU2 era), reopened in 2024 post STU1.2 publication. The C-CDA on FHIR IG has undergone significant evolution. The workgroup should verify whether the current R4 version of the IG already addresses this concern.

5. **Batch Treatment:** The sibling tickets (FHIR-10193, FHIR-10194, FHIR-10195) are all in the same Triaged/Unresolved state. The workgroup may wish to address all four as a group.

## Proposed Dispositions

### Disposition A: Not Persuasive (Recommend)

**Rationale:** The underlying request — that section entries reference DAF profiles — is outdated. DAF profiles have been superseded by US Core. The C-CDA on FHIR IG has evolved significantly since 2016, and the related ticket FHIR-11883 (resolved as Persuasive, Published) already addressed the Social History section's overly restrictive entry references. The current R4 version of the IG likely reflects appropriate profile references. The specific DAF-based change requested is no longer applicable.

**Suggested Comment:** "The DAF profiles referenced in this ticket have been superseded by US Core profiles. The related concern about Social History section entries being too restrictive was addressed in FHIR-11883 (resolved Persuasive). The current C-CDA on FHIR R4 IG reflects updated profile references. Closing as Not Persuasive given the request is no longer actionable in its original form."

### Disposition B: Persuasive with Modification

**Rationale:** The spirit of the request — that section entries should reference the appropriate US Realm profiles rather than generic FHIR resources — remains valid even though the specific DAF profiles cited are obsolete. Reinterpret the request to ensure the Social History Section's `Composition.section.entry` references the appropriate US Core profiles (e.g., US Core Smoking Status Observation, US Core Simple Observation) rather than base FHIR resources.

**Suggested Comment:** "The original request referenced DAF profiles which are now obsolete. Reinterpreting in the context of current US Core: the Social History Section entry references should align with applicable US Core profiles. Verify current IG and update if needed."

### Disposition C: Duplicate / Refer to FHIR-11883

**Rationale:** FHIR-11883 addressed the same fundamental issue (Social History section entry references) and was resolved as Persuasive. This ticket may be a duplicate of that resolved concern.

**Suggested Comment:** "This concern overlaps substantially with FHIR-11883, which was resolved as Persuasive and published. Closing as duplicate."

## Recommendation

**→ Disposition A: Not Persuasive**

The specific change requested (referencing DAF profiles) is no longer applicable as DAF has been superseded. The broader concern about Social History section entry restrictions was already addressed via FHIR-11883. The workgroup should verify whether the current IG state satisfies the intent, but the ticket as written is not actionable. The sibling tickets (FHIR-10193, FHIR-10194, FHIR-10195) should be reviewed together under similar reasoning.
