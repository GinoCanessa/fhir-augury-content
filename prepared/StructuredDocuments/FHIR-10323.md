## Description

The reporter requests that `Composition.type` in the C-CDA on FHIR Consultation Note profile be bound to the value set `http://hl7.org/fhir/ValueSet/2.16.840.1.113883.11.20.9.31` (the OID-based LOINC value set for Consultation Note document types from C-CDA). The reporter also notes the value set itself needs to be added to the IG build so that it resolves correctly.

The original related URL points to the C-CDA Consultation Note Composition profile: `http://build.fhir.org/ccda/ccda-consultation-note-composition.html`.

## Analysis

This is a longstanding issue (filed 2016, re-triaged 2024) from a batch of related tickets (FHIR-10321, FHIR-10322, FHIR-10323, FHIR-10338) all filed by Corey Spears on the same day. All concern missing or unbound `Composition.type` value sets in the US C-CDA on FHIR IG.

**Key considerations:**

1. **Age and IG evolution:** The C-CDA on FHIR IG has undergone significant changes since 2016. The referenced OID-based value set URL (`http://hl7.org/fhir/ValueSet/2.16.840.1.113883.11.20.9.31`) uses a pattern that may no longer be the preferred approach for FHIR value set references.
2. **Ecosystem alignment:** FHIR-28078 (Applied/Persuasive) already aligned `DocumentReference.type` with `Composition.type` using LOINC doc-typecodes in FHIR Core. The newer FHIR-55633 addresses terminology binding alignment for type and category with US Core and FHIR Clinical Document.
3. **Sibling tickets:** FHIR-10321, FHIR-10322, and FHIR-10338 are essentially the same class of issue for different C-CDA document types. They should be resolved together or consolidated.
4. **Current state of the IG:** The Consultation Note profile may already have been updated in more recent versions of the C-CDA on FHIR IG. The workgroup should verify the current state of `Composition.type` bindings in the active build.

## Proposed Dispositions

### Disposition A: Persuasive — Apply the requested binding
**Action:** Add a `Composition.type` binding to the appropriate LOINC document type value set on the C-CDA Consultation Note Composition profile, and ensure the value set is included in the IG build. Use the current FHIR-canonical form for the value set rather than the OID-based URL originally requested.
**Rationale:** The request is valid — `Composition.type` should be bound to a document-type value set to ensure conformance and validation. The approach should be updated to use current FHIR value set conventions rather than the OID-based URL from 2016.

### Disposition B: Not Persuasive (Duplicate / Covered Elsewhere)
**Action:** Close as addressed by more recent work (e.g., FHIR-55633 or the current C-CDA on FHIR IG build). Consolidate with sibling tickets FHIR-10321, FHIR-10322, and FHIR-10338.
**Rationale:** Given the age of this ticket (8+ years) and the extensive IG evolution since then, the binding may already be in place or superseded by newer terminology alignment efforts. If the current IG build already binds `Composition.type` correctly, this can be closed without further action.

### Disposition C: Considered for Future Use — Defer to next IG update cycle
**Action:** Defer resolution to the next C-CDA on FHIR IG update cycle, and address all sibling tickets (FHIR-10321, FHIR-10322, FHIR-10338) together as a batch.
**Rationale:** The issue is valid but has been open since 2016 without blocking implementation. A coordinated review of all `Composition.type` bindings across all C-CDA document profiles would be more thorough than addressing them individually.

## Recommendation

**Disposition B: Not Persuasive (Duplicate / Covered Elsewhere)** is recommended. This ticket is over 8 years old and the C-CDA on FHIR IG has undergone multiple revision cycles since it was filed. The workgroup should verify whether the current IG build already includes appropriate `Composition.type` bindings for the Consultation Note profile. If bindings are already present, close this and the sibling tickets. If not, apply Disposition A using current FHIR value set conventions rather than the 2016-era OID-based URL. The batch of related tickets (FHIR-10321, FHIR-10322, FHIR-10338) should be reviewed and resolved together regardless of the chosen disposition.
