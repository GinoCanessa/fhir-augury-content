## Summary

This change request asks for section-specific pages in the International Patient Summary (IPS) Implementation Guide. The reporter (Giorgio Cangioli) proposes that each IPS composition section should have its own dedicated page with details about the section and the assets (profiles, value sets, etc.) used by that section. The goal is to improve readability and navigation of the IG.

The ticket was originally filed in May 2018 and was resolved as "Considered for Future Use" in August 2018 by a vote of 16-0-3 (Rob Hausam / Gay Dolin), with the note "Will defer for consideration in the next release." It was subsequently re-opened by Lynn Laakso in November 2022 following an STU update publication for renewed consideration.

## Proposed Dispositions

### Disposition A: Accept (Persuasive)

**Rationale:** Accept the request and create dedicated pages for each IPS composition section (e.g., Medications, Allergies, Problems, Immunizations, etc.), with each page documenting the section's purpose, constraints, referenced profiles, value sets, and examples. This aligns with modern IG authoring best practices where large monolithic pages are broken into focused, navigable sections. The companion ticket FHIR-16199 (splitting the profiles page) supports the same readability concern. The IPS IG has matured significantly since the original 2018 filing, and section-specific pages would benefit implementers navigating an increasingly complex specification.

**Action:** Editors create individual pages per IPS composition section, including section-level narrative, applicable profiles, terminology bindings, and usage guidance. Coordinate with FHIR-16199 to avoid duplication of effort.

### Disposition B: Persuasive with Modification

**Rationale:** Rather than creating fully independent pages for every section, add an enhanced "Sections" overview page with anchor-linked subsections, each providing a summary, the relevant profile(s), and links to the profile detail pages. This achieves the readability improvement with less editorial overhead and avoids fragmenting the IG into too many small pages that could themselves become hard to navigate.

**Action:** Editors create a structured "Sections" page with per-section anchors and summary content, linking out to existing profile pages. Add a table of contents or sidebar navigation to aid discoverability.

### Disposition C: Decline (Not Persuasive)

**Rationale:** The IPS IG already provides artifact listings on the profiles and terminology pages, and the IG publisher's standard navigation provides access to individual profile pages. Creating additional section-specific pages would increase editorial maintenance burden without clear evidence that implementers are struggling with the current structure. The ticket was previously deferred in 2018 by a strong vote margin (16-0-3), and no new community feedback or ballot comments have reinforced this need since re-opening.

**Action:** Close as Not Persuasive. Recommend the reporter raise specific usability concerns with concrete examples if the issue persists.

## Recommendation

**Disposition A: Accept (Persuasive)** is recommended.

The IPS IG has grown substantially since 2018 and now covers many composition sections, each with multiple profiles and terminology bindings. Section-specific pages would significantly improve implementer experience by providing focused, self-contained documentation for each clinical domain (medications, allergies, problems, etc.). This is consistent with the direction taken by other mature IGs (e.g., US Core) that organize content by clinical area. The companion ticket FHIR-16199 reinforces that the current monolithic page structure is a readability concern. Re-opening the ticket post-STU publication signals editorial intent to address this in the next cycle, making now the appropriate time to act.
