## Description

The reporter identifies that the History & Physical (H&P) Note Composition profile in the US C-CDA on FHIR Implementation Guide is missing constraints on `Composition.encounter`. Per C-CDA R2.1, the H&P Note **SHALL** contain exactly one `componentOf` (CONF:1198-8338) with exactly one `encompassingEncounter` (CONF:1198-8339) containing at least one `id` (CONF:1198-8340). The FHIR profile should therefore constrain `Composition.encounter` to a cardinality of `1..1` and type it as a `Reference` to the DAF Encounter profile (`http://hl7.org/fhir/StructureDefinition/daf-encounter`).

The original related page was: http://build.fhir.org/ccda/ccda-history-and-physical-composition.html

A comment from lynn_laakso (2024-03-12) notes the ticket was re-opened post STU1.2 publication.

## Analysis

### Context
This ticket is part of a batch of related issues filed by Corey Spears in mid-2016 against the US C-CDA on FHIR IG (STU1 era). The common theme is that many C-CDA R2.1 SHALL constraints were not translated into the corresponding FHIR StructureDefinition profiles. Specifically for the H&P Note, C-CDA requires a mandatory `componentOf/encompassingEncounter`, but the FHIR Composition profile left `Composition.encounter` at its base cardinality of `0..1` and did not reference the DAF-encounter profile.

### Key Considerations
1. **Age and ecosystem evolution**: The ticket is nearly 9 years old. The DAF profiles referenced in the request have been superseded by **US Core** profiles. Any fix would need to reference current US Core Encounter rather than the deprecated `daf-encounter`.
2. **Near-duplicates**: FHIR-10336 and FHIR-10340 cover the same issue (missing daf-encounter reference on `Composition.encounter`) without the cardinality detail. These should be resolved together.
3. **Specification status**: The US C-CDA on FHIR IG has undergone significant evolution since STU1. The comment from lynn_laakso (2024-03-12) indicates the ticket was re-opened post STU1.2, suggesting it was considered but not resolved during that cycle.
4. **Broader pattern**: This is one of many tickets (FHIR-10195, FHIR-10193, FHIR-10320, FHIR-10324, FHIR-10325, FHIR-10326) reporting the same class of issue across different sections and document types.

## Proposed Dispositions

### Disposition A: Apply as Submitted (with modernization)
**Action:** Constrain `Composition.encounter` to `1..1` in the H&P Note Composition profile and type it as a `Reference` to the current US Core Encounter profile (replacing the deprecated DAF-encounter reference). Coordinate resolution with near-duplicates FHIR-10336 and FHIR-10340.

**Rationale:** The C-CDA R2.1 requirement is clear — the H&P Note SHALL have an `encompassingEncounter`. The FHIR profile should faithfully represent this constraint. The profile reference needs updating to US Core Encounter to reflect the current ecosystem.

**Pros:**
- Maintains fidelity to C-CDA R2.1 conformance rules
- Ensures FHIR-based C-CDA documents enforce the same encounter requirements
- Addresses a long-standing gap

**Cons:**
- Requires review of whether US Core Encounter is the correct replacement target
- May need to assess impact on existing implementations that rely on `0..1` cardinality

### Disposition B: Resolve as duplicate / consolidate
**Action:** Mark FHIR-10337 as a duplicate of FHIR-10336 (or vice versa), consolidating the cardinality and profile-reference concerns into a single tracking ticket. Defer the actual fix to a future C-CDA on FHIR revision.

**Rationale:** Three tickets (FHIR-10336, FHIR-10337, FHIR-10340) cover the same underlying issue. Consolidating avoids redundant work and allows the workgroup to address the concern holistically in a future ballot cycle.

**Pros:**
- Reduces ticket clutter
- Ensures all aspects (cardinality + profile reference) are captured in one place

**Cons:**
- Continues to defer the fix
- Does not address the underlying conformance gap

### Disposition C: Close as out of scope / won't fix
**Action:** Close the ticket, noting that the DAF-encounter profile is deprecated and the US C-CDA on FHIR IG has been substantially reworked since STU1. Direct the reporter to file a new ticket against the current version if the constraint gap persists.

**Rationale:** The original request references deprecated DAF profiles and a build URL from the STU1 era. The IG landscape has changed significantly. If the constraint gap still exists in the current version, a fresh ticket with current context would be more actionable.

**Pros:**
- Clears an aging ticket from the backlog
- Encourages assessment against current IG state

**Cons:**
- Risk of losing track of a valid conformance gap if no new ticket is filed
- The 2024 re-open suggests the issue may still be relevant

---

## Recommendation

**Disposition A: Apply as Submitted (with modernization)**

The underlying C-CDA R2.1 conformance requirement is valid and well-defined. The 2024 re-opening by lynn_laakso confirms continued relevance. The fix should constrain `Composition.encounter` to `1..1` in the H&P Note profile, reference the current US Core Encounter profile, and be coordinated with resolution of the near-duplicate tickets FHIR-10336 and FHIR-10340. The workgroup should also review the batch of related tickets (FHIR-10195, FHIR-10193, FHIR-10320, etc.) to address this class of missing constraint systematically.
