## Description

The reporter identifies that `Composition.encounter` in the C-CDA Consultation Note Composition profile does not reference the `daf-encounter` profile. The request is to add a `<type>` element with a profile reference to `http://hl7.org/fhir/StructureDefinition/daf-encounter` so that the encounter element is constrained to the appropriate profile.

## Important Context

1. **DAF → US Core evolution:** The `daf-encounter` profile referenced in this 2016 ticket no longer exists in its original form. DAF profiles were refactored into US Core profiles. A previous resolution attempt on the related FHIR-10193 (reverted) noted: *"DAF-Encounter was removed from US-Core. Will link to an un-profiled Encounter resource."*

2. **Batch of related issues:** This ticket is one of a cluster filed by Corey Spears in June–July 2016 identifying missing DAF profile references across multiple C-CDA on FHIR Composition profiles. The pattern is consistent: section entries and header elements reference DAF profiles in their `definition` text but lack the corresponding `<type><profile>` constraint in the StructureDefinition.

3. **Re-opened in 2024:** All tickets in this cluster were re-opened by lynn_laakso on 2024-03-12 following STU1.2 publication, suggesting they remain relevant to the current C-CDA on FHIR IG.

## Proposed Dispositions

### Disposition A: Persuasive with Modification
**Action:** Accept the underlying request but update the target profile reference. Constrain `Composition.encounter` in the C-CDA Consultation Note Composition to reference the current US Core Encounter profile (or the base Encounter resource if no US Core Encounter profile is appropriate for the C-CDA on FHIR context) rather than the defunct `daf-encounter` profile. Consolidate this ticket with FHIR-10340 (Transfer Summary) and FHIR-10337 (H&P Note) to apply the fix consistently across all affected Composition profiles.

**Rationale:** The core issue — that `Composition.encounter` lacks a profile-level type constraint — is a valid profiling gap. However, the specific profile URL (`daf-encounter`) is obsolete. The fix should reference whatever Encounter profile is currently appropriate for C-CDA on FHIR (e.g., US Core Encounter), and should be applied uniformly across the related composition profiles.

### Disposition B: Not Persuasive (Outdated)
**Action:** Close as outdated. The C-CDA on FHIR IG has undergone significant revision since 2016, and the DAF ecosystem has been superseded by US Core. The specific profiling approach suggested (referencing `daf-encounter`) is no longer applicable, and the current IG may already handle encounter references appropriately in its latest revision.

**Rationale:** This ticket is over 8 years old and references infrastructure (DAF profiles, DSTU2-era StructureDefinition syntax) that no longer exists. If the issue persists in the current IG, a new ticket with updated context would be more actionable than attempting to resolve this legacy item.

### Disposition C: Duplicate — Consolidate into FHIR-10320
**Action:** Close as duplicate. FHIR-10320 ("Sections with DAF defined entries do not have a proper profile reference") describes the same class of issue more broadly, covering multiple sections and compositions. Resolve the encounter profile reference gap as part of that broader ticket.

**Rationale:** FHIR-10320 captures the systemic pattern of missing `<type><profile>` constraints across the C-CDA on FHIR IG. Addressing encounter references through that broader ticket avoids piecemeal fixes and ensures consistency across all affected elements.

## Recommendation

**Disposition A: Persuasive with Modification** is recommended. The underlying profiling gap is real and was explicitly re-opened for current work in 2024. The fix should update the profile reference to the modern equivalent (US Core Encounter or base Encounter) rather than the obsolete `daf-encounter`, and should be coordinated with the cluster of related tickets (FHIR-10340, FHIR-10337, FHIR-10320) to ensure consistent handling across all C-CDA on FHIR Composition profiles. The workgroup should determine the appropriate target Encounter profile for the current C-CDA on FHIR IG edition.
