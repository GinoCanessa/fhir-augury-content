# FHIR-17156: External links are broken

## Summary

**Ticket:** [FHIR-17156](https://jira.hl7.org/browse/FHIR-17156)
**Status:** Triaged | **Type:** Change Request | **Priority:** Medium
**Specification:** US C-CDA on FHIR (FHIR)
**Work Group:** Structured Documents
**Reporter:** Rick Geimer | **Assignee:** Unassigned
**Created:** 2018-05-14 | **Updated:** 2019-11-21

The reporter identifies that external links to FHIR core within the published US C-CDA on FHIR specification are broken. The specific example given is a malformed URL — `http://hl7.org/hl7.org/fhirindex.html` — which contains a duplicated `hl7.org` path segment and a missing slash before `index.html`. The correct URL should follow the pattern `http://hl7.org/fhir/index.html`. This is a publishing/link generation defect that would affect any reader attempting to navigate from the C-CDA on FHIR IG to FHIR core resources.

## Proposed Dispositions

### Disposition A: Persuasive (Fix the Links)

**Rationale:** The reported defect is clear and verifiable — the URL `http://hl7.org/hl7.org/fhirindex.html` is objectively malformed with a duplicated path segment. Broken links degrade the usability of the published specification and prevent readers from navigating to referenced FHIR core content. This is a straightforward technical correction with no clinical or conformance impact. Multiple analogous tickets (FHIR-40188, FHIR-17236, FHIR-52232) have been resolved as Persuasive.

**Action:** Audit all external links to FHIR core in the US C-CDA on FHIR IG source and correct them to use properly formed URLs (e.g., `http://hl7.org/fhir/index.html`). Consider also addressing FHIR-13634 at the same time since it affects the same specification.

### Disposition B: Persuasive with Modification (Fix with Broader Link Audit)

**Rationale:** While the specific example is clearly broken, the issue may indicate a systemic link generation problem in the C-CDA on FHIR IG build tooling. Rather than fixing only the reported link, the work group should conduct a comprehensive link audit of the entire specification to identify and fix all broken external links in a single pass. This would be more efficient and prevent future individual link-fix tickets.

**Action:** Perform a full link validation pass on the US C-CDA on FHIR IG, fix all broken external links, and address FHIR-13634 as part of the same effort.

### Disposition C: Not Persuasive (Obsolete)

**Rationale:** This ticket was filed in 2018 against a version of the C-CDA on FHIR IG that may since have been republished or superseded. If subsequent IG publications have already regenerated all links (potentially fixing this defect), the issue may no longer be reproducible. The work group may prefer to close this as obsolete and request a new ticket if the problem persists in the current published version.

**Action:** Verify whether the broken link still exists in the current published version. If resolved by subsequent publication, close as Not Persuasive. If still broken, reopen or file a new ticket against the current version.

## Recommendation

**Disposition A: Persuasive** is recommended. The defect is a clear, objectively verifiable URL malformation (`hl7.org/hl7.org/` duplication) with no ambiguity about its correctness. Broken links are a well-established category of technical correction that the community consistently resolves as Persuasive (see FHIR-40188, FHIR-17236, FHIR-52232). Even though the ticket is from 2018, the fact that it remains Triaged and Unresolved suggests it has not been addressed by subsequent publications. Fixing the links is low-risk, high-value, and consistent with prior precedent. The work group should also consider bundling the fix with FHIR-13634, which reports another broken link in the same specification.
