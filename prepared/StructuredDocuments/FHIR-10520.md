## Description

The request asks for inline examples to be added for DAF (Data Access Framework) profiles that are referenced within the US C-CDA on FHIR implementation guide. The Composition resources in C-CDA on FHIR reference various DAF profiles (e.g., for observations, conditions, family member history), but no inline examples are provided to illustrate how these referenced resources should be populated.

## History and Context

- **2016-08-29:** Filed by Sarah Gaunt (minigrrl) requesting inline examples for DAF profiles referenced in C-CDA on FHIR.
- **2016-10-27:** Previously resolved as **"Considered for Future Use"** by vote of Brett Marquard/Sean Muir (3-0-0). The resolution noted: *"Will not add inline examples to Composition resources since they are intended to be references. However, will consider supplying full Document Bundle examples that include the DAF resource instances as well as the Composition examples."*
- **2024-03-12:** Reopened by Lynn Laakso post STU1.2.0 publication, reverting the previous resolution.

**Key evolution since original filing:** The DAF profiles referenced in this ticket have been deprecated and superseded by US Core profiles. Any implementation of this request would need to use current US Core profile references rather than the original DAF references.

## Proposed Dispositions

### Disposition A: Persuasive — Create Document Bundle Examples with Referenced Resources
**Action:** Accept the request with the approach originally contemplated in the 2016 resolution — create full Document Bundle examples that include instances of the referenced profiles (now US Core, not DAF) alongside the Composition resources. This provides concrete, runnable examples showing how a complete C-CDA on FHIR document is assembled.

**Rationale:** The original 2016 vote acknowledged the value of complete Document Bundle examples. Since the ticket was reopened post-STU1.2.0, the work group has the opportunity to fulfill this commitment. Full bundle examples are the most implementer-friendly approach, showing the Composition and all its referenced resources in context.

### Disposition B: Not Persuasive — Inline Examples Not Appropriate for Composition References
**Action:** Close the ticket as not persuasive, affirming that Composition section entries are references by design and inline examples within Composition profiles are not architecturally appropriate.

**Rationale:** The original 2016 resolution correctly identified that Composition resources use references, not inline/contained resources. Adding inline examples to individual Composition profiles would be misleading and could encourage contained resources rather than proper referencing. Implementers should look to US Core profiles directly for examples of referenced resources.

### Disposition C: Considered for Future Use — Defer Pending IG Restructuring
**Action:** Defer this request until the next planned update cycle of C-CDA on FHIR, allowing it to be addressed as part of a broader examples improvement effort.

**Rationale:** While examples are valuable, the DAF-to-US Core transition means significant rework is needed. This should be coordinated with the overall IG update to ensure examples reference current profiles. Deferring avoids creating examples that may immediately become outdated.

## Recommendation

**Disposition A: Persuasive — Create Document Bundle Examples with Referenced Resources**

This is recommended because:
1. The ticket was deliberately reopened in 2024, signaling active work group interest in addressing it.
2. The 2016 resolution already agreed in principle that full Document Bundle examples would be valuable — this fulfills that commitment.
3. All four related tickets (FHIR-11881, -11825, -11874, -11888) about DAF profile references have been resolved as Persuasive, showing a consistent pattern of the work group improving profile references in C-CDA on FHIR.
4. Complete Document Bundle examples showing Composition + referenced US Core resources would significantly aid implementers in understanding the full document assembly pattern, which is one of the more complex aspects of C-CDA on FHIR.

The scope should be updated to reference current US Core profiles rather than the deprecated DAF profiles.
