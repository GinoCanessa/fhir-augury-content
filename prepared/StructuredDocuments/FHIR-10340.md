## Description

The reporter identifies that the `Composition.encounter` element in the C-CDA on FHIR Transfer Summary Composition profile does not include a reference to the DAF Encounter profile (`http://hl7.org/fhir/StructureDefinition/daf-encounter`). The profile's `<type>` element should include a `<profile>` value pointing to `daf-encounter`, ensuring the encounter reference is constrained to the appropriate DAF profile rather than allowing any generic Encounter resource.

## Analysis

### Context

This ticket is part of a batch of issues filed on 2016-07-14 by Corey Spears against the original C-CDA on FHIR STU1 content. All tickets in the batch (FHIR-10320, FHIR-10336, FHIR-10337, FHIR-10340, FHIR-10193–10195) report the same structural deficiency: profile references to DAF resources were described in human-readable text (e.g., `<definition>`) but not formally expressed in the `<type><profile>` element, meaning validators could not enforce them.

The DAF (Data Access Framework) profiles referenced here (`daf-encounter`, `daf-allergyintolerance`, etc.) were part of the DSTU2-era US Core predecessor. Since 2016, the US landscape has shifted significantly:
- **DAF profiles have been superseded by US Core** (STU3+), which defines `us-core-encounter` and other profiles.
- **C-CDA on FHIR** has continued to evolve. The referenced URL (`build.fhir.org/ccda/ccda-transfer-summary-composition.html`) points to an older build.
- Lynn Laakso's 2024-03-12 comment ("re-open post STU1.2 publication") suggests the ticket was reconsidered but remains unresolved.

### Key Considerations

1. **Near-duplicate of FHIR-10336**: Both tickets request the same fix — adding a `daf-encounter` profile reference to `Composition.encounter`. FHIR-10336 was filed minutes earlier with nearly identical text. These should be consolidated.
2. **DAF is obsolete**: The `daf-encounter` profile no longer exists in current specifications. Any fix must reference the current equivalent (e.g., US Core Encounter) or the appropriate C-CDA on FHIR encounter profile.
3. **Age of the ticket**: Filed in 2016 (nearly 9 years old) against STU1 content. The specification has undergone multiple ballot cycles since then.
4. **Systemic pattern**: This is one of ~10 tickets reporting missing profile references across C-CDA on FHIR sections. A coordinated resolution of the entire batch would be more efficient than individual dispositions.
