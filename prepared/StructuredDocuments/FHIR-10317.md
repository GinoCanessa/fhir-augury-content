## Description

The reporter identifies that all code system references to LOINC throughout the US C-CDA on FHIR implementation guide use OID URNs (e.g., `urn:oid:2.16.840.1.113883.6.1`) instead of the preferred FHIR-canonical URI (`http://loinc.org`). The request is to update all LOINC code system URIs across the entire guide to be consistent with FHIR conventions.

## Zulip Discussions

### 1. [terminology > Use of OID identifier when a system url is defined](https://chat.fhir.org/#narrow/stream/terminology/topic/Use%20of%20OID%20identifier%20when%20a%20system%20url%20is%20defined) (Jan 2025)
Discussion about whether terminology servers should accept OID URNs when a non-OID canonical URL exists for a CodeSystem. **Grahame Grieve** stated that OIDs should not be accepted in place of the canonical URL (except in CDA context). **Lloyd McKenzie** confirmed: "If you use OIDs when there's a non-OID URI defined, you're non-conformant." This strongly supports the need to replace OIDs with canonical URLs in FHIR IGs.

### 2. [implementers > converting source codeSystem OIDs to FHIR CodeSystem.url](https://chat.fhir.org/#narrow/stream/implementers/topic/converting%20source%20codeSystem%20OIDs%20to%20FHIR%20CodeSystem.url) (Feb 2018)
John Moehrke raised the general question of converting OIDs from source material (CDA, XDS, v2) to FHIR URIs. **Lloyd McKenzie** gave clear guidance: "take all reasonable steps to avoid OIDs appearing in FHIR instances" and "convert unless it's simply not possible." **Grahame Grieve** pointed to the NamingSystem `$preferred-id` operation as a mapping tool. This directly supports the requested change.

## Analysis

This is a longstanding issue (filed 2016) requesting that the US C-CDA on FHIR IG replace all LOINC code system references from OID URNs (`urn:oid:2.16.840.1.113883.6.1`) to the FHIR-canonical URL (`http://loinc.org`). The core issue is straightforward: FHIR specifies canonical URIs for well-known code systems, and LOINC's canonical URI is `http://loinc.org`, not its OID.

**Key considerations:**
- FHIR-10318 and FHIR-10319 are exact duplicates already resolved as such.
- FHIR-11886 is essentially the same request, filed by the same reporter, and was resolved **Persuasive** and **Published**. This strongly suggests the workgroup already agreed this change should be made.
- Comment from lynn_laakso (2024-03-12) indicates the ticket was re-opened "post STU1.2 publication," implying the fix may not have been fully applied in that publication cycle.
- Community consensus from Zulip (Grahame Grieve, Lloyd McKenzie) is unambiguous: OIDs should not be used when a canonical URL exists.

## Proposed Dispositions

### Disposition A: Persuasive
**Rationale:** Accept the change request. Replace all instances of `"system":"urn:oid:2.16.840.1.113883.6.1"` with `"system":"http://loinc.org"` throughout the US C-CDA on FHIR IG. This aligns with FHIR's canonical code system URI conventions, is consistent with the resolution of the closely related FHIR-11886 (already resolved Persuasive), and reflects clear community consensus that OIDs should not appear where canonical URLs are defined. The guide-wide scope is appropriate since the reporter noted the issue persists "across the entire guide."

### Disposition B: Duplicate
**Rationale:** Resolve as a duplicate of FHIR-11886, which was already resolved Persuasive and Published. Both tickets were filed by the same reporter (Corey Spears) requesting the same change—replacing LOINC OID URNs with `http://loinc.org` in the C-CDA on FHIR IG. If FHIR-11886's resolution was meant to cover the full guide, this ticket is redundant.

### Disposition C: Persuasive with Modification
**Rationale:** Accept the change but broaden scope: in addition to LOINC, audit and fix all code system references across the C-CDA on FHIR IG that use OID URNs where a FHIR-canonical URL exists (e.g., SNOMED CT should use `http://snomed.info/sct`, not its OID). This addresses the root cause comprehensively rather than fixing only LOINC while leaving other OID-based references in place.

## Recommendation

**Disposition A: Persuasive**

The change is well-justified, non-controversial, and consistent with both FHIR convention and established community consensus. The closely related FHIR-11886 was already resolved Persuasive, confirming the workgroup's agreement. The 2024 re-opening indicates residual OID references remain in the guide and need to be addressed. Resolving as Duplicate (B) risks the remaining instances being overlooked if FHIR-11886 is considered closed. Disposition C (broadening scope) is worthwhile but may be better tracked as a separate effort to keep this ticket focused and actionable.
