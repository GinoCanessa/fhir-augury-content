## Description

The reporter identifies that the ValueSet URL `http://hl7.org/fhir/ValueSet/2.16.840.1.113883.1.11.20.22` referenced by `Composition.type` in the C-CDA on FHIR Implementation Guide does not resolve to a valid ValueSet. The OID `2.16.840.1.113883.1.11.20.22` corresponds to a CDA-era value set identifier (HPDocumentType) that was used as a FHIR ValueSet canonical URL, but the ValueSet was never created or published at that URL.

The original related page was the [C-CDA History and Physical Composition](http://build.fhir.org/ccda/ccda-history-and-physical-composition.html) profile page.

Per comments from lynn_laakso (2024-03-12), the ticket was re-opened post STU1.2 publication, indicating the issue persists.

## Analysis

This ticket is part of a cluster of four closely related tickets (FHIR-10321, FHIR-10322, FHIR-10323, FHIR-10338) all filed by Corey Spears on 2016-07-14, all reporting that `Composition.type` in various C-CDA on FHIR composition profiles referenced OID-based ValueSet URLs that do not resolve. These OIDs are CDA-era value set identifiers that were used verbatim as FHIR ValueSet canonical URLs without the corresponding ValueSet resources being published on the FHIR server.

Since 2016, significant changes have occurred:
- **FHIR Core** (via FHIR-28078, applied): `Composition.type` now uses a preferred binding to LOINC document-type codes rather than OID-based value sets.
- **C-CDA on FHIR IG** has gone through multiple STU updates. The re-opening post STU1.2 (2024-03-12) suggests the issue was not fully addressed in that publication cycle.

The key question for the workgroup is whether the C-CDA on FHIR IG's History and Physical composition profile still references this non-existent ValueSet, or whether the binding has been updated in more recent versions. Given the cluster of similar tickets and the evolution of FHIR Core terminology bindings, a coordinated resolution across FHIR-10321, FHIR-10322, FHIR-10323, and FHIR-10338 would be most efficient.

## Proposed Dispositions

### Disposition A: Persuasive — Update binding to use FHIR-native ValueSet
**Action:** Replace the broken OID-based ValueSet URL (`http://hl7.org/fhir/ValueSet/2.16.840.1.113883.1.11.20.22`) with the current FHIR-native document-type codes ValueSet (e.g., `http://hl7.org/fhir/ValueSet/doc-typecodes`) in the C-CDA on FHIR History and Physical composition profile. Align with the approach taken in FHIR-28078 for Composition.type bindings.

**Rationale:** The OID-based URL was carried over from CDA and never published as a FHIR ValueSet. FHIR Core has since resolved this at the base resource level (FHIR-28078). The C-CDA on FHIR IG profiles should follow suit and reference valid, published ValueSets.

### Disposition B: Persuasive with Modification — Create the missing ValueSet
**Action:** Create and publish the ValueSet at `http://hl7.org/fhir/ValueSet/2.16.840.1.113883.1.11.20.22` with the HPDocumentType concepts, making the existing reference valid. Coordinate with the related tickets (FHIR-10321, FHIR-10323, FHIR-10338) to create all missing OID-based ValueSets.

**Rationale:** This preserves backward compatibility with existing implementations that may already reference this URL, and maintains alignment with the CDA value set definitions. However, this approach is less aligned with the direction FHIR Core has taken.

### Disposition C: Not Persuasive — Defer as potentially already resolved
**Action:** Close the ticket as no longer applicable if the C-CDA on FHIR IG's current published version has already resolved the binding (e.g., through STU updates post-STU1.2). Verify current state of the IG before closing.

**Rationale:** Significant terminology binding work has occurred since 2016, and FHIR-28078 addressed the core resource binding. If the C-CDA on FHIR IG profiles have been updated accordingly, this ticket may already be moot.

## Recommendation

**Disposition A: Persuasive — Update binding to use FHIR-native ValueSet.** This aligns with the direction FHIR Core has taken (FHIR-28078, applied), avoids creating new OID-based ValueSets that conflict with modern FHIR terminology practices, and should be coordinated with the sibling tickets (FHIR-10321, FHIR-10323, FHIR-10338) for a consistent resolution across all affected C-CDA on FHIR composition profiles. The workgroup should verify the current state of the IG and resolve all four tickets together.
