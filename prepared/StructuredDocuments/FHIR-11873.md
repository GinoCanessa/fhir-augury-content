## Description

The ticket requests that hyperlinks be added to the text that references other C-CDA on FHIR document profiles. The existing wording in the specification states:

> "More specific use cases, such as a Discharge Summary, Transfer Summary, Referral Note, Consultation Note, or Progress Note, are available as alternative profiles"

These document type names are listed as plain text with no hyperlinks to their respective profile pages. The submitter requests that clickable links be added so readers can navigate directly to the referenced profiles.

## Proposed Dispositions

### Option A: Persuasive (Recommended)

**Action:** Accept the request and add hyperlinks from the listed document type names to their corresponding C-CDA on FHIR profile pages.

**Rationale:** This is a straightforward editorial/navigation improvement. The original deferral (2016) was explicitly because IG URLs had not yet stabilized. Now that STU1.2.0 has been published and URLs are stable, the stated blocker is resolved. Several analogous link-fix tickets from the same ballot cycle (FHIR-11869, FHIR-11893, FHIR-11913) were resolved as Persuasive and have been published. Adding hyperlinks to the Discharge Summary, Transfer Summary, Referral Note, Consultation Note, and Progress Note profile pages improves IG usability at minimal cost, with no semantic or conformance impact.

### Option B: Persuasive with Modification

**Action:** Accept the request but expand scope to audit and add hyperlinks to all document profile references across the specification, not just the single paragraph cited in the ticket.

**Rationale:** If other locations in the IG also reference document types as plain text without links, it would be more consistent to address them all at once rather than fixing a single paragraph. This option extends the improvement but requires a broader review pass.

### Option C: Not Persuasive

**Action:** Reject the request on the grounds that the IG publisher tooling or modern rendering already auto-generates links to profiles where appropriate.

**Rationale:** If the current IG publisher tooling already resolves profile references into hyperlinks in the rendered output, the manual addition of links may be unnecessary or could create maintenance burden if URLs change again. However, this seems unlikely given the ticket was explicitly re-opened in 2024 to address this gap.

## Recommendation

**Option A: Persuasive** — The request is a clear, low-risk editorial improvement. The original reason for deferral (unstable URLs) no longer applies. Peer tickets requesting similar link fixes were all resolved as Persuasive and published. Adding hyperlinks to the five named document profiles improves navigation with no conformance impact.
