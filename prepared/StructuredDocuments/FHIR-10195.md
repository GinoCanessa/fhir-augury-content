## Summary

The Immunization section of the C-CDA on FHIR Implementation Guide does not include a `Composition.section.entry` reference to the DAF-Immunization profile (`http://hl7.org/fhir/StructureDefinition/daf-immunization`). The submitter expected the section to constrain entries to reference the DAF Immunization profile with a `<type>` element containing a `<code value="Reference"/>` and `<profile>` pointing to `daf-immunization`, but this constraint is missing from the IG's StructureDefinition.

The ticket title contains a typo ("Immunizationy") — the actual issue concerns the Immunization section.

## Context & Analysis

This ticket is part of a family of issues (FHIR-10193, FHIR-10194, FHIR-10195, FHIR-10196, FHIR-10320) all filed by Corey Spears in June–July 2016, reporting that various C-CDA on FHIR Composition sections lack proper `<type>` constraints on `Composition.section.entry` to reference the corresponding DAF profiles. The underlying concern is that without these type constraints, conformance tooling cannot validate that section entries actually reference the correct DAF-profiled resources.

Key considerations:
1. **DAF has been superseded by US Core.** The DAF profiles referenced in this ticket (`daf-immunization`) are no longer maintained. The modern equivalent would be US Core Immunization profiles.
2. **C-CDA on FHIR has evolved significantly** since 2016. The current C-CDA on FHIR IG (STU 2.x+) likely uses US Core profiles rather than DAF, so the specific fix requested may already be addressed or may need to be reframed in terms of current profiles.
3. **FHIR-10320 is a superset** that covers the same issue across multiple sections including Immunizations, which may make this individual ticket redundant.
4. The ticket was re-opened by lynn_laakso in March 2024 "post STU1.2 publication," suggesting it is still considered relevant.

---

## Proposed Dispositions

### Disposition A: Persuasive — Update entry constraints to reference current profiles
**Action:** Accept the request in principle, but update the Immunization section's `Composition.section.entry` to reference the current US Core Immunization profile (rather than the obsolete DAF-Immunization profile). Add a `<type>` element with `<code value="Reference"/>` and the appropriate US Core Immunization profile URL.

**Rationale:** The underlying issue is valid — section entries should be constrained to reference the appropriate profiled resources for validation purposes. The specific DAF profile is outdated, but the fix should use the current US Core equivalent.

### Disposition B: Persuasive — Resolve as duplicate of FHIR-10320
**Action:** Mark this ticket as a duplicate of FHIR-10320 ("Sections with DAF defined entries do not have a proper profile reference"), which covers the same issue across all affected sections including Immunizations. Resolve FHIR-10320 with a comprehensive fix that adds proper profile references to all affected section entries using current US Core profiles.

**Rationale:** FHIR-10320 explicitly lists the Immunizations Section among the 8 sections with this issue. Consolidating the fix in one ticket avoids redundant work and ensures consistency across all sections.

### Disposition C: Not Persuasive — Already addressed in current IG
**Action:** Close as not persuasive if the current version of the C-CDA on FHIR IG already constrains the Immunization section entries to reference the appropriate profile (US Core Immunization or equivalent). Verify the current IG before closing.

**Rationale:** Given the significant evolution of the C-CDA on FHIR IG since 2016, the issue may have been resolved in subsequent versions without explicitly closing this ticket.

---

## Recommendation

**Disposition B: Persuasive — Resolve as duplicate of FHIR-10320.**

FHIR-10320 is a comprehensive ticket that covers the exact same issue across all affected sections, including Immunizations. Consolidating the resolution there ensures a consistent fix across all sections and avoids piecemeal changes. The workgroup should verify whether the current C-CDA on FHIR IG already addresses this (in which case both tickets can be closed as applied), and if not, apply the fix using current US Core profile references rather than the obsolete DAF profiles.
