## Description

The "History of Past Illness" section in the C-CDA on FHIR Composition profiles is missing an entry definition that should reference the DAF Condition profile (`daf-condition`). In C-CDA, the History of Past Illness section uses the same Problem Observation as the Problem Section entries. The reporter requests that the `Composition.section.entry` element be constrained to reference the U.S. DAF Condition Profile (`http://hl7.org/fhir/StructureDefinition/daf-condition`) with cardinality `[0..*]`.

The proposed fix adds a `<type>` element with `<code value="Reference" />` and `<profile value="http://hl7.org/fhir/StructureDefinition/daf-condition" />` to the entry definition.

## Analysis

### Context

This ticket dates from July 2016 and targets the original US C-CDA on FHIR IG. The core issue is that the "History of Past Illness" section's `Composition.section.entry` element lacked a type constraint referencing the DAF Condition profile, unlike the analogous Problem Section which correctly constrained entries to the DAF Condition profile.

### Key Considerations

1. **Closely related ticket was resolved:** FHIR-11888 (same topic, same reporter) was resolved as **Persuasive** and marked **Published**, indicating the workgroup accepted the change and it was applied. This raises the question of whether FHIR-10324 is effectively a duplicate.

2. **Evidence of fix applied:** The 2017-11-05 IG build message confirms that Rick Geimer updated the "History of Past Illness section in multiple document types" in the HL7/ccda-on-fhir repository.

3. **Re-opened post STU1.2:** Lynn Laakso's March 2024 comment explicitly states "re-open post STU1.2 publication," suggesting the workgroup believes this issue was not fully resolved or has resurfaced in the STU 1.2 publication.

4. **DAF → US Core evolution:** The original DAF profiles (`daf-condition`) have been superseded by US Core profiles. Any fix applied today would need to reference the current US Core Condition profile rather than the obsolete DAF Condition profile.

5. **Part of a broader pattern:** FHIR-10320 identifies the same class of issue across 8+ sections. Addressing FHIR-10324 in isolation may not be sufficient; a systematic review of all section entry profile references may be warranted.

## Proposed Dispositions

### Disposition A: Persuasive (Resolve with updated profile references)

Accept the change request. Update the "History of Past Illness" section's `Composition.section.entry` to reference the current US Core Condition profile (replacing the obsolete DAF Condition profile reference). Verify the fix across all C-CDA on FHIR document types that include a History of Past Illness section. Coordinate with FHIR-10320 to ensure consistency across all sections.

**Rationale:** The original issue is valid — the section entry should be constrained to reference the appropriate Condition profile. FHIR-11888 was resolved as Persuasive for the same concern, establishing precedent. The re-opening post STU1.2 indicates the fix needs to be verified or reapplied with updated profile references.

### Disposition B: Duplicate (of FHIR-11888)

Close as duplicate of FHIR-11888, which was resolved as Persuasive and Published. The fix was already applied as evidenced by the 2017 IG build. If the issue has resurfaced in STU1.2, a new ticket should be filed with updated context referencing the current US Core profiles rather than the obsolete DAF profiles.

**Rationale:** FHIR-11888 covers the same issue with the same scope and was already resolved. Re-processing this 2016-era ticket with obsolete DAF profile references is less efficient than creating a fresh ticket if the problem persists in the current IG.

### Disposition C: Not Persuasive (Out of scope / obsolete)

Close as not persuasive. The DAF profiles referenced in this ticket no longer exist — they have been replaced by US Core profiles. The C-CDA on FHIR IG has undergone significant restructuring since 2016, and the specific profile URL cited (`daf-condition`) is no longer valid. Any remaining section entry issues should be filed as new tickets against the current version of the IG.

**Rationale:** The ticket references an obsolete profile and an older version of the IG. The workgroup should address current-state issues with current-state tickets rather than resolving a 2016 ticket whose technical details no longer apply.

## Recommendation

**Disposition A: Persuasive** — The underlying issue (History of Past Illness section entries should reference a Condition profile) remains valid regardless of the DAF → US Core migration. FHIR-11888's resolution as Persuasive establishes that the workgroup agreed with this change. Lynn Laakso's 2024 re-opening indicates the fix is still needed in the current publication. The workgroup should accept this as persuasive, update the profile reference to the current US Core Condition profile, and coordinate with FHIR-10320 for a consistent fix across all sections.
