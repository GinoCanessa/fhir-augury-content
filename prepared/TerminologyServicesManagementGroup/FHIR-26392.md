## Description

The reporter identifies that OID `2.16.840.1.113883.1.11.16621`, assigned to the Act Substance Admin Substitution value set, does not appear to be a valid OID. Looking it up in the HL7 OID Registry returns a 500 error. The reporter asks whether the correct OID might instead be `2.16.840.1.113883.11.19717`, and requests that the OID information on this value set be corrected.

## Proposed Dispositions

### Disposition A: Not Persuasive
**Rationale:** The OID `2.16.840.1.113883.1.11.16621` listed in THO is the correct OID inherited from the V3 harmonization process. The 500 error in the HL7 OID Registry is an OID Registry infrastructure issue, not a FHIR specification error. The FHIR specification itself correctly reflects the OID assigned through the harmonization process. Carmela confirmed Vocab would register the OID, which is an administrative action outside the scope of the FHIR specification. All sibling tickets (FHIR-26418, FHIR-26412, FHIR-26417, FHIR-26394) from the same reporter were resolved as Not Persuasive or Retracted for similar reasons.

### Disposition B: Persuasive with Modification
**Rationale:** While the OID value in THO may be correct, the workgroup could verify whether the OID has since been registered in the HL7 OID Registry as Carmela committed to in 2021. If registration has occurred, resolve as completed. If not, re-escalate to the Vocab team to complete the registration. The modification would be to confirm OID correctness and ensure registry alignment, without changing the FHIR specification itself. This acknowledges the reporter's valid concern about verifiability.

### Disposition C: Not Persuasive with Modification
**Rationale:** The request to change the OID in the FHIR specification is not persuasive — the OID in THO is the authoritative source. However, the workgroup could add a note acknowledging that OID registration in the HL7 OID Registry is a known gap and direct future OID registration requests to the Vocabulary team rather than filing FHIR specification change requests. This helps avoid duplicate tickets of this nature.

## Recommendation

**Disposition A: Not Persuasive**

This ticket is part of a batch of OID registration requests from the same reporter, all of which have been resolved without changes to the FHIR specification. The core issue — that the OID was not registered in the HL7 OID Registry — is an OID Registry administration concern, not a FHIR specification defect. The OID `2.16.840.1.113883.1.11.16621` listed in THO is the correct identifier from the V3 harmonization process, as confirmed by carmela_couderc. The Vocab team committed to registering the OID, which is the appropriate remediation path. The precedent set by sibling tickets (FHIR-26394 resolved Not Persuasive, FHIR-26418 and FHIR-26417 Retracted) strongly supports this disposition.
