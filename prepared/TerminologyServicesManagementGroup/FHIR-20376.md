## Details

Alexander Henket reported that 24 V3 ValueSets have OIDs assigned in V3 but those OIDs are missing from their FHIR representations. The affected value sets span a wide range of clinical and administrative domains including Confidentiality, MaritalStatus, Race, Ethnicity, RouteOfAdministration, ReligiousAffiliation, and others.

**Key Comment History:**

- **carmela_couderc (2021-02-11):** Suggested this is a UTG ticket and asked Jessica Bota (jsnell) to confirm.
- **jsnell (2021-02-11 & 2021-02-19):** Confirmed the OIDs are correct on terminology.hl7.org (THO). Stated the FHIR spec pages are out of sync with UTG and that the sync issue would be addressed in a separate ticket.
- **tedhl7geek (2021-12-20):** Clarified that FHIR objects migrated to THO that are copies of V3 objects should have been merged with THO content from the V3 coremif. Any that were not merged is an error requiring manual fix. Also noted that V3 objects copied into FHIR R4 did not have their OIDs carried over, and there is no decision to retroactively add them to R4.

The core issue appears to be a **sync gap**: the OIDs exist in THO but were not propagated back to the FHIR spec pages when V3 content was migrated.

## Proposed Dispositions

### Disposition A: Accept (Persuasive with Modification)

**Action:** Verify each of the 24 listed ValueSets in the current THO publication to confirm their OIDs are present. For any ValueSets where OIDs are still missing in THO, add the V3-sourced OIDs. Close this ticket with a note that the FHIR spec pages will reflect the OIDs once the THO sync is complete.

**Rationale:** The reporter identified a real gap. While comments from 2021 indicate the OIDs are correct in THO, the issue may not be fully resolved for all 24 value sets, and formal verification would close the loop. Accepting ensures these OIDs are tracked and confirmed present in THO artifacts.

### Disposition B: Accept as Duplicate / Subsume into FHIR-31621

**Action:** Close this ticket as a duplicate of or subsumed by [FHIR-31621](https://jira.hl7.org/browse/FHIR-31621) (Inconsistent usage of OIDs/URIs overview), which is a broader ticket by the same reporter covering OID/URI inconsistencies across the specification. Note that the specific V3 ValueSet OIDs listed here should be tracked as part of FHIR-31621's resolution.

**Rationale:** FHIR-31621 is a comprehensive OID/URI inconsistency tracker for the same work group that encompasses this ticket's scope. Consolidating avoids duplicate tracking. Comments on this ticket already indicate the OIDs are correct in THO, suggesting the remaining issue is the broader sync problem addressed by FHIR-31621.

### Disposition C: Decline (Not Persuasive)

**Action:** Close with no change. Note that the OIDs are confirmed present in THO (as verified by jsnell in Feb 2021), and that V3 objects copied into FHIR R4 intentionally did not carry OIDs. The FHIR spec pages are generated from THO and will reflect the OIDs once the publication pipeline is updated.

**Rationale:** The underlying data issue (missing OIDs) has already been resolved in THO. The FHIR spec page rendering is a separate publication concern, not a specification content issue. No specification change is needed. Similar tickets (FHIR-26412, FHIR-3620) were resolved as Not Persuasive for related reasons.

## Recommendation

**Disposition B: Accept as Duplicate / Subsume into FHIR-31621**

This is recommended because:

1. **The OIDs are already correct in THO** — as confirmed by jsnell in February 2021, the underlying data issue is resolved for the sampled value sets.
2. **FHIR-31621 is a broader, active tracker** — filed by the same reporter and assigned to the same work group (TSMG), it comprehensively catalogs OID/URI inconsistencies and would naturally encompass the 24 value sets listed here.
3. **Avoids duplicate tracking** — maintaining two open tickets for overlapping OID concerns in the same work group creates unnecessary administrative overhead.
4. **Preserves the specific list** — by noting these 24 value sets in the closure comment and linking to FHIR-31621, the detailed OID mapping is not lost but is consolidated into a single actionable ticket.
