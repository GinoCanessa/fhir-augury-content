## Summary

The reporter (Alexander Henket) identifies a significant gap in the alignment between FHIR's NamingSystem bundles, the terminology-systems HTML pages, and the HL7 OID Registry. Specifically:

1. The relationship between NamingSystem resources and the terminology pages is **unclear and inconsistent** — items present on the HTML pages may be missing from the NamingSystem bundle (and vice versa).
2. The **V3/V2 OID ↔ FHIR URI mappings** are incomplete — OID columns on terminology pages are empty and corresponding data is absent from NamingSystem resources.
3. It is "virtually impossible" to connect OIDs to URIs using only what is published in the OID Registry.

The request has four concrete asks:
- Make the NamingSystem bundle the **single source of truth** for every code/identifier system in FHIR.
- Ensure every NamingSystem references the HL7 OID Registry as the **authoritative source for OIDs**.
- **Generate** the terminology HTML pages from NamingSystems so they cannot diverge.
- Populate **V2/V3 value set pages** with relevant OIDs (instead of "??") using NamingSystem data.

## Details

This ticket has been open since February 2019 and has been discussed across Vocab and TSMG calls. Key comment history:

- **2020-10-08**: Carmela Couderc asked whether this belongs in the UTG project. Jessica Bota (after consulting Ted) said it should be discussed on the FHIR tracker call — some items may be addressed within UTG but others are FHIR-specific.
- **2021-08-10**: Carmela disagreed with Ted's assessment and noted Vocab can't do much without quality control in UTG and updated FHIR spec references.
- **2021-10-20**: Carmela asked Reuben Daniels for guidance — specifically what can be done beyond referencing THO work and related tickets.
- **2021-10-27**: Discussed on the 2021-10-26 Vocab FHIR Tracker call; the topic was assigned to TSMG for resolution.

The ticket references **FHIR-20329** (Missing NamingSystems for NUCC Taxonomy / Health Canada Drug codes), which was resolved as "Not Persuasive" under Terminology Infrastructure. That ticket highlighted specific NamingSystems missing from the bundle despite being listed on the terminology-systems page.

Since this ticket was last updated in January 2022, significant progress has occurred via **FHIR-37853** (Published, Persuasive), which updated the Terminologies — Code Systems page to establish **HL7 Terminology (THO)** as the source of truth for external code and identifier system information. That ticket directed implementers to THO's External Code Systems page and removed rows from the FHIR spec's "Defined Externally" tab that lacked "Using X With FHIR" guides.

## Proposed Dispositions

### Disposition A: Accept — Partially Satisfied, Resolve Remainder via THO Alignment

**Resolution:** Persuasive with Modification

**Rationale:** The core concern — that there is no single source of truth for terminology system identifiers — has been **substantially addressed** by FHIR-37853, which established THO as the authoritative source for external terminology information. However, the specific asks about NamingSystem bundle completeness and generating terminology pages from NamingSystems remain unaddressed.

**Action:**
1. Acknowledge that THO now serves as the source of truth for external terminology system URIs and OIDs (per FHIR-37853).
2. For FHIR-defined code systems, ensure the NamingSystem bundle in the FHIR specification is complete and includes all OIDs from the HL7 OID Registry.
3. Evaluate whether terminology HTML pages can be generated from NamingSystem data to prevent divergence for FHIR-defined systems.
4. Update the ticket to reference FHIR-37853 as having resolved the external terminology source-of-truth concern.

---

### Disposition B: Accept — Defer to THO as Full Resolution

**Resolution:** Persuasive — Referred to THO/TSMG

**Rationale:** Given that THO is now the authoritative source of truth for external terminologies (per HTA, TSC, and TSMG decisions codified in FHIR-37853), and that the NamingSystem bundle in the FHIR spec is effectively superseded by THO for external systems, the remaining gap is narrow. The FHIR spec's NamingSystem bundle and terminology pages for FHIR-defined systems should be maintained, but external system alignment is a THO concern.

**Action:**
1. Mark this ticket as resolved, noting that FHIR-37853 addressed the primary source-of-truth concern.
2. For any remaining NamingSystem/OID gaps specific to FHIR-defined terminology, create a new targeted ticket under TSMG or Terminology Infrastructure.
3. Close with a note directing implementers to THO for authoritative OID↔URI mappings.

---

### Disposition C: Decline — Superseded by THO and FHIR-37853

**Resolution:** Not Persuasive (Superseded)

**Rationale:** The landscape has changed significantly since 2019. FHIR-37853 (Published, Persuasive) has redirected implementers to THO as the authoritative source. The specific request to make the FHIR NamingSystem bundle the "source of truth" for all systems conflicts with the current HL7 policy that THO fulfills this role. The remaining asks (generating terminology pages from NamingSystems, populating V2/V3 OIDs) are either resolved by THO or represent implementation details that are no longer strategically aligned.

**Action:**
1. Close as superseded by FHIR-37853.
2. Note that THO's External Code Systems page provides the authoritative OID↔URI mappings.
3. Any specific remaining gaps should be filed as new, focused tickets.

---

## Recommendation

**Disposition B: Accept — Defer to THO as Full Resolution** is recommended.

This ticket's core concern — the lack of a single, authoritative source for terminology system identifiers and OID mappings — has been substantively addressed by FHIR-37853 and the broader THO initiative. The FHIR spec's terminology pages now defer to THO for external terminologies. Rather than keeping this broad 2019 ticket open, it is cleaner to acknowledge the THO resolution, close this ticket with appropriate references, and create any remaining focused tickets for FHIR-defined system gaps. This avoids the complexity of partially implementing a 2019 vision that has been superseded by organizational decisions.
