## Description

The submitter identifies that Section 3.4 of the PHD IG uses overly prescriptive language regarding PHD current-time reporting. The current text states:

> "This IG requires that PHDs that generate measurements that have timestamps also be able to report their sense of current time."

The submitter argues this should not be a hard requirement. The proposed rewording is:

> "This IG recommends that PHDs that generate measurements that have timestamps should be able to report their sense of current time to allow creating accurate timestamps."

The change shifts the conformance level from **requires** (mandatory) to **recommends** (advisory), which would lower the barrier for PHD devices that provide timestamps but lack the ability to report their current time to the PHG.

## Analysis

This ticket addresses a conformance-level question in Section 3.4 of the PHD IG. The core issue is whether PHDs that provide timestamped measurements **must** (SHALL) or **should** (SHOULD) be able to report their sense of current time to the PHG.

**Key considerations:**

1. **Implementability**: Not all PHDs that provide timestamps can report their current time. Making this a hard requirement excludes such devices from conformance, which conflicts with the submitter's (and arguably the IG's) goal of broad adoption.

2. **Timestamp accuracy**: The PHG uses the PHD's reported current time to correct for clock drift in measurement timestamps. Without this, timestamps may be inaccurate. Relaxing to SHOULD means some timestamps may go uncorrected.

3. **Pattern of feedback**: The submitter (MalcolmClarke) filed a cluster of related tickets (FHIR-53327, FHIR-53328, FHIR-53329, FHIR-53330, FHIR-53335) all addressing overly prescriptive or confusing language in the PHD IG. FHIR-53330 is closely related and addresses overlapping text in the same section.

4. **Prior history**: This was previously resolved as "Considered for Future Use" and deferred to the next PHD IG revision per the 2025-11-12 Devices WG call. It was then re-opened post-STU2 publication, indicating the WG wants to address it in the upcoming cycle.

5. **Precedent**: FHIR-23748 and FHIR-15474 show that the WG has previously accepted editorial and accuracy improvements to this same area of the PHD IG.

## Proposed Dispositions

### Disposition A: Persuasive (Accept as proposed)

Change the wording in Section 3.4 from "requires" to "recommends" as the submitter proposes:

> "This IG recommends that PHDs that generate measurements that have timestamps should be able to report their sense of current time to allow creating accurate timestamps."

**Rationale**: This broadens IG applicability to PHDs that cannot report current time while still encouraging the practice. Aligns with the submitter's intent and the WG's re-opening of this ticket for the next revision.

**Risk**: Some implementers may interpret the relaxed language as an invitation to skip current-time reporting entirely, potentially degrading timestamp accuracy across the ecosystem.

### Disposition B: Persuasive with Modification

Accept the intent but use formal FHIR conformance language. Change Section 3.4 to:

> "PHDs that generate measurements with timestamps **SHOULD** be able to report their sense of current time. Reporting current time allows the PHG to create accurate timestamps by correcting for clock drift."

Add an explanatory note that devices unable to report current time may still conform to the IG but that timestamp accuracy may be reduced.

**Rationale**: Uses standard RFC 2119 / FHIR conformance vocabulary (SHOULD) instead of informal "recommends." Preserves the clear recommendation while providing implementers with context on the tradeoff. Coordinates well with FHIR-53330 which also requests clarification in the same section.

**Risk**: Minimal. The SHOULD keyword is well-understood in the FHIR ecosystem and provides the correct conformance signal.

### Disposition C: Not Persuasive

Retain the current requirement language. The ability for the PHG to obtain the PHD's sense of current time is essential for accurate timestamp correction and is a core design principle of the PHD IG.

**Rationale**: PHD time clocks are often unreliable. Without the ability to compare PHD and PHG times, measurements may carry significantly inaccurate timestamps, undermining clinical data quality. The requirement exists for a well-founded technical reason.

**Risk**: Continues to exclude PHDs that provide timestamps but cannot report current time, limiting IG adoption.

## Recommendation

**Disposition B: Persuasive with Modification**

The submitter's concern is valid — the current mandatory language is overly restrictive for devices that provide timestamps but lack current-time reporting capability. However, the proposed replacement text uses informal language ("recommends...should") rather than standard FHIR conformance keywords. Using **SHOULD** (RFC 2119) provides a clear, well-understood conformance signal that preserves the recommendation while correctly lowering the requirement level. Adding an explanatory note about the timestamp accuracy tradeoff gives implementers the information needed to make informed decisions. This disposition should be coordinated with FHIR-53330, which addresses overlapping language in the same section.
