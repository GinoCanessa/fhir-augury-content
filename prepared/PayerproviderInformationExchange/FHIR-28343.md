## Summary

The submitter requests that the Dental Referral Note and Dental Consult Note Composition profiles be harmonized so that shared sections (plan_of_treatment, assessment, procedures, payer information) are organized consistently—using the same ordering and hierarchy—across both profiles.

The proposed resolution (Persuasive) explains that both profiles derive from their respective CCDAonFHIR base profiles. Dependent profiles must preserve the top-down ordering of snapshot elements from the base StructureDefinition; additional element constraints in the differential are not order-dependent. Sections will be found in the same order across Composition profiles to the extent allowed by the FHIR IG publisher.

## Details

**Problem:** The Dental Referral Note and Dental Consult Note profiles both include sections for plan_of_treatment, assessment, procedures, and payer information, but organize them differently. This inconsistency creates confusion for implementers working with both documents.

**Resolution Approach:** The resolution acknowledges the concern and notes that:
1. Both Dental Referral Note and Dental Consult Note profiles are based on their respective CCDAonFHIR profiles (Referral Note and Consultation Note).
2. Dependent profiles must follow the same top-down ordering of snapshot elements as the base StructureDefinition.
3. Additional element constraints defined in the differential are not order-dependent.
4. Sections will appear in the same order across Composition profiles to the extent the FHIR IG publisher allows.

This means harmonization is constrained by the base CCDAonFHIR profile ordering and the IG publisher tooling, but within those constraints, section ordering will be made consistent.

## Proposed Dispositions

### Disposition A: Accept the Persuasive Resolution as Applied

**Recommendation:** Accept the existing Persuasive resolution.

**Rationale:** The resolution correctly identifies the structural constraint: both Dental Composition profiles inherit section ordering from their CCDAonFHIR base profiles, and the FHIR IG publisher enforces snapshot element ordering from the base StructureDefinition. The resolution commits to aligning section order across profiles to the extent tooling allows. This addresses the submitter's concern while acknowledging real technical limitations. The related ticket FHIR-28462 (also Persuasive, now Published) addressed clarity of these same sections, suggesting this area has received adequate attention.

### Disposition B: Accept with Additional Guidance

**Recommendation:** Accept the Persuasive resolution but request the IG include a narrative table or comparison showing the section ordering of both the Dental Referral Note and Dental Consult Note side by side, so implementers can quickly see the mapping.

**Rationale:** While the resolution explains *why* ordering may differ (base profile constraints), implementers would benefit from explicit documentation showing how sections map between the two profiles. This would address the underlying usability concern without requiring structural changes to the profiles themselves.

### Disposition C: Decline – No Further Action Needed

**Recommendation:** Mark as resolved with no further action.

**Rationale:** The ticket was resolved in September 2022 as Persuasive, the related ticket FHIR-28462 has already been Published, and the specification has progressed. The IG publisher's constraints on snapshot ordering are well-understood. Reopening this for further harmonization work would require changes to the CCDAonFHIR base profiles, which is out of scope for the Dental Data Exchange IG.

## Recommendation

**Disposition A: Accept the Persuasive Resolution as Applied.**

The resolution is technically sound—it correctly identifies that section ordering is constrained by the CCDAonFHIR base profiles and the IG publisher. The commitment to align sections within those constraints directly addresses the submitter's request. The companion ticket FHIR-28462, from the same reporter about the same Consult Note sections, has already been published, indicating this area of the specification has been actively improved. No further action is required beyond accepting the existing resolution.
