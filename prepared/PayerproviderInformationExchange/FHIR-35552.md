### Description

The request asks that Value Set canonical URLs used in the CARIN Digital Insurance Card (DIC) specification be anchored in terminology.hl7.org (THO) rather than hl7.org/fhir. The submitter states that a UTG ticket should be created to register the Value Set in THO. Only Value Sets with a required binding to a 'code' data type should retain hl7.org/fhir anchoring.

This is one of a series of related ballot comments (Comments 1–5, imported by Ron G. Parker) all requesting the same change for different Value Sets within the DIC specification.

## 6. Proposed Dispositions

### Disposition A: Persuasive

**Rationale:** The request aligns with established HL7 terminology governance policy that Value Sets should be registered in THO (terminology.hl7.org) and use THO-anchored canonical URLs. The sibling ticket FHIR-35551 (Comment 1 in the same series) has already been resolved as **Persuasive**, establishing a precedent for this family of tickets. The issue has been reopened twice after STU publications, indicating the WG has acknowledged the change is needed but deferred it. Resolving as Persuasive commits the WG to migrating the affected Value Set canonical URL to THO and creating the corresponding UTG ticket.

**Action required:** Identify the specific Value Set referenced by "Comment 2," update its canonical URL to use a terminology.hl7.org base, and submit a UTG registration ticket to the Vocabulary WG.

### Disposition B: Persuasive with Modification

**Rationale:** Accept the request to move Value Set canonical URLs to THO, but modify the scope to clarify that the migration will occur as part of a coordinated batch effort across all five sibling tickets (FHIR-35551 through FHIR-35555) rather than individually. This avoids piecemeal changes and ensures consistency. The modification would also note that the timing depends on Vocabulary WG readiness and the formalized UTG process mentioned in the prior disposition rationale.

**Action required:** Same as Disposition A, but coordinate with the batch of sibling tickets and the Vocabulary WG to establish a timeline. Document that all five Value Sets will be migrated together.

### Disposition C: Considered – No Change (Deferred)

**Rationale:** Defer the change again, noting that while THO anchoring is the correct long-term direction, the Vocabulary WG process for registering IG-specific Value Sets in THO is still maturing. The WG has already deferred this twice (2022 and 2024), and the most recent comment (2025-09-23) suggests checking STU publication status before acting. A third deferral may be appropriate if the current publication cycle does not include this change.

**Action required:** No specification change. Document the deferral rationale and reassess for the next ballot cycle.

## 7. Recommendation

**Disposition A: Persuasive** is recommended.

The sibling ticket FHIR-35551 has already been resolved as Persuasive, establishing clear precedent. This ticket makes the same request for a different Value Set, and there is no reason to treat it differently. The THO anchoring policy is well-established HL7 practice. This ticket has been deferred twice over three years; continued deferral undermines the ballot process and the community feedback. Resolving as Persuasive and proceeding with the UTG registration is the most appropriate path forward.
