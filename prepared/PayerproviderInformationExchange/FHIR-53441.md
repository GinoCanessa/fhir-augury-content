## Summary

This change request asks to add a new "Best Practice" page as an entry under the Guidance menu in the CARIN Digital Insurance Card (DIC) Implementation Guide. The resolution description states: "Apply change as proposed." This is part of a broader set of IG restructuring changes for the DIC STU2 update, which includes menu bar reorganization (FHIR-53289), a new Support menu (FHIR-53028), use case page restructuring (FHIR-53096), and additional guidance/narrative pages.

## Details

The request is straightforward: introduce a dedicated "Best Practice" page that will be accessible from the Guidance menu in the IG's navigation bar. The page itself is not described in the ticket content — only the structural change to add the menu entry is specified.

This ticket is one of several filed by Mark Scrimshire on 2025-11-14 as part of a coordinated set of DIC STU2 changes. Related structural changes include:

- **FHIR-53289** (Persuasive): Restructure Menu Bar and Add Use Cases — moved Guidance before Use Cases in the menu bar for better reading flow.
- **FHIR-53028** (Persuasive): Create Support Menu in IG Menu Bar — added a Support menu with links to Discussion Forum, Project Page, Implementer Support, Jira Dashboard, Propose a Change, and Downloads.
- **FHIR-53096** (Persuasive with Modification): Restructure Use Case Page to connect multiple use case pages — changed the use case section to a multi-page structure to accommodate new STU2 narrative.
- **FHIR-53442** (Persuasive): Add Use Case for Support for Multiple Digital Insurance Cards.
- **FHIR-53443** (Persuasive): Add Guidance on RxIIN in Coverage Class ValueSet.

The ticket is tagged `Block-Vote-1` and `Fit-For-Teleconference`, indicating it is part of a block vote and suitable for teleconference discussion.

## Proposed Dispositions

### Disposition A: Accept (Apply as Proposed)

Accept the change as proposed and add a Best Practice page as a new entry under the Guidance menu.

**Rationale:** The resolution is "Persuasive" with direction to "Apply change as proposed." Adding a best practice page is a low-risk navigational enhancement that provides a dedicated location for implementation guidance. It aligns with the broader STU2 IG restructuring effort (FHIR-53289, FHIR-53028, FHIR-53096) and improves discoverability of practical guidance for implementers.

**Action:** Add a new `bestpractice.md` (or equivalent) page to the IG source and register it as an entry under the Guidance menu in the IG's `sushi-config.yaml` or `menu.xml` configuration.

### Disposition B: Accept with Modification — Require Content Outline

Accept the structural change but request that an initial content outline or stub content be provided for the Best Practice page before publication.

**Rationale:** The ticket requests adding the page but does not describe what content will appear on it. Publishing a Best Practice page with no content or only placeholder text could confuse readers. Requiring at least a content outline (e.g., section headings such as "Card Display Best Practices," "Data Transmission Guidance," "Security Considerations") would ensure the page adds value from its first appearance.

**Action:** Accept the menu entry addition and ask the submitter to provide initial content or section headings before the page goes live in the published IG.

### Disposition C: Decline — Insufficient Detail

Decline the change request due to insufficient specification of what content will appear on the Best Practice page.

**Rationale:** While the structural concept is sound, the ticket provides no detail on what best practices will be documented. Without content requirements, there is a risk of an empty or low-value page being added. The content could instead be integrated into existing Guidance pages rather than creating a new standalone page.

**Action:** Return to submitter requesting a description of the intended best practice content, with the option to re-submit.

## Recommendation

**Disposition A: Accept (Apply as Proposed)**

This ticket is part of a coordinated block of DIC STU2 IG restructuring changes, all filed by the same reporter and all resolved as Persuasive. The related tickets (FHIR-53289, FHIR-53028, FHIR-53096) establish the broader navigation restructuring pattern. Adding a Best Practice page under the Guidance menu is a straightforward, low-risk change that improves the IG's organizational structure. The content for the page is expected to be developed as part of the STU2 update process — this ticket addresses the structural addition. Accepting as proposed maintains consistency with the resolution and the block vote approach.
