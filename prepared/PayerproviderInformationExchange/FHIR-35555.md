### Description

The ticket requests that Value Set canonical URLs in the CARIN Digital Insurance Card (DIC) IG be anchored in `terminology.hl7.org` (THO) rather than `hl7.org/fhir`. A UTG ticket should be created to register the Value Set in THO. The exception is value sets with required binding to a `code` data type, which should remain anchored in `hl7.org/fhir`.

This is one of a series of similar tickets filed by Joan Harper (imported by Ron G. Parker) across multiple IGs requesting the same THO migration pattern.

### Ticket History

## 4. Proposed Dispositions

### Disposition A: Persuasive — Migrate Value Set URLs to THO

**Action:** Accept the change request. Update all Value Set canonical URLs in the CARIN DIC IG to be anchored at `terminology.hl7.org` (THO) instead of `hl7.org/fhir`. Create corresponding UTG tickets to register each Value Set in THO. Retain `hl7.org/fhir` anchoring only for value sets with required binding to a `code` data type.

**Rationale:** This aligns with HL7 terminology governance best practices. Sibling ticket FHIR-35551 in the same specification was resolved as "Persuasive," establishing precedent within this IG. FHIR-35566 in another IG successfully applied a similar change. The Vocabulary WG processes that were previously cited as a reason for deferral have had time to mature since the original 2022 deferral. The ticket has highest priority and has been deferred twice already.

**Risks:** Requires coordination with the Vocabulary WG and UTG process. Changing canonical URLs is a breaking change for implementers who have already adopted the current URLs; a migration strategy and versioning approach would be needed.

### Disposition B: Not Persuasive with Modification — Selective THO Migration with Version Boundary

**Action:** Partially accept. Identify which DIC Value Sets are appropriate candidates for THO migration (e.g., those with broad applicability beyond the DIC IG) versus those that are domain-specific and may remain IG-anchored. Migrate only the appropriate candidates, deferring remaining value sets. Document the decision criteria.

**Rationale:** Not all value sets benefit equally from THO registration. Domain-specific, rapidly evolving value sets (as noted in FHIR-38529's resolution) may be better managed within the IG during early maturity phases. A selective approach balances governance compliance with practical considerations for an IG that has gone through STU1 and STU1.1 but may still be evolving.

**Risks:** Requires analysis effort to categorize each value set. May not fully satisfy the reporter's intent. Inconsistency in URL anchoring within the same IG could cause implementer confusion.

### Disposition C: Considered for Future Use — Defer to Next Major Version

**Action:** Defer the change to a future ballot cycle (e.g., STU2) when the IG is undergoing a major revision that would already introduce breaking changes. Document the intent to migrate in the IG's roadmap.

**Rationale:** This is consistent with prior resolutions of this ticket (twice resolved as "Considered for Future Use" with 11-0-1 votes). Changing canonical URLs mid-lifecycle introduces breaking changes for existing implementers. Bundling this with a major version boundary reduces disruption. The Vocabulary WG process may be more mature by that time.

**Risks:** Continues to defer a repeatedly-raised governance concern. The ticket has already been deferred twice and re-opened twice, suggesting the WG needs to make a definitive decision. Further deferral may erode confidence in the ballot process.

## 5. Recommendation

**Recommended Disposition: A — Persuasive**

This ticket has been through two full cycles of deferral and reopening since January 2022. The sibling ticket FHIR-35551 in the same DIC specification was already resolved as "Persuasive," establishing that the WG has accepted this principle. Continuing to defer creates inconsistency within the IG (some value sets migrated, others not) and undermines the ballot process given the ticket's "Highest" priority and "Block-Vote-6" label. The IG has matured through STU1 and STU1.1, reducing the argument that value sets are too fluid. Coordinating with the Vocabulary WG to execute the THO migration now would resolve a long-standing governance gap and align with HL7 terminology best practices.
