The CARIN Digital Insurance Card IG already defines a **SupportingImage** extension on the Coverage resource. This ticket requests that IG documentation be enhanced to explicitly describe how this extension supports inclusion of full front-and-back card images — a common real-world need for digital insurance card implementations where members expect a visual representation of their physical card.

### Impact Assessment
- **Scope:** Documentation-only change (narrative/guidance); no structural changes to profiles, extensions, or value sets.
- **Risk:** Low. The extension already exists; this merely adds clarifying text about an intended use case.
- **Pre-Applied:** The label indicates the change has already been applied to the IG source, so this is a retrospective ballot vote to ratify an existing edit.

### Considerations
1. The documentation should clearly state whether **both** front and back images are expected in a single extension instance or as separate extension instances on the Coverage resource.
2. Guidance on image format (e.g., MIME types — JPEG, PNG), size constraints, and accessibility (alt text) would strengthen the documentation.
3. Coordination with FHIR-55562 (experimental Summary of Benefits profile) is advisable since both modify extensions on the same Coverage profile.
