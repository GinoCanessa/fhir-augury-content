## Problem Description

The reporter observed that the Obligation profiles in the Laboratory Report IG contain tables in the "Description & Constraints" section where the **"Actor" column is blank**. The IG defines three actor roles, and each obligation table row presumably corresponds to one of those actors, but the empty Actor column makes it impossible to determine which actors have which obligations. The reporter requested that these tables be populated or that usage guidance be improved.

## Discussion & Comment Summary

- **hbuitendijk (2025-01-27):** Identified this as a **publisher rendering issue**, not a data problem. Provided JSON evidence from the StructureDefinition showing that actor canonical URLs (e.g., `http://hl7.org/fhir/uv/laboratory/actor-repos-uv-lab`) are present in the obligation extensions alongside obligation codes (e.g., `SHALL:handle`). Directed Jose Costa-Teixeira to investigate the publishing toolchain. Referenced the [Patient Obligations profile](https://hl7.org/fhir/uv/lab-report/2024Sep/StructureDefinition-Patient-obl-uv-lab.html) as an example.

- **giorgio (2025-02-21):** Confirmed the publisher issue has been fixed. Suggested additionally adding a **Title** to the Actor definition so that the rendered table displays a human-readable title instead of just the actor name/URL.

## Resolution Description

> The issue with the Publisher has now been resolved. Will also add the title for the actors for completeness (and human readability).

The fix has two parts:
1. **Publisher fix** — The FHIR IG Publisher was corrected to properly render actor information from obligation extensions in profile tables.
2. **Actor title addition** — The Laboratory Report IG will add titles to ActorDefinition resources so that the rendered obligation tables show human-readable actor names rather than canonical URLs.

## Proposed Dispositions

### Disposition A: Persuasive with Modification (Confirm Resolution)
**Accept the existing resolution.** The root cause was a FHIR IG Publisher bug that failed to render actor information already present in obligation extensions. The publisher has been fixed, and the IG will additionally add human-readable titles to ActorDefinition resources. This fully addresses the reporter's concern — obligation tables will now display which actors have which obligations, with readable names rather than canonical URLs.

### Disposition B: Persuasive
**Accept as-is without modification.** The reporter's concern about blank Actor columns is valid and the fix (publisher correction) directly addresses it. However, this disposition would not include the additional enhancement of adding actor titles for human readability, relying solely on the publisher fix to display whatever actor identifiers already exist in the extensions.

### Disposition C: Not Persuasive
**Reject the change request.** Argue that the obligation data was always present in the underlying StructureDefinition and the rendering gap was a transient tooling issue, not a specification defect. The publisher has since been updated independently of this ticket. However, this disposition ignores the valid point that actor titles improve readability and the IG itself benefits from the enhancement.

---

## Recommendation

**Disposition A: Persuasive with Modification** — This aligns with the existing resolution and the consensus in the comments. The issue was real (blank Actor columns), the root cause was identified (publisher rendering bug), the fix has been implemented (publisher correction), and an additional enhancement (actor titles for readability) adds genuine value. The modification to add human-readable titles to ActorDefinition resources goes beyond the pure publisher fix and improves the IG's usability, making "Persuasive with Modification" the appropriate disposition.
