## Summary

The submitter (Gay Dolin) requests that the **Laboratory Report (FHIR) IG** add a "Plain Language Summary about HL7 and this Guide" to its home page. The request cites the MCC IG as an exemplar ([https://hl7.org/fhir/us/mcc/#plain-language-summary-about-hl7-and-this-guide](https://hl7.org/fhir/us/mcc/#plain-language-summary-about-hl7-and-this-guide)) and notes the growing trend in peer-reviewed journals to require plain language summaries (PLS) to make technical artifacts accessible to non-professional audiences such as patients and the general public.

This is part of a **bulk ballot submission** by Gay Dolin across many IGs — nearly identical tickets were filed against numerous specifications in September 2024 (e.g., FHIR-47555 for Patient Administration, FHIR-47560 for Structured Documents, FHIR-47566 for Public Health/NHSN dQM, and many others).

## Resolution Description

The ticket was resolved as **Persuasive** with the following modifications:

> - Review with the template developers to have the About HL7 paragraphs part of the standard IG publications process in either the footer or header
> - Always include the About the Guide as visible text with the header "About the Guide" instead of "Plain Language" etc., at the start of the Home Page.

## Related Zulip Discussions

### 1. [IG creation > plain-language](https://chat.fhir.org/#narrow/stream/IG%20creation/topic/plain-language)

Active thread (Mar–May 2025) discussing the **operational mechanics** of adding plain language summaries to IGs. Key points:
- HL7 maintains a centralized repository at [github.com/HL7/plain-language](https://github.com/HL7/plain-language) for standardized summaries.
- Grahame Grieve provided an AI prompt for generating summaries via ChatGPT, targeting ~200 words for a general audience.
- IG authors can either manually write a summary or use AI-generated drafts that require human review and editing.
- The summaries use the `# General` heading prefix to be recognized by hl7.org/fhir.org publishing.
- Mark Scrimshire reported issues with the plain-language template causing IG publication crashes (script injection errors).

### 2. [patient empowerment > Plain language IG summaries](https://chat.fhir.org/#narrow/stream/patient%20empowerment/topic/Plain%20language%20IG%20summaries)

Active thread (Sep–Oct 2024) where Josh Mandel presented LLM-generated IG summaries:
- TSC approved the approach of hosting summaries externally and injecting them into published IGs.
- Summaries are maintained as markdown files in the [HL7/plain-language](https://github.com/HL7/plain-language) repo.
- Grahame Grieve confirmed summaries would not live in the IG source itself but would be injected during publication.
- Dave deBronkart provided patient-perspective feedback on readability and formatting.

## Analysis

This ticket is one instance of a broad cross-IG initiative to improve accessibility for non-technical audiences. Since this ticket was filed:

1. **Infrastructure has matured**: The HL7/plain-language repository now provides a centralized, standardized approach to managing plain language summaries across all IGs. Summaries are maintained externally and injected at publication time by the IG publisher tooling.
2. **The resolution aligns with the evolved approach**: The resolution's two modifications — (a) having "About HL7" in the standard template header/footer, and (b) including an "About the Guide" section — are consistent with the centralized plain-language repository approach.
3. **No IG source changes needed**: Per Grahame Grieve's clarification, the summaries do not go into the IG source. They are managed in the HL7/plain-language repo and injected during publication.
4. **Comment from hbuitendijk** flags the need for coordination with OO Main and template developers, which is appropriate given the cross-cutting nature.

The primary implementation action is for the Laboratory Report IG team to:
- Create or review an AI-generated plain language summary for the Lab Report IG
- Submit it as a PR to the [HL7/plain-language](https://github.com/HL7/plain-language) repo following the documented process

## Proposed Dispositions

### Disposition A: Accept as Persuasive (affirm existing resolution)

**Action:** Affirm the existing "Persuasive" resolution. The Laboratory Report IG team creates a plain language summary for the IG and submits a PR to [HL7/plain-language](https://github.com/HL7/plain-language). The "About HL7" content will be handled by the standard IG template, and "About the Guide" text will be visible on the home page.

**Rationale:** The resolution is reasonable and aligns with the now-standardized HL7 approach. The centralized HL7/plain-language repo and IG publisher tooling handle the mechanics. This is consistent with how other work groups have resolved identical tickets.

**Trade-offs:** Requires Lab Report IG team effort to draft/review the summary and submit a PR. Coordination with template developers (per hbuitendijk's comment) may still be pending but is an infrastructure concern, not specific to this ticket.

### Disposition B: Accept as Persuasive with Modification — defer to centralized process only

**Action:** Modify the resolution to state that no changes are needed in the IG source itself; the Lab Report IG team should only submit a plain language summary to the HL7/plain-language repository. Remove the requirement for "About the Guide" text in the IG home page, as the centralized system handles display.

**Rationale:** Since the HL7 plain-language infrastructure now handles injection at publication time, requiring IG-level home page changes is redundant. This simplifies the implementer's burden and is consistent with Grahame Grieve's stated approach that summaries should not be baked into the IG source.

**Trade-offs:** May diverge from the original resolution's specific wording about "About the Guide" as visible text in the home page. However, this better reflects current HL7 infrastructure reality.

### Disposition C: Defer — pending template standardization

**Action:** Defer resolution until the IG template and plain-language publication infrastructure are fully stabilized. Noted issues include publication crashes (Mark Scrimshire's report in May 2025) and ongoing FMG discussions about standardizing the approach.

**Rationale:** The infrastructure is still maturing. Committing the Lab Report IG to a specific implementation approach while the tooling is still evolving could create rework.

**Trade-offs:** Delays progress on an initiative with broad HL7 support. The ticket has been open since September 2024 and already has a resolution. Other work groups have successfully moved forward despite the same infrastructure status.

## Recommendation

**Disposition A: Accept as Persuasive (affirm existing resolution).**

The resolution is sound, aligns with the now-standardized HL7 approach, and is consistent with how other work groups have resolved identical bulk-filed tickets (FHIR-47555, FHIR-47560, FHIR-47566). The HL7/plain-language repository and publication tooling provide a clear path for implementation. The Lab Report IG team should create a summary using the documented AI-assisted process and submit a PR to the HL7/plain-language repo. The "About HL7" template content and "About the Guide" section are reasonable additions that improve accessibility for non-technical readers.
