## Description

This change request asks the IG to consider methods for delivering at-home in-vitro test results to the patient's physician—potentially via consumer health platforms like Apple Health or Google Health—and to consider adding physician information to the message sent to Public Health Authorities (PHAs).

The ticket addresses two related gaps in the current At-Home In-Vitro Test Report IG:

1. **Patient-to-physician result delivery:** The IG currently focuses on reporting at-home test results to PHAs. There is no defined mechanism for the patient to share those same results with their personal physician or care team.
2. **Physician identification in PHA messages:** The current message bundle sent to PHAs does not include the patient's physician, which could facilitate downstream clinical follow-up.

## Proposed Dispositions

### Disposition A: Considered for Future Use
**Rationale:** This request involves significant architectural complexity (HIGH_COMPLEXITY label) requiring interoperability with consumer health platforms (Apple Health, Google Health) and EHR integration pathways that are largely undeveloped across the industry. The companion ticket FHIR-38203 confirmed (Jan 2026) that no stakeholders are currently ready to work out these details. Deferring allows the ecosystem—including consumer health APIs, SMART on FHIR app frameworks, and TEFCA infrastructure—to mature before the IG attempts to define this pathway. The IG could add a "Future Considerations" section documenting this use case and potential approaches without normative content.

**Vote text:** "Considered for Future Use. This change request involves high complexity and requires industry-wide infrastructure (consumer health platform APIs, EHR ingestion pathways) that is not yet mature enough to standardize. No stakeholders are currently available to develop the detailed specifications. Add a non-normative 'Future Considerations' section to the IG documenting the patient-to-physician result delivery use case and potential mechanisms."

### Disposition B: Persuasive with Modification
**Rationale:** Address the lower-complexity portion of the request—adding the patient's physician/ordering provider to the PHA-bound message bundle—without tackling the consumer health platform integration. This is achievable by adding an optional `Practitioner` reference (e.g., the patient's primary care provider) to the existing Bundle profile. The broader question of Apple/Google Health integration can be deferred separately.

**Vote text:** "Persuasive with Modification. Address the portion of this request relating to including physician information in the PHA message bundle by adding an optional Practitioner reference to the At-Home In-Vitro Test Report Bundle profile. The consumer health platform integration aspect (Apple Health, Google Health, patient-to-physician delivery) is deferred as a separate future consideration pending ecosystem readiness."

### Disposition C: Not Persuasive with Modification
**Rationale:** The IG's scope is specifically at-home test reporting to PHAs. Patient-to-physician communication is a fundamentally different data flow that belongs in a separate IG or in the broader US Core/USCDI framework. Adding physician info to PHA messages is also of limited value since PHAs do not typically use physician identifiers for at-home test result processing. However, acknowledge the valid use case and recommend cross-referencing with ongoing work in the CGM IG (which addresses a similar consumer-to-provider data flow pattern).

**Vote text:** "Not Persuasive with Modification. The patient-to-physician result delivery use case is outside the scope of this IG, which focuses on at-home test reporting to PHAs. Recommend this use case be addressed in a separate project or through the PHR/EHR use case work tracked in FHIR-38203. Add an informative note in the IG referencing FHIR-38203 and the CGM IG as related work."

## Recommendation

**Disposition A: Considered for Future Use** is recommended.

The request is conceptually sound and addresses a real gap in the at-home testing workflow. However, the HIGH_COMPLEXITY label is well-justified: delivering results via consumer health platforms requires standardized APIs and EHR integration pathways that do not yet exist in a mature, implementable form. The Jan 2026 discussion on the companion ticket (FHIR-38203) confirmed that no stakeholders are ready to develop the details. Re-scoping to only add a physician reference to PHA messages (Disposition B) would be a partial solution that may not see meaningful implementation. The most productive path is to formally document this use case as a future direction in the IG, allowing the work to proceed when industry readiness improves—potentially coordinating with the CGM IG and broader PHR/EHR integration efforts.
