## Analysis

### Context

The Laboratory Report (FHIR) IG currently uses a design where a FHIR "document" Bundle includes both a Composition resource (required for a valid FHIR Document) and a DiagnosticReport resource (used for searchability and clinical content). This dual-resource approach enables European jurisdictions to treat the lab report as both a formal document and a queryable DiagnosticReport. An STU note in the IG solicits ballot feedback on this design choice.

FHIR-47201 is a ballot comment from Giorgio Cangioli (a key contributor to the European FHIR laboratory work) explicitly supporting the current design. He recommends maintaining the Composition + DiagnosticReport pattern for consistency with the European FHIR document model.

### Tension in the Community

This ticket sits at the center of a significant design debate:

- **Supporters of the current dual approach** (like this ticket and FHIR-43179) value the European document model and want to maintain or extend it.
- **Critics** (FHIR-48059, Keith Boone) argue that combining Composition and DiagnosticReport creates ambiguity in interpretation and redundancy.
- **Pragmatists** (FHIR-46865, Hans Buitendijk) propose clarifying that there are actually three design choices, giving implementers explicit options.
- **Long-term simplifiers** (FHIR-55368, Robert Hausam) propose aligning DiagnosticReport and Composition at the core FHIR level so DiagnosticReport can itself anchor a FHIR Document.

The resolved ticket FHIR-46865 (Persuasive with Modification) already establishes that the IG should describe three design approaches, which partially addresses the feedback in FHIR-47201 by preserving the document Bundle approach as one valid option.
