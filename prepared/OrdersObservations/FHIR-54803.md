## Summary

The text summaries (human-readable narrative descriptions) for certain vital signs profiles contain errors where `Observation.category` is referenced instead of `Observation.code`. Specifically:

- **Oxygen Saturation (OxSat) profile:** The text summary incorrectly states a fixed `Observation.category.coding.code` = `'2708-6'`, when it should read `Observation.code.coding.code` = `'2708-6'`.
- **Blood Pressure (BP) profile:** The text summary incorrectly states a fixed `Observation.category.coding.system` = `'http://loinc.org'`, when it should read `Observation.code.coding.system` = `'http://loinc.org'`.

The reporter also notes that the text structure around these statements needs reworking, and that all other vital sign profiles should be audited for similar remnants of a previous modeling approach.

---

## Details

### Description

The ticket identifies leftover text from a previous approach to vital signs profiling that incorrectly attributes LOINC code/system bindings to `Observation.category` rather than `Observation.code`. These are purely errors in the human-readable text summaries — the underlying profile constraints likely already target the correct element paths. Two specific instances are called out:

1. **OxSat profile** — Text says `Observation.category.coding.code` = `'2708-6'`; should say `Observation.code.coding.code` = `'2708-6'`.
2. **BP profile** — Text says `Observation.category.coding.system` = `'http://loinc.org'`; should say `Observation.code.coding.system` = `'http://loinc.org'`.

The reporter requests:
- Correcting these two specific errors.
- Reworking the text structure around these statements for clarity.
- Auditing all other vital sign profiles for similar remnants.

## Proposed Dispositions

### Disposition A: Accept (Persuasive)

**Resolution:** Persuasive

Correct the text summaries for the Oxygen Saturation (OxSat) and Blood Pressure (BP) vital signs profiles as requested:

1. **OxSat profile:** Change the text from `Observation.category.coding.code` = `'2708-6'` to `Observation.code.coding.code` = `'2708-6'`.
2. **BP profile:** Change the text from `Observation.category.coding.system` = `'http://loinc.org'` to `Observation.code.coding.system` = `'http://loinc.org'`.
3. Rework the surrounding text structure in both profiles for clarity and consistency.
4. Audit all other vital signs profiles (Body Temperature, Heart Rate, Respiratory Rate, Body Height, Body Weight, Head Circumference, BMI) for similar remnants referencing `Observation.category` where `Observation.code` is intended, and correct any found.

This is a straightforward documentation fix that corrects objectively wrong element path references in the narrative text summaries.

### Disposition B: Accept with Broader Scope (Persuasive with Modification)

**Resolution:** Persuasive with Modification

Accept the corrections as described in Disposition A, but also expand the scope to address the broader pattern of text summary issues identified in related ticket FHIR-54521 (which reports additional text summary errors in the vital signs base profile). Coordinate with FHIR-54521 to perform a comprehensive review and correction of all vital signs profile text summaries in a single pass, ensuring:

- All element path references are correct (`Observation.code` vs `Observation.category` vs `Observation.component.code`).
- Value set bindings referenced in text match the actual profile constraints.
- Text structure is consistent across all vital signs profiles.

This consolidates related text summary fixes and reduces the risk of repeated piecemeal corrections.

### Disposition C: Decline (Not Persuasive)

**Resolution:** Not Persuasive

Decline on the basis that the text summaries are auto-generated narrative and do not constitute normative content. The underlying profile constraints (StructureDefinition elements, bindings, and invariants) are the authoritative source and are correct. Implementers should rely on the formal profile definitions rather than the text summaries. The effort to manually audit and correct all vital signs profile text summaries is better spent on tooling improvements to ensure text summaries are generated correctly from the formal definitions.

---

## Recommendation

**Disposition A: Accept (Persuasive)** is recommended.

The errors are clear-cut — the text summaries reference `Observation.category` where `Observation.code` is intended, creating confusion for implementers who read the narrative descriptions. These are not matters of opinion or design philosophy; they are factually incorrect element path references. The fix is low-risk and high-value, particularly given that the vital signs profiles are widely used and the R6 ballot cycle has seen significant restructuring of category vs. code usage (per FHIR-45617 and FHIR-50705), making accurate text summaries especially important for clarity. The audit of other profiles is a reasonable scope addition given the known pattern of similar errors (as evidenced by FHIR-54521).
