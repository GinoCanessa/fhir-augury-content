## Description

When using the R5 extension for `Observation.value[x]` (which adds the Attachment data type) in the Laboratory Report IG, SUSHI v3.14.0 incorrectly URL-encodes the square bracket characters in the extension URL. Specifically, SUSHI transforms:

- **Correct URL:** `http://hl7.org/fhir/5.0/StructureDefinition/extension-Observation.value[x]`
- **Encoded URL:** `http://hl7.org/fhir/5.0/StructureDefinition/extension-Observation.value%5Bx%5D`

The IG Publisher cannot process the percent-encoded form, resulting in a build error. As a workaround, the ticket proposes temporarily replacing the `observation-lab.fsh` FSH source file with a hand-crafted StructureDefinition JSON resource (including the snapshot required by SUSHI), where the bracket characters in the extension URL are preserved correctly. The intent is to revert to the FSH source once SUSHI fixes the encoding issue.

## Analysis

### Core Issue
This is a **tooling workaround** that was applied to the Laboratory Report IG to avoid a SUSHI bug. SUSHI v3.14.0 incorrectly percent-encoded square brackets in extension URLs referencing polymorphic elements (e.g., `Observation.value[x]`), causing downstream IG Publisher build failures. The workaround replaced the FSH source file (`observation-lab.fsh`) with a pre-built JSON StructureDefinition.

### Current State
A comment from October 2025 indicates that **SUSHI v3.16.5 has resolved the underlying encoding bug**. This means the workaround (JSON StructureDefinition in lieu of FSH) is likely no longer needed and should be reverted to restore the FSH source as the canonical definition.

### Impact
- The workaround introduces a maintenance burden: changes to the Observation-lab profile must be made in raw JSON rather than FSH.
- If the workaround JSON is left in place unnecessarily, it diverges from the standard FSH-based authoring workflow used by the rest of the IG.
- Reverting to FSH would restore consistency and reduce maintenance complexity.

## Proposed Dispositions

### Disposition A: Persuasive — Revert Workaround Now
**Rationale:** The underlying SUSHI bug (bracket URL encoding) has been confirmed fixed in SUSHI v3.16.5. The workaround should be reverted by restoring `observation-lab.fsh` as the source for the Observation-lab profile and removing the substitute JSON StructureDefinition. The ticket should be resolved as Persuasive, acknowledging that the workaround was appropriate at the time and the fix has been delivered upstream.

**Action Required:** Restore the FSH source file for the Observation-lab profile in the Laboratory Report IG, verify the build succeeds with current SUSHI (≥ v3.16.5), and remove the JSON workaround artifact.

### Disposition B: Persuasive with Modification — Verify Fix Then Revert
**Rationale:** While the comment asserts the SUSHI fix is in v3.16.5, the workgroup may want explicit verification before reverting. Accept the ticket as Persuasive with Modification, where the modification is to first confirm that the current SUSHI version correctly handles the `Observation.value[x]` extension URL in the Laboratory Report IG build before reverting the workaround.

**Action Required:** Build the Laboratory Report IG with `observation-lab.fsh` restored using SUSHI ≥ v3.16.5. If the build passes, remove the JSON workaround. If not, investigate further and keep the workaround in place.

**Rationale:** The workaround was a pragmatic interim fix applied by the reporter, and the underlying SUSHI issue is now resolved upstream. **Action Required:** None. Close as resolved with no specification change; note that the SUSHI fix is available.
