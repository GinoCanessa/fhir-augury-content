## Analysis

This ticket addresses a gap in the Laboratory Report (FHIR) IG where obligations are either absent or incomplete for the defined profiles. Obligations in FHIR specify what actors (e.g., content creators, content consumers) are expected to do with particular data elements — whether they SHALL populate them, SHALL be able to process them, etc. Without comprehensive obligation definitions, implementers lack clear guidance on conformance expectations for each actor role.

The resolution ("Persuasive") accepted the request and the resolution description directs that obligations be specified for all defined profiles. This is a significant editorial and conformance undertaking, as it requires reviewing every profile in the Laboratory Report IG and defining appropriate obligations for each relevant actor.

**Key considerations:**
- The Zulip thread on "Obligations extension does not meet slice" indicates that IGs adopting obligations (IPS, IPA, AU IGs) encounter tooling artifacts (snapshot-source sub-extensions) that generate QA messages. Implementers of this change should be aware of this known tooling behavior.
- Related tickets from other specifications (FHIR-52412, FHIR-52413) show that incomplete or ambiguous obligation definitions lead to interoperability confusion, reinforcing the value of this ticket's request.
- FHIR-51128 from the same reporter addresses a parallel concern in the EU Imaging Study Report, suggesting this is part of a broader effort to improve obligation coverage across European FHIR IGs.

## Proposed Dispositions

### Disposition A: Apply as Persuasive — Define obligations for all profiles

**Action:** Accept the resolution as stated. Systematically review all profiles in the Laboratory Report IG and define obligations for each, specifying actor expectations (e.g., content creator SHALL populate, content consumer SHALL be able to process) for constrained elements.

**Rationale:** The resolution is clear and addresses a real conformance gap. Obligations are a core part of FHIR IG conformance and the Laboratory Report IG should have them for all profiled resources to provide unambiguous implementation guidance.

**Risk:** Medium effort — requires careful review of every profile in the IG and consensus on actor definitions. Tooling QA noise from obligation extensions is a known issue but has no functional impact.

### Disposition B: Apply as Persuasive with Modification — Prioritize key profiles first

**Action:** Accept the intent but scope the initial work to the most critical profiles (e.g., DiagnosticReport, Observation, Specimen, Patient) and defer less central profiles (e.g., Device, Media) to a follow-up cycle. Document the prioritization and create follow-up tickets for remaining profiles.

**Rationale:** Defining obligations for every profile in one pass may delay the ballot cycle. A phased approach delivers the highest-value obligations sooner while keeping the full scope on the roadmap.

**Risk:** Partial coverage could still leave ambiguity for less common profiles. Requires discipline to follow through on deferred work.

### Disposition C: Defer — Request more specific guidance from submitter

**Action:** Return to the submitter for clarification on which actors and which specific obligation patterns are expected, before beginning the work.

**Rationale:** The description is broad ("obligations should be defined for all the specified profiles" and "their revision is needed") but does not specify which actor roles or obligation codes are intended. Clearer requirements would reduce rework.

**Risk:** Delays the work further. The resolution description already provides sufficient direction ("Specify obligations for all the defined profiles"), so deferral may be unnecessary.

## Recommendation

**Disposition A: Apply as Persuasive** — The resolution is clear and the need is well-established. The Laboratory Report IG should have obligations defined for all its profiles to ensure implementers have unambiguous conformance guidance. The ticket has already been resolved as Persuasive with a clear resolution description, and the work should proceed as directed. The workgroup should plan for the editorial effort required to review and annotate all profiles with appropriate actor-specific obligations.
