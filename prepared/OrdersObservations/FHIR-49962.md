## Summary

The reporter requests that the Order Catalog Implementation Guide (IG) be updated to include proper ImplementationGuide resource extensions and parameters, and to correct the contact and publisher metadata elements. This is a technical correction filed as part of a broader effort by Robert Hausam to align the Order Catalog IG with the final FHIR R5 release, alongside companion tickets FHIR-49913 (updating ObservationDefinition element names) and FHIR-49964 (updating ActivityDefinition canonical data types). The IG Publisher tool generates ImplementationGuide resources that reference extensions (such as `ig-parameter` and `resource-information` from the `http://hl7.org/fhir/tools/StructureDefinition/` namespace) and use parameter codes not present in the bound `guide-parameter-code` ValueSet. This ticket asks that the Order Catalog IG properly declare these extensions and parameters, and that the `contact` and `publisher` elements be corrected to reflect accurate metadata for the specification.

## Details

**Description:**

No detailed description body was provided in the Jira ticket beyond the title. The title itself specifies three actions:

1. **Add ImplementationGuide resource extensions** — Ensure that the IG resource includes the necessary extensions used by the IG Publisher (e.g., `http://hl7.org/fhir/tools/StructureDefinition/ig-parameter`, `http://hl7.org/fhir/tools/StructureDefinition/resource-information`), so the ImplementationGuide resource validates correctly.
2. **Add parameters** — Include proper parameter definitions in the ImplementationGuide resource for IG Publisher configuration parameters (e.g., `path-expansion-params`, `releaselabel`, `copyrightyear`, `apply-publisher`, `apply-contact`, etc.).
3. **Fix contact and publisher** — Correct the `ImplementationGuide.contact` and `ImplementationGuide.publisher` elements to accurately reflect the responsible work group and organization metadata.

## Proposed Dispositions

### Disposition A: Accept As Requested

#### Proposal

Accept the ticket and update the Order Catalog IG's `ImplementationGuide` resource to:
1. Add the standard IG Publisher extensions (`ig-parameter`, `resource-information`, and any other required tool extensions) to the ImplementationGuide resource definition.
2. Add all necessary IG Publisher parameter entries (e.g., `copyrightyear`, `releaselabel`, `path-expansion-params`, `apply-publisher`, `apply-contact`, `apply-version`) to `ImplementationGuide.definition.parameter`.
3. Correct the `ImplementationGuide.publisher` and `ImplementationGuide.contact` elements to accurately reflect the Orders & Observations work group and HL7 organizational metadata.

#### Justification

This is a straightforward technical correction that aligns the Order Catalog IG with established IG Publisher conventions and the final R5 release. The same reporter (Robert Hausam) has filed companion tickets (FHIR-49913, FHIR-49964) performing analogous R5 alignment corrections for the same IG. A related ticket (FHIR-35778, resolved Persuasive) previously established that IG Publisher-generated extensions and parameter codes need formal definitions to enable proper validation by other tools. Applying this correction ensures the IG resource validates without errors and that its metadata accurately identifies the responsible parties.

---

### Disposition B: Accept with Broader Metadata Review

#### Proposal

Accept the ticket but expand the scope to perform a comprehensive metadata review of the entire Order Catalog IG, including not only the ImplementationGuide resource extensions, parameters, contact, and publisher, but also verifying all other conformance metadata (jurisdiction, copyright, version, dependencies, and resource references) against the final R5 release and current IG Publisher requirements. Consolidate this work with the related tickets FHIR-49913 and FHIR-49964 into a single coordinated update.

#### Justification

Since multiple technical correction tickets have been filed against the Order Catalog IG for R5 alignment (FHIR-49962, FHIR-49913, FHIR-49964), it may be more efficient and thorough to address all metadata and conformance issues in one pass. This would reduce the risk of discovering additional issues after each individual fix is applied. However, this approach increases scope and may delay resolution if the broader review uncovers additional problems.

---

### Disposition C: Decline

#### Proposal

Decline the ticket on the grounds that the ImplementationGuide resource extensions and parameters are auto-generated by the IG Publisher tool, and that publisher/contact metadata corrections are editorial in nature and do not affect the clinical or technical content of the IG.

#### Justification

One could argue that IG Publisher tool extensions are internal build artifacts that do not need to be formally specified in the IG source, and that contact/publisher metadata discrepancies are cosmetic. However, this position is weak: FHIR-35778 (resolved Persuasive for FHIR Core) established the precedent that these extensions and parameters need formal definitions for proper validation. Declining would leave the IG in a state that produces validation errors and contains inaccurate metadata.

---

### Recommendation

**Recommended disposition:** A (Accept As Requested)

This ticket requests a well-defined, narrowly scoped technical correction to bring the Order Catalog IG's ImplementationGuide resource into conformance with standard IG Publisher practices and the final R5 release. The three requested changes — adding extensions, adding parameters, and fixing publisher/contact — are all necessary for the IG to validate cleanly and present accurate metadata. The precedent set by FHIR-35778 (Persuasive) confirms that these IG Publisher extensions and parameters need formal definitions. The reporter, Robert Hausam, is actively working on R5 alignment for this IG and has demonstrated domain knowledge through companion tickets. Accepting as requested provides a clean, targeted fix without unnecessary scope expansion, while Disposition B's broader review, though potentially beneficial, risks delaying a straightforward correction. Disposition C is not viable given the established precedent.
