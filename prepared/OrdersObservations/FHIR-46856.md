## Description

The submitter (Hans Buitendijk) requests that the "Navigating the profiles" section diagram in the Laboratory Report IG be updated to show optionality of profile relationships. Currently, the diagram implies that a Composition **MUST** be used in all circumstances, whereas the design approach is to **allow** its use optionally. Specifically:

- The relationship between **Composition: Laboratory Report** and **DiagnosticReport: Laboratory Report** should be marked as optional in both directions.
- Other relationships representing choices (e.g., animal vs. person) or elements that may not always be present should also be annotated for optionality where appropriate.
- The change should not substantially increase diagram complexity beyond what is needed for clarity.

## Analysis

This ticket addresses a documentation clarity issue in the Laboratory Report IG's "Navigating the profiles" diagram. The core problem is that the current diagram does not indicate which profile relationships are optional vs. required, leading readers to incorrectly assume that Composition is mandatory in all lab report implementations.

**Key context from related tickets:**
- **FHIR-46865** (same reporter) establishes that there are *three* design choices for lab reporting, with Composition being optional in two of them. This was resolved as Persuasive with Modification.
- **FHIR-50691** and **FHIR-51275** confirm a cross-IG consensus that DiagnosticReport is the primary resource and Composition is supplemental/optional.
- **FHIR-43246** demonstrates that the lack of clarity around optionality has caused real confusion for implementers (Belgian spec incompatibility concerns).
- **FHIR-51157** shows another IG (EU Imaging Study Report) already making the DR reference optional (0..) and aligning with the Lab Report approach.

The request is narrowly scoped: annotate the existing diagram with optionality indicators (e.g., "0..1" cardinalities, dashed lines, or similar visual cues). This is a low-risk, high-value documentation improvement.

## Proposed Dispositions

### Disposition A: Persuasive
Accept the request as stated. Update the "Navigating the profiles" diagram to annotate optionality on profile relationships — particularly marking the Composition ↔ DiagnosticReport relationship as optional in both directions. Extend optionality annotations to other choice-based relationships (e.g., animal vs. person profiles) where appropriate. This aligns with the design philosophy confirmed by FHIR-46865 and the approach already adopted by FHIR-51157.

### Disposition B: Persuasive with Modification
Accept the request for the Composition ↔ DiagnosticReport optionality annotation, but defer annotation of other relationships (animal vs. person, optional elements) to a separate ticket or future ballot cycle. Add a brief legend or note to the diagram explaining the optionality notation. This limits scope to the most impactful change and avoids overloading the diagram.

### Disposition C: Not Persuasive
The diagram is intentionally simplified to aid navigation, and adding optionality notation would increase visual complexity. The design choices section and profile definitions already convey optionality through cardinalities and conformance language. Readers should refer to those sections for detailed conformance requirements rather than relying on the navigation diagram.

## Recommendation

**Disposition A: Persuasive.** The diagram currently misleads readers into thinking Composition is required, which contradicts the IG's stated design philosophy and has caused documented confusion (FHIR-43246). Multiple related tickets across IGs (FHIR-46865, FHIR-50691, FHIR-51157, FHIR-51275) confirm that the Composition–DiagnosticReport relationship is optional by design. Annotating optionality on the diagram is a straightforward documentation fix that aligns with the existing design intent and improves implementer understanding without significant complexity increase.
