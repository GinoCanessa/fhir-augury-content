## Proposed Dispositions

### Disposition A: Persuasive (Correct the resource type label)

**Action:** Accept the change request and correct the created resource type from "Device" to "Practitioner" in the relevant example section of the DICOM SR to FHIR Resource Mapping IG.

**Rationale:** This is a straightforward editorial/errata correction. The reporter has identified a mislabeled resource type in the examples. The pattern is consistent with the sibling ticket FHIR-47513, which reports the same class of error (Device listed where Observation should be). The IG editors should verify the specific example section and update the resource type to Practitioner where the context requires a person performing or interpreting a study, not a device.

**Impact:** Minimal — editorial correction to an example. No structural or normative changes.

### Disposition B: Persuasive with Modification (Correct and audit all example resource labels)

**Action:** Accept the change request, correct the specific instance, and additionally audit all example subsections in the IG for similar resource type mislabeling errors.

**Rationale:** Multiple tickets from the same ballot review cycle (FHIR-47506, FHIR-47513, FHIR-47515) report incorrect resource types or example mistakes. A broader audit of the examples section would address these systematically rather than one-by-one, reducing the risk of lingering errors.

**Impact:** Low — editorial corrections across the examples section. Slightly more effort but improves overall IG quality.

### Disposition C: Not Persuasive (Device is intentionally listed)

**Action:** Reject the change request if the IG editors confirm that Device is the correct resource type for this particular example context.

**Rationale:** In some DICOM SR mapping scenarios, a Device resource is legitimately created (e.g., for algorithm identification or equipment). If the example in question is intentionally showing Device creation and the reporter has misunderstood the context, no change is needed. However, the IG should add clarifying text to explain why Device is used.

**Impact:** None if rejected; minor editorial addition if clarifying text is added.

## Recommendation

**Disposition A: Persuasive** — This appears to be a clear editorial error where the wrong FHIR resource type is listed in an example. The reporter's observation is consistent with a pattern of mislabeled resource types found by the same reviewer in the same ballot cycle (see FHIR-47513). The workgroup should verify the specific example and correct "Device" to "Practitioner" where the mapping context involves a person (e.g., performing physician, interpreting practitioner) rather than equipment. If the broader audit in Disposition B is feasible within the current cycle, it would be beneficial but is not strictly required for this ticket.
