## Description

Eric Haas requests changing the binding strength of `ImagingStudy.modality` and `ImagingStudy.series.modality` from **extensible** to **preferred**. The rationale is that the current extensible binding locks implementers into the DICOM code system, which may not be used by all jurisdictions or implementation communities. The reporter acknowledges not being a domain expert and is raising this as a hypothetical concern to prevent future implementers from being trapped into a code system they may not use or want.

## Analysis

This ticket is part of a batch of similar requests from Eric Haas seeking to relax extensible bindings to preferred across multiple resources. The core argument is that extensible bindings may constrain jurisdictions that use alternative code systems.

**For changing to preferred:**
- Sibling tickets FHIR-55649 and FHIR-55651 were accepted as Persuasive by their respective work groups (Patient Administration, Orders & Observations), establishing a pattern of acceptance for this request type.
- FHIR-36120 already changed these elements to CodeableConcept, which provides a mechanism for local codes via `.text` and additional codings, partially addressing the concern.

**Against changing to preferred:**
- The II WG member (jwhitby) has stated the DICOM code system is the logical and only viable code system for imaging modalities—there is no known competing terminology.
- FHIR-47481 demonstrates a WG precedent of carefully considering binding changes for ImagingStudy terminology; that ticket to tighten bindings was rejected as Not Persuasive.
- Imaging modality codes are inherently tied to DICOM standards. Unlike encounter participant types or observation interpretations, imaging modalities are a domain where DICOM is universally adopted.
- Weakening the binding to preferred removes interoperability assurance without a demonstrated real-world need from any jurisdiction.

## Proposed Dispositions

### Disposition A: Not Persuasive
**Rationale:** The DICOM modality code system is the universally adopted standard for imaging modalities. There is no known competing terminology that could represent imaging modality data, making the concern hypothetical rather than practical. The extensible binding already permits additional codes beyond the value set while requiring DICOM codes to be used when they exist—this is the appropriate level of constraint for interoperability in the imaging domain. The prior change to CodeableConcept (FHIR-36120) already gives implementers flexibility to include local codes alongside DICOM codes. Relaxing to preferred would weaken interoperability without demonstrated need.

### Disposition B: Persuasive
**Rationale:** For consistency with sibling tickets FHIR-55649 and FHIR-55651 that were resolved as Persuasive, change the binding from extensible to preferred. This gives jurisdictions maximum flexibility to adopt alternative terminologies if they emerge. The practical impact is small since most implementations will continue using DICOM codes regardless of binding strength.

### Disposition C: Persuasive with Modification
**Rationale:** Keep the binding as extensible but add a clarifying note in the element definition acknowledging that DICOM is the expected code system for imaging modalities, and that jurisdictions using alternative imaging classification systems should coordinate with the Imaging Integration work group. This preserves interoperability while documenting the expectation and providing a path for jurisdictions with genuine alternative needs.

## Recommendation

**Disposition A: Not Persuasive**

The imaging domain is fundamentally different from the resources addressed by the sibling tickets. Unlike encounter participant types or observation interpretations, imaging modalities are universally defined by DICOM with no known competing terminologies. The II WG member (jwhitby) has confirmed this assessment. The reporter (ehaas) has acknowledged this is a hypothetical concern and that he is not a domain expert. An extensible binding is the correct choice here: it requires use of DICOM codes when they exist while still permitting supplemental codes—exactly the right balance for a domain with a single dominant universal standard. The prior adoption of CodeableConcept (FHIR-36120) already provides a mechanism for local code augmentation.
