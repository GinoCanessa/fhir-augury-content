## Description

The submitter suggests adding the word "resource" after each FHIR resource name in the subsection headings of the DICOM SR to FHIR Resource Mapping IG. This would make it clear that the subsection titles refer to FHIR concepts (e.g., "Observation Resource", "Patient Resource") rather than DICOM concepts, reducing ambiguity for readers working across both standards.

## Analysis

This is a straightforward editorial request to improve clarity in the DICOM SR to FHIR Resource Mapping IG. The IG's example sections contain subsection headings that use bare FHIR resource type names (e.g., "Observation", "Patient", "Device"). Because the IG bridges DICOM and FHIR domains, these bare names can be ambiguous — readers may not immediately understand whether the heading refers to a FHIR resource or a DICOM concept. Adding "Resource" as a suffix (e.g., "Observation Resource") would disambiguate.

This aligns with the closely related ticket FHIR-47510, which requests renaming a section title for the same reason. Both tickets reflect a pattern of feedback from the same reporter about improving naming consistency and clarity across the IG's example sections. Addressing this ticket alongside FHIR-47510 would be efficient and yield consistent results.

The change is low-risk and editorial in nature — it does not affect conformance, profiles, or technical content.

## Proposed Dispositions

### Disposition A: Persuasive (Recommended)
**Action:** Accept the change request and add "Resource" after each FHIR resource type name in subsection headings throughout the IG's example sections.

**Rationale:** The request is reasonable, low-effort, and improves readability for an IG that inherently bridges two standards with overlapping terminology. The disambiguation benefit is clear, and the change is consistent with feedback in FHIR-47510. This should be addressed in coordination with FHIR-47510 to ensure naming consistency across all sections.

**Suggested wording:** Update subsection headings from bare resource names (e.g., "Observation", "Patient", "ImagingStudy") to qualified names (e.g., "Observation Resource", "Patient Resource", "ImagingStudy Resource") throughout the example mapping sections.

### Disposition B: Persuasive with Modification
**Action:** Accept the request but use a different qualification pattern, such as "FHIR Observation" or "Observation (FHIR Resource)" instead of "Observation Resource".

**Rationale:** While the intent is correct, simply appending "Resource" may not be sufficient disambiguation in all contexts since DICOM also has the concept of resources. Using a "FHIR" prefix or a parenthetical "(FHIR Resource)" would provide even stronger disambiguation. However, this approach is more verbose and may conflict with established HL7 IG conventions where the FHIR context is implicit.

### Disposition C: Not Persuasive
**Action:** Decline the change request.

**Rationale:** The IG is inherently a FHIR IG, and readers should understand that resource names within it refer to FHIR resources by default. Adding "Resource" to every heading is redundant and adds unnecessary verbosity. The FHIR context is already established by the IG title and introduction.

## Recommendation

**Disposition A: Persuasive** — The change is minimal, improves clarity for implementers working across DICOM and FHIR, and should be coordinated with FHIR-47510 for consistency. The IG's dual-standard nature makes the disambiguation genuinely helpful, not redundant.
