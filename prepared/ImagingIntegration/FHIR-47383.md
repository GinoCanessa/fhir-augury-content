## Analysis

The DICOM SR to FHIR Resource Mapping IG defines a profile for "Device - DICOM SR TID 4019 Algorithm Identification Mapping" (referenced in FHIR-47504's naming discussion), and guidance states that observations with Algorithm Name concepts (DCM#111001) are mapped to an Algorithm Identification Device (per FHIR-47367). However, the submitter has identified that despite this profile existing, there is no corresponding **mapping section** in the IG that explains how DICOM SR content maps to this Device resource.

This gap is substantive — without a mapping section, implementers lack guidance on how to construct the Algorithm Identification Device from DICOM SR data. The reclassification to Change Request by jwhitby confirms this requires actual content addition, not just clarification.

The cluster of related tickets (FHIR-47367, FHIR-47482, FHIR-47504) all concern different aspects of the Algorithm Identification Device mapping, suggesting this is an area of the IG that needs holistic attention.

## Proposed Dispositions

### Disposition A: Persuasive — Add the missing mapping section
Add a dedicated mapping section for Device Algorithm Identification to the IG, paralleling the structure of other mapping sections. This section should define how DICOM SR TID 4019 Algorithm Identification content maps to the FHIR Device resource, including concept mappings (e.g., DCM#111001 "Algorithm Name"), parameters, and version information. Consider coordinating with FHIR-47367 (Derived Imaging Measurement inclusion) and FHIR-47504 (naming consistency) to ensure the new section is comprehensive and consistently named.

### Disposition B: Persuasive with Modification — Add a condensed mapping note rather than a full section
If the Algorithm Identification Device mapping is sufficiently simple (e.g., a small number of direct attribute mappings), add mapping guidance as an inline note or subsection within the existing Algorithm Identification Device profile page, rather than creating a standalone mapping section. This reduces IG bloat while still providing implementer guidance.

### Disposition C: Not Persuasive — No mapping section is needed
If the Algorithm Identification Device profile is fully self-documenting through its StructureDefinition and the existing narrative guidance (i.e., the conditional mapping from DCM#111001 to Device is sufficient), then no additional mapping section is needed. The existing guidance in the IG text already covers when and how to create the Device. In this case, the resolution should clarify why a separate section is unnecessary.

## Recommendation

**Disposition A: Persuasive — Add the missing mapping section.**

The reclassification to Change Request by jwhitby already signals that a substantive addition is expected. Other mapping targets in the IG (Observations, BodyStructure, etc.) have dedicated mapping sections, so the Algorithm Identification Device should follow the same pattern for consistency. The cluster of related tickets (FHIR-47367, FHIR-47482, FHIR-47504) indicates implementer confusion around this area, which a proper mapping section would help resolve. This should be coordinated with the resolution of FHIR-47367 to ensure Derived Imaging Measurements are also addressed.
