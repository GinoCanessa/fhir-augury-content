## Summary

The `basic-resource-type` code system in THO (HL7 Terminology) is considered obsolete and slated for retirement. However, the FHIR Core example file `basic/referral` (Basic-example.xml) still references this code system. This ticket requests removing that usage from the example so the code system can be retired from THO without leaving dangling references in the FHIR specification.

## Details

### Description

The reporter (Marc Duteau) wants to remove (or retire) the `basic-resource-type` code system from THO. This code system was originally intended to define codes for resources not yet supported by FHIR, but it has become outdated as those resource types have since been added to the specification. The only remaining reference to this code system in FHIR Core is the `basic/referral` example (Basic-example.xml), which needs to be updated to stop using it.

## Proposed Dispositions

### Disposition A: Accept (Persuasive)

**Action:** Accept the request. Remove or replace the reference to the `basic-resource-type` code system in the `basic/referral` example (Basic-example.xml) in FHIR Core. The example should either use a different code system for the `Basic.code` element or be restructured so it no longer depends on `basic-resource-type`.

**Rationale:** The `basic-resource-type` code system is widely regarded as obsolete (confirmed by Grahame Grieve and Lloyd McKenzie in Zulip). The companion ticket FHIR-52925 has already been resolved as Persuasive to retire the code system from THO. Removing the example's dependency is a necessary prerequisite for that retirement. The binding on `Basic.code` is an example binding, so there is no normative impact.

### Disposition B: Accept with Modification (Persuasive with Mod)

**Action:** Accept, but instead of simply removing the code system reference, replace the `basic/referral` example with a more modern and useful example of the Basic resource that demonstrates current best practices — for instance, using a custom code system defined in the example itself, or referencing a well-established code system. Update the example narrative accordingly.

**Rationale:** Since the example must be changed anyway, this is an opportunity to provide a more illustrative and up-to-date example of the Basic resource. The current referral example is a relic from early FHIR development and could benefit from modernization.

### Disposition C: Decline (Not Persuasive)

**Action:** Decline the request. Keep the `basic/referral` example as-is.

**Rationale:** This would only be appropriate if there were a reason to retain the `basic-resource-type` code system in THO. However, since FHIR-52925 has already been resolved as Persuasive to retire the code system, declining this ticket would leave a broken reference in FHIR Core examples. This disposition is not recommended.

## Recommendation

**Disposition A: Accept (Persuasive)** is recommended. The community consensus (Zulip discussion, companion ticket FHIR-52925 already resolved) is clear that the `basic-resource-type` code system should be retired. Removing its use from the `basic/referral` example is a straightforward, low-risk change that unblocks the THO retirement. The example binding means there is no normative or breaking impact.
