## Summary

The FHIR R6 Ballot 4 home page ([index.html](https://hl7.org/fhir/6.0.0-ballot4/index.html)) contains an **Implementation Links** table with three entries: "Registry of Software Implementations", "Public Test Servers", and "Useful Tools for Implementers". The reporter identified that the "Useful Tools for Implementers" link incorrectly points to the same URL as "Public Test Servers" (`https://confluence.hl7.org/spaces/FHIR/pages/35718859/Public+Test+Servers`). This appears to be a copy-paste error where the link target was duplicated rather than pointing to the correct Confluence page for implementer tools.

---

## Details

### Description

The reporter observed that on the FHIR 6.0.0-ballot4 home page, the Implementation Links table has three links:

1. **Registry of Software Implementations** — presumably links to a registry page
2. **Public Test Servers** — links to `https://confluence.hl7.org/spaces/FHIR/pages/35718859/Public+Test+Servers`
3. **Useful Tools for Implementers** — also links to `https://confluence.hl7.org/spaces/FHIR/pages/35718859/Public+Test+Servers`

The third link should point to a different Confluence page listing tools for implementers, not the Public Test Servers page. The reporter asks whether this is intentional or a mistake.

## Proposed Dispositions

### Disposition A: Accept as Technical Correction (Persuasive)

**Action:** Correct the "Useful Tools for Implementers" link in `index.html` to point to the appropriate Confluence page for implementer tools (e.g., `https://confluence.hl7.org/display/FHIR/Useful+Tools+for+Implementers` or the correct target URL), instead of duplicating the Public Test Servers link.

**Rationale:** This is a clear copy-paste error where two of the three Implementation Links entries point to the same URL. The "Useful Tools for Implementers" link should point to a distinct page listing tools relevant to FHIR implementers. This is a straightforward technical correction that does not change any normative content—only a hyperlink target in the home page table.

**Impact:** Minimal. Only the `href` attribute of one link in `index.html` changes. No specification content, resource definitions, or behavior is affected.

### Disposition B: Accept with Note (Persuasive with Mod)

**Action:** Correct the link as in Disposition A, and additionally verify that all three Implementation Links entries in the home page table point to valid, distinct, and currently maintained Confluence pages. Add a comment or internal note to the build process to validate external links on the home page during publishing.

**Rationale:** Given the history of link maintenance issues on the FHIR home page (see related Zulip threads and FHIR-13008/FHIR-8659), a one-time audit of the Implementation Links section would prevent similar issues in future ballots. This goes slightly beyond what was reported but addresses the root cause.

**Impact:** Low. Requires verifying 2–3 links and possibly correcting others. Could be combined with ongoing editorial maintenance.

### Disposition C: Decline (Not Persuasive)

**Action:** Close the ticket without changes, noting that the link targets are managed externally and may be updated independently of the specification build.

**Rationale:** If the Confluence "Public Test Servers" page is intentionally used as a landing page that also covers tools, or if the tools content has been merged into that page, the current behavior may be by design.

**Impact:** None. The duplicate link remains in the published spec.

---

## Recommendation

**Disposition A: Accept as Technical Correction (Persuasive)** is recommended. This is a clear and unambiguous copy-paste error: two distinct rows in the Implementation Links table ("Public Test Servers" and "Useful Tools for Implementers") point to the same URL, which is titled "Public Test Servers" on Confluence. There is no indication this was intentional. The fix is minimal—changing a single `href`—and aligns with the type classification of "Technical Correction." The correct target URL should be identified during resolution (likely a Confluence page specifically listing implementer tools).
