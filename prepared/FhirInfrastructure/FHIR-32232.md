### Description

The ValueSet "Patient Contact Relationship" (bound to `Patient.contact.role`) includes four additional codes in R5 that are not present in R4. At least one of these is duplicative with existing R4 codes. The reporter suspects the R5 expansion is including deprecated codes from CodeSystem v2-0131, which would be confusing to implementers unless clearly marked as deprecated.

### Root Cause (from comments)

Brian Postlethwaite and Grahame Grieve identified that the underlying CodeSystem v2-0131 in HL7 Terminology (THO) uses a custom property (`http://terminology.hl7.org/CodeSystem/utg-concept-properties#v2-table-deprecated`) to mark concepts as deprecated, rather than the standard FHIR concept property (`http://hl7.org/fhir/concept-properties#status` with value `deprecated`). The expansion logic does not recognize this custom property and therefore includes these codes as if they were active.

Marc Duteau (2023-06-21) confirmed that "the concepts weren't properly deprecated in THO" and committed to creating a UP (Unified Process) ticket to correct the deprecation properties in THO once the current release was completed.

## 5. Proposed Dispositions

### Disposition A: Resolve as Applied — Correct the deprecation properties in THO

**Action:** Mark the four deprecated concepts in CodeSystem v2-0131 using the standard FHIR concept properties (`inactive` = `true` and/or `status` = `deprecated`) instead of the non-standard `v2-table-deprecated` property. This would cause standard expansion logic to properly exclude or flag these codes when expanding the PatientContactRelationship ValueSet.

**Rationale:** The root cause is well-understood and was confirmed by Marc Duteau (THO maintainer) in June 2023. This is a data correction in THO, not a specification change. Once the CodeSystem properly declares these concepts as deprecated/inactive, the expansion will behave correctly without any change to the FHIR specification or tooling.

**Vote text:** "Correct the deprecation properties for the affected concepts in CodeSystem v2-0131 in THO to use standard FHIR concept properties (inactive=true / status=deprecated) so expansion logic properly handles them. Create a UP ticket if one does not already exist."

### Disposition B: Resolve as Persuasive with Modification — Add explicit exclude to the ValueSet

**Action:** Modify the ValueSet definition for PatientContactRelationship to explicitly exclude the four deprecated codes, independent of how they are marked in the CodeSystem.

**Rationale:** This provides an immediate fix in the FHIR specification itself, regardless of the timeline for THO corrections. However, it is a workaround rather than a root-cause fix and requires ongoing maintenance as the CodeSystem evolves. It also doesn't fix the broader problem of non-standard deprecation properties in v2-derived CodeSystems.

**Vote text:** "Add explicit exclude clauses for the four deprecated v2-0131 codes (BP and others) to the PatientContactRelationship ValueSet definition in FHIR Core to prevent them from appearing in expansions."

### Disposition C: Resolve as Not Persuasive — Defer to THO process

**Action:** Close the FHIR Core ticket as not actionable in this context; the fix belongs entirely in THO via the Unified Process.

**Rationale:** Since the issue is in the underlying CodeSystem data in THO, not in the FHIR specification, this ticket may not be the correct venue. Marc Duteau already committed to creating a UP ticket. However, this approach risks the issue remaining unresolved if the UP ticket is not created or prioritized.

**Vote text:** "Close as not a FHIR Core issue. The root cause is incorrect deprecation properties in CodeSystem v2-0131 in THO. Ensure a UP ticket exists and is tracked for the correction."

## 6. Recommendation

**Recommended: Disposition A — Correct the deprecation properties in THO.**

This is the right approach because:

1. **Root cause is well-understood.** The concepts use a non-standard `v2-table-deprecated` property that expansion logic cannot interpret. The fix is to use standard FHIR concept properties (`inactive`, `status`).
2. **Broad impact.** Fixing the CodeSystem properly in THO resolves this issue for *all* ValueSets that reference v2-0131, not just PatientContactRelationship.
3. **Expert consensus.** Brian Postlethwaite, Grahame Grieve, Marc Duteau, and Carmela Couderc all agree the fix belongs in the THO CodeSystem data.
4. **Actionable.** Marc Duteau confirmed he would create a UP ticket. FHIR Infrastructure should verify a UP ticket exists (or create one) and track it to completion for R6.

The workgroup should vote to approve the THO correction and ensure a UP ticket is filed and tracked. The FHIR-32232 ticket can then be resolved as Applied once the UP change is published.
