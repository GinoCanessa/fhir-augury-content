## Summary

The FHIR Core specification contains a broken link to the Testing Resources Implementation Guide. The link `https://build.fhir.org/ig/hl7/fhir-testing-ig/index.html` returns a 404 error. This link appears in the `[additional-ig-list]` section of `source/fhir.ini` in the HL7/fhir repository (line 9), which generates a "see also Testing Resources" hyperlink on the resource type pages in the published specification (e.g., in the R6 ballot content).

The root cause is a URL case-sensitivity issue: the link uses lowercase `hl7` in the path, but the GitHub organization (and therefore the build.fhir.org CI build URL) uses uppercase `HL7`. The correct base URL is `https://build.fhir.org/ig/HL7/fhir-testing-ig/`. Additionally, the CI-built IG may serve its index page at a different path (e.g., `/en/index.html` rather than `/index.html`).

A duplicate ticket [FHIR-53898](https://jira.hl7.org/browse/FHIR-53898) ("Link to Testing Incubator broken") was filed by Reinhard Egelkraut on 2026-01-09 reporting the same issue.

## Details

### Description (from ticket)

> \+ see also [Testing Resources](https://build.fhir.org/ig/hl7/fhir-testing-ig/index.html) — this link is broken and results in error "No webpage was found for the web address: https://build.fhir.org/ig/hl7/fhir-testing-ig/index.html"

### Source of the Broken Link

The link originates in the HL7/fhir repository at `source/fhir.ini`:

```ini
[additional-ig-list]
testing=https://build.fhir.org/ig/hl7/fhir-testing-ig/index.html
```

This entry causes the FHIR build tooling to render a "see also Testing Resources" link on the resource type pages. The URL uses lowercase `hl7`, but the build.fhir.org CI system resolves paths using the exact GitHub organization/repo casing — `HL7/fhir-testing-ig`.

### Duplicate: FHIR-53898

[FHIR-53898](https://jira.hl7.org/browse/FHIR-53898) ("Link to Testing Incubator broken", Reporter: Reinhard Egelkraut, Priority: Medium) reports the identical issue: "The Resource Type Page contains a link to the new FHIR Testing incubator … But the link to https://build.fhir.org/ig/hl7/fhir-testing-ig/index.html is broken. it results in an HTTP ERROR 404."

## Proposed Dispositions

### Disposition A: Accept — Fix URL casing in `fhir.ini` (Recommended)

**Resolution:** Applied — Technical Correction

Update the `[additional-ig-list]` entry in `source/fhir.ini` in the HL7/fhir repository to use the correct URL casing and path:

```ini
[additional-ig-list]
testing=https://build.fhir.org/ig/HL7/fhir-testing-ig/
```

**Changes:**
1. Correct `hl7` → `HL7` to match the GitHub organization name and build.fhir.org CI path convention.
2. Remove the explicit `/index.html` suffix, using the base URL instead (consistent with how Grahame Grieve originally posted the URL, and consistent with how the ballot-intro.html links reference the IG).

**Additionally:** Mark [FHIR-53898](https://jira.hl7.org/browse/FHIR-53898) as a duplicate of this ticket, as it reports the identical broken link.

**Rationale:** This is a straightforward one-line URL correction. The ballot-intro.html page already uses the correct casing, confirming `HL7` is the correct form.

---

### Disposition B: Accept — Fix URL and add `/en/` path prefix

**Resolution:** Applied — Technical Correction

Update the URL in `source/fhir.ini` to include the language-prefixed path that the IG publisher generates:

```ini
[additional-ig-list]
testing=https://build.fhir.org/ig/HL7/fhir-testing-ig/en/index.html
```

**Rationale:** The IG publisher may serve multi-language IGs with a language prefix (`/en/`). The Zulip "Testing IG" thread shows a community member using `https://build.fhir.org/ig/HL7/fhir-testing-ig/en/` successfully. This ensures the link lands on the correct English-language index page.

**Risk:** If the IG publisher configuration changes or the `/en/` prefix is removed, the link would break again. The base URL (Disposition A) may be more resilient.

---

### Disposition C: Decline — Defer until Testing IG publication stabilizes

**Resolution:** Not Persuasive — no change

Defer the fix because the Testing IG is still under active development and its CI build may not be consistently available (per Lloyd McKenzie's 2026-01-25 Zulip comment). The link should be fixed only once the IG has a stable publication URL.

**Rationale:** Fixing the URL now may result in a link that intermittently resolves depending on the CI build status. However, this argument is weak — the repository exists, the IG structure is valid, and the CI build system should serve it once the casing is corrected.

---

## Recommendation

**Disposition A: Accept — Fix URL casing in `fhir.ini`**

This is a clear-cut technical correction. The broken link is caused by a lowercase/uppercase mismatch (`hl7` vs `HL7`) in the GitHub organization name portion of the build.fhir.org URL. The fix is a single-character change in `source/fhir.ini`. The `ballot-intro.html` file in the same repository already uses the correct uppercase `HL7` casing for links to the same IG, confirming the correct form. Using the base URL without `/index.html` is the most resilient approach and matches the original URL shared by Grahame Grieve. FHIR-53898 should be marked as a duplicate.
