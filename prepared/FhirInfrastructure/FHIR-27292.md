## Summary

Keith W. Boone requests that the FHIR Shorthand (FSH) specification documentation take a more pedagogical (teaching-oriented) approach rather than serving solely as a reference specification. The reporter notes that while the reference material is adequate as a spec, it lacks introductory material to teach practical usage. He specifically cites the first half of Kernighan & Ritchie's "The C Programming Language" as a model — a narrative, tutorial-style introduction before the formal reference. The reporter also notes that the examples are "very limited" and calls for more practical, worked examples of using SUSHI.

## Proposed Dispositions

### Disposition A: Persuasive — Add Tutorial/Getting-Started Content

**Resolution:** Persuasive

**Rationale:** The request is well-founded. The FSH specification has matured significantly since STU1, and user adoption depends heavily on accessible onboarding material. The workgroup should commit to adding a tutorial-style "Getting Started" section to the specification itself (not just external Confluence pages), including step-by-step worked examples that walk a new user from zero to a basic profile definition. This addresses the reporter's concern directly and aligns with how successful language specifications (like K&R for C) combine tutorial and reference material. External tutorials on Confluence are valuable but are not a substitute for in-spec pedagogical content, since external links can rot and new users naturally look at the spec first.

**Vote Suggestion:** Motion to accept as Persuasive; assign to FSH editors to add a tutorial/getting-started section in a future ballot cycle.

### Disposition B: Persuasive with Modification — Improve Existing Overview + Link to External Tutorials

**Resolution:** Persuasive with Modification

**Rationale:** Rather than a full K&R-style tutorial rewrite, the workgroup could take a lighter-weight approach: enhance the existing Overview section with additional practical examples and more narrative guidance, and add a prominent, curated "Learning Resources" section that links to the Confluence tutorials, FSH School (https://fshschool.org), and other community resources. This acknowledges the legitimate need for better onboarding without requiring a major rewrite of the specification document itself. The modification is that the spec does not need to become a textbook — it needs to better guide users to the right learning resources while providing enough inline examples to be self-sufficient for common tasks.

**Vote Suggestion:** Motion to accept as Persuasive with Modification; editors to expand the Overview with more examples and add a "Learning Resources" section.

### Disposition C: Considered for Future Use — Defer Pending Resource Availability

**Resolution:** Considered for Future Use

**Rationale:** This is the same resolution previously applied (voted 11-0-1 in 2020) and then reverted at STU1 publication. Since STU1, significant community resources have been created (FSH School, updated Confluence tutorials, SUSHI documentation). If the workgroup determines that in-spec tutorial content is not the highest priority compared to other pending FSH enhancements, deferral remains a reasonable option. However, the workgroup should note that this ticket has already been deferred once and re-opened, so a second deferral should include a concrete timeline or milestone for when it will be addressed.

**Vote Suggestion:** Motion to defer as Considered for Future Use with a note to revisit at the next major ballot cycle.

## Recommendation

**Disposition B: Persuasive with Modification** is recommended. The underlying concern — that the spec is hard for newcomers to learn from — is valid and has persisted since 2020. However, a full K&R-style tutorial rewrite within the spec itself would be a large undertaking that may not be the best use of editor time when excellent external resources like FSH School already exist. The pragmatic middle ground is to enhance the in-spec Overview with more worked examples and add a dedicated "Learning Resources" section that curates links to tutorials, FSH School, and other community material. This directly addresses the reporter's concern while being achievable within a reasonable timeline.
