## Summary

This ticket requests adding FSH syntax to set the human-readable `Quantity.unit` field when using the UCUM shortcut notation for Quantity assignments. Currently, the FSH Quantity shortcut syntax (`* valueQuantity = 155.0 '[lb_av]'`) sets the `value`, `system`, and `code` fields but provides no way to specify the `unit` (display) string. The proposal is to allow an optional quoted string after the UCUM code:

```fsh
* valueQuantity = 155.0 '[lb_av]' "lb"
```

This would set `valueQuantity.unit` to `"lb"`. The reporter argues this is intuitive since FSH already uses trailing quoted strings for human-readable display values in other contexts.

## Details

**Background:** FHIR's `Quantity` datatype has both a computable `code` (e.g., UCUM code `[lb_av]`) and a human-readable `unit` string (e.g., `"lb"`). FSH provides a convenient shorthand for setting UCUM-based quantities using single-quoted codes, but prior to this request the `unit` field could only be set via a separate assignment rule (e.g., `* valueQuantity.unit = "lb"`).

**Implementation Status:** This feature was **already implemented** in SUSHI 1.1.0 (released January 2021), approximately two months after this ticket was filed. The SUSHI 1.1.0 release notes explicitly state: _"Extends Quantity syntax to include unit string (proposed FSH STU2 feature)."_ The syntax is now in widespread use in the FSH community, as evidenced by recent Zulip discussions (2024) demonstrating usage such as `* valueQuantity = '1/mL' "{copies}/mL"`.

**Current Spec Status:** Although the feature has been implemented in SUSHI and is in active use, this Jira ticket remains in "Triaged" status and has never been formally resolved. The FSH specification (Shorthand STU2 and beyond) already documents this syntax.

## Proposed Dispositions

### Disposition A: Accept (Persuasive — Already Applied)

Resolve as **Persuasive** and close the ticket. The requested syntax extension has already been implemented in SUSHI since version 1.1.0 (January 2021) and is formally documented in the FSH specification (STU2+). The syntax `* valueQuantity = 155.0 '[lb_av]' "lb"` works as described and is in widespread community use. No further specification or tooling changes are needed. This is a bookkeeping closure.

**Specific action:** Update the ticket status to "Applied" or "Published" to reflect that the change was implemented as proposed and is part of the current FSH specification.

### Disposition B: Accept with Modification (Persuasive with Modification)

Resolve as **Persuasive with Modification**. Accept the proposal as implemented but note that the actual syntax implemented may have minor differences from the original proposal (e.g., the syntax also works for non-UCUM code system assignments like `$UCUM#mm "millimeters"`, not just the single-quote UCUM shorthand). Update any specification language if needed to ensure the documented syntax exactly matches the implementation.

**Specific action:** Verify the FSH spec text matches the implemented behavior in SUSHI and GoFSH, then close.

### Disposition C: Decline (Not Persuasive)

Resolve as **Not Persuasive**. The syntax was implemented without formal ballot resolution of this ticket, and the work group may prefer to retroactively decline the original ticket since the feature was handled through the FSH STU2 development process independently.

**Specific action:** Close with a note that the feature was addressed through FSH STU2 specification development and SUSHI 1.1.0 implementation, outside the scope of this individual tracker item.
