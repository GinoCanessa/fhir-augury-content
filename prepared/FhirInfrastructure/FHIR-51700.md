## Summary

There are approximately 34 occurrences of the incorrectly spaced text "H L7" (instead of "HL7") throughout the FHIR spec source code. All occurrences are in files matching `source/[Resource]/implementationguide-[Resource]-core.xml`. The reporter (Elliot Silver) believes this was introduced by an erroneous conversion process. The ticket is filed as a technical correction directed to FHIR-I for implementation rather than filing separate tickets per work group.

Additionally, the reporter notes a similar issue in the apparently no-longer-used `imagingmanifest/codesystem-kos-title.xml`, where the title reads "K O Stitle" instead of "KOS title".

A follow-up comment from the reporter also identifies that double spaces exist around these incorrectly split words and should be corrected as part of the fix.

## Proposed Dispositions

### Disposition A: Applied — Technical Correction (Recommended ✓)

**Action:** Accept as a technical correction. Correct all ~34 instances of "H L7" to "HL7" in the `implementationguide-[Resource]-core.xml` files, fix the "K O Stitle" → "KOS title" issue in `imagingmanifest/codesystem-kos-title.xml`, and remove the associated double spaces around the corrected words.

**Rationale:** This is a straightforward text correction of an obvious encoding/conversion error. The misspelling "H L7" is clearly unintentional — "HL7" is the correct organization name that should appear throughout the spec. A commit on master (2d2e185c) has already partially addressed the imagingmanifest file. The original draft PR (HL7/fhir#3653) was closed without merge but demonstrated that the fix is a simple 1:1 replacement across 34 files. The reporter's additional observation about double spaces should also be addressed. As a technical correction, this does not change normative or substantive content and is appropriate for FHIR-I to implement directly.

### Disposition B: Deferred — Awaiting Comprehensive Audit

**Action:** Defer the ticket until a broader audit of encoding/conversion artifacts is completed across the entire FHIR spec source.

**Rationale:** The "H L7" issue suggests a systemic conversion problem that may have introduced other, as-yet-undetected, text corruption. Deferring would allow for a single comprehensive sweep to catch all such issues at once, rather than applying piecemeal fixes.

### Disposition C: Not Persuasive — Cosmetic, Low Impact

**Action:** Close without action, noting that the misspellings occur only in source XML files (not rendered spec text) and do not affect spec functionality.

**Rationale:** If the misspellings only appear in XML source metadata and are not rendered in the published specification, the practical impact is negligible. The partial commit already on master may be sufficient.

---

## Recommendation

**Disposition A: Applied — Technical Correction** is recommended. The issue is clearly an error (nobody intends to spell "H L7"), the fix is mechanical and well-understood (34 simple string replacements plus double-space cleanup), and a draft PR already proved the scope of changes needed. Partial work has already been committed to master. This should be completed as a technical correction with a clean PR that incorporates the reporter's feedback about double spaces.
