## Summary

The reporter identified that the link to the Citation Resource from the FHIR Core specification's "immature resources" list is broken. The list in `ballot-intro.html` (and several other pages) links to `https://build.fhir.org/ig/HL7/ebm/StructureDefinition-Citation.html`, but the EBM IG repository was renamed from `HL7/ebm` to `HL7/ebm-incubator`. The correct URL is `https://build.fhir.org/ig/HL7/ebm-incubator/StructureDefinition-Citation.html`. The Citation StructureDefinition exists in the `ebm-incubator` repo at `input/resources/citation/StructureDefinition-Citation.xml`, confirming the link target is valid once the repo name is corrected.

---

## Details

### Description

The reporter states: Following the text "Immature resources have been moved out of this specification into other IGs, often labelled as 'incubator IGs'. Here's the list:", the link to the [Citation Resource](https://build.fhir.org/ig/HL7/ebm/StructureDefinition-Citation.html) at `https://build.fhir.org/ig/HL7/ebm/StructureDefinition-Citation.html` does not work.

### Analysis

The broken link appears in **8 locations** across the FHIR Core specification (repo `HL7/fhir`):

1. **`source/ballot-intro.html:74`** — immature resources list
2. **`source/clinicalreasoning-module.html:87`** — clinical reasoning module table
3. **`source/clinicalreasoning-evidence-and-statistics.html:80`** — evidence and statistics narrative
4. **`source/history.html:282`** — R5 history entry (new resources)
5. **`source/history.html:304`** — R5 history entry (add new resources)
6. **`source/history.html:330`** — R5 history entry (add Citation)
7. **`source/metadataresource/metadataresource-introduction.xml:29`** — MetadataResource introduction
8. **`source/resource/resource-notes.xml:157`** — Resource notes

All occurrences use the URL `https://build.fhir.org/ig/HL7/ebm/...` but the GitHub repository is `HL7/ebm-incubator` (canonical: `http://hl7.org/fhir/uv/ebm-incubator`, IG id: `hl7.fhir.uv.ebm-incubator`). The CI build URL therefore resolves under `https://build.fhir.org/ig/HL7/ebm-incubator/`.

## Proposed Dispositions

### Disposition A: Accept — Fix all broken Citation links to use `ebm-incubator`

**Resolution:** Technical Correction — Applied

Replace all 8 occurrences of `https://build.fhir.org/ig/HL7/ebm/StructureDefinition-Citation.html` with `https://build.fhir.org/ig/HL7/ebm-incubator/StructureDefinition-Citation.html` in the `HL7/fhir` repository:

- `source/ballot-intro.html` (line 74)
- `source/clinicalreasoning-module.html` (line 87)
- `source/clinicalreasoning-evidence-and-statistics.html` (line 80)
- `source/history.html` (lines 282, 304, 330)
- `source/metadataresource/metadataresource-introduction.xml` (line 29)
- `source/resource/resource-notes.xml` (line 157)

This is a straightforward URL correction with no semantic or normative impact.

### Disposition B: Accept — Broader link audit for all incubator IG references

**Resolution:** Technical Correction — Applied

In addition to fixing the 8 Citation links above, perform a broader audit of all `build.fhir.org/ig/HL7/` links in the FHIR Core spec to verify that every incubator IG link uses the correct current repository name. This would catch any similar issues with other moved resources (e.g., checking that links to `oo-incubator`, `admin-incubator`, `cg-incubator`, `immunization-incubator`, `pc-incubator`, `phx-incubator`, and `api-incubator` all resolve correctly). File a separate tracker if additional broken links are found.

### Disposition C: Decline — Defer until EBM IG publication

**Resolution:** Not Persuasive — Deferred

Decline the fix on the grounds that the `build.fhir.org` CI build URLs are inherently transient and may change again when the EBM IG is formally published. Wait until the EBM IG has a stable published URL (e.g., `http://hl7.org/fhir/uv/ebm-incubator/`) and update all links at that time. However, this leaves users with broken links in the current ballot specification.

---

## Recommendation

**Disposition A** is recommended. This is a clear technical correction — the repo was renamed from `ebm` to `ebm-incubator` and the links were never updated. The fix is mechanical (find-and-replace across 8 locations), carries zero normative risk, and directly improves the usability of the current ballot specification. A broader audit (Disposition B) is worthwhile but can be tracked separately rather than blocking this simple fix.
